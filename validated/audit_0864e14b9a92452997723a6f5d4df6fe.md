### Title
Shared Slot Timestamp in `CompressedOracle` Allows Stale Per-Position Prices to Bypass Downstream Staleness Checks — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1` packs four oracle positions into a single 256-bit storage slot with **one shared timestamp**. When any position in a slot is updated, the slot-level `timestampMs` advances for all four positions simultaneously. Downstream consumers (`AnchoredPriceProvider`, `PriceProvider`, `ProtectedPriceProvider`) use this slot timestamp as the freshness indicator for a specific position. A creator who updates position 0 frequently and position 1 infrequently will inadvertently make position 1's stale price appear fresh, allowing it to pass the staleness gate and reach a live pool swap.

---

### Finding Description

The slot layout packs four 48-bit oracle data words and one 56-bit timestamp into a single 256-bit storage word:

```
[data0:6][data1:6][data2:6][data3:6][ts:7][slotId:1]
```

`_loadSlotLayout` extracts one timestamp for the entire slot: [1](#0-0) 

`getOracleData` then assigns that single slot timestamp to whichever position is being read: [2](#0-1) 

The push path (`fallback`) writes the entire 32-byte word atomically. The monotonicity guard only checks whether the incoming slot timestamp is newer than the stored one — it does not track per-position freshness: [3](#0-2) 

`_price` converts the slot timestamp to seconds and returns it as `refTime`: [4](#0-3) 

`AnchoredPriceProvider._readLeg` uses this `refTime` as the sole freshness signal: [5](#0-4) 

The same pattern applies in `PriceProvider._getBidAndAskPrice` and `ProtectedPriceProvider._computeBidAsk`: [6](#0-5) [7](#0-6) 

Because the timestamp belongs to the slot and not to the individual position, updating position 0 silently refreshes the staleness clock for positions 1–3, even though their price bytes are unchanged.

---

### Impact Explanation

A pool whose price provider reads position 1 of a slot will receive a `refTime` equal to the last time **any** position in that slot was updated. If position 0 is a high-frequency feed (e.g., updated every 30 seconds) and position 1 is a low-frequency feed (e.g., updated every 10 minutes), the pool's staleness check — governed by `MAX_REF_STALENESS` — will always pass for position 1, even when its price is many minutes old. The pool then executes swaps at a stale mid-price, enabling arbitrageurs to extract value from LPs or causing the pool to settle trades at incorrect rates. This is a direct loss of LP principal and a swap conservation failure.

---

### Likelihood Explanation

The compressed slot format is explicitly designed to hold multiple independent feeds in one slot (e.g., four different token pairs). A creator who operates feeds with different update cadences — a common and expected pattern — will trigger this condition in normal operation without any malicious intent. No special permissions beyond owning a namespace (which requires zero setup) are needed. Any delegated pusher who updates the slot for one position also silently refreshes the timestamp for all others.

---

### Recommendation

Replace the single slot-level timestamp with per-position timestamps. One approach: store four independent 14-bit (or similar) delta timestamps relative to a slot base, so each position carries its own freshness. Alternatively, enforce that all four positions in a slot must be updated atomically with fresh data on every push, and document this invariant explicitly so that downstream consumers can rely on it. At minimum, the `getOracleData` and `_price` functions should not return the slot timestamp as the position's `refTime` without a clear guarantee that the position's price was updated at that time.

---

### Proof of Concept

```
Setup:
  - CompressedOracle deployed with MAX_TIME_DRIFT = 60s
  - Creator pushes slot 0 with:
      position 0 = ETH/USD (updated every 30s)
      position 1 = BTC/USD (updated every 10 min)
  - AnchoredPriceProvider configured with:
      offchainOracle = CompressedOracle
      baseFeedId     = feedIdOf(creator, 0, 1)  // BTC/USD at position 1
      MAX_REF_STALENESS = 5 minutes

Timeline:
  T+0:00  Pusher writes slot 0: ETH=3000, BTC=60000, ts=T
  T+0:30  Pusher writes slot 0: ETH=3001, BTC=60000 (unchanged), ts=T+30s
  T+1:00  Pusher writes slot 0: ETH=3002, BTC=60000 (unchanged), ts=T+60s
  ...
  T+4:59  Pusher writes slot 0: ETH=3010, BTC=60000 (unchanged), ts=T+4:59

  At T+5:00, a swap is triggered on the pool using BTC/USD:
    getOracleData(feedIdOf(creator, 0, 1)) returns:
      price     = 60000  (from T+0:00, now 5 minutes old)
      timestampMs = T+4:59  (slot was updated 1 second ago)
      refTime   = T+4:59 in seconds

    AnchoredPriceProvider._readLeg:
      _isStale(T+4:59, T+5:00, 300s) → (5:00 - 4:59) = 1s ≤ 300s → NOT stale ✓

  BTC has moved from 60000 to 62000 in the last 5 minutes.
  Pool executes swap at 60000 instead of 62000.
  LPs absorb the 3.3% price discrepancy as a loss.
```

The corrupted value is `data.timestampMs` returned by `getOracleData` for position 1: it reflects the slot's last write time (driven by position 0's high-frequency updates) rather than the time position 1's price bytes were last set. [8](#0-7)

    function _loadSlotLayout(bytes32 slotIndex) internal view returns (SlotLayout memory _layout) {
        uint256 slotValue;
        assembly ("memory-safe") {
            slotValue := sload(slotIndex)
        }

        _layout.timestampMs = toTimeMs(slotValue >> 8 & X56);

        _layout.oracle0 = _decodeCompressedOracleData(uint48((slotValue >> 208) & X48));
        _layout.oracle1 = _decodeCompressedOracleData(uint48((slotValue >> 160) & X48));
        _layout.oracle2 = _decodeCompressedOracleData(uint48((slotValue >> 112) & X48));
        _layout.oracle3 = _decodeCompressedOracleData(uint48((slotValue >> 64) & X48));
    }

    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
