### Title
Stale `namespaceRemapping` Allows a Revoked Contract Pusher to Inject Arbitrary Prices into a Creator's Namespace — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowContractPushers` validates a contract pusher's consent via a one-time live `isPusher(creator)` staticcall at delegation time and writes `namespaceRemapping[pusher] = creator`. The `fallback()` push path reads only `namespaceRemapping[msg.sender]` to resolve the target namespace and **never re-validates `isPusher()`**. If the pusher contract's internal authorization is subsequently revoked (i.e., `isPusher(creator)` now returns `false`), the stale mapping persists and the pusher continues to overwrite the creator's price slots with arbitrary data, feeding bad prices to every pool that consumes those feeds.

---

### Finding Description

In `CompressedOracleV1.allowContractPushers()`, the comment states:

> "Contract-pusher variant: consent is proven by a LIVE `isPusher(creator)` staticcall instead of a signature, so there is nothing to replay and no deadline is needed."

The live check is performed exactly once — at delegation time:

```solidity
// CompressedOracle.sol lines 226-231
(bool ok, bytes memory res) = pusher.staticcall(
    abi.encodeWithSignature("isPusher(address)", msg.sender)
);
require(ok);
bool allowed = abi.decode(res, (bool));
require(allowed);
namespaceRemapping[pusher] = msg.sender;   // persists indefinitely
```

The `fallback()` push path resolves the target namespace solely from the stored mapping:

```solidity
// CompressedOracle.sol lines 315-316
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
```

There is no call to `isPusher()` here. If the pusher contract's internal state changes so that `isPusher(creator)` returns `false` — for example, the pusher's own admin revokes the creator's authorization, the pusher is upgraded, or its allowlist is modified — the oracle has no mechanism to detect this. `namespaceRemapping[pusher]` still points to the creator, and every subsequent fallback call from the pusher overwrites the creator's price slots.

This is the direct analog of the "deactivated passport" bug: the passport (delegation) was valid when issued, but the system only checks validity at issuance time, not at use time. The old credential continues to work even after it has been deactivated in the issuing system.

---

### Impact Explanation

A pusher whose on-chain authorization has been silently revoked can push arbitrary `(price, spread0, spread1, timestampMs)` tuples into the creator's namespace. Specifically, it can push:

- A manipulated mid price (non-zero, within any configured `priceGuard` range, with a fresh timestamp) that passes all `AnchoredPriceProvider._readLeg()` guards (staleness, `mid == 0`, `spreadBps >= ORACLE_BPS`, price guard).
- The corrupted price then reaches `_computeBidAsk()`, producing a bad bid/ask pair that is delivered to the pool's swap path.

This satisfies the **bad-price execution** impact: a manipulated bid/ask quote reaches a live pool swap. Depending on the magnitude of the price manipulation, this can cause:
- Traders to receive more tokens than the oracle curve permits (swap conservation failure).
- LP positions to be drained at incorrect rates (direct loss of LP principal).

---

### Likelihood Explanation

The scenario requires:
1. A pusher contract with mutable `isPusher()` logic (e.g., an upgradeable contract, a contract with internal access control, or a multi-sig-managed allowlist).
2. The pusher's internal authorization being revoked without the creator being notified.
3. The creator failing to call `removePushers()` before the pusher pushes bad data.

In production deployments where pusher contracts are managed by separate teams or are upgradeable, this is a realistic operational gap. The creator has no on-chain signal that `isPusher()` has changed; they must monitor it off-chain.

---

### Recommendation

Re-validate the pusher's authorization at push time inside `fallback()`. When a non-zero `namespaceRemapping` entry is found for a contract pusher, call `isPusher(creator)` before routing the write:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator != address(0)) {
    // Re-validate contract-pusher consent at push time.
    (bool ok, bytes memory res) = msg.sender.staticcall(
        abi.encodeWithSignature("isPusher(address)", creator)
    );
    if (!ok || !abi.decode(res, (bool))) {
        // Consent revoked: fall back to own namespace and clear stale mapping.
        namespaceRemapping[msg.sender] = address(0);
        creator = msg.sender;
    }
}
if (creator == address(0)) creator = msg.sender;
```

Alternatively, if the gas cost of a per-push staticcall is unacceptable, document explicitly that `allowContractPushers` is a one-time snapshot and that creators **must** monitor their pushers' `isPusher()` state and call `removePushers()` immediately upon any change.

---

### Proof of Concept

```
1. Deploy MutablePusher with isPusher(creator) = true.
2. creator calls allowContractPushers([MutablePusher]).
   → namespaceRemapping[MutablePusher] = creator  ✓
3. MutablePusher.admin() revokes creator: isPusher(creator) now returns false.
   (Creator is not notified; removePushers is never called.)
4. MutablePusher.admin() calls oracle.fallback() with a crafted slot word:
     price  = 1 (near-zero, far below true market)
     spread = 3  (valid codebook index, passes spreadBps < ORACLE_BPS)
     ts     = block.timestamp * 1000  (fresh, passes staleness check)
5. fallback() resolves: namespaceRemapping[MutablePusher] = creator → writes into creator's slot.
6. AnchoredPriceProvider._readLeg(feedId) reads the corrupted price:
     _isStale → false (fresh timestamp)
     mid == 0 → false (mid = 1)
     spreadBps >= ORACLE_BPS → false (spread = 3)
     priceGuard → passes (no guard set, or 1 is within range)
     ok = true  ← bad price accepted
7. _computeBidAsk(1, 3) produces a near-zero bid/ask.
8. Pool swap executes at the manipulated price → LP funds drained.
```

    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
