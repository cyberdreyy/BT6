### Title
Pusher Revocation Bypass via Signature Replay in `allowPushers` — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

A pusher who calls `revokePusher()` to self-revoke their namespace delegation can have that delegation silently re-established by the creator replaying the original EIP-191 consent signature before its deadline expires. The revocation is therefore not final, breaking the invariant that a pusher can unilaterally exit a delegation.

### Finding Description

`allowPushers` requires each pusher to supply an EIP-191 signature committing to `(block.chainid, address(this), deadline, pusher, msg.sender)`. [1](#0-0) 

The function contains no nonce, no "already-used" bitmap, and no check for a prior revocation. It simply overwrites `namespaceRemapping[pusher] = msg.sender` unconditionally on every call that passes signature verification and the deadline check. [2](#0-1) 

`revokePusher()` clears the mapping to `address(0)`: [3](#0-2) 

Because the original signature is still cryptographically valid (same `deadline`, same `pusher`, same `creator`), the creator can immediately call `allowPushers` again with the identical signature and re-establish the delegation. The pusher's revocation is silently undone.

The protocol's own documentation acknowledges the deadline is required precisely because "an undated signature could re-establish a delegation after the pusher revoked it," but the deadline only bounds the replay window — it does not prevent replay within that window. [4](#0-3) 

The fallback push path resolves the namespace from `namespaceRemapping[msg.sender]`, so once the mapping is restored the pusher's calldata lands in the creator's namespace again, not the pusher's own. [5](#0-4) 

### Impact Explanation

**Bad-price execution / oracle feed hijack.** The concrete harm path:

1. A pusher's private key is compromised. The pusher calls `revokePusher()` to stop the attacker from writing into the creator's price feeds.
2. The creator (unaware, or colluding) calls `allowPushers` with the original signature before the deadline expires. Delegation is restored.
3. The attacker (holding the pusher key) resumes pushing arbitrary prices into the creator's namespace slots.
4. `PriceProvider` / `AnchoredPriceProvider` reads those slots via `getOracleData` and forwards the attacker-controlled bid/ask to the pool's `getBidAskPrice`.
5. Swaps execute at the attacker-supplied price, causing direct loss of trader or LP principal.

The `namespaceRemapping` invariant — "pusher consent must prevent replay or silent feed hijack" — is broken. [6](#0-5) 

### Likelihood Explanation

**Medium.** The creator must still hold the original signature bytes and act before the deadline. Deadlines are typically set days in the future (tests use `block.timestamp + 1 days`), giving a wide replay window. The pusher has no on-chain way to invalidate the signature early. [7](#0-6) 

### Recommendation

Add a per-pusher revocation nonce or a "revoked-at" timestamp to `allowPushers` validation:

```solidity
mapping(address => uint256) public pusherRevocationNonce;

// In revokePusher():
pusherRevocationNonce[msg.sender]++;

// In allowPushers(), include the nonce in the signed hash:
keccak256(abi.encode(
    block.chainid, address(this), deadline,
    pusher, msg.sender,
    pusherRevocationNonce[pusher]   // <-- added
))
```

This makes every revocation invalidate all previously issued signatures for that pusher, regardless of their deadline.

### Proof of Concept

```
T=0   Pusher signs: sig = EIP191(chainid, oracle, deadline=T+1day, pusher, creator)
T=1   Creator calls allowPushers(T+1day, [pusher], [sig])
         → namespaceRemapping[pusher] = creator  ✓

T=2   Pusher's key is stolen. Pusher calls revokePusher()
         → namespaceRemapping[pusher] = address(0)  ✓ (pusher believes they are safe)

T=3   Creator calls allowPushers(T+1day, [pusher], [sig])  ← SAME sig, deadline not expired
         → namespaceRemapping[pusher] = creator  ← delegation silently restored

T=4   Attacker (with stolen pusher key) calls oracle fallback with crafted slot word
         → price lands in creator's namespace
         → PriceProvider returns attacker price to pool
         → swap executes at bad price → loss of funds
```

    ///         signature is REQUIRED — without it anyone could remap a foreign pusher
    ///         wallet into their own namespace and silently swallow its pushes. The
    ///         deadline is likewise required: the signed consent carries no timestamp of
    ///         its own, so an undated signature could re-establish a delegation AFTER the
    ///         pusher revoked it.
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
