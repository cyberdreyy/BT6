### Title
Shared Slot-Level Timestamp in `CompressedOracleV1` Allows Stale Per-Lane Price Data to Bypass Provider Staleness Check — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1` packs four feeds into one 256-bit storage slot with a **single shared 56-bit timestamp**. When a creator performs a read-modify-write to update only one lane, the shared timestamp advances for all four lanes even though the other lanes' price data is unchanged. Every downstream price provider (`PriceProviderL2`, `ProtectedPriceProviderL2`, `PriceProvider`) uses that shared `refTime` as the sole staleness gate. A lane whose price has not been refreshed for longer than `MAX_TIME_DELTA` will pass the staleness check because a sibling lane's recent update advanced the slot timestamp, delivering a stale bid/ask quote to a live pool swap.

---

### Finding Description

**Slot layout — one timestamp for four feeds:**

```
bits 255 … 208 : oracle[0] (48 bits)
bits 207 … 160 : oracle[1] (48 bits)
bits 159 … 112 : oracle[2] (48 bits)
bits 111 …  64 : oracle[3] (48 bits)
bits  63 …   8 : timestamp (uint56, unix milliseconds)   ← shared
bits   7 …   0 : reserved
``` [1](#0-0) 

Both push paths (`fallback` and `updateBySignature`) overwrite the **entire** 256-bit slot in one `sstore`. The timestamp field is part of that word, so it is always the timestamp of the most-recent push to the slot, regardless of which lane was actually changed. [2](#0-1) 

**`getOracleData` assigns the slot-level timestamp to every lane it returns:**

```solidity
data.timestampMs = _layout.timestampMs;   // same value for all four positions
``` [3](#0-2) 

**Price providers treat that timestamp as the freshness proof for the feed they are reading:**

```solidity
(uint256 mid, uint256 spread, , uint256 refTime) =
    IPricedOracle(address(offchainOracle)).price(offchainFeedId, msg.sender);

if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA, FUTURE_TOLERANCE)) {
    return (0, type(uint128).max);
}
``` [4](#0-3) 

**Concrete sequence:**

| Time | Event |
|------|-------|
| T₀ | Creator pushes slot 0 with all four lanes (ETH/USD @ pos 0, BTC/USD @ pos 1, …). Slot timestamp = T₀. |
| T₀ + Δ (Δ > `MAX_TIME_DELTA`) | Creator pushes slot 0 again, updating only BTC/USD (pos 1) with a new price. The read-modify-write preserves the ETH/USD price bytes from T₀ but writes a new timestamp T₁ = T₀ + Δ. |
| T₁ | Pool reads ETH/USD (pos 0). `getOracleData` returns `price = ETH_price_at_T₀`, `timestampMs = T₁`. Staleness check: `T₁ − block.timestamp < MAX_TIME_DELTA` → **passes**. Pool executes swap at the T₀ price. |

The documentation itself acknowledges the per-slot (not per-lane) nature of the timestamp:

> "A sibling lane in the SAME slot shares the slot's 56-bit timestamp word, but its own 48-bit price lane was never written → price 0 (the timestamp is per-slot, not per-lane)." [5](#0-4) 

The documentation acknowledges the structural fact but does not address the staleness-bypass consequence when a lane carries a non-zero stale price.

---

### Impact Explanation

A pool whose `PriceProvider` is bound to a `CompressedOracleV1` feed at position P will receive a stale mid-price that passes the `MAX_TIME_DELTA` guard whenever any sibling lane in the same slot was updated more recently than P. The pool will compute bid/ask from the stale mid and execute the swap, violating the oracle-anchored pricing invariant. Traders can extract value by arbitraging the stale quote against the true market price; LPs bear the loss.

This matches the allowed impact gate: **bad-price execution — stale bid/ask quote reaches a pool swap**.

---

### Likelihood Explanation

The trigger is a routine operational pattern: a creator maintains multiple feeds in one slot at different update frequencies (e.g., a high-volatility asset updated every 30 s and a low-volatility asset updated every 5 min). Every high-frequency push silently refreshes the staleness clock for all co-located lanes. No malicious intent is required; the creator may be acting in good faith. The `CompressedOracleV1` is open and registrationless, so any creator can deploy this configuration.

---

### Recommendation

**Option A — Per-lane timestamps.** Restructure the slot to give each of the four lanes its own timestamp field. This requires a wider storage word (e.g., two slots) or a reduced number of lanes per slot.

**Option B — Sentinel on unchanged lanes.** When a pusher supplies a read-modify-write word, require that unchanged lanes carry a reserved "carry-forward" marker. The oracle then preserves the per-lane timestamp from the previous slot value rather than overwriting it with the new slot timestamp.

**Option C — Provider-side per-feed timestamp cache.** Each `PriceProvider` stores the last `refTime` it observed for its specific `feedId`. On each read it compares the new `refTime` against the cached value; if the slot timestamp advanced but the lane's price bytes are identical to the previous read, the provider treats the feed as not updated and applies the cached (older) timestamp for the staleness check.

---

### Proof of Concept

```solidity
// Extend CompressedOracle.t.sol
function test_sharedTimestampBypassesStaleness() public {
    uint256 MAX_TIME_DELTA_SEC = 60; // provider configured with 60 s

    // T0: creator pushes all four lanes
    uint56 tsMs0 = uint56(block.timestamp * 1000);
    uint48 raw0 = _packRaw(1_000_000, 3, 3); // ETH/USD at pos 0
    uint48 raw1 = _packRaw(2_000_000, 4, 4); // BTC/USD at pos 1
    uint48 raw2 = _packRaw(3_000_000, 5, 5);
    uint48 raw3 = _packRaw(4_000_000, 6, 6);

    vm.prank(creator);
    (bool ok,) = address(oracle).call(_slotWord(0, raw0, raw1, raw2, raw3, tsMs0));
    assertTrue(ok);

    // Advance past MAX_TIME_DELTA — ETH/USD (pos 0) is now stale
    skip(MAX_TIME_DELTA_SEC + 1);

    // T1: creator updates only BTC/USD (pos 1), read-modify-write preserves pos 0 price
    uint56 tsMs1 = uint56(block.timestamp * 1000);
    uint48 newRaw1 = _packRaw(2_500_000, 4, 4);

    vm.prank(creator);
    (ok,) = address(oracle).call(_slotWord(0, raw0, newRaw1, raw2, raw3, tsMs1));
    assertTrue(ok);

    // Read ETH/USD (pos 0)
    IOffchainOracle.OracleData memory d = oracle.getOracleData(oracle.feedIdOf(creator, 0, 0));

    // Price is the T0 value — stale
    assertEq(d.price, U64x32.decode(uint32(raw0 >> 16)), "price is from T0");

    // But timestamp is T1 — fresh
    assertEq(TimeMs.unwrap(d.timestampMs), uint256(tsMs1), "timestamp is T1");

    // Provider staleness check: refTime = tsMs1/1000 = block.timestamp → passes
    // Pool executes swap at the T0 ETH/USD price despite MAX_TIME_DELTA being exceeded
    uint256 refTime = uint256(tsMs1) / 1000;
    assertFalse(
        (block.timestamp - refTime) > MAX_TIME_DELTA_SEC,
        "staleness check incorrectly passes for stale lane"
    );
}
```

The test demonstrates that after `MAX_TIME_DELTA` seconds, the ETH/USD lane (position 0) carries a T₀ price but a T₁ timestamp. Any price provider configured with `MAX_TIME_DELTA = 60 s` will accept this feed as fresh and deliver the stale quote to the pool's swap execution path. [6](#0-5) [3](#0-2) [4](#0-3)

    function _loadSlotLayout(bytes32 slotIndex) internal view returns (SlotLayout memory _layout) {
        uint256 slotValue;
        assembly ("memory-safe") {
            slotValue := sload(slotIndex)
        }

        _layout.timestampMs = toTimeMs(slotValue >> 8 & X56);

        _layout.oracle0 = _decodeCompressedOracleData(uint48((slotValue >> 208) & X48));

        _layout.oracle2 = _decodeCompressedOracleData(uint48((slotValue >> 112) & X48));
        _layout.oracle3 = _decodeCompressedOracleData(uint48((slotValue >> 64) & X48));
