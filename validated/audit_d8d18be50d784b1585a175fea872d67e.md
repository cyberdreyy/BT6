### Title
`PriceProvider._getBidAndAskPrice()` Silently Produces Zero-Spread or Stall Sentinel When `confidenceParam == 0` (Default State) вЂ” (`smart-contracts-poc/contracts/PriceProvider.sol`)

---

### Summary

`PriceProvider` zero-initializes `confidenceParam` and permits it to be reset to `0` via `setConfidenceParam`. When `confidenceParam == 0` and `marginStep == 0`, the function computes `bid == ask == mid`, producing either a permanent `FeedStalled()` revert (pool unusable) or a 1-unit Q64 spread (effectively zero spread, stripping LP fees). This is the direct analog of the external bug: a zero-valued parameter causes the function to return wrong/sentinel values instead of correct prices.

---

### Finding Description

**Step 1 вЂ” Zero initialization and unrestricted reset to zero.**

`confidenceParam` is zero-initialized in storage and is never set in the constructor. [1](#0-0) 

`setConfidenceParam` only rejects values `> CONFIDENCE_MAX`; `newValue == 0` passes silently. [2](#0-1) 

**Step 2 вЂ” Zero `confidenceParam` collapses spread to zero.**

```solidity
// comment: "0 means no spread"
uint256 adjustedSpread = spread * confidenceParam;   // = 0
(uint256 bid, uint256 ask) = _getBidAskFrom(mid, adjustedSpread);
``` [3](#0-2) 

Inside `_getBidAskFrom`, with `confidence == 0`:

```solidity
uint256 delta = midPrice * 0 / CONFIDENCE_BASE;  // = 0
bid = midPrice;   // delta < midPrice
ask = midPrice;   // bid == ask
``` [4](#0-3) 

**Step 3 вЂ” Step adjustment with `marginStep == 0` produces `bidOut в‰€ askOut`.**

With `marginStep == 0`, `stepBidFactor == stepAskFactor == BPS_BASE_U`. The only difference between `bidOut` and `askOut` is `Math.Rounding.Floor` vs `Math.Rounding.Ceil` applied to the same value `mid * Q64 / ORACLE_DECIMALS`. [5](#0-4) 

- If `mid * Q64` is exactly divisible by `ORACLE_DECIMALS (1e8)`: `bidOut == askOut` в†’ the guard at line 228 fires в†’ returns `(0, type(uint128).max)` в†’ `getBidAndAskPrice()` reverts with `FeedStalled()`.
- Otherwise: `askOut = bidOut + 1` (one Q64 unit в‰€ 5.4 Г— 10вЃ»ВІвЃ° in oracle-decimal space) вЂ” a quote with effectively zero spread passes through. [6](#0-5) 

**Step 4 вЂ” Contrast with `AnchoredPriceProvider`.**

`AnchoredPriceProvider` explicitly documents and handles the `confidence == 0` case: its band clamp (`min(refBid, cBid)` / `max(refAsk, cAsk)`) restores ordering even when the shaped quote degenerates to `bid == ask`. `PriceProvider` has no such clamp and silently fails. [7](#0-6) 

---

### Impact Explanation

**Stall path (exact divisibility):** `getBidAndAskPrice()` reverts with `FeedStalled()`. Every swap through the pool fails. Core pool swap functionality is completely broken вЂ” matching the "Broken core pool functionality causing loss of funds or unusable withdraw/swap/liquidity flows" impact gate.

**Zero-spread path (typical case):** The pool quotes at the exact oracle mid with a 1-unit Q64 spread. LPs earn no spread-based compensation. Traders can swap at mid with no cost, draining LP value вЂ” matching the "Swap conservation failure" impact gate.

---

### Likelihood Explanation

- `confidenceParam` is **zero by default** in every freshly deployed `PriceProvider`. Any pool that begins accepting liquidity or swaps before the factory calls `setConfidenceParam` is immediately affected.
- The factory is permitted to call `setConfidenceParam(0)` at any time (no lower-bound guard), re-triggering the condition on a live pool.
- `marginStep == 0` is the natural default (no CEX step bias), making the combination common.

---

### Recommendation

1. **Reject zero in `setConfidenceParam`:** Add `if (newValue == 0) revert ConfidenceParamOutOfBounds();` alongside the upper-bound check.
2. **Initialize `confidenceParam` in the constructor** to a safe non-zero default (e.g., `1`) so the pool is never in the broken state before the factory acts.
3. **Or** add an explicit guard in `_getBidAndAskPrice` that returns the stall sentinel immediately when `confidenceParam == 0`, with a clear revert reason, rather than silently producing a degenerate quote.

---

    function _getBidAndAskPrice() internal returns (uint128, uint128) {
        // 1. Read via the unified price(feedId, pool) path, forwarding the pool (msg.sender).
        //    refTime is already in seconds.
        (uint256 mid, uint256 spread, , uint256 refTime) =
            IPricedOracle(address(offchainOracle)).price(offchainFeedId, msg.sender);

        // 2. Staleness check
        if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA)) {
            return (0, type(uint128).max);
        }

        // 3. Basic validity вЂ” price must be positive, spread must not be stalled marker
        if (mid == 0 || spread >= ORACLE_BPS) {
            return (0, type(uint128).max);
        }

        // 4. Price guard check (moved from oracle)
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(offchainFeedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) {
            return (0, type(uint128).max);
        }

        // 5. Compute bid/ask from mid + confidence-adjusted spread
        //    confidenceParam multiplies oracle spread; 0 means no spread
        uint256 adjustedSpread = spread * confidenceParam;
        (uint256 bid, uint256 ask) = _getBidAskFrom(mid, adjustedSpread);

        // 6. Apply marginStep adjustment
        (uint256 bidOut, bool bidOk) = _applyBidAdjustments(bid);
        if (!bidOk || bidOut > type(uint128).max) return (0, type(uint128).max);

        (uint256 askOut, bool askOk) = _applyAskAdjustments(ask);
        if (!askOk || askOut > type(uint128).max) return (0, type(uint128).max);

        // 7. Hard invariant: bid must be strictly less than ask.
        //    Can be violated when marginStep < 0 and confidence is too small.
        if (bidOut >= askOut) return (0, type(uint128).max);

        return (uint128(bidOut), uint128(askOut));
    }
