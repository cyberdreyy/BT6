### Title
`allowPushers` re-establishes a revoked delegation using the same signature, neutralising `revokePusher()` as a safety mechanism — (File: `smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1::allowPushers` does not check whether a pusher has already self-revoked via `revokePusher()`. Because the EIP-191 consent signature covers only `(chainid, oracle, deadline, pusher, creator)` — none of which change after revocation — the creator can replay the exact same signature to re-establish the delegation immediately after the pusher clears it, for as long as the deadline has not expired. The code comment on line 190 explicitly acknowledges the risk ("an undated signature could re-establish a delegation AFTER the pusher revoked it") but the deadline only prevents replay *after* the deadline, not *within* the deadline window.

---

### Finding Description

`revokePusher()` clears `namespaceRemapping[msg.sender]` to `address(0)`:

```solidity
// CompressedOracle.sol:238-243
function revokePusher() external {
    address creator = namespaceRemapping[msg.sender];
    if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
    namespaceRemapping[msg.sender] = address(0);
    emit PusherRevoked(msg.sender, creator);
}
```

`allowPushers` then unconditionally overwrites `namespaceRemapping[pusher]` back to `msg.sender` whenever the signature verifies and `block.timestamp <= deadline`:

```solidity
// CompressedOracle.sol:192-212
function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
    _ensureDeadline(deadline);
    ...
    bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
        keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
    );
    require(pusher == ECDSA.recover(hash, signatures[i]));
    namespaceRemapping[pusher] = msg.sender;   // ← no check for prior revocation
    ...
}
```

The signed message contains no revocation nonce, no "used" flag, and no per-revocation counter. The same `(chainid, oracle, deadline, pusher, creator)` tuple is valid for the entire lifetime of the deadline, so the creator can call `allowPushers` with the original signature an unlimited number of times within the deadline window, each time undoing the pusher's self-revocation.

The `fallback()` push path then routes the pusher's calldata into the creator's namespace:

```solidity
// CompressedOracle.sol:315-316
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
```

If the pusher is an automated off-chain system that called `revokePusher()` because its data source became compromised, it will continue emitting slot words into what it believes is its own namespace. Because the creator has silently re-established the remapping, those slot words land in the creator's namespace instead, and any `PriceProvider` / `AnchoredPriceProvider` backed by that namespace will serve the corrupted prices to pools.

---

### Impact Explanation

A creator (semi-trusted party) can force a pusher's slot writes to continue landing in the creator's namespace even after the pusher explicitly revoked consent. If the pusher's data source is compromised at the time of revocation, the creator can keep the bad-price stream alive in the oracle slot that backs a live pool's `IPriceProvider`. Every subsequent `pool.swap()` call will read the corrupted mid price through `_readLeg` / `_getBidAndAskPrice`, producing an incorrect bid/ask pair that causes traders to receive more or less than the true market price — a direct bad-price execution impact.

---

### Likelihood Explanation

The trigger is a semi-trusted creator acting in bad faith (or a creator who is itself compromised). The pusher must be an automated system that does not re-check `namespaceRemapping` after calling `revokePusher()`. Both conditions are realistic in production deployments where pushers are bots and creators are protocol operators. The deadline window is typically set to hours or days, giving the creator ample time to replay the signature before the pusher's automated system detects the re-establishment.

---

### Recommendation

Introduce a per-pusher revocation nonce and include it in the signed digest:

```solidity
mapping(address => uint256) public pusherRevocationNonce;

function revokePusher() external {
    ...
    namespaceRemapping[msg.sender] = address(0);
    pusherRevocationNonce[msg.sender]++;          // invalidate all prior consents
    emit PusherRevoked(msg.sender, creator);
}

// In allowPushers, include the nonce in the hash:
keccak256(abi.encode(
    block.chainid, address(this), deadline,
    pusher, msg.sender,
    pusherRevocationNonce[pusher]   // ← binds consent to current revocation epoch
))
```

After `revokePusher()` increments the nonce, any previously issued signature becomes invalid because the recovered digest no longer matches.

---

### Proof of Concept

```
T=0  Creator calls allowPushers(deadline=T+1day, [pusher], [sig])
       → namespaceRemapping[pusher] = creator

T=1  Pusher's data source is compromised; pusher calls revokePusher()
       → namespaceRemapping[pusher] = address(0)
       → pusher believes its fallback pushes now go to its OWN namespace

T=2  Creator calls allowPushers(deadline=T+1day, [pusher], [sig])  ← SAME sig, still valid
       → namespaceRemapping[pusher] = creator  (re-established, no revert)

T=3  Pusher's automated bot pushes a corrupted slot word (bad price)
       → fallback() resolves namespaceRemapping[pusher] = creator
       → slot written into creator's namespace with bad price

T=4  pool.swap() → provider.getBidAndAskPrice() → oracle.price(feedId, pool)
       → _readLeg returns corrupted mid; bid/ask computed from bad price
       → trader executes swap at wrong price
```

The creator needs only one transaction at T=2 to undo the pusher's safety action. No new signature from the pusher is required. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)

    /// @notice Delegates pusher wallets into the caller's namespace. The pusher's EIP-191
    ///         signature is REQUIRED — without it anyone could remap a foreign pusher
    ///         wallet into their own namespace and silently swallow its pushes. The
    ///         deadline is likewise required: the signed consent carries no timestamp of
    ///         its own, so an undated signature could re-establish a delegation AFTER the
    ///         pusher revoked it.
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }

    fallback() override external {
        uint256 end;
        uint256 namespace;

        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0)) creator = msg.sender;

        assembly ("memory-safe") {
            end := calldatasize()
            namespace := shl(96, creator) // [creator:20][zeros:12]
        }

        // 4 * 6 + 7 + 1 = 32 bytes per slot
        if (end == 0 || end % 32 != 0) revert BadCalldataLength();

        for (uint256 ptr = 0; ptr < end; ptr += 32) {
            uint256 word;
            assembly ("memory-safe") {
                word := calldataload(ptr)
            }
            // casting to 'uint8' is safe we want LSB
            // forge-lint: disable-next-line(unsafe-typecast)

    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
