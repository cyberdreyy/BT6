### Title
Slot-Level Timestamp in `CompressedOracleV1` Masks Per-Lane Price Staleness, Allowing Stale Prices to Reach Pool Swaps — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1` packs four oracle feed lanes into a single 256-bit storage slot with **one shared 56-bit timestamp**. When a pusher updates the slot to refresh any lane, the timestamp advances for **all four lanes simultaneously**. A consumer's staleness check (`_isStale`) uses this slot-level timestamp as `refTime`, so lanes whose price data has not actually changed appear fresh. Stale prices bypass the staleness guard and reach pool swaps via `AnchoredPriceProvider`.

---

### Finding Description

**Root cause — shared slot timestamp, no per-lane timestamp:**

`CompressedOracleV1._loadSlotLayout` reads one 256-bit storage word and extracts a single `timestampMs` field shared by all four lanes: [1](#0-0) 

`getOracleData` then assigns this slot-level timestamp to every lane it returns, regardless of which lane is being read: [2](#0-1) 

`_price` surfaces this as `refTime`: [3](#0-2) 

**Push path — entire slot overwritten atomically:**

Both `fallback()` and `updateBySignature()` overwrite the entire 256-bit slot word in one `sstore`. There is no mechanism to update a single lane independently: [4](#0-3) 

The slot-structure documentation confirms this explicitly: *"The entire slot is overwritten on update: if you update a single lane, you must still supply correct values for the other lanes."* [5](#0-4) 

**Consumer staleness check uses the slot-level timestamp:**

`AnchoredPriceProvider._readLeg` calls `oracle.price(feedId, pool)` and checks `_isStale(refTime, block.timestamp, MAX_REF_STALENESS)`. `refTime` is the slot-level timestamp, not a per-lane timestamp: [6](#0-5) 

The staleness check itself: [7](#0-6) 

**The gap:** If a pusher updates the slot frequently to keep lane 0 (e.g., ETH/USD) fresh, the slot timestamp advances on every push. Lane 2 (e.g., stETH/ETH) receives the same fresh timestamp even though its 48-bit price field was simply copied from the previous push. The consumer's `_isStale` check passes, and the stale price for lane 2 is consumed by the pool as if it were current.

The test suite itself acknowledges the per-slot timestamp design: [8](#0-7) 

---

### Impact Explanation

A pool whose `AnchoredPriceProvider` is bound to a `CompressedOracleV1` feed that shares a slot with a more-frequently-updated lane will receive stale bid/ask quotes at swap time. The `MAX_REF_STALENESS` guard — the only on-chain freshness protection — is defeated because the slot timestamp is refreshed by pushes to sibling lanes. This satisfies the **bad-price execution** impact gate: a stale, unclamped bid/ask quote reaches a pool swap, allowing traders to extract value at the expense of LPs or causing the pool to accept incorrect input amounts.

---

### Likelihood Explanation

The `CompressedOracleV1` design explicitly encourages packing multiple feeds into one slot (4 lanes per slot, 256 slots per creator). A natural operational configuration places feeds with different liquidity profiles — one liquid (updated every few seconds) and one illiquid (updated every few hours) — in the same slot. The pusher, acting correctly for the liquid feed, inadvertently refreshes the timestamp for the illiquid feed. No malicious intent is required; the structural constraint (whole-slot overwrite) makes this the default outcome. The trigger is a valid semi-trusted pusher operating normally.

---

### Recommendation

1. **Per-lane timestamps:** Extend the slot layout to include a per-lane timestamp (e.g., reduce each lane from 48 bits to 40 bits and use the freed 8 bits × 4 lanes = 32 bits for per-lane timestamps, or use a second storage slot for timestamps). Consumers must validate freshness against the lane's own timestamp, not the slot's.

2. **Alternatively, enforce same-slot update discipline:** Document and enforce (via off-chain tooling or an on-chain check) that all lanes in a slot must be updated with data no older than `MAX_REF_STALENESS` on every push. This is a weaker mitigation because it relies on off-chain correctness.

3. **Stalled-marker propagation:** If a pusher cannot supply fresh data for a lane, they should write the explicit stalled marker (`s0 = 0xFF, s1 = 0xFF`) for that lane. The consumer already rejects this marker (`spreadBps >= ORACLE_BPS`). Pushers should be required to use this marker rather than repeating old prices.

### Proof of Concept

```
Setup:
  creator deploys CompressedOracleV1 with MAX_TIME_DRIFT = 60s
  creator assigns slot 0, lane 0 → ETH/USD feed
  creator assigns slot 0, lane 2 → stETH/ETH feed
  AnchoredPriceProvider bound to lane 2 (stETH/ETH), MAX_REF_STALENESS = 120s

T=0:
  pusher pushes slot 0 with:
    lane 0: ETH/USD = 3000e8, fresh
    lane 2: stETH/ETH = 0.998e8, fresh
  slot timestamp = T=0

T=3600 (1 hour later, stETH/ETH has drifted to 0.95e8 on-market):
  pusher pushes slot 0 with:
    lane 0: ETH/USD = 3100e8, fresh
    lane 2: stETH/ETH = 0.998e8  ← repeated old value (pusher has no fresh stETH data)
  slot timestamp = T=3600

Pool swap at T=3600:
  AnchoredPriceProvider._readLeg(lane2_feedId)
    → oracle.price() returns mid=0.998e8, refTime=3600
    → _isStale(3600, 3600, 120) = false  ← PASSES
    → pool quotes stETH/ETH at 0.998e8 instead of true 0.95e8
    → trader swaps stETH for ETH at inflated stETH price, extracting ~5% excess value from LPs
```

    function _loadSlotLayout(bytes32 slotIndex) internal view returns (SlotLayout memory _layout) {
        uint256 slotValue;
        assembly ("memory-safe") {
            slotValue := sload(slotIndex)
        }

        _layout.timestampMs = toTimeMs(slotValue >> 8 & X56);

        _layout.oracle0 = _decodeCompressedOracleData(uint48((slotValue >> 208) & X48));
        _layout.oracle1 = _decodeCompressedOracleData(uint48((slotValue >> 160) & X48));
        _layout.oracle2 = _decodeCompressedOracleData(uint48((slotValue >> 112) & X48));
        _layout.oracle3 = _decodeCompressedOracleData(uint48((slotValue >> 64) & X48));
    }

Updates overwrite the entire storage slot value.

- `fallback()` accepts one or more 32-byte slot words (no prefix; calldata length must
  be a non-zero multiple of 32).
- `updateBySignature(feedCreator, newSlotValue, signature)` accepts a slot word signed
  by the feed creator over `keccak256(abi.encode(chainid, oracleAddress, feedCreator,
  newSlotValue))`, and is callable by anyone.

There is **no deadline** on either push path: each word carries its own timestamp and
the per-slot monotonicity check neutralizes replay (a replayed word is "not newer" and
is skipped).

    function _isStale(
        uint256 refTime,
        uint256 nowTs,
        uint256 maxDelta
    ) internal pure returns (bool) {
        if (refTime == 0) return true;
        if (refTime > nowTs) return true;
        return (nowTs - refTime) > maxDelta;
    }

    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
