Looking at the `allowPushers` and `revokePusher` functions in detail:

### Title
Signature Replay After `revokePusher` Allows Creator to Permanently Re-delegate Pusher — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowPushers` has no nonce, no used-signature bitmap, and no on-chain invalidation path. A pusher's EIP-191 consent is replayable by the creator as many times as the deadline permits. With `deadline = type(uint256).max` the signature is valid forever, so every `revokePusher` call by the pusher can be immediately undone by the creator replaying the same bytes.

---

### Finding Description

`allowPushers` constructs the signed digest as:

```
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [1](#0-0) 

The only freshness gate is `_ensureDeadline`, which is `block.timestamp <= deadline`. [2](#0-1) 

There is no nonce, no used-signature set, and no state that is mutated to invalidate a previously consumed signature. `revokePusher` only zeroes `namespaceRemapping[msg.sender]`: [3](#0-2) 

It does not touch any signature-level state. The creator therefore holds a replayable credential and can call `allowPushers` again with the identical `(deadline, pusher, signature)` tuple to restore `namespaceRemapping[pusher] = creator` immediately after every revocation.

The code comment in `allowPushers` explicitly acknowledges the design intent:

> "an undated signature could re-establish a delegation AFTER the pusher revoked it" [4](#0-3) 

The deadline is the intended guard, but the protocol imposes no upper bound on `deadline`, so a pusher who signs with `deadline = type(uint256).max` (or any far-future value) provides a credential that satisfies the guard indefinitely.

---

### Impact Explanation

After re-delegation, `fallback()` resolves the pusher's namespace as:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [5](#0-4) 

Every push the pusher makes lands in the creator's storage slots, not the pusher's own namespace. The pusher's own feeds (used by pools via `feedIdOf(pusher, ...)`) receive no updates and go stale. Any pool or provider consuming those feeds will read `timestampMs = 0` or an old timestamp, which every consumer rejects as stale — effectively bricking the pusher's own oracle feeds and any pool that depends on them.

---

### Likelihood Explanation

- The pusher must have signed with a far-future or max deadline. This is a realistic operational choice (e.g., a bot operator signs once and never wants to re-sign).
- The creator must be adversarial and willing to replay. This is a semi-trusted but not fully trusted party.
- No privileged role (factory owner, oracle admin) is required — `allowPushers` is a public function callable by any `msg.sender`.
- The attack is free (only gas cost) and repeatable indefinitely.

---

### Recommendation

Add a per-pusher nonce or a used-signature mapping. On `revokePusher`, increment the pusher's nonce (or record the signature hash as spent). Include the nonce in the signed digest so any previously issued signature is automatically invalidated:

```solidity
mapping(address => uint256) public pusherNonce;

// In allowPushers digest:
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender, pusherNonce[pusher]))

// In revokePusher:
pusherNonce[msg.sender]++;
```

Alternatively, enforce a protocol-level maximum deadline (e.g., 30 days from `block.timestamp` at signing time) so that any issued credential eventually expires.

---

### Proof of Concept

```solidity
// Foundry test sketch
function test_replayAfterRevoke() public {
    uint256 deadline = type(uint256).max;
    // pusher signs consent once
    bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
        keccak256(abi.encode(block.chainid, address(oracle), deadline, pusher, creator))
    );
    (uint8 v, bytes32 r, bytes32 s) = vm.sign(pusherKey, hash);
    bytes memory sig = abi.encodePacked(r, s, v);

    address[] memory pushers = new address[](1);
    pushers[0] = pusher;
    bytes[] memory sigs = new bytes[](1);
    sigs[0] = sig;

    for (uint256 i = 0; i < 3; i++) {
        // creator delegates
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);
        assertEq(oracle.namespaceRemapping(pusher), creator);

        // pusher revokes
        vm.prank(pusher);
        oracle.revokePusher();
        assertEq(oracle.namespaceRemapping(pusher), address(0));
    }

    // creator re-delegates a 4th time — revocation is not final
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs);
    assertEq(oracle.namespaceRemapping(pusher), creator); // pusher trapped
}
```
