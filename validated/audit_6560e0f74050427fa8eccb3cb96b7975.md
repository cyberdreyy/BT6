### Title
Creator Replays Pusher Consent Signature to Nullify `revokePusher()` and Permanently Redirect Pushes Until Deadline Expires — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowPushers()` in `CompressedOracleV1` contains no nonce, no used-signature bitmap, and no on-chain record that a given consent signature has already been consumed. A creator who retains a pusher's still-valid EIP-191 consent signature can call `allowPushers()` again after the pusher has self-revoked via `revokePusher()`, overwriting `namespaceRemapping[pusher] = address(0)` back to `namespaceRemapping[pusher] = creator`. The pusher's revocation is silently nullified, their future fallback pushes continue to land in the creator's namespace, and the pusher's own namespace receives no updates — exactly the state-overwrite-with-stale-value pattern from the external report.

---

### Finding Description

`allowPushers()` signs and verifies:

```
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [1](#0-0) 

The signed tuple contains no nonce, no "already-used" flag, and no reference to the current `namespaceRemapping` state. The only replay guard is `_ensureDeadline(deadline)`, which rejects signatures whose deadline has already passed. [2](#0-1) 

The code comment explicitly acknowledges the concern — "an undated signature could re-establish a delegation AFTER the pusher revoked it" — and claims the deadline is the mitigation: [3](#0-2) 

But the deadline only blocks *expired* signatures. A signature whose deadline is still in the future passes `_ensureDeadline` unconditionally, regardless of whether the pusher has already revoked. The creator can therefore call `allowPushers()` a second time with the identical `(deadline, [pusher], [sig])` arguments, and the assignment

```solidity
namespaceRemapping[pusher] = msg.sender;   // overwrites address(0) → creator
``` [4](#0-3) 

silently undoes the pusher's `revokePusher()` call:

```solidity
namespaceRemapping[msg.sender] = address(0);   // pusher's intent
``` [5](#0-4) 

The fallback push path resolves the namespace at call time:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [6](#0-5) 

With the remapping restored to `creator`, every subsequent fallback push from the pusher lands in the creator's namespace, not the pusher's own. The pusher's own namespace (`feedIdOf(pusher, slotIndex, positionIndex)`) receives no updates and its stored `timestampMs` stays at whatever value it had before — or zero if it was never written.

---

### Impact Explanation

Any pool or `AnchoredPriceProvider` configured to read from the pusher's own namespace will observe a stale or zero `refTime`. The staleness check in `_readLeg` treats `refTime == 0` as stale:

```solidity
if (refTime == 0) return true;   // _isStale
``` [7](#0-6) 

`_readLeg` then returns `ok = false`, and `getBidAndAskPrice()` reverts with `FeedStalled`:

```solidity
if (bid == 0 || ask == type(uint128).max) revert FeedStalled();
``` [8](#0-7) 

Every swap through that pool reverts for the entire window between the pusher's revocation and the deadline expiry — broken core pool functionality, unusable swap flow.

---

### Likelihood Explanation

- The pusher must have signed a consent with a future deadline (normal operational practice).
- The creator retains the signature (it was submitted on-chain in the original `allowPushers` call, so it is trivially recoverable from transaction history).
- The creator calls `allowPushers` again with the same arguments — one transaction, no special privilege.
- The window of exposure equals the remaining time until the deadline, which can be days or weeks depending on how the consent was originally signed.

The trigger is fully unprivileged from the creator's perspective (creators are semi-trusted, not oracle admins) and requires no malicious initial setup beyond retaining a public on-chain signature.

---

### Recommendation

Track consumed signatures with a used-signature bitmap or include a monotonic nonce in the signed payload:

```solidity
// Option A: nonce per (creator, pusher) pair
mapping(address => mapping(address => uint256)) public delegationNonce;

// signed payload becomes:
keccak256(abi.encode(
    block.chainid, address(this), deadline,
    pusher, msg.sender,
    delegationNonce[msg.sender][pusher]++
))
```

Alternatively, record the signature hash and reject re-use:

```solidity
mapping(bytes32 => bool) public usedDelegationSigs;
// after recovery: require(!usedDelegationSigs[hash]); usedDelegationSigs[hash] = true;
```

Either approach ensures that a pusher's `revokePusher()` cannot be silently overwritten by replaying the original consent.

---

### Proof of Concept

```
1. creator calls allowPushers(deadline = now + 7 days, [pusher], [sig])
   → namespaceRemapping[pusher] = creator  ✓

2. pusher calls revokePusher()
   → namespaceRemapping[pusher] = address(0)  ✓ (pusher's intent)

3. creator calls allowPushers(deadline = now + 7 days, [pusher], [sig])
   (same arguments, deadline still valid, no nonce check)
   → namespaceRemapping[pusher] = creator  ← overwrites revocation

4. pusher sends fallback push (e.g., fresh price data)
   → fallback resolves namespaceRemapping[pusher] = creator
   → push lands in creator's namespace, NOT pusher's own

5. Pool reading feedIdOf(pusher, slotIndex, positionIndex):
   → timestampMs = 0 (never written)
   → _isStale returns true → _readLeg ok = false
   → getBidAndAskPrice() reverts FeedStalled
   → all swaps through that pool revert
```

The replay succeeds because `_ensureDeadline` only checks `deadline > block.timestamp` — it has no memory of prior invocations with the same signature. [9](#0-8)

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
```
