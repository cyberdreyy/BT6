### Title
`allowContractPushers` Overwrites an Existing Namespace Mapping Without Checking the Prior Owner, Enabling Any Caller to Silently Hijack a Live Pusher and Freeze Pool Prices - (File: smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol)

### Summary

`CompressedOracleV1.allowContractPushers` maps a pusher contract into the caller's namespace after a live `isPusher(creator)` staticcall. It never checks whether `namespaceRemapping[pusher]` is already set to a *different* creator. Any address that can make the pusher contract return `true` for itself can unconditionally overwrite the existing mapping, silently redirecting all future pushes away from the legitimate creator's namespace. Because the `CompressedOracleV1` is the oracle backing `AnchoredPriceProvider`, the creator's feeds immediately go stale, the provider halts with `FeedStalled`, and every pool that depends on those feeds becomes unusable.

### Finding Description

`allowContractPushers` performs exactly one authorization check — a live `isPusher(msg.sender)` staticcall on the pusher contract — and then unconditionally writes `namespaceRemapping[pusher] = msg.sender`:

```solidity
// CompressedOracle.sol lines 217-234
function allowContractPushers(address[] calldata pushers) external {
    uint256 l = pushers.length;
    for (uint256 i; i < l; i++) {
        address pusher = pushers[i];

        if (pusher == msg.sender) {
            revert NoSelfRemapping();
        }

        (bool ok, bytes memory res) = pusher.staticcall(
            abi.encodeWithSignature("isPusher(address)", msg.sender)
        );
        require(ok);
        bool allowed = abi.decode(res, (bool));
        require(allowed);

        namespaceRemapping[pusher] = msg.sender;   // ← no prior-owner check
        emit PusherAuthorized(pusher, msg.sender);
    }
}
``` [1](#0-0) 

There is no guard of the form `require(namespaceRemapping[pusher] == address(0) || namespaceRemapping[pusher] == msg.sender)`. The contrast with the EOA path (`allowPushers`) is instructive: that path binds the signature to the specific `msg.sender` (creator), so a signature issued to Creator A cannot be replayed by Creator B. The contract-pusher path has no equivalent binding — the live `isPusher` call is the *only* gate, and if the pusher contract returns `true` for more than one address, any of those addresses can overwrite the mapping at will. [2](#0-1) 

Once the mapping is overwritten, the `fallback` push path resolves the namespace from `namespaceRemapping[msg.sender]`:

```solidity
// CompressedOracle.sol lines 315-316
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
```
