### Title
`allowPushers` Lacks Revocation Nonce, Enabling Creator to Replay Pusher Consent After `revokePusher` Within Deadline Window — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary
`allowPushers` accepts any valid EIP-191 signature over `(block.chainid, address(this), deadline, pusher, creator)` as long as `block.timestamp <= deadline`. After a pusher self-revokes via `revokePusher`, the creator can immediately replay the original signature (unchanged) to re-establish `namespaceRemapping[pusher] = creator`. There is no revocation nonce, no consumed-signature bitmap, and no check that the pusher has not previously revoked.

### Finding Description
The signed consent tuple is `(chainid, oracle, deadline, pusher, creator)`. It contains no monotonic counter or revocation epoch. `revokePusher` writes `namespaceRemapping[pusher] = address(0)` but does not invalidate the signature — the signature remains cryptographically valid until `deadline`. A creator holding the original `bytes` value can call `allowPushers(deadline, [pusher], [sig])` a second time, passing both guards (`_ensureDeadline` and `ECDSA.recover`), and the mapping is silently restored.

Attack sequence:
1. Pusher signs consent; creator calls `allowPushers` → `namespaceRemapping[pusher] = creator`.
2. Pusher calls `revokePusher` → `namespaceRemapping[pusher] = address(0)`.
3. Creator calls `allowPushers` again with the identical `(deadline, sig)` before expiry → `namespaceRemapping[pusher] = creator` restored.
4. Pusher, believing revocation succeeded, pushes data intended for their own namespace.
5. The fallback dispatch reads `namespaceRemapping[pusher]` and routes the write into the creator's namespace instead.

### Impact Explanation
The pusher's data lands in the creator's feed slots rather than the pusher's own. If the pusher is operating their own feed (e.g., for a different token pair or price range), their own namespace receives no update (stale/zero price), while the creator's feed receives data the pusher never intended for it. Any pool consuming the creator's feed via `AnchoredPriceProvider` or `CompressedOracleV1.price()` will execute swaps against a corrupted mid/spread, constituting bad-price execution and potential loss of user principal.

### Likelihood Explanation
Requires a creator who retains the pusher's original signature bytes (trivially available from calldata of the first `allowPushers` call on-chain) and acts within the deadline window. No privileged role is needed — the creator is an ordinary user. The pusher has no on-chain way to detect the re-delegation without polling `namespaceRemapping` before every push.

### Recommendation
Introduce a per-pusher revocation nonce stored in contract state. Include the nonce in the signed digest and increment it on every successful `revokePusher` call. `allowPushers` must read the current nonce and reject any signature whose embedded nonce does not match.

```solidity
// storage
mapping(address => uint256) public pusherNonce;

// in allowPushers:
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(this), deadline,
                         pusher, msg.sender, pusherNonce[pusher]))
);

// in revokePusher:
namespaceRemapping[msg.sender] = address(0);
pusherNonce[msg.sender]++;          // invalidates all prior signatures
```

    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs);  // succeeds — no revert

    // Invariant violated: revoked pusher is re-delegated without fresh consent
    assertEq(oracle.namespaceRemapping(pusher), creator);

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
