### Title
`revokePusher()` is ineffective within the deadline window — creator can replay pusher's consent signature to re-establish delegation after revocation - (File: smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol)

### Summary
`allowPushers` contains no nonce and no used-signature tracking. A pusher's EIP-191 consent signature, once used to establish delegation, can be replayed by the creator at any time before the deadline expires — including after the pusher has called `revokePusher()`. This breaks the only self-protection mechanism a pusher has against a compromised key writing bad prices into the creator's namespace.

### Finding Description
`allowPushers` verifies a pusher's consent signature whose domain is `(chainid, oracle_address, deadline, pusher, creator)` and then unconditionally writes `namespaceRemapping[pusher] = msg.sender`. [1](#0-0) 

There is no nonce, no `usedSignatures` bitmap, and no per-pusher revocation counter. The only replay barrier is the deadline check: [2](#0-1) 

`revokePusher()` clears the mapping to `address(0)`: [3](#0-2) 

Because the signature is not consumed or invalidated on first use, the creator can call `allowPushers` again with the identical `(deadline, pusher, sig)` tuple at any point while `block.timestamp <= deadline`, restoring `namespaceRemapping[pusher] = creator` and silently undoing the revocation.

The code comment at line 189–191 explicitly acknowledges this risk and claims the deadline is the mitigation: [4](#0-3) 

However, the deadline only prevents replay **after** it expires. Within the window it provides no protection at all — the same signature is valid for the entire `[0, deadline]` interval.

### Impact Explanation
Once delegation is re-established against the pusher's will, the pusher's key (or an attacker who has compromised it) can call the `fallback()` push path and write arbitrary prices into the creator's namespace: [5](#0-4) 

Any pool whose `AnchoredPriceProvider` is bound to a feed in that namespace will consume the corrupted price through `_readLeg → oracle.price(feedId, pool)`: [6](#0-5) 

A sufficiently skewed mid price that still passes the `priceGuard` and `MAX_SPREAD_BPS` checks will produce a bad bid/ask that reaches live pool swaps, enabling the attacker to extract user principal from the pool.

### Likelihood Explanation
The trigger is semi-trusted: the creator must call `allowPushers` a second time with the old signature. This can happen through an automated keeper that periodically re-establishes delegation without checking whether the pusher has revoked, or through a creator who is unaware of the revocation. The pusher's key compromise is the external precondition; the re-delegation is the on-chain enabler. Consent signatures are broadcast on-chain in calldata and are trivially recoverable by any observer, so the creator does not need to store them separately.

### Recommendation
Track consumed signatures to make each consent single-use. The simplest approach is a per-pusher revocation nonce:

```solidity
mapping(address => uint256) public pusherNonce;

// include nonce in the signed domain:
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender, pusherNonce[pusher]))

// revokePusher increments the nonce, invalidating all prior signatures:
function revokePusher() external {
    ...
    pusherNonce[msg.sender]++;
    namespaceRemapping[msg.sender] = address(0);
    ...
}
```

Alternatively, store a `usedSignatures` mapping keyed on the signature hash and revert on reuse.

### Proof of Concept
```
1. Pusher signs consent: sig = sign(chainid, oracle, deadline=T+1day, pusher, creator)
2. Creator calls allowPushers(T+1day, [pusher], [sig])
   → namespaceRemapping[pusher] = creator  ✓

3. Pusher's key is compromised; pusher calls revokePusher()
   → namespaceRemapping[pusher] = address(0)  ✓ (revocation succeeds)

4. Creator (automated keeper) calls allowPushers(T+1day, [pusher], [sig]) again
   — same deadline, same pusher, same sig, block.timestamp still ≤ T+1day
   → namespaceRemapping[pusher] = creator  ← revocation silently undone

5. Attacker (holding compromised pusher key) calls oracle.fallback() with
   a crafted slot word containing an inflated price and a fresh timestamp.
   → _writeStorage(key, badSlotValue)  ← bad price stored in creator namespace

6. Pool's AnchoredPriceProvider reads oracle.price(feedId, pool)
   → _readLeg returns inflated mid, passes priceGuard and MAX_SPREAD_BPS
   → getBidAndAskPrice() returns skewed bid/ask
   → swap executes at bad price, attacker extracts user principal
``` [7](#0-6)

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }

        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0)) creator = msg.sender;

            end := calldatasize()

        // 4 * 6 + 7 + 1 = 32 bytes per slot

    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
