### Title
Pusher Revocation Bypassed by Creator Re-Using Original Consent Signature Before Deadline — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowPushers` validates only that the pusher's EIP-191 consent signature is unexpired, but never checks whether the pusher has already self-revoked. A creator can replay the original signature at any time before its deadline to silently re-establish a delegation the pusher explicitly cancelled, restoring write access to the creator's namespace for a potentially compromised pusher key.

---

### Finding Description

`allowPushers` enforces two conditions before writing `namespaceRemapping[pusher] = msg.sender`:

1. The deadline has not passed (`_ensureDeadline`).
2. The pusher's ECDSA signature is valid over `(chainid, address(this), deadline, pusher, msg.sender)`. [1](#0-0) 

`revokePusher` clears the mapping: [2](#0-1) 

There is **no check** that `namespaceRemapping[pusher] == address(0)` (i.e., that the pusher has not already revoked) before overwriting the mapping. Because the original signature is permanently valid until its deadline, the creator can call `allowPushers` a second time with the identical `(deadline, [pusher], [sig])` arguments to re-establish the delegation immediately after the pusher revokes.

The documentation itself acknowledges this gap: [3](#0-2) 

The analog to the external bug is exact: `allowPushers` checks the minimum validity condition (deadline not expired) but ignores the revocation state, just as `createLeveragedPosition` checked the minimum ARCH amount but ignored the excess — in both cases the "extra" resource (here, the pusher's write authority) is consumed without the expected guard.

---

### Impact Explanation

Once the delegation is re-established, a party holding the compromised pusher key can call the `fallback` push path and write arbitrary packed slot words into the creator's namespace: [4](#0-3) 

Each word overwrites all four oracle lanes and the shared 56-bit timestamp for a slot. The attacker can set any `U64x32`-encoded price and any `Codebook256` spread indices. Those values are then returned verbatim by `getOracleData` → `price`, consumed by `PriceProviderL2._getBidAndAskPrice`, and forwarded to pool swaps as bid/ask quotes: [5](#0-4) 

An attacker-controlled price that passes the staleness and guard checks (trivially achievable by setting a fresh timestamp and a price within any configured guard range) reaches live pool swaps, constituting **bad-price execution** and enabling the attacker to extract value from traders or LPs.

---

### Likelihood Explanation

- The pusher's original consent signature is in public calldata from the first `allowPushers` transaction; anyone can replay it on behalf of the creator.
- The creator need not be malicious — they may re-delegate in good faith after a pusher revokes (e.g., assuming the revocation was accidental).
- The window is the full remaining lifetime of the deadline, which the documentation notes can be arbitrarily long since the signed consent carries no internal timestamp. [6](#0-5) 

---

### Recommendation

Add a guard in `allowPushers` that rejects re-delegation when the pusher has an active revocation (i.e., `namespaceRemapping[pusher] == address(0)` after a prior revoke). Alternatively, require a fresh signature that commits to a nonce or the current block number so that a revoked pusher's old consent cannot be replayed.

```solidity
// Before: namespaceRemapping[pusher] = msg.sender;
require(namespaceRemapping[pusher] == address(0), "Pusher already delegated or self-revoked");
namespaceRemapping[pusher] = msg.sender;
```

This mirrors the fix for the external bug: just as the ARCH contract should have returned excess tokens, `allowPushers` should refuse to overwrite a cleared mapping without fresh consent.

---

### Proof of Concept

```
1. Creator calls allowPushers(deadline=T+365days, [pusher], [sig])
   → namespaceRemapping[pusher] = creator

2. Pusher's key is compromised.
   Pusher calls revokePusher()
   → namespaceRemapping[pusher] = address(0)

3. Creator (unaware of compromise) calls allowPushers(deadline=T+365days, [pusher], [sig])
   with the IDENTICAL arguments from step 1 (signature still valid, deadline not expired).
   → namespaceRemapping[pusher] = creator  ← revocation silently undone

4. Attacker (holding compromised pusher key) calls oracle.fallback() with a crafted
   32-byte slot word encoding an arbitrary price and fresh timestamp.
   → Slot in creator's namespace overwritten with attacker-controlled price.

5. Pool calls PriceProviderL2.getBidAndAskPrice()
   → offchainOracle.price(feedId, pool) returns attacker's price
   → Swap executes at manipulated bid/ask.
``` [7](#0-6) [2](#0-1) [8](#0-7)

    /// @notice Delegates pusher wallets into the caller's namespace. The pusher's EIP-191
    ///         signature is REQUIRED — without it anyone could remap a foreign pusher
    ///         wallet into their own namespace and silently swallow its pushes. The
    ///         deadline is likewise required: the signed consent carries no timestamp of
    ///         its own, so an undated signature could re-establish a delegation AFTER the
    ///         pusher revoked it.
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }

    fallback() override external {
        uint256 end;
        uint256 namespace;

        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0)) creator = msg.sender;

        assembly ("memory-safe") {
            end := calldatasize()
            namespace := shl(96, creator) // [creator:20][zeros:12]
        }

        // 4 * 6 + 7 + 1 = 32 bytes per slot
        if (end == 0 || end % 32 != 0) revert BadCalldataLength();

        for (uint256 ptr = 0; ptr < end; ptr += 32) {
            uint256 word;
            assembly ("memory-safe") {
                word := calldataload(ptr)
            }
            // casting to 'uint8' is safe we want LSB
            // forge-lint: disable-next-line(unsafe-typecast)
            uint8 slotId = uint8(word);
            TimeMs timestampMs = toTimeMs(word >> 8 & X56);
            timestampMs.revertIfAfterBlockTimeWithDrift(MAX_TIME_DRIFT);
            bytes32 key = bytes32(namespace | uint256(slotId));
            uint256 old = uint256(_loadStorage(key));
            TimeMs oldTimestampMs = toTimeMs(old >> 8 & X56);

            bool newer = timestampMs.isAfter(oldTimestampMs);
            if (!newer) continue;

            _writeStorage(key, bytes32(bytes32(word & ~uint256(0xff))));

    function _getBidAndAskPrice() internal returns (uint128, uint128) {
        // 1. Read via the unified price(feedId, pool) path, forwarding the pool (msg.sender).
        //    refTime is already in seconds.
        (uint256 mid, uint256 spread, , uint256 refTime) =
            IPricedOracle(address(offchainOracle)).price(offchainFeedId, msg.sender);

        // 2. Staleness check
        if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA, FUTURE_TOLERANCE)) {
            return (0, type(uint128).max);
        }

        // 3. Basic validity — price must be positive, spread must not be stalled marker
        if (mid == 0 || spread >= ORACLE_BPS) {
            return (0, type(uint128).max);
        }

        // 4. Price guard check (moved from oracle)
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(offchainFeedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) {
            return (0, type(uint128).max);
        }

        // 5. Compute bid/ask from mid + confidence-adjusted spread
        //    confidenceParam multiplies oracle spread; 0 means no spread
        uint256 adjustedSpread = spread * confidenceParam;
        (uint256 bid, uint256 ask) = _getBidAskFrom(mid, adjustedSpread);

        // 6. Apply marginStep adjustment
        (uint256 bidOut, bool bidOk) = _applyBidAdjustments(bid);
        if (!bidOk || bidOut > type(uint128).max) return (0, type(uint128).max);

        (uint256 askOut, bool askOk) = _applyAskAdjustments(ask);
        if (!askOk || askOut > type(uint128).max) return (0, type(uint128).max);

        // 7. Hard invariant: bid must be strictly less than ask.
        //    Can be violated when marginStep < 0 and confidence is too small.
        if (bidOut >= askOut) return (0, type(uint128).max);

        return (uint128(bidOut), uint128(askOut));
    }
