### Title
Stale `namespaceRemapping` after contract-pusher self-destruction enables arbitrary price injection into creator namespace - (File: `smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`allowContractPushers` verifies pusher consent via a live `isPusher(creator)` staticcall **at registration time only**. The `fallback` push path never re-verifies that the registered contract still exists or still returns `isPusher = true`. If a registered contract pusher is destroyed via `selfdestruct` and a new contract is deployed at the same address (via CREATE2), the stale `namespaceRemapping` entry lets the new contract push arbitrary prices into the creator's namespace with no further authorization check.

### Finding Description

`allowContractPushers` (lines 217–234) proves consent by calling:

```solidity
(bool ok, bytes memory res) = pusher.staticcall(
    abi.encodeWithSignature("isPusher(address)", msg.sender)
);
require(ok);
bool allowed = abi.decode(res, (bool));
require(allowed);
namespaceRemapping[pusher] = msg.sender;
``` [1](#0-0) 

The design comment explicitly states consent is proven by a **live** staticcall. However, the `fallback` push path (lines 311–344) only reads `namespaceRemapping[msg.sender]` to resolve the namespace — it performs no re-verification of the pusher contract's existence or its `isPusher` response:

```solidity
fallback() override external {
    address creator = namespaceRemapping[msg.sender];
    if (creator == address(0)) creator = msg.sender;
    // ... writes directly into creator's namespace
}
``` [2](#0-1) 

Once `namespaceRemapping[pusherAddress] = creator` is written, it persists indefinitely. If the pusher contract is destroyed and a new contract is redeployed at the same address (possible via CREATE2 on any chain, and via `selfdestruct` without EIP-6780 restrictions on HyperEVM), the new contract inherits the stale delegation and can push arbitrary slot words into the creator's namespace.

The `_writeStorage` call overwrites the entire slot with attacker-controlled data: [3](#0-2) 

### Impact Explanation

The `CompressedOracle` implements `price(feedId, pool)` and `getOracleData(feedId)`, making it a valid `IOffchainOracle` source for price providers and pools. Arbitrary prices written into the creator's namespace are returned verbatim by `_price` → `getOracleData`: [4](#0-3) 

A pool consuming this feed will execute swaps at the attacker-controlled mid/spread, causing bad-price execution: traders receive more than the oracle/bin curve permits, or the pool fails to receive owed input. The `AnchoredPriceProvider`'s staleness and `priceGuard` checks do not prevent a freshly-timestamped, in-range but manipulated price from passing through. [5](#0-4) 

### Likelihood Explanation

- **HyperEVM** (no EIP-6780): `selfdestruct` destroys code in a separate transaction. Attacker deploys P via CREATE2, gets creator to call `allowContractPushers([P])`, selfdestructs P, redeploys malicious contract at P. Fully reachable with no privileged access beyond controlling the pusher contract.
- **Ethereum / Base** (EIP-6780 post-Cancun): `selfdestruct` only destroys code if called in the same transaction as deployment. The attack still applies if the pusher contract is an upgradeable proxy — the `isPusher` check is done on the proxy at registration time, but the implementation can be swapped to malicious logic afterward. The `namespaceRemapping` entry remains valid.

The trigger is semi-trusted (requires a creator to have registered the pusher), but the pusher contract owner is a distinct party from the creator, and the oracle provides no mechanism to detect or block post-registration behavioral changes.

### Recommendation

**Short term**: In the `fallback` push path, re-verify that a contract-pusher's `isPusher(creator)` still returns `true` before accepting its push, or add an `EXTCODESIZE` check to revert if the pusher address has no code:

```solidity
// In fallback, before writing:
if (creator != msg.sender) { // delegated push
    uint256 codeSize;
    assembly { codeSize := extcodesize(msg.sender) }
    require(codeSize > 0, "pusher contract destroyed");
    // Optionally: re-call isPusher(creator) for full re-verification
}
```

**Long term**: Emit a `PusherAuthorized` event with a block number and add an on-chain mechanism for creators to re-attest contract pushers periodically, or require contract pushers to re-register after any code change (detectable via codehash comparison stored at registration time).

    /// @notice Contract-pusher variant: consent is proven by a LIVE `isPusher(creator)`
    ///         staticcall instead of a signature, so there is nothing to replay and no
    ///         deadline is needed.
    function allowContractPushers(address[] calldata pushers) external {
        uint256 l = pushers.length;
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            (bool ok, bytes memory res) = pusher.staticcall(abi.encodeWithSignature("isPusher(address)", msg.sender));
            require(ok);
            bool allowed = abi.decode(res, (bool));
            require(allowed);

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }

    fallback() override external {
        uint256 end;
        uint256 namespace;

        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0)) creator = msg.sender;

        assembly ("memory-safe") {
            end := calldatasize()
            namespace := shl(96, creator) // [creator:20][zeros:12]
        }

        // 4 * 6 + 7 + 1 = 32 bytes per slot
        if (end == 0 || end % 32 != 0) revert BadCalldataLength();

        for (uint256 ptr = 0; ptr < end; ptr += 32) {
            uint256 word;
            assembly ("memory-safe") {
                word := calldataload(ptr)
            }
            // casting to 'uint8' is safe we want LSB
            // forge-lint: disable-next-line(unsafe-typecast)
            uint8 slotId = uint8(word);
            TimeMs timestampMs = toTimeMs(word >> 8 & X56);
            timestampMs.revertIfAfterBlockTimeWithDrift(MAX_TIME_DRIFT);
            bytes32 key = bytes32(namespace | uint256(slotId));
            uint256 old = uint256(_loadStorage(key));
            TimeMs oldTimestampMs = toTimeMs(old >> 8 & X56);

            bool newer = timestampMs.isAfter(oldTimestampMs);
            if (!newer) continue;

            _writeStorage(key, bytes32(bytes32(word & ~uint256(0xff))));
        }
    }
