### Title
Single-Step `transferProviderOwnership` Lacks the Two-Step Safeguard Present Everywhere Else in the Protocol — (`smart-contracts-poc/contracts/PriceProviderFactory.sol`, `smart-contracts-poc/contracts/PriceProviderFactoryL2.sol`, `smart-contracts-poc/contracts/AnchoredProviderFactory.sol`)

---

### Summary

`MetricOmmPoolFactory` inherits `Ownable2Step` and implements a two-step pool-admin transfer (`proposePoolAdminTransfer` → `acceptPoolAdmin`). Every privileged role in the core protocol therefore carries a pending-accept safeguard. The three oracle-layer factory contracts — `PriceProviderFactory`, `PriceProviderFactoryL2`, and `AnchoredProviderFactory` — expose `transferProviderOwnership`, which is a **single-step, instant transfer** with no pending-accept pattern. A provider owner who has internalized the two-step norm from the rest of the protocol will perceive a safeguard that does not exist, and a single typo or incompatible multisig address permanently and irrecoverably strips all management rights from the provider.

---

### Finding Description

`MetricOmmPoolFactory` is declared as:

```solidity
contract MetricOmmPoolFactory is Ownable2Step, IMetricOmmPoolFactory, ReentrancyGuardTransient {
``` [1](#0-0) 

Pool-admin ownership follows the same two-step discipline:

```solidity
function proposePoolAdminTransfer(address pool, address newAdmin) external ...
function acceptPoolAdmin(address pool) external ...
``` [2](#0-1) 

By contrast, all three oracle factory contracts implement provider ownership transfer as a single atomic write:

```solidity
// PriceProviderFactory.sol  (identical pattern in PriceProviderFactoryL2 and AnchoredProviderFactory)
function transferProviderOwnership(address provider, address newOwner)
    external override onlyProviderOwner(provider)
{
    require(_providers.contains(provider), ProviderNotTracked());
    require(newOwner != address(0));
    address previousOwner = providerOwner[provider];

    providerOwner[provider] = newOwner;          // ← instant, no pending/accept
    _providersByCreator[previousOwner].remove(provider);
    _providersByCreator[newOwner].add(provider);

    emit ProviderOwnershipTransferred(provider, previousOwner, newOwner);
}
``` [3](#0-2) [4](#0-3) [5](#0-4) 

The `providerOwner` mapping is the sole key to every privileged provider action:

- `grantUpdater` / `revokeUpdater` — controls who may tune `confidenceParam`
- `transferProviderOwnership` itself — the only recovery path
- `setSource` (AnchoredProviderFactory only) — swaps the custom quote source [6](#0-5) [7](#0-6) 

There is no `pendingProviderOwner` mapping, no `acceptProviderOwnership` function, and no cancellation path. Once `providerOwner[provider]` is written to an inoperable address the state is permanent.

---

### Impact Explanation

**Loss of oracle management rights (Medium).**

1. **Updater revocation is permanently blocked.** If a previously-granted updater is later compromised, the owner can no longer call `revokeUpdater`. The compromised updater retains the ability to call `setConfidence` on the factory, which forwards to `PriceProvider.setConfidenceParam`. Setting `confidenceParam` to `CONFIDENCE_MAX` (1 000 000) multiplies the oracle spread by 100×, producing an extremely wide bid/ask that degrades swap quality for every pool backed by this provider. [8](#0-7) 

2. **Source cannot be changed (AnchoredProviderFactory).** The `setSource` path on `AnchoredProviderFactory` is `onlyProviderOwner`. If the source contract becomes stale, malicious, or is upgraded away, the provider owner cannot redirect it. The reference-band clamp in `_computeBidAsk` limits how far a bad source can move prices, but it cannot substitute for the ability to replace the source entirely. [9](#0-8) 

3. **No recovery path exists.** Unlike `Ownable2Step` (where the pending owner has not yet accepted and the transfer can be abandoned), a single-step write to an inoperable address is final. The provider is permanently orphaned.

---

### Likelihood Explanation

The protocol deliberately trains operators to expect two-step ownership transfers: `Ownable2Step` at the factory level and `proposePoolAdminTransfer`/`acceptPoolAdmin` at the pool-admin level. A provider owner who has internalized this norm will not double-check whether `transferProviderOwnership` is also two-step — the perceived safeguard creates false confidence. A single address typo, a multisig that has not yet been deployed on the target chain, or a contract address that cannot call `acceptOwnership` is sufficient to trigger permanent loss of control. This is the exact failure mode described in M-05.

---

### Recommendation

Replace the single-step transfer with a two-step pending/accept pattern in all three factory contracts, consistent with the rest of the protocol:

```solidity
// Storage additions
mapping(address provider => address) public pendingProviderOwner;

function proposeProviderOwnershipTransfer(address provider, address newOwner)
    external onlyProviderOwner(provider)
{
    require(_providers.contains(provider), ProviderNotTracked());
    require(newOwner != address(0));
    pendingProviderOwner[provider] = newOwner;
    emit ProviderOwnershipTransferProposed(provider, providerOwner[provider], newOwner);
}

function acceptProviderOwnership(address provider) external {
    address pending = pendingProviderOwner[provider];
    require(msg.sender == pending, NotPendingProviderOwner());
    address previous = providerOwner[provider];
    providerOwner[provider] = pending;
    _providersByCreator[previous].remove(provider);
    _providersByCreator[pending].add(provider);
    delete pendingProviderOwner[provider];
    emit ProviderOwnershipTransferred(provider, previous, pending);
}
```

Apply the same pattern to `PriceProviderFactory`, `PriceProviderFactoryL2`, and `AnchoredProviderFactory`.

---

### Proof of Concept

1. Alice deploys a `PriceProvider` via `PriceProviderFactory.createPriceProvider`. She becomes `providerOwner[provider]`.
2. Alice calls `transferProviderOwnership(provider, typoAddress)` — a single transaction, no confirmation required.
3. `providerOwner[provider]` is now `typoAddress`, which is an EOA that does not exist or a multisig not yet deployed on this chain.
4. Alice attempts to call `revokeUpdater` or `transferProviderOwnership` again — both revert with `NotProviderOwner()` because `msg.sender != providerOwner[provider]`.
5. A previously-granted updater calls `setConfidence([provider], [1_000_000])` — the factory forwards to `PriceProvider.setConfidenceParam(1_000_000)`, setting `confidenceParam` to `CONFIDENCE_MAX`. Every pool backed by this provider now receives a 100× spread-multiplied bid/ask, permanently degrading swap quality with no remediation path. [10](#0-9) [11](#0-10)

    function _computeBidAsk(uint256 mid, uint256 spreadBps)
        internal view returns (uint128, uint128)
    {
        // Circuit breaker: extreme (combined) uncertainty means the feed is clearly broken.
        if (spreadBps > MAX_SPREAD_BPS) {
            return (0, type(uint128).max);
        }

        // Reference band: mid ± (spreadBps + minMargin), bid rounded down, ask rounded up.
        uint256 half = spreadBps * ONE_BPS_E18 + minMargin; // < BPS_BASE_U by construction (spreadBps <= MAX_SPREAD_BPS here)
        uint256 refBid = _bandEdge(mid, BPS_BASE_U - half, Math.Rounding.Floor);
        uint256 refAsk = _bandEdge(mid, BPS_BASE_U + half, Math.Rounding.Ceil);
        if (refBid == 0 || refAsk > type(uint128).max || refBid >= refAsk) {
            return (0, type(uint128).max);
        }

        // Custom quote: source (both variants) or shaped reference quote (customizable variant).
        //    Immutable reference mode quotes the band directly — zero knob SLOADs.
        address _source = source;
        uint256 cBid;
        uint256 cAsk;
        if (_source != address(0)) {
            // 7a. Source mode: any failure (revert, OOG, garbage, zero, inverted) halts — fail
            //     closed. Knobs do NOT post-process the source output (the source shapes itself).
            bool ok;
            (ok, cBid, cAsk) = _readSource(_source);
            if (!ok) {
                return (0, type(uint128).max);
            }
        } else if (MUTABLE_PARAMS) {
            // 7b. Shaped reference quote: mid ± mid·spreadBps·confidence, then the marginStep step
            //     factors — PriceProvider semantics, clamped into the band below.
            bool ok;
            (ok, cBid, cAsk) = _shapedQuote(mid, spreadBps);
            if (!ok) {
                return (0, type(uint128).max);
            }
        } else {
            return (uint128(refBid), uint128(refAsk));
        }

        // 8. Clamp: out-of-band custom quotes are clipped silently to the band edge.
        //    bid ≤ refBid < refAsk ≤ ask, so bid < ask holds by construction.
        uint256 bidOut = Math.min(refBid, cBid);
        uint256 askOut = Math.max(refAsk, cAsk);
        if (bidOut == 0 || bidOut >= askOut) {
            return (0, type(uint128).max);
        }

        return (uint128(bidOut), uint128(askOut));
