### Title
`CompressedOracleV1` `stateGuard` is irrevocable — creator permanently loses feed-guard authority once a guard is accepted - (File: `smart-contracts-poc/contracts/oracles/compressed/OracleBase.sol`)

### Summary

The `OracleBase` used by `CompressedOracleV1` exposes `acceptStateGuardRole` to install a per-feed `stateGuard`, but — unlike the providers oracle's `OracleBase` — it provides no `purgeStateGuardRole` function. Once a guard is accepted, `stateGuard[feedId]` is permanently non-zero and the creator-derived default authority is unreachable. If the guard address becomes inaccessible, the feed's `priceGuard` is frozen forever, which can either block valid prices (causing `FeedStalled` and an unusable pool) or leave the creator permanently unable to tighten a too-wide price band.

### Finding Description

`CompressedOracleV1` inherits from `smart-contracts-poc/contracts/oracles/compressed/OracleBase.sol`. The `checkRole` modifier resolves authority as follows:

```solidity
modifier checkRole(bytes32 feedId) {
    address guard = stateGuard[feedId];
    if (guard == address(0)) guard = _defaultGuard(feedId);
    require(guard == msg.sender, InvalidGuard(msg.sender));
    _;
}
``` [1](#0-0) 

When `stateGuard[feedId] == address(0)`, `_defaultGuard` returns the creator decoded from the feedId: [2](#0-1) 

Once `acceptStateGuardRole` is called, `stateGuard[feedId]` is set to the new guard and the creator-based default is permanently bypassed: [3](#0-2) 

The compressed oracle's `OracleBase` provides `setPendingStateGuardRole`, `purgePendingStateGuardRole`, and `acceptStateGuardRole`, but **does not** provide `purgeStateGuardRole`. There is no way to set `stateGuard[feedId]` back to `address(0)`.

By contrast, the providers oracle's `OracleBase` does include this escape hatch: [4](#0-3) 

This asymmetry means:
1. Once a `stateGuard` is accepted in `CompressedOracleV1`, the creator permanently loses authority over the feed's `priceGuard` settings.
2. The guard can only transfer to another guard; it can never revert to the creator-based default.
3. If the guard becomes inaccessible (lost key, self-destructed contract, governance failure), the `priceGuard` is permanently frozen.

The `priceGuard` is the per-feed min/max price bound checked by `AnchoredPriceProvider._readLeg()`: [5](#0-4) 

### Impact Explanation

If the `stateGuard` becomes inaccessible after setting a tight `priceGuard`:

- The `priceGuard` cannot be updated or removed.
- When the market price moves outside the frozen bounds, `_readLeg()` returns `ok=false`.
- `getBidAndAskPrice()` reverts with `FeedStalled`.
- The pool is permanently unusable for swaps — a broken core pool functionality.

Alternatively, if the `priceGuard` was set to wide bounds (or never set), the creator is permanently unable to tighten it, allowing any price to pass through to the pool without a circuit-breaker.

### Likelihood Explanation

Requires the creator to have called `setPendingStateGuardRole` and the guard to have called `acceptStateGuardRole` — both deliberate actions. The guard subsequently becoming inaccessible (multisig key loss, DAO governance failure, contract self-destruct) is a realistic operational scenario. The combination is low-to-medium likelihood, but the impact is a permanently bricked pool.

### Recommendation

Add `purgeStateGuardRole` to the compressed oracle's `OracleBase`, mirroring the providers oracle:

```solidity
function purgeStateGuardRole(bytes32 feedId) external checkRole(feedId) {
    delete stateGuard[feedId];
    emit StateGuardDeleted(feedId);
}
```

This restores the creator-based default authority when the guard is cleared, exactly as the providers oracle already does. [6](#0-5) 

### Proof of Concept

1. Creator creates a feed in `CompressedOracleV1` (e.g., `feedIdOf(creator, 5, 1)`).
2. Creator calls `setPendingStateGuardRole(feedId, guardContract)`.
3. `guardContract` calls `acceptStateGuardRole(feedId)` — now `stateGuard[feedId] = guardContract`.
4. `guardContract` calls `setPriceGuard(feedId, 100e8, 200e8)` — tight bounds.
5. `guardContract` self-destructs or its key is lost.
6. Creator calls `setPendingStateGuardRole(feedId, newGuard)` — reverts `InvalidGuard(creator)` because `stateGuard[feedId] != address(0)` and `creator != guardContract`.
7. Market price moves to `250e8` — outside the frozen `priceGuard` bounds.
8. `AnchoredPriceProvider._readLeg()` evaluates `mid > guardMax` → returns `ok=false`.
9. `getBidAndAskPrice()` reverts `FeedStalled`.
10. Pool is permanently unusable for swaps; no path exists to restore it. [7](#0-6) [8](#0-7)

    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
