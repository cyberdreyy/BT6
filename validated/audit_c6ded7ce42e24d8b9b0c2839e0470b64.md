### Title
`priceGuard` bounds are defined and settable in `CompressedOracleV1` but are never enforced during price reads, allowing a compromised pusher to feed unbounded prices into pools — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`, `smart-contracts-poc/contracts/oracles/compressed/OracleBase.sol`)

---

### Summary

The compressed oracle's `OracleBase` inherits a `priceGuard` mapping and exposes `setPriceGuard()` so feed creators can configure absolute `[min, max]` price bounds. However, `CompressedOracle.getOracleData()` — the sole read path for every consumer — never consults `priceGuard`. Any price value packed into a slot by an authorized pusher is returned verbatim. This is the direct analog of the Rescuable bug: a safety override exists in the interface but is never applied, so the privileged actor (pusher) can exceed the boundary that was supposed to protect user assets.

---

### Finding Description

`OracleBase` (compressed variant) declares:

```solidity
mapping(bytes32 => PriceGuard) public priceGuard;
```

and exposes a setter guarded by the feed's creator:

```solidity
function setPriceGuard(bytes32 feedId, uint128 minPrice, uint128 maxPrice)
    external checkRole(feedId)
{
    require(minPrice < maxPrice);
    priceGuard[feedId] = PriceGuard({min: minPrice, max: maxPrice});
    emit PriceGuardUpdated(feedId, minPrice, maxPrice);
}

The entire on-chain read path flows through `getOracleData()`:

```solidity

    data.spread1 = _decodeCodebookIndex(compressed.s1);

``` [3](#0-2) 

`_price()` delegates directly to `getOracleData()` with no post-check:

```solidity
function _price(bytes32 feedId) internal view
    returns (uint256 mid, uint256 spread, uint16 spread1, uint256 refTime)
{
    OracleData memory data = getOracleData(feedId);
    return (uint256(data.price), uint256(data.spread0), data.spread1, data.timestampMs.toSeconds());
}
``` [4](#0-3) 

The public `price(feedId, pool)` entry point — called by every PriceProvider — simply calls `_price()`:

```solidity
function price(bytes32 feedId, address /* pool */)
    external view
    returns (uint256 mid, uint256 spread, uint16 spread1, uint256 refTime)
{
    return _price(feedId);
}
``` [5](#0-4) 

`priceGuard` is therefore **dead state**: it can be written but is never read. The compressed `OracleBase` has no `_readPrice` override (unlike the providers `OracleBase` whose child contracts are documented to apply the guard), so there is no downstream enforcement point either.

The push path — both the `fallback()` direct push and `updateBySignature()` — only enforces monotonic timestamp and `maxTimeDrift`; it does not reject slot values whose decoded price falls outside `priceGuard`:

```solidity
fallback() override external {
    ...
    timestampMs.revertIfAfterBlockTimeWithDrift(MAX_TIME_DRIFT);
    ...
    bool newer = timestampMs.isAfter(oldTimestampMs);
    if (!newer) continue;
    _writeStorage(key, bytes32(bytes32(word & ~uint256(0xff))));
    // no priceGuard check
}
``` [6](#0-5)

### Impact Explanation

A pusher authorized via `allowPushers` or `allowContractPushers` can write any U64x32-representable price (spanning many orders of magnitude) into a slot. That price propagates through `getOracleData → _price → price(feedId, pool)` to the `PriceProvider`, which applies only a staleness check before computing bid/ask in Q64 and returning them to the pool. A price of, say, 1 (instead of 200 000 000 for ETH/USD at 8 decimals) would let an attacker drain the pool by swapping at a rate orders of magnitude below fair value, causing direct loss of LP principal. This satisfies the **bad-price execution** and **pool insolvency** impact gates.

---

### Likelihood Explanation

The trigger is a compromised or rogue pusher — a semi-trusted entity explicitly authorized by the feed creator. This is structurally identical to the Rescuable scenario: the rescuer role is privileged but not fully trusted, and a takeover of that account enables the attack. The `allowPushers` delegation path uses EIP-191 signatures with a deadline; once a pusher is authorized, the creator can re-add them after a `revokePusher()` call using the same unexpired signature, making it harder to contain a compromised key:

```solidity
function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
    _ensureDeadline(deadline);
    ...
    bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
        keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
    );
    require(pusher == ECDSA.recover(hash, signatures[i]));
    namespaceRemapping[pusher] = msg.sender;
``` [7](#0-6) 

A pusher who calls `revokePusher()` to stop an attacker using their compromised key can be immediately re-added by the creator (who may be unaware of the compromise) using the still-valid old signature, keeping the attack window open until the deadline expires.

---

### Recommendation

**Short term:** In `CompressedOracle.getOracleData()` (or in `_price()`), read `priceGuard[feedId]` and revert or return a stalled result when the decoded price falls outside `[min, max]`:

```solidity
PriceGuard memory guard = priceGuard[feedId];
if (guard.max != 0) {
    require(data.price >= guard.min && data.price <= guard.max, PriceOutOfBounds(feedId));
}
```

**Short term (delegation):** Introduce a per-pusher nonce or a revocation bitmap so that `revokePusher()` permanently invalidates all prior signatures for that pusher, preventing the creator from replaying an old consent after revocation.

**Long term:** Apply the same principle as the Rescuable recommendation: design the system so that even if a pusher account is compromised, the damage is bounded. The `priceGuard` was intended to be that bound — enforce it.

---

### Proof of Concept

1. Creator deploys a feed at `feedIdOf(creator, 0, 0)` and sets `setPriceGuard(feedId, 1e8, 3000e8)` (ETH/USD bounds: $1–$3000).
2. Creator calls `allowPushers(deadline, [pusherBot], [sig])` to authorize an automated bot.
3. Attacker compromises `pusherBot`'s private key.
4. `pusherBot` calls `revokePusher()`. Creator, unaware, immediately calls `allowPushers` again with the same unexpired signature, restoring the delegation.
5. Attacker calls the oracle's `fallback()` from `pusherBot`, pushing a slot word encoding price = `U64x32.encode(1)` (i.e., $0.00000001) with a fresh timestamp.
6. The monotonicity check passes (new timestamp > old). The slot is written. `priceGuard` is never consulted.
7. A pool's `swap()` calls `PriceProvider.getBidAndAskPrice()`, which calls `CompressedOracle.price(feedId, pool)`, which returns `mid = 1`.
8. The PriceProvider's staleness check passes (timestamp is fresh). It computes `bid ≈ ask ≈ 1` in Q64 and returns them to the pool.
9. The attacker swaps token B (e.g., USDC) for token A (e.g., WETH) at the manipulated price, receiving WETH at a tiny fraction of its real value, draining LP funds. [8](#0-7) [2](#0-1)

    fallback() override external {
        uint256 end;
        uint256 namespace;

        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0)) creator = msg.sender;

        assembly ("memory-safe") {
            end := calldatasize()
            namespace := shl(96, creator) // [creator:20][zeros:12]
        }

        // 4 * 6 + 7 + 1 = 32 bytes per slot
        if (end == 0 || end % 32 != 0) revert BadCalldataLength();

        for (uint256 ptr = 0; ptr < end; ptr += 32) {
            uint256 word;
            assembly ("memory-safe") {
                word := calldataload(ptr)
            }
            // casting to 'uint8' is safe we want LSB
            // forge-lint: disable-next-line(unsafe-typecast)
            uint8 slotId = uint8(word);
            TimeMs timestampMs = toTimeMs(word >> 8 & X56);
            timestampMs.revertIfAfterBlockTimeWithDrift(MAX_TIME_DRIFT);
            bytes32 key = bytes32(namespace | uint256(slotId));
            uint256 old = uint256(_loadStorage(key));
            TimeMs oldTimestampMs = toTimeMs(old >> 8 & X56);

            bool newer = timestampMs.isAfter(oldTimestampMs);
            if (!newer) continue;

            _writeStorage(key, bytes32(bytes32(word & ~uint256(0xff))));
        }
