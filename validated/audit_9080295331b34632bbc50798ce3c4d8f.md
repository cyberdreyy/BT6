### Title
Single-Step Provider Ownership Transfer Allows Attacker to Permanently Brick Pool Swaps via Malicious Source — (`smart-contracts-poc/contracts/AnchoredProviderFactory.sol`, `smart-contracts-poc/contracts/PriceProviderFactory.sol`)

---

### Summary

`AnchoredProviderFactory.transferProviderOwnership` and `PriceProviderFactory.transferProviderOwnership` both implement a **single-step** ownership transfer: the new owner is written immediately with no pending/accept step. If the current provider owner accidentally supplies a wrong `newOwner` address — including an attacker-controlled address — the legitimate owner permanently loses the sole key to `setSource`. An attacker who receives ownership can immediately point the provider at a malicious source contract, causing every downstream pool swap to revert with `FeedStalled()`. For pools whose price provider is marked **immutable** at creation, this DoS is irreversible at the pool level.

---

### Finding Description

`AnchoredProviderFactory.transferProviderOwnership` writes the new owner in a single atomic step:

```solidity
// AnchoredProviderFactory.sol lines 230-239
function transferProviderOwnership(address provider, address newOwner)
    external override onlyProviderOwner(provider)
{
    require(_providers.contains(provider), ProviderNotTracked());
    require(newOwner != address(0));
    address previousOwner = providerOwner[provider];

    providerOwner[provider] = newOwner;          // ← immediate, no accept step
    _providersByCreator[previousOwner].remove(provider);
    _providersByCreator[newOwner].add(provider);

    emit ProviderOwnershipTransferred(provider, previousOwner, newOwner);
}
```

`PriceProviderFactory.transferProviderOwnership` is byte-for-byte identical in structure (lines 92–101).

The only function gated on `providerOwner` is `setSource` (via `onlyProviderOwner`):

```solidity
// AnchoredProviderFactory.sol lines 224-228
function setSource(address provider, address newSource)
    external override onlyProviderOwner(provider)
{
    require(_providers.contains(provider), ProviderNotTracked());
    AnchoredPriceProvider(provider).setSource(newSource);
```

`setSource` is the **only** way to revert a provider from source mode back to reference mode (zero address). Once ownership is lost, the source is frozen at whatever address it currently holds.

**Attack path (attacker receives ownership by admin typo):**

1. Provider owner calls `transferProviderOwnership(provider, attackerAddress)` by mistake.
2. Attacker immediately calls `setSource(provider, maliciousContract)`.
3. `maliciousContract.getBidAndAskPrice()` always reverts (or returns `retSize != 64`).
4. `AnchoredPriceProvider._readSource` returns `ok = false`.
5. `_computeBidAsk` returns `(0, type(uint128).max)`.
6. `getBidAndAskPrice()` reverts with `FeedStalled()`.
7. Every pool swap that calls this provider halts.

**Irreversibility for immutable-provider pools:**

At pool creation, if `params.priceProviderTimelock == type(uint256).max`, the pool stores `immutablePriceProvider` and `proposePoolPriceProvider` reverts with `PriceProviderImmutable()`. The pool admin has no escape hatch; swaps are permanently bricked.

```solidity
// MetricOmmPoolFactory.sol line 164
bool immutablePriceProvider = params.priceProviderTimelock == type(uint256).max;
```

```solidity
// MetricOmmPoolFactory.sol line 482
if (p.immutablePriceProvider != address(0)) revert PriceProviderImmutable();
```

---

### Impact Explanation

For any pool whose `AnchoredPriceProvider` (or `PriceProvider`) has had its ownership accidentally transferred to an attacker:

- The attacker sets a malicious source → every swap on that pool reverts with `FeedStalled()`.
- For pools with `immutablePriceProvider`, the pool admin cannot replace the provider → **permanent swap DoS**, LP funds are locked (withdrawals still work, but the pool is commercially dead).
- For mutable-provider pools, the pool admin can propose a new provider, but must wait out the timelock (up to 7 days of swap unavailability).

This satisfies: *Broken core pool functionality causing loss of funds or unusable withdraw/swap/liquidity flows.*

---

### Likelihood Explanation

Low. Requires the provider owner to supply a wrong address to `transferProviderOwnership`. This is the same admin-error class as the seeded external report (M-02). The protocol already recognized this risk for the factory owner (using `Ownable2Step` in `MetricOmmPoolFactory`) but did not apply the same pattern to provider ownership in the oracle layer.

---

### Recommendation

Replace the single-step transfer in both `AnchoredProviderFactory` and `PriceProviderFactory` with a two-step pattern:

1. Add `mapping(address provider => address) public pendingProviderOwner`.
2. `transferProviderOwnership` writes only `pendingProviderOwner[provider] = newOwner` and emits a proposal event.
3. Add `acceptProviderOwnership(address provider)` callable only by `pendingProviderOwner[provider]`, which then commits the transfer and clears the pending slot.

This mirrors the `proposePoolAdminTransfer` / `acceptPoolAdmin` pattern already present in `MetricOmmPoolFactory` (lines 510–526) and the `Ownable2Step` pattern used for the factory owner itself.

---

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

// Assume: provider P is owned by Alice; source is currently address(0) (reference mode).
// Alice mistypes and sends ownership to Attacker.

anchoredFactory.transferProviderOwnership(P, attacker); // Alice's typo

// Attacker's turn:
vm.startPrank(attacker);

// Deploy a source that always reverts
MaliciousSource ms = new MaliciousSource();

// Set it — no timelock, instant
anchoredFactory.setSource(P, address(ms));

vm.stopPrank();

// Now any pool using P as an immutable price provider:
// pool.swap(...) → AnchoredPriceProvider.getBidAndAskPrice()
//               → _computeBidAsk → _readSource(ms) → ok=false
//               → returns (0, type(uint128).max)
//               → revert FeedStalled()
// Pool swaps are permanently bricked; pool admin cannot replace the provider.

contract MaliciousSource {
    function getBidAndAskPrice() external pure returns (uint128, uint128) {
        revert("griefed");
    }
}

  function proposePoolAdminTransfer(address pool, address newAdmin) external override nonReentrant onlyPoolAdmin(pool) {
    if (newAdmin == address(0)) revert InvalidAdmin();
    if (newAdmin == poolAdmin[pool]) revert InvalidAdmin();
    pendingPoolAdmin[pool] = newAdmin;
    emit PoolAdminTransferProposed(pool, poolAdmin[pool], newAdmin);
  }

  /// @inheritdoc IMetricOmmPoolFactoryPoolAdmin
  function acceptPoolAdmin(address pool) external override nonReentrant {
    address pending = pendingPoolAdmin[pool];
    if (pending == address(0)) revert NoPendingPoolAdminTransfer();
    if (msg.sender != pending) revert NotPendingPoolAdmin(pool, msg.sender, pending);
    address previousAdmin = poolAdmin[pool];
    poolAdmin[pool] = pending;
    delete pendingPoolAdmin[pool];
    emit PoolAdminTransferred(pool, previousAdmin, pending);
  }
