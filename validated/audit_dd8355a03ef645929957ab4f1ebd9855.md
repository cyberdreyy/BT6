### Title
Creator Can Replay Pusher's Signed Consent to Re-Establish a Revoked Delegation Within the Deadline Window — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`CompressedOracleV1.allowPushers` includes a `deadline` in the EIP-191 signature hash specifically to prevent a creator from re-establishing a pusher's delegation after the pusher has revoked it. However, the deadline only prevents replay *after* it expires — it does not prevent the creator from replaying the same signature to re-establish the delegation *within* the deadline window. A pusher's `revokePusher()` call can be silently undone by the creator at any time before the deadline, causing the pusher's subsequent slot writes to land in the creator's namespace instead of their own.

### Finding Description

The natspec for `allowPushers` explicitly states:

> *"The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it."*

This is the stated security rationale for requiring a deadline. However, the implementation does not include a nonce, revocation counter, or any per-delegation invalidation bit in the signed message. The signature hash is:

```solidity
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [1](#0-0) 

After `revokePusher()` clears `namespaceRemapping[pusher]` to `address(0)`: [2](#0-1) 

…the creator can immediately call `allowPushers` again with the **identical** `(deadline, pushers, signatures)` arguments. Because the signature is still valid (deadline has not expired), `ECDSA.recover` succeeds and `namespaceRemapping[pusher]` is written back to `creator`: [3](#0-2) 

There is no state that records "this pusher has revoked" and no mechanism to invalidate the old signature short of waiting for the deadline to expire.

The `fallback()` push path resolves the namespace at call time:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [4](#0-3) 

So every push the pusher makes after the creator re-establishes the delegation lands in the creator's namespace, not the pusher's own.

### Impact Explanation

**Bad-price execution / stale price reaching a pool swap.**

A pusher who has revoked their delegation and is now pushing prices into their own namespace (e.g., for a pool that reads `feedIdOf(pusher, slot, pos)`) will have those writes silently redirected to the creator's namespace. The pusher's own namespace receives no update and its timestamp remains at the last pre-revocation value. Any pool or price provider reading the pusher's own feed will see a stale price and revert with `FeedStalled` — or, if `MAX_TIME_DELTA` is generous, will consume the last stale quote.

Conversely, if the pusher is unaware the delegation was re-established and continues pushing, the creator's namespace receives price data the pusher did not intend to publish there, potentially feeding a pool with prices from a pusher who believed they had withdrawn consent.

### Likelihood Explanation

**Medium.** The creator must have retained the pusher's original signed consent (trivially true — they submitted it on-chain and it is visible in calldata). The attack requires no special privilege beyond being the namespace owner. The window of vulnerability is the entire remaining lifetime of the deadline, which the natspec encourages to be set to a meaningful future timestamp to avoid "undated signature" problems — making the window potentially days or weeks long.

### Recommendation

Include a per-pusher revocation nonce in the signed message so that a revocation permanently invalidates all prior signatures for that pusher/creator pair:

```solidity
mapping(address => uint256) public pusherNonce; // incremented on revoke

// In allowPushers:
keccak256(abi.encode(
    block.chainid, address(this), deadline,
    pusher, msg.sender, pusherNonce[pusher]   // ← add nonce
))

// In revokePusher:
pusherNonce[msg.sender]++;
namespaceRemapping[msg.sender] = address(0);
```

Alternatively, record a `revokedAt` timestamp per pusher and reject any signature whose deadline predates the revocation.

### Proof of Concept

```
1. creator calls allowPushers(deadline=T, [pusher], [sig])
   → namespaceRemapping[pusher] = creator  ✓

2. pusher calls revokePusher()
   → namespaceRemapping[pusher] = address(0)  ✓

3. creator calls allowPushers(deadline=T, [pusher], [sig])  ← SAME call, block.timestamp < T
   → _ensureDeadline passes (T not expired)
   → ECDSA.recover returns pusher (signature still valid)
   → namespaceRemapping[pusher] = creator  ← revocation silently undone

4. pusher calls oracle.call(slotWord)  ← intending to push into own namespace
   → fallback resolves creator = namespaceRemapping[pusher] = creator
   → slot written to creator's namespace, NOT pusher's own

5. feedIdOf(pusher, slot, pos) returns price=0 / timestamp=stale
   → any pool reading the pusher's own feed gets FeedStalled
``` [5](#0-4) [6](#0-5)

    /// @notice Delegates pusher wallets into the caller's namespace. The pusher's EIP-191
    ///         signature is REQUIRED — without it anyone could remap a foreign pusher
    ///         wallet into their own namespace and silently swallow its pushes. The
    ///         deadline is likewise required: the signed consent carries no timestamp of
    ///         its own, so an undated signature could re-establish a delegation AFTER the
    ///         pusher revoked it.
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }

    fallback() override external {
        uint256 end;
        uint256 namespace;

        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0)) creator = msg.sender;

        assembly ("memory-safe") {
            end := calldatasize()
            namespace := shl(96, creator) // [creator:20][zeros:12]
        }

        // 4 * 6 + 7 + 1 = 32 bytes per slot
        if (end == 0 || end % 32 != 0) revert BadCalldataLength();

        for (uint256 ptr = 0; ptr < end; ptr += 32) {
            uint256 word;
            assembly ("memory-safe") {
                word := calldataload(ptr)
            }
            // casting to 'uint8' is safe we want LSB
            // forge-lint: disable-next-line(unsafe-typecast)
            uint8 slotId = uint8(word);
            TimeMs timestampMs = toTimeMs(word >> 8 & X56);
            timestampMs.revertIfAfterBlockTimeWithDrift(MAX_TIME_DRIFT);
            bytes32 key = bytes32(namespace | uint256(slotId));
            uint256 old = uint256(_loadStorage(key));
            TimeMs oldTimestampMs = toTimeMs(old >> 8 & X56);

            bool newer = timestampMs.isAfter(oldTimestampMs);
            if (!newer) continue;

            _writeStorage(key, bytes32(bytes32(word & ~uint256(0xff))));
        }
