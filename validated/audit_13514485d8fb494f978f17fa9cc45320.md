### Title
`updateBySignature` omits a deadline from the signed message, allowing withheld signed price updates to be submitted at attacker-chosen timing to feed stale prices into pool swaps — (File: smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol)

---

### Summary

`CompressedOracle.updateBySignature` is callable by **anyone** and its EIP-191 signed message contains no deadline. A signed slot word is valid indefinitely until a strictly-newer timestamp is stored on-chain. An attacker who obtains a signed price update — e.g., from a public keeper network — can withhold it and submit it at the optimal moment (just before a large swap) to force the pool to execute at a stale, attacker-chosen price.

---

### Finding Description

`updateBySignature` writes a creator-signed slot word to storage if its embedded timestamp is strictly newer than the stored one: [1](#0-0) 

The signed digest is:

```solidity
keccak256(abi.encode(block.chainid, address(this), feedCreator, newSlotValue))
``` [2](#0-1) 

There is **no deadline field**. The in-code justification is:

> "The signed slot word carries its own 56-bit timestamp, so replay is neutralized by the monotonicity check below — no deadline is needed." [3](#0-2) 

This reasoning prevents **replay** of a single old message, but it does **not** prevent **selective ordering** of multiple valid signed messages. Because `updateBySignature` is permissionless (any `msg.sender`), anyone who holds a signed slot word can submit it at any future block, as long as its timestamp is newer than whatever is currently stored.

The contract itself acknowledges the deadline is load-bearing in the analogous `allowPushers` path:

> "The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it." [4](#0-3) 

The same timing-manipulation risk applies to price updates: a signed slot word with no deadline can be submitted at any moment the attacker finds profitable.

**Concrete attack path:**

1. Creator signs slot word **W1** (timestamp `T1`, price `P1 = $1 000`) and **W2** (timestamp `T2 > T1`, price `P2 = $2 000`). Both are broadcast to a public keeper network.
2. Attacker obtains both signed messages. Current on-chain stored timestamp is `T0 < T1`.
3. Attacker observes a large pending swap that benefits from the `$1 000` price (e.g., buying the base token cheaply).
4. Attacker front-runs the keeper: calls `updateBySignature(creator, W1_slotValue, W1_sig)`. Monotonicity check passes (`T1 > T0`). Price is written as `$1 000`.
5. Swap executes at the stale `$1 000` price; pool pays out excess base tokens.
6. Attacker then submits W2 (`T2 > T1`), restoring the correct price.
7. Attacker profits from the delta; LPs bear the loss.

The `fallback()` push path is **not** affected because it requires `msg.sender` to be the creator or a delegated pusher — only `updateBySignature` is permissionless. [5](#0-4) 

---

### Impact Explanation

`AnchoredPriceProvider._readLeg` consumes the oracle price and applies a staleness check bounded by `MAX_REF_STALENESS`: [6](#0-5) 

W1's timestamp `T1` is recent (it was signed recently and passes the future-drift check), so it clears the staleness gate. The pool then executes a swap at the stale `$1 000` price. LPs suffer a **direct loss of principal** equal to the price delta times the swap size — a swap conservation failure where the pool pays out more base tokens than the current oracle price permits.

---

### Likelihood Explanation

- `updateBySignature` is explicitly designed for keeper/relayer networks where the creator pre-signs slot words and third parties submit them.
- Any participant in such a network (or anyone who monitors the mempool for pending `updateBySignature` calls) can obtain valid signed messages.
- The attacker needs only two signed messages with consecutive timestamps — a normal occurrence whenever the price moves between two keeper cycles.
- No privileged access, no malicious setup, no non-standard tokens required.

---

### Recommendation

Add a `deadline` parameter to the signed digest and enforce it on-chain, mirroring the `allowPushers` pattern:

```solidity
// In updateBySignature:
function updateBySignature(
    address feedCreator,
    uint256 deadline,          // NEW
    uint256 newSlotValue,
    bytes calldata signature
) external override returns (bool) {
    _ensureDeadline(deadline); // NEW — revert if block.timestamp > deadline

    // ... existing monotonicity check ...

    bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
        keccak256(abi.encode(
            block.chainid,
            address(this),
            feedCreator,
            deadline,          // NEW — bind deadline into the signed message
            newSlotValue
        ))
    );
    require(feedCreator == ECDSA.recover(hash, signature));
    // ...
}
```

This is consistent with `allowPushers`, which already requires a deadline for the same reason. A short deadline (e.g., 60–120 seconds) eliminates the attacker's ability to withhold and selectively time the submission.

### Proof of Concept

```solidity
// Attacker holds two signed messages obtained from the keeper network:
//   W1: slotValue encodes (timestamp=T1, price=$1_000)  — older
//   W2: slotValue encodes (timestamp=T2, price=$2_000)  — newer, T2 > T1
// On-chain stored timestamp is T0 < T1.

// Step 1 — attacker submits the OLDER signed price (stale $1 000):
oracle.updateBySignature(creator, W1_slotValue, W1_sig);
// Passes: T1 > T0 (monotonicity), T1 <= block.timestamp + drift (future check).

// Step 2 — attacker (or victim) executes a large swap; pool prices at $1 000:
pool.swap(/* buy base token */);
// Pool pays out 2× the base tokens it should at the correct $2 000 price.

// Step 3 — attacker submits the NEWER signed price to restore state:
oracle.updateBySignature(creator, W2_slotValue, W2_sig);
// Passes: T2 > T1.

// Net result: attacker extracted the $1 000 vs $2 000 delta from LP reserves.
```

    /// @notice Single-slot update authorized by the creator's signature. The signed slot
    ///         word carries its own 56-bit timestamp, so replay is neutralized by the
    ///         monotonicity check below — no deadline is needed.
    function updateBySignature(address feedCreator, uint256 newSlotValue, bytes calldata signature)
        external
        override
        returns (bool)
    {
        require(feedCreator != address(0), InvalidNamespace());

        uint256 namespace;
        assembly ("memory-safe") {
            namespace := shl(96, feedCreator) // [creator:20][zeros:12]
        }

        uint8 slotId = uint8(newSlotValue); // LSB
        TimeMs timestampMs = toTimeMs(newSlotValue >> 8 & X56);
        timestampMs.revertIfAfterBlockTimeWithDrift(MAX_TIME_DRIFT);
        bytes32 key = bytes32(namespace | uint256(slotId));
        uint256 old = uint256(_loadStorage(key));
        TimeMs oldTimestampMs = toTimeMs(old >> 8 & X56);

        bool newer = timestampMs.isAfter(oldTimestampMs);
        if (!newer) {
            return false;
        }

        bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
            keccak256(abi.encode(block.chainid, address(this), feedCreator, newSlotValue))
        );
        require(feedCreator == ECDSA.recover(hash, signature));

        _writeStorage(key, bytes32(newSlotValue & ~uint256(0xff)));

        return true;
    }

    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
