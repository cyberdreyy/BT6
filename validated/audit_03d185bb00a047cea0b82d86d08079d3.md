### Title
No Upper Bound on `allowPushers` Deadline Allows Creator to Permanently Defeat Pusher Revocation, Enabling Compromised Key to Feed Bad Prices — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`allowPushers` accepts an arbitrary `deadline` with no upper-bound check. A pusher who signed consent with a far-future deadline cannot effectively revoke their delegation: the creator can immediately re-establish it using the same still-valid signature. If the pusher's key is later compromised, the attacker retains write authority over the creator's namespace and can push fresh bad prices into pools.

### Finding Description

`allowPushers` signs and stores a namespace delegation. The signed message binds `(chainid, oracle, deadline, pusher, creator)`: [1](#0-0) 

The only deadline validation is `_ensureDeadline(deadline)`, which checks `block.timestamp <= deadline` (confirmed by the `DeadlineExceeded` revert in tests). There is **no upper-bound check** — a deadline of `type(uint256).max` is accepted.

The protocol's own NatSpec explicitly states the deadline is the sole protection against re-establishing delegation after revocation: [2](#0-1) 

A pusher who signed with a far-future deadline and later calls `revokePusher()`: [3](#0-2) 

…clears `namespaceRemapping[pusher] = address(0)`. But the creator can immediately call `allowPushers` again with the **identical signature** (same `deadline`, still in the future), writing `namespaceRemapping[pusher] = creator` again. The revocation is silently undone.

### Impact Explanation

Once delegation is re-established, any entity controlling the pusher's private key can call the `fallback` push path and overwrite the creator's namespace slots with arbitrary price data: [4](#0-3) 

The `CompressedOracle` is open and permissionless — `price()` has no pool-attribution gate: [5](#0-4) 

Price providers (`PriceProvider`, `PriceProviderL2`, `ProtectedPriceProvider`) consume this data and apply only a staleness check (`MAX_TIME_DELTA`) and a price guard (optional, off by default). A compromised pusher can push **fresh** bad prices (valid timestamp, within drift), bypassing the staleness gate: [6](#0-5) 

These bad prices propagate to `getBidAndAskPrice()` and into live pool swaps, causing traders to receive more than the oracle curve permits or the pool to receive less than owed — a direct swap conservation failure.

### Likelihood Explanation

The trigger requires: (1) a pusher who signed with a far-future deadline, (2) a compromised pusher key, and (3) a creator who re-establishes delegation (possibly unaware of the compromise). The protocol provides no warning that a long deadline defeats revocation, and no UI/factory enforces a short deadline. The `allowPushers` path is fully permissionless and publicly callable. Likelihood is **Low** but the impact is **High**, placing this at **Medium** severity.

### Recommendation

Add an upper bound on the `deadline` parameter in `allowPushers`, analogous to the `MAX_TIME_DELTA` and `MAX_REF_STALENESS` caps elsewhere in the codebase:

```solidity
uint256 public constant MAX_DELEGATION_DEADLINE = 30 days;

function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
    _ensureDeadline(deadline);
    require(deadline <= block.timestamp + MAX_DELEGATION_DEADLINE, DeadlineExceeded());
    // ...
}
```

This mirrors the `MAX_REF_STALENESS` cap in `AnchoredPriceProvider`: [7](#0-6) 

A bounded deadline ensures that a pusher's `revokePusher()` call becomes permanently effective once the original signature expires, restoring the revocation guarantee the protocol's NatSpec promises.

### Proof of Concept

```solidity
// 1. Pusher signs consent with deadline = type(uint256).max
uint256 deadline = type(uint256).max;
bytes memory sig = pusher.sign(keccak256(abi.encode(
    block.chainid, address(oracle), deadline, pusherAddr, creatorAddr
)));

// 2. Creator establishes delegation
vm.prank(creator);
oracle.allowPushers(deadline, [pusherAddr], [sig]);
// namespaceRemapping[pusher] == creator ✓

// 3. Pusher's key is compromised; pusher revokes
vm.prank(pusher);
oracle.revokePusher();
// namespaceRemapping[pusher] == address(0) ✓

// 4. Creator re-establishes with the SAME signature (deadline still valid)
vm.prank(creator);
oracle.allowPushers(deadline, [pusherAddr], [sig]); // succeeds — no revert
// namespaceRemapping[pusher] == creator again ✓

// 5. Attacker (holding compromised key) pushes manipulated price into creator's namespace
uint56 tsMs = uint56(block.timestamp * 1000);
uint48 badPrice = _packRaw(1, 0, 0); // near-zero price
vm.prank(attacker); // attacker == pusher address
(bool ok,) = address(oracle).call(_wordAt(slotId, pos, badPrice, tsMs));
assertTrue(ok); // bad price accepted into creator's namespace

// 6. Price provider reads the bad price and feeds it to the pool swap
```
