### Title
Pusher Delegation Signature Replay Nullifies `revokePusher()` — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`allowPushers` accepts an EIP-191 consent signature from a pusher but never marks it as used. A creator can replay the same signature — before its deadline expires — to silently re-establish delegation immediately after the pusher calls `revokePusher()`. The revocation is therefore ineffective for the entire remaining deadline window, and the pusher's subsequent price pushes continue to land in the creator's namespace without fresh consent.

### Finding Description

`allowPushers` signs over `(chainid, address(this), deadline, pusher, creator)` and writes `namespaceRemapping[pusher] = msg.sender`: [1](#0-0) 

`revokePusher` clears the mapping: [2](#0-1) 

There is no nonce, no `usedSignatures` mapping, and no mechanism to invalidate the signature upon revocation. The code's own comment acknowledges the risk — "an undated signature could re-establish a delegation AFTER the pusher revoked it" — and claims the deadline is the fix: [3](#0-2) 

But the deadline only prevents using an *expired* signature. Within its validity window (which the creator controls and can set to months or years), the creator can call `allowPushers` again with the identical `(deadline, pusher, sig)` tuple to restore `namespaceRemapping[pusher] = creator` immediately after every revocation.

The fallback push path resolves the namespace at call time: [4](#0-3) 

So once the creator replays the signature, every subsequent push from the pusher's address lands in the creator's namespace again, not the pusher's own.

### Impact Explanation

A pusher who revokes (e.g., because they have discovered the creator is malicious, or because they intend to push prices for a different asset) believes their pushes now go to their own namespace. In reality, the creator has already replayed the signature and the pushes still land in the creator's namespace. If the pusher begins pushing prices for a different asset or pair to the same `(slotIndex, positionIndex)`, those prices overwrite the creator's feed slot. Any pool whose `AnchoredPriceProvider` or `PriceProvider` is bound to that creator feed will consume the wrong prices at swap time, satisfying the "bad-price execution" impact gate. [5](#0-4) 

### Likelihood Explanation

Medium. The creator sets the deadline and can make it arbitrarily long (up to the `uint256` maximum). The pusher has no on-chain way to detect that their revocation was replayed. The creator's replay call is a single public transaction with no special privilege beyond possessing the already-broadcast signature.

### Recommendation

Track each consent signature as a one-time token. Add a `mapping(bytes32 => bool) public usedDelegationSignatures` and, inside `allowPushers`, compute the hash and `require(!usedDelegationSignatures[hash])` before writing `namespaceRemapping`, then set `usedDelegationSignatures[hash] = true`. This ensures each pusher consent is consumed exactly once and cannot be replayed after revocation.

### Proof of Concept

```
1. Creator calls allowPushers(deadline = block.timestamp + 365 days, [pusher], [sig])
   → namespaceRemapping[pusher] = creator

2. Pusher calls revokePusher()
   → namespaceRemapping[pusher] = address(0)
   → Pusher believes they now push to their own namespace

3. Creator calls allowPushers(deadline, [pusher], [sig])  // SAME sig, SAME deadline
   → require(pusher == ECDSA.recover(hash, sig)) passes (signature is still valid)
   → namespaceRemapping[pusher] = creator  // delegation silently restored

4. Pusher pushes prices for asset Y to (slotId=0, pos=0)
   → fallback resolves creator = namespaceRemapping[pusher] = creator
   → prices land in creator's namespace at slot 0 / position 0
   → creator's pool reads asset-Y prices from an asset-X feed → bad-price execution
```

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }

    function revokePusher() external {
        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
        namespaceRemapping[msg.sender] = address(0);
        emit PusherRevoked(msg.sender, creator);
    }
