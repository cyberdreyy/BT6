### Title
`allowPushers` Signature Replay Within Deadline Window Allows Creator to Re-establish a Revoked Pusher Delegation, Enabling Bad-Price Injection into Oracle Feeds — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1.allowPushers()` contains no nonce, no used-signature tracking, and no on-chain record that a consent signature was already consumed. The only replay guard is `_ensureDeadline(deadline)`, which only blocks replay **after** the deadline expires. Within the deadline window the identical signature is unconditionally re-accepted, so a creator can replay a pusher's original consent to silently undo a `revokePusher()` call. The protocol's own inline comment acknowledges this exact risk but the deadline-only mitigation does not close it.

---

### Finding Description

When a pusher calls `revokePusher()`, `namespaceRemapping[pusher]` is set to `address(0)`: [1](#0-0) 

The creator can immediately replay the original `allowPushers` call with the **same** `deadline` and `signature` values (both still on-chain in calldata history / mempool): [2](#0-1) 

The signed digest is:

```
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
```

It contains no nonce and no "already-used" flag. `_ensureDeadline` only checks `block.timestamp <= deadline`; it does not record that the signature was consumed. Every call that arrives before `deadline` passes the check identically, so `namespaceRemapping[pusher] = msg.sender` is written again, restoring the revoked delegation.

The protocol's own comment on `allowPushers` explicitly names this risk:

> *"the deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it."* [3](#0-2) 

The stated intent is that the deadline prevents re-establishment after revocation. It does not: it only prevents replay after the deadline itself expires. Within the window the revocation is fully reversible by the creator.

Once `namespaceRemapping[pusher]` is restored, the `fallback()` push path routes the pusher's calldata back into the creator's namespace: [4](#0-3) 

The only freshness gate on the push path is the per-slot monotonicity check (`timestampMs.isAfter(oldTimestampMs)`). An attacker holding a compromised pusher key can push any price that is newer than the last stored value — no additional authorization is required.

---

### Impact Explanation

`CompressedOracleV1` is designed to serve as the `offchainOracle` for `AnchoredPriceProvider` (deployed via `AnchoredProviderFactory`): [5](#0-4) 

`AnchoredPriceProvider._readLeg` calls `IPricedOracle(address(offchainOracle)).price(feedId, msg.sender)` and uses the returned `mid` as the anchor for the bid/ask band: [6](#0-5) 

If the attacker pushes a manipulated `mid` that passes the staleness, spread, and `priceGuard` checks, the entire band is shifted around that manipulated anchor. The pool's swap then executes at a bad price: traders receive more than the true oracle/bin curve permits, or the pool receives less input than owed.

---

### Likelihood Explanation

**Medium.** The attack requires two independent conditions:

1. A pusher's private key is compromised (realistic for hot-wallet pushers).
2. The creator replays `allowPushers` with the original signature — either unknowingly (e.g., an automated re-delegation script) or because the creator is unaware of the compromise.

Deadlines are typically set days or weeks in the future (the signed consent is meant to be long-lived to avoid repeated signing ceremonies), so the replay window is wide. The creator retains the original calldata from the first `allowPushers` transaction and can replay it with zero additional cost.

---

### Recommendation

Track consumed signatures on-chain so each pusher-consent can only be accepted once:

```solidity
mapping(bytes32 => bool) public usedDelegationSignatures;

function allowPushers(...) external {
    _ensureDeadline(deadline);
    for (...) {
        bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
            keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
        );
        require(!usedDelegationSignatures[hash], SignatureAlreadyUsed());
        require(pusher == ECDSA.recover(hash, signatures[i]));
        usedDelegationSignatures[hash] = true;
        namespaceRemapping[pusher] = msg.sender;
        emit PusherAuthorized(pusher, msg.sender);
    }
}
```

Alternatively, include a per-pusher nonce in the signed digest and increment it on every successful `allowPushers` or `revokePusher` call, so a revoked pusher's old signature is permanently invalidated.

---

### Proof of Concept

1. Pusher P signs consent for creator A with `deadline = block.timestamp + 30 days`.
2. Creator A calls `allowPushers([P], [sig])` → `namespaceRemapping[P] = creatorA`.
3. Pusher P's key is compromised; P calls `revokePusher()` → `namespaceRemapping[P] = address(0)`.
4. Creator A (or an automated script) replays the **identical** `allowPushers([P], [sig])` call. `_ensureDeadline` passes (deadline is 30 days away). No nonce check exists. `namespaceRemapping[P] = creatorA` is written again.
5. Attacker uses the compromised key to call `CompressedOracleV1.fallback()` with a crafted slot word containing a fresh `timestampMs` and a manipulated price. The monotonicity check passes (new timestamp > stored timestamp). The price is written to creator A's namespace.
6. `AnchoredPriceProvider._readLeg` reads the manipulated `mid` from the oracle. Staleness, spread, and `priceGuard` checks pass (attacker chose values within bounds). The band is anchored to the manipulated mid.
7. The pool swap executes at the manipulated bid/ask — bad-price execution confirmed.

    ///         signature is REQUIRED — without it anyone could remap a foreign pusher
    ///         wallet into their own namespace and silently swallow its pushes. The
    ///         deadline is likewise required: the signed consent carries no timestamp of
    ///         its own, so an undated signature could re-establish a delegation AFTER the
    ///         pusher revoked it.
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }

        uint256 end;
        uint256 namespace;

        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0)) creator = msg.sender;

        assembly ("memory-safe") {
            end := calldatasize()
            namespace := shl(96, creator) // [creator:20][zeros:12]

        // 4 * 6 + 7 + 1 = 32 bytes per slot

    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
