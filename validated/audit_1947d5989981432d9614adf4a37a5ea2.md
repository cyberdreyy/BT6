### Title
Delegated Pusher Can Push Arbitrary Prices Into Creator Namespace With No Per-Update Creator Authorization — (File: `smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

The `fallback()` function in `CompressedOracleV1` is the primary high-frequency push path. It resolves the effective namespace from `namespaceRemapping[msg.sender]` and writes the caller-supplied 32-byte slot word directly to EVM storage, subject only to a timestamp monotonicity check. There is **no per-update signature from the creator (namespace owner)**. Once a pusher is delegated via `allowPushers` or `allowContractPushers`, they can push any price, spread, or stalled-marker value into the creator's namespace indefinitely, until explicitly revoked. A single compromised pusher key is sufficient to feed manipulated prices into every pool that consumes that creator's feeds.

---

### Finding Description

**Root cause — `fallback()` has no content or authorization guard:** [1](#0-0) 

The `fallback()` loop:
1. Resolves `creator = namespaceRemapping[msg.sender]` (or `msg.sender` if not delegated).
2. Checks only that the incoming `timestampMs` is strictly greater than the stored one (`isAfter`).
3. Writes the raw caller-supplied word to `creator`'s storage slot.

There is **no signature from the creator** and **no validation of the encoded price or spread indices**. The only alternative path that requires a creator signature is `updateBySignature`: [2](#0-1) 

`updateBySignature` verifies `feedCreator == ECDSA.recover(hash, signature)` before writing, but `fallback()` — the primary gas-efficient path — does not.

**Delegation path — once granted, consent is open-ended:** [3](#0-2) 

`allowPushers` maps `namespaceRemapping[pusher] = msg.sender`. After this, the pusher can call `fallback()` with any slot word at any time. The pusher's EIP-191 signature proves only that the pusher *consented to be delegated*; it does not authorize any specific price update.

**No price-range guard in the compressed oracle:**

Unlike the providers' `OracleBase`, which exposes a `priceGuard` mapping with `minPrice`/`maxPrice` bounds: [4](#0-3) 

`CompressedOracleV1` has no equivalent. A pusher can write `p = 0` (U64x32 zero → decoded price 0) or `p = 0xFFFFFFFF` (maximum pseudo-float) with no on-chain rejection.

**Stalled-marker abuse:** [5](#0-4) 

A pusher can set `s0 = 0xFF, s1 = 0xFF` in any lane, causing `getOracleData` to return `price = 0` and `spread = BPS_BASE (10 000 bps = 100%)`. Every consumer that checks for a stalled feed will reject the price, freezing swaps for all pools on that feed.

---

### Impact Explanation

- **Bad-price execution:** A compromised pusher pushes `p ≈ 0` with a fresh timestamp. `PriceProvider.getBidAndAskPrice()` reads `CompressedOracleV1.getOracleData()`, receives a near-zero mid-price, and computes a near-zero bid/ask. The pool executes swaps at that price, allowing the attacker (or any trader) to drain the pool's token0 reserves for near-zero token1 input.
- **Pool insolvency:** LP claims cannot be covered after the pool's reserves are drained at a manipulated price.
- **Swap DoS:** Pushing the stalled marker freezes all swaps on the affected feed until the creator pushes a valid update.

Pools that use `AnchoredPriceProvider` receive partial protection (price is clamped to the anchor oracle's uncertainty band). Pools that use `PriceProvider` or `ProtectedPriceProvider` directly with `CompressedOracleV1` as the backend receive **no clamping** and are fully exposed.

---

### Likelihood Explanation

Delegated pushers are automated bots whose private keys are stored on internet-connected servers — exactly the class of key that bridge-protocol attackers have repeatedly compromised (as noted in the external report). A single compromised pusher key is sufficient; no admin or creator cooperation is required after delegation. The attacker only needs to submit a `fallback()` call with a timestamp one millisecond newer than the current stored value.

---

### Recommendation

1. **Require per-update creator authorization in `fallback()`** — either a creator co-signature on each slot word, or a threshold of independent pushers that must agree before a slot is written (analogous to the external report's "both maker and taker must sign" recommendation).
2. **Add a price-range guard to `CompressedOracleV1`** — mirror the `priceGuard` mechanism from `OracleBase` so that pushes outside `[minPrice, maxPrice]` are rejected on-chain.
3. **Enforce `AnchoredPriceProvider` for all public pools** — make the factory reject pool deployments that use `CompressedOracleV1` without an anchor oracle clamping the output.

---

### Proof of Concept

```
1. Creator C deploys a pool backed by CompressedOracleV1 using PriceProvider (no anchor).
2. C calls allowPushers(deadline, [pusherP], [pusherP_sig]) → namespaceRemapping[pusherP] = C.
3. Attacker steals pusherP's private key.
4. Current stored slot for (C, slotIndex=0): timestampMs = T, price = 50_000_00000000 (BTC ~$50k).
5. Attacker calls CompressedOracleV1.fallback() with calldata =
       [lane0: p=1, s0=0, s1=0][lane1..3: zeros][timestampMs = T+1][slotId = 0]
   (32 bytes, timestamp T+1 > T → monotonicity passes, no signature required).
6. Storage slot for (C, 0) is overwritten: price = U64x32.decode(1) = 1 (1 unit of 8-decimal price).
7. Pool calls PriceProvider.getBidAndAskPrice() → getOracleData(feedId) → price = 1.
8. Staleness check passes (timestampMs = T+1, fresh).
9. Pool quotes bid ≈ 1, ask ≈ 1 for BTC/DAI.
10. Attacker swaps 1 wei DAI → receives 1 BTC from pool at price = 1 wei/BTC.
    Pool is drained; LPs suffer total loss of token0 reserves.
``` [6](#0-5)

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }

    fallback() override external {
        uint256 end;
        uint256 namespace;

        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0)) creator = msg.sender;

        assembly ("memory-safe") {
            end := calldatasize()
            namespace := shl(96, creator) // [creator:20][zeros:12]
        }

        // 4 * 6 + 7 + 1 = 32 bytes per slot
        if (end == 0 || end % 32 != 0) revert BadCalldataLength();

        for (uint256 ptr = 0; ptr < end; ptr += 32) {
            uint256 word;
            assembly ("memory-safe") {
                word := calldataload(ptr)
            }
            // casting to 'uint8' is safe we want LSB
            // forge-lint: disable-next-line(unsafe-typecast)
            uint8 slotId = uint8(word);
            TimeMs timestampMs = toTimeMs(word >> 8 & X56);
            timestampMs.revertIfAfterBlockTimeWithDrift(MAX_TIME_DRIFT);
            bytes32 key = bytes32(namespace | uint256(slotId));
            uint256 old = uint256(_loadStorage(key));
            TimeMs oldTimestampMs = toTimeMs(old >> 8 & X56);

            bool newer = timestampMs.isAfter(oldTimestampMs);
            if (!newer) continue;

            _writeStorage(key, bytes32(bytes32(word & ~uint256(0xff))));
        }
