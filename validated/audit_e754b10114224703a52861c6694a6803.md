### Title
Pusher delegation consent signature lacks a nonce, allowing a creator to replay a revoked consent and permanently nullify pusher self-revocation within the deadline window — (`File: smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowPushers` signs pusher consent with a domain that does **not** include a nonce. After a pusher calls `revokePusher()` to clear `namespaceRemapping[pusher]`, the creator can immediately replay the original signature to re-establish the delegation. This cycle can repeat indefinitely until the deadline expires, making the pusher's self-revocation permanently ineffective within the deadline window and allowing bad prices to continue reaching pools that consume the creator's feeds.

---

### Finding Description

The `allowPushers` function constructs the signed digest as:

```solidity
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
);
require(pusher == ECDSA.recover(hash, signatures[i]));
namespaceRemapping[pusher] = msg.sender;
``` [1](#0-0) 

The signed message binds `chainid`, `address(this)`, `deadline`, `pusher`, and `creator` (`msg.sender`). It does **not** bind a per-pusher nonce or any monotonic counter. As a result, the same `(deadline, pusher, creator)` tuple produces the same valid signature for every call to `allowPushers` until `block.timestamp > deadline`.

The code comment explicitly acknowledges the risk:

> "The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it." [2](#0-1) 

However, the deadline only bounds the replay window — it does **not** prevent replay within that window. After `revokePusher()` clears `namespaceRemapping[pusher] = address(0)`: [3](#0-2) 

…the creator can immediately call `allowPushers` again with the identical signature, restoring `namespaceRemapping[pusher] = creator`. The pusher's revocation is silently undone. This is structurally identical to the external report's "duplicate index" class: the same credential (signature) can be consumed multiple times, making one instance of the intended state (revocation) permanently unreachable until an external timer expires.

The `fallback` push path resolves the namespace at call time:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [4](#0-3) 

So every push after the creator's replay lands in the creator's namespace, not the pusher's own namespace, regardless of how many times the pusher revokes.

---

### Impact Explanation

If a pusher's private key is compromised, the pusher calls `revokePusher()` as an emergency stop. The creator — who may be running an automated re-delegation bot, or who may be the attacker — replays the original signature in the same block or the next block, restoring the delegation. The compromised pusher key continues to push arbitrary prices into the creator's namespace. Any pool registered against a feed in that namespace will execute swaps at those bad prices, causing direct loss of trader principal and LP assets. The `AnchoredPriceProvider` staleness and spread guards are the only downstream protection, but they cannot reject a freshly-timestamped, in-range price pushed by the compromised key. [5](#0-4) 

---

### Likelihood Explanation

The trigger is semi-trusted: the creator must replay the signature. This can happen automatically if the creator runs a delegation-maintenance bot (a common operational pattern for oracle infrastructure), or deliberately if the creator is adversarial. The pusher must also have a compromised key. The combination is low-probability but the window can be arbitrarily long if the original consent was signed with a far-future deadline (e.g., `type(uint256).max`), making the pusher permanently unable to revoke.

---

### Recommendation

Add a per-pusher nonce to the signature domain and increment it on every successful `allowPushers` call:

```solidity
mapping(address => uint256) public pusherNonce;

// inside allowPushers loop:
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(
        block.chainid, address(this), deadline,
        pusher, msg.sender, pusherNonce[pusher]   // <-- nonce added
    ))
);
require(pusher == ECDSA.recover(hash, signatures[i]));
namespaceRemapping[pusher] = msg.sender;
pusherNonce[pusher]++;                            // <-- invalidate after use
```

This ensures each consent signature is single-use: once consumed (or once the pusher revokes and the nonce advances), the old signature is permanently invalid and cannot re-establish the delegation.

### Proof of Concept

```
T=0  Pusher P signs: keccak256(chainid, oracle, deadline=T+365d, P, creatorA) → sig
T=1  creatorA calls allowPushers(T+365d, [P], [sig])
       → namespaceRemapping[P] = creatorA  ✓

T=2  P's key is compromised; attacker pushes bad price to creatorA's slot 0.

T=3  P calls revokePusher()
       → namespaceRemapping[P] = address(0)  ✓ (revocation succeeds)

T=4  creatorA (or attacker) calls allowPushers(T+365d, [P], [sig])  ← SAME sig, still valid
       → namespaceRemapping[P] = creatorA  ✗ (revocation undone)

T=5  Attacker pushes bad price again; pool swap executes at bad price → trader loss.

T=6  P calls revokePusher() again → undone again at T=7 by replay.
     ... cycle continues until T+365d.
```

The `namespaceRemapping` write at line 209 is the corrupted state: it is set back to `creatorA` after the pusher legitimately cleared it, because the signature domain contains no monotonic counter that would distinguish the first `allowPushers` call from the replay. [6](#0-5)

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
