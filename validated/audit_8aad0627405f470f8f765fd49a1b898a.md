### Title
Metamorphic Source Contract Bypasses AnchoredPriceProvider Band Clamp, Enabling Unbounded Bid/Ask Execution — (`smart-contracts-poc/contracts/AnchoredPriceProvider.sol`)

---

### Summary

`AnchoredPriceProvider` stores a mutable `source` address that is called at swap-time to supply custom bid/ask quotes. The band clamp in `_computeBidAsk` only prevents the source from quoting *tighter* than the reference band; it does not prevent the source from quoting *wider*. If the source contract is metamorphic (self-destructs and is redeployed with different bytecode at the same address), the stored `source` pointer remains valid but now executes malicious logic that returns arbitrarily extreme bid/ask values. These extreme values pass all `_readSource` validity checks and flow through the clamp unconstrained, delivering unbounded prices to the pool.

---

### Finding Description

`AnchoredPriceProvider` holds a mutable `source` address in storage:

```solidity
address public source;
```

The factory sets this once via `setSource`. At swap-time, `_computeBidAsk` calls `_readSource(source)` and then applies the band clamp:

```solidity
uint256 bidOut = Math.min(refBid, cBid);   // source can only lower bid
uint256 askOut = Math.max(refAsk, cAsk);   // source can only raise ask
```

The clamp is **one-directional**: it prevents the source from quoting *inside* the reference band (tighter), but it does not cap how far *outside* the band the source can push the quote. A source returning `cBid = 1` and `cAsk = type(uint128).max - 1` passes every check in `_readSource`:

```solidity
if (srcBid == 0 || srcBid >= srcAsk || srcAsk > type(uint128).max) return (false, 0, 0);
// 1 != 0, 1 < (type(uint128).max-1), (type(uint128).max-1) <= type(uint128).max → all pass
```

After the clamp, `bidOut = 1` and `askOut = type(uint128).max - 1`, and the final guard `bidOut >= askOut` also passes. The pool receives a completely broken price.

**Metamorphic path:** The factory legitimately sets `source = X` where X is a well-behaved curator contract. The curator later self-destructs X and redeploys a malicious contract to the same address via CREATE2 (metamorphic pattern). `AnchoredPriceProvider.source` still holds X, but `_readSource(X)` now executes the malicious contract. No factory action is required after the initial `setSource` call — the attack is invisible to the factory.

This is the direct analog of the Basin Aquifer finding: a stored address is treated as immutable by the registry (`source` in APP, `wellImplementations` in Aquifer), but the code at that address can be silently replaced if it contains self-destruct logic.

---

### Impact Explanation

The pool executes swaps at `bid = 1` and `ask = type(uint128).max - 1`. Traders buying pay an astronomically high ask price; traders selling receive near-zero bid proceeds. This constitutes **bad-price execution** (unbounded bid/ask reaching a pool swap) and **swap conservation failure** (traders receive far less than the oracle/bin curve permits). LP assets and trader principal are directly at risk.

---

### Likelihood Explanation

Requires the source contract to contain self-destruct logic. This is the same precondition as the Basin M-09 finding (rated Medium). The curator who deploys the source contract controls whether it has self-destruct capability. Because the source is described as "never reviewed" by the protocol, there is no on-chain enforcement preventing a curator from deploying a metamorphic source. The factory's one-time `setSource` call provides no ongoing protection once the source is replaced.

---

### Recommendation

1. **Document** that source contracts must not contain self-destruct (or `SELFDESTRUCT`/`DELEGATECALL`-to-selfdestruct) logic — analogous to the Basin mitigation.
2. **Optionally**, record the `extcodehash` of the source at the time `setSource` is called and verify it has not changed before each `_readSource` call:
   ```solidity
   bytes32 public sourceCodeHash;
   function setSource(address newSource) external {
       require(msg.sender == factory);
       source = newSource;
       sourceCodeHash = newSource.codehash;
   }
   // In _readSource:
   require(_source.codehash == sourceCodeHash, "source replaced");
   ```
3. **Alternatively**, cap the source output symmetrically: clamp `cBid` to `[refBid, type(uint128).max]` and `cAsk` to `[0, refAsk]` so the source can only tighten, never widen, the reference band.

---

### Proof of Concept

1. Factory deploys `AnchoredPriceProvider` and calls `setSource(X)` where X is a legitimate curator contract.
2. Curator calls `selfdestruct` on X (X must contain this path).
3. Curator redeploys a malicious contract to X via CREATE2 with the same salt. Malicious contract's `getBidAndAskPrice()` returns `(1, type(uint128).max - 1)`.
4. Pool calls `getBidAndAskPrice()` on the provider during a swap.
5. `_readSource(X)` staticcalls the malicious contract → `srcBid = 1`, `srcAsk = type(uint128).max - 1`.
6. Validity check: `1 != 0`, `1 < type(uint128).max - 1`, `type(uint128).max - 1 <= type(uint128).max` → all pass.
7. Clamp: `bidOut = min(refBid, 1) = 1`; `askOut = max(refAsk, type(uint128).max - 1) = type(uint128).max - 1`.
8. Final guard: `1 < type(uint128).max - 1` → passes.
9. Pool executes swap at `bid = 1`, `ask = type(uint128).max - 1` — unbounded bad-price execution. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)

    function _readSource(address _source)
        internal view returns (bool ok, uint256 srcBid, uint256 srcAsk)
    {
        bytes4 sel = IAnchorSource.getBidAndAskPrice.selector;
        bool success;
        uint256 retSize;
        uint256 b;
        uint256 a;
        assembly ("memory-safe") {
            // Scratch beyond the free-memory pointer; never updated, so this is memory-safe.
            let ptr := mload(0x40)
            mstore(ptr, sel) // 4-byte selector, left-aligned
            // Input is consumed before output is written, so in/out may share ptr. Output is capped
            // at 0x40 bytes: a larger returndata is NOT copied (only returndatasize() reports it).
            success := staticcall(SOURCE_GAS_LIMIT, _source, ptr, 0x04, ptr, 0x40)
            retSize := returndatasize()
            b := mload(ptr)
            a := mload(add(ptr, 0x20))
        }
        if (!success || retSize != 64) return (false, 0, 0);

        srcBid = b;
        srcAsk = a;
        if (srcBid == 0 || srcBid >= srcAsk || srcAsk > type(uint128).max) return (false, 0, 0);

        return (true, srcBid, srcAsk);
    }
