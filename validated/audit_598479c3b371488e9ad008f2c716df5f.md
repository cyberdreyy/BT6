### Title
Unconsumed Pusher Consent Signature in `allowPushers` Allows Creator to Re-Establish Revoked Delegation, Enabling Compromised Pusher Key to Feed Bad Prices to Pools — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowPushers()` in `CompressedOracle.sol` does not consume or invalidate the pusher's EIP-191 consent signature after delegation is established. When a pusher calls `revokePusher()` to stop their delegation (e.g., because their key is compromised), the creator can immediately replay the same signature — before the deadline expires — to silently re-establish the delegation. The pusher's revocation is therefore not final until the deadline window closes, and a compromised pusher key can continue writing arbitrary prices into the creator's namespace.

---

### Finding Description

`allowPushers()` verifies the pusher's signature and writes `namespaceRemapping[pusher] = msg.sender`: [1](#0-0) 

The signature domain is:

```
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [2](#0-1) 

No nonce, no used-signature mapping, and no state change marks the signature as consumed. `revokePusher()` only zeroes `namespaceRemapping[msg.sender]`: [3](#0-2) 

The code's own NatSpec acknowledges the risk: *"The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it."* [4](#0-3) 

The deadline mitigates re-establishment **after** expiry, but not **before**. Within the deadline window the creator can call `allowPushers()` with the identical signature an unlimited number of times, each time overwriting the zero written by `revokePusher()`.

The `fallback()` push path resolves the namespace from `namespaceRemapping[msg.sender]` at call time: [5](#0-4) 

So once the creator re-establishes the mapping, every subsequent `fallback()` call from the compromised pusher key writes into the creator's namespace, overwriting legitimate price data with attacker-controlled values.

---

The band clamp is relative to the oracle mid — if the oracle mid is attacker-controlled, the entire band shifts.     function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }

    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
