### Title
Missing Sequencer Uptime Feed Check in L2 Price Providers Allows Immediate Bad-Price Execution After Sequencer Recovery ‚Äî (`smart-contracts-poc/contracts/PriceProviderL2.sol`, `smart-contracts-poc/contracts/ProtectedPriceProviderL2.sol`)

---

### Summary

`PriceProviderL2` and `ProtectedPriceProviderL2` are deployed on L2 chains (Arbitrum, Base, Avalanche, etc.) via `PriceProviderFactoryL2`, but neither contract checks the Chainlink Sequencer Uptime Feed nor enforces a post-recovery grace period. When the sequencer goes down and then recovers, oracle pushers can immediately submit fresh prices and swaps resume at the new price ‚Äî with no window for LPs to adjust their positions.

---

### Finding Description

`PriceProviderL2._getBidAndAskPrice()` reads the oracle price and applies only a staleness check (`_isStale`) against `MAX_TIME_DELTA`:

```solidity
// PriceProviderL2.sol L208-217
function _getBidAndAskPrice() internal returns (uint128, uint128) {
    (uint256 mid, uint256 spread, , uint256 refTime) =
        IPricedOracle(address(offchainOracle)).price(offchainFeedId, msg.sender);

    // 2. Staleness check
    if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA, FUTURE_TOLERANCE)) {
        return (0, type(uint128).max);
    }
    ...
}
```

`ProtectedPriceProviderL2._computeBidAsk()` is identical in structure:

```solidity
// ProtectedPriceProviderL2.sol L206-209
if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA, FUTURE_TOLERANCE)) {
    return (0, type(uint128).max);
}
```

Neither contract:
1. Reads the Chainlink Sequencer Uptime Feed to detect whether the sequencer is currently up.
2. Enforces a grace period after the sequencer recovers (e.g., `block.timestamp - sequencerStartedAt < GRACE_PERIOD`).

The `PriceProviderFactoryL2.createPriceProvider()` constructor call confirms no sequencer uptime feed is ever wired in:

```solidity
// PriceProviderFactoryL2.sol L50-59
PriceProviderL2 p = new PriceProviderL2(
    address(this),
    _oracle,
    _feedId,
    _marginStep,
    _maxTimeDelta,
    _futureTolerance,   // ‚Üê no _sequencerUptimeFeed parameter
    _baseToken,
    _quoteToken
);
```

By contrast, the compiled ABI in `contract-registry/versions/registry.json` shows a separate contract (`ChainlinkVerifierL2`) that does expose `GRACE_PERIOD` and `sequencerUptimeFeed`, and a more complex provider variant whose constructor accepts `_sequencerUptimeFeed`. However, `PriceProviderL2` and `ProtectedPriceProviderL2` ‚Äî the contracts actually deployed by `PriceProviderFactoryL2` on Arbitrum, Base, and other L2s ‚Äî contain no such check.

**Attack path:**

1. Arbitrum sequencer goes down. LPs cannot call `removeLiquidity` or adjust positions. Oracle pushers cannot submit new reports.
2. Sequencer recovers. A pusher immediately calls `ChainlinkOracle.updateReport()` or pushes via `CompressedOracle.fallback()` with the current (post-downtime) price, which may have moved significantly.
3. The new price passes `_isStale` (it is fresh). `getBidAndAskPrice()` returns the new bid/ask.
4. An arbitrageur immediately swaps against the pool at the new oracle price. LPs who could not act during the downtime absorb the loss.

---

### Impact Explanation

LPs suffer direct principal loss. The pool's bid/ask is set entirely by the oracle price; when that price jumps on sequencer recovery, arbitrageurs extract value from LP positions that could not be adjusted during the outage. This is a bad-price execution scenario: the oracle quote is technically fresh but reflects a price movement that LPs had no opportunity to react to.

---

### Likelihood Explanation

Arbitrum sequencer outages have occurred historically. The attack requires no special permissions ‚Äî any EOA can push a fresh oracle report the moment the sequencer recovers, and any EOA can immediately swap. The `PriceProviderFactoryL2` is deployed on Arbitrum, Base, Avalanche, Optimism, BSC, Polygon, and others, all of which use sequencers.

---

### Recommendation

Add a sequencer uptime check and grace period to `PriceProviderL2` and `ProtectedPriceProviderL2`, mirroring the pattern already present in `ChainlinkVerifierL2`:

```solidity
AggregatorV3Interface public immutable sequencerUptimeFeed;
uint256 public constant GRACE_PERIOD = 3600; // 1 hour

function _checkSequencer() internal view {
    if (address(sequencerUptimeFeed) == address(0)) return;
    (, int256 answer, uint256 startedAt,,) = sequencerUptimeFeed.latestRoundData();
    // answer == 0 means sequencer is up
    require(answer == 0, SequencerDown());
    require(block.timestamp - startedAt >= GRACE_PERIOD, GracePeriodNotElapsed());
}
```

Call `_checkSequencer()` at the top of `_getBidAndAskPrice()` / `_computeBidAsk()`. Pass `_sequencerUptimeFeed` through `PriceProviderFactoryL2.createPriceProvider()` (zero address for non-L2 deployments).

---

### Proof of Concept

1. Deploy `PriceProviderL2` on Arbitrum fork via `PriceProviderFactoryL2`.
2. Warp time forward; simulate sequencer downtime by blocking oracle pushes.
3. Warp time forward again (sequencer recovery); push a fresh oracle report with a 10% higher price.
4. Call `getBidAndAskPrice()` ‚Äî it returns the new price immediately, no revert.
5. Execute a swap against the pool at the new price; LP balance decreases by the full price-move delta with no grace period having elapsed. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)

        if (refTime > nowTs) {
            // refTime in the future: tolerate only within futureTol
            return (refTime - nowTs) > futureTol;
        }

        // refTime in the past or equal: check age
        return (nowTs - refTime) > maxDelta;
    }
