After exhaustive tracing through all four pivot paths ‚Äî compressed oracle encoding, delegation, provider math, and abuse-protection ‚Äî here is the complete analysis.

**Compressed path (`CompressedOracle`, `U64x32`, `Codebook256`, `TimeMs`)**

The stalled sentinel (`s0 == 0xff && s1 == 0xff`) returns early without setting `data.timestampMs`, so `refTime = 0`. Both the staleness guard (`refTime == 0 ‚Üí stale`) and the spread guard (`spread >= ORACLE_BPS = 10_000`) independently reject it before any price reaches a pool. [1](#0-0) 

`U64x32.decode` is a pure left-shift with no division; it cannot produce a zero from a non-zero mantissa. `Codebook256.decode` reverts on an out-of-range index via `CodebookDecodeFailed`, which is a DoS on the read path but not a bad-price pass-through. [2](#0-1) [3](#0-2) 

**Delegation path (`allowPushers`, `allowContractPushers`, `revokePusher`, `updateBySignature`)**

`allowPushers` signs over `(chainid, address(this), deadline, pusher, msg.sender)`. The creator address is bound in the hash, so a signature issued for one creator cannot be replayed for another. The deadline prevents re-establishment after revocation. `updateBySignature` performs the monotonicity check before signature verification (gas optimisation), but the signature covers the full `newSlotValue` including the embedded timestamp, so a replayed older word is rejected by the monotonicity gate before the signature is even checked. No namespace hijack path exists. [4](#0-3) [5](#0-4) 

**Provider path (`ProtectedPriceProvider`, `PriceProviderL2`, `_computeBidAsk`)**

`_computeBidAsk` applies four sequential guards before any price reaches the pool: staleness, zero-price/spread-sentinel, price-guard bounds, and `bidOut >= askOut`. `_applyStepAdjustment` returns `(0, false)` if the Q64-scaled result is zero, which causes `FeedStalled()` before the pool sees any value. For any non-zero 8-decimal price and a `stepFactor` in the valid constructor range, the numerator `Q64 * stepFactor ‚âà 1.8e37` divided by `STEP_DENOM = 1e26` yields at least `1.8e11 √ó price`, which is non-zero for all representable prices.     function _computeBidAsk(uint256 price, uint256 spread, uint256 refTime)
        internal view returns (uint128, uint128)
    {
        // 1. Staleness check
        if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA)) {
            return (0, type(uint128).max);
        }

        // 2. Basic validity ‚Äî price must be positive, spread must not be stalled marker
        if (price == 0 || spread >= ORACLE_BPS) {
            return (0, type(uint128).max);
        }

        // 3. Price guard check
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(offchainFeedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (price < guardMin || price > guardMax) {
            return (0, type(uint128).max);
        }

        // 4. Compute bid/ask from mid + confidence-adjusted spread
        uint256 adjustedSpread = spread * confidenceParam;
        (uint256 bid, uint256 ask) = _getBidAskFrom(price, adjustedSpread);

        // 5. Apply marginStep adjustment
        (uint256 bidOut, bool bidOk) = _applyBidAdjustments(bid);
        if (!bidOk || bidOut > type(uint128).max) return (0, type(uint128).max);

        (uint256 askOut, bool askOk) = _applyAskAdjustments(ask);
        if (!askOk || askOut > type(uint128).max) return (0, type(uint128).max);

        // 6. Hard invariant: bid must be strictly less than ask.
        if (bidOut >= askOut) return (0, type(uint128).max);

        return (uint128(bidOut), uint128(askOut));
    }
