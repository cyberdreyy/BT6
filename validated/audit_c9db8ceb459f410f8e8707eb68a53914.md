### Title
Creator can replay a non-expired `allowPushers` consent signature to silently re-establish pusher delegation after `revokePusher`, redirecting future price pushes into the creator's namespace — (File: `smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`CompressedOracle.allowPushers` verifies a deadline-bound EIP-191 signature from the pusher and writes `namespaceRemapping[pusher] = creator`. After the pusher calls `revokePusher()` to clear that mapping, the creator can immediately replay the **same** signature (provided `block.timestamp ≤ deadline`) to re-write `namespaceRemapping[pusher] = creator`. The pusher's revocation is therefore silently undone, and every subsequent fallback push the pusher makes — believing it lands in their own namespace — is instead written into the creator's namespace and consumed by any pool that reads the creator's feeds.

### Finding Description

`allowPushers` builds the signed digest as:

```solidity
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [1](#0-0) 

The only replay guard is `_ensureDeadline(deadline)`, which reverts only after the deadline has passed. There is no used-signature registry, no per-pusher nonce, and no check that `namespaceRemapping[pusher]` is currently zero. The code comment itself acknowledges the risk:

> "The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it." [2](#0-1) 

`revokePusher` clears the mapping:

```solidity
namespaceRemapping[msg.sender] = address(0);
``` [3](#0-2) 

But `allowPushers` unconditionally overwrites it again with no check on prior revocation state. Because the deadline is chosen by the pusher at signing time and is typically set far in the future (production scripts show no enforced maximum), the creator retains the ability to replay the consent for the entire deadline window — potentially months or years.

The fallback push path resolves the namespace at call time:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [4](#0-3) 

So every push the pusher makes after revoking — intending to populate their own namespace — is silently redirected into the creator's namespace and overwrites the feed data that pools read via `price(feedId, pool)` or `getOracleData`. [5](#0-4) 

### Impact Explanation

A pusher typically revokes because they are changing asset coverage, data source, or scale (e.g., switching from ETH/USD to BTC/USD pushes). After revocation, the pusher's new pushes carry prices calibrated for their own namespace. If the creator silently re-establishes delegation, those mis-calibrated prices overwrite the creator's live feed slots. Any pool registered to the creator's `feedId` then executes swaps against the wrong mid-price and spread, satisfying the "bad-price execution" impact gate: a stale or wrong-asset bid/ask quote reaches a live pool swap, causing traders to receive more or less than the oracle/bin curve permits and potentially leaving the pool insolvent relative to LP claims.

### Likelihood Explanation

Medium. The creator must hold a non-expired pusher signature (common when deadlines are set to days/weeks) and must call `allowPushers` after the pusher's `revokePusher`. The pusher must continue pushing (believing they are writing to their own namespace). This is a realistic operational sequence: a pusher rotates asset coverage, revokes, and resumes pushing while the creator quietly re-delegates to preserve their feed. No privileged role is required beyond being the creator who originally called `allowPushers`.

### Recommendation

Track consumed signatures with a `mapping(bytes32 => bool) usedSignatures` keyed on the digest, and revert if the digest has already been used. Alternatively, add a per-pusher nonce to the signed payload so that `revokePusher` can increment the nonce, invalidating all prior signatures for that pusher regardless of deadline.

```solidity
mapping(address => uint256) public pusherNonce; // incremented on revoke

// In revokePusher:
pusherNonce[msg.sender]++;

// In allowPushers digest:
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender, pusherNonce[pusher]))
```

### Proof of Concept

1. Pusher P signs consent with `deadline = block.timestamp + 365 days`, targeting creator C.
2. Creator C calls `allowPushers(deadline, [P], [sig])` → `namespaceRemapping[P] = C`.
3. P decides to push BTC/USD into their own namespace; calls `revokePusher()` → `namespaceRemapping[P] = address(0)`.
4. Creator C immediately calls `allowPushers(deadline, [P], [sig])` with the **identical** signature → `namespaceRemapping[P] = C` again (deadline not yet expired, no revert).
5. P pushes BTC/USD slot words via `fallback`. The oracle resolves `namespaceRemapping[P] = C` and writes BTC/USD prices into `feedIdOf(C, slotIndex, positionIndex)` — the ETH/USD feed that C's pool reads.
6. C's pool calls `getBidAndAskPrice()` → `price(feedId, pool)` → `_price(feedId)` → `getOracleData(feedId)` returns BTC/USD mid and spread.
7. Pool executes a swap at BTC/USD price instead of ETH/USD price; trader receives a grossly wrong token amount. [6](#0-5) [7](#0-6)

    /// @notice Delegates pusher wallets into the caller's namespace. The pusher's EIP-191
    ///         signature is REQUIRED — without it anyone could remap a foreign pusher
    ///         wallet into their own namespace and silently swallow its pushes. The
    ///         deadline is likewise required: the signed consent carries no timestamp of
    ///         its own, so an undated signature could re-establish a delegation AFTER the
    ///         pusher revoked it.
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }

    fallback() override external {
        uint256 end;
        uint256 namespace;

        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0)) creator = msg.sender;

        assembly ("memory-safe") {
            end := calldatasize()
            namespace := shl(96, creator) // [creator:20][zeros:12]
        }

        // 4 * 6 + 7 + 1 = 32 bytes per slot
        if (end == 0 || end % 32 != 0) revert BadCalldataLength();

        for (uint256 ptr = 0; ptr < end; ptr += 32) {
            uint256 word;
            assembly ("memory-safe") {
                word := calldataload(ptr)
            }
            // casting to 'uint8' is safe we want LSB
            // forge-lint: disable-next-line(unsafe-typecast)

            timestampMs.revertIfAfterBlockTimeWithDrift(MAX_TIME_DRIFT);

            if (!newer) continue;
