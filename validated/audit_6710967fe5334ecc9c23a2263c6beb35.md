### Title
Pusher Consent Signature Replay Allows Creator to Override `revokePusher`, Permanently Hijacking Pusher's Namespace Writes Until Deadline - (File: smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol)

---

### Summary

`allowPushers` in `CompressedOracleV1` does not track consumed signatures. A creator who already holds a valid pusher-consent signature can call `allowPushers` again with the same signature after the pusher has called `revokePusher`, silently re-establishing the delegation. The pusher cannot permanently escape the creator's namespace until the deadline in the original signature expires. Any pools backed by the pusher's own feeds receive no further price updates during this window, producing stale prices that halt swaps.

---

### Finding Description

`allowPushers` requires each pusher to supply an EIP-191 signature over:

```
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [1](#0-0) 

The only replay guard is the deadline check via `_ensureDeadline(deadline)`. There is no nonce, no used-signature bitmap, and no check that `namespaceRemapping[pusher]` was previously zero (i.e., that the pusher has not already revoked).

`revokePusher` clears the mapping:

```solidity
namespaceRemapping[msg.sender] = address(0);
``` [2](#0-1) 

But because `allowPushers` performs no check against a prior revocation and does not invalidate the signature, the creator can immediately call `allowPushers` again with the identical `(deadline, [pusher], [sig])` tuple, writing `namespaceRemapping[pusher] = creator` again. The pusher's `revokePusher` is silently undone.

The `fallback` push path resolves the namespace at call time:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [3](#0-2) 

So every subsequent push by the pusher lands in the creator's namespace, not the pusher's own. The pusher's own feeds (identified by `feedIdOf(pusher, slotIndex, positionIndex)`) receive no new data.

---

### Impact Explanation

Any `AnchoredPriceProvider` or pool backed by the pusher's own compressed-oracle feeds will read a timestamp that never advances. `_isStale` in `AnchoredPriceProvider` returns `true` once `(block.timestamp - refTime) > MAX_REF_STALENESS`:

```solidity
if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);
``` [4](#0-3) 

`getBidAndAskPrice` then returns the `(0, type(uint128).max)` sentinel and reverts with `FeedStalled`:

```solidity
if (bid == 0 || ask == type(uint128).max) revert FeedStalled();
``` [5](#0-4) 

Every swap through those pools is blocked for the entire duration of the deadline window. This is a direct "stale bid/ask quote reaches a pool swap" impact and "broken core pool functionality" within the allowed gate.

---

### Likelihood Explanation

- The creator already possesses the signature (they submitted it in the original `allowPushers` call).
- No additional off-chain material is needed; the creator simply re-submits the same calldata.
- The window is as long as the deadline the pusher originally agreed to (potentially hours or days).
- The pusher's only escape is to stop pushing entirely, which itself causes staleness in their own pools.

Likelihood: **Medium** (requires a creator who is adversarial toward the pusher, but the mechanism is trivially executable with no additional cost or capability).

---

### Recommendation

Add a per-pusher revocation nonce or a used-signature set. The simplest fix is a `mapping(bytes32 => bool) private _usedConsentSig` keyed on the signature hash, set to `true` on first use and checked at the top of the loop. Alternatively, include a monotonic per-pusher nonce in the signed message so that each consent can only be used once regardless of deadline.

---

### Proof of Concept

```
1. Pusher signs: keccak256(abi.encode(chainid, oracle, deadline=T+1day, pusher, creator))
2. Creator calls allowPushers(T+1day, [pusher], [sig])
   → namespaceRemapping[pusher] = creator  ✓
3. Pusher calls revokePusher()
   → namespaceRemapping[pusher] = address(0)  ✓
4. Creator calls allowPushers(T+1day, [pusher], [sig])  ← same calldata, no revert
   → namespaceRemapping[pusher] = creator  ← revocation silently undone
5. Pusher's fallback pushes land in creator's namespace; pusher's own feeds stay at
   timestamp 0 (or last pre-revocation value).
6. After MAX_REF_STALENESS seconds, AnchoredPriceProvider._readLeg returns ok=false
   for every feed in the pusher's namespace → getBidAndAskPrice reverts FeedStalled
   → all swaps through those pools are blocked until deadline T+1day expires.
```

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
