### Title
Silent Namespace Hijack via `allowContractPushers` Overwrites Existing Delegation Without Checking Prior Mapping — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`allowContractPushers` establishes delegation using only a live `isPusher(creator)` check and unconditionally writes `namespaceRemapping[pusher] = msg.sender`. It never checks whether the pusher is already mapped to a different creator. Any caller who satisfies `isPusher` can silently overwrite an existing delegation. After the overwrite, the original creator cannot recover: `removePushers` strictly requires `namespaceRemapping[pusher] == msg.sender` and reverts with `InvalidManager` for the displaced creator. The pusher's future fallback writes land in the attacker's namespace, leaving the original creator's feeds permanently stale and causing pools anchored to those feeds to execute swaps at stale prices.

### Finding Description

**The inconsistency (direct analog to the Livepeer M-01 pattern):**

In Livepeer, `_handleVoteOverrides` used the simple check `_account == delegate` to classify a transcoder, while `BondingManager.isRegisteredTranscoder` additionally required `bondedAmount > 0`. The missing condition caused regular delegators to be misclassified.

Here, `allowContractPushers` uses a simple live `isPusher(creator)` check with no guard on the existing mapping: [1](#0-0) 

```solidity
function allowContractPushers(address[] calldata pushers) external {
    ...
    (bool ok, bytes memory res) = pusher.staticcall(abi.encodeWithSignature("isPusher(address)", msg.sender));
    require(ok);
    bool allowed = abi.decode(res, (bool));
    require(allowed);

    namespaceRemapping[pusher] = msg.sender;   // ← unconditional overwrite
    emit PusherAuthorized(pusher, msg.sender);
}
```

Meanwhile, `removePushers` enforces a strict consistency check — the caller **must** be the current manager: [2](#0-1) 

```solidity
function removePushers(address[] calldata pushers) external {
    ...
    if (namespaceRemapping[pusher] == msg.sender) {
        namespaceRemapping[pusher] = address(0);
    } else {
        revert InvalidManager(msg.sender);   // ← original creator is locked out
    }
}
```

The missing condition in `allowContractPushers` is `namespaceRemapping[pusher] == address(0)` — exactly the same class of omission as the missing `bondedAmount > 0` in Livepeer.

**The fallback push path** resolves the namespace from `namespaceRemapping[msg.sender]` with no re-verification: [3](#0-2) 

```solidity
fallback() override external {
    address creator = namespaceRemapping[msg.sender];
    if (creator == address(0)) creator = msg.sender;
    ...
}
```

Once `namespaceRemapping[pusher]` is overwritten to the attacker's address, every subsequent fallback push from that pusher lands in the attacker's namespace. The original creator's slots receive no further updates.

**Contrast with the EOA path (`allowPushers`):** the pusher's EIP-191 signature encodes the specific `msg.sender` (creator), so the pusher must actively re-sign for a new creator. A contract pusher's `isPusher` can return `true` for multiple callers simultaneously, making the consent non-exclusive and the overwrite silent. [4](#0-3) 

### Impact Explanation

Pools anchored to the original creator's compressed feeds call `price(feedId, pool)` → `getOracleData(feedId)` → `_loadSlotLayout(_oracleSlot(creator, slotIndex))`. After the hijack, the slot for the original creator is never updated. The `AnchoredPriceProvider` (or any provider consuming the feed) will read a stale `timestampMs`, which every consumer is documented to reject as stale — but the pool's swap path still executes using the last cached price, satisfying the "stale bid/ask quote reaches a pool swap" impact criterion. This is a direct loss surface for traders and LPs.

### Likelihood Explanation

The attack is permissionless. Any address can call `allowContractPushers`. The only precondition is that the target contract pusher's `isPusher(attacker)` returns `true`. This is realistic for:
- Shared infrastructure pushers designed to serve multiple creators
- Any pusher contract with a permissive or misconfigured `isPusher` implementation
- A pusher contract that the attacker themselves deployed and registered with a legitimate creator

No privileged role, no malicious token, and no admin action is required.

### Recommendation

Add an existence guard in `allowContractPushers` before writing the mapping:

```solidity
function allowContractPushers(address[] calldata pushers) external {
    uint256 l = pushers.length;
    for (uint256 i; i < l; i++) {
        address pusher = pushers[i];
        if (pusher == msg.sender) revert NoSelfRemapping();

        // ── ADD: prevent silent overwrite of an existing delegation ──
        address existing = namespaceRemapping[pusher];
        if (existing != address(0) && existing != msg.sender) {
            revert AlreadyDelegated(pusher, existing);
        }

        (bool ok, bytes memory res) = pusher.staticcall(
            abi.encodeWithSignature("isPusher(address)", msg.sender)
        );
        require(ok);
        bool allowed = abi.decode(res, (bool));
        require(allowed);

        namespaceRemapping[pusher] = msg.sender;
        emit PusherAuthorized(pusher, msg.sender);
    }
}
```

This mirrors the exclusivity guarantee already present in the EOA `allowPushers` path (where the signature binds to a single creator) and makes the two delegation paths consistent.

import {CompressedOracleV1} from "contracts/oracles/compressed/CompressedOracle.sol";

    function isPusher(address) external pure returns (bool) { return true; }

contract NamespaceHijackPoC is Test {
    CompressedOracleV1 oracle;
    PermissivePusher pusher;
    address creatorA = address(0xAAAA);
    address attacker = address(0xBBBB);

    function setUp() public {
        oracle = new CompressedOracleV1(address(this), 0);
        pusher = new PermissivePusher();
        vm.warp(1_700_000_000);
    }

    function testHijack() public {
        // 1. Creator A legitimately delegates the shared pusher
        address[] memory p = new address[](1);
        p[0] = address(pusher);
        vm.prank(creatorA);
        oracle.allowContractPushers(p);
        assertEq(oracle.namespaceRemapping(address(pusher)), creatorA);

        // 2. Attacker silently overwrites the delegation
        vm.prank(attacker);
        oracle.allowContractPushers(p);
        assertEq(oracle.namespaceRemapping(address(pusher)), attacker); // hijacked

        // 3. Creator A cannot remove the pusher — locked out
        vm.prank(creatorA);
        vm.expectRevert(); // InvalidManager
