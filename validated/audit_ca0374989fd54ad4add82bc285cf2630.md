### Title
Signature Replay in `allowPushers` Allows Creator to Undo a Pusher's `revokePusher()` Within the Deadline Window — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1.allowPushers` verifies a pusher's EIP-191 consent signature but never marks it as consumed. Because `_ensureDeadline` only checks `block.timestamp <= deadline` and no nonce or used-signature bitmap exists, the creator can replay the identical `(deadline, pusher, sig)` tuple any number of times before the deadline expires. A pusher who calls `revokePusher()` to clear `namespaceRemapping[pusher]` has that revocation silently undone the moment the creator re-submits the original calldata.

---

### Finding Description

`allowPushers` constructs the signed digest as:

```
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [1](#0-0) 

The tuple contains no nonce, no per-use counter, and no consumed-flag. `_ensureDeadline` performs only a timestamp comparison:

```solidity
require(block.timestamp <= deadline, DeadlineExceeded());
``` [2](#0-1) 

`revokePusher` clears the mapping:

```solidity
namespaceRemapping[msg.sender] = address(0);
``` [3](#0-2) 

But `allowPushers` unconditionally overwrites it on every valid call:

```solidity
namespaceRemapping[pusher] = msg.sender;
``` [4](#0-3) 

The code's own NatSpec acknowledges the deadline is the sole guard against post-revocation re-delegation:

> *"an undated signature could re-establish a delegation AFTER the pusher revoked it"* [5](#0-4) 

The deadline limits the *window* but does not prevent replay *within* that window. The invariant is broken.

---

### Impact Explanation

The primary bad-price execution path:

1. Pusher's EOA key is compromised. Pusher calls `revokePusher()` to protect the creator's namespace from the attacker.
2. Creator (unaware of the compromise, or acting in bad faith) replays the original `allowPushers` calldata before the deadline.
3. `namespaceRemapping[pusher]` is restored to `creator`.
4. The attacker controlling the compromised pusher key pushes manipulated prices into the creator's namespace.
5. Pools consuming those feeds via `price(feedId, pool)` execute swaps at the attacker-controlled price → direct loss of user principal or swap conservation failure.

The creator already controls the namespace directly, so the replay does not grant the creator new powers — but it does nullify the pusher's self-protection mechanism and re-opens the compromised key's write path into the creator's feeds.

---

### Likelihood Explanation

- The replay requires only that the creator re-submit the original transaction calldata before the deadline, which is trivially achievable (e.g., from mempool history or off-chain storage).
- Deadlines are typically set hours to days in the future (the test suite uses `block.timestamp + 1 days`), giving a wide replay window.
- The pusher has no on-chain way to force the creator to stop replaying; `revokePusher()` can be undone repeatedly. [6](#0-5) 

---

### Recommendation

Mark each signature as consumed after first use. Add a `mapping(bytes32 => bool) private _usedSignatures` to `CompressedOracleV1` and, inside `allowPushers`, after recovering the signer:

```solidity
bytes32 sigHash = keccak256(signatures[i]);
require(!_usedSignatures[sigHash], "signature already used");
_usedSignatures[sigHash] = true;
```

Alternatively, include a per-pusher nonce in the signed digest so each consent is single-use by construction.

---

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

import "forge-std/Test.sol";
import {CompressedOracleV1} from "contracts/oracles/compressed/CompressedOracle.sol";
import {MessageHashUtils} from "@openzeppelin/contracts/utils/cryptography/MessageHashUtils.sol";

contract ReplayTest is Test {
    CompressedOracleV1 oracle;
    uint256 constant PUSHER_KEY = 0xBEEF;
    address pusher;
    address creator = address(0xC0FFEE);

    function setUp() public {
        oracle = new CompressedOracleV1(address(this), 0);
        pusher = vm.addr(PUSHER_KEY);
        vm.warp(1_700_000_000);
    }

    function testReplayAfterRevoke() public {
        uint256 deadline = block.timestamp + 1 hours;

        // Pusher signs consent
        bytes32 digest = MessageHashUtils.toEthSignedMessageHash(
            keccak256(abi.encode(block.chainid, address(oracle), deadline, pusher, creator))
        );
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(PUSHER_KEY, digest);
        bytes memory sig = abi.encodePacked(r, s, v);

        address[] memory pushers = new address[](1);
        pushers[0] = pusher;
        bytes[] memory sigs = new bytes[](1);
        sigs[0] = sig;

        // Step 1: creator delegates pusher
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);
        assertEq(oracle.namespaceRemapping(pusher), creator);

        // Step 2: pusher revokes
        vm.prank(pusher);
        oracle.revokePusher();
        assertEq(oracle.namespaceRemapping(pusher), address(0));

        // Step 3: creator replays the SAME signature — succeeds, revocation undone
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);

        // Invariant violated: revoked pusher is re-delegated without a fresh signature
        assertEq(oracle.namespaceRemapping(pusher), creator, "replay re-established revoked delegation");
    }
}
```
