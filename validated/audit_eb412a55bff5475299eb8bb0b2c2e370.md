### Title
Residual Contract-Pusher Delegation Allows Compromised Pusher to Inject Arbitrary Prices into Creator's Namespace — (File: smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol)

---

### Summary

`allowContractPushers` grants permanent namespace write authority to a contract pusher based on a **one-time** live `isPusher(creator)` staticcall. Unlike the EOA path (`allowPushers`), which requires a signed consent with a mandatory deadline, the contract-pusher path has no deadline and no expiry. The resulting `namespaceRemapping[pusher] = creator` entry persists indefinitely. If the pusher contract is later upgraded or compromised, it retains full write authority over the creator's namespace and can push arbitrary prices into any feed slot. Those prices propagate to pools via price providers, causing bad-price execution.

This is the direct analog of M-07: just as the fCash wrapper's residual token allowance could be exploited if the wrapper (an upgradeable contract) is compromised, a contract pusher's residual namespace delegation can be exploited if the pusher contract is upgraded or compromised after delegation.

---

### Finding Description

`allowContractPushers` checks consent once at delegation time:

```solidity
// CompressedOracle.sol lines 217–234
function allowContractPushers(address[] calldata pushers) external {
    uint256 l = pushers.length;
    for (uint256 i; i < l; i++) {
        address pusher = pushers[i];
        if (pusher == msg.sender) { revert NoSelfRemapping(); }

        (bool ok, bytes memory res) = pusher.staticcall(
            abi.encodeWithSignature("isPusher(address)", msg.sender)
        );
        require(ok);
        bool allowed = abi.decode(res, (bool));
        require(allowed);

        namespaceRemapping[pusher] = msg.sender;   // ← permanent, never re-verified
        emit PusherAuthorized(pusher, msg.sender);
    }
}
``` [1](#0-0) 

After this call, the `fallback()` push path resolves the namespace unconditionally from the stored mapping:

```solidity
// CompressedOracle.sol lines 315–316
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [2](#0-1) 

There is no re-verification of `isPusher` on each push. The delegation is permanent until the creator calls `removePushers` or the pusher calls `revokePusher`. The contract-pusher path explicitly omits the deadline that the EOA path (`allowPushers`) requires:

```solidity
// allowPushers — has deadline + signature
function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
    _ensureDeadline(deadline);
    ...
    bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
        keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
    );
    require(pusher == ECDSA.recover(hash, signatures[i]));
    namespaceRemapping[pusher] = msg.sender;
}
``` [3](#0-2) 

The comment on `allowContractPushers` states "there is nothing to replay and no deadline is needed" because consent is live — but consent is only live **at delegation time**. After that, the `namespaceRemapping` entry is the sole authority gate, and it is never re-checked against the pusher contract's current `isPusher` state.

The production deployment script confirms this path is used in production:

```solidity
// SetAllowance.sol
oracle.allowContractPushers(pushers);
``` [4](#0-3) 

---

### Impact Explanation

A compromised or upgraded contract pusher retains full write authority over the creator's namespace. It can call the oracle's `fallback()` with crafted slot words containing arbitrary prices, spreads, and timestamps. Because the timestamp monotonicity check only requires the new timestamp to be strictly after the stored one, the attacker simply uses a future-valid timestamp to overwrite any feed slot:

```
feedId = creator << 96 | chainid << 16 | slotIndex << 8 | positionIndex
``` [5](#0-4) 

The corrupted price then propagates through the price provider chain to the pool's `getBidAndAskPrice()` call during a swap. The `AnchoredPriceProvider` or `PriceProvider` reads the feed via `price(feedId, pool)` and uses the returned mid/spread to compute bid/ask: [6](#0-5) 

A manipulated mid price that passes the `priceGuard` bounds (or where no guard is set) reaches the pool swap, causing bad-price execution: traders receive more than the oracle/bin curve permits, or LPs receive less than owed. This is a direct loss of LP principal or protocol fees.

---

### Likelihood Explanation

The trigger requires:
1. A creator delegates to a contract pusher that is an upgradeable proxy (common in production systems).
2. The proxy admin key is compromised, or the proxy is upgraded to a malicious implementation.
3. A pool uses a price provider backed by a feed in the creator's namespace.

All three conditions are realistic. The production deployment script (`SetAllowance.sol`) shows `allowContractPushers` is used in production. Upgradeable proxies are the standard pattern for production pusher infrastructure. The creator may not detect the compromise before the attacker pushes bad prices.

The asymmetry with the EOA path is the key signal: `allowPushers` requires a deadline precisely because "an undated signature could re-establish a delegation AFTER the pusher revoked it." The same logic applies to contract pushers — a stale delegation can be exploited after the pusher contract changes behavior — but no deadline is enforced.

---

### Recommendation

Add a deadline to `allowContractPushers`, mirroring the EOA path:

```diff
- function allowContractPushers(address[] calldata pushers) external {
+ function allowContractPushers(uint256 deadline, address[] calldata pushers) external {
+     _ensureDeadline(deadline);
      uint256 l = pushers.length;
      for (uint256 i; i < l; i++) {
          address pusher = pushers[i];
          if (pusher == msg.sender) { revert NoSelfRemapping(); }

          (bool ok, bytes memory res) = pusher.staticcall(
              abi.encodeWithSignature("isPusher(address)", msg.sender)
          );
          require(ok);
          bool allowed = abi.decode(res, (bool));
          require(allowed);

          namespaceRemapping[pusher] = msg.sender;
          emit PusherAuthorized(pusher, msg.sender);
      }
  }
```

This forces creators to re-authorize contract pushers periodically, eliminating the residual delegation window after a pusher contract is compromised or upgraded. The deadline also aligns the contract-pusher path with the documented security rationale already applied to the EOA path.

### Proof of Concept

```solidity
// 1. Deploy an upgradeable pusher proxy that initially returns isPusher=true
MockUpgradeablePusher pusherProxy = new MockUpgradeablePusher();
pusherProxy.setIsPusher(true);

// 2. Creator delegates to the proxy
address[] memory pushers = new address[](1);
pushers[0] = address(pusherProxy);
vm.prank(creator);
oracle.allowContractPushers(pushers);
assertEq(oracle.namespaceRemapping(address(pusherProxy)), creator);

// 3. Proxy is "upgraded" — isPusher now returns false, but delegation persists
pusherProxy.setIsPusher(false);
assertEq(oracle.namespaceRemapping(address(pusherProxy)), creator); // still mapped!

// 4. Compromised proxy pushes a manipulated price into the creator's namespace
uint56 tsMs = uint56(block.timestamp * 1000 + 1);
uint48 manipulatedRaw = _packRaw(type(uint32).max, 0, 0); // extreme price, zero spread
vm.prank(address(pusherProxy));
(bool ok,) = address(oracle).call(_wordAt(0, 0, manipulatedRaw, tsMs));
assertTrue(ok, "compromised pusher can still write");

// 5. The manipulated price is now in the creator's namespace
IOffchainOracle.OracleData memory data = oracle.getOracleData(oracle.feedIdOf(creator, 0, 0));
assertEq(data.price, U64x32.decode(uint32(manipulatedRaw >> 16)), "bad price injected");
// Any pool using a price provider backed by feedIdOf(creator, 0, 0) now receives the bad price.
```

    fallback() override external {
        uint256 end;
        uint256 namespace;

        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0)) creator = msg.sender;

        assembly ("memory-safe") {
            end := calldatasize()
            namespace := shl(96, creator) // [creator:20][zeros:12]
        }

        // 4 * 6 + 7 + 1 = 32 bytes per slot
        if (end == 0 || end % 32 != 0) revert BadCalldataLength();

        for (uint256 ptr = 0; ptr < end; ptr += 32) {
            uint256 word;
            assembly ("memory-safe") {
                word := calldataload(ptr)
            }
            // casting to 'uint8' is safe we want LSB
            // forge-lint: disable-next-line(unsafe-typecast)
            uint8 slotId = uint8(word);
            TimeMs timestampMs = toTimeMs(word >> 8 & X56);
            timestampMs.revertIfAfterBlockTimeWithDrift(MAX_TIME_DRIFT);
            bytes32 key = bytes32(namespace | uint256(slotId));
            uint256 old = uint256(_loadStorage(key));
            TimeMs oldTimestampMs = toTimeMs(old >> 8 & X56);

            bool newer = timestampMs.isAfter(oldTimestampMs);
            if (!newer) continue;

            _writeStorage(key, bytes32(bytes32(word & ~uint256(0xff))));
        }

        (uint256 mid, uint256 spreadBps, , bool ok) = _readLeg(baseFeedId);
        if (!ok) return (0, type(uint128).max);

        bytes32 _quote = quoteFeedId;
        if (_quote != bytes32(0)) {
            (uint256 mid2, uint256 spreadBps2, , bool ok2) = _readLeg(_quote);
            if (!ok2 || mid2 == 0) return (0, type(uint128).max);
            // Synthetic ratio (8-decimal): mid1 / mid2. Relative uncertainties of a ratio add.
            mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);
            spreadBps += spreadBps2;
        }

        return _computeBidAsk(mid, spreadBps);
    }

    /// @dev Reads one feed and runs its per-leg guards. ok=false (→ caller halts, fail closed) on:
    ///      stale reference, mid == 0, spreadBps == the off-hours/stall marker (spreadBps >= ORACLE_BPS), or a
    ///      priceGuard violation. Each leg is read through the attributed path independently.
    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
