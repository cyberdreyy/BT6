### Title
Pusher Revocation Is Silently Reversible via Signature Replay — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`revokePusher()` sets `namespaceRemapping[msg.sender] = address(0)`, but the consent signature accepted by `allowPushers` contains no nonce. A creator who still holds a valid (non-expired) signed consent can immediately replay it to re-establish the delegation, silently undoing the pusher's revocation. The pusher believes they are pushing into their own namespace; their data lands in the creator's namespace instead, feeding wrong prices to any pool that reads those feeds.

---

### Finding Description

`allowPushers` authenticates pusher consent with an EIP-191 signature over `(chainid, oracle, deadline, pusher, creator)`: [1](#0-0) 

There is no nonce, no single-use flag, and no on-chain record that the signature was consumed. The only expiry is the `deadline` field. `revokePusher` writes `address(0)` back to the mapping: [2](#0-1) 

Because the same `(deadline, pusher, creator)` tuple is still valid until `block.timestamp >= deadline`, the creator can call `allowPushers` a second time with the identical signature bytes, restoring `namespaceRemapping[pusher] = creator` in the same block the pusher revoked.

The push fallback resolves the namespace at call time: [3](#0-2) 

So every push the pusher makes after their (silently undone) revocation writes into the creator's slot, not their own.

---

### Impact Explanation

The pusher, believing they have revoked, begins pushing data calibrated for their own feeds (different asset, different scale, different spread). That data is written into the creator's slot. Any `AnchoredPriceProvider` or pool that reads the creator's feed receives a structurally wrong mid-price and spread. This satisfies the **bad-price execution** impact gate: a stale or inverted bid/ask quote reaches a pool swap, enabling a trader to receive more than the oracle/bin curve permits or causing the pool to accept less input than owed.

---

### Likelihood Explanation

The creator must have retained the original signature bytes (trivially true — they submitted the transaction and can read it from calldata in the mempool or chain history). The deadline must not yet have expired. Deadlines are set by the creator and are typically long enough to be operationally convenient, making the replay window wide. No special privilege beyond being the original creator is required.

---

### Recommendation

**Short term:** Record a per-pusher nonce or a "used" flag for each signature in `allowPushers`. After `revokePusher()` increments (or sets) the nonce, any previously signed message is invalidated even before its deadline.

**Long term:** Add a `revokedAt[pusher]` timestamp and reject any `allowPushers` call whose signed deadline predates `revokedAt[pusher]`, ensuring a pusher's explicit revocation cannot be undone by replaying a pre-revocation consent.

---

### Proof of Concept

```
1. Pusher signs: sig = sign(chainid || oracle || deadline=T || pusher || creator)
2. Creator calls allowPushers(T, [pusher], [sig])
   → namespaceRemapping[pusher] = creator  ✓

3. Pusher calls revokePusher()
   → namespaceRemapping[pusher] = address(0)
   → Pusher believes they now push to their own namespace

4. Creator calls allowPushers(T, [pusher], [sig])  // same sig, same deadline, block.timestamp < T
   → namespaceRemapping[pusher] = creator  ← silently restored

5. Pusher calls fallback() with data for their own feed (e.g., ETH/USD at $3000)
   → creator = namespaceRemapping[msg.sender] = creator (not pusher)
   → data written to creator's slot (e.g., a BTC/USD feed expecting ~$60000)
   → pool reads creator's BTC/USD feed, receives $3000 mid-price
   → swap executes at ~20× wrong price
``` [4](#0-3) [2](#0-1) [5](#0-4)

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }

        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
        namespaceRemapping[msg.sender] = address(0);
        emit PusherRevoked(msg.sender, creator);
    }

    fallback() override external {
        uint256 end;
        uint256 namespace;

        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0)) creator = msg.sender;

        assembly ("memory-safe") {
            end := calldatasize()
            namespace := shl(96, creator) // [creator:20][zeros:12]
        }

        // 4 * 6 + 7 + 1 = 32 bytes per slot
        if (end == 0 || end % 32 != 0) revert BadCalldataLength();

        for (uint256 ptr = 0; ptr < end; ptr += 32) {
            uint256 word;
            assembly ("memory-safe") {
                word := calldataload(ptr)
            }
            // casting to 'uint8' is safe we want LSB
            // forge-lint: disable-next-line(unsafe-typecast)
            uint8 slotId = uint8(word);
            TimeMs timestampMs = toTimeMs(word >> 8 & X56);
            timestampMs.revertIfAfterBlockTimeWithDrift(MAX_TIME_DRIFT);
            bytes32 key = bytes32(namespace | uint256(slotId));
            uint256 old = uint256(_loadStorage(key));
            TimeMs oldTimestampMs = toTimeMs(old >> 8 & X56);

            bool newer = timestampMs.isAfter(oldTimestampMs);
            if (!newer) continue;

            _writeStorage(key, bytes32(bytes32(word & ~uint256(0xff))));
        }
