### Title
PriceGuard bounds are never enforced in `CompressedOracleV1` — any authorized pusher can write an out-of-bounds price that reaches pool swaps unchecked — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

The `priceGuard` mapping is stored and configurable in `CompressedOracleV1` via `setPriceGuard`, but neither the write path (`fallback()`, `updateBySignature()`) nor the read path (`getOracleData()` → `_price()` → `price()`) ever consults the configured `[min, max]` bounds. A compromised or malicious authorized pusher can write any price value, and that value is returned verbatim to any consuming `PriceProvider` and ultimately to pool swaps.

---

### Finding Description

`CompressedOracleV1` inherits from the compressed `OracleBase`, which stores a `priceGuard` mapping and exposes `setPriceGuard`: [1](#0-0) 

The creator can configure per-feed bounds via `setPriceGuard(feedId, minPrice, maxPrice)`. However, the bounds are never consulted anywhere in the data path:

**Write path — `fallback()`** (lines 311–344): Only checks timestamp monotonicity (`isAfter`) and `maxTimeDrift`. No PriceGuard check before `_writeStorage`. [2](#0-1) 

**Write path — `updateBySignature()`** (lines 271–303): Same — only monotonicity and drift, then writes unconditionally. [3](#0-2) 

**Read path — `getOracleData()`** (lines 101–117): Decodes and returns the raw price without checking `priceGuard[feedId]`. [4](#0-3) 

**Read path — `_price()` / `price()`** (lines 163–178): Calls `getOracleData()` and forwards the result with no bounds check. [5](#0-4) 

The same gap exists in the provider-side `OracleBase._readPrice()`, which is also not `virtual` and returns raw `oracleData` without consulting `priceGuard`: [6](#0-5) 

The `setPriceGuard` function and `priceGuard` mapping are completely dead code — they have no effect on the actual price values returned to consumers.

---

### Impact Explanation

A compromised or malicious authorized pusher writes a price of `1` (or any extreme value) into a feed. The `PriceProvider` consuming this feed receives the unclamped price, computes bid/ask from it, and the pool executes swaps at the manipulated price. LPs receive far less than owed (or a trader drains the pool), causing direct loss of user principal. This matches the impact gate: **bad-price execution — unbounded/unclamped bid/ask quote reaches a pool swap**.

The `PriceProvider` staleness check (`MAX_TIME_DELTA`) does not save the pool here because the attacker pushes a fresh timestamp alongside the manipulated price, so `refTime` is recent and passes the staleness gate. [7](#0-6) 

---

### Likelihood Explanation

**Medium.** The trigger requires an authorized pusher to be compromised or act maliciously. However, the `allowPushers` delegation path (EIP-191 signatures with a deadline) is explicitly designed to support third-party bots and automated pushers — a realistic operational pattern. The PriceGuard is specifically documented as the safety net against exactly this scenario. Its non-enforcement means the protection is illusory: any pusher authorized with a far-future deadline can push an extreme price at any time, including front-running a large pending swap observed in the mempool (the direct analog to the "last-second vote" in the seed report). [8](#0-7) 

---

### Recommendation

Apply the PriceGuard check in the read path inside `getOracleData()`, after decoding the price:

```solidity
PriceGuard memory guard = priceGuard[feedId];
if (guard.min != 0 || guard.max != 0) {
    require(
        data.price >= guard.min && data.price <= guard.max,
        PriceOutOfBounds(feedId, data.price)
    );
}
```

Alternatively, enforce the check at write time in `fallback()` and `updateBySignature()` before calling `_writeStorage`, so out-of-bounds data is rejected at ingestion. Either location is sufficient; applying it at read time is safer because it also catches data written before a guard was configured.

---

### Proof of Concept

1. Creator Alice deploys a `CompressedOracleV1` feed and calls `setPriceGuard(feedId, 90e8, 110e8)` — price should stay between \$90 and \$110.
2. Alice authorizes Pusher Bob via `allowPushers(farFutureDeadline, [bob], [bobSig])`.
3. Bob's private key is compromised. The attacker calls `fallback()` with a 32-byte slot word encoding `price = 1` (U64x32-encoded) and a timestamp strictly greater than the current stored timestamp.
4. `fallback()` checks only monotonicity and drift — both pass. `_writeStorage` writes the manipulated slot.
5. Any call to `getOracleData(feedId)` now returns `price = 1` with a fresh `timestampMs`.
6. The consuming `PriceProvider` reads `mid = 1`, computes `bid ≈ 1`, `ask ≈ 1` in Q64, and returns these to the pool.
7. The pool executes swaps at `price ≈ 1`, allowing a trader to drain LP-deposited assets at a fraction of fair value. [9](#0-8) [4](#0-3)

```text
    fallback() override external {
        uint256 end;
        uint256 namespace;

        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0)) creator = msg.sender;

        assembly ("memory-safe") {
            end := calldatasize()
            namespace := shl(96, creator) // [creator:20][zeros:12]
        }

        // 4 * 6 + 7 + 1 = 32 bytes per slot
        if (end == 0 || end % 32 != 0) revert BadCalldataLength();

        for (uint256 ptr = 0; ptr < end; ptr += 32) {
            uint256 word;
            assembly ("memory-safe") {
                word := calldataload(ptr)
            }
            // casting to 'uint8' is safe we want LSB
            // forge-lint: disable-next-line(unsafe-typecast)
            uint8 slotId = uint8(word);
            TimeMs timestampMs = toTimeMs(word >> 8 & X56);
            timestampMs.revertIfAfterBlockTimeWithDrift(MAX_TIME_DRIFT);
            bytes32 key = bytes32(namespace | uint256(slotId));
            uint256 old = uint256(_loadStorage(key));
            TimeMs oldTimestampMs = toTimeMs(old >> 8 & X56);

            bool newer = timestampMs.isAfter(oldTimestampMs);
            if (!newer) continue;

            _writeStorage(key, bytes32(bytes32(word & ~uint256(0xff))));
        }
```
