### Title
Slot-level shared timestamp in `CompressedOracleV1` masks per-lane price staleness, allowing stale prices to bypass freshness checks and reach pool swaps — (File: `smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1` packs four price feeds (lanes) into a single 256-bit storage slot with **one shared 56-bit timestamp**. When a pusher updates a slot, all four lanes receive the same new timestamp regardless of whether each lane's price data is actually fresh. A pusher who has fresh data for only one lane but must write all four simultaneously can stamp stale prices with a current timestamp. Downstream consumers — including `AnchoredPriceProvider` and pools — use the returned `refTime` (the slot-level timestamp) for staleness validation; because that timestamp is fresh, the check passes and the stale price reaches swap execution.

---

### Finding Description

**Slot layout — one timestamp for four lanes**

The storage slot layout is:

```
[255:208] Lane 0 (48 bits: price32 | s0 | s1)
[207:160] Lane 1
[159:112] Lane 2
[111: 64] Lane 3
[ 63:  8] Timestamp (56-bit milliseconds) ← shared by all four lanes
[  7:  0] Reserved / slotId on push
``` [1](#0-0) 

`getOracleData` returns the slot-level timestamp for every lane:

```solidity
data.timestampMs = _layout.timestampMs;   // same value for lane 0, 1, 2, 3
``` [2](#0-1) 

**Push path — monotonicity is slot-level only**

`fallback()` accepts one 32-byte word per slot. The only freshness gate is a comparison of the new slot timestamp against the stored slot timestamp:

```solidity
bool newer = timestampMs.isAfter(oldTimestampMs);
if (!newer) continue;
_writeStorage(key, bytes32(bytes32(word & ~uint256(0xff))));
``` [3](#0-2) 

There is no per-lane timestamp, no per-lane monotonicity check, and no on-chain enforcement that the price data for each lane is actually fresh at the time of the push.

**Read path — `refTime` propagates the slot timestamp**

`price()` returns `refTime = data.timestampMs.toSeconds()`, which is the slot-level timestamp:

```solidity
return (uint256(data.price), uint256(data.spread0), data.spread1, data.timestampMs.toSeconds());
``` [4](#0-3) 

Consumers (`AnchoredPriceProvider`, pools) use `refTime` to decide whether the price is within the acceptable staleness window. Because the slot timestamp is fresh (it was just written), the staleness check passes for every lane in the slot — including lanes whose price data was not actually updated.

---

### Impact Explanation

A pool consuming any lane whose price was not genuinely refreshed in the latest push will execute swaps against a stale mid-price that carries a fresh `refTime`. This is a **bad-price execution** impact: the pool's bid/ask is derived from an outdated oracle value while the staleness guard is silently bypassed. LPs bear the resulting arbitrage loss; in extreme cases (e.g., a large price move in one asset while the pusher's data source for that asset is temporarily unavailable) the loss can be material.

---

### Likelihood Explanation

The trigger does not require a malicious pusher. A legitimate pusher who:

1. Needs to urgently update lane 0 (e.g., ETH/USD moved sharply), **and**
2. Has a temporary data-source outage for lanes 1–3 (BTC/USD, SOL/USD, AVAX/USD)

is forced by the protocol's slot-level write granularity to either skip the urgent update entirely or write the slot with stale prices for the unavailable lanes. Option 2 is the natural choice and produces the vulnerability. Off-chain pusher infrastructure failures, network partitions, or feed-provider outages are realistic operational events.

---

### Recommendation

1. **Per-lane timestamps**: Extend the slot layout to store a timestamp per lane (e.g., reduce each lane from 48 bits to 40 bits and allocate 8 bits per lane for a coarse timestamp delta, or use a separate storage slot for per-lane timestamps). Staleness checks on the read path should use the per-lane timestamp, not the slot-level one.

2. **Sentinel on absent data**: If a pusher cannot supply a fresh price for a lane, require them to write the stalled sentinel (`s0 = 0xFF, s1 = 0xFF, p = 0`) for that lane. Consumers already treat this sentinel as a "stalled" feed and widen the spread to `BPS_BASE`. This prevents a stale price from masquerading as fresh. [6](#0-5) 

3. **Documentation / invariant enforcement**: At minimum, document that all four lanes in a slot must be updated with genuinely fresh data on every push, and add an off-chain linting step in the pusher tooling to enforce this.

### Proof of Concept

```

─────
Slot 0, creator Alice, four lanes:
  Lane 0 → ETH/USD
  Lane 1 → BTC/USD
  Lane 2 → SOL/USD
  Lane 3 → AVAX/USD

Step 1 (T = T0)
───────────────
Alice's pusher calls fallback() with a slot word:
  Lane0 = ETH $3 000, Lane1 = BTC $60 000, Lane2 = SOL $100, Lane3 = AVAX $30
  Timestamp = T0
All four lanes stored; slot timestamp = T0.

Step 2 (T = T0 + 1 h)
──────────────────────
BTC price has moved to $65 000.
Alice's pusher's BTC data source is temporarily unavailable.
ETH has also moved to $3 100 and must be updated urgently.

The pusher constructs a slot word:
  Lane0 = ETH $3 100  ← fresh
  Lane1 = BTC $60 000 ← STALE (copied from last known value)
  Lane2 = SOL $100    ← stale
  Lane3 = AVAX $30    ← stale
  Timestamp = T0 + 1 h  ← fresh

Monotonicity check: (T0 + 1 h) > T0 → passes.
Slot written. All four lanes now carry timestamp T0 + 1 h.

Step 3 (T = T0 + 1 h + ε)
──────────────────────────
A pool configured on feedId = Alice/slot0/lane1 (BTC/USD) calls:
  CompressedOracleV1.price(feedId, pool)
  → getOracleData(feedId)
  → data.price      = $60 000  (stale)
  → data.timestampMs = T0 + 1 h (fresh)
  → refTime          = T0 + 1 h

Pool's staleness guard: (T0 + 1 h) is within maxTimeDrift → passes.
Pool executes swaps at BTC $60 000 instead of $65 000.
Arbitrageurs drain the pool's BTC-side liquidity at the $5 000 discount.

        (address creator, uint8 slotIndex, uint8 positionIndex) = _unpackFeedId(feedId);

        SlotLayout memory _layout = _loadSlotLayout(_oracleSlot(creator, slotIndex));
        CompressedOracleData memory compressed = _selectCompressedData(_layout, positionIndex);

        if (compressed.s1 == 0xff && compressed.s0 == 0xff) {
            data.spread1 = BPS_BASE;
            data.spread0 = BPS_BASE;
            return data;

        data.spread0 = _decodeCodebookIndex(compressed.s0);
        data.spread1 = _decodeCodebookIndex(compressed.s1);
        data.timestampMs = _layout.timestampMs;

    function _loadSlotLayout(bytes32 slotIndex) internal view returns (SlotLayout memory _layout) {
        uint256 slotValue;
        assembly ("memory-safe") {
            slotValue := sload(slotIndex)

        _layout.oracle2 = _decodeCompressedOracleData(uint48((slotValue >> 112) & X48));
