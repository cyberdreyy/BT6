### Title
Permissionless `register()` unconditionally clears ADMIN-set blacklist, allowing any actor to re-enable a blocked pool's oracle read path — (File: `smart-contracts-poc/contracts/oracles/providers/OracleBase.sol`)

---

### Summary

`OracleBase.register()` is a permissionless, fee-gated function whose primary purpose is to whitelist a pool for a feedId. As a side-effect it unconditionally clears any existing blacklist on that pool. Because the blacklist is otherwise exclusively controlled by `ADMIN_ROLE`, this creates an unprivileged path that lets anyone override an admin security decision for the cost of the registration fee (default: **1 wei**).

---

### Finding Description

`setBlacklist()` is the only intended write path for the `blacklisted` mapping and is gated by `onlyRole(ADMIN_ROLE)`: [1](#0-0) 

However, `register()` — which is `external payable` with no role check — contains this block: [2](#0-1) 

Lines 207-210 write `blacklisted[pool] = false` without any access-control guard. The only preconditions are:

1. `msg.value >= registrationFee` — default is **1 wei** (`registrationFee = 1 wei`).
2. `approvedFactories.contains(factory)` — factory must be admin-approved.
3. `IPoolFactory(factory).isPool(pool)` — pool must be recognized by that factory. [3](#0-2) 

All three conditions are satisfiable by any unprivileged actor who knows a valid `(pool, factory)` pair — both of which are public on-chain.

The `price()` function enforces the blacklist at read time: [4](#0-3) 

Once `register()` clears `blacklisted[pool]`, the pool passes both the `notBlacklisted` modifier on `msg.sender` (the price provider) and the explicit `require(!blacklisted[pool])` check, restoring full oracle read access.

---

### Impact Explanation

The blacklist is the oracle's primary abuse-protection lever. An admin blacklists a pool when it is detected performing price-read abuse, sandwich attacks, or other manipulation. With the blacklist bypassed:

- The pool can call `price(feedId, pool)` again during swaps.
- Any bad-price execution the admin was trying to halt resumes immediately.
- LP assets and trader principal in that pool are exposed to the same manipulation the admin was blocking.

This maps directly to the allowed impact categories: **admin-boundary break** (oracle role check bypassed by an unprivileged path) and **bad-price execution** (stale or manipulated bid/ask reaching a pool swap).

---

### Likelihood Explanation

**High.** The registration fee starts at 1 wei and is tunable by ADMIN, but even if raised, the attacker pays once to permanently re-enable the pool. The pool and factory addresses are public. No special role, signature, or off-chain coordination is required. The attack is a single transaction.

---

### Recommendation

Remove the blacklist-clearing side-effect from `register()`. Blacklist management should remain exclusively in `setBlacklist()` (ADMIN_ROLE). If the intent is to allow a pool to "pay its way out" of a blacklist, that path must itself be gated by `onlyRole(ADMIN_ROLE)` or a separate two-step admin approval:

```solidity
// Remove these lines from register():
// if (blacklisted[pool]) {
//     blacklisted[pool] = false;
//     emit BlacklistUpdated(pool, false);
// }
```

---

### Proof of Concept

```
1. Admin calls setBlacklist(poolP, true)  →  blacklisted[poolP] = true
   (pool P was abusing oracle reads)

2. Attacker calls register{value: 1 wei}(feedId, poolP, approvedFactory)
   Preconditions pass:
     - msg.value (1 wei) >= registrationFee (1 wei)  ✓
     - approvedFactories.contains(approvedFactory)    ✓
     - approvedFactory.isPool(poolP)                  ✓  (pool is legitimate)

3. Inside register():
     blacklisted[poolP] = false   ← no role check, no guard

4. poolP.swap() → provider.getBidAndAskPrice()
              → oracle.price(feedId, poolP)
   Both blacklist checks now pass.
   Oracle returns live price data to the previously-blocked pool.

5. Attacker executes the same manipulation the admin was blocking.
```

```text
    function register(bytes32 feedId, address pool, address factory) external payable {
        require(msg.value >= registrationFee, InsufficientFee(msg.value, registrationFee));
        require(pool != address(0));
        require(approvedFactories.contains(factory), FactoryNotApproved(factory));
        require(IPoolFactory(factory).isPool(pool), NotAPool(pool));

        if (blacklisted[pool]) {
            blacklisted[pool] = false;
            emit BlacklistUpdated(pool, false);
        }

        registeredPool[feedId][pool] = true;
        emit PoolRegistered(feedId, pool, msg.sender, msg.value);
    }
```
