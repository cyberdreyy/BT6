### Title
Slot-level timestamp shared across all 4 positions allows stale per-position prices to bypass staleness checks — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`CompressedOracleV1` packs 4 oracle feeds into one 256-bit storage slot and stores **one shared `timestampMs`** for the entire slot. `getOracleData()` returns that slot-level timestamp as `refTime` for every position. A pusher who updates any one position with a fresh timestamp silently "refreshes" the staleness signal for all three other positions in the same slot, even when those positions carry price data that has not changed since a much earlier push. Any consumer that enforces a `MAX_REF_STALENESS` window — including `AnchoredPriceProvider` — will accept those stale prices as current.

### Finding Description

The slot layout packs four 48-bit oracle lanes and one 56-bit timestamp into a single EVM storage word:

```
bits 255…208 : oracle[0]   (48 bits)
bits 207…160 : oracle[1]   (48 bits)
bits 159…112 : oracle[2]   (48 bits)
bits 111… 64 : oracle[3]   (48 bits)
bits  63…  8 : timestampMs (56 bits, shared)
bits   7…  0 : reserved / slotId in calldata
``` [1](#0-0) 

`getOracleData` reads the position-specific price bytes but assigns the **slot-level** timestamp to every position unconditionally:

```solidity
data.timestampMs = _layout.timestampMs;   // same for all 4 positions
``` [2](#0-1) 

The `fallback()` push path overwrites the **entire** 256-bit slot word on every accepted update. The monotonicity guard only checks whether the incoming timestamp is strictly newer than the stored one — it does not verify that the price bytes for each lane have actually changed:

```solidity
bool newer = timestampMs.isAfter(oldTimestampMs);
if (!newer) continue;
_writeStorage(key, bytes32(bytes32(word & ~uint256(0xff))));
``` [3](#0-2) 

The documentation itself acknowledges the overwrite-all semantics: *"The entire slot is overwritten on update: if you update a single lane, you must still supply correct values for the other lanes."* [4](#0-3) 

This is a design-level enforcement gap: the contract accepts a slot word that carries a fresh timestamp for position 0 and **carries forward the old, stale price bytes** for positions 1–3. After the write, `getOracleData` for positions 1–3 returns the new timestamp T2 even though their price data was last set at T1.

`AnchoredPriceProvider._readLeg()` uses the returned `refTime` as the sole staleness signal:

```solidity
if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);
``` [5](#0-4) 

Because `refTime` is now T2 (fresh), the staleness gate passes for positions 1–3 even though their prices are from T1.

The same flaw exists in the `updateBySignature` path, which is callable by **anyone** who holds a creator-signed slot word: [6](#0-5) 

A creator who signs a slot word that only refreshes one lane (leaving the others at old values) produces a replayable artifact that any third party can submit, permanently refreshing the staleness clock for all four positions.

### Impact Explanation

Any pool whose `IPriceProvider` is backed by `CompressedOracleV1` will receive a stale mid-price and spread for the affected positions while the staleness check reports them as current. The `AnchoredPriceProvider` band is computed from that stale mid, so the bid/ask quotes presented to swappers are anchored to an outdated price. Traders can exploit the divergence between the stale on-chain quote and the true market price to extract value from LP reserves — a direct loss of LP principal that satisfies the "bad-price execution" impact gate.

### Likelihood Explanation

The trigger is a semi-trusted pusher (authorized via `allowPushers` or `allowContractPushers`). Two realistic paths exist:

1. **Operational**: A pusher whose upstream data source delivers updates for different feeds at different cadences naturally carries forward the last-known value for slower feeds. The slot timestamp advances with every push, silently masking the staleness of the slower lanes.
2. **Adversarial**: A delegated pusher who wishes to lock in a favorable price for one position can push a slot word with a fresh timestamp for that position and stale (advantageous) prices for the others, bypassing `MAX_REF_STALENESS` for all four.

Neither path requires any privileged role beyond the semi-trusted pusher authorization that the protocol explicitly supports.

### Recommendation

Track freshness **per position**, not per slot. The simplest fix is to store a separate 56-bit timestamp for each of the four lanes (consuming 4 × 56 = 224 bits, still fitting in one slot alongside the four 48-bit price lanes if the layout is reorganized). Alternatively, enforce at write time that any lane whose price bytes differ from the stored value must carry a timestamp strictly newer than the current slot timestamp, and that any lane whose price bytes are unchanged must carry the **same** timestamp as the stored value (i.e., no silent carry-forward with a newer timestamp).

Step 1 – T = 0

Step 3 – T = 61 s (same block)
Pusher pushes a new slot word for slot 0 with timestamp T2 = 61_000 ms:
  pos 0: price = 1_000_100  (fresh ETH/USD)

  pos 2: price = 100_100    (fresh SOL/USD)

Step 4 – Pool swap

    function getOracleData(bytes32 feedId) public view override returns (OracleData memory data) {
        (address creator, uint8 slotIndex, uint8 positionIndex) = _unpackFeedId(feedId);

        SlotLayout memory _layout = _loadSlotLayout(_oracleSlot(creator, slotIndex));
        CompressedOracleData memory compressed = _selectCompressedData(_layout, positionIndex);

        if (compressed.s1 == 0xff && compressed.s0 == 0xff) {
            data.spread1 = BPS_BASE;
            data.spread0 = BPS_BASE;
            return data;

        data.price = U64x32.decode(compressed.p);
        data.spread0 = _decodeCodebookIndex(compressed.s0);
        data.spread1 = _decodeCodebookIndex(compressed.s1);
        data.timestampMs = _layout.timestampMs;
    }

    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
