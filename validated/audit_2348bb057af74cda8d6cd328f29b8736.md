### Title
Missing Oracle Allowlist in `PriceProviderFactory` and `PriceProviderFactoryL2` Enables Arbitrary Oracle Injection — (File: `smart-contracts-poc/contracts/PriceProviderFactory.sol`, `smart-contracts-poc/contracts/PriceProviderFactoryL2.sol`)

---

### Summary

`PriceProviderFactory.createPriceProvider` and `PriceProviderFactoryL2.createPriceProvider` are permissionless and accept any `_oracle` address without validation against an allowlist. `AnchoredProviderFactory` — the parallel factory — correctly enforces an admin-curated oracle allowlist before deployment. The missing guard allows any caller to register a `PriceProvider` backed by a malicious oracle that returns attacker-controlled prices, which then reach pool swaps through the standard `getBidAndAskPrice()` path.

---

### Finding Description

`AnchoredProviderFactory.createAnchoredProvider` enforces:

```solidity
if (!_oracles.contains(oracle)) revert OracleNotAllowed(oracle);
```

before deploying any provider. The allowlist is admin-curated via `addOracle` / `removeOracle`.

`PriceProviderFactory.createPriceProvider` contains no equivalent check:

```solidity
function createPriceProvider(
    address _oracle,
    bytes32 _feedId,
    int256  _marginStep,
    uint256 _maxTimeDelta,
    address _baseToken,
    address _quoteToken
) external override returns (address provider) {
    PriceProvider p = new PriceProvider(
        address(this),
        _oracle,          // ← any address accepted, no allowlist
        _feedId,
        ...
    );
    _providers.add(provider);          // immediately tracked as isProvider() == true
    providerOwner[provider] = creator;
```

`PriceProviderFactoryL2.createPriceProvider` has the identical gap.

Once a `PriceProvider` is deployed against a malicious oracle and tracked by the factory (`isProvider() == true`), the malicious oracle controls every value returned through `_getBidAndAskPrice()`:

- **`mid`** — the oracle returns an attacker-chosen price.
- **`spread`** — returned below `ORACLE_BPS` to pass the stall check.
- **`refTime`** — returned as `block.timestamp` to pass the staleness check.
- **`priceGuard(feedId)`** — the oracle returns `(0, 0)`, which the provider converts to `(0, type(uint128).max)`, so any `mid` passes the guard range.

All downstream checks in `_getBidAndAskPrice` (`_isStale`, `mid == 0`, `spread >= ORACLE_BPS`, `priceGuard`) are satisfied by the attacker-controlled oracle, and the provider emits an arbitrary `(bid, ask)` pair to the pool.

---

### Impact Explanation

A pool that uses a `PriceProvider` from `PriceProviderFactory` backed by a malicious oracle receives attacker-controlled bid/ask prices at swap time. The pool's `SwapMath` uses these prices to determine how many tokens a trader receives. With an inverted or artificially wide spread the pool can be drained: the attacker swaps at a price far from market, extracting more output tokens than the pool is owed, causing direct loss of LP principal. This matches the **bad-price execution** and **direct loss of user principal** impact classes.

---

### Likelihood Explanation

`createPriceProvider` is fully permissionless — no role, no allowlist, no admin approval. Any EOA can:
1. Deploy a malicious oracle contract.
2. Call `createPriceProvider` with it.
3. Point a pool at the resulting provider.

The only friction is that a pool must be created or reconfigured to use the malicious provider. Given permissionless pool creation (any `token0 < token1` pair), this is reachable without any privileged action. Likelihood is **Medium** (requires a pool to adopt the provider, but no privileged step blocks the attacker from creating both the provider and the pool).

---

### Recommendation

Add an admin-curated oracle allowlist to `PriceProviderFactory` and `PriceProviderFactoryL2`, mirroring `AnchoredProviderFactory`:

```solidity
EnumerableSet.AddressSet private _oracles;

function addOracle(address oracle) external onlyRole(ADMIN_ROLE) {
    require(oracle != address(0));
    require(_oracles.add(oracle));
}

function createPriceProvider(address _oracle, ...) external returns (address provider) {
    if (!_oracles.contains(_oracle)) revert OracleNotAllowed(_oracle);
    // ... rest of deployment
}
```

This ensures every `PriceProvider` is anchored to a vetted oracle, closing the injection path at the factory boundary before any provider is tracked as `isProvider() == true`.
