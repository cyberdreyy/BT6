### Title
Delegation Signature Not Consumed After Use in `allowPushers` Allows Creator to Re-Establish Revoked Pusher Delegation, Enabling Continued Bad-Price Injection — (File: `smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowPushers` in `CompressedOracleV1` verifies a pusher's EIP-191 consent signature but never marks it as used. There is no nonce, no used-signature set, and no per-pusher revocation counter. The only replay guard is the deadline. Within the deadline window, the creator can call `allowPushers` again with the identical signature to re-establish a delegation the pusher explicitly revoked via `revokePusher()`. If the pusher's key is compromised and the pusher revokes to stop bad-price injection, the creator (unaware or colluding) can immediately re-delegate using the same bytes, restoring the attacker's write authority over the creator's namespace and allowing bad prices to reach pool swaps.

---

### Finding Description

`allowPushers` performs the following steps for each pusher:

1. Checks `block.timestamp <= deadline` via `_ensureDeadline`.
2. Recovers the signer from `keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))`.
3. Writes `namespaceRemapping[pusher] = msg.sender`. [1](#0-0) 

No state is updated to record that this particular `(pusher, creator, deadline, signature)` tuple has been consumed. The signature is a pure off-chain bytes value; the contract has no `usedSignatures` mapping, no per-pusher nonce, and no revocation counter.

`revokePusher` clears the mapping:

```solidity
namespaceRemapping[msg.sender] = address(0);
``` [2](#0-1) 

But this write has no effect on the validity of the original signature. As long as `block.timestamp <= deadline`, the creator can call `allowPushers` again with the same `(deadline, [pusher], [sig])` arguments and the mapping is restored to `creator`.

The code's own NatSpec acknowledges the risk:

> "The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it." [3](#0-2) 

The deadline mitigates the infinite-replay window but does not prevent replay within the deadline window. The pusher's revocation is therefore not final until the deadline expires.

The `fallback` push path resolves the namespace at call time:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [4](#0-3) 

So restoring `namespaceRemapping[pusher] = creator` immediately restores the attacker's ability to write into the creator's namespace via the compromised pusher key.

---

### Impact Explanation

The `CompressedOracleV1` is an open oracle — reads are permissionless and any price provider can consume prices from any creator's namespace. Bad prices pushed into the creator's namespace flow directly to pool swaps through the `price(feedId, pool)` path. A compromised pusher key that is re-delegated after revocation can inject arbitrary mid-prices, spread0, and spread1 values into the creator's slots, causing pool swaps to execute at attacker-controlled bid/ask quotes. This is a direct bad-price execution impact above Sherlock thresholds.

---

### Likelihood Explanation

Medium. The attack requires:
1. A pusher's key to be compromised (realistic for hot-wallet pushers).
2. The creator to call `allowPushers` again after the pusher revokes — this is plausible if the creator runs automated re-delegation scripts or is unaware of the compromise.
3. The original deadline to still be in the future — deadlines are typically set days or weeks ahead.

All three conditions are reachable without any privileged assumption. The creator is not an oracle admin; `allowPushers` is a permissionless call requiring only a valid signature.

---

### Recommendation

Track consumed delegation signatures to prevent replay. The simplest fix is a `mapping(bytes32 => bool) public usedDelegations` keyed on the signature hash, set to `true` on first acceptance and checked before writing `namespaceRemapping`:

```solidity
bytes32 sigHash = keccak256(signatures[i]);
require(!usedDelegations[sigHash], SignatureAlreadyUsed());
usedDelegations[sigHash] = true;
namespaceRemapping[pusher] = msg.sender;
```

Alternatively, include a per-pusher nonce in the signed payload and increment it on each accepted delegation, so a revoked pusher can advance their nonce to invalidate all prior signatures.

### Proof of Concept

```
T=0:   Pusher signs consent: keccak256(abi.encode(chainid, oracle, deadline=T+30days, pusher, creator))
       → sig = <bytes>

T=1:   creator calls allowPushers(T+30days, [pusher], [sig])
       → namespaceRemapping[pusher] = creator  ✓

T=2:   Pusher's key is compromised.
       Attacker pushes bad price P_bad into creator's slot via fallback().
       → oracle slot for (creator, slotId) now contains P_bad.

T=3:   Pusher calls revokePusher()
       → namespaceRemapping[pusher] = address(0)
       → Attacker's fallback pushes now land in pusher's OWN namespace (harmless).

T=4:   Creator (unaware of compromise) calls allowPushers(T+30days, [pusher], [sig])
       with the IDENTICAL sig bytes — deadline still valid, signature still verifies.
       → namespaceRemapping[pusher] = creator  ← delegation restored

T=5:   Attacker calls fallback() again with P_bad2.
       → namespaceRemapping[pusher] == creator → write lands in creator's namespace.
       → oracle slot for (creator, slotId) = P_bad2.

T=6:   Price provider reads oracle.price(feedIdOf(creator, slotId, pos), pool)
       → returns P_bad2 as mid price.
       → Pool swap executes at attacker-controlled bid/ask.
```

The cycle at T=4–T=6 can repeat for the entire remaining deadline window. The pusher's `revokePusher()` call at T=3 is completely nullified by the creator's replay of the original signature.

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
```
