### Title
Creator can re-delegate a revoked EOA pusher indefinitely within the deadline window via consent-signature replay — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1.allowPushers()` accepts a pusher's EIP-191 consent signature and writes `namespaceRemapping[pusher] = creator`. The signature is never consumed or invalidated. After a pusher calls `revokePusher()` to clear the mapping, the creator can immediately replay the original signature (provided `block.timestamp <= deadline`) to re-establish the delegation. This cycle repeats indefinitely until the deadline expires, trapping the pusher in the creator's namespace and causing the pusher's own namespace to receive no updates — producing stale prices for any pool or provider reading from that namespace.

---

### Finding Description

`allowPushers` verifies the pusher's EIP-191 consent over `(chainid, oracle, deadline, pusher, creator)` and sets `namespaceRemapping[pusher] = creator`: [1](#0-0) 

`revokePusher` clears the mapping back to `address(0)`: [2](#0-1) 

The only replay guard is `_ensureDeadline`, which checks `block.timestamp <= deadline`: [3](#0-2) [4](#0-3) 

There is **no nonce, no used-signature bitmap, and no per-pusher revocation counter**. The same `(deadline, sig)` tuple passes `_ensureDeadline` and `ECDSA.recover` every time it is replayed, as long as the deadline has not expired.

The code comment at lines 186–191 explicitly acknowledges the risk of post-revocation re-delegation but treats the deadline as the sole mitigation: [5](#0-4) 

The deadline limits the *window* of the attack but does not prevent re-delegation within that window. A pusher who signed a consent with a 30-day deadline is trapped for up to 30 days regardless of how many times they call `revokePusher()`.

The `fallback()` push path resolves the namespace at call time: [6](#0-5) 

While the pusher is re-delegated, every fallback push lands in the creator's namespace. The pusher's own namespace (`feedIdOf(pusher, slotIndex, positionIndex)`) receives no updates and its stored timestamp stagnates.

---

### Impact Explanation

Any `PriceProvider` or pool configured to read from the pusher's own namespace (i.e., a `feedId` whose creator bits equal the pusher's address) will observe a stale `timestampMs`. Downstream consumers that enforce a `maxTimeDrift` staleness check will revert or return a zero/sentinel price, breaking swap execution. Pools that do not enforce staleness will execute swaps against an arbitrarily old mid-price, allowing traders to extract value from LPs — a direct loss of LP principal matching the Metric OMM Allowed Impact Gate criterion for bad-price execution.

---

### Likelihood Explanation

- Pushers routinely sign consents with deadlines measured in days or weeks (the test suite uses `block.timestamp + 1 days` as the canonical example).
- The creator is the party who calls `allowPushers`; a creator who wants to retain a pusher's data stream has a direct economic incentive to replay the signature after every revocation.
- The attack requires no privileged role, no special token, and no off-chain infrastructure beyond holding the original `(deadline, sig)` tuple — which the creator already possesses from the initial delegation.

---

### Recommendation

Track consumed signatures with a `mapping(bytes32 => bool) private _usedConsentSigs` keyed on the signature hash (or the full digest). In `allowPushers`, after a successful `ECDSA.recover`, assert the digest has not been used and then mark it:

```solidity
bytes32 digest = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
);
require(!_usedConsentSigs[digest], SignatureAlreadyUsed());
require(pusher == ECDSA.recover(digest, signatures[i]));
_usedConsentSigs[digest] = true;
namespaceRemapping[pusher] = msg.sender;
```

Alternatively, include a per-pusher monotonic nonce in the signed message so each consent is single-use by construction.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

// Foundry test demonstrating that a creator can re-delegate a pusher
// who has already called revokePusher(), using the same signature.

function testCreatorTrapsRevokedPusherIndefinitely() public {
    uint256 deadline = block.timestamp + 30 days; // long-lived consent

    // 1. Pusher signs consent once.
    bytes memory sig = _signConsent(PUSHER_KEY, deadline, pusher, creator);

    address[] memory pushers = new address[](1);
    pushers[0] = pusher;
    bytes[] memory sigs = new bytes[](1);
    sigs[0] = sig;

    // 2. Creator establishes delegation.
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs);
    assertEq(oracle.namespaceRemapping(pusher), creator);

    // 3. Pusher revokes.
    vm.prank(pusher);
    oracle.revokePusher();
    assertEq(oracle.namespaceRemapping(pusher), address(0));

    // 4. Creator replays the SAME signature — delegation is restored.
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs); // no revert
    assertEq(oracle.namespaceRemapping(pusher), creator); // pusher is trapped again

    // 5. Pusher's fallback pushes now land in creator's namespace, not their own.
    uint56 tsMs = uint56(block.timestamp * 1000);
    uint48 raw = _packRaw(999_999, 1, 1);
    vm.prank(pusher);
    (bool ok,) = address(oracle).call(_wordAt(0, 0, raw, tsMs));
    assertTrue(ok);

    // Pusher's own namespace is still empty → stale price for any pool reading it.
    assertEq(oracle.getOracleData(oracle.feedIdOf(pusher, 0, 0)).price, 0);
}
```

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
