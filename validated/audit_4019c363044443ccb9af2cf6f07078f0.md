### Title
Consent Signature Replay in `allowPushers` Allows Creator to Re-establish Delegation After Pusher Self-Revokes — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowPushers` contains no nonce or used-signature tracking. A creator who holds a still-unexpired consent signature can call `allowPushers` a second time with the identical signature after the pusher has called `revokePusher()`, silently re-establishing delegation and redirecting the pusher's subsequent pushes into the creator's namespace without the pusher's current consent.

---

### Finding Description

`allowPushers` validates the pusher's consent by recovering the signer from an EIP-191 hash of `(block.chainid, address(this), deadline, pusher, msg.sender)` and checking `block.timestamp <= deadline`: [1](#0-0) 

The only liveness check is `_ensureDeadline`: [2](#0-1) 

There is no nonce, no per-pusher sequence counter, and no used-signature bitmap. The same `(deadline, sig)` pair passes all checks identically on every call as long as `block.timestamp <= deadline`.

`revokePusher` clears the mapping: [3](#0-2) 

But nothing prevents the creator from immediately calling `allowPushers` again with the same signature to overwrite the cleared mapping back to `creator`.

The code's own NatDoc at lines 189–191 acknowledges the deadline is the sole guard against post-revocation replay:

> *"an undated signature could re-establish a delegation AFTER the pusher revoked it"*

The deadline prevents replay **after** it expires, but not replay **within** the deadline window — which is exactly the post-revocation window the pusher cannot control. [4](#0-3) 

---

### Impact Explanation

After the creator replays the signature, `namespaceRemapping[pusher] = creator` is restored. The pusher's `fallback()` pushes are silently routed into the creator's namespace: [5](#0-4) 

The pusher's own namespace receives nothing. Any pool or provider reading the pusher's own feed ID gets stale/zero data. The creator's namespace receives price updates the pusher no longer consents to provide, which can be consumed by pools for pricing decisions. This is an admin-boundary break: an unprivileged creator bypasses the pusher's self-revocation, a core access-control invariant of the delegation system.

---

### Likelihood Explanation

The creator already holds the signature (they used it in the first `allowPushers` call). The replay requires only a single transaction before the deadline expires. Deadlines are typically set days in the future, giving the creator a large window. No special privilege or off-chain coordination is needed beyond what the creator already possesses.

---

### Recommendation

Track consumed signatures. The simplest fix is a per-pusher nonce that is incremented on each successful `allowPushers` and included in the signed digest, or a `mapping(bytes32 => bool) usedSignatures` that marks each digest as consumed on first use. Either approach makes the signature single-use and closes the replay window entirely.

function testReplayAfterRevoke() public {
    uint256 deadline = block.timestamp + 1 days;

    // 1. Pusher signs consent
    bytes memory sig = _signConsent(PUSHER_KEY, deadline, pusher, creator);

    address[] memory pushers = new address[](1);
    pushers[0] = pusher;
    bytes[] memory sigs = new bytes[](1);
    sigs[0] = sig;

    // 2. Creator establishes delegation
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs);
    assertEq(oracle.namespaceRemapping(pusher), creator);

    // 3. Pusher self-revokes
    vm.prank(pusher);
    oracle.revokePusher();
    assertEq(oracle.namespaceRemapping(pusher), address(0));

    // 4. Creator replays the SAME signature — still before deadline
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs); // should revert, but does not

    // 5. Delegation is re-established without pusher's current consent
    assertEq(oracle.namespaceRemapping(pusher), creator); // invariant violated
}

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
