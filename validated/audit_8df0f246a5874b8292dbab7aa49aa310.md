### Title
Creator Can Re-Establish Revoked Pusher Delegation Using Replayed Consent Signature, Allowing Bad Prices Into Pools — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1.allowPushers` does not check whether a pusher has previously self-revoked their delegation. Because the consent signature commits to no nonce and no revocation state, a creator can replay the same valid (deadline-not-yet-expired) signature to re-establish a delegation that the pusher intentionally terminated via `revokePusher()`. This is the direct analog of the external report's "missing abort-status check": a state transition (revocation) is not verified before a subsequent privileged action (re-delegation) is permitted.

---

### Finding Description

`allowPushers` requires a pusher's EIP-191 consent signature and a deadline: [1](#0-0) 

The signed message commits to `(chainid, oracle, deadline, pusher, creator)` — no nonce, no revocation counter: [2](#0-1) 

`revokePusher()` clears the mapping to `address(0)`: [3](#0-2) 

After revocation, `allowPushers` can be called again by the creator with the **identical** signature (same deadline, still in the future) and will unconditionally overwrite `namespaceRemapping[pusher]` back to the creator: [4](#0-3) 

The code's own comment acknowledges the concern but claims the deadline prevents it: [5](#0-4) 

This claim is incorrect. The deadline only prevents using an **expired** signature. Within the deadline window, the same signature is replayable an unlimited number of times, making the pusher's revocation non-final.

The `fallback()` push path resolves the effective namespace at call time: [6](#0-5) 

So any push made by the pusher after the creator re-establishes the delegation lands in the creator's namespace, not the pusher's own.

---

### Impact Explanation

The creator's namespace feeds are consumed by `AnchoredPriceProvider` via the attributed `price(feedId, pool)` path: [7](#0-6) 

`AnchoredPriceProvider` applies staleness, spread, and `priceGuard` checks, but if the bad price falls within those bounds it is accepted and used to compute the pool's bid/ask: [8](#0-7) 

Attack path:

1. Creator delegates pusher P with a 30-day deadline signature.
2. P's key is compromised; P calls `revokePusher()` to protect the creator's namespace.
3. Creator (unaware, or malicious) calls `allowPushers` again with the **same** signature — deadline still valid — re-establishing the delegation.
4. Attacker (holding P's compromised key) pushes a manipulated price into the creator's namespace via `fallback()`.
5. `AnchoredPriceProvider.getBidAndAskPrice()` reads the manipulated price, computes a skewed bid/ask, and the pool executes swaps at the wrong price — direct loss of user principal.

---

### Likelihood Explanation

- The deadline window is set by the creator and can be arbitrarily long (e.g., 30 days). Within that window, the replay is unconditionally valid.
- The creator need not be malicious: an honest creator who sees the pusher's revocation as a mistake will naturally call `allowPushers` again with the cached signature.
- The pusher has no on-chain mechanism to invalidate their prior consent short of waiting for the deadline to expire.
- No special privileges beyond being the creator of the feed are required.

---

### Recommendation

Include a per-pusher revocation nonce in the signed message and increment it on every `revokePusher` / `removePushers` call:

```diff
+ mapping(address => uint256) public pusherRevocationNonce;

  bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
-     keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
+     keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender, pusherRevocationNonce[pusher]))
  );

  // in revokePusher() and removePushers():
+ pusherRevocationNonce[pusher]++;
  namespaceRemapping[pusher] = address(0);
```

This makes every prior consent signature invalid after a revocation, matching the protocol's stated intent.

### Proof of Concept

```solidity
function test_revoked_pusher_redelegated_via_replayed_signature() public {
    uint256 PUSHER_KEY = 0xBEEF;
    address pusher = vm.addr(PUSHER_KEY);
    address creator = address(0xC0FFEE);

    uint256 deadline = block.timestamp + 30 days;

    // Pusher signs consent once
    bytes32 digest = MessageHashUtils.toEthSignedMessageHash(
        keccak256(abi.encode(block.chainid, address(oracle), deadline, pusher, creator))
    );
    (uint8 v, bytes32 r, bytes32 s) = vm.sign(PUSHER_KEY, digest);
    bytes memory sig = abi.encodePacked(r, s, v);

    address[] memory pushers = new address[](1);
    pushers[0] = pusher;
    bytes[] memory sigs = new bytes[](1);
    sigs[0] = sig;

    // Step 1: Creator establishes delegation
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs);
    assertEq(oracle.namespaceRemapping(pusher), creator);

    // Step 2: Pusher revokes (e.g., key compromised)
    vm.prank(pusher);
    oracle.revokePusher();
    assertEq(oracle.namespaceRemapping(pusher), address(0));

    // Step 3: Creator replays the SAME signature — deadline still valid
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs); // succeeds — no revert
    assertEq(oracle.namespaceRemapping(pusher), creator); // delegation restored

    // Step 4: Attacker (with compromised pusher key) pushes bad price into creator namespace
    uint56 tsMs = uint56(block.timestamp * 1000);
    uint48 badRaw = _packRaw(9_999_999, 4, 2); // manipulated price
    vm.prank(pusher); // attacker controls this key
    (bool ok,) = address(oracle).call(_wordAt(0, 0, badRaw, tsMs));
    assertTrue(ok);

    // Bad price now lives in creator's namespace, readable by AnchoredPriceProvider
    IOffchainOracle.OracleData memory data = oracle.getOracleData(oracle.feedIdOf(creator, 0, 0));
    assertEq(data.price, U64x32.decode(uint32(badRaw >> 16)));
}
```

```text
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();

            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
```

```text
    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
```
