### Title
Delegation Signature Replay in `allowPushers` Bypasses Pusher Revocation Within Deadline Window — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1::allowPushers` verifies a pusher's EIP-191 consent signature and enforces a deadline, but never marks the signature as consumed. Any caller who holds the original calldata can replay the same `(deadline, pusher, signature)` tuple an unlimited number of times before the deadline expires. This lets a creator re-establish a delegation that the pusher already revoked, permanently overriding `revokePusher()` for the entire deadline window and silently redirecting the pusher's fallback writes into the creator's namespace against the pusher's will.

---

### Finding Description

`allowPushers` is the only EOA-pusher delegation path in `CompressedOracleV1`:

```solidity
// CompressedOracle.sol L192-211
function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
    _ensureDeadline(deadline);                          // only checks block.timestamp <= deadline
    ...
    bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
        keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
    );
    require(pusher == ECDSA.recover(hash, signatures[i]));
    namespaceRemapping[pusher] = msg.sender;            // no "used" flag ever set
    ...
}
```

The code's own NatSpec acknowledges the deadline is the sole guard against post-revocation replay:

> *"The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it."*

But `_ensureDeadline` only checks `block.timestamp <= deadline`. It does **not** record that the signature was consumed. There is no `usedSignatures` mapping, no nonce, and no per-pusher revocation epoch anywhere in the contract. The `revokePusher` path clears `namespaceRemapping[msg.sender]` to `address(0)`, but that state write is immediately undone the next time the creator calls `allowPushers` with the identical, still-valid signature.

The analog to the MysteryBox bug is exact:
- MysteryBox: `fulfilled` is checked but never set → same VRF request can be re-fulfilled.
- CompressedOracle: deadline is checked but signature is never consumed → same delegation can be re-established after revocation.

---

### Impact Explanation

**Pusher's own namespace becomes permanently stale during the deadline window.**

After a pusher revokes and resumes pushing into their own namespace, the creator replays the signature. The fallback path resolves the namespace at call time:

```solidity
// CompressedOracle.sol L315-316
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
```

Once the remapping is restored, every subsequent fallback push from the pusher lands in the creator's namespace, not the pusher's. Any pool or price provider reading from the pusher's namespace (`feedIdOf(pusher, slotIndex, positionIndex)`) now receives a timestamp that never advances — the oracle data is frozen at the last value written before the hijack. Downstream `AnchoredPriceProvider` or pool swap calls that enforce `maxTimeDrift` will revert or return a stale quote, breaking swap and liquidity flows for any pool backed by the pusher's namespace.

The creator can repeat the replay on every block, making `revokePusher()` completely ineffective for the entire deadline window (which can be set to any future timestamp the pusher agreed to at signing time).

---

### Likelihood Explanation

- The creator already holds the exact calldata needed (they submitted it in the original `allowPushers` call, which is on-chain and trivially retrievable from logs or mempool history).
- No privileged role is required; `allowPushers` is a public function.
- The pusher's signature is valid until the deadline; a typical off-chain consent flow uses deadlines of hours to days.
- The pusher has no on-chain recourse other than waiting for the deadline to expire.

---

### Recommendation

Mark each `(pusher, creator, deadline)` tuple as consumed on first use:

```solidity
mapping(bytes32 => bool) private _usedDelegationSignatures;

function allowPushers(...) external {
    _ensureDeadline(deadline);
    ...
    bytes32 hash = ...; // existing digest
    require(pusher == ECDSA.recover(hash, signatures[i]));

    // NEW: prevent replay within the deadline window
    require(!_usedDelegationSignatures[hash], SignatureAlreadyUsed());
    _usedDelegationSignatures[hash] = true;

    namespaceRemapping[pusher] = msg.sender;
    ...
}
```

Alternatively, include a per-pusher nonce in the signed message so each consent is single-use by construction.

### Proof of Concept

```solidity
// 1. Pusher signs consent with deadline = now + 7 days
uint256 deadline = block.timestamp + 7 days;
bytes memory sig = _signConsent(PUSHER_KEY, deadline, pusher, creator);

// 2. Creator delegates pusher
vm.prank(creator);
oracle.allowPushers(deadline, _arr(pusher), _arr(sig));
assertEq(oracle.namespaceRemapping(pusher), creator);

// 3. Pusher revokes
vm.prank(pusher);
oracle.revokePusher();
assertEq(oracle.namespaceRemapping(pusher), address(0));

// 4. Creator replays the SAME signature — succeeds, revocation undone
vm.prank(creator);
oracle.allowPushers(deadline, _arr(pusher), _arr(sig));
assertEq(oracle.namespaceRemapping(pusher), creator); // pusher's revoke is nullified

// 5. Pusher's fallback pushes now land in creator's namespace, not pusher's own.
//    Pusher's own namespace timestamp never advances → stale oracle for any pool
//    reading feedIdOf(pusher, slotIndex, positionIndex).
```

    ///         signature is REQUIRED — without it anyone could remap a foreign pusher
    ///         wallet into their own namespace and silently swallow its pushes. The
    ///         deadline is likewise required: the signed consent carries no timestamp of
    ///         its own, so an undated signature could re-establish a delegation AFTER the
    ///         pusher revoked it.
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
