### Title
`revokePusher()` Revocation Is Silently Bypassed by Creator Replaying the Same EIP-191 Signature — (`File: smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`revokePusher()` clears `namespaceRemapping[pusher]` so the pusher's future fallback writes land in their own namespace instead of the creator's. However, `allowPushers` contains no nonce, no "used-signature" bitmap, and no check that the pusher has not already revoked. A creator can immediately replay the original EIP-191 consent signature (which remains cryptographically valid until its deadline expires) to re-establish the delegation, silently undoing the pusher's revocation. If the pusher revoked because their key was compromised, the creator's replay re-opens the creator's namespace to attacker-controlled price writes that downstream pools consume.

### Finding Description

`allowPushers` signs over `(block.chainid, address(this), deadline, pusher, msg.sender)`. [1](#0-0) 

After `revokePusher()` clears the mapping: [2](#0-1) 

the creator calls `allowPushers` again with the **identical** `(deadline, pusher, signature)` tuple. Because none of those four signed fields changed, `ECDSA.recover` returns the pusher's address, the `require` passes, and `namespaceRemapping[pusher]` is overwritten back to `msg.sender`: [3](#0-2) 

The contract's own comment acknowledges the replay risk but treats the deadline as the sole mitigation:

> *"The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it."*

The deadline prevents replay **after** it expires, but does nothing to prevent replay **within** the deadline window — which is exactly when revocation is most urgent (e.g., immediately after a key compromise is detected).

The fallback push path resolves the namespace at call time: [4](#0-3) 

So once the creator replays the delegation, every subsequent fallback push from the compromised key lands in the creator's namespace and is immediately readable by `price()` and `getOracleData()`, which price providers consume to compute bid/ask quotes for pools.

### Impact Explanation

A compromised pusher key can push arbitrary prices into the creator's namespace even after the legitimate pusher operator calls `revokePusher()`, provided the creator (possibly unaware of the compromise) replays the original `allowPushers` call. The creator's namespace is what `PriceProvider` / `ProtectedPriceProvider` read from via `getOracleData` / `price(feedId, pool)`. Pools executing swaps against those quotes receive attacker-controlled bid/ask prices, enabling bad-price execution: traders receive more than the oracle curve permits or the pool receives less than owed input, directly violating the swap conservation invariant. [5](#0-4) 

### Likelihood Explanation

The trigger is the creator calling `allowPushers` a second time — a normal operational action a creator might take if they believe the pusher accidentally self-revoked or if they are unaware of the compromise. The attacker needs only to wait for the creator to replay the delegation; no additional permissions are required. Deadlines are typically set days in the future (the test suite uses `block.timestamp + 1 days`), giving a wide replay window. [6](#0-5) 

### Recommendation

Track consumed signatures with a per-pusher nonce or a `mapping(bytes32 => bool) usedConsents` bitmap. Alternatively, include a monotonically increasing per-pusher nonce in the signed payload so that `revokePusher()` increments the nonce, invalidating all previously issued signatures for that pusher regardless of their deadline.

```solidity
// In allowPushers, after recovering the signer:
bytes32 consentKey = keccak256(signatures[i]);
require(!usedConsents[consentKey], ConsentAlreadyUsed());
usedConsents[consentKey] = true;
```

Or, simpler: store a per-pusher revocation nonce and include it in the signed message; `revokePusher()` increments it, making all prior signatures invalid.

### Proof of Concept

```solidity
// 1. Creator delegates pusher with deadline T+1day
uint256 deadline = block.timestamp + 1 days;
bytes memory sig = _signConsent(PUSHER_KEY, deadline, pusher, creator);
vm.prank(creator);
oracle.allowPushers(deadline, [pusher], [sig]);
// namespaceRemapping[pusher] == creator ✓

// 2. Pusher detects key compromise, self-revokes
vm.prank(pusher);
oracle.revokePusher();
// namespaceRemapping[pusher] == address(0) ✓

// 3. Creator (unaware of compromise) replays the SAME signature
vm.prank(creator);
oracle.allowPushers(deadline, [pusher], [sig]); // same sig, no revert
// namespaceRemapping[pusher] == creator again ✓

// 4. Attacker (holding compromised key) pushes bad price into creator's namespace
uint56 badTs = uint56(block.timestamp * 1000);
uint48 badRaw = _packRaw(999_999_999, 0, 0); // extreme price
vm.prank(pusher); // attacker controls this key
(bool ok,) = address(oracle).call(_wordAt(0, 0, badRaw, badTs));
assertTrue(ok);

// 5. Bad price is now in creator's namespace, readable by pools
IOffchainOracle.OracleData memory data =
    oracle.getOracleData(oracle.feedIdOf(creator, 0, 0));
assertEq(data.price, U64x32.decode(uint32(badRaw >> 16))); // attacker's price
```

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }

    function revokePusher() external {
        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
        namespaceRemapping[msg.sender] = address(0);
        emit PusherRevoked(msg.sender, creator);
