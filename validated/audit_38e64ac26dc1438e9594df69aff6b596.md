### Title
Pusher Consent Signature Has No Nonce — Creator Can Silently Re-Establish Delegation After Pusher Revokes, Enabling Bad-Price Injection Into Pools - (File: smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol)

---

### Summary

`allowPushers` signs consent over `(chainid, oracle, deadline, pusher, creator)` with no nonce or single-use flag. The same signature is valid for every call before the deadline. A pusher who calls `revokePusher()` to stop bad-price pushes can have their delegation silently re-established by the creator in the same block using the identical signature, making revocation ineffective until the deadline expires and allowing a compromised pusher key to write arbitrary prices into the creator's namespace.

---

### Finding Description

`allowPushers` in `CompressedOracleV1` verifies an EIP-191 signature over:

```solidity
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [1](#0-0) 

There is no nonce, no "used" flag, and no per-consent counter. The only expiry is the deadline. The same `(deadline, pusher, creator)` tuple produces the same hash every time, so the same `bytes` signature passes `ECDSA.recover` on every call until `block.timestamp > deadline`.

`revokePusher` clears the mapping:

```solidity
namespaceRemapping[msg.sender] = address(0);
``` [2](#0-1) 

But the creator can immediately call `allowPushers` again with the identical signature and restore `namespaceRemapping[pusher] = creator`. The code's own comment acknowledges the intent:

> *"The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it."* [3](#0-2) 

The deadline addresses the *post-deadline* replay but leaves the *pre-deadline* replay fully open. Revocation is therefore a no-op for the entire window `[revoke_time, deadline]`.

Once delegation is re-established, the `fallback` push path resolves the namespace from `namespaceRemapping[msg.sender]`:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [4](#0-3) 

Any push from the (now re-delegated) pusher address lands in the creator's namespace and overwrites the slot if the timestamp is newer than the stored one.

The `CompressedOracleV1.price()` function is the open read path consumed by `AnchoredPriceProvider._readLeg`:

```solidity
(mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);
``` [5](#0-4) 

The provider's band clamp is anchored on the oracle's own `mid`. A manipulated `mid` shifts the entire band, so the clamp does not protect against a compromised oracle mid price — it only prevents the *source* from quoting outside the band derived from that same mid.

---

### Impact Explanation

A compromised pusher key can write arbitrary prices (within timestamp-monotonicity and drift limits) into the creator's namespace. Those prices flow through `CompressedOracleV1.price()` → `AnchoredPriceProvider._getBidAndAskPrice()` → `MetricOmmPool.swap()`. The `AnchoredPriceProvider` band is computed as `mid ± (spreadBps + minMargin)` where `mid` comes from the manipulated oracle, so the band itself is shifted. Traders executing swaps against the pool receive output amounts computed from the manipulated bid/ask, causing direct loss of principal to LPs or to the counterparty of the swap.

---

### Likelihood Explanation

The attack requires two conditions: (1) the pusher's signing key is compromised, and (2) the creator re-establishes delegation — either unknowingly (routine maintenance) or because they are unaware of the compromise. Condition (2) is the limiting factor; however, the creator has no on-chain signal that the pusher revoked due to a key compromise versus a routine operational change, so re-establishment is a plausible mistake. The window is the full `[revoke_time, deadline]` interval, which can be up to the maximum deadline the pusher originally signed (e.g., 30 days). During that window the attacker can push at any time.

---

### Recommendation

Include a per-pusher nonce in the signed message and increment it on each successful `allowPushers` call (or on each `revokePusher`). This makes every consent single-use:

```solidity
mapping(address => uint256) public pusherNonce;

// In allowPushers:
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(
        block.chainid, address(this), deadline,
        pusher, msg.sender, pusherNonce[pusher]   // ← add nonce
    ))
);
pusherNonce[pusher]++;   // invalidate after use
```

Alternatively, record a `bytes32` digest of each used consent and revert on reuse.

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }

        uint256 end;
        uint256 namespace;

        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0)) creator = msg.sender;

        assembly ("memory-safe") {
            end := calldatasize()
            namespace := shl(96, creator) // [creator:20][zeros:12]

        // 4 * 6 + 7 + 1 = 32 bytes per slot
        if (end == 0 || end % 32 != 0) revert BadCalldataLength();

    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
