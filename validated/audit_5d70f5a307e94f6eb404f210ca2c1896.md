### Title
Pusher Self-Revocation Can Be Silently Undone by Creator via Consent-Signature Replay — (`File: smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`CompressedOracleV1.allowPushers` accepts a pusher's EIP-191 consent signature that commits to `(chainid, oracle, deadline, pusher, creator)`. There is no nonce, no revocation flag, and no on-chain record that the signature was consumed. After a pusher calls `revokePusher()` to clear their delegation, the creator can immediately replay the original consent signature — as long as the deadline has not expired — to silently re-establish the delegation. The pusher's revocation is therefore ineffective for the entire remaining lifetime of the deadline.

### Finding Description

`allowPushers` verifies the pusher's EIP-191 consent and writes `namespaceRemapping[pusher] = msg.sender`:

```solidity
// CompressedOracle.sol L192-211
function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
    _ensureDeadline(deadline);
    ...
    bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
        keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
    );
    require(pusher == ECDSA.recover(hash, signatures[i]));
    namespaceRemapping[pusher] = msg.sender;
    ...
}
``` [1](#0-0) 

`revokePusher` clears the mapping:

```solidity
// CompressedOracle.sol L238-243
function revokePusher() external {
    address creator = namespaceRemapping[msg.sender];
    if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
    namespaceRemapping[msg.sender] = address(0);
    emit PusherRevoked(msg.sender, creator);
}
``` [2](#0-1) 

The signed consent message contains no nonce and no "used" flag. The contract never records that a given `(deadline, pusher, creator)` tuple has been consumed. After `revokePusher()` sets `namespaceRemapping[pusher] = address(0)`, the creator can call `allowPushers` again with the **identical** signature and deadline, passing both `_ensureDeadline` and `ECDSA.recover`, and the mapping is restored. The protocol's own comment acknowledges the structural risk:

> "The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it." [3](#0-2) 

The deadline is the **only** guard, but it is chosen by the creator at call time, not by the pusher. A pusher who signs consent with `deadline = block.timestamp + 365 days` (a realistic production value) gives the creator a one-year replay window. The pusher has no mechanism to invalidate the signature short of waiting for the deadline to expire.

The `fallback` push path resolves the namespace at call time:

```solidity
// CompressedOracle.sol L315-316
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [4](#0-3) 

Once the delegation is re-established, every subsequent `fallback` push from the pusher's address lands in the creator's namespace, overwriting live feed slots consumed by `AnchoredPriceProvider`.

`AnchoredPriceProvider._readLeg` reads the compressed oracle's price and feeds it directly into the bid/ask band computation that pools consume during swaps:

```solidity
// AnchoredPriceProvider.sol L280
(mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);
``` [5](#0-4) 

### Impact Explanation

A pusher whose signing key is compromised calls `revokePusher()` to stop the attacker. The creator — unaware of the compromise, or acting negligently — replays the original consent signature. The attacker's pushes resume, writing attacker-controlled prices into the creator's feed slots. `AnchoredPriceProvider` reads those prices, computes bid/ask from them, and the pool executes swaps at the corrupted quote. Traders receive incorrect amounts; the pool's token balances diverge from what the oracle-anchored invariant requires. This is a direct, quantifiable loss of user principal matching the "bad-price execution" impact gate.

### Likelihood Explanation

The trigger requires the creator to call `allowPushers` a second time with the same signature. This can happen:
- **Negligently**: the creator's key-management tooling re-submits the delegation after detecting the cleared mapping, without knowing the pusher revoked intentionally.
- **Maliciously**: a creator who wants to force a pusher to remain delegated (e.g., to keep a compromised key active).

The pusher has no on-chain mechanism to prevent either scenario before the deadline expires. The consent signature is a public transaction once submitted; anyone who observed the original `allowPushers` call can extract it from calldata.

### Recommendation

Track consumed consent signatures with a `mapping(bytes32 => bool) usedConsentHashes` keyed on `keccak256(abi.encode(chainid, oracle, deadline, pusher, creator))`. Mark the hash as used on first acceptance and revert on replay. Alternatively, include a monotonically increasing per-pusher nonce in the signed message and store the last accepted nonce, so a revoked pusher can increment their nonce to invalidate all prior signatures.

### Proof of Concept

```
1. Pusher signs: sig = EIP191(keccak256(abi.encode(chainid, oracle, deadline=T+365d, pusher, creator)))
2. Creator calls allowPushers(T+365d, [pusher], [sig])
   → namespaceRemapping[pusher] = creator  ✓
3. Pusher's key is compromised; pusher calls revokePusher()
   → namespaceRemapping[pusher] = address(0)  ✓
4. Creator (or attacker with calldata from step 2) calls allowPushers(T+365d, [pusher], [sig]) again
   → _ensureDeadline passes (deadline still future)
   → ECDSA.recover returns pusher  ✓
   → namespaceRemapping[pusher] = creator  ← delegation silently restored
5. Attacker pushes crafted slot word via fallback():
   word = pack(price=0, s0=0, s1=0, ts=now_ms, slotId=X)
   → stored in creator's namespace slot X
6. AnchoredPriceProvider._readLeg reads price=0 → ok=false → getBidAndAskPrice returns (0, type(uint128).max)
   → pool swap reverts with FeedStalled, or if price is non-zero but manipulated, swap executes at attacker price
```

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
