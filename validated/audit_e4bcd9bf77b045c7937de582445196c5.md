### Title
`revokePusher()` Bypass via Creator-Replayed Consent Signature Within Deadline Window — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowPushers()` stores no nonce and marks no signature as used. After a pusher calls `revokePusher()` to clear their delegation, the creator can immediately replay the original EIP-191 consent signature — with the same `deadline` — to re-establish the mapping. The pusher's revocation is therefore ineffective for the entire remaining lifetime of the deadline, allowing a compromised pusher key to continue writing arbitrary prices into the creator's namespace and into live pool swaps.

---

### Finding Description

`allowPushers` verifies a pusher's EIP-191 consent and writes `namespaceRemapping[pusher] = msg.sender`:

```solidity
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
);
require(pusher == ECDSA.recover(hash, signatures[i]));
namespaceRemapping[pusher] = msg.sender;
``` [1](#0-0) 

The signed tuple is `(chainid, oracle, deadline, pusher, creator)`. There is no nonce, no per-pusher counter, and no "signature consumed" mapping. The same bytes can be submitted to `allowPushers` an unlimited number of times before `block.timestamp > deadline`.

`revokePusher` clears the mapping:

```solidity
namespaceRemapping[msg.sender] = address(0);
``` [2](#0-1) 

But because `allowPushers` performs no used-signature check, the creator can call it again immediately with the identical `(deadline, [pusher], [sig])` arguments, restoring `namespaceRemapping[pusher] = creator`. The code comment itself acknowledges the replay risk but treats the deadline as the sole mitigation:

> "The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it." [3](#0-2) 

The deadline prevents replay only after it expires. Within the window it provides no protection at all.

Once re-delegated, the pusher's `fallback()` writes land in the creator's namespace:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
// ...
bytes32 key = bytes32(namespace | uint256(slotId));
// ...
_writeStorage(key, bytes32(bytes32(word & ~uint256(0xff))));
``` [4](#0-3) 

Those writes are then read by `getOracleData` → `_price` → `AnchoredPriceProvider._readLeg` → `_computeBidAsk` → `getBidAndAskPrice`, which feeds the bid/ask directly into `MetricOmmPool.swap`. [5](#0-4) 

---

### Impact Explanation

A pusher whose private key is compromised calls `revokePusher()` as an emergency measure. The creator — whether malicious or simply unaware of the compromise — replays the original consent signature before the deadline expires. The compromised key is re-delegated and can push arbitrary `(price, spread0, spread1)` tuples into the creator's namespace. `AnchoredPriceProvider` reads those values, passes them through staleness and spread guards (which the attacker satisfies by supplying a fresh timestamp and a valid spread index), and delivers a manipulated bid/ask to every pool swap that uses this feed. Traders receive more output than the true oracle price permits, draining pool reserves.

---

### Likelihood Explanation

The deadline is typically set days in the future (the test suite uses `block.timestamp + 1 days`). Any creator who holds the original signature bytes — which are public calldata from the first `allowPushers` transaction — can replay them at any time within that window. No special privilege is required beyond being the creator (namespace owner), which is a semi-trusted, non-admin role explicitly in scope per the delegation-path audit pivot. [6](#0-5) 

---

### Recommendation

Track consumed signatures with a `mapping(bytes32 => bool) private _usedSignatures` keyed on the signature hash (or the consent digest), and revert if the same digest is submitted twice. Alternatively, include a per-pusher monotonic nonce in the signed payload so each consent is single-use by construction.

---

### Proof of Concept

```
1. Pusher signs consent:
   digest = keccak256(abi.encode(chainid, oracle, deadline=T+1day, pusher, creator))
   sig = sign(pusherKey, digest)

2. Creator calls allowPushers(T+1day, [pusher], [sig])
   → namespaceRemapping[pusher] = creator  ✓

3. Pusher's key is compromised; pusher calls revokePusher()
   → namespaceRemapping[pusher] = address(0)  ✓ (revoked)

4. Creator (or attacker with the public calldata) calls
   allowPushers(T+1day, [pusher], [sig])  ← SAME sig, deadline not yet expired
   → namespaceRemapping[pusher] = creator  ✗ (re-established without fresh consent)

5. Compromised pusher key calls fallback() with:
   word = (arbitraryPrice << 208+16) | (validSpreadIdx << 208+8) | (validSpreadIdx << 208)
         | (freshTimestampMs << 8) | slotId
   → _writeStorage(creatorNamespace | slotId, word)  ← bad price in creator namespace

6. AnchoredPriceProvider.getBidAndAskPrice()
   → _readLeg(baseFeedId) → oracle.price(feedId, pool)
   → getOracleData returns manipulated mid/spread
   → _computeBidAsk delivers wrong bid/ask to MetricOmmPool.swap
   → trader extracts excess output at pool's expense
```

    function _getBidAndAskPrice() internal returns (uint128, uint128) {
        (uint256 mid, uint256 spreadBps, , bool ok) = _readLeg(baseFeedId);
        if (!ok) return (0, type(uint128).max);

        bytes32 _quote = quoteFeedId;
        if (_quote != bytes32(0)) {
            (uint256 mid2, uint256 spreadBps2, , bool ok2) = _readLeg(_quote);
            if (!ok2 || mid2 == 0) return (0, type(uint128).max);
            // Synthetic ratio (8-decimal): mid1 / mid2. Relative uncertainties of a ratio add.
            mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);
            spreadBps += spreadBps2;
        }

        return _computeBidAsk(mid, spreadBps);
    }

    /// @dev Reads one feed and runs its per-leg guards. ok=false (→ caller halts, fail closed) on:
    ///      stale reference, mid == 0, spreadBps == the off-hours/stall marker (spreadBps >= ORACLE_BPS), or a
    ///      priceGuard violation. Each leg is read through the attributed path independently.
    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
