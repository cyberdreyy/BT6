### Title
Replayable Pusher-Consent Signature Lets Creator Nullify a Revoked Delegation — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowPushers` verifies an EIP-191 signature but never marks it as consumed. A creator who retains a pusher's consent signature can call `allowPushers` again after the pusher has explicitly revoked via `revokePusher()`, silently re-establishing the delegation. The code's own comment acknowledges the deadline is the sole guard against this, but the deadline only blocks *expired* signatures — it does nothing to prevent a still-valid signature from being replayed.

---

### Finding Description

`allowPushers` constructs and verifies the following hash:

```solidity
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
);
require(pusher == ECDSA.recover(hash, signatures[i]));
namespaceRemapping[pusher] = msg.sender;
``` [1](#0-0) 

The signed message contains no nonce and no per-use marker. After a successful call, the signature is not recorded anywhere. The only freshness gate is `_ensureDeadline(deadline)`, which rejects signatures whose deadline has already passed. [2](#0-1) 

`revokePusher` zeroes the mapping entry:

```solidity
function revokePusher() external {
    address creator = namespaceRemapping[msg.sender];
    if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
    namespaceRemapping[msg.sender] = address(0);
    emit PusherRevoked(msg.sender, creator);
}
``` [3](#0-2) 

Because the original signature is still cryptographically valid (deadline not yet expired), the creator can immediately call `allowPushers` again with the identical `(deadline, pusher, signature)` tuple, writing `namespaceRemapping[pusher] = creator` back into storage. The revocation is silently undone.

The code's own NatSpec acknowledges the deadline is the intended protection:

> *"the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it."* [4](#0-3) 

The deadline prevents *expired* signatures from being replayed, but it does not prevent a *live* signature from being replayed an unlimited number of times within the deadline window. The protection is therefore incomplete.

---

### Impact Explanation

The `fallback` push path resolves the effective namespace by reading `namespaceRemapping[msg.sender]`:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [5](#0-4) 

When the delegation is re-established against the pusher's will, every subsequent push from that pusher's EOA is written into the **creator's** namespace, not the pusher's own. Two concrete harms follow:

1. **Pusher's own feeds are corrupted.** If the pusher revoked in order to start pushing into their own namespace (for their own pools), those updates are silently hijacked into the creator's namespace. The pusher's own `feedId`s receive no updates and appear stale to consumers.

2. **Compromised-key scenario feeds bad prices to pools.** If the pusher revoked because their signing key was compromised, the creator can re-establish the delegation, allowing the attacker controlling the compromised key to continue writing price data into the creator's slots. Those slots are consumed by `PriceProvider` / `AnchoredPriceProvider` during pool swaps, potentially delivering manipulated bid/ask prices to the pool. [6](#0-5) 

---

### Likelihood Explanation

- The creator must have retained the original signature bytes — trivially true since they submitted it on-chain and it is visible in calldata/logs.
- The deadline must not yet have expired. Pushers are expected to sign with generous deadlines (the system is designed for long-running delegation relationships), so the window is typically large.
- No special privilege is required; the creator is a normal user of the system.

---

### Recommendation

Record each signature hash as consumed on first use and reject any replay:

```solidity
mapping(bytes32 => bool) public usedDelegationSignatures;

function allowPushers(...) external {
    _ensureDeadline(deadline);
    ...
    bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
        keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
    );
    require(!usedDelegationSignatures[hash], SignatureAlreadyUsed());
    require(pusher == ECDSA.recover(hash, signatures[i]));
    usedDelegationSignatures[hash] = true;
    namespaceRemapping[pusher] = msg.sender;
    emit PusherAuthorized(pusher, msg.sender);
}
```

This ensures each signed consent can establish the delegation exactly once, making `revokePusher` irrevocable by the creator for that signature.

### Proof of Concept

```solidity
// 1. Pusher signs consent with a 30-day deadline (off-chain).
//    sig = sign(keccak256(abi.encode(chainid, oracle, deadline, pusher, creator)))

// 2. Creator establishes delegation.
vm.prank(creator);
oracle.allowPushers(deadline, [pusher], [sig]);
assertEq(oracle.namespaceRemapping(pusher), creator); // delegation active

// 3. Pusher revokes.
vm.prank(pusher);
oracle.revokePusher();
assertEq(oracle.namespaceRemapping(pusher), address(0)); // revoked

// 4. Creator replays the SAME signature — deadline still valid.
vm.prank(creator);
oracle.allowPushers(deadline, [pusher], [sig]); // succeeds, no revert
assertEq(oracle.namespaceRemapping(pusher), creator); // delegation silently restored

// 5. Pusher's subsequent fallback() pushes now write into creator's namespace,
//    not the pusher's own — revocation is nullified.
```

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }

    /// @notice Push path. Calldata = N × 32-byte slot words:
    ///         [data0:6][data1:6][data2:6][data3:6][ts:7][slotId:1]
    ///         The sender pushes into `namespaceRemapping[msg.sender]`, falling back to
    ///         its OWN namespace — a creator needs zero setup transactions to start
    ///         pushing. Each word carries its own timestamp (monotonicity is the only
    ///         freshness gate), so there is no deadline prefix.
    fallback() override external {
        uint256 end;
        uint256 namespace;

        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0)) creator = msg.sender;

        assembly ("memory-safe") {
            end := calldatasize()
            namespace := shl(96, creator) // [creator:20][zeros:12]
        }

        // 4 * 6 + 7 + 1 = 32 bytes per slot
        if (end == 0 || end % 32 != 0) revert BadCalldataLength();

        for (uint256 ptr = 0; ptr < end; ptr += 32) {
            uint256 word;
            assembly ("memory-safe") {
                word := calldataload(ptr)
            }
            // casting to 'uint8' is safe we want LSB
            // forge-lint: disable-next-line(unsafe-typecast)
            uint8 slotId = uint8(word);
            TimeMs timestampMs = toTimeMs(word >> 8 & X56);
            timestampMs.revertIfAfterBlockTimeWithDrift(MAX_TIME_DRIFT);
            bytes32 key = bytes32(namespace | uint256(slotId));
            uint256 old = uint256(_loadStorage(key));
            TimeMs oldTimestampMs = toTimeMs(old >> 8 & X56);

            bool newer = timestampMs.isAfter(oldTimestampMs);
            if (!newer) continue;

            _writeStorage(key, bytes32(bytes32(word & ~uint256(0xff))));
        }
