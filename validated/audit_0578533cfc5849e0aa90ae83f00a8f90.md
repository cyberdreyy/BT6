### Title
`allowPushers` consent signature is replayable within the deadline window, making `revokePusher()` ineffective — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

The `allowPushers` function uses a deadline as the only replay protection for pusher consent signatures. The deadline only prevents replay *after* it expires. Within the deadline window, the same consent signature can be replayed an unlimited number of times by the creator. A pusher's revocation via `revokePusher()` can therefore be silently undone by the creator replaying the original consent, re-authorizing a compromised pusher wallet to push bad prices into the creator's namespace.

### Finding Description

`allowPushers` verifies a pusher's EIP-191 consent over:

```
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [1](#0-0) 

The code comment explicitly acknowledges the replay concern:

> "The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it." [2](#0-1) 

However, the deadline only prevents replay **after it expires**. There is no nonce, no used-signature bitmap, and no per-pusher counter. Within the deadline window the identical `(deadline, pusher, creator)` tuple produces the identical hash, so the same `bytes` signature passes `ECDSA.recover` every time.

`revokePusher()` clears `namespaceRemapping[msg.sender]` to `address(0)`: [3](#0-2) 

But `allowPushers` unconditionally overwrites it back to `msg.sender` (the creator) on every successful call: [4](#0-3) 

**Attack path:**

1. Pusher signs consent with `deadline = block.timestamp + 30 days`.
2. Creator calls `allowPushers` → `namespaceRemapping[pusher] = creator`.
3. Pusher's wallet is compromised; pusher calls `revokePusher()` → `namespaceRemapping[pusher] = address(0)`.
4. Creator (in good faith, or colluding) calls `allowPushers` again with the **same** consent bytes — deadline still valid → `namespaceRemapping[pusher] = creator` again.
5. Attacker (holding the pusher's key) calls `fallback()` with an extreme price word; it lands in the **creator's** namespace.
6. Any pool whose `IPriceProvider` reads that feed executes swaps at the attacker-controlled price.

The `fallback()` push path has no additional authorization check beyond `namespaceRemapping`: [5](#0-4) 

### Impact Explanation

Once the delegation is re-established, the attacker can push an arbitrary `U64x32`-encoded price and arbitrary `Codebook256` spread indices into the creator's namespace. Any `AnchoredPriceProvider` or pool that reads that feed will receive the attacker-controlled bid/ask, causing swaps to execute at a fabricated price — direct loss of trader principal and LP assets.

### Likelihood Explanation

Medium. The pusher must have signed a consent with a non-trivial deadline (common for operational convenience). The creator must replay the signature — either inadvertently (thinking the pusher accidentally revoked) or deliberately. No privileged protocol role is required; the creator is an ordinary user address.

### Recommendation

Add a per-pusher nonce to the signed digest and increment it on every successful `allowPushers` call:

```solidity
mapping(address => uint256) public pusherNonce;

// inside allowPushers, replace the hash with:
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(
        block.chainid, address(this), deadline,
        pusher, msg.sender, pusherNonce[pusher]++
    ))
);
```

This makes every consent single-use: after the first `allowPushers` call the nonce advances, so the same signature cannot satisfy the new hash even if the deadline has not expired.

### Proof of Concept

```solidity
// 1. Pusher signs consent with a 30-day deadline
uint256 deadline = block.timestamp + 30 days;
bytes32 digest = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(oracle), deadline, pusher, creator))
);
(uint8 v, bytes32 r, bytes32 s) = vm.sign(PUSHER_KEY, digest);
bytes memory sig = abi.encodePacked(r, s, v);

// 2. Creator establishes delegation
vm.prank(creator);
oracle.allowPushers(deadline, _pushers(pusher), _sigs(sig));
assertEq(oracle.namespaceRemapping(pusher), creator);

// 3. Pusher revokes (e.g. wallet compromise detected)
vm.prank(pusher);
oracle.revokePusher();
assertEq(oracle.namespaceRemapping(pusher), address(0));

// 4. Creator replays the IDENTICAL consent bytes — still within deadline
vm.prank(creator);
oracle.allowPushers(deadline, _pushers(pusher), _sigs(sig)); // ← succeeds, no revert
assertEq(oracle.namespaceRemapping(pusher), creator);        // ← delegation restored

// 5. Attacker (holding pusher key) pushes an extreme price into creator's namespace
uint56 tsMs = uint56(block.timestamp * 1000);
uint48 badRaw = _packRaw(9_999_999, 0, 0);
vm.prank(pusher);                                            // attacker controls this key
(bool ok,) = address(oracle).call(_wordAt(0, 0, badRaw, tsMs));
assertTrue(ok);                                              // bad price accepted
// oracle.getOracleData(oracle.feedIdOf(creator, 0, 0)).price == U64x32.decode(9_999_999)
```
