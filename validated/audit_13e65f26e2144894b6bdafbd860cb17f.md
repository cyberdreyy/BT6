### Title
Creator can replay pusher consent signature to re-establish delegation after `revokePusher()` — (File: smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol)

### Summary

`allowPushers` signs consent over `(chainid, oracle, deadline, pusher, creator)` with no nonce or revocation counter. After a pusher calls `revokePusher()`, the creator can immediately replay the original signature — if the deadline has not yet expired — to re-establish the delegation. The pusher's self-revocation is silently undone.

### Finding Description

`revokePusher()` clears `namespaceRemapping[msg.sender]` to `address(0)`: [1](#0-0) 

`allowPushers` verifies the pusher's EIP-191 signature and unconditionally writes `namespaceRemapping[pusher] = msg.sender`: [2](#0-1) 

The signed message is `keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))`. There is **no nonce, no revocation counter, and no "already-used" flag**. The only replay guard is the deadline. Within the deadline window the exact same `(deadline, pusher, sig)` triple is accepted an unlimited number of times.

The code comment acknowledges the risk but frames the deadline as the fix: [3](#0-2) 

The deadline prevents replay *after* it expires, but does **not** prevent replay *before* it expires. A pusher who signed a 1-year consent and revokes after 1 day is still bound for the remaining 364 days: the creator can re-establish the delegation in the same block as the revocation.

This is the direct analog of the withdrawal-queue bug: just as the queue checkpoint did not account for withdrawal size and allowed smaller withdrawals to bypass priority, the delegation checkpoint (`namespaceRemapping`) does not account for revocation events and allows the creator to bypass the pusher's self-protection.

### Impact Explanation

After the creator replays the signature:

1. The pusher's `fallback` pushes are routed to the **creator's namespace** again (via `namespaceRemapping[pusher] = creator`).
2. The pusher's **own namespace** receives no new data.
3. Any pool whose `AnchoredPriceProvider` is bound to a `feedIdOf(pusher, slotId, posId)` feed will read a stale timestamp, causing `_isStale` to return `true` and `getBidAndAskPrice` to revert with `FeedStalled`. [4](#0-3) 

The pool becomes permanently unusable for swaps until the pusher either re-signs a new consent (giving the creator what they want) or the deadline expires. This is a broken core pool functionality impact: swap flows are blocked for the duration of the deadline window.

### Likelihood Explanation

- The creator is a semi-trusted party who controls the namespace and has the original signature in hand.
- The pusher's only recourse is to stop pushing entirely, which itself causes the pool to stall.
- Any pusher who signed a long-lived deadline (days to months, which is operationally normal) is exposed for the full remaining window.
- No privileged role or special setup is required beyond the creator retaining the original calldata.

### Recommendation

Add a per-pusher revocation nonce to the signed message:

```solidity
mapping(address => uint256) public pusherNonce;

// In allowPushers:
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(
        block.chainid, address(this), deadline,
        pusher, msg.sender,
        pusherNonce[pusher]   // <-- bind to current nonce
    ))
);

// In revokePusher:
namespaceRemapping[msg.sender] = address(0);
pusherNonce[msg.sender]++;     // <-- invalidate all prior signatures
```

This ensures that once a pusher revokes, every previously issued signature is permanently invalid regardless of its deadline.

### Proof of Concept

```
1. Pusher signs: keccak256(abi.encode(chainid, oracle, deadline=T+365days, pusher, creator))
   → sig

2. creator calls allowPushers(T+365days, [pusher], [sig])
   → namespaceRemapping[pusher] = creator  ✓

3. pusher calls revokePusher()
   → namespaceRemapping[pusher] = address(0)  ✓

4. creator calls allowPushers(T+365days, [pusher], [sig])  // same args, same sig
   → _ensureDeadline passes (deadline still in future)
   → ECDSA.recover returns pusher  ✓
   → namespaceRemapping[pusher] = creator  ← revocation silently undone

5. pusher's subsequent fallback pushes land in creator's namespace, not pusher's own.
   Pool bound to feedIdOf(pusher, ...) reads stale data → FeedStalled on every swap.
``` [5](#0-4)

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
```

    function revokePusher() external {
        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
        namespaceRemapping[msg.sender] = address(0);
        emit PusherRevoked(msg.sender, creator);
    }

    fallback() override external {
        uint256 end;
        uint256 namespace;

        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0)) creator = msg.sender;

        assembly ("memory-safe") {
            end := calldatasize()
            namespace := shl(96, creator) // [creator:20][zeros:12]
        }

        // 4 * 6 + 7 + 1 = 32 bytes per slot
        if (end == 0 || end % 32 != 0) revert BadCalldataLength();

        for (uint256 ptr = 0; ptr < end; ptr += 32) {
            uint256 word;
            assembly ("memory-safe") {
                word := calldataload(ptr)
            }
            // casting to 'uint8' is safe we want LSB
            // forge-lint: disable-next-line(unsafe-typecast)
            uint8 slotId = uint8(word);
            TimeMs timestampMs = toTimeMs(word >> 8 & X56);
            timestampMs.revertIfAfterBlockTimeWithDrift(MAX_TIME_DRIFT);

            uint256 old = uint256(_loadStorage(key));
            TimeMs oldTimestampMs = toTimeMs(old >> 8 & X56);

            bool newer = timestampMs.isAfter(oldTimestampMs);
            if (!newer) continue;

            _writeStorage(key, bytes32(bytes32(word & ~uint256(0xff))));
