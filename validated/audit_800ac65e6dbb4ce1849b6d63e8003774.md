### Title
`allowPushers` Deadline-Replay Allows Creator to Silently Re-Establish Revoked Pusher Delegation — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`allowPushers` has no nonce, no used-signature set, and no check for whether the pusher has already revoked. The only replay guard is the deadline. A creator who holds a still-valid signed consent can call `allowPushers` again after the pusher has called `revokePusher()`, re-writing `namespaceRemapping[pusher] = creator` without the pusher's current consent.

### Finding Description

`allowPushers` performs exactly three checks before writing `namespaceRemapping[pusher] = msg.sender`:

1. `_ensureDeadline(deadline)` — `block.timestamp <= deadline` only.
2. Signature validity over `(chainid, oracle, deadline, pusher, creator)`.
3. `pusher != msg.sender`. [1](#0-0) 

There is no nonce, no consumed-signature bitmap, and no check that `namespaceRemapping[pusher]` is currently zero (i.e., that the pusher has not already revoked). `revokePusher` simply sets `namespaceRemapping[msg.sender] = address(0)`: [2](#0-1) 

`_ensureDeadline` is `require(block.timestamp <= deadline)` — nothing more: [3](#0-2) 

The NatSpec comment on `allowPushers` explicitly acknowledges the deadline is the intended guard against post-revocation replay ("an undated signature could re-establish a delegation AFTER the pusher revoked it"), but the implementation only prevents using an *expired* signature — it does not prevent replaying a still-valid one after revocation: [4](#0-3) 

### Impact Explanation

The `fallback` push path resolves the effective namespace as:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [5](#0-4) 

After the creator replays the consent, `namespaceRemapping[pusher] = creator` again. Any subsequent push by the pusher (who believes they are now writing to their own namespace) is silently redirected into the creator's namespace. The pusher's own namespace (`feedIdOf(pusher, ...)`) receives no update and goes stale. Any pool or provider reading from `feedIdOf(pusher, ...)` will consume a stale or zero price, which constitutes bad-price execution under the contest's impact gate.

### Likelihood Explanation

The creator must be adversarial and must have retained the original signed consent bytes. The pusher must push again after revoking (believing they are writing to their own namespace). Both conditions are realistic in a production oracle setup where pushers rotate or change affiliations. The attack requires no privileged role — `allowPushers` is a public, permissionless function callable by any `msg.sender` who holds a valid signature.

### Recommendation

Track consumed consents. The simplest fix is a `mapping(bytes32 => bool) public usedConsents` keyed on the signature hash (or on `keccak256(abi.encode(deadline, pusher, creator))`), and revert if the hash has already been used. Alternatively, add a per-pusher revocation nonce that is incremented on `revokePusher` and included in the signed message, so any prior signature becomes invalid after revocation.

### Proof of Concept

```solidity
// Foundry test (pseudocode)
uint256 deadline = block.timestamp + 1 days;
bytes memory sig = _signConsent(PUSHER_KEY, deadline, pusher, creator);

// Step 1: creator establishes delegation
vm.prank(creator);
oracle.allowPushers(deadline, _arr(pusher), _arr(sig));
assertEq(oracle.namespaceRemapping(pusher), creator);

// Step 2: pusher revokes
vm.prank(pusher);
oracle.revokePusher();
assertEq(oracle.namespaceRemapping(pusher), address(0));

// Step 3: creator replays the SAME signature before deadline
vm.prank(creator);
oracle.allowPushers(deadline, _arr(pusher), _arr(sig)); // succeeds — no guard
assertEq(oracle.namespaceRemapping(pusher), creator);   // delegation silently restored

// Step 4: pusher pushes, believing they write to own namespace
vm.prank(pusher);
(bool ok,) = address(oracle).call(_wordAt(slotId, pos, raw, tsMs));
assertTrue(ok);

// Pusher's own namespace is empty; creator's namespace received the push
assertEq(oracle.getOracleData(oracle.feedIdOf(pusher, slotId, pos)).price, 0);
assertEq(oracle.getOracleData(oracle.feedIdOf(creator, slotId, pos)).price, expectedPrice);
```

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
