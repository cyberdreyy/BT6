### Title
Pusher consent signature replay within deadline window allows creator to override `revokePusher()`, enabling resumed bad-price injection into pools - (File: `smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`allowPushers` signs consent over `(chainid, oracle, deadline, pusher, creator)` but tracks no per-pusher nonce and maintains no used-signature bitmap. The `namespaceRemapping[pusher]` slot is unconditionally **assigned** on every call. A creator who retains the original consent bytes can replay them any number of times while the deadline is still valid, silently overwriting a pusher's `revokePusher()` call. This is the direct structural analog of the external "assign instead of accumulate" bug: the revocation write (`address(0)`) is overwritten by the replayed delegation write (`creator`), exactly as `vaultWithdraw.userStrategyShares` was overwritten instead of accumulated.

### Finding Description [1](#0-0) 

The code comment at lines 189–191 explicitly acknowledges the risk:

> *"the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it"*

The deadline is stated as the guard. But the deadline only blocks replay **after it expires**; within the window it provides zero protection. There is no nonce, no `usedConsents` mapping, and no check that `namespaceRemapping[pusher]` is currently `address(0)` before writing. [2](#0-1) 

After a pusher calls `revokePusher()`: [3](#0-2) 

…the creator calls `allowPushers` again with the **identical** `(deadline, [pusher], [sig])` tuple. The signature is still cryptographically valid, the deadline is still in the future, and line 209 unconditionally overwrites the revocation:

```
namespaceRemapping[pusher] = msg.sender;   // ← overwrites address(0) set by revokePusher
```

The fallback push path then resolves the namespace from this restored mapping: [4](#0-3) 

So any subsequent push from the pusher's address lands in the creator's namespace again, as if the revocation never happened.

### Impact Explanation

If a pusher's private key is compromised, the pusher calls `revokePusher()` as an emergency stop. The creator, unaware of the compromise, replays the original consent signature to restore the feed pipeline. The attacker controlling the compromised key resumes pushing manipulated slot words into the creator's namespace. Those prices flow through `AnchoredPriceProvider._readLeg` → `IPricedOracle.price(feedId, pool)` into live pool swaps: [5](#0-4) 

Traders receive wrong bid/ask quotes on every swap until the creator manually calls `removePushers`, causing direct loss of user principal. The `AnchoredPriceProvider` band clamp limits how far the price can deviate, but within the band the attacker can still push the worst-allowed price on every update.

### Likelihood Explanation

Creators routinely store consent signatures off-chain for operational use (re-delegation after key rotation). Deadlines are commonly set days or weeks in the future. A compromised pusher key is a realistic operational incident. The creator replaying the signature is a plausible mistake when the compromise is not immediately communicated to the creator. No privileged role is required beyond the creator's own namespace authority.

### Recommendation

Track consumed consent signatures with a `mapping(bytes32 => bool) usedConsents` keyed on the full signature hash, or add a per-pusher revocation nonce incremented on each `revokePusher` / `removePushers` call and include it in the signed message. Either approach ensures a revoked pusher's original consent cannot be replayed within the deadline window.

### Proof of Concept

```
1. Pusher P signs:
     keccak256(abi.encode(chainid, oracle, deadline=T+7days, P, creator))
   → sig

2. Creator calls allowPushers(T+7days, [P], [sig])
   → namespaceRemapping[P] = creator  ✓

3. P's key is compromised; P calls revokePusher()
   → namespaceRemapping[P] = address(0)  ✓  (emergency stop)

4. Creator calls allowPushers(T+7days, [P], [sig])   ← SAME sig, deadline still valid
   → namespaceRemapping[P] = creator                 ← revocation silently overwritten

5. Attacker (holding P's key) calls oracle.fallback with a manipulated price word
   → fallback resolves namespaceRemapping[attacker_addr] = creator
   → price lands in creator's namespace

6. AnchoredPriceProvider._readLeg reads manipulated price
   → pool swap executes at wrong bid/ask
   → trader loses principal
```

    /// @notice Delegates pusher wallets into the caller's namespace. The pusher's EIP-191
    ///         signature is REQUIRED — without it anyone could remap a foreign pusher
    ///         wallet into their own namespace and silently swallow its pushes. The
    ///         deadline is likewise required: the signed consent carries no timestamp of
    ///         its own, so an undated signature could re-establish a delegation AFTER the
    ///         pusher revoked it.
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }

    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
