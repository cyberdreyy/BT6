### Title
Missing `baseFeedId != quoteFeedId` Identity Check in Synthetic Ratio Mode Produces Constant Bad Price — (`File: smart-contracts-poc/contracts/AnchoredPriceProvider.sol`)

---

### Summary

`AnchoredPriceProvider` supports a two-feed synthetic ratio mode (`baseFeedId / quoteFeedId`). The constructor enforces `_baseToken != _quoteToken` but contains **no analogous check that `_baseFeedId != _quoteFeedId`**. When both feed IDs are identical and non-zero, the synthetic ratio computation collapses to a constant price of `ORACLE_DECIMALS` (i.e., 1.0 in 8-decimal format) with a doubled spread, regardless of the actual oracle price. This bad price is then clamped and forwarded to the pool's swap math.

---

### Finding Description

In `_getBidAndAskPrice()`, the synthetic path reads both legs independently and divides:

```solidity
mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);
spreadBps += spreadBps2;
``` [1](#0-0) 

When `baseFeedId == quoteFeedId`, both `_readLeg` calls hit the same oracle slot and return the same `mid` value. The division `mid / mid` yields exactly `ORACLE_DECIMALS` (1.0 in 8-decimal), and `spreadBps` is doubled. The constructor enforces token distinctness but not feed distinctness:

```solidity
require(_baseToken != address(0) && _quoteToken != address(0) && _baseToken != _quoteToken);
baseFeedId = _baseFeedId;
quoteFeedId = _quoteFeedId;
``` [2](#0-1) 

There is no `require(_baseFeedId != _quoteFeedId)` guard anywhere in the constructor. [3](#0-2) 

The resulting `mid = ORACLE_DECIMALS` then flows into `_computeBidAsk`, which computes a band around price 1.0 and forwards it to the pool via `getBidAndAskPrice()`. [4](#0-3) 

---

### Impact Explanation

Any pool configured with this provider receives a constant oracle mid of 1.0 (in Q64 after `_bandEdge`) instead of the true market price. For any non-1:1 pair (e.g., BTC/ETH, ETH/USDC), the pool's bid/ask quotes are completely detached from reality. A trader can:

- **Buy the underpriced asset**: if the true price is BTC/ETH ≈ 20, the pool quotes ≈ 1.0, so the trader buys BTC for 1 ETH instead of 20 ETH, draining the pool's BTC reserves.
- **Sell the overpriced asset**: sell ETH to the pool at a price of 1 BTC each, draining BTC.

This constitutes **direct loss of LP principal** and **pool insolvency** — LP claims cannot be covered after the exploit.

---

### Likelihood Explanation

The likelihood depends on whether the factory (`AnchoredProviderFactory`) validates `baseFeedId != quoteFeedId` before deployment. The constructor itself does not. If the factory passes through caller-supplied feed IDs without this check, any actor with permission to deploy a provider (which may be semi-trusted or permissionless depending on factory access control) can trigger this. The analog to the Code-VM bug is exact: the missing identity check between two roles that must be distinct.

---

### Recommendation

Add an explicit identity check in the `AnchoredPriceProvider` constructor:

```solidity
require(
    _quoteFeedId == bytes32(0) || _quoteFeedId != _baseFeedId,
    "baseFeedId and quoteFeedId must differ"
);
``` [5](#0-4) 

Additionally, enforce the same check in the factory deployment path.

---

### Proof of Concept

1. Deploy `AnchoredPriceProvider` with `_baseFeedId = X` and `_quoteFeedId = X` (same non-zero feed ID), pointing to a real oracle (e.g., ETH/USD Pyth feed), with `_baseToken = WBTC`, `_quoteToken = USDC`.
2. Deploy a pool using this provider.
3. Call `getBidAndAskPrice()`. Both `_readLeg(X)` calls return the same `mid` (e.g., `3000_00000000` for ETH/USD at $3000). The ratio: `Math.mulDiv(3000e8, 1e8, 3000e8) = 1e8`. The pool receives bid/ask around price 1.0 (in 8-decimal).
4. The true WBTC/USDC price is ~$60,000. The pool quotes ≈ $1. A trader calls `swap()` buying WBTC at $1 each, draining all WBTC from the pool for near-zero USDC cost. [4](#0-3)

    function _getBidAndAskPrice() internal returns (uint128, uint128) {
        (uint256 mid, uint256 spreadBps, , bool ok) = _readLeg(baseFeedId);
        if (!ok) return (0, type(uint128).max);

        bytes32 _quote = quoteFeedId;
        if (_quote != bytes32(0)) {
            (uint256 mid2, uint256 spreadBps2, , bool ok2) = _readLeg(_quote);
            if (!ok2 || mid2 == 0) return (0, type(uint128).max);
            // Synthetic ratio (8-decimal): mid1 / mid2. Relative uncertainties of a ratio add.
            mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);
            spreadBps += spreadBps2;
        }

        return _computeBidAsk(mid, spreadBps);
    }
