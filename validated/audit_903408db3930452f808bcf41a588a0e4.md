### Title
Synthetic-ratio provider with `baseFeedId == quoteFeedId` always quotes price 1.0, enabling LP drain — (`smart-contracts-poc/contracts/AnchoredPriceProvider.sol`)

---

### Summary

`AnchoredPriceProvider` supports a two-feed synthetic ratio mode (`baseFeedId / quoteFeedId`). The constructor enforces `baseToken != quoteToken` but has **no corresponding guard** that `baseFeedId != quoteFeedId`. When both feed IDs are identical the ratio collapses to a constant `1e8` (price = 1.0) regardless of actual market conditions, feeding a permanently wrong price into every pool swap.

---

### Finding Description

In `_getBidAndAskPrice`, when `quoteFeedId != bytes32(0)` the provider reads both legs and computes:

```solidity
mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);   // mid / mid2
spreadBps += spreadBps2;                          // spread1 + spread2
``` [1](#0-0) 

If `baseFeedId == quoteFeedId`, both `_readLeg` calls return the same `(mid, spreadBps)` from the oracle. The division then yields:

```
mid = Math.mulDiv(X, 1e8, X) = 1e8   (always, for any real price X)
spreadBps = 2 × spreadBps
```

The constructor that enforces token distinctness but omits the feed-ID check:

```solidity
require(_baseToken != address(0) && _quoteToken != address(0) && _baseToken != _quoteToken);
// ← no require(_baseFeedId != _quoteFeedId)
``` [2](#0-1) 

The immutables are set unconditionally:

```solidity
baseFeedId = _baseFeedId;
quoteFeedId = _quoteFeedId;
``` [3](#0-2) 

The doubled spread (`2 × spreadBps`) may or may not exceed `MAX_SPREAD_BPS` depending on the constructor argument chosen by the deployer. If `MAX_SPREAD_BPS` is set generously (e.g. 9 999), the circuit breaker does not fire and `_computeBidAsk` proceeds with `mid = 1e8`. [4](#0-3) 

The resulting bid/ask is anchored around price 1.0 in 8-decimal units. For any real token pair whose true price differs from 1.0 (e.g. ETH/BTC ≈ 0.033, BTC/USD ≈ 100 000), every pool swap executes at a wildly wrong price.

---

### Impact Explanation

Any pool that consumes this provider's `getBidAndAskPrice()` will execute swaps at a constant price of 1.0. Arbitrageurs can immediately drain LP assets by buying the underpriced side. LP principal is directly and completely at risk. This satisfies the "bad-price execution" and "direct loss of user principal" impact gates.

---

### Likelihood Explanation

`AnchoredProviderFactory` (and `PriceProviderFactory`) are permissionless — anyone can deploy a provider with arbitrary feed IDs. [5](#0-4) 

There is no on-chain validation that `baseFeedId != quoteFeedId` at any point in the deployment path. A misconfigured or malicious deployer can create such a provider, attach it to a pool, and the pool will silently quote 1.0 forever. This is directly analogous to M-5, where the permissionless OTLM pool creation with `stakeToken == payoutToken` was accepted as medium severity by the sponsor.

---

### Recommendation

Add a guard in the `AnchoredPriceProvider` constructor:

```solidity
require(_quoteFeedId == bytes32(0) || _baseFeedId != _quoteFeedId,
        "base and quote feed must differ");
```

This mirrors the existing `_baseToken != _quoteToken` check and closes the gap at the only point where the immutables are set.

---

### Proof of Concept

1. Deploy `AnchoredPriceProvider` with:
   - `_baseFeedId = keccak256("ETH/USD")`
   - `_quoteFeedId = keccak256("ETH/USD")` ← same feed
   - `_baseToken = WETH`, `_quoteToken = WBTC` (distinct, passes constructor)
   - `MAX_SPREAD_BPS = 9999`, `MAX_REF_STALENESS = 7 days`
2. Attach provider to a WETH/WBTC pool.
3. Call `getBidAndAskPrice()`. Both `_readLeg` calls return the same live ETH/USD price `X`. The ratio `Math.mulDiv(X, 1e8, X) = 1e8` → mid = 1.0.
4. Pool quotes WETH/WBTC at 1.0. True price ≈ 0.033.
5. Attacker swaps WBTC → WETH at 1:1, receiving ~30× fair value. LP loses the difference. [6](#0-5)

    function _getBidAndAskPrice() internal returns (uint128, uint128) {
        (uint256 mid, uint256 spreadBps, , bool ok) = _readLeg(baseFeedId);
        if (!ok) return (0, type(uint128).max);

        bytes32 _quote = quoteFeedId;

            spreadBps += spreadBps2;

    }
