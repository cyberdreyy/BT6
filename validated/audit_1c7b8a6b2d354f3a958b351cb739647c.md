### Title
Signature Replay in `allowPushers` Nullifies `revokePusher()` Within the Deadline Window — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary
`allowPushers` accepts any EIP-191 signature whose deadline has not yet expired. There is no nonce, no used-signature set, and no check against the current `namespaceRemapping` state. A creator who holds a pusher's original consent signature can call `allowPushers` again with the identical `(deadline, pusher, sig)` tuple at any point before `deadline` passes, unconditionally overwriting `namespaceRemapping[pusher] = creator` even after the pusher explicitly called `revokePusher()`.

### Finding Description
`allowPushers` computes the signed hash as:

```
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
```

and the only liveness check is `_ensureDeadline`:

```solidity
function _ensureDeadline(uint256 deadline) internal view virtual {
    require(block.timestamp <= deadline, DeadlineExceeded());
}
``` [1](#0-0) 

After `revokePusher()` zeroes the mapping:

```solidity
namespaceRemapping[msg.sender] = address(0);
``` [2](#0-1) 

the creator can immediately call `allowPushers` again with the same `(deadline, pusher, sig)` tuple. Because the hash is identical and the deadline has not expired, both `_ensureDeadline` and `ECDSA.recover` pass, and the mapping is restored:

```solidity
namespaceRemapping[pusher] = msg.sender;
``` [3](#0-2) 

The code comment in `allowPushers` explicitly acknowledges the concern — *"an undated signature could re-establish a delegation AFTER the pusher revoked it"* — but the deadline alone does not close this window; it only prevents using an expired signature. [4](#0-3) 

No nonce, used-signature mapping, or revocation-state guard exists anywhere in the compressed oracle contracts. [5](#0-4) 

### Impact Explanation
Once the creator replays the signature, `namespaceRemapping[pusher] = creator` is live again. Every subsequent fallback push from the pusher's address is routed into the creator's namespace:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [6](#0-5) 

If the pusher revoked in order to push into their own namespace (e.g., to operate an independent oracle feed), those pushes are silently hijacked into the creator's namespace. The creator's feed receives data the pusher intended for their own feed, and the pusher's own feed receives nothing. Any pool consuming the creator's compressed oracle feed via `getOracleData` / `price` will read the hijacked data, which may be stale, wrong-scale, or intended for a different asset pair — constituting bad-price execution.

The pusher has no recourse until `deadline` expires: every call to `revokePusher()` can be immediately undone by the creator replaying the same signature.

### Likelihood Explanation
The creator already holds the pusher's signed consent from the original `allowPushers` call. Replaying it is a single on-chain transaction with no additional prerequisites. The window is as wide as the original deadline (commonly hours to days). The creator is not a privileged protocol role — they are any address that created an oracle feed.

### Recommendation
Track consumed consent signatures. The simplest fix is a `mapping(bytes32 => bool) private _usedConsents` keyed on the full signed hash. Mark it `true` on first use in `allowPushers` and revert if it is already set. This makes each pusher consent single-use, so a replayed signature after `revokePusher()` is rejected regardless of the deadline.

Alternatively, include a per-pusher nonce in the signed payload and increment it on every successful `allowPushers` or `revokePusher` call, invalidating all prior signatures.

### Proof of Concept
```solidity
// Foundry unit test (pseudo-code)
function testReplayAfterRevoke() public {
    uint256 deadline = block.timestamp + 1 days;

    // 1. Pusher signs consent
    bytes memory sig = _signConsent(PUSHER_KEY, deadline, pusher, creator);

    // 2. Creator delegates pusher
    vm.prank(creator);
    oracle.allowPushers(deadline, _arr(pusher), _arr(sig));
    assertEq(oracle.namespaceRemapping(pusher), creator);

    // 3. Pusher self-revokes
    vm.prank(pusher);
    oracle.revokePusher();
    assertEq(oracle.namespaceRemapping(pusher), address(0));

    // 4. Creator replays the SAME signature before deadline — succeeds
    vm.prank(creator);
    oracle.allowPushers(deadline, _arr(pusher), _arr(sig));

    // Invariant violated: pusher is re-delegated without fresh consent
    assertEq(oracle.namespaceRemapping(pusher), creator);
}

        uint256 l = pushers.length;
        require(l == signatures.length);
