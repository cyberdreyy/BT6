### Title
Revoked Pusher Delegation Can Be Silently Re-Established via Replayed Signed Consent — (File: `smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`revokePusher()` clears `namespaceRemapping[pusher]` but does not invalidate previously issued signed consents. Because the consent signature commits only to a deadline (not a revocation nonce), any party holding the original signed consent can call `allowPushers` again after revocation to silently re-establish the delegation. A re-delegated, potentially compromised pusher can then push manipulated prices into the oracle, which are consumed by pools.

---

### Finding Description

The `CompressedOracleV1` delegation model works as follows:

- A feed creator calls `allowPushers(pushers[], consents[])`, where each consent is an EIP-191 signature from the pusher over `(deadline, pusher, creator)` (or similar binding). This writes `namespaceRemapping[pusher] = creator`.
- The `fallback()` push path resolves the effective namespace via `namespaceRemapping[msg.sender]`, falling back to `msg.sender` itself.
- `revokePusher(pusher)` clears `namespaceRemapping[pusher]`.

The critical gap: the signed consent contains a **deadline** but **no revocation nonce or per-pusher sequence number**. The protocol documentation explicitly acknowledges this:

> *"the signed consent has no data timestamp, so an undated signature could re-establish a delegation after the pusher revoked it."*
> — `smart-contracts-poc/contracts/oracles/compressed/docs/en/slot-structure.md`

Because `revokePusher` only clears the mapping entry without incrementing any per-pusher nonce or recording the revocation on-chain, the original signed consent remains cryptographically valid until its deadline expires. Anyone who possesses the consent bytes (the pusher themselves, or an attacker who observed the original `allowPushers` calldata on-chain) can replay the call to `allowPushers` and restore the delegation.

This is structurally identical to the `ActivityRewardDistributor` bug: a state-changing operation (`revokePusher` / `withdrawPmx`) modifies one piece of state (the mapping / `totalReward`) but leaves a related piece of state (the signed consent validity / `lastUpdatedRewardTimestamp`) stale, allowing a subsequent actor to exploit the inconsistency.

---

### Impact Explanation

Once re-delegated, the pusher writes into the creator's namespace via the `fallback()` path. The `CompressedOracleV1` slot is the upstream data source for `PriceProvider` / `AnchoredPriceProvider`, which deliver bid/ask quotes to `MetricOmmPool.swap()`. A manipulated slot value (crafted `U64x32` price, sentinel or extreme `Codebook256` spread indexes) reaches the pool as a valid oracle price, enabling:

- **Bad-price execution**: swapper receives more output than the true mid-price permits, draining pool reserves.
- **Pool insolvency**: LP claims exceed actual token balances after a sequence of adversarial swaps at the manipulated price.

---

### Likelihood Explanation

- The original `allowPushers` calldata is public on-chain; any observer can extract the consent bytes.
- The attacker only needs to call `allowPushers` again before the deadline expires — no special privilege is required.
- The creator has no on-chain mechanism to detect that revocation was bypassed until a bad price is already stored.
- Likelihood is **Medium**: requires the attacker to act within the consent deadline window, but the window can be months or years depending on the deadline chosen at delegation time.

---

### Recommendation

1. **Add a per-pusher revocation nonce** (`mapping(address pusher => uint256 nonce) public pusherNonce`). Include the nonce in the signed consent message: `keccak256(abi.encode(chainid, oracleAddress, creator, pusher, deadline, nonce))`.
2. In `revokePusher(pusher)`, increment `pusherNonce[pusher]` in addition to clearing `namespaceRemapping[pusher]`. This invalidates all previously issued consents for that pusher.
3. In `allowPushers`, verify the nonce in the consent matches the current `pusherNonce[pusher]`.

---

### Proof of Concept

```
1. Creator signs consent for Pusher with deadline = block.timestamp + 365 days.
   Consent bytes are broadcast on-chain via allowPushers([pusher], [consent]).
   → namespaceRemapping[pusher] = creator  ✓

2. Pusher's key is compromised. Creator calls revokePusher(pusher).
   → namespaceRemapping[pusher] = address(0)  ✓ (revoked)

3. Attacker extracts the original consent bytes from step-1 calldata.
   Attacker calls allowPushers([pusher], [old_consent]).
   Deadline check: block.timestamp < deadline → passes.
   No nonce check exists.
   → namespaceRemapping[pusher] = creator  ✓ (silently re-established)

4. Compromised pusher calls oracle fallback() with a crafted slot word:
   - U64x32 price set to 10× the true market price
   - Codebook256 spread indexes set to 0x00/0x00 (zero spread, no staleness marker)
   - timestampMs = current time (passes monotonicity check)
   → Creator's namespace slot is overwritten with the manipulated price.

5. PriceProvider reads the slot, AnchoredPriceProvider passes it within
   its confidence band (if the anchor feed is also stale or the deviation
   param is wide), and MetricOmmPool.swap() executes at the bad price.
   → Trader receives excess output; pool reserves are drained.

    fallback() override external {
        uint256 end;
        uint256 namespace;

        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0)) creator = msg.sender;

        assembly ("memory-safe") {
            end := calldatasize()
            namespace := shl(96, creator) // [creator:20][zeros:12]
        }

        // 4 * 6 + 7 + 1 = 32 bytes per slot
        if (end == 0 || end % 32 != 0) revert BadCalldataLength();

        for (uint256 ptr = 0; ptr < end; ptr += 32) {
            uint256 word;
            assembly ("memory-safe") {
                word := calldataload(ptr)
            }
            // casting to 'uint8' is safe we want LSB
            // forge-lint: disable-next-line(unsafe-typecast)

            if (!newer) continue;

        uint56 tsMs = uint56(block.timestamp * 1000);
