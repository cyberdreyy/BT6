### Title
Single-Step `transferProviderOwnership` Enables Irrecoverable Oracle Source Hijack — (File: `smart-contracts-poc/contracts/AnchoredProviderFactory.sol`)

---

### Summary

`AnchoredProviderFactory`, `PriceProviderFactory`, and `PriceProviderFactoryL2` all implement `transferProviderOwnership` as a single atomic write to `providerOwner[provider]`. A typo or wrong address permanently transfers the oracle-source management role to an unintended address with no recovery path. The new "owner" can immediately call `setSource` to redirect the `AnchoredPriceProvider` to a malicious contract, causing every pool that uses that provider to revert on swap with `FeedStalled`.

---

### Finding Description

All three provider factories share the same single-step pattern:

```solidity
// AnchoredProviderFactory.sol L230-239
function transferProviderOwnership(address provider, address newOwner)
    external override onlyProviderOwner(provider)
{
    require(_providers.contains(provider), ProviderNotTracked());
    require(newOwner != address(0));
    address previousOwner = providerOwner[provider];

    providerOwner[provider] = newOwner;                        // ← immediate, irrevocable
    _providersByCreator[previousOwner].remove(provider);
    _providersByCreator[newOwner].add(provider);

    emit ProviderOwnershipTransferred(provider, previousOwner, newOwner);
}
``` [1](#0-0) 

The `providerOwner` role is the sole key to `setSource`, which swaps the live oracle source feeding prices to every pool that uses the provider:

```solidity
// AnchoredProviderFactory.sol L224-228
function setSource(address provider, address newSource)
    external override onlyProviderOwner(provider)
{
    require(_providers.contains(provider), ProviderNotTracked());
    AnchoredPriceProvider(provider).setSource(newSource);
``` [2](#0-1) 

The same single-step pattern is replicated in `PriceProviderFactory` and `PriceProviderFactoryL2`: [3](#0-2) [4](#0-3) 

**The protocol already knows the two-step pattern and applies it everywhere else:**

- `MetricOmmPoolFactory` inherits `Ownable2Step` for its own ownership transfer. [5](#0-4) 
- Pool-admin transfer uses `proposePoolAdminTransfer` / `acceptPoolAdmin`. [6](#0-5) 
- `OracleBase` (compressed) uses `setPendingStateGuardRole` / `acceptStateGuardRole`. [7](#0-6) 
- `OracleBase` (providers) uses the same two-step guard pattern. [8](#0-7) 

`transferProviderOwnership` is the only privileged transfer in the entire system that lacks this protection.

---

### Impact Explanation

Once `providerOwner[provider]` is set to a wrong address, the attacker calls `setSource(provider, maliciousSource)`. The malicious source contract returns `(bid=0, ask=type(uint128).max)`. Inside `AnchoredPriceProvider._computeBidAsk`:

```solidity
uint256 bidOut = Math.min(refBid, cBid);   // = min(refBid, 0) = 0
uint256 askOut = Math.max(refAsk, cAsk);   // = max(refAsk, max) = max
if (bidOut == 0 || bidOut >= askOut) {
    return (0, type(uint128).max);          // FeedStalled sentinel
}
``` [9](#0-8) 

`getBidAndAskPrice` then reverts with `FeedStalled`:

```solidity
if (bid == 0 || ask == type(uint128).max) revert FeedStalled();
``` [10](#0-9) 

Every pool that uses this provider becomes permanently unable to execute swaps. LPs are trapped: they cannot rebalance or exit via swaps. The previous legitimate owner has no recovery path because `transferProviderOwnership` has no cancel or reclaim mechanism.

---

### Likelihood Explanation

The `providerOwner` is expected to be a multisig or EOA managing multiple providers across chains. A single address typo, clipboard error, or phishing substitution during a routine ownership handoff permanently transfers control. The likelihood is low-to-medium (human error in a privileged operation), but the consequence is irrecoverable and affects all pools downstream of the provider.

---

### Recommendation

Implement a two-step transfer in all three factories, mirroring the existing `proposePoolAdminTransfer` / `acceptPoolAdmin` pattern already present in `MetricOmmPoolFactory`:

1. `proposeProviderOwnership(provider, newOwner)` — stores `pendingProviderOwner[provider] = newOwner` and emits an event.
2. `acceptProviderOwnership(provider)` — callable only by `pendingProviderOwner[provider]`; completes the transfer and clears the pending slot.
3. `cancelProviderOwnership(provider)` — callable by the current owner to abort a pending transfer.

---

### Proof of Concept

1. Alice owns `provider` (an `AnchoredPriceProvider` for ETH/USDC, used by a live pool).
2. Alice calls `AnchoredProviderFactory.transferProviderOwnership(provider, 0xDEAD...BEEF)` with a typo.
3. `providerOwner[provider]` is immediately set to `0xDEAD...BEEF`. Alice has no recovery path.
4. The attacker at `0xDEAD...BEEF` calls `setSource(provider, maliciousOracle)`.
5. `maliciousOracle.getBidAndAskPrice()` returns `(0, type(uint128).max)`.
6. `AnchoredPriceProvider._computeBidAsk` returns the `FeedStalled` sentinel `(0, type(uint128).max)`.
7. `getBidAndAskPrice()` reverts with `FeedStalled` on every subsequent swap attempt.
8. The ETH/USDC pool is permanently bricked for swaps; LPs cannot exit via the normal swap path.

  function acceptPoolAdmin(address pool) external override nonReentrant {
    address pending = pendingPoolAdmin[pool];
    if (pending == address(0)) revert NoPendingPoolAdminTransfer();
    if (msg.sender != pending) revert NotPendingPoolAdmin(pool, msg.sender, pending);
