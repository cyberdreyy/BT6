### Title
Signature Replay in `allowPushers` Allows Creator to Override Pusher's `revokePusher()` Within the Deadline Window — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`allowPushers` has no used-signature tracking and no nonce. The only replay guard is the deadline, which only prevents replay **after** the deadline expires. Within the deadline window, a creator can call `allowPushers` a second time with the exact same pusher signature to silently re-establish a delegation that the pusher explicitly revoked via `revokePusher()`.

### Finding Description

`allowPushers` constructs the signed digest as:

```solidity
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [1](#0-0) 

The only liveness check is `_ensureDeadline`, which passes as long as `block.timestamp <= deadline`: [2](#0-1) 

`revokePusher` clears `namespaceRemapping[msg.sender]` to `address(0)`: [3](#0-2) 

There is no mapping of consumed signatures, no per-pusher nonce, and no check that the pusher has not previously revoked. The code comment explicitly acknowledges the risk ("an undated signature could re-establish a delegation AFTER the pusher revoked it") but the deadline only closes the window **after expiry** — not within it. [4](#0-3) 

**Attack sequence:**

1. Pusher signs consent for creator with `deadline = block.timestamp + 1 days`.
2. Creator calls `allowPushers(deadline, [pusher], [sig])` → `namespaceRemapping[pusher] = creator`.
3. Pusher calls `revokePusher()` → `namespaceRemapping[pusher] = address(0)`.
4. Creator calls `allowPushers(deadline, [pusher], [sig])` again with the **identical** signature and deadline (still valid since `block.timestamp <= deadline`).
5. `namespaceRemapping[pusher] = creator` is restored. No revert occurs.

### Impact Explanation

After step 5, the pusher believes they are pushing into their own namespace (the registrationless default after revocation). Instead, every subsequent `fallback()` push is silently redirected to the creator's namespace:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [5](#0-4) 

Consequences:
- **Pusher's own pools** receive no updates (stale oracle data → bad-price execution or DoS of swap/liquidity flows).
- **Creator's namespace** receives data from a pusher who explicitly withdrew consent, potentially with mismatched feed IDs, stale timestamps, or prices intended for a different asset pair — enabling bad-price execution in pools consuming the creator's namespace.

This is a direct admin-boundary break: `revokePusher()` is the pusher's only self-protection mechanism, and it is rendered non-final by the creator.

### Likelihood Explanation

The creator needs only to retain the original signature bytes (trivially available from the original `allowPushers` calldata on-chain) and call `allowPushers` again before the deadline expires. No privileged protocol role is required. The deadline window is creator-chosen and can be set to days or weeks.

### Recommendation

Mark each signature as consumed after first use:

```solidity
mapping(bytes32 => bool) private _usedDelegationSigs;

// inside allowPushers loop:
bytes32 sigHash = keccak256(signatures[i]);
require(!_usedDelegationSigs[sigHash], "signature already used");
_usedDelegationSigs[sigHash] = true;
```

Alternatively, introduce a per-pusher revocation nonce that is incremented on `revokePusher()` and included in the signed digest, so any previously issued signature becomes invalid after revocation.

### Proof of Concept

```solidity
// Foundry test sketch
function testReplayAfterRevoke() public {
    uint256 deadline = block.timestamp + 1 days;

    // 1. Pusher signs consent
    bytes memory sig = _signConsent(PUSHER_KEY, deadline, pusher, creator);

    // 2. Creator delegates pusher
    address[] memory pushers = new address[](1);
    pushers[0] = pusher;
    bytes[] memory sigs = new bytes[](1);
    sigs[0] = sig;
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs);
    assertEq(oracle.namespaceRemapping(pusher), creator);

    // 3. Pusher revokes
    vm.prank(pusher);
    oracle.revokePusher();
    assertEq(oracle.namespaceRemapping(pusher), address(0));

    // 4. Creator replays the SAME signature — succeeds, no revert
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs);

    // 5. Revocation finality violated
    assertEq(oracle.namespaceRemapping(pusher), creator); // pusher silently re-delegated

    // 6. Pusher's subsequent push lands in creator's namespace, not their own
    uint56 tsMs = uint56(block.timestamp * 1000);
    vm.prank(pusher);
    (bool ok,) = address(oracle).call(_wordAt(0, 0, _packRaw(9_000_000, 1, 1), tsMs));
    assertTrue(ok);
    assertEq(oracle.getOracleData(oracle.feedIdOf(creator, 0, 0)).price, ...); // creator ns updated
    assertEq(oracle.getOracleData(oracle.feedIdOf(pusher,   0, 0)).price, 0);  // pusher ns empty
}
```
