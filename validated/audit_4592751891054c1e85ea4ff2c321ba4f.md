### Title
`priceGuard` Min/Max Bounds Are Stored But Never Enforced During Price Reads, Allowing Unclamped Prices to Reach Pool Swaps — (`smart-contracts-poc/contracts/oracles/providers/OracleBase.sol`, `smart-contracts-poc/contracts/oracles/compressed/OracleBase.sol`)

---

### Summary

Both oracle base contracts expose `setPriceGuard` to configure absolute min/max price bounds for a feed. Neither the `_readPrice` path (providers oracle) nor the `getOracleData` path (compressed oracle) ever reads or enforces those bounds before returning a price to a consuming pool. The guard is written-only dead code. A compromised or malfunctioning pusher can push any price that passes the monotonicity/timestamp check, and that price will be forwarded to pools unclamped.

---

### Finding Description

**Providers oracle (`providers/OracleBase.sol`):**

`setPriceGuard` writes a `PriceGuard({min, max})` struct: [1](#0-0) 

The sole internal price-serving function `_readPrice`, called by both `price()` and `integratorPrice()`, returns raw storage data with no guard check: [2](#0-1) 

`priceGuard` is never read anywhere in the read path.

**Compressed oracle (`compressed/OracleBase.sol` + `CompressedOracle.sol`):**

`setPriceGuard` stores bounds identically: [3](#0-2) 

`CompressedOracle.getOracleData` decodes and returns the raw U64x32 price without consulting `priceGuard[feedId]`: [4](#0-3) 

`price()` calls `_price()` → `getOracleData()` — the guard is never reached: [5](#0-4) 

Additionally, the `pool` argument to `CompressedOracle.price()` is explicitly discarded (comment: "pool is unused"), so reads are fully permissionless — no in-swap binding, no registration check — making the unclamped price reachable by any caller: [6](#0-5) 

---

### Impact Explanation

Pools using `PriceProvider` (not `AnchoredPriceProvider`) receive the raw oracle price directly. If a pusher is compromised or malfunctions and pushes a price outside the configured `priceGuard` bounds, the pool executes swaps at that bad price. LPs lose principal as traders receive more output than the oracle/bin curve permits — a direct swap conservation failure.

For the `CompressedOracle`, the surface is wider: because reads are permissionless and the `pool` parameter is ignored, any contract (including a malicious one) can consume unclamped prices without registration.

For pools using `AnchoredPriceProvider`, the clamping layer narrows the impact but does not eliminate it: the anchor oracle's unclamped mid-price shifts the entire reference band, so the "safe" bid/ask edges are themselves anchored to a bad value.

---

### Likelihood Explanation

Medium. The trigger requires a compromised pusher key (providers oracle) or a malicious/compromised creator or delegated pusher (compressed oracle). The `priceGuard` feature is explicitly designed to be the on-chain mitigation for exactly this scenario. Its non-enforcement means the mitigation is absent while operators believe it is active — the same single-point-of-failure risk identified in the external report.

---

### Recommendation

Enforce `priceGuard` bounds inside `_readPrice` (providers) and `getOracleData` (compressed) before returning the price:

```solidity
PriceGuard memory guard = priceGuard[feedId];
if (guard.min != 0 || guard.max != 0) {
    require(
        data.price >= guard.min && data.price <= guard.max,
        PriceOutOfBounds(feedId, data.price)
    );
}
```

This mirrors the on-chain check the external report recommends: the system must not rely solely on off-chain pusher honesty when an on-chain bound mechanism already exists.

---

### Proof of Concept

1. Admin calls `setPriceGuard(feedId, 1e8, 2e8)` — intending to bound the feed to $1–$2.
2. A compromised pusher pushes a price of `100` (i.e., $0.000001) with a fresh timestamp. The only on-chain gate is the monotonicity check (`newTs > oldTs`) and the `maxTimeDrift` check — both pass. [7](#0-6) 

3. A pool calls `price(feedId, pool)` → `_readPrice(feedId)` → returns `100` without consulting `priceGuard`.
4. `PriceProvider` converts `100` to a Q64.64 bid/ask and the pool executes swaps at $0.000001.
5. Traders drain LP reserves at a tiny fraction of the true price; LPs suffer direct principal loss.
6. The `priceGuard` mapping holds `{min: 1e8, max: 2e8}` in storage throughout — it is never read.

    function getOracleData(bytes32 feedId) public view override returns (OracleData memory data) {
        (address creator, uint8 slotIndex, uint8 positionIndex) = _unpackFeedId(feedId);

        SlotLayout memory _layout = _loadSlotLayout(_oracleSlot(creator, slotIndex));
        CompressedOracleData memory compressed = _selectCompressedData(_layout, positionIndex);

        if (compressed.s1 == 0xff && compressed.s0 == 0xff) {
            data.spread1 = BPS_BASE;
            data.spread0 = BPS_BASE;
            return data;
        }

        data.price = U64x32.decode(compressed.p);
        data.spread0 = _decodeCodebookIndex(compressed.s0);
        data.spread1 = _decodeCodebookIndex(compressed.s1);
        data.timestampMs = _layout.timestampMs;
    }
