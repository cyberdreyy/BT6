### Title
`MetricOmmPoolFactory._validatePriceProvider` accepts any contract as price provider without verifying factory provenance, allowing unclamped bid/ask prices to reach pool swaps — (File: `metric-core/contracts/MetricOmmPoolFactory.sol`)

---

### Summary

`MetricOmmPoolFactory._validatePriceProvider` only checks that the provider's `token0()`/`token1()` match the pool's tokens. It never verifies that the provider was deployed by the approved `AnchoredProviderFactory`. Any contract that exposes matching `token0()`/`token1()` is accepted, bypassing every envelope, clamp, and staleness guarantee that `AnchoredProviderFactory` enforces.

---

### Finding Description

`AnchoredProviderFactory` explicitly documents its public-pool eligibility predicate:

> *"public-pool eligibility is then the machine-checkable predicate `recognizedFactory.isProvider(p)`"* [1](#0-0) 

The factory enforces this by requiring every provider to use an admin-curated oracle, parameters within admin-set envelopes (staleness, spread, margin), and an immutable clamp that keeps every quote no tighter than the audited band. [2](#0-1) 

However, `MetricOmmPoolFactory._validatePriceProvider` — the sole guard called both at pool creation and on every price-provider update — performs only a token-address match:

```solidity
function _validatePriceProvider(address token0, address token1, address priceProvider) internal view {
    if (priceProvider == address(0)) revert InvalidPriceProvider();
    if (IPriceProvider(priceProvider).token0() != token0 || IPriceProvider(priceProvider).token1() != token1) {
      revert PriceProviderTokenMismatch();
    }
}
``` [3](#0-2) 

It is called unconditionally at pool creation: [4](#0-3) 

And again when a pool admin proposes or executes a provider update: [5](#0-4) [6](#0-5) 

There is no call to `AnchoredProviderFactory.isProvider()` anywhere in the factory. A malicious provider needs only to return the correct token addresses from `token0()`/`token1()` to pass validation. It can then return any values from `getBidAndAskPrice()`:

- `(0, type(uint128).max)` — halts all swaps in the pool permanently.
- An inverted pair `(bid ≥ ask)` — causes `AnchoredPriceProvider`'s own clamp to fail closed, but a raw malicious provider has no such clamp.
- Extreme or stale prices — executes swaps at rates that drain LP balances.

The `AnchoredPriceProvider`'s band clamp and staleness guards exist only inside providers deployed by `AnchoredProviderFactory`. A provider deployed outside that factory carries none of those guarantees. [7](#0-6) 

---

### Impact Explanation

**Bad-price execution / pool insolvency.** A pool admin (any user who created a pool, since `createPool` is permissionless) can set a price provider that returns unclamped, inverted, or extreme bid/ask quotes. Swaps execute against those quotes: traders receive more output than the oracle permits, or the pool receives less input than owed, violating the swap-conservation invariant. Repeated exploitation drains LP balances below what LPs can claim, causing pool insolvency. The `AnchoredProviderFactory`'s envelope and clamp guarantees — the protocol's stated safety net for public pools — are entirely bypassed.

---

### Likelihood Explanation

`createPool` is permissionless. Any caller can deploy a pool, appoint themselves as pool admin, and supply a malicious price provider at creation time (zero timelock is valid: `priceProviderTimelock` is caller-supplied). No privileged role is required. The only prerequisite is that the malicious contract returns the correct token addresses from `token0()`/`token1()`. [8](#0-7) 

---

### Recommendation

Add a factory-provenance check inside `_validatePriceProvider`. The factory should store the address of the approved `AnchoredProviderFactory` and require:

```solidity
require(
    IAnchoredProviderFactory(anchoredProviderFactory).isProvider(priceProvider),
    "Provider not from approved factory"
);
```

Alternatively, maintain an admin-curated allowlist of approved provider factories and verify membership before accepting any provider address. This mirrors the oracle allow-list already enforced inside `AnchoredProviderFactory.createAnchoredProvider`. [9](#0-8) 

---

### Proof of Concept

```solidity
// Malicious price provider: passes token check, returns inverted prices
contract MaliciousProvider {
    address public immutable token0;
    address public immutable token1;
    constructor(address t0, address t1) { token0 = t0; token1 = t1; }
    // Returns bid >= ask → pool swap reverts or executes at bad rate
    function getBidAndAskPrice() external pure returns (uint128 bid, uint128 ask) {
        return (type(uint128).max, 1); // inverted: bid > ask
    }
}

// Attack:
// 1. Deploy MaliciousProvider(token0, token1)
// 2. Call factory.createPool(..., priceProvider = address(malicious), ...)
//    → _validatePriceProvider passes (token0/token1 match)
// 3. Pool is live with unclamped, inverted price provider
// 4. Any swap call reads getBidAndAskPrice() → bad execution or revert
```

`_validatePriceProvider` accepts the malicious provider because it only checks `token0()`/`token1()` return values, never calling `AnchoredProviderFactory.isProvider()`. The pool is deployed with a provider that carries none of the envelope, clamp, or staleness guarantees the protocol relies on. [3](#0-2)

    ///      Returns the (0, max) sentinel on any failure — fail closed, withdrawals unaffected.
    function _computeBidAsk(uint256 mid, uint256 spreadBps)
        internal view returns (uint128, uint128)
    {
        // Circuit breaker: extreme (combined) uncertainty means the feed is clearly broken.
        if (spreadBps > MAX_SPREAD_BPS) {
            return (0, type(uint128).max);
        }

        // Reference band: mid ± (spreadBps + minMargin), bid rounded down, ask rounded up.
        uint256 half = spreadBps * ONE_BPS_E18 + minMargin; // < BPS_BASE_U by construction (spreadBps <= MAX_SPREAD_BPS here)
        uint256 refBid = _bandEdge(mid, BPS_BASE_U - half, Math.Rounding.Floor);
        uint256 refAsk = _bandEdge(mid, BPS_BASE_U + half, Math.Rounding.Ceil);
        if (refBid == 0 || refAsk > type(uint128).max || refBid >= refAsk) {
            return (0, type(uint128).max);
        }

        // Custom quote: source (both variants) or shaped reference quote (customizable variant).
        //    Immutable reference mode quotes the band directly — zero knob SLOADs.
        address _source = source;
        uint256 cBid;
        uint256 cAsk;
        if (_source != address(0)) {
            // 7a. Source mode: any failure (revert, OOG, garbage, zero, inverted) halts — fail
            //     closed. Knobs do NOT post-process the source output (the source shapes itself).
            bool ok;
            (ok, cBid, cAsk) = _readSource(_source);
            if (!ok) {
                return (0, type(uint128).max);
            }
        } else if (MUTABLE_PARAMS) {
            // 7b. Shaped reference quote: mid ± mid·spreadBps·confidence, then the marginStep step
            //     factors — PriceProvider semantics, clamped into the band below.
            bool ok;
            (ok, cBid, cAsk) = _shapedQuote(mid, spreadBps);
            if (!ok) {
                return (0, type(uint128).max);
            }
        } else {
            return (uint128(refBid), uint128(refAsk));
        }

        // 8. Clamp: out-of-band custom quotes are clipped silently to the band edge.
        //    bid ≤ refBid < refAsk ≤ ask, so bid < ask holds by construction.
        uint256 bidOut = Math.min(refBid, cBid);
        uint256 askOut = Math.max(refAsk, cAsk);
        if (bidOut == 0 || bidOut >= askOut) {
            return (0, type(uint128).max);
        }

        return (uint128(bidOut), uint128(askOut));
    }
