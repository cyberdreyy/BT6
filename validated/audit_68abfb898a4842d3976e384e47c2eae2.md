### Title
Creator Can Replay Pusher Consent Signature to Silently Re-Establish Revoked Delegation — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowPushers` in `CompressedOracle.sol` contains no nonce, no used-signature bitmap, and no per-pusher revocation timestamp. After a pusher calls `revokePusher()` to self-revoke their delegation, the creator can immediately call `allowPushers` again with the **identical original signature and deadline** to restore `namespaceRemapping[pusher] = creator`. The pusher's self-revocation is completely ineffective for the entire remaining lifetime of the original deadline window. The code's own inline comment claims the deadline prevents this replay, but it does not.

---

### Finding Description

`allowPushers` validates a pusher's EIP-191 consent signature over the tuple `(chainid, oracle, deadline, pusher, creator)`:

```solidity
// CompressedOracle.sol lines 192–212
function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
    _ensureDeadline(deadline);
    ...
    bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
        keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
    );
    require(pusher == ECDSA.recover(hash, signatures[i]));
    namespaceRemapping[pusher] = msg.sender;
    emit PusherAuthorized(pusher, msg.sender);
}
``` [1](#0-0) 

`revokePusher` clears the mapping:

```solidity
// CompressedOracle.sol lines 238–243
function revokePusher() external {
    address creator = namespaceRemapping[msg.sender];
    if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
    namespaceRemapping[msg.sender] = address(0);
    emit PusherRevoked(msg.sender, creator);
}
``` [2](#0-1) 

There is **no nonce, no used-signature set, and no per-pusher revocation counter** anywhere in the contract. The signature domain `(chainid, oracle, deadline, pusher, creator)` is identical on every call. After `revokePusher()` zeroes the mapping, the creator can call `allowPushers` again with the same `(deadline, [pusher], [sig])` arguments and the `require(pusher == ECDSA.recover(...))` check passes identically, restoring `namespaceRemapping[pusher] = creator`.

The inline comment at lines 186–191 explicitly claims the deadline prevents this:

> *"The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it."* [3](#0-2) 

This claim is incorrect. The deadline only prevents replay **after** the deadline expires. Before expiry, the creator can replay the signature an unlimited number of times, including immediately after the pusher revokes. The stated invariant — that a pusher can exit a delegation — is broken.

---

### Impact Explanation

Every fallback push resolves the effective namespace via `namespaceRemapping`:

```solidity
// CompressedOracle.sol fallback — namespace resolution
address ns = namespaceRemapping[msg.sender];
if (ns == address(0)) ns = msg.sender;
``` [4](#0-3) 

After the creator replays the signature, the pusher's writes continue to land in the **creator's namespace**. Any `PriceProvider` or pool reading `feedIdOf(creator, slotIndex, positionIndex)` receives prices from a pusher who has explicitly revoked consent. If the pusher revoked because they are compromised, switching data sources, or deliberately stopping service, the pool continues to consume their (now stale, wrong, or adversarially controlled) prices. This satisfies the **bad-price execution** impact gate: a stale or unclamped bid/ask quote reaches a live pool swap via the creator's namespace.

---

### Likelihood Explanation

- The creator holds the original signature from the initial delegation setup — no additional off-chain action is required.
- The creator is a semi-trusted but **unprivileged** party with respect to the pusher's revocation right; no admin role is needed to call `allowPushers`.
- Deadlines in practice are set days to weeks in the future (as seen in tests using `block.timestamp + 1 days`), giving the creator a large replay window.
- The pusher has **no on-chain recourse** within the deadline window other than ceasing to push entirely.

---

### Recommendation

Track consumed signatures with a used-signature set, or include a per-pusher monotonic nonce in the signed domain so that revocation increments the nonce and invalidates all prior signatures:

```solidity
mapping(address => uint256) public pusherNonce;

// In allowPushers, include nonce in the hash:
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender, pusherNonce[pusher]))

// In revokePusher, increment the nonce:
pusherNonce[msg.sender]++;
namespaceRemapping[msg.sender] = address(0);
```

Alternatively, record a `revokedAt` timestamp per pusher and reject any signature whose deadline predates the revocation timestamp.

---

### Proof of Concept

```
1. Pusher signs consent: sig = sign(chainid, oracle, deadline=T+30days, pusher, creator)
2. Creator calls allowPushers(T+30days, [pusher], [sig])
   → namespaceRemapping[pusher] = creator  ✓
3. Pusher calls revokePusher()
   → namespaceRemapping[pusher] = address(0)  ✓ (pusher believes they are free)
4. Creator immediately calls allowPushers(T+30days, [pusher], [sig])  ← SAME sig, SAME deadline
   → _ensureDeadline passes (deadline not yet expired)
   → ECDSA.recover returns pusher  (hash is identical)
   → namespaceRemapping[pusher] = creator  ← delegation silently restored
5. Pusher's subsequent fallback pushes land in creator's namespace
   → pool reading feedIdOf(creator, ...) receives pusher's prices without consent
```

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
