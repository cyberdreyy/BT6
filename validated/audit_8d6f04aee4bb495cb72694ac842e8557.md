### Title
Signature Replay in `allowPushers` Allows Creator to Re-Establish Revoked Pusher Delegation Within Deadline Window — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary
`allowPushers` uses a deadline-scoped EIP-191 signature with no nonce and no used-signature registry. After a pusher calls `revokePusher()`, the creator can replay the original signature (while `block.timestamp <= deadline`) to unconditionally overwrite `namespaceRemapping[pusher]` back to the creator's address, violating pusher-revocation finality.

### Finding Description
The attack sequence in `PUSHER_REVOKED_BUT_DEADLINE_VALID_STATE`:

1. Pusher signs `keccak256(abi.encode(chainid, oracle, deadline, pusher, creator))` with `deadline = block.timestamp + 1 days`.
2. Creator calls `allowPushers(deadline, [pusher], [sig])` → `namespaceRemapping[pusher] = creator`.
3. Pusher calls `revokePusher()` → `namespaceRemapping[pusher] = address(0)`.
4. Creator calls `allowPushers(deadline, [pusher], [sig])` again with the **identical** signature and deadline (still valid) → `namespaceRemapping[pusher] = creator` again.

No new signature from the pusher is required. The contract has no mechanism to detect or reject a replayed signature.

### Impact Explanation
After step 4, every call to the `fallback()` push path by the pusher routes into the creator's namespace. If the pusher has since moved to a different creator's namespace (or their own), that namespace receives no further updates and its oracle data goes stale. Pools consuming that namespace's `feedId` will have their `getOracleData` return a `timestampMs` that fails the `maxTimeDrift` check, rendering those pools' swap/liquidity flows unusable. Additionally, if the pusher is now pushing data for different token pairs than what the creator's pools expect, the redirected pushes land in the creator's slots with mismatched price data, constituting bad-price execution.

### Likelihood Explanation
Requires a creator who retains a not-yet-expired pusher signature and is willing to act adversarially. Deadlines of hours-to-days are typical in delegation flows, making the replay window practically exploitable. The pusher has no on-chain recourse during the window other than waiting for expiry.

### Recommendation
Track consumed signatures with a `mapping(bytes32 => bool) usedSignatures` and mark each signature hash as used on first acceptance. Alternatively, include a monotonically increasing per-pusher nonce in the signed payload and store the last-used nonce, so a replayed signature is rejected even within the deadline window.

function test_replayAfterRevoke() public {
    uint256 deadline = block.timestamp + 1 days;
    bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
        keccak256(abi.encode(block.chainid, address(oracle), deadline, pusher, creator))
    );
    (uint8 v, bytes32 r, bytes32 s) = vm.sign(pusherKey, hash);
    bytes memory sig = abi.encodePacked(r, s, v);

    address[] memory pushers = new address[](1);
    pushers[0] = pusher;
    bytes[] memory sigs = new bytes[](1);
    sigs[0] = sig;

    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs);
    assertEq(oracle.namespaceRemapping(pusher), creator);

    vm.prank(pusher);
    oracle.revokePusher();
    assertEq(oracle.namespaceRemapping(pusher), address(0));

    // Replay same sig — no nonce, no used-sig check
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs);
    assertEq(oracle.namespaceRemapping(pusher), creator); // re-established
}

    ///         signature is REQUIRED — without it anyone could remap a foreign pusher
    ///         wallet into their own namespace and silently swallow its pushes. The
    ///         deadline is likewise required: the signed consent carries no timestamp of
    ///         its own, so an undated signature could re-establish a delegation AFTER the
    ///         pusher revoked it.
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
