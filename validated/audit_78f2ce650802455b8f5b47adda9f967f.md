### Title
`PriceProvider.confidenceParam` Zero-Initialized Default Causes Unclamped Bid/Ask Spread That Ignores Oracle Uncertainty — (File: `smart-contracts-poc/contracts/PriceProvider.sol`)

---

### Summary

`PriceProvider.confidenceParam` is a Solidity storage variable that zero-initializes by default. With `confidenceParam = 0`, the oracle's reported `spreadBps` (its own uncertainty signal) is multiplied by zero and completely discarded. The resulting bid/ask spread is determined solely by the immutable `marginStep`, which can be arbitrarily small. This is the direct analog of `amountOutMinimum = 0`: just as zero minimum output removes slippage protection for a swap caller, zero `confidenceParam` removes oracle-uncertainty protection for LPs, allowing traders to execute against prices that carry no oracle spread compensation.

---

### Finding Description

In `PriceProvider._getBidAndAskPrice()`, the oracle spread is incorporated as:

```solidity
uint256 adjustedSpread = spread * confidenceParam;
(uint256 bid, uint256 ask) = _getBidAskFrom(mid, adjustedSpread);
``` [1](#0-0) 

When `confidenceParam = 0` (the Solidity storage default for every newly deployed `PriceProvider`), `adjustedSpread = spread * 0 = 0` regardless of what the oracle reports for `spreadBps`. `_getBidAskFrom` then computes:

```solidity
uint256 delta = midPrice * confidence / CONFIDENCE_BASE; // = 0
bid = delta >= midPrice ? 0 : midPrice - delta;          // = mid
ask = midPrice + delta;                                   // = mid
``` [2](#0-1) 

After the step-factor adjustments with a positive `marginStep`, `bidOut < askOut` holds and a valid-looking bid/ask pair is returned to the pool — but with a spread determined only by `marginStep`, not by oracle uncertainty. The oracle's own `spreadBps` signal is silently zeroed out.

Critically, `PriceProvider` has **no band clamp**. `AnchoredPriceProvider` explicitly guards against this with:

```solidity
uint256 half = spreadBps * ONE_BPS_E18 + minMargin;
uint256 refBid = _bandEdge(mid, BPS_BASE_U - half, Math.Rounding.Floor);
uint256 refAsk = _bandEdge(mid, BPS_BASE_U + half, Math.Rounding.Ceil);
``` [3](#0-2) 

…which ensures the final quote is never tighter than `mid ± (spreadBps + minMargin)`. `PriceProvider` has no equivalent floor; the only spread is `marginStep`.

The `PriceProviderFactory.createPriceProvider()` is **permissionless** — any caller can deploy a `PriceProvider` and attach it to a pool:

```solidity
function createPriceProvider(
    address _oracle,
    bytes32 _feedId,
    int256  _marginStep,
    uint256 _maxTimeDelta,
    address _baseToken,
    address _quoteToken
) external override returns (address provider) {
``` [4](#0-3) 

No `confidenceParam` is set at construction. Every provider starts in the bad state.

---

### Impact Explanation

LPs in any pool backed by a `PriceProvider` with `confidenceParam = 0` are exposed to unbounded adverse selection. When the oracle reports high uncertainty (e.g., `spreadBps = 500`, meaning 5% uncertainty), the pool still quotes at the `marginStep`-only spread (e.g., 2 bps). Informed traders can trade against the pool at prices that do not compensate for oracle uncertainty, extracting LP principal. This matches the impact gate criterion: **"Bad-price execution: unclamped bid/ask quote reaches a pool swap."**

---

### Likelihood Explanation

Every `PriceProvider` instance starts with `confidenceParam = 0`. Pool creators who deploy a provider and open the pool before calling `setConfidenceParam` (which requires a separate factory transaction and a 1-minute cooldown) expose LPs immediately. There is no constructor guard, no deployment-time minimum, and no on-chain warning. The factory's `setConfidence` batch call is the only remediation path, and it is not enforced. [5](#0-4) [6](#0-5) 

---

### Recommendation

1. Require `confidenceParam > 0` as a constructor parameter in `PriceProvider` (passed through `createPriceProvider`), so no provider can be deployed in the zero-spread state.
2. Alternatively, add a band clamp analogous to `AnchoredPriceProvider`'s `refBid`/`refAsk` floor, ensuring the spread is always at least `spreadBps * ONE_BPS_E18` wide regardless of `confidenceParam`.
3. At minimum, revert in `getBidAndAskPrice()` if `confidenceParam == 0` and `marginStep == 0`, and document the zero-default risk prominently.

---

### Proof of Concept

1. Deploy `PriceProvider` via `PriceProviderFactory.createPriceProvider(oracle, feedId, marginStep=1e14, maxTimeDelta=3600, baseToken, quoteToken)`. `confidenceParam` is 0 by default.
2. Oracle reports `mid = 1e8` (price = 1.00), `spreadBps = 500` (5% uncertainty).
3. `adjustedSpread = 500 * 0 = 0`.
4. `_getBidAskFrom(1e8, 0)` → `bid = 1e8`, `ask = 1e8`.
5. After step adjustments with `marginStep = 1e14` (1 bps): `bidOut ≈ mid * Q64 * (1e18 - 1e14) / 1e26`, `askOut ≈ mid * Q64 * (1e18 + 1e14) / 1e26`. Spread ≈ 2 bps.
6. Pool quotes at 2 bps spread while oracle uncertainty is 5%.
7. An informed trader sells the overpriced asset into the pool at the ask (which is 5% below fair value relative to oracle uncertainty), extracting LP funds. The pool has no minimum-output equivalent to reject the trade. [7](#0-6)

    function _getBidAndAskPrice() internal returns (uint128, uint128) {
        // 1. Read via the unified price(feedId, pool) path, forwarding the pool (msg.sender).
        //    refTime is already in seconds.
        (uint256 mid, uint256 spread, , uint256 refTime) =
            IPricedOracle(address(offchainOracle)).price(offchainFeedId, msg.sender);

        // 2. Staleness check
        if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA)) {
            return (0, type(uint128).max);
        }

        // 3. Basic validity — price must be positive, spread must not be stalled marker
        if (mid == 0 || spread >= ORACLE_BPS) {
            return (0, type(uint128).max);
        }

        // 4. Price guard check (moved from oracle)
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(offchainFeedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) {
            return (0, type(uint128).max);
        }

        // 5. Compute bid/ask from mid + confidence-adjusted spread
        //    confidenceParam multiplies oracle spread; 0 means no spread
        uint256 adjustedSpread = spread * confidenceParam;
        (uint256 bid, uint256 ask) = _getBidAskFrom(mid, adjustedSpread);

        // 6. Apply marginStep adjustment
        (uint256 bidOut, bool bidOk) = _applyBidAdjustments(bid);
        if (!bidOk || bidOut > type(uint128).max) return (0, type(uint128).max);

        (uint256 askOut, bool askOk) = _applyAskAdjustments(ask);
        if (!askOk || askOut > type(uint128).max) return (0, type(uint128).max);

        // 7. Hard invariant: bid must be strictly less than ask.
        //    Can be violated when marginStep < 0 and confidence is too small.
        if (bidOut >= askOut) return (0, type(uint128).max);

        return (uint128(bidOut), uint128(askOut));
    }
```
