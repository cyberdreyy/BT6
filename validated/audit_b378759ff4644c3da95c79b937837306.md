### Title
Creator Can Replay Pusher Consent Signature to Re-Establish Delegation After `revokePusher()` — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowPushers` in `CompressedOracleV1` contains no nonce or revocation flag in its signature domain. A creator who holds a pusher's valid consent signature can call `allowPushers` repeatedly within the deadline window — including immediately after the pusher calls `revokePusher()` — silently re-establishing write authority over the creator's namespace. The pusher's self-revocation is therefore unilaterally reversible by the creator, leaving a compromised or unwanted pusher key permanently active until the deadline expires.

---

### Finding Description

`allowPushers` verifies a pusher's EIP-191 consent signature whose domain is:

```
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [1](#0-0) 

The domain commits to `(chainid, oracle, deadline, pusher, creator)` but contains **no nonce and no revocation flag**. Once a pusher signs consent with `deadline = T`, that single signature is valid for every call to `allowPushers` made before `T`, regardless of how many times `revokePusher()` has been called in between.

`revokePusher()` only clears `namespaceRemapping[pusher]` to `address(0)`:

```solidity
function revokePusher() external {
    address creator = namespaceRemapping[msg.sender];
    if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
    namespaceRemapping[msg.sender] = address(0);
    emit PusherRevoked(msg.sender, creator);
}
``` [2](#0-1) 

There is no on-chain record that the pusher ever revoked. The creator can immediately call `allowPushers` with the original signature to write `namespaceRemapping[pusher] = creator` again: [3](#0-2) 

The code's own comment acknowledges the risk but treats the deadline as the sole mitigation:

> *"The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it."* [4](#0-3) 

The deadline only bounds the outer window; it does not prevent replay **within** that window. A pusher who signed a 30-day or 1-year consent cannot unilaterally terminate their delegation before the deadline expires.

After re-establishment, the `fallback()` push path resolves the namespace from `namespaceRemapping[msg.sender]` and writes directly into the creator's storage slots:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
...
bytes32 key = bytes32(namespace | uint256(slotId));
...
_writeStorage(key, bytes32(bytes32(word & ~uint256(0xff))));
``` [5](#0-4) 

Those storage slots are the canonical source for `getOracleData` → `price()`, which pools consume directly during swaps: [6](#0-5) 

---

### Impact Explanation

A pusher whose private key is compromised calls `revokePusher()` to halt the damage. The creator — unaware of the compromise, or acting in bad faith — replays the original consent signature via `allowPushers`. The attacker retains write authority and pushes an arbitrary price/spread word into the creator's namespace. That corrupted value propagates through `getOracleData` → `price()` into any pool configured to use the affected feed, causing swaps to execute at a fabricated bid/ask. Traders receive more output than the true oracle permits (swap conservation failure) or the pool receives less input than owed, directly draining LP assets.

---

### Likelihood Explanation

- Pusher consent signatures with multi-day or multi-week deadlines are the normal operational pattern (the test suite uses `block.timestamp + 1 days`).
- The creator needs only to replay a signature they already hold — no new off-chain interaction with the pusher is required.
- The window is as long as the original deadline, which can be months.
- The scenario where a creator unknowingly re-establishes a compromised pusher (e.g., thinking the revocation was accidental) is realistic in production.

---

### Recommendation

Include a per-pusher revocation nonce in the consent signature domain and increment it on every `revokePusher()` / `removePushers()` call:

```solidity
mapping(address => uint256) public pusherNonce;

// In allowPushers:
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(
        block.chainid, address(this), deadline,
        pusher, msg.sender,
        pusherNonce[pusher]   // <-- add nonce
    ))
);

// In revokePusher / removePushers:
pusherNonce[pusher]++;
```

This invalidates all previously signed consents the moment the pusher revokes, making `revokePusher()` a true unilateral termination.

### Proof of Concept

```solidity
// 1. Pusher signs consent for creator with a 30-day deadline.
uint256 deadline = block.timestamp + 30 days;
bytes memory sig = _signConsent(PUSHER_KEY, deadline, pusher, creator);

// 2. Creator establishes delegation.
vm.prank(creator);
oracle.allowPushers(deadline, _arr(pusher), _arr(sig));
assertEq(oracle.namespaceRemapping(pusher), creator);

// 3. Pusher's key is compromised; pusher self-revokes.
vm.prank(pusher);
oracle.revokePusher();
assertEq(oracle.namespaceRemapping(pusher), address(0)); // revoked

// 4. Creator replays the SAME signature — no nonce prevents it.
vm.prank(creator);
oracle.allowPushers(deadline, _arr(pusher), _arr(sig));
assertEq(oracle.namespaceRemapping(pusher), creator);    // re-established!

// 5. Attacker uses compromised key to push a fabricated price.
uint56 tsMs = uint56(block.timestamp * 1000);
uint48 badRaw = _packRaw(1, 5, 0); // near-zero price
vm.prank(pusher /* attacker */);
(bool ok,) = address(oracle).call(_wordAt(0, 0, badRaw, tsMs));
assertTrue(ok);

// 6. Pool reads the corrupted feed and executes swaps at the bad price.
IOffchainOracle.OracleData memory data = oracle.getOracleData(
    oracle.feedIdOf(creator, 0, 0)
);
// data.price is now the attacker-controlled value — bad-price execution follows.
```

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
