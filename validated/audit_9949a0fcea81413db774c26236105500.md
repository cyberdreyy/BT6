### Title
Stale Updater Retains `confidenceParam` Write Access After Provider Ownership Transfer — (`File: smart-contracts-poc/contracts/PriceProviderFactory.sol`, `AnchoredProviderFactory.sol`, `PriceProviderFactoryL2.sol`)

---

### Summary

`transferProviderOwnership` updates `providerOwner[provider]` to the new owner but never clears the `isUpdater[provider][*]` entries granted by the previous owner. After the transfer, every address the old owner had authorized via `grantUpdater` retains the ability to call `setConfidence`, which directly writes `confidenceParam` on the live price provider. Because `isUpdater` is a plain nested mapping with no enumeration, the new owner has no on-chain mechanism to discover which stale updaters exist, and therefore cannot reliably revoke them.

---

### Finding Description

`_requireUpdater` passes for any caller that is either the current `providerOwner` **or** has `isUpdater[provider][caller] == true`: [1](#0-0) 

`transferProviderOwnership` replaces `providerOwner` and updates the `_providersByCreator` index, but makes no change to `isUpdater`: [2](#0-1) 

The identical omission exists in both sibling factories: [3](#0-2) [4](#0-3) 

`setConfidence` then routes through `_requireUpdater` and calls `setConfidenceParam` on the provider: [5](#0-4) 

`confidenceParam` is the multiplier applied to the oracle spread before the bid/ask is computed. In `PriceProvider` (no reference-band clamp), the shaped quote is:

```
delta = mid * (spreadBps * confidenceParam) / CONFIDENCE_BASE
bid   = mid - delta   (then stepBidFactor applied)
ask   = mid + delta   (then stepAskFactor applied)
``` [6](#0-5) 

`CONFIDENCE_MAX` is `1_000_000` (100× multiplier); the lower bound is `0` (spread collapses to `marginStep` only): [7](#0-6) 

Because `isUpdater` is a `mapping(address provider => mapping(address updater => bool))` with no enumeration support, the new owner cannot iterate over previously granted updaters: [8](#0-7) 

The test suite confirms the old **owner** loses access after transfer (`testOldOwnerCannotUpdateAfterTransfer`), but no test covers the case where an **updater** granted by the old owner retains access post-transfer: [9](#0-8) 

---

### Impact Explanation

For `PriceProvider` (used by `PriceProviderFactory` and `PriceProviderFactoryL2`), there is no reference-band clamp. A stale updater can set `confidenceParam = 0`, collapsing the effective spread to only `marginStep`. If `marginStep` is small (e.g., 1 bps), the pool quotes near-mid prices, exposing LPs to adverse selection and direct loss of principal. The stale updater can repeat this every `CONFIDENCE_COOLDOWN` (1 minute), continuously undermining the new owner's risk parameters.

For `AnchoredPriceProvider` (used by `AnchoredProviderFactory`), the reference-band clamp limits how tight the quote can be, so the impact is bounded to spread widening beyond the band — still a pool-usability and LP-value concern, but less severe.

The corrupted value is `confidenceParam`, which directly sets the bid/ask spread delivered to every pool swap that calls `getBidAndAskPrice()`. [10](#0-9) 

---

### Likelihood Explanation

The trigger requires: (1) a provider ownership transfer has occurred, (2) the previous owner had granted at least one updater, and (3) that updater acts against the new owner's interest. Ownership transfers are an explicit, documented feature (`transferProviderOwnership`). Updater grants are also a documented feature (`grantUpdater`). The combination is a realistic operational scenario (e.g., a provider sold or reassigned between teams). The stale updater does not need any privileged position beyond the `isUpdater` flag that was legitimately set before the transfer.

---

### Recommendation

Clear all `isUpdater` entries for the provider at transfer time, or — since the mapping is not enumerable — require the transferring owner to supply the list of updaters to revoke as part of the transfer call. A minimal fix is to add a `revokeAll` step or to change `transferProviderOwnership` to accept an explicit `address[] calldata updatersToRevoke` parameter that is processed atomically with the ownership change.

---

### Proof of Concept

1. Owner A deploys a `PriceProvider` via `PriceProviderFactory.createPriceProvider`. `providerOwner[p] = A`.
2. A calls `grantUpdater(p, U)`. `isUpdater[p][U] = true`.
3. A calls `transferProviderOwnership(p, B)`. `providerOwner[p] = B`. `isUpdater[p][U]` is **not cleared** — still `true`.
4. B sets `confidenceParam` to a risk-appropriate value (e.g., `500_000`) via `setConfidence`.
5. After `CONFIDENCE_COOLDOWN` (1 minute), U calls `setConfidence([p], [0])`. `_requireUpdater` passes because `isUpdater[p][U] == true`. `confidenceParam` is reset to `0`.
6. The pool's next swap calls `getBidAndAskPrice()`. With `confidenceParam = 0`, `delta = 0`, and the spread collapses to `marginStep` only. Traders receive near-mid prices; LPs suffer adverse-selection losses.
7. B cannot enumerate which updaters U exists to revoke, because `isUpdater` has no iteration interface. U can repeat step 5 every minute indefinitely. [11](#0-10)

        require(_providers.contains(provider), ProviderNotTracked());
        require(newOwner != address(0));
        address previousOwner = providerOwner[provider];

        providerOwner[provider] = newOwner;
        _providersByCreator[previousOwner].remove(provider);
        _providersByCreator[newOwner].add(provider);

    function setConfidenceParam(uint256 newValue) external override {
        require(msg.sender == factory, OnlyFactory());
        if (!MUTABLE_PARAMS) revert ImmutableProvider();
        if (newValue > CONFIDENCE_MAX) {
            revert ConfidenceParamOutOfBounds();
        }
        if (block.timestamp < lastConfidenceUpdate + CONFIDENCE_COOLDOWN) {
            revert CooldownNotElapsed();
        }

        confidenceParam = newValue;
        lastConfidenceUpdate = block.timestamp;
        emit ConfidenceParamSet(newValue);
    }
