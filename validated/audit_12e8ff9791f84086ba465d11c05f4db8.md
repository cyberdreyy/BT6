### Title
Atomic Oracle Price Sandwich via Deadline-Free `updateBySignature` Allows LP Value Extraction — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1.updateBySignature` is callable by **any** address that holds a valid feed-creator signature. The function carries **no deadline** and no per-call nonce; the only replay guard is the per-slot timestamp monotonicity check. Because signed slot words are intended to be publicly submittable (the whole point of the function), an attacker can intercept a pending signed price update from the off-chain feed and atomically sandwich it: swap at the old price, push the new price via `updateBySignature`, then swap back at the new price — extracting value from LPs in a single transaction.

---

### Finding Description

`updateBySignature` is a permissionless, state-mutating function: [1](#0-0) 

The signed message binds only `(chainid, oracleAddress, feedCreator, newSlotValue)`: [2](#0-1) 

There is **no deadline field** in the signature and **no deadline parameter** in the call. The documentation explicitly confirms this design: [3](#0-2) 

The only guards are:
1. `timestampMs.revertIfAfterBlockTimeWithDrift(MAX_TIME_DRIFT)` — prevents future-dated pushes.
2. Monotonicity — skips non-newer timestamps.

Neither guard prevents an attacker from **holding** a legitimately signed, current-timestamped slot word and submitting it at the exact moment that maximises their sandwich profit.

At swap time, `PriceProvider._getBidAndAskPrice()` reads the oracle state live: [4](#0-3) 

So a price update committed within the same transaction is immediately visible to the second swap leg.

---

### Impact Explanation

An attacker who monitors the off-chain signed-update stream can, in a single atomic transaction:

1. **Swap A→B** at the current oracle price P₁ (buy cheap).
2. **Call `updateBySignature`** with the intercepted signed slot word (price jumps to P₂ > P₁).
3. **Swap B→A** at the new oracle price P₂ (sell expensive).
4. **Pocket** `(P₂ − P₁) × notional − gas` at the direct expense of LPs.

This is a direct loss of LP principal — the pool receives less token-in than the oracle-anchored curve requires, or pays out more token-out than it should, depending on swap direction. The loss is bounded by the magnitude of the price jump and the liquidity depth, but for pools using `PriceProvider` (no anchor clamping) with a wide `priceGuard` or none at all, the loss is unbounded within a single block. [5](#0-4) 

---

### Likelihood Explanation

- `updateBySignature` is explicitly designed for **public submission** — signed slot words are broadcast off-chain so that any relayer can push them. An attacker needs only to monitor the same public channel.
- The attack requires no privileged role, no special token, and no malicious setup.
- It is profitable whenever a price jump exceeds gas cost, which is routine for any asset with infrequent but large oracle updates (the exact scenario the external report identifies as problematic).
- Pools on Ethereum mainnet using `PriceProvider` (not `AnchoredPriceProvider`) are directly exposed; `AnchoredPriceProvider` limits but does not eliminate the attack within the anchor band. [6](#0-5) 

---

### Recommendation

Add a **caller-supplied deadline** to `updateBySignature` and bind it into the signed digest, mirroring the pattern already used in `allowPushers`: [7](#0-6) 

Concretely:

```solidity
function updateBySignature(
    address feedCreator,
    uint256 deadline,          // NEW
    uint256 newSlotValue,
    bytes calldata signature
) external returns (bool) {
    _ensureDeadline(deadline); // NEW
    ...
    bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
        keccak256(abi.encode(block.chainid, address(this), feedCreator, deadline, newSlotValue))
    );
    ...
}
```

This forces the feed creator to issue short-lived signed updates (e.g., 30–60 s), making it impossible for an attacker to hold a valid signature across the window of a price jump. The registry ABI already anticipates this — it lists `updateBySignature(address,uint256,uint256,bytes)` with a `deadline` field — but the current source code omits it. [8](#0-7)

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

// Attacker contract — executes the full sandwich atomically.
contract OracleSandwichAttack {
    IMetricOmmPool   pool;
    CompressedOracleV1 oracle;

    // Pre-obtained from the public off-chain feed:
    address  feedCreator;
    uint256  newSlotValue;   // encodes price P2 > current P1, timestamp T2 > T1
    bytes    signature;      // signed by feedCreator over (chainid, oracle, feedCreator, newSlotValue)

    function attack(uint256 amountIn) external {
        // 1. Swap A→B at current oracle price P1
        pool.swap(/* A→B, amountIn */);

        // 2. Push the intercepted signed price update (P1 → P2)
        //    Anyone can call this — no deadline, no nonce.
        oracle.updateBySignature(feedCreator, newSlotValue, signature);

        // 3. Swap B→A at new oracle price P2 > P1
        pool.swap(/* B→A, full balance */);

        // 4. Net profit = (P2 - P1) * notional - gas
    }
}
```

The attack succeeds because:
- Step 2 is permissionless and the signature has no expiry.
- Step 3 reads the oracle state freshly, now reflecting P2.
- The pool's LP reserves absorb the loss.

```text
    /// @notice Single-slot update authorized by the creator's signature. The signed slot
    ///         word carries its own 56-bit timestamp, so replay is neutralized by the
    ///         monotonicity check below — no deadline is needed.
    function updateBySignature(address feedCreator, uint256 newSlotValue, bytes calldata signature)
        external
        override
        returns (bool)
    {
        require(feedCreator != address(0), InvalidNamespace());

        uint256 namespace;
        assembly ("memory-safe") {
            namespace := shl(96, feedCreator) // [creator:20][zeros:12]
        }

        uint8 slotId = uint8(newSlotValue); // LSB
        TimeMs timestampMs = toTimeMs(newSlotValue >> 8 & X56);
        timestampMs.revertIfAfterBlockTimeWithDrift(MAX_TIME_DRIFT);
        bytes32 key = bytes32(namespace | uint256(slotId));
        uint256 old = uint256(_loadStorage(key));
        TimeMs oldTimestampMs = toTimeMs(old >> 8 & X56);

        bool newer = timestampMs.isAfter(oldTimestampMs);
        if (!newer) {
            return false;
        }

        bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
            keccak256(abi.encode(block.chainid, address(this), feedCreator, newSlotValue))
        );
        require(feedCreator == ECDSA.recover(hash, signature));

        _writeStorage(key, bytes32(newSlotValue & ~uint256(0xff)));

        return true;
    }
```
