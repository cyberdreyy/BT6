### Title
Single-Step `transferProviderOwnership` Permanently Locks Provider Controls, Enabling Irreversible Pool Halt - (`smart-contracts-poc/contracts/AnchoredProviderFactory.sol`, `PriceProviderFactory.sol`, `PriceProviderFactoryL2.sol`)

---

### Summary

`transferProviderOwnership()` in all three provider factory contracts is a single-step operation that immediately overwrites `providerOwner[provider]` with the supplied address. There is no pending-accept step. If the current owner supplies a non-zero but inaccessible address (typo, wrong network address, burned key), ownership is irrecoverably lost. The new "owner" cannot call `setSource`, `grantUpdater`, `revokeUpdater`, or `setConfidence`. For any `AnchoredPriceProvider` that has a custom source set, the legitimate owner can no longer call `setSource(address(0))` to restore reference mode. If that source subsequently fails, every pool using the provider is permanently halted with `FeedStalled`, with no recovery path short of redeployment.

---

### Finding Description

`AnchoredProviderFactory.transferProviderOwnership` (and the identical implementations in `PriceProviderFactory` and `PriceProviderFactoryL2`) performs a direct, unconditional write:

```solidity
// AnchoredProviderFactory.sol lines 230-240
function transferProviderOwnership(address provider, address newOwner)
    external override onlyProviderOwner(provider)
{
    require(_providers.contains(provider), ProviderNotTracked());
    require(newOwner != address(0));
    address previousOwner = providerOwner[provider];

    providerOwner[provider] = newOwner;          // ← immediate, irreversible
    _providersByCreator[previousOwner].remove(provider);
    _providersByCreator[newOwner].add(provider);

    emit ProviderOwnershipTransferred(provider, previousOwner, newOwner);
}
```

The only guard is `newOwner != address(0)`. Any non-zero address — including one for which no private key exists — is accepted and immediately installed as the new owner.

By contrast, the pool admin transfer in `MetricOmmPoolFactory` is explicitly two-step:

```solidity
// MetricOmmPoolFactory.sol lines 510-526
function proposePoolAdminTransfer(address pool, address newAdmin) external ...
{
    pendingPoolAdmin[pool] = newAdmin;
    emit PoolAdminTransferProposed(...);
}

function acceptPoolAdmin(address pool) external ...
{
    if (msg.sender != pending) revert NotPendingPoolAdmin(...);
    poolAdmin[pool] = pending;
    ...
}
```

The protocol already knows the correct pattern for high-value role transfers; it was not applied to provider ownership.

The `providerOwner` role is the exclusive gatekeeper for:

- `setSource(provider, newSource)` — switches the `AnchoredPriceProvider` between custom-source mode and reference mode
- `grantUpdater` / `revokeUpdater` — controls who may update confidence parameters
- `setConfidence` — shapes the bid/ask spread delivered to pools
- `transferProviderOwnership` — the only recovery path

`AnchoredPriceProvider.setSource` is callable only through the factory (`OnlyFactory` guard), so there is no bypass:

```solidity
// AnchoredPriceProvider.sol line 178-181
function setSource(address newSource) external {
    require(msg.sender == factory, OnlyFactory());
    source = newSource;
}
```

---

### Impact Explanation

**Scenario leading to pool halt:**

1. Provider owner deploys an `AnchoredPriceProvider` and sets a custom source via `AnchoredProviderFactory.setSource`.
2. Owner calls `transferProviderOwnership(provider, wrongAddress)` — a non-zero address for which they hold no key (e.g., a checksummed address with a single character transposed).
3. Ownership is immediately and irrecoverably transferred.
4. The custom source later begins to fail (reverts, returns garbage, burns gas, or returns an ask > `uint128.max`).
5. `AnchoredPriceProvider._getBidAndAskPrice` fails closed on any of these conditions and `getBidAndAskPrice` reverts with `FeedStalled`.
6. Every pool whose `IPriceProvider` is this `AnchoredPriceProvider` cannot execute swaps — `getBidAndAsk` is called inside the swap path.
7. The legitimate owner cannot call `setSource(address(0))` to restore reference mode because `onlyProviderOwner` now points to the inaccessible address.
8. The pool is permanently halted for swaps. LP positions are stranded until the pool is redeployed.

The `AnchoredPriceProvider` band clamp prevents a bad source from quoting prices outside the reference band, but it does not prevent a failing source from halting the pool entirely — the fail-closed design is correct for price safety but makes the lockout permanent.

---

### Likelihood Explanation

- **Trigger**: The current `providerOwner` — a permissionless, semi-trusted actor who created the provider — makes a single address typo or pastes the wrong network's address.
- **No malicious intent required**: This is an operational error, identical in character to the `changeUberOwner()` scenario in the reference report.
- **Probability**: Low per individual transfer, but the function is callable by any provider creator across all three factory contracts on three deployment targets (Ethereum, Base, HyperEVM), multiplying exposure.
- **Irreversibility**: Once set, there is no admin override, no factory-level recovery, and no timelock to cancel.

---

### Recommendation

Replace the single-step transfer with a two-step propose/accept pattern, consistent with `MetricOmmPoolFactory.proposePoolAdminTransfer` / `acceptPoolAdmin`:

```solidity
mapping(address provider => address) public pendingProviderOwner;

function proposeProviderOwnership(address provider, address newOwner)
    external onlyProviderOwner(provider)
{
    require(newOwner != address(0));
    pendingProviderOwner[provider] = newOwner;
    emit ProviderOwnershipProposed(provider, providerOwner[provider], newOwner);
}

function acceptProviderOwnership(address provider) external {
    address pending = pendingProviderOwner[provider];
    require(msg.sender == pending, NotPendingProviderOwner());
    address previous = providerOwner[provider];
    providerOwner[provider] = pending;
    _providersByCreator[previous].remove(provider);
    _providersByCreator[pending].add(provider);
    delete pendingProviderOwner[provider];
    emit ProviderOwnershipTransferred(provider, previous, pending);
}
```

Apply identically to `PriceProviderFactory` and `PriceProviderFactoryL2`.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

// Demonstrates: single-step transferProviderOwnership locks out the owner,
// preventing source restoration when the custom source fails.

function test_lockout_halts_pool() public {
    // 1. Curator creates an AnchoredPriceProvider and sets a custom source
    vm.prank(curator);
    address provider = factory.createAnchoredProvider(...);
    vm.prank(curator);
    factory.setSource(provider, address(customSource));

    // 2. Curator accidentally transfers ownership to an inaccessible address
    address wrongAddress = address(0xDEAD000000000000000000000000000000000001);
    vm.prank(curator);
    factory.transferProviderOwnership(provider, wrongAddress);

    // 3. providerOwner is now wrongAddress — curator is locked out
    assertEq(factory.providerOwner(provider), wrongAddress);

    // 4. Custom source begins to fail
    customSource.setMode(MockAnchorSource.Mode.Revert);

    // 5. Pool swap halts — getBidAndAskPrice reverts with FeedStalled
    vm.expectRevert(AnchoredPriceProvider.FeedStalled.selector);
    pool.getBidAndAskPrice();

    // 6. Curator cannot restore reference mode — locked out
    vm.prank(curator);
    vm.expectRevert(IAnchoredProviderFactory.NotProviderOwner.selector);
    factory.setSource(provider, address(0)); // ← permanently blocked

    // Pool is permanently halted. No recovery without redeployment.
}

    address pending = pendingPoolAdmin[pool];

    if (msg.sender != pending) revert NotPendingPoolAdmin(pool, msg.sender, pending);

    poolAdmin[pool] = pending;
    delete pendingPoolAdmin[pool];
    emit PoolAdminTransferred(pool, previousAdmin, pending);
