### Title
`allowPushers` Consent Signature Replay Allows Revoked Creator to Re-Establish Pusher Delegation and Stale Downstream Pool Feeds — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracle.allowPushers` accepts a pusher's EIP-191 consent signature and unconditionally overwrites `namespaceRemapping[pusher]` with `msg.sender`. There is no nonce and no check that the mapping is currently unset. A creator who holds a pusher's old (still-deadline-valid) consent signature can replay it after the pusher has revoked and re-delegated to a new creator, silently redirecting all future pushes back to the old creator's namespace. The new creator's feeds stop updating, become stale, and every pool whose `AnchoredPriceProvider` reads those feeds halts with `FeedStalled`.

---

### Finding Description

`allowPushers` signs consent as:

```
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
```

The signed tuple binds `(chainid, oracle, deadline, pusher, creator)`. There is no per-pusher nonce and no revocation counter. The function body is:

```solidity
_ensureDeadline(deadline);
...
require(pusher == ECDSA.recover(hash, signatures[i]));
namespaceRemapping[pusher] = msg.sender;   // ← unconditional overwrite
emit PusherAuthorized(pusher, msg.sender);
``` [1](#0-0) 

`revokePusher` clears the mapping:

```solidity
namespaceRemapping[msg.sender] = address(0);
``` [2](#0-1) 

But `allowPushers` never checks whether `namespaceRemapping[pusher]` is already zero before writing. A creator who retains the pusher's old signature (deadline still in the future) can call `allowPushers` again at any time to overwrite the mapping back to themselves — even after the pusher has revoked and re-delegated to a different creator.

The code comment acknowledges the replay risk but frames the deadline as the sole mitigation:

> *"The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it."* [3](#0-2) 

The deadline prevents replay only after it expires. Within the deadline window — which the pusher may have set to days, weeks, or longer — the old creator can replay freely. The pusher has no on-chain mechanism to invalidate a specific consent before its deadline.

The `fallback` push path resolves the namespace at call time:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [4](#0-3) 

Once the mapping is overwritten back to C1, every subsequent push from P lands in C1's namespace. C2's namespace receives no further updates.

---

### Impact Explanation

After the replay:

1. `namespaceRemapping[P] = C1` (overwriting C2).
2. P's fallback pushes write to C1's storage slots; C2's slots freeze at their last value.
3. `AnchoredPriceProvider._isStale` returns `true` for C2's feeds once `MAX_REF_STALENESS` seconds elapse:

```solidity
function _isStale(uint256 refTime, uint256 nowTs, uint256 maxDelta) internal pure returns (bool) {
    if (refTime == 0) return true;
    if (refTime > nowTs) return true;
    return (nowTs - refTime) > maxDelta;
}
``` [5](#0-4) 

4. `_getBidAndAskPrice` returns `(0, type(uint128).max)` for the stale feed.
5. `getBidAndAskPrice` reverts with `FeedStalled`, blocking all swaps through every pool whose provider reads C2's feeds. [6](#0-5) 

This is broken core pool functionality: swap flows are permanently unusable until the pusher re-delegates (which C1 can immediately undo again with the same replay).

---

### Likelihood Explanation

- C1 must be malicious and must have retained P's signed consent bytes (trivially true — the signature was submitted on-chain in the original `allowPushers` call and is visible in calldata/events).
- The deadline must not have expired. Deadlines are caller-chosen with no enforced maximum; a pusher who signed a 30-day or 1-year deadline gives C1 a correspondingly long replay window.
- The pusher has no on-chain recourse: `revokePusher` clears the mapping but cannot invalidate the off-chain signature bytes that C1 already holds.
- The attack is a single transaction with no ETH cost beyond gas.

---

### Recommendation

Add a per-pusher revocation nonce to the consent signature domain:

```solidity
mapping(address => uint256) public pusherNonce;

// In allowPushers:
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(
        block.chainid, address(this), deadline,
        pusher, msg.sender,
        pusherNonce[pusher]   // ← bind to current nonce
    ))
);
...
namespaceRemapping[pusher] = msg.sender;

// In revokePusher / removePushers:
pusherNonce[pusher]++;   // ← invalidate all prior signatures
namespaceRemapping[pusher] = address(0);
```

Incrementing the nonce on every revocation makes all previously-signed consents immediately invalid, regardless of their deadline.

### Proof of Concept

```
T=0  P signs: sig1 = sign(chainid, oracle, deadline=T+365d, P, C1)
T=1  C1 calls allowPushers(T+365d, [P], [sig1])
       → namespaceRemapping[P] = C1  ✓

T=2  P calls revokePusher()
       → namespaceRemapping[P] = address(0)

T=3  P signs: sig2 = sign(chainid, oracle, deadline=T+30d, P, C2)
T=4  C2 calls allowPushers(T+30d, [P], [sig2])
       → namespaceRemapping[P] = C2  ✓

T=5  C1 replays: allowPushers(T+365d, [P], [sig1])   ← deadline still valid
       → namespaceRemapping[P] = C1  (overwrites C2)

T=6  P pushes slot word via fallback
       → creator = namespaceRemapping[P] = C1
       → data lands in C1's namespace; C2's namespace frozen

T=6 + MAX_REF_STALENESS + 1
     Pool calls AnchoredPriceProvider.getBidAndAskPrice()
       → _isStale(C2_refTime, now, MAX_REF_STALENESS) = true
       → returns (0, type(uint128).max)
       → revert FeedStalled()   ← all swaps blocked
``` [7](#0-6) [8](#0-7)

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }

        address creator = namespaceRemapping[msg.sender];

            end := calldatasize()

        // 4 * 6 + 7 + 1 = 32 bytes per slot

                word := calldataload(ptr)
