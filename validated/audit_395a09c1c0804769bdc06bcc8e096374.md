### Title
`PriceProvider` silently zeroes oracle spread when `confidenceParam` is at its default zero value, delivering an unclamped tight bid/ask to pool swaps — (File: `smart-contracts-poc/contracts/PriceProvider.sol`)

---

### Summary

`PriceProvider._getBidAndAskPrice()` multiplies the oracle-reported spread by `confidenceParam` before computing bid/ask. Because `confidenceParam` is a plain storage slot that is **zero-initialized** and has no minimum-value guard, any provider that is deployed (or intentionally left) with `confidenceParam == 0` silently discards the oracle's spread entirely. The resulting bid/ask is derived purely from the immutable `marginStep`, which can be far tighter than the oracle's actual uncertainty, delivering an unclamped quote to every pool swap.

---

### Finding Description

In `PriceProvider._getBidAndAskPrice()`, step 5 computes:

```solidity
uint256 adjustedSpread = spread * confidenceParam;   // confidenceParam == 0 → always 0
(uint256 bid, uint256 ask) = _getBidAskFrom(mid, adjustedSpread);
``` [1](#0-0) 

`_getBidAskFrom` with `confidence == 0` returns `bid = mid, ask = mid` (delta is zero):

```solidity
uint256 delta = midPrice * confidence / CONFIDENCE_BASE;  // 0
bid = delta >= midPrice ? 0 : midPrice - delta;           // mid
ask = midPrice + delta;                                   // mid
``` [2](#0-1) 

After `_applyBidAdjustments` / `_applyAskAdjustments` apply the immutable `marginStep`, if `marginStep > 0` the provider returns a **valid but spread-free** bid/ask whose width is determined solely by `marginStep`, completely ignoring the oracle's reported uncertainty. [3](#0-2) 

Unlike `AnchoredPriceProvider`, `PriceProvider` has **no** `minMargin` floor, **no** `MAX_SPREAD_BPS` circuit breaker, and **no** band clamp that would ensure the final quote is at least as wide as the oracle's spread: [4](#0-3) 

The code's own comment acknowledges the zero case — *"confidenceParam multiplies oracle spread; 0 means no spread"* — but provides no guard against it: [5](#0-4) 

The analog to the Lombard bug is exact: just as `validateLombardTransferRequest()` approved a transfer without verifying that unspent inputs were returned as change, `_getBidAndAskPrice()` approves a quote without verifying that the oracle's spread is conserved in the output.

---

### Impact Explanation

A pool using `PriceProvider` with `confidenceParam == 0` and `marginStep > 0` receives quotes whose spread is fixed at `marginStep` regardless of oracle-reported uncertainty. During high-volatility periods the oracle may report a large spread (e.g. 500 bps), but the pool quotes only `marginStep` (e.g. 50 bps). Traders can execute at prices the oracle considers uncertain, extracting value from LPs. This is a **bad-price execution** impact: the bid/ask reaching the pool swap is unclamped relative to oracle uncertainty, satisfying the allowed-impact gate.

---

### Likelihood Explanation

`confidenceParam` is a plain `uint256` storage slot initialized to zero by the EVM. Every freshly deployed `PriceProvider` with `marginStep > 0` is immediately vulnerable until the factory calls `setConfidenceParam`. There is no constructor-time requirement, no `require(confidenceParam > 0)` guard in `_getBidAndAskPrice`, and no timelock or deployment ceremony that forces the factory to configure it before the pool opens. The trigger requires no privileged action by an attacker — the default state is the vulnerable state. [6](#0-5) 

---

### Recommendation

Add a minimum-spread floor in `_getBidAndAskPrice()` analogous to `AnchoredPriceProvider`'s `minMargin`, or require `confidenceParam > 0` before computing the bid/ask:

```solidity
// Option A: require non-zero confidence before quoting
if (confidenceParam == 0) return (0, type(uint128).max);

// Option B: enforce a minimum half-width (minMargin, set at construction)
uint256 effectiveSpread = Math.max(adjustedSpread, minSpread);
```

Either change ensures the oracle's uncertainty is conserved in the output quote, closing the analog to the Lombard change-validation gap.

---

### Proof of Concept

1. Deploy `PriceProvider` with `marginStep = 5e16` (50 bps) and leave `confidenceParam` at its default `0`.
2. Oracle reports `mid = 1_000e8`, `spread = 500` (500 bps, high-volatility session).
3. `adjustedSpread = 500 * 0 = 0` → `bid = ask = 1_000e8`.
4. After `marginStep` adjustment: `bidOut ≈ 950 * Q64 / 1e8`, `askOut ≈ 1_050 * Q64 / 1e8` — a 50 bps spread.
5. Pool opens for swaps. Oracle uncertainty is 500 bps but pool quotes 50 bps.
6. Attacker swaps at the tight quote, receiving tokens at a price the oracle considers 450 bps outside its uncertainty band.
7. LPs absorb the loss; the pool's swap conservation invariant is broken relative to the oracle's own reported uncertainty. [7](#0-6)

    function _getBidAndAskPrice() internal returns (uint128, uint128) {
        // 1. Read via the unified price(feedId, pool) path, forwarding the pool (msg.sender).
        //    refTime is already in seconds.
        (uint256 mid, uint256 spread, , uint256 refTime) =
            IPricedOracle(address(offchainOracle)).price(offchainFeedId, msg.sender);

        // 2. Staleness check
        if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA)) {
            return (0, type(uint128).max);
        }

        // 3. Basic validity — price must be positive, spread must not be stalled marker
        if (mid == 0 || spread >= ORACLE_BPS) {
            return (0, type(uint128).max);
        }

        // 4. Price guard check (moved from oracle)
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(offchainFeedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) {
            return (0, type(uint128).max);
        }

        // 5. Compute bid/ask from mid + confidence-adjusted spread
        //    confidenceParam multiplies oracle spread; 0 means no spread
        uint256 adjustedSpread = spread * confidenceParam;
        (uint256 bid, uint256 ask) = _getBidAskFrom(mid, adjustedSpread);

        // 6. Apply marginStep adjustment
        (uint256 bidOut, bool bidOk) = _applyBidAdjustments(bid);
        if (!bidOk || bidOut > type(uint128).max) return (0, type(uint128).max);

        (uint256 askOut, bool askOk) = _applyAskAdjustments(ask);
        if (!askOk || askOut > type(uint128).max) return (0, type(uint128).max);

        // 7. Hard invariant: bid must be strictly less than ask.
        //    Can be violated when marginStep < 0 and confidence is too small.
        if (bidOut >= askOut) return (0, type(uint128).max);

        return (uint128(bidOut), uint128(askOut));
    }
```
