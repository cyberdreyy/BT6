### Title
Identical `baseFeedId` and `quoteFeedId` in Synthetic Mode Locks Ratio Price at 1.0, Enabling LP Fund Drain — (`smart-contracts-poc/contracts/AnchoredPriceProvider.sol`)

---

### Summary

`AnchoredPriceProvider` supports a two-feed "synthetic ratio" mode where `mid = price(baseFeedId) / price(quoteFeedId)`. Neither the provider constructor nor `AnchoredProviderFactory.createAnchoredProvider` checks that `baseFeedId != quoteFeedId` when `quoteFeedId != bytes32(0)`. If both feeds are set to the same value, the ratio is always exactly `ORACLE_DECIMALS` (1e8 = 1.0 in 8-decimal format), regardless of actual market prices. Every swap in a pool using this provider executes at price 1.0, allowing traders to drain LP principal.

---

### Finding Description

`AnchoredPriceProvider._getBidAndAskPrice()` computes the synthetic ratio as:

```solidity
mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);
``` [1](#0-0) 

When `baseFeedId == quoteFeedId`, both `_readLeg` calls hit the same oracle slot, returning identical values. The division `mid / mid * ORACLE_DECIMALS` collapses to exactly `ORACLE_DECIMALS = 1e8` for any non-zero price. The resulting Q64 bid/ask is computed from this constant 1.0 mid, completely decoupled from the real market price.

The constructor stores both feed IDs without any equality guard:

```solidity
baseFeedId = _baseFeedId;
quoteFeedId = _quoteFeedId;
``` [2](#0-1) 

The factory's `createAnchoredProvider` is permissionless (any caller may invoke it once the admin has added an oracle to the allow-list). It validates envelope bounds and oracle membership but performs no check on feed identity: [3](#0-2) 

The token-pair check (`_baseToken != _quoteToken`) exists but guards only the ERC-20 addresses, not the feed IDs: [4](#0-3) 

---

### Impact Explanation

A pool paired with this provider (e.g., BTC/ETH) receives a constant bid/ask centered on 1.0 instead of the real BTC/ETH ratio (~15–20). Traders can buy BTC from the pool at 1 ETH and immediately sell it on any external market for 15+ ETH. Every swap extracts value from LP deposits until the pool is insolvent. This is a direct, unbounded loss of LP principal — matching the "pool insolvency / bad-price execution" impact gate.

---

### Likelihood Explanation

`createAnchoredProvider` is permissionless once an admin has added any oracle to the allow-list (a routine, legitimate action). Any caller — including one who makes an honest configuration mistake — can deploy a provider with `baseFeedId == quoteFeedId`. The resulting provider passes `AnchoredProviderFactory.isProvider()` and is eligible for use in a public pool. No further privileged step is required after the oracle is allow-listed. [5](#0-4) 

---

### Recommendation

Add the following guard in `AnchoredPriceProvider`'s constructor (and mirror it in `createAnchoredProvider`):

```solidity
require(
    _quoteFeedId == bytes32(0) || _baseFeedId != _quoteFeedId,
    "baseFeedId and quoteFeedId must differ in synthetic mode"
);
``` [6](#0-5) 

---

### Proof of Concept

1. Admin calls `addOracle(pythOracle)` and `setEnvelope(DEFAULT_CLASS, ...)` — routine setup.
2. Attacker (or misconfiguring user) calls:
   ```
   createAnchoredProvider(
       oracle    = pythOracle,
       baseFeedId  = BTC_USD_FEED,
       quoteFeedId = BTC_USD_FEED,   // ← same feed
       baseToken = BTC,
       quoteToken = ETH,
       ...
   )
   ```
3. Provider is deployed and `isProvider() == true`.
4. A pool is created with this provider for the BTC/ETH pair.
5. `_getBidAndAskPrice()` reads `mid1 = mid2 = P_BTC_USD`; computes `mid = P_BTC_USD * 1e8 / P_BTC_USD = 1e8` (price = 1.0).
6. Pool quotes 1 BTC = 1 ETH. Trader buys 1 BTC for 1 ETH, sells on Uniswap for 15 ETH. Repeats until pool is drained. [7](#0-6)

    function _getBidAndAskPrice() internal returns (uint128, uint128) {
        (uint256 mid, uint256 spreadBps, , bool ok) = _readLeg(baseFeedId);
        if (!ok) return (0, type(uint128).max);

        bytes32 _quote = quoteFeedId;
        if (_quote != bytes32(0)) {
            (uint256 mid2, uint256 spreadBps2, , bool ok2) = _readLeg(_quote);
            if (!ok2 || mid2 == 0) return (0, type(uint128).max);
            // Synthetic ratio (8-decimal): mid1 / mid2. Relative uncertainties of a ratio add.
            mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);
            spreadBps += spreadBps2;
        }

        return _computeBidAsk(mid, spreadBps);
    }
