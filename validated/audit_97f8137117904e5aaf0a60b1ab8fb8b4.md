### Title
Pusher Revocation Bypass via Signature Replay in `allowPushers` Allows Compromised Pusher to Continue Feeding Bad Prices — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1.allowPushers` accepts the same EIP-191 pusher-consent signature an unlimited number of times before its deadline expires. A pusher who calls `revokePusher()` to stop a compromised key from writing prices can have that revocation immediately undone by the creator re-submitting the original signature. The broken invariant is that `revokePusher` is supposed to be a terminal, pusher-initiated stop; instead it is a no-op whenever the creator (or an automated keeper) holds a still-valid signature.

---

### Finding Description

`allowPushers` commits the signed consent to `(chainid, oracle, deadline, pusher, creator)` and checks only that `deadline >= block.timestamp`:

```solidity
// CompressedOracle.sol L192-L211
function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
    _ensureDeadline(deadline);
    ...
    bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
        keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
    );
    require(pusher == ECDSA.recover(hash, signatures[i]));
    namespaceRemapping[pusher] = msg.sender;   // ← re-establishes delegation
    ...
}
```

There is no nonce, no used-signature bitmap, and no per-pusher revocation flag. `revokePusher` only zeroes the mapping:

```solidity
// CompressedOracle.sol L238-L243
function revokePusher() external {
    address creator = namespaceRemapping[msg.sender];
    if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
    namespaceRemapping[msg.sender] = address(0);
    emit PusherRevoked(msg.sender, creator);
}
```

Because `allowPushers` carries no replay protection, the creator can call it again with the identical `(deadline, pusher, sig)` tuple and restore `namespaceRemapping[pusher] = creator` in the very next block. The documentation itself flags the risk ("an undated signature could re-establish a delegation AFTER the pusher revoked it") but the deadline only closes the window after expiry — it does nothing to prevent replay within the window.

Once the delegation is live again, the compromised pusher key writes into the creator's namespace via the `fallback` push path:

```solidity
// CompressedOracle.sol L315-L316
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
```

The written slot value is then consumed by `PriceProvider._getBidAndAskPrice()` (or `AnchoredPriceProvider._readLeg`) through `IPricedOracle.price(feedId, pool)`, which decodes the compressed price and passes it through staleness, price-guard, and spread checks before producing the bid/ask delivered to the pool swap.

---

### Impact Explanation

An attacker who has obtained a pusher's private key can push arbitrary prices into the creator's oracle namespace. If the creator (or an automated keeper bot that monitors and re-establishes delegations) calls `allowPushers` with the original signature after the pusher's self-revocation, the attacker regains write access. Prices that pass the staleness window and fall within a wide or unset price guard reach `getBidAndAskPrice()` and are used directly for swap execution, causing traders to receive worse-than-market fills and LPs to suffer adverse-selection losses.

---

### Likelihood Explanation

Requires two conditions: (1) a compromised pusher EOA key, and (2) the creator or an automated keeper re-submitting the original consent signature after revocation. Keeper bots that maintain delegations are a common operational pattern; they would re-establish the delegation without awareness of the compromise. The window of exposure equals the remaining lifetime of the original deadline, which can be up to the maximum the creator chose (no protocol cap on deadline length is enforced).

---

### Recommendation

Track consumed signatures with a `mapping(bytes32 => bool) private _usedConsents` keyed on the full consent hash, and revert if the hash has already been recorded. Alternatively, introduce a per-pusher monotonic nonce that the pusher signs, so each consent is single-use and a revocation permanently invalidates all prior signatures for that pusher–creator pair.

---

### Proof of Concept

```
T=0  Creator calls allowPushers(deadline=T+30days, [pusher], [sig])
     → namespaceRemapping[pusher] = creator  ✓

T=1  Attacker (holding pusher key) pushes manipulated slot word via fallback()
     → creator's oracle slot overwritten with bad price

T=2  Pusher (legitimate owner) calls revokePusher()
     → namespaceRemapping[pusher] = address(0)

T=3  Creator's keeper bot detects revocation, calls allowPushers(deadline=T+30days, [pusher], [sig])
     (same deadline, same sig — no replay guard)
     → namespaceRemapping[pusher] = creator  ← delegation restored

T=4  Attacker pushes another bad price via fallback()
     → PriceProvider reads bad mid, passes staleness + guard checks,
       delivers manipulated bid/ask to pool swap
     → trader receives worse-than-market execution; LP suffers adverse selection

        uint256 namespace;

        address creator = namespaceRemapping[msg.sender];

        // 4 * 6 + 7 + 1 = 32 bytes per slot

    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
