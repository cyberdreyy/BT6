### Title
Pusher consent signature has no nonce, allowing creator to replay `allowPushers` and override a pusher's `revokePusher()` before deadline expiry — (File: `smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1.allowPushers` verifies a pusher's EIP-191 consent signature and maps the pusher into the caller's (creator's) namespace. The signature domain is `(chainid, oracle, deadline, pusher, creator)` — there is no nonce. Because the same signature remains valid for any call before the deadline, a creator can replay it immediately after the pusher calls `revokePusher()`, silently re-establishing the delegation. `revokePusher()` is therefore ineffective until the deadline expires. If the pusher is actively pushing prices for their own namespace after revocation, those pushes are redirected into the creator's namespace, feeding wrong prices into the creator's pool.

---

### Finding Description

`allowPushers` is a permissionless function — any address may call it — but the signature binds the pusher's consent to a specific `(chainid, oracle, deadline, pusher, msg.sender)` tuple, so only the intended creator can submit a valid signature. [1](#0-0) 

The code's own comment acknowledges the replay risk:

> "The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it." [2](#0-1) 

However, the deadline only prevents replay **after** it expires. Before expiry, the same signature can be submitted an unlimited number of times. `revokePusher()` clears `namespaceRemapping[msg.sender]` to `address(0)`: [3](#0-2) 

But the creator can immediately call `allowPushers` again with the identical `(deadline, pusher, sig)` tuple, writing `namespaceRemapping[pusher] = creator` again: [4](#0-3) 

The fallback push path resolves the namespace at call time: [5](#0-4) 

So every push the pusher makes after revocation — intended for `feedIdOf(pusher, slotId, pos)` — is silently written to `feedIdOf(creator, slotId, pos)` instead.

**Attack sequence:**

1. Pusher signs consent for creator with `deadline = T` (e.g. 7 days out).
2. Creator calls `allowPushers(T, [pusher], [sig])` → `namespaceRemapping[pusher] = creator`.
3. Pusher calls `revokePusher()` → `namespaceRemapping[pusher] = address(0)`.
4. Creator calls `allowPushers(T, [pusher], [sig])` again with the **same** signature → `namespaceRemapping[pusher] = creator` restored.
5. Steps 3–4 repeat indefinitely until `block.timestamp > T`.

After step 4, the pusher's fallback pushes land in the creator's namespace. If the pusher has started pushing prices for a different asset pair (their own pool), those prices overwrite the creator's feed slots. The creator's pool then reads the wrong prices on the next swap.

---

### Impact Explanation

- **Bad-price execution:** The creator's pool reads prices from `feedIdOf(creator, slotId, pos)`. After the delegation is forcibly re-established, the pusher's pushes (intended for a different asset or different parameters) overwrite those slots. A pool swap consuming the corrupted feed executes at a wrong bid/ask, causing direct loss of trader principal.
- **Stale prices on pusher's own pool:** The pusher's own pool reads from `feedIdOf(pusher, slotId, pos)`, which receives no updates because all pushes are redirected. Swaps on the pusher's pool revert on staleness checks or execute at stale prices.
- **`revokePusher()` is a no-op:** The pusher has no on-chain mechanism to permanently withdraw consent before the deadline; the creator can always replay the old signature.

---

### Likelihood Explanation

- The original `allowPushers` transaction is public on-chain; the creator trivially retains the calldata.
- Re-calling `allowPushers` with the same arguments costs one transaction and no additional off-chain work.
- The scenario is realistic whenever a pusher relationship ends mid-deadline (key rotation, business dispute, pusher switching to their own pool).
- No privileged role is required; the creator is a normal user.

---

### Recommendation

Add a per-`(pusher, creator)` nonce to the signed message and increment it on each successful delegation. This makes every consent signature single-use:

```solidity
// storage
mapping(address => mapping(address => uint256)) public delegationNonce; // pusher => creator => nonce

// inside allowPushers, replace the hash construction:
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(
        block.chainid,
        address(this),
        deadline,
        pusher,
        msg.sender,
        delegationNonce[pusher][msg.sender]++   // <-- consume nonce
    ))
);
```

The pusher signs over the current nonce; after one successful `allowPushers` call the nonce advances and the old signature is permanently invalid, so `revokePusher()` cannot be overridden.

### Proof of Concept

```solidity
// Foundry test — extends the existing CompressedOracleTest helpers

function testAllowPushersReplayOverridesRevoke() public {
    uint256 deadline = block.timestamp + 7 days;

    // 1. Pusher signs consent for creator
    bytes memory sig = _signConsent(PUSHER_KEY, deadline, pusher, creator);
    address[] memory pushers = new address[](1);
    pushers[0] = pusher;
    bytes[] memory sigs = new bytes[](1);
    sigs[0] = sig;

    // 2. Creator establishes delegation
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs);
    assertEq(oracle.namespaceRemapping(pusher), creator, "delegation set");

    // 3. Pusher self-revokes
    vm.prank(pusher);
    oracle.revokePusher();
    assertEq(oracle.namespaceRemapping(pusher), address(0), "revoked");

    // 4. Creator replays the SAME signature — revocation overridden
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs);
    assertEq(oracle.namespaceRemapping(pusher), creator, "VULN: revoke overridden");

    // 5. Pusher pushes prices intended for their own pool (different asset)
    uint56 tsMs = uint56(block.timestamp * 1000);
    uint48 wrongPrice = _packRaw(999_999_999, 5, 0); // pusher's own asset price
    vm.prank(pusher);
    (bool ok,) = address(oracle).call(_wordAt(0, 0, wrongPrice, tsMs));
    assertTrue(ok);

    // Creator's pool reads the wrong price
    IOffchainOracle.OracleData memory data =
        oracle.getOracleData(oracle.feedIdOf(creator, 0, 0));
    assertEq(
        data.price,
        U64x32.decode(uint32(wrongPrice >> 16)),
        "VULN: wrong price in creator namespace"
    );

    // Pusher's own namespace is empty — their pool gets stale prices
    IOffchainOracle.OracleData memory own =
        oracle.getOracleData(oracle.feedIdOf(pusher, 0, 0));
    assertEq(own.price, 0, "pusher's pool starved of updates");
}
```

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
