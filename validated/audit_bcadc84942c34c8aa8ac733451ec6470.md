### Title
Pusher Delegation Signature Replay Allows Creator to Silently Override `revokePusher()` — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowPushers` in `CompressedOracle.sol` has no nonce or used-signature tracking. A creator who holds a pusher's valid consent signature can replay it after the pusher calls `revokePusher()`, immediately re-establishing the delegation. The pusher's revocation is rendered ineffective for the entire remaining deadline window, and every subsequent push the pusher makes — believing it lands in their own namespace — is silently redirected into the creator's namespace and consumed by any pool backed by that creator's feeds.

---

### Finding Description

The `allowPushers` function performs two checks before writing `namespaceRemapping[pusher] = msg.sender`:

1. `_ensureDeadline(deadline)` — the deadline has not yet passed.
2. ECDSA recovery over `keccak256(abi.encode(chainid, oracle, deadline, pusher, creator))` — the signature is authentic. [1](#0-0) 

Neither check prevents the same `(deadline, sig)` pair from being submitted a second time. There is no nonce, no `usedSignatures` bitmap, and no check that `namespaceRemapping[pusher]` is currently `address(0)` before overwriting it.

The code comment on lines 186–191 explicitly acknowledges the risk:

> *"the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it."* [2](#0-1) 

Yet no mitigation is implemented. `_ensureDeadline` only prevents use after expiry; it does nothing to prevent replay within the deadline window. [3](#0-2) 

`revokePusher` clears the mapping to `address(0)`: [4](#0-3) 

But the creator can immediately call `allowPushers` again with the identical `(deadline, sig)` tuple, restoring `namespaceRemapping[pusher] = creator` without any new consent from the pusher.

The `fallback` push path resolves the namespace at call time: [5](#0-4) 

So every push the pusher makes after their (ineffective) revocation still lands in the creator's namespace, not their own.

---

### Impact Explanation

The creator's namespace feeds are the direct input to price providers (`PriceProvider`, `ProtectedPriceProvider`, `PriceProviderL2`, `ProtectedPriceProviderL2`) which call `IPricedOracle.price(offchainFeedId, pool)` and forward the result as the live bid/ask to pool swaps. [6](#0-5) 

A pusher who revokes because they believe the creator's pool is misusing their data — or because they intend to push into their own namespace for a different purpose — will unknowingly continue feeding the creator's pool. Any price the pusher pushes (intended for their own namespace) is silently written into the creator's namespace and consumed by the creator's pool swaps. This is a direct bad-price-execution path: the pool receives prices from a pusher who explicitly withdrew consent, and the pusher has no on-chain mechanism to stop it until the deadline expires.

---

### Likelihood Explanation

The trigger is fully unprivileged: the creator is a regular namespace owner, not an oracle admin. The only precondition is that the creator retained the pusher's original consent bytes (trivially available from calldata of the first `allowPushers` call). Any creator who issued a delegation with a multi-day or multi-week deadline can exploit this window. The pusher has no way to detect the re-delegation without monitoring chain state.

---

### Recommendation

Track consumed signatures to prevent replay. The minimal fix is a `mapping(bytes32 => bool) public usedDelegationSignatures` keyed on the signature hash, checked and set inside `allowPushers`:

```solidity
bytes32 sigHash = keccak256(signatures[i]);
require(!usedDelegationSignatures[sigHash], SignatureAlreadyUsed());
usedDelegationSignatures[sigHash] = true;
```

Alternatively, include a per-pusher nonce in the signed digest and increment it on each successful delegation, so a revoked pusher can invalidate all prior signatures by incrementing their nonce.

---

### Proof of Concept

```solidity
// Setup: pusher signs consent with a 30-day deadline
uint256 deadline = block.timestamp + 30 days;
bytes memory sig = _signConsent(PUSHER_KEY, deadline, pusher, creator);

// Step 1: creator establishes delegation
vm.prank(creator);
oracle.allowPushers(deadline, _arr(pusher), _arr(sig));
assertEq(oracle.namespaceRemapping(pusher), creator);

// Step 2: pusher revokes
vm.prank(pusher);
oracle.revokePusher();
assertEq(oracle.namespaceRemapping(pusher), address(0)); // cleared

// Step 3: creator replays the IDENTICAL (deadline, sig) — succeeds with no revert
vm.prank(creator);
oracle.allowPushers(deadline, _arr(pusher), _arr(sig));
assertEq(oracle.namespaceRemapping(pusher), creator); // re-established!

// Step 4: pusher pushes, believing they are writing to their own namespace
uint56 tsMs = uint56(block.timestamp * 1000);
uint48 raw = _packRaw(1_000_000, 3, 3);
vm.prank(pusher);
(bool ok,) = address(oracle).call(_wordAt(1, 0, raw, tsMs));
assertTrue(ok);

// Data lands in CREATOR's namespace, not pusher's own
assertEq(
    oracle.getOracleData(oracle.feedIdOf(creator, 1, 0)).price,
    U64x32.decode(uint32(raw >> 16))   // creator's feed updated
);
assertEq(
    oracle.getOracleData(oracle.feedIdOf(pusher, 1, 0)).price,
    0                                   // pusher's own namespace empty
);
// Any pool backed by creator's feedIdOf(creator,1,0) now receives
// prices the pusher intended to stop providing.

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
