### Title
Revoked Pusher Delegation Can Be Silently Re-Established via Old Consent Signature Replay — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`allowPushers` in `CompressedOracleV1` accepts any EIP-191 consent signature whose deadline has not yet expired. Because the signed payload contains no nonce and no "used" flag is ever recorded, a creator can replay the pusher's original consent signature an unlimited number of times within the deadline window — including after the pusher has called `revokePusher()`. The pusher's revocation is therefore ineffective until the original deadline expires, and any price data the pusher continues to push lands in the creator's namespace rather than the pusher's own.

### Finding Description

The consent message signed by a pusher is:

```
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [1](#0-0) 

There is no nonce, no per-signature consumed flag, and no on-chain record that this exact `(deadline, pusher, creator)` tuple was already used. The only guard is `_ensureDeadline`, which checks `block.timestamp <= deadline`: [2](#0-1) 

After `revokePusher()` clears `namespaceRemapping[pusher] = address(0)`: [3](#0-2) 

…the creator can immediately call `allowPushers` again with the identical `(deadline, [pusher], [oldSig])` arguments. Because the deadline is still valid and the signature still verifies, `namespaceRemapping[pusher]` is set back to `creator`: [4](#0-3) 

The code comment on `allowPushers` explicitly acknowledges the concern ("an undated signature could re-establish a delegation AFTER the pusher revoked it") and states the deadline is the mitigation — but the deadline only prevents use *after* expiry; it does not prevent replay *within* the deadline window. [5](#0-4) 

The `fallback` push path resolves the namespace at call time:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [6](#0-5) 

So every push the pusher makes after their revocation — believing they are writing into their own namespace — continues to land in the creator's namespace, feeding the creator's oracle feeds and ultimately the pools that consume them.

### Impact Explanation

A pusher who revokes their delegation (e.g., because their signing key is suspected compromised, or because they are ending their service relationship) cannot actually stop their pushes from reaching the creator's namespace until the original consent deadline expires. If the pusher's key is compromised, an attacker holding that key can continue to push arbitrary prices into the creator's oracle feeds. Those feeds are consumed by `AnchoredPriceProvider` / `ProtectedPriceProvider` → `getBidAndAskPrice()` → pool swaps. Bad prices reaching live swaps constitute bad-price execution and direct loss of user principal.

### Likelihood Explanation

The trigger requires the creator to call `allowPushers` with the old signature after the pusher revokes. This is a semi-trusted actor action, but it is a single permissionless transaction with no additional privilege beyond holding the creator address. In a key-compromise scenario the creator may not even be aware the pusher revoked, and may re-establish the delegation innocently. Deadlines in practice are set days to weeks in the future (the test suite uses `block.timestamp + 1 days`), leaving a wide replay window. [7](#0-6) 

### Recommendation

Track consumed `(pusher, creator, deadline)` tuples with a `mapping(bytes32 => bool) usedConsents` and revert if the same tuple is replayed:

```solidity
bytes32 consentKey = keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender));
require(!usedConsents[consentKey], ConsentAlreadyUsed());
usedConsents[consentKey] = true;
```

Alternatively, include a per-pusher nonce in the signed payload and increment it on each successful `allowPushers` call, so that any prior signature is invalidated the moment the delegation is established (and therefore also after any subsequent revocation).

### Proof of Concept

```solidity
// 1. Pusher signs consent with deadline = now + 1 day
uint256 deadline = block.timestamp + 1 days;
bytes memory sig = _signConsent(PUSHER_KEY, deadline, pusher, creator);

// 2. Creator establishes delegation
address[] memory pushers = new address[](1); pushers[0] = pusher;
bytes[] memory sigs = new bytes[](1);        sigs[0] = sig;
vm.prank(creator);
oracle.allowPushers(deadline, pushers, sigs);
assertEq(oracle.namespaceRemapping(pusher), creator); // delegated

// 3. Pusher self-revokes
vm.prank(pusher);
oracle.revokePusher();
assertEq(oracle.namespaceRemapping(pusher), address(0)); // cleared

// 4. Creator replays the SAME old signature — deadline still valid, no nonce check
vm.prank(creator);
oracle.allowPushers(deadline, pushers, sigs);           // succeeds — BUG
assertEq(oracle.namespaceRemapping(pusher), creator);   // delegation silently restored

// 5. Pusher's next fallback push lands in creator's namespace, not their own
uint56 tsMs = uint56(block.timestamp * 1000);
uint48 raw  = _packRaw(BAD_PRICE, 5, 0);
vm.prank(pusher);
(bool ok,) = address(oracle).call(_wordAt(0, 0, raw, tsMs));
assertTrue(ok);
// BAD_PRICE is now live in creator's feed, consumed by pools on next swap
assertGt(oracle.getOracleData(oracle.feedIdOf(creator, 0, 0)).price, 0);
assertEq(oracle.getOracleData(oracle.feedIdOf(pusher,  0, 0)).price, 0);
```

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
```
