### Title
Contract-Pusher `isPusher` Consent Verified Only at Delegation Time; Stale `namespaceRemapping` Survives Pusher-Contract Ownership Transfer, Enabling Arbitrary Bad-Price Injection Into Pools — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowContractPushers` proves consent by a one-time live `isPusher(creator)` staticcall and then permanently writes `namespaceRemapping[pusher] = creator`. The fallback push path reads only `namespaceRemapping[msg.sender]` — it never re-checks `isPusher`. If the pusher contract's internal authorization later changes (ownership transfer, proxy upgrade, or any mutable `isPusher` logic), the new controller retains full write authority over the creator's namespace and can push arbitrary prices that reach `AnchoredPriceProvider` and live pool swaps.

---

### Finding Description

`allowContractPushers` is the contract-pusher delegation path:

```solidity
// CompressedOracle.sol L217-233
function allowContractPushers(address[] calldata pushers) external {
    ...
    (bool ok, bytes memory res) = pusher.staticcall(
        abi.encodeWithSignature("isPusher(address)", msg.sender)
    );
    require(ok);
    bool allowed = abi.decode(res, (bool));
    require(allowed);

    namespaceRemapping[pusher] = msg.sender;   // ← written once, never re-checked
    emit PusherAuthorized(pusher, msg.sender);
}
```

The fallback push path resolves the namespace exclusively from the stored mapping:

```solidity
// CompressedOracle.sol L315-316
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
```

No re-verification of `isPusher` occurs at push time. The comment on `allowContractPushers` states "there is nothing to replay and no deadline is needed" — but this reasoning only holds if the live check is re-evaluated on every push. Because it is not, the delegation is effectively permanent until the creator calls `removePushers` or the pusher calls `revokePusher`.

**Exact analog to the external report:** In `LimitOrderManager`, the ERC721 operator set via `permit` is not cleared on token transfer, so the old operator retains authority over the new owner's position. Here, the `namespaceRemapping` entry set via a one-time `isPusher` check is not cleared when the pusher contract's authorization changes, so the new controller of the pusher contract retains write authority over the creator's namespace.

**Attack path:**

1. Creator calls `allowContractPushers([pusherContract])`. At this moment `pusherContract.isPusher(creator)` returns `true`. `namespaceRemapping[pusherContract] = creator` is written.
2. The pusher contract's ownership is transferred (or its proxy is upgraded, or its `isPusher` logic is mutated) so that `isPusher(creator)` would now return `false`.
3. The new controller of `pusherContract` calls the `CompressedOracleV1` fallback directly, pushing a crafted slot word with an arbitrary price and a fresh timestamp.
4. The fallback resolves `namespaceRemapping[pusherContract] == creator` and writes the bad price into the creator's namespace.
5. `AnchoredPriceProvider._readLeg` calls `offchainOracle.price(feedId, pool)`, which decodes the creator's namespace and returns the attacker-controlled mid price and spread.
6. `_computeBidAsk` produces a bad bid/ask that reaches the pool swap.

The creator has no on-chain signal that the pusher contract's authorization changed; `removePushers` is the only remedy, but it requires the creator to actively monitor the pusher contract's internal state.

---

### Impact Explanation

The new controller of the pusher contract can write any price value (within the 56-bit timestamp monotonicity constraint and `MAX_TIME_DRIFT`) into the creator's namespace. `AnchoredPriceProvider` applies a staleness check, a `priceGuard` min/max, and a `MAX_SPREAD_BPS` circuit breaker — but the attacker can craft a slot word that passes all three (fresh timestamp, price within guard bounds, low spread index). The resulting bad bid/ask reaches `MetricOmmPool.swap`, causing traders to receive more than the oracle curve permits or the pool to receive less than owed — direct loss of LP assets and protocol fees.

---

### Likelihood Explanation

Any pusher contract with mutable `isPusher` logic (proxy pattern, ownable access control, or any upgradeable contract) is vulnerable. This is a common pattern for production oracle pusher infrastructure. The creator has no on-chain notification mechanism and must proactively monitor the pusher contract's state. The trigger is unprivileged from the perspective of the oracle contract: the fallback is open to any caller, and the only gate is the stale `namespaceRemapping` entry.

---

### Recommendation

Re-verify `isPusher(creator)` inside the fallback push path for contract pushers, or add a deadline to `allowContractPushers` delegations (mirroring the `allowPushers` EOA path). The simplest fix is to re-check consent at push time:

```solidity
// In fallback, after resolving creator from namespaceRemapping:
if (creator != msg.sender) {
    // Re-verify live consent for contract pushers
    (bool ok, bytes memory res) = msg.sender.staticcall(
        abi.encodeWithSignature("isPusher(address)", creator)
    );
    if (!ok || !abi.decode(res, (bool))) revert UnauthorizedPusher();
}
```

Alternatively, require a deadline on `allowContractPushers` delegations (matching the `allowPushers` EOA path), forcing periodic re-authorization and bounding the window of stale delegation.

    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
