### Title
`updateBySignature` Permits Anyone to Push Two Different Prices Within the Same Transaction, Enabling Risk-Free Arbitrage Against LPs — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1.updateBySignature()` is callable by **any** address that holds a valid creator-signed slot word. Because the only freshness gate is a per-slot monotonicity check (new timestamp > stored timestamp), an attacker who has collected two legitimately-signed slot words with different prices and strictly-increasing timestamps can push both within a single transaction, sandwiching a pool swap between them. The pool reads the oracle price from storage at swap time with no intra-block update guard, so each swap executes at a different price — producing risk-free arbitrage at LP expense.

---

### Finding Description

`updateBySignature` is an open relay path: the creator signs a slot word off-chain and any address can submit it on-chain. [1](#0-0) 

The only guards are:

1. **Future-timestamp check** — the embedded timestamp must not exceed `block.timestamp + MAX_TIME_DRIFT`.
2. **Monotonicity check** — the new timestamp must be strictly greater than the currently stored timestamp for that slot.

Neither guard prevents an attacker from calling `updateBySignature` twice in the same transaction with two different signed slot words (T₁ < T₂, both ≤ `block.timestamp + MAX_TIME_DRIFT`). [2](#0-1) 

The price provider (`AnchoredPriceProvider`, `PriceProvider`, `PriceProviderL2`) reads the oracle price from storage at swap time via `CompressedOracleV1.price()`, which is a `view` that simply returns the current stored value. The `pool` argument is explicitly ignored — there is no in-swap attribution binding for the compressed oracle path. [3](#0-2) 

The pool reads the price once at the start of each `swap()` call: [4](#0-3) 

Because `swap()` is protected only by a reentrancy guard (not an intra-block price-update guard), sequential calls within the same transaction each read whatever is currently in oracle storage.

---

### Impact Explanation

An attacker can execute the following within a single atomic transaction:

1. **Push P₁ (low price)** — `updateBySignature(creator, slotValue₁, sig₁)` → oracle storage = P₁  
2. **Swap A** — `pool.swap(zeroForOne=true, …)` → pool reads P₁, attacker buys token0 cheaply  
3. **Push P₂ (high price)** — `updateBySignature(creator, slotValue₂, sig₂)` → oracle storage = P₂ (T₂ > T₁)  
4. **Swap B** — `pool.swap(zeroForOne=false, …)` → pool reads P₂, attacker sells token0 at inflated price  

The profit is `(P₂ − P₁) × amount`, extracted entirely from LP reserves. This is a direct, quantifiable loss of LP principal — a swap conservation failure where the pool receives less input than the oracle-derived price permits.

---

### Likelihood Explanation

The creator signs slot words regularly as part of normal oracle operation (the `updateBySignature` path is explicitly designed for permissionless relaying). An attacker needs only to:

- Observe two legitimately-signed slot words with different prices and increasing timestamps (trivially available from the oracle's off-chain infrastructure or mempool).
- Bundle both `updateBySignature` calls and two `pool.swap` calls into a single transaction via a flash-loan contract.

No privileged access, no malicious setup, and no non-standard token behavior is required. The attacker is a fully unprivileged third party exploiting a design gap in the push path.

---

### Recommendation

Add a per-slot, per-block update guard inside `updateBySignature` (and the `fallback` path) that records the last block number in which a slot was written and reverts if a second write is attempted in the same block:

```solidity
mapping(bytes32 => uint256) private _lastUpdateBlock;

// inside updateBySignature / fallback, before _writeStorage:
require(_lastUpdateBlock[key] < block.number, "OracleUpdatedThisBlock");
_lastUpdateBlock[key] = block.number;
```

Alternatively, the price provider can cache the price read at the start of a block and reject any swap whose oracle timestamp matches a slot that was written in the current block.

### Proof of Concept

```solidity
// Attacker contract — no privileged role required
contract Exploit {
    CompressedOracleV1 oracle;
    IMetricOmmPool    pool;

    // sig1/sig2 collected off-chain from the creator's regular update stream
    function attack(
        address creator,
        uint256 slotValue1, bytes calldata sig1,   // P_low,  T1
        uint256 slotValue2, bytes calldata sig2,   // P_high, T2 > T1
        bytes calldata cbData
    ) external {
        // 1. Push low price
        oracle.updateBySignature(creator, slotValue1, sig1);

        // 2. Buy token0 cheaply at P_low
        pool.swap(address(this), true, AMOUNT, 0, cbData, "");

        // 3. Push high price (T2 > T1 satisfies monotonicity)
        oracle.updateBySignature(creator, slotValue2, sig2);

        // 4. Sell token0 at P_high — profit = (P_high - P_low) × AMOUNT
        pool.swap(address(this), false, AMOUNT, type(uint128).max, cbData, "");
    }
}
```

The two `updateBySignature` calls succeed because T₂ > T₁ satisfies the monotonicity check, and both timestamps are within `MAX_TIME_DRIFT` of `block.timestamp`. The pool reads a different price for each swap, and the attacker captures the spread at LP expense. [1](#0-0) [3](#0-2) [5](#0-4)

    function updateBySignature(address feedCreator, uint256 newSlotValue, bytes calldata signature)
        external
        override
        returns (bool)
    {
        require(feedCreator != address(0), InvalidNamespace());

        uint256 namespace;
        assembly ("memory-safe") {
            namespace := shl(96, feedCreator) // [creator:20][zeros:12]
        }

        uint8 slotId = uint8(newSlotValue); // LSB
        TimeMs timestampMs = toTimeMs(newSlotValue >> 8 & X56);
        timestampMs.revertIfAfterBlockTimeWithDrift(MAX_TIME_DRIFT);
        bytes32 key = bytes32(namespace | uint256(slotId));
        uint256 old = uint256(_loadStorage(key));
        TimeMs oldTimestampMs = toTimeMs(old >> 8 & X56);

        bool newer = timestampMs.isAfter(oldTimestampMs);
        if (!newer) {
            return false;
        }

        bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
            keccak256(abi.encode(block.chainid, address(this), feedCreator, newSlotValue))
        );
        require(feedCreator == ECDSA.recover(hash, signature));

        _writeStorage(key, bytes32(newSlotValue & ~uint256(0xff)));

        return true;
    }
