### Title
`allowContractPushers` Silently Overwrites Existing Pusher Delegation, Causing Creator Feed Staleness and Bad-Price Execution — (`File: smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowContractPushers` unconditionally overwrites `namespaceRemapping[pusher]` without checking whether the pusher is already delegated to another creator. Any address that satisfies a contract pusher's `isPusher(attacker)` check can silently steal that pusher from its current creator, causing the creator's feeds to go stale and delivering bad prices to every pool anchored to those feeds.

---

### Finding Description

`CompressedOracleV1` maintains a single-slot delegation mapping:

```solidity
mapping(address => address) public namespaceRemapping;
``` [1](#0-0) 

When a creator calls `allowContractPushers`, the function verifies consent via a live `isPusher(creator)` staticcall and then unconditionally writes the new creator into the mapping:

```solidity
(bool ok, bytes memory res) = pusher.staticcall(
    abi.encodeWithSignature("isPusher(address)", msg.sender)
);
require(ok);
bool allowed = abi.decode(res, (bool));
require(allowed);

namespaceRemapping[pusher] = msg.sender;   // ← overwrites any existing mapping
emit PusherAuthorized(pusher, msg.sender);
``` [2](#0-1) 

There is no guard of the form `require(namespaceRemapping[pusher] == address(0))` or any notification/consent from the existing creator. The mapping is a flat `address → address` store — exactly one creator per pusher — mirroring the ERC-721 `_tokenApprovals` single-slot design that the external report identified as the root cause of allowance loss.

Once overwritten, the original creator A can no longer call `removePushers([P])` because that function checks `namespaceRemapping[pusher] == msg.sender`:

```solidity
if (namespaceRemapping[pusher] == msg.sender) {
    namespaceRemapping[pusher] = address(0);
    ...
} else {
    revert InvalidManager(msg.sender);
}
``` [3](#0-2) 

Creator A has permanently lost their pusher with no on-chain recourse.

The `fallback` push path resolves the namespace at call time from the current mapping:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [4](#0-3) 

After the hijack, every subsequent push from P lands in B's namespace. A's slots receive no further updates; their timestamps freeze, and every consumer that checks staleness will see `timestamp = 0` (or the last pre-hijack value) against the current block time.

---

### Impact Explanation

A pool anchored to creator A's feeds via `AnchoredPriceProvider` will receive a stale mid-price and stale spread once the pusher is stolen. The `maxTimeDrift` guard in `OracleBase` will eventually reject the stale data, but until that threshold is crossed the pool executes swaps at an outdated price. After the threshold is crossed the pool's price read reverts entirely, making swaps and liquidity operations unusable. Both outcomes are in-scope:

- **Bad-price execution**: stale bid/ask quote reaches a pool swap before the drift guard fires.
- **Broken core pool functionality**: once the drift guard fires, every swap that reads the feed reverts, freezing the pool.

---

### Likelihood Explanation

The attack requires a contract pusher P where `P.isPusher(attacker)` returns `true`. This is realistic for:

1. **Shared oracle relay contracts** that allow any registered creator to use them (returning `true` for all callers, or for a set that includes the attacker).
2. **Multi-tenant pusher services** where the attacker is a legitimate but malicious participant.

No privileged role, no signature from the victim creator, and no deadline are required. The attacker calls one public function. The `allowContractPushers` path has no deadline at all (unlike `allowPushers`), making it permanently replayable.

---

### Recommendation

Add an existence check before overwriting the mapping:

```solidity
function allowContractPushers(address[] calldata pushers) external {
    uint256 l = pushers.length;
    for (uint256 i; i < l; i++) {
        address pusher = pushers[i];
        if (pusher == msg.sender) revert NoSelfRemapping();

        // Prevent silent hijack of an already-delegated pusher
        address existing = namespaceRemapping[pusher];
        if (existing != address(0) && existing != msg.sender) {
            revert PusherAlreadyDelegated(pusher, existing);
        }

        (bool ok, bytes memory res) = pusher.staticcall(
            abi.encodeWithSignature("isPusher(address)", msg.sender)
        );
        require(ok);
        bool allowed = abi.decode(res, (bool));
        require(allowed);

        namespaceRemapping[pusher] = msg.sender;
        emit PusherAuthorized(pusher, msg.sender);
    }
}
```

Apply the same guard to `allowPushers`. Alternatively, require the existing creator to call `removePushers` before a new creator can claim the same pusher.

### Proof of Concept

```
State before attack:
  namespaceRemapping[P] = creatorA   // P is a shared contract; isPusher returns true for all

Attack:
  vm.prank(attacker);
  oracle.allowContractPushers([P]);  // isPusher(attacker) == true → succeeds

State after attack:
  namespaceRemapping[P] = attacker   // creatorA's delegation silently erased

Consequence:
  P pushes slot word → lands in attacker's namespace, not creatorA's
  creatorA's feeds: timestamp frozen at last pre-hijack value
  Pool anchored to creatorA's feedId reads stale price until maxTimeDrift fires
  After maxTimeDrift: every swap on that pool reverts → pool frozen
```

The `allowContractPushers` function contains no check against `namespaceRemapping[pusher] != address(0)`, so the overwrite succeeds unconditionally whenever `isPusher(attacker)` returns `true`. [5](#0-4)
