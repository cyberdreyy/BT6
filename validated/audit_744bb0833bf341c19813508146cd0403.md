### Title
Pusher Delegation Revocation Bypassed via Signature Replay in `allowPushers` — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`allowPushers` contains no nonce or "used" flag. A creator who holds a valid (non-expired) consent signature can replay it after the pusher has called `revokePusher()`, silently re-establishing the delegation the pusher intended to terminate. The code's own NatSpec acknowledges the risk ("an undated signature could re-establish a delegation AFTER the pusher revoked it") but relies solely on the deadline — which only prevents expired signatures, not replay within the deadline window.

### Finding Description

`revokePusher()` sets `namespaceRemapping[msg.sender] = address(0)`, returning the pusher to the "never delegated" default state. Because `address(0)` is both the "never delegated" sentinel and the post-revocation state, `allowPushers` cannot distinguish between the two. It performs no check that the pusher has previously revoked, and no nonce is consumed on use:

```solidity
// OracleBase (compressed) — _ensureDeadline
function _ensureDeadline(uint256 deadline) internal view virtual {
    require(block.timestamp <= deadline, DeadlineExceeded()); // only checks expiry
}

// CompressedOracle.allowPushers — no nonce, no "already-used" guard
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
);
require(pusher == ECDSA.recover(hash, signatures[i]));
namespaceRemapping[pusher] = msg.sender;   // overwrites address(0) set by revokePusher
```

This is the direct analog of the Derby M-45 pattern: a zero-value sentinel (`address(0)`) that is also the natural post-revocation state, allowing the "already done" guard to be bypassed by replaying the original action.

### Impact Explanation

After a pusher revokes, the creator replays the original consent signature (deadline still valid) to force `namespaceRemapping[pusher] = creator` again. Every subsequent fallback push by the pusher lands in the creator's namespace instead of the pusher's own. If the pusher stops pushing to escape the creator's feeds, the creator's oracle slots go stale. `AnchoredPriceProvider._isStale` returns `true` for any `refTime` older than `MAX_REF_STALENESS`, causing `getBidAndAskPrice()` to return `(0, type(uint128).max)` and `getBidAndAskPrice()` to revert with `FeedStalled`, making every pool swap that depends on those feeds unusable.

### Likelihood Explanation

Medium. The creator must hold the pusher's original consent bytes. Long-lived deadlines (days or weeks, common for operational convenience) give the creator an extended replay window. The pusher has no on-chain way to invalidate the signature short of waiting for the deadline to expire.

### Recommendation

Introduce a per-pusher nonce that is incremented on every successful `allowPushers` call and included in the signed digest. Alternatively, record a `revokedAt` timestamp per pusher and reject any signature whose deadline predates the most recent revocation.

```solidity
mapping(address => uint256) public pusherNonce;

// in allowPushers:
uint256 nonce = pusherNonce[pusher]++;
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(this), deadline, nonce, pusher, msg.sender))
);
```

    /// @notice Delegates pusher wallets into the caller's namespace. The pusher's EIP-191
    ///         signature is REQUIRED — without it anyone could remap a foreign pusher
    ///         wallet into their own namespace and silently swallow its pushes. The
    ///         deadline is likewise required: the signed consent carries no timestamp of
    ///         its own, so an undated signature could re-establish a delegation AFTER the
    ///         pusher revoked it.
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }

    /// @notice Allows a pusher to self-revoke their delegation. After revocation the
    ///         wallet pushes into its OWN namespace again (the registrationless default).
    function revokePusher() external {
        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
        namespaceRemapping[msg.sender] = address(0);
        emit PusherRevoked(msg.sender, creator);
    }
