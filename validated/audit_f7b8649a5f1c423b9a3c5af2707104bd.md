### Title
Shared per-slot timestamp in `CompressedOracleV1` allows stale co-lane prices to bypass `MAX_TIME_DELTA`/`MAX_REF_STALENESS` staleness checks, feeding bad prices into pools - (File: `smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1` packs four oracle lanes (positions 0–3) into a single 256-bit storage slot with **one shared timestamp** for all four lanes. When any lane is updated via `fallback()` or `updateBySignature()`, the entire slot — including all four lane prices — is overwritten with the new timestamp. `PriceProvider` and `AnchoredPriceProvider` check staleness using this per-slot timestamp. A pusher that updates one lane (e.g., ETH/USD) while leaving another lane's price (e.g., BTC/USD) unchanged will silently refresh the staleness clock for the stale lane, allowing an arbitrarily old price to pass the `MAX_TIME_DELTA` / `MAX_REF_STALENESS` guard and reach pool swaps.

---

### Finding Description

**Slot layout — one timestamp for four lanes:**

`CompressedOracleV1` stores four 48-bit oracle lanes and one 56-bit timestamp in a single EVM storage word:

```
bits 255…208 : oracle[0] (price 32b | s0 8b | s1 8b)
bits 207…160 : oracle[1]
bits 159…112 : oracle[2]
bits 111…64  : oracle[3]
bits  63…8   : timestamp (uint56, unix ms)   ← shared across all four lanes
bits   7…0   : reserved
``` [1](#0-0) 

**`getOracleData` returns the per-slot timestamp for every lane:**

```solidity
data.timestampMs = _layout.timestampMs;   // line 116 — slot-level, not lane-level
``` [2](#0-1) 

**`fallback()` overwrites the entire slot on every push:**

When a pusher sends a 32-byte slot word, the monotonicity check compares the incoming timestamp against the stored slot timestamp. If newer, the **entire** 256-bit slot is replaced — all four lane prices and the shared timestamp — in one `sstore`:

```solidity
bool newer = timestampMs.isAfter(oldTimestampMs);
if (!newer) continue;
_writeStorage(key, bytes32(bytes32(word & ~uint256(0xff))));
``` [3](#0-2) 

The documentation explicitly acknowledges this: *"The entire slot is overwritten on update: if you update a single lane, you must still supply correct values for the other lanes."* [4](#0-3) 

**Staleness checks in providers use the per-slot timestamp:**

`PriceProvider._getBidAndAskPrice()` calls `IPricedOracle.price(feedId, pool)`, which resolves to `CompressedOracleV1._price()` → `getOracleData()` → returns `data.timestampMs = _layout.timestampMs`. The provider then checks:

```solidity
if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA)) {
    return (0, type(uint128).max);
}
``` [5](#0-4) 

`AnchoredPriceProvider._readLeg()` performs the identical check:

```solidity
if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);
``` [6](#0-5) 

Because `refTime` is the **slot-level** timestamp, not a per-lane timestamp, updating lane 0 with a fresh timestamp causes lane 1's price — which may be hours old — to report `refTime = block.timestamp`, passing the staleness guard unconditionally.

---

### Impact Explanation

A pool backed by `PriceProvider` or `AnchoredPriceProvider` reading lane N of a `CompressedOracleV1` slot will execute swaps against a price that is arbitrarily stale whenever any other lane in the same slot is updated. The `MAX_TIME_DELTA` / `MAX_REF_STALENESS` circuit breaker — the sole on-chain guard against stale oracle data — is silently bypassed. Traders can observe the divergence between the stale on-chain price and the true market price and extract value from LPs through arbitrage, causing direct loss of LP principal. This matches the allowed impact gate: **bad-price execution — stale bid/ask quote reaches a pool swap**.

---

### Likelihood Explanation

The trigger is a valid, authorized pusher (semi-trusted) that updates one lane in a slot more frequently than another. This is a normal operational pattern: a creator may host ETH/USD (lane 0, updated every second) and a less-liquid asset (lane 1, updated every minute) in the same slot to save gas. Every ETH/USD push refreshes the slot timestamp, making the less-liquid lane appear perpetually fresh regardless of how old its price actually is. No malicious intent is required; the bug fires under routine multi-feed operation.

---

### Recommendation

Introduce per-lane timestamps. Replace the single 56-bit slot-level timestamp with four independent per-lane timestamps. Each lane's 48 bits can be reduced to 32 bits of price + 16 bits of spread index, freeing 64 bits for four 16-bit relative timestamps (delta from a slot epoch), or the slot layout can be extended to two storage words. Alternatively, the provider layer can be hardened: `PriceProvider` / `AnchoredPriceProvider` should require the caller to supply the expected `feedId` lane index and compare the lane's own last-written timestamp (stored separately) rather than the slot-level timestamp.

---

### Proof of Concept

```
Setup:
  - CompressedOracleV1 deployed with MAX_TIME_DRIFT = 5 s
  - Creator pushes slot 0 with four lanes at T=0:
      lane 0 (ETH/USD): price = 3000e8, ts = T
      lane 1 (BTC/USD): price = 60000e8, ts = T   ← same slot timestamp
  - PriceProvider deployed reading lane 1 (BTC/USD), MAX_TIME_DELTA = 60 s

Attack:
  1. At T = 0, both lanes are fresh. BTC/USD = 60000e8.
  2. At T = 3600 (1 hour later), BTC/USD true market price = 50000e8.
     Without any update, the provider would revert FeedStalled (age > 60 s).
  3. Pusher sends a slot word updating only lane 0 (ETH/USD) with ts = T+3600,
     supplying the OLD lane 1 value (60000e8) for BTC/USD.
  4. Slot is overwritten: lane 1 price = 60000e8, slot timestamp = T+3600.
  5. Provider reads lane 1: refTime = T+3600, age = 0 s → staleness check passes.
  6. Provider returns bid/ask anchored on 60000e8 (stale, 20% above market).
  7. Attacker sells BTC at the inflated 60000e8 pool price, draining LP token1.
``` [7](#0-6) [2](#0-1) [5](#0-4) [8](#0-7)

    /// @notice Push path. Calldata = N × 32-byte slot words:
    ///         [data0:6][data1:6][data2:6][data3:6][ts:7][slotId:1]
    ///         The sender pushes into `namespaceRemapping[msg.sender]`, falling back to
    ///         its OWN namespace — a creator needs zero setup transactions to start
    ///         pushing. Each word carries its own timestamp (monotonicity is the only
    ///         freshness gate), so there is no deadline prefix.
    fallback() override external {
        uint256 end;
        uint256 namespace;

        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0)) creator = msg.sender;

        assembly ("memory-safe") {
            end := calldatasize()
            namespace := shl(96, creator) // [creator:20][zeros:12]
        }

        // 4 * 6 + 7 + 1 = 32 bytes per slot
        if (end == 0 || end % 32 != 0) revert BadCalldataLength();

        for (uint256 ptr = 0; ptr < end; ptr += 32) {
            uint256 word;
            assembly ("memory-safe") {
                word := calldataload(ptr)
            }
            // casting to 'uint8' is safe we want LSB
            // forge-lint: disable-next-line(unsafe-typecast)
            uint8 slotId = uint8(word);
            TimeMs timestampMs = toTimeMs(word >> 8 & X56);
            timestampMs.revertIfAfterBlockTimeWithDrift(MAX_TIME_DRIFT);
            bytes32 key = bytes32(namespace | uint256(slotId));
            uint256 old = uint256(_loadStorage(key));
            TimeMs oldTimestampMs = toTimeMs(old >> 8 & X56);

            bool newer = timestampMs.isAfter(oldTimestampMs);
            if (!newer) continue;

            _writeStorage(key, bytes32(bytes32(word & ~uint256(0xff))));
        }

    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
