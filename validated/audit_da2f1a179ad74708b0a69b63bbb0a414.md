### Title
`allowPushers` EIP-191 signature replay re-establishes a revoked delegation within the deadline window, bypassing pusher revocation — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary
`CompressedOracle.allowPushers` contains no guard preventing a creator from replaying the same pusher-signed EIP-191 consent after the pusher has called `revokePusher()`. Because the signature domain contains no nonce and the only expiry is the deadline, the creator can re-establish the delegation an unlimited number of times within the deadline window, making `revokePusher()` ineffective and forcing the pusher's fallback writes to continue landing in the creator's namespace.

### Finding Description
`allowPushers` verifies a pusher-signed EIP-191 message whose domain is:

```
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [1](#0-0) 

The domain binds chain id, oracle address, deadline, pusher address, and creator address — but **no nonce**. Within the deadline window the same bytes are a valid input to `ECDSA.recover` on every call. After the signature passes, the function unconditionally overwrites `namespaceRemapping[pusher]`:

```solidity
namespaceRemapping[pusher] = msg.sender;
``` [2](#0-1) 

There is no check that `namespaceRemapping[pusher] == address(0)` before writing. When a pusher calls `revokePusher()`:

```solidity
namespaceRemapping[msg.sender] = address(0);
``` [3](#0-2) 

the creator can immediately call `allowPushers` again with the **identical** signature and deadline to restore `namespaceRemapping[pusher] = creator`. The code comment explicitly acknowledges the risk ("an undated signature could re-establish a delegation AFTER the pusher revoked it") and claims the deadline prevents it — but the deadline only blocks signatures whose expiry has already passed; it does nothing to prevent replay of a still-valid signature within the window. [4](#0-3) 

The fallback push path resolves the namespace at call time:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [5](#0-4) 

So every push the pusher makes after the re-established delegation lands in the creator's namespace, not the pusher's own.

### Impact Explanation
The `CompressedOracle` is a registrationless price source consumed by `AnchoredPriceProvider` / `AnchoredProviderFactory`-deployed providers, which in turn supply bid/ask quotes to `MetricOmmPool` swaps. A creator whose namespace is backed by a pool can force a pusher's prices to continue feeding that pool even after the pusher has explicitly revoked consent. If the pusher is a keeper bot that cannot simply stop pushing, or if the creator is acting adversarially (e.g., selectively replaying the delegation only when the pusher is about to push a price that benefits the creator), the pool receives prices attributed to a pusher who no longer consents — a bad-price execution path and an admin-boundary break (pusher revocation bypassed by an unprivileged creator call).

### Likelihood Explanation
Any creator who holds a still-valid pusher signature (deadline in the future) can execute this with a single transaction immediately after the pusher's `revokePusher()`. No privileged role is required. Deadlines are typically set days or weeks in the future, giving a wide replay window. The test suite covers expired-deadline reverts and successful revocation but has no test asserting that a revoked delegation cannot be re-established within the deadline window. [6](#0-5) 

### Recommendation
Add a nonce to the signature domain (e.g., a per-pusher counter incremented on every successful `allowPushers` or `revokePusher` call) so that each consent is single-use. Alternatively, require `namespaceRemapping[pusher] == address(0)` before writing, so that re-delegation requires a fresh signature obtained after revocation. The comment's stated intent — that the deadline prevents post-revocation re-establishment — should be enforced in code, not just documentation.

```solidity
// Option A: require the slot is empty
require(namespaceRemapping[pusher] == address(0), AlreadyDelegated());

// Option B: include a nonce in the signed message
mapping(address => uint256) public pusherNonce;
// ... in allowPushers:
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender, pusherNonce[pusher]++))
```

### Proof of Concept
```
1. Pusher P signs: keccak256(abi.encode(chainid, oracle, deadline=T+30days, P, creatorA))
   → sig_A

2. creatorA calls allowPushers(T+30days, [P], [sig_A])
   → namespaceRemapping[P] = creatorA  ✓

3. P calls revokePusher()
   → namespaceRemapping[P] = address(0)  (P believes delegation is gone)

4. creatorA calls allowPushers(T+30days, [P], [sig_A])  // SAME call, SAME sig
   → ECDSA.recover succeeds (deadline not expired, domain unchanged)
   → namespaceRemapping[P] = creatorA  ✓  (revocation silently undone)

5. P's next fallback push lands in creatorA's namespace, not P's own.
   Pool backed by creatorA's namespace receives P's price feed.
```

```text
    /// @notice Delegates pusher wallets into the caller's namespace. The pusher's EIP-191
    ///         signature is REQUIRED — without it anyone could remap a foreign pusher
    ///         wallet into their own namespace and silently swallow its pushes. The
    ///         deadline is likewise required: the signed consent carries no timestamp of
    ///         its own, so an undated signature could re-establish a delegation AFTER the
    ///         pusher revoked it.
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
