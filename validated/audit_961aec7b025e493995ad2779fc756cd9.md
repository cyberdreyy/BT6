### Title
Missing Zero-Address Validation for `_oracle` in Price Provider Constructors Allows Permanently Broken Provider Deployment — (`smart-contracts-poc/contracts/PriceProvider.sol`, `ProtectedPriceProvider.sol`, `PriceProviderL2.sol`)

### Summary
`PriceProvider`, `ProtectedPriceProvider`, and `PriceProviderL2` constructors accept `_oracle = address(0)` without reverting. The permissionless `PriceProviderFactory.createPriceProvider` also performs no zero-address check on `_oracle`. The resulting provider is tracked as valid by the factory (`isProvider == true`) but permanently reverts with `FeedStalled` on every swap, making any pool that adopts it completely unusable for swaps.

### Finding Description

In all three L1/L2 price provider constructors the `_factory` address is validated but `_oracle` is not:

```solidity
// PriceProvider.sol constructor (lines 70–73)
require(_factory != address(0));
factory = _factory;

offchainOracle = IOffchainOracle(_oracle);   // ← no zero-address check
offchainFeedId = _offchainFeedId;
``` [1](#0-0) 

The same pattern appears verbatim in `ProtectedPriceProvider.sol` and `PriceProviderL2.sol`: [2](#0-1) [3](#0-2) 

`PriceProviderFactory.createPriceProvider` forwards `_oracle` directly to the constructor with no pre-flight check:

```solidity
// PriceProviderFactory.sol lines 49–57
PriceProvider p = new PriceProvider(
    address(this),
    _oracle,          // ← caller-supplied, never validated
    _feedId,
    ...
);
``` [4](#0-3) 

After deployment the provider is added to `_providers` and `providerOwner` is set, so `isProvider(provider)` returns `true` — the factory considers it a legitimate provider.

**Contrast with `AnchoredProviderFactory`**, which correctly guards against this via an oracle allow-list:

```solidity
// AnchoredProviderFactory.sol line 71
require(oracle != address(0), ZeroOracle());
``` [5](#0-4) 

and at call time:

```solidity
if (!_oracles.contains(oracle)) revert OracleNotAllowed(oracle);
``` [6](#0-5) 

`PriceProviderFactory` has no equivalent guard.

### Impact Explanation

When `offchainOracle == address(0)`, every call to `getBidAndAskPrice()` executes a high-level interface call to `address(0)`. In Solidity ≥ 0.8 a call to an address with no code succeeds and returns zero bytes, decoded as `(mid=0, spread=0, refTime=0)`. The staleness check immediately fires (`refTime == 0 → true`) and the function returns the stall sentinel `(0, type(uint128).max)`, causing `getBidAndAskPrice` to revert with `FeedStalled`:

```solidity
// PriceProvider.sol lines 118–120
function getBidAndAskPrice() external override returns (uint128 bid, uint128 ask) {
    (bid, ask) = _getBidAndAskPrice();
    if (bid == 0 || ask == type(uint128).max) revert FeedStalled();
}
``` [7](#0-6) 

Because `offchainOracle` is `immutable`, the provider is permanently broken. Any pool that adopts it cannot execute swaps for its entire lifetime — a complete loss of core swap functionality.

### Likelihood Explanation

`PriceProviderFactory.createPriceProvider` is permissionless. Any caller can supply `_oracle = address(0)`. A user who accidentally passes a zero oracle (e.g., from a misconfigured deployment script) receives a provider that appears valid (`isProvider == true`, `providerOwner` set) but is silently broken. There is no on-chain signal at deploy time that the provider is unusable; the failure only surfaces at the first swap attempt.

### Recommendation

Add a zero-address guard for `_oracle` in each provider constructor, mirroring the existing `_factory` check:

```solidity
require(_oracle != address(0), "PriceProvider: zero oracle");
offchainOracle = IOffchainOracle(_oracle);
```

Apply the same guard in `PriceProviderFactory.createPriceProvider` before the `new PriceProvider(...)` call, consistent with `AnchoredProviderFactory.addOracle`'s `ZeroOracle()` pattern.

### Proof of Concept

1. Call `PriceProviderFactory.createPriceProvider(address(0), feedId, 0, 1 hours, tokenA, tokenB)`.
2. Factory deploys `PriceProvider` with `offchainOracle = address(0)` (immutable). `isProvider(provider) == true`.
3. Create a pool pointing to this provider.
4. Any swap calls `provider.getBidAndAskPrice()` → `IPricedOracle(address(0)).price(...)` → returns `(0,0,0,0)` → `_isStale(0,...) == true` → returns `(0, type(uint128).max)` → reverts `FeedStalled`.
5. Pool is permanently unable to execute swaps.
