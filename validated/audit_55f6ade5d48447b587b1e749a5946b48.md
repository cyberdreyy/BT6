### Title
`revokePusher` Delegation Revocation Bypassed by Creator Signature Replay — (File: `smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary
The `allowPushers` delegation signature contains no nonce. After a pusher calls `revokePusher` to exit a delegation, the creator can immediately replay the original signed message to re-establish `namespaceRemapping[pusher] = creator`, nullifying the revocation. The pusher cannot effectively exit the delegation until the signed deadline expires. If the pusher's key is compromised, the attacker retains the ability to push arbitrary prices into the creator's namespace for the full deadline window.

### Finding Description

`allowPushers` constructs the signed digest as:

```solidity
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [1](#0-0) 

There is no nonce, no per-delegation counter, and no consumed-signature registry. The same `(deadline, pusher, creator)` tuple produces the same hash every time it is presented, so the signature is reusable for the entire lifetime of the deadline.

`revokePusher` sets `namespaceRemapping[msg.sender] = address(0)`:

```solidity
function revokePusher() external {
    address creator = namespaceRemapping[msg.sender];
    if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
    namespaceRemapping[msg.sender] = address(0);
    emit PusherRevoked(msg.sender, creator);
}
``` [2](#0-1) 

Immediately after, the creator calls `allowPushers` again with the identical `(deadline, [pusher], [signature])` arguments. `_ensureDeadline` passes (deadline is still in the future), ECDSA recovery succeeds (same hash, same signature), and `namespaceRemapping[pusher]` is written back to `creator`. The revocation is undone in a single follow-up transaction.

The `fallback()` push path resolves the namespace from `namespaceRemapping[msg.sender]`:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [3](#0-2) 

So once the delegation is re-established, every push from the compromised pusher key lands in the creator's namespace and overwrites the live price data for all four positions in the targeted slot, subject only to the monotonicity check on `timestampMs`.

The code comment in `allowPushers` explicitly acknowledges the deadline's role:

> "The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it." [4](#0-3) 

The comment treats the deadline as the guard against post-revocation replay, but the deadline only prevents replay *after it expires*. Before expiry, the same signature is unconditionally valid and the revocation is trivially undone.

### Impact Explanation

A compromised pusher key can push arbitrary `(price, spread0, spread1, timestampMs)` tuples into the creator's namespace for any slot. The `AnchoredPriceProvider` consuming a `CompressedOracle` feed reads price and spread directly:

```solidity
(mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);
``` [5](#0-4) 

The `CompressedOracle.price` ignores the `pool` argument entirely — there is no blacklist, no pool-registration check, and no `inSwap` binding:

```solidity
function price(bytes32 feedId, address /* pool */)
    external view
    returns (uint256 mid, uint256 spread, uint16 spread1, uint256 refTime)
{
    return _price(feedId);
}
``` [6](#0-5) 

A manipulated mid price within the `AnchoredPriceProvider`'s `MAX_REF_STALENESS` window and below `MAX_SPREAD_BPS` passes all guards and reaches `_computeBidAsk`, producing a shifted bid/ask band. Swaps execute against this band, causing traders to receive more output than the true price permits or LPs to receive less input than owed — a direct swap conservation failure.

### Likelihood Explanation

Likelihood is **Medium**. The attack requires two concurrent conditions: (1) a pusher key is compromised, and (2) the creator replays the delegation signature (possibly without knowing the key is compromised, e.g., in an automated keeper setup). Delegation signatures with multi-month deadlines are operationally normal. The creator's replay call is a single permissionless transaction requiring only the original calldata, which is on-chain and trivially retrievable from the `PusherAuthorized` event log.

### Recommendation

Introduce a per-`(creator, pusher)` nonce that is incremented on every successful `allowPushers` call and included in the signed digest:

```solidity
mapping(address creator => mapping(address pusher => uint256)) public delegationNonce;

// In allowPushers:
uint256 nonce = delegationNonce[msg.sender][pusher]++;
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender, nonce))
);
```

This makes each delegation consent single-use: after `revokePusher`, the old signature is permanently invalid because the nonce has advanced.

### Proof of Concept

```
T=0  Pusher signs: keccak256(abi.encode(chainid, oracle, deadline=T+365d, pusher, creator))
     → sig_A

T=1  Creator calls allowPushers(T+365d, [pusher], [sig_A])
     → namespaceRemapping[pusher] = creator  ✓

T=2  Pusher key is compromised; attacker pushes manipulated slot word via fallback()
     → creator's feedId slot now holds attacker-controlled price

T=3  Pusher calls revokePusher()
     → namespaceRemapping[pusher] = address(0)  (revocation)

T=4  Creator calls allowPushers(T+365d, [pusher], [sig_A])   ← SAME sig_A, deadline still valid
     → namespaceRemapping[pusher] = creator  (revocation undone)

T=5  Attacker pushes another manipulated slot word (timestampMs > previous)
     → monotonicity check passes, bad price stored

T=6  Pool swap reads CompressedOracle via AnchoredPriceProvider
     → bad mid price within staleness window → shifted bid/ask band → swap executes at wrong price
```

The cycle at T=3→T=4 can repeat indefinitely until `deadline` expires, giving the attacker a window equal to the original deadline duration to push bad prices into any pool anchored to the creator's feedId.
