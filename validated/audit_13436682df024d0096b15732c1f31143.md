**Tracing `allowPushers` → `revokePusher` → `allowPushers` replay:**

`allowPushers` at [1](#0-0)  constructs the signed hash as:


There is **no nonce** and **no used-signature registry**. The only replay protection is `_ensureDeadline(deadline)` at [2](#0-1) , which only rejects calls after the deadline, not within it.

### Title
`allowPushers` signature replay within deadline silently undoes `revokePusher`, forcing pusher back into creator's namespace — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary
A pusher who calls `revokePusher()` can be silently re-delegated by the creator replaying the original EIP-191 signature before its deadline expires. The pusher's revocation is overwritten without any renewed consent, and subsequent `fallback()` pushes land in the creator's slots.

### Finding Description
`allowPushers` verifies the pusher's EIP-191 signature over `(chainid, oracle, deadline, pusher, creator)` and writes `namespaceRemapping[pusher] = creator`. There is no nonce, no per-signature consumed flag, and no check that the pusher has not since revoked. `revokePusher` only zeroes the mapping entry; it does not invalidate the signed message. Because `_ensureDeadline` only rejects calls where `block.timestamp > deadline`, the creator can call `allowPushers` a second time with the identical signature at any point before the deadline, restoring `namespaceRemapping[pusher] = creator` and silently undoing the revocation. [6](#0-5) 

### Impact Explanation
After the silent re-delegation, every `fallback()` call from the pusher writes into the creator's storage slots. If the pusher, believing they revoked, continues pushing data intended for their own namespace (different asset, stale feed, or test data), those values overwrite the creator's production price slots. Any pool or provider reading the creator's feed via `getOracleData` / `price` will consume the corrupted price, leading to bad-price execution on swaps.

### Likelihood Explanation
The window is bounded by the deadline the pusher originally signed (e.g., 1 hour). Within that window the creator needs only one additional transaction. No privileged role is required — the creator is a normal user of the oracle. The pusher has no on-chain way to detect or prevent the replay.

### Recommendation
Track consumed signatures. Add a `mapping(bytes32 => bool) private _usedDelegationHashes` and mark the hash as used on the first successful `allowPushers` call. Reject any subsequent call with the same hash, regardless of deadline. Alternatively, include a per-pusher nonce in the signed payload and increment it on every `revokePusher`, invalidating all prior signatures.

```solidity
function test_replayAfterRevoke() public {
    uint256 deadline = block.timestamp + 1 hours;
    // pusher signs consent
    bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
        keccak256(abi.encode(block.chainid, address(oracle), deadline, pusher, creator))
    );
    (uint8 v, bytes32 r, bytes32 s) = vm.sign(pusherKey, hash);
    bytes memory sig = abi.encodePacked(r, s, v);

    address[] memory pushers = new address[](1);
    pushers[0] = pusher;
    bytes[] memory sigs = new bytes[](1);
    sigs[0] = sig;

    // Step 1: creator delegates
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs);
    assertEq(oracle.namespaceRemapping(pusher), creator);

    // Step 2: pusher revokes
    vm.prank(pusher);
    oracle.revokePusher();
    assertEq(oracle.namespaceRemapping(pusher), address(0));

    // Step 3: creator replays same signature before deadline
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs); // succeeds — no nonce check
    assertEq(oracle.namespaceRemapping(pusher), creator); // revocation silently undone

    // Step 4: pusher's fallback push lands in creator's slot
    vm.prank(pusher);
    (bool ok,) = address(oracle).call(craftSlotWord(slotId, badPrice, freshTs));
    assertTrue(ok);
    // creator's feed now contains pusher's data
}
```
