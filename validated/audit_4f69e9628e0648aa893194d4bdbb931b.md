### Title
`revokePusher` fails to invalidate the pusher's signed consent, allowing the creator to silently re-establish delegation after revocation — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1.revokePusher()` clears `namespaceRemapping[msg.sender]` but does not invalidate the pusher's previously signed EIP-191 consent. Because `allowPushers` has no nonce or used-signature tracking, the creator can replay the original consent signature (before its deadline expires) to silently re-establish the delegation. The pusher's revocation is therefore ineffective for the entire remaining lifetime of the signed consent.

---

### Finding Description

The delegation flow is:

1. Pusher signs: `keccak256(abi.encode(block.chainid, address(this), deadline, pusher, creator))`
2. Creator calls `allowPushers(deadline, [pusher], [sig])` → `namespaceRemapping[pusher] = creator`
3. Pusher calls `revokePusher()` → `namespaceRemapping[pusher] = address(0)`
4. Creator calls `allowPushers(deadline, [pusher], [sig])` **again with the identical signature** → `namespaceRemapping[pusher] = creator` is silently restored

`allowPushers` performs no nonce check and no used-signature bookkeeping: [1](#0-0) 

`revokePusher` only zeroes the mapping; it does not record a revocation timestamp or burn the consent: [2](#0-1) 

The code's own comment acknowledges the deadline is the sole guard against post-revocation re-establishment: [3](#0-2) 

But the deadline only prevents replay **after** it expires. Before expiry the creator can replay the same bytes indefinitely. `_ensureDeadline` only checks `block.timestamp <= deadline`: [4](#0-3) 

**Invariant broken:** After `revokePusher()` succeeds, every subsequent push by that pusher MUST land in the pusher's own namespace. The re-established delegation silently redirects those pushes back into the creator's namespace without any new consent from the pusher.

---

### Impact Explanation

After the creator replays the consent, the pusher's future fallback pushes resolve the namespace as:

```solidity
address creator = namespaceRemapping[msg.sender]; // now creator again
if (creator == address(0)) creator = msg.sender;
``` [5](#0-4) 

The pusher, believing they have revoked, may continue pushing data intended for their own namespace (e.g., for a different pool or a different asset pair). That data is written into the creator's namespace slots instead, overwriting the creator's pool's live price feed with unintended values. Any pool whose `AnchoredPriceProvider` reads from the creator's compressed feed will then execute swaps against a corrupted bid/ask, satisfying the **bad-price execution** impact gate.

---

### Likelihood Explanation

- The creator already holds the pusher's signed consent bytes (they submitted them in the original `allowPushers` call).
- No privileged role is required; `allowPushers` is fully permissionless.
- The window is the full remaining lifetime of the deadline. If the pusher signed with a 7-day deadline and revokes on day 1, the creator has 6 days to replay.
- The pusher has no on-chain way to detect the re-establishment (no event distinguishes a first delegation from a replay).
- The pusher continuing to push after revocation is the normal expected behaviour (they are now pushing for their own namespace or a new creator).

---

### Recommendation

Track consumed consent signatures with a per-pusher nonce or a `mapping(bytes32 => bool) usedConsents` set. Burn the consent hash on first use in `allowPushers`:

```solidity
mapping(bytes32 => bool) private _usedConsents;

// inside allowPushers, after recovering the signer:
bytes32 consentHash = keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender));
require(!_usedConsents[consentHash], ConsentAlreadyUsed());
_usedConsents[consentHash] = true;
```

Alternatively, bind the consent to a per-pusher nonce that increments on every `revokePusher` / `removePushers` call, so any previously signed consent is automatically invalidated on revocation.

---

### Proof of Concept

```solidity
// 1. Pusher signs consent for creator, deadline = now + 7 days
uint256 deadline = block.timestamp + 7 days;
bytes memory sig = pusherSign(deadline, pusher, creator);

// 2. Creator establishes delegation
vm.prank(creator);
oracle.allowPushers(deadline, toArray(pusher), toArray(sig));
assertEq(oracle.namespaceRemapping(pusher), creator);

// 3. Pusher revokes
vm.prank(pusher);
oracle.revokePusher();
assertEq(oracle.namespaceRemapping(pusher), address(0)); // cleared

// 4. Creator replays the SAME signature — no revert, delegation restored
vm.prank(creator);
oracle.allowPushers(deadline, toArray(pusher), toArray(sig));
assertEq(oracle.namespaceRemapping(pusher), creator); // silently re-established

// 5. Pusher pushes data they believe goes to their OWN namespace
uint56 tsMs = uint56(block.timestamp * 1000);
uint48 raw = _packRaw(9_000_000, 4, 2); // extreme price intended for pusher's own feed
vm.prank(pusher);
(bool ok,) = address(oracle).call(_wordAt(0, 0, raw, tsMs));
assertTrue(ok);

// Data lands in CREATOR's namespace, not pusher's
assertEq(oracle.getOracleData(oracle.feedIdOf(creator, 0, 0)).price, U64x32.decode(uint32(raw >> 16)));
assertEq(oracle.getOracleData(oracle.feedIdOf(pusher,  0, 0)).price, 0); // pusher's own ns untouched
// Creator's pool now reads the corrupted price on next swap
```

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
```
