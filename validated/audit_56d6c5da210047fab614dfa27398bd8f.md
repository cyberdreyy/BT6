### Title
Pusher Delegation Signature Replay After `revokePusher` Allows Creator to Permanently Override Revocation — (File: `smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

The `allowPushers` function signs consent over `(chainid, oracle_address, deadline, pusher, creator)` with no nonce. After a pusher calls `revokePusher()`, the creator can replay the original consent signature — while the deadline is still valid — to silently re-establish the delegation and redirect the pusher's future price pushes back into the creator's namespace, permanently overriding the revocation.

---

### Finding Description

The code comment in `allowPushers` explicitly acknowledges the replay risk:

> "The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it." [1](#0-0) 

The comment treats the deadline as the complete mitigation. It is not. The deadline only prevents replay **after it expires**. While the deadline is still in the future, the creator holds a fully valid, replayable credential.

The signed hash is:

```solidity
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [2](#0-1) 

There is no nonce, no current-state binding, and no check that `namespaceRemapping[pusher]` is currently `address(0)` before the overwrite:

```solidity
namespaceRemapping[pusher] = msg.sender;
``` [3](#0-2) 

`revokePusher()` clears the mapping to zero:

```solidity
namespaceRemapping[msg.sender] = address(0);
``` [4](#0-3) 

But `allowPushers` has no guard against re-establishing a delegation that was just revoked. The creator can immediately call `allowPushers` again with the identical signature to restore `namespaceRemapping[pusher] = creator`. The pusher's revocation is silently undone.

This is the direct analog to the M03 `safeApprove` bypass: just as `UniERC20` bypassed the non-zero→non-zero allowance restriction by first setting to zero (revoke) then setting to a new value (re-delegate), here the pusher's `revokePusher` sets the mapping to zero, but the creator immediately replays the old signature to set it back to non-zero — bypassing the revocation invariant.

The `_ensureDeadline` check in `OracleBase`:

```solidity
require(block.timestamp <= deadline, DeadlineExceeded());
``` [5](#0-4) 

only rejects expired deadlines. A deadline of `block.timestamp + 30 days` passes this check for 30 days after the pusher revokes.

---

### Impact Explanation

The `fallback` push path resolves the namespace at call time:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [6](#0-5) 

If the creator has replayed the old signature, `namespaceRemapping[pusher]` is non-zero again, so every subsequent push from the pusher's bot lands in the **creator's namespace** — not the pusher's own namespace as the pusher intended after revocation. The creator's oracle feeds (consumed by pools via `getOracleData` → `price` → `getBidAndAsk`) continue to receive live price updates from a pusher who believed they had revoked. The pusher cannot stop feeding the creator's pool until the deadline expires. This is an oracle role-check bypass: the pusher's revocation boundary is overridden by an unprivileged creator action.

---

### Likelihood Explanation

Pushers routinely sign consents with multi-day or multi-week deadlines (the test suite uses `block.timestamp + 1 days`). [7](#0-6) 

Any creator who retains the original consent bytes can replay at any time before expiry. No special privilege is required — `allowPushers` is a public function. The trigger is a valid semi-trusted creator acting on a legitimately obtained signature.

---

### Recommendation

Add a per-pusher revocation nonce to the signed message and increment it on every `revokePusher` / `removePushers` call:

```solidity
mapping(address => uint256) public pusherNonce;

// In allowPushers:
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender, pusherNonce[pusher]))

// In revokePusher / removePushers:
pusherNonce[pusher]++;
```

This makes every previously issued consent signature invalid the moment the pusher revokes, matching the security guarantee the comment claims the deadline provides.

### Proof of Concept

```solidity
// 1. Pusher signs consent for Creator A with a 30-day deadline
uint256 deadline = block.timestamp + 30 days;
bytes memory sig = pusher.sign(
    keccak256(abi.encode(block.chainid, oracle, deadline, pusher, creatorA))
);

// 2. Creator A establishes delegation
vm.prank(creatorA);
oracle.allowPushers(deadline, [pusher], [sig]);
// namespaceRemapping[pusher] == creatorA ✓

// 3. Pusher revokes
vm.prank(pusher);
oracle.revokePusher();
// namespaceRemapping[pusher] == address(0) ✓

// 4. Creator A replays the SAME signature (deadline still valid)
vm.prank(creatorA);
oracle.allowPushers(deadline, [pusher], [sig]);
// namespaceRemapping[pusher] == creatorA again — revocation bypassed ✗

// 5. Pusher's bot pushes a price — lands in creatorA's namespace, not pusher's own
vm.prank(pusher);
(bool ok,) = address(oracle).call(slotWord);
assertTrue(ok);
assertEq(oracle.getOracleData(oracle.feedIdOf(creatorA, slotId, pos)).price, pushedPrice);
assertEq(oracle.getOracleData(oracle.feedIdOf(pusher,   slotId, pos)).price, 0);
``` [8](#0-7) [9](#0-8)

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
