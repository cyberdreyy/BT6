### Title
Pusher Consent Signature Has No Nonce/Invalidation, Allowing Creator to Replay a Revoked Delegation and Feed Arbitrary Prices into Pools — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1.allowPushers` accepts an EIP-191 consent signature from a pusher but stores no nonce and marks no signature as used. A pusher's `revokePusher()` call clears `namespaceRemapping[pusher]` but does not invalidate the original consent bytes. The creator can immediately replay the identical signature (within the deadline window) to re-establish the delegation, making `revokePusher()` ineffective as a security mechanism. If the pusher's key is compromised, an attacker retains write authority over the creator's namespace and can push arbitrary prices that flow through `AnchoredPriceProvider` into live pool swaps.

---

### Finding Description

`allowPushers` hashes the consent as:

```
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [1](#0-0) 

There is no nonce field, no per-pusher revocation counter, and no "used signatures" set. The only replay guard is the deadline check:

```solidity
_ensureDeadline(deadline);
``` [2](#0-1) 

`revokePusher()` only zeroes the mapping entry:

```solidity
namespaceRemapping[msg.sender] = address(0);
``` [3](#0-2) 

It does not touch the consent signature, which remains cryptographically valid until `block.timestamp > deadline`. The creator can call `allowPushers` again with the exact same `(deadline, pusher, signature)` tuple and the check at line 207 passes identically, writing `namespaceRemapping[pusher] = creator` again.

The code comment at lines 186–191 acknowledges the deadline is meant to bound this window, but the deadline only prevents using an *expired* signature — it does **not** prevent replaying a still-valid signature after revocation. The pusher cannot permanently revoke until the deadline expires. [4](#0-3) 

Once the delegation is re-established, the `fallback` push path resolves the namespace from `namespaceRemapping[msg.sender]` and writes directly into the creator's storage slot:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
...
bytes32 key = bytes32(namespace | uint256(slotId));
...
_writeStorage(key, bytes32(bytes32(word & ~uint256(0xff))));
``` [5](#0-4) 

Those slot values are then decoded by `getOracleData` and consumed by `AnchoredPriceProvider._readLeg` → `getBidAndAskPrice` → pool swap: [6](#0-5) 

---

### Impact Explanation

An attacker who holds a compromised pusher key can push any `(price, spread0, spread1, timestamp)` tuple into the creator's namespace. The only on-chain guards are:

- **Timestamp monotonicity** — the attacker simply uses a timestamp newer than the last legitimate push.
- **`maxTimeDrift`** — the timestamp must not be in the future; the attacker uses `block.timestamp * 1000`.
- **`priceGuard`** — only enforced if the creator set one; most feeds have none.
- **`MAX_REF_STALENESS` / `MAX_SPREAD_BPS`** in `AnchoredPriceProvider` — the attacker sets a fresh timestamp and a spread below `MAX_SPREAD_BPS`.

A crafted slot word with an arbitrary mid price and a valid spread index passes all guards and reaches `_computeBidAsk`, producing a bad bid/ask that the pool executes against. Traders receive more output than the oracle permits, or the pool receives less input than owed — direct loss of pool assets.

---

### Likelihood Explanation

The scenario requires two conditions:

1. A pusher's EOA key is compromised (realistic for any hot-wallet pusher).
2. The creator replays the original consent signature — either unknowingly (thinking the pusher accidentally revoked) or because the creator is unaware of the compromise.

Condition 2 is the only friction. Because `allowPushers` is a public function and the signature is already on-chain (emitted or visible in the original transaction), the creator needs only one transaction to re-establish the delegation. The window is the full deadline duration (up to any value the creator chose at delegation time). This is a medium-likelihood, high-impact path.

---

### Recommendation

Add a per-pusher revocation nonce or a "used signatures" mapping so that `revokePusher()` permanently invalidates any previously issued consent:

```solidity
mapping(address => uint256) public pusherNonce; // incremented on every revoke

// In allowPushers, include the nonce in the signed hash:
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender, pusherNonce[pusher]))

// In revokePusher / removePushers, increment the nonce:
pusherNonce[pusher]++;
namespaceRemapping[pusher] = address(0);
```

This ensures that any signature issued before a revocation is permanently invalid, regardless of whether the deadline has elapsed.

bytes memory sig = sign(PUSHER_KEY, keccak256(abi.encode(

vm.prank(creator);

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }

    /// @notice Push path. Calldata = N × 32-byte slot words:
    ///         [data0:6][data1:6][data2:6][data3:6][ts:7][slotId:1]
    ///         The sender pushes into `namespaceRemapping[msg.sender]`, falling back to
    ///         its OWN namespace — a creator needs zero setup transactions to start
    ///         pushing. Each word carries its own timestamp (monotonicity is the only
    ///         freshness gate), so there is no deadline prefix.
    fallback() override external {
        uint256 end;
        uint256 namespace;

        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0)) creator = msg.sender;

        assembly ("memory-safe") {
            end := calldatasize()
            namespace := shl(96, creator) // [creator:20][zeros:12]
        }

        // 4 * 6 + 7 + 1 = 32 bytes per slot
        if (end == 0 || end % 32 != 0) revert BadCalldataLength();

        for (uint256 ptr = 0; ptr < end; ptr += 32) {
            uint256 word;
            assembly ("memory-safe") {
                word := calldataload(ptr)
            }
            // casting to 'uint8' is safe we want LSB
            // forge-lint: disable-next-line(unsafe-typecast)
            uint8 slotId = uint8(word);
            TimeMs timestampMs = toTimeMs(word >> 8 & X56);
            timestampMs.revertIfAfterBlockTimeWithDrift(MAX_TIME_DRIFT);
            bytes32 key = bytes32(namespace | uint256(slotId));
            uint256 old = uint256(_loadStorage(key));
            TimeMs oldTimestampMs = toTimeMs(old >> 8 & X56);

            bool newer = timestampMs.isAfter(oldTimestampMs);
            if (!newer) continue;

            _writeStorage(key, bytes32(bytes32(word & ~uint256(0xff))));
        }

    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
