### Title
Pusher consent signature is replayable within the deadline window, allowing creator to silently re-establish delegation after pusher revokes — (`CompressedOracleV1.allowPushers`)

### Summary
`CompressedOracleV1.allowPushers` accepts an EIP-191 pusher-consent signature that covers `(chainid, oracle, deadline, pusher, creator)` but includes **no nonce**. The same signature is valid for every call to `allowPushers` until the deadline expires. After a pusher calls `revokePusher()`, the creator can replay the original consent signature to silently re-establish the delegation — exactly the scenario the code comment warns about but fails to prevent.

### Finding Description

`allowPushers` hashes the consent as:

```solidity
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [1](#0-0) 

There is no nonce, no used-signature bitmap, and no per-pusher revocation counter. The code comment explicitly acknowledges the risk:

> "the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it." [2](#0-1) 

The comment claims the `deadline` is the mitigation. It is not: the deadline only prevents using the signature *after* it expires. Within the window `[now, deadline]`, the identical bytes can be submitted to `allowPushers` an unlimited number of times, each time overwriting `namespaceRemapping[pusher] = creator`.

`revokePusher` clears the mapping:

```solidity
namespaceRemapping[msg.sender] = address(0);
``` [3](#0-2) 

But the creator can immediately call `allowPushers` again with the same `(deadline, pusher, sig)` tuple, restoring `namespaceRemapping[pusher] = creator` without any new consent from the pusher.

After re-delegation, every `fallback()` push from the pusher's address lands in the creator's namespace:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
// ...
namespace := shl(96, creator)
``` [4](#0-3) 

The `CompressedOracleV1.price()` function is **permissionless** — it has no in-swap binding and no registration check — so any pool consuming a feed in the creator's namespace immediately receives the corrupted price:

```solidity
function price(bytes32 feedId, address /* pool */)
    external view
    returns (uint256 mid, uint256 spread, uint16 spread1, uint256 refTime)
{
    return _price(feedId);
}
``` [5](#0-4) 

### Impact Explanation

A pusher whose key is compromised calls `revokePusher()` to stop the attacker from writing bad prices. The creator — acting in good faith or maliciously — replays the original consent signature before the deadline, re-establishing the delegation. The attacker (holding the compromised pusher key) resumes pushing arbitrary `(price, s0, s1)` tuples into the creator's namespace. Because `CompressedOracleV1.price()` is permissionless, any pool whose `AnchoredPriceProvider` or `PriceProvider` is bound to a feed in that namespace receives the attacker-controlled mid price and spread, causing bad-price execution: swaps execute at an attacker-chosen rate, and traders or LPs suffer direct loss of principal.

### Likelihood Explanation

The creator is a semi-trusted, unprivileged actor (any address can be a creator). The replay requires only that the creator still holds the original `(deadline, pusher, sig)` tuple — a normal operational artifact. Pusher key compromise is a realistic operational risk. The deadline window is set by the creator and can be arbitrarily long (e.g., 30 days in tests). The attack requires no special on-chain state beyond the original delegation having occurred.

### Recommendation

Track consumed signatures with a `mapping(bytes32 => bool) usedConsents` keyed on the full digest, or add a per-pusher revocation nonce to the signed payload. Either approach makes a replayed signature produce a different or already-consumed digest after revocation:

```solidity
mapping(bytes32 => bool) private _usedConsents;

// inside allowPushers loop:
require(!_usedConsents[hash], "consent already used");
_usedConsents[hash] = true;
```

Alternatively, include a monotonically increasing `nonce[pusher]` in the signed message and increment it on every successful `allowPushers` or `revokePusher` call.

### Proof of Concept

```solidity
// 1. Pusher signs consent with deadline = now + 30 days
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(oracle), deadline, pusher, creator))
);
bytes memory sig = sign(PUSHER_KEY, hash);

// 2. Creator establishes delegation
vm.prank(creator);
oracle.allowPushers(deadline, _arr(pusher), _arr(sig));
assertEq(oracle.namespaceRemapping(pusher), creator); // delegated

// 3. Pusher revokes (e.g., key compromised)
vm.prank(pusher);
oracle.revokePusher();
assertEq(oracle.namespaceRemapping(pusher), address(0)); // cleared

// 4. Creator replays the SAME signature — no new consent needed
vm.prank(creator);
oracle.allowPushers(deadline, _arr(pusher), _arr(sig)); // succeeds!
assertEq(oracle.namespaceRemapping(pusher), creator);   // re-delegated

// 5. Attacker (with compromised pusher key) pushes a manipulated price
uint48 badPrice = _packRaw(0xFFFFFF, 0, 0); // extreme price
vm.prank(pusher); // attacker controls this key
(bool ok,) = address(oracle).call(_wordAt(0, 0, badPrice, uint56(block.timestamp * 1000 + 1)));
assertTrue(ok);

// 6. Pool reads the corrupted price — no registration or in-swap check on CompressedOracleV1
(uint256 mid,,,) = oracle.price(oracle.feedIdOf(creator, 0, 0), address(pool));
// mid is now attacker-controlled → bad-price execution in pool swap

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }

    /// @notice Allows a pusher to self-revoke their delegation. After revocation the
    ///         wallet pushes into its OWN namespace again (the registrationless default).
    function revokePusher() external {
        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
        namespaceRemapping[msg.sender] = address(0);
        emit PusherRevoked(msg.sender, creator);
    }
