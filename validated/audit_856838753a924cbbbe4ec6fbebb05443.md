### Title
`AnchoredProviderFactory.createAnchoredProvider` Registers Pool-Eligible Providers With Unvalidated `quoteFeedId`, Enabling Bad Synthetic-Ratio Prices in Pools — (`smart-contracts-poc/contracts/AnchoredProviderFactory.sol`)

---

### Summary

`AnchoredProviderFactory.createAnchoredProvider` validates the `oracle` address (must be in the admin allow-list) and `baseFeedId` (must have an envelope class with staleness/spread bounds), but performs **zero validation on `quoteFeedId`**. Any unprivileged caller can deploy an `AnchoredPriceProvider` in synthetic-ratio mode with an arbitrary second feed and have it registered as `isProvider(provider) == true` — the machine-checkable predicate that grants public-pool eligibility. When the pool calls `getBidAndAskPrice()`, the synthetic mid is computed as `price(baseFeedId) / price(quoteFeedId)`, where `quoteFeedId` is completely unaudited, producing an arbitrarily wrong bid/ask that reaches swap execution.

---

### Finding Description

`AnchoredProviderFactory.createAnchoredProvider` enforces three checks before deploying a provider:

1. `oracle` must be in the admin-curated allow-list (`_oracles.contains(oracle)`).
2. `baseFeedId` must resolve to an envelope class (explicit or `DEFAULT_CLASS`), and the caller's parameters must fall within that envelope's staleness/spread bounds.
3. Parameters are validated against the envelope corners. [1](#0-0) 

`quoteFeedId` is passed directly to the `AnchoredPriceProvider` constructor with no corresponding check: [2](#0-1) 

The `AnchoredPriceProvider` constructor stores `quoteFeedId` as an immutable without calling the oracle to verify the feed exists or is appropriate: [3](#0-2) 

The factory's own NatSpec comment claims "the provider constructor reverts `FeedNotFound` otherwise," but inspecting the constructor confirms **no such call is made** — the feed ID is stored unconditionally. [4](#0-3) 

At swap time, `_getBidAndAskPrice` reads both legs independently and computes the synthetic mid:

```solidity
mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);   // baseFeed / quoteFeed
spreadBps += spreadBps2;
``` [5](#0-4) 

Because `quoteFeedId` is never envelope-checked, an attacker can choose any feed present in the approved oracle — including one for a completely different asset — making the synthetic ratio arbitrarily wrong. The reference band in `_computeBidAsk` is derived from this already-wrong `mid`, so the clamp does not correct the error; it only prevents the custom source from quoting *tighter* than the (wrong) band. [6](#0-5) 

After deployment the provider is unconditionally added to `_providers`: [7](#0-6) 

`isProvider(provider)` therefore returns `true`, satisfying the public-pool eligibility predicate described in the factory's own documentation. [8](#0-7) 

---

### Impact Explanation

A pool created with this provider will execute every swap against a wrong synthetic price. Concretely:

- **Bad-price execution**: if `quoteFeedId` is a feed whose price is much higher than the intended quote asset (e.g., BTC/USD used as the quote leg of an ETH/USDC pool), the synthetic mid `ETH_USD / BTC_USD ≈ 0.03` is ~100 000× too low. Swappers can buy the base token for a tiny fraction of its true cost, draining the pool's base-token reserves.
- **Pool insolvency**: LP balances fail to cover the base-token claims of remaining LPs after the drain.
- **Broken swap flow**: if `quoteFeedId` is a feed that does not exist in the oracle, `_readLeg` returns `ok = false`, `getBidAndAskPrice` reverts with `FeedStalled`, and the pool is permanently DoS'd for swaps.

Both outcomes are within the allowed impact gate (bad-price execution reaching a pool swap; broken core pool functionality causing loss of LP principal).

---

### Likelihood Explanation

`createAnchoredProvider` is permissionless — any address may call it provided it supplies an allow-listed oracle and a `baseFeedId` with a configured envelope. No admin action is required after the initial oracle/envelope setup. The attacker only needs to:

1. Know which oracle addresses are allow-listed (public on-chain state).
2. Know which feed IDs exist in that oracle (observable from oracle events or the oracle's own registry).
3. Call `createAnchoredProvider` with a valid `baseFeedId` and a chosen `quoteFeedId` for a different asset.

The provider is immediately pool-eligible. Creating a pool with it and seeding initial liquidity is also permissionless. Likelihood is **medium**: the attack requires a social-engineering step (convincing LPs to deposit into the pool), but the technical preconditions are fully unprivileged.

---

### Recommendation

Validate `quoteFeedId` in `createAnchoredProvider` with the same rigour applied to `baseFeedId`:

1. **Require `quoteFeedId` to have an explicit feed class** (reject `DEFAULT_CLASS` fallback for the quote leg, or require an explicit admin assignment via `setFeedClass`). This ensures the quote feed has been reviewed and bounded.
2. **Apply envelope staleness/spread bounds to `quoteFeedId`** independently, or at minimum require that `feedClass[quoteFeedId] != bytes32(0)` so the admin has explicitly acknowledged the feed.
3. **Correct the misleading NatSpec** that claims the constructor validates feed existence — it does not.

Alternatively, restrict synthetic-ratio mode (`quoteFeedId != 0`) to admin-only deployment, consistent with the elevated risk of two-feed providers.

### Proof of Concept

```

  anchoredFactory.setEnvelope(DEFAULT_CLASS, {stalenessMin:0, stalenessMax:3600, ...})

  // quoteFeedId = BTC/USD (wrong quote leg — no validation performed)
  provider = anchoredFactory.createAnchoredProvider(
      approvedOracle,
      ETH_USD_FEED_ID,
      BTC_USD_FEED_ID,   // ← unvalidated; no revert

  //   Pool prices 1 WETH at ~0.00003 USDC instead of ~3000 USDC.

    function createAnchoredProvider(
        address oracle,
        bytes32 baseFeedId,
        bytes32 quoteFeedId,
        uint256 minMargin,
        uint256 maxRefStaleness,
        uint16  maxSpreadBps,
        bool    mutableParams,
        int256  marginStep,
        address baseToken,
        address quoteToken
    ) external override returns (address provider) {
        if (!_oracles.contains(oracle)) revert OracleNotAllowed(oracle);

        // Feeds without an explicit class fall back to the admin-configured DEFAULT_CLASS envelope.
        bytes32 classId = feedClass[baseFeedId];
        if (classId == bytes32(0)) classId = DEFAULT_CLASS;

        Envelope storage env = envelopes[classId];
        if (!env.exists) revert EnvelopeNotFound(classId);
        if (
            minMargin < env.minMarginMin || minMargin > env.minMarginMax
            || maxRefStaleness < env.stalenessMin || maxRefStaleness > env.stalenessMax
            || maxSpreadBps < env.maxSpreadMin || maxSpreadBps > env.maxSpreadMax
        ) revert ParamsOutOfEnvelope();

        AnchoredPriceProvider p = new AnchoredPriceProvider(
            address(this),
            oracle,
            baseFeedId,
            quoteFeedId,
            minMargin,
            maxRefStaleness,
            maxSpreadBps,
            mutableParams,
            marginStep,
            baseToken,
            quoteToken
        );

        provider = address(p);
        address creator = msg.sender;

        _providers.add(provider);
        _providersByCreator[creator].add(provider);
        providerOwner[provider] = creator;

        emit ProviderDeployed(
            provider,
            creator,
            baseFeedId,
            quoteFeedId,
            classId,
            p.baseToken(),
            p.quoteToken(),
            minMargin,
            maxRefStaleness,
            maxSpreadBps,
            mutableParams,
            marginStep,
            oracle
        );
    }

    function _getBidAndAskPrice() internal returns (uint128, uint128) {
        (uint256 mid, uint256 spreadBps, , bool ok) = _readLeg(baseFeedId);
        if (!ok) return (0, type(uint128).max);

        bytes32 _quote = quoteFeedId;
        if (_quote != bytes32(0)) {
            (uint256 mid2, uint256 spreadBps2, , bool ok2) = _readLeg(_quote);
            if (!ok2 || mid2 == 0) return (0, type(uint128).max);
            // Synthetic ratio (8-decimal): mid1 / mid2. Relative uncertainties of a ratio add.
            mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);
            spreadBps += spreadBps2;
        }

        return _computeBidAsk(mid, spreadBps);
    }

    function _computeBidAsk(uint256 mid, uint256 spreadBps)
        internal view returns (uint128, uint128)
    {
        // Circuit breaker: extreme (combined) uncertainty means the feed is clearly broken.
        if (spreadBps > MAX_SPREAD_BPS) {
            return (0, type(uint128).max);
        }

        // Reference band: mid ± (spreadBps + minMargin), bid rounded down, ask rounded up.
        uint256 half = spreadBps * ONE_BPS_E18 + minMargin; // < BPS_BASE_U by construction (spreadBps <= MAX_SPREAD_BPS here)
        uint256 refBid = _bandEdge(mid, BPS_BASE_U - half, Math.Rounding.Floor);
        uint256 refAsk = _bandEdge(mid, BPS_BASE_U + half, Math.Rounding.Ceil);
        if (refBid == 0 || refAsk > type(uint128).max || refBid >= refAsk) {
            return (0, type(uint128).max);
