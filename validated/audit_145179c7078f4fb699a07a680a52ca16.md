### Title
Creator Can Replay Pusher's EIP-191 Signature to Permanently Override `revokePusher()` Within the Deadline Window — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1.allowPushers()` has no nonce and no used-signature bitmap. A creator who holds a pusher's valid EIP-191 signature can call `allowPushers` repeatedly with the identical `(deadline, pusher, sig)` tuple, including after the pusher has called `revokePusher()`, re-writing `namespaceRemapping[pusher] = creator` and nullifying the revocation for as long as the deadline has not expired.

---

### Finding Description

`allowPushers` verifies the pusher's signature and writes `namespaceRemapping[pusher] = msg.sender`: [1](#0-0) 

The signed message binds `(chainid, address(this), deadline, pusher, creator)`. There is no per-signature nonce, no consumed-signature set, and no state that changes after the first call. The only replay guard is `_ensureDeadline`, which only checks `block.timestamp <= deadline`: [2](#0-1) 

`revokePusher()` zeroes the mapping: [3](#0-2) 

Because `allowPushers` re-writes the same slot unconditionally on every valid call, the creator can immediately undo the revocation by replaying the original signature. The code's own comment acknowledges the risk but claims the deadline is the mitigation: [4](#0-3) 

The deadline only prevents replay *after* it expires. Within the window — which can be hours or days depending on the off-chain signer — the creator can replay indefinitely.

---

### Impact Explanation

After the pusher revokes and begins pushing into their own namespace (`namespaceRemapping[pusher] == address(0)` → `fallback` uses `msg.sender` as creator): [5](#0-4) 

The creator replays the signature → `namespaceRemapping[pusher] = creator` again → the pusher's subsequent pushes are silently redirected into the creator's namespace. The pusher's own feeds receive no updates and go stale (price 0 / timestamp 0). Any pool consuming the pusher's own feeds will have its price reads rejected as stale, breaking swap/liquidity flows for those pools. This satisfies the **admin-boundary break** criterion (pusher's revocation right is bypassed by an unprivileged creator) and the **broken core pool functionality** criterion (pools depending on the pusher's own feeds become non-functional).

---

### Likelihood Explanation

The creator already holds the pusher's signature (obtained legitimately during the original `allowPushers` call). No additional privilege is needed. The attack is a single on-chain transaction callable by any creator who has a still-valid deadline. Deadlines are set off-chain and can span hours to days, making the window practically exploitable.

---

### Recommendation

Maintain a `mapping(bytes32 => bool) private _usedDelegations` keyed on the signature hash (or a `mapping(address => uint256) public pusherNonce` incremented on each `allowPushers` and included in the signed message). Either approach ensures a signature can only establish a delegation once, and a subsequent `revokePusher()` cannot be undone by replaying the original signature.

// Foundry property test sketch
function test_replayOverridesRevocation(uint256 deadline, uint8 N) public {
    vm.assume(deadline > block.timestamp && N > 1);

    // pusher signs consent
    bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
        keccak256(abi.encode(block.chainid, address(oracle), deadline, pusher, creator))
    );
    (uint8 v, bytes32 r, bytes32 s) = vm.sign(pusherKey, hash);
    bytes memory sig = abi.encodePacked(r, s, v);

    address[] memory pushers = new address[](1);
    pushers[0] = pusher;
    bytes[] memory sigs = new bytes[](1);
    sigs[0] = sig;

    // creator calls allowPushers N times with the same sig — all succeed
    for (uint256 i; i < N; i++) {
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);
        assertEq(oracle.namespaceRemapping(pusher), creator);
    }

    // pusher revokes
    vm.prank(pusher);
    oracle.revokePusher();
    assertEq(oracle.namespaceRemapping(pusher), address(0));

    // creator replays — revocation is undone
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs);
    assertEq(oracle.namespaceRemapping(pusher), creator); // revocation nullified
}

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
