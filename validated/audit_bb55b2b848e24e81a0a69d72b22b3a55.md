### Title
Delegation Consent Signature Replayable Within Deadline Window Bypasses Pusher Revocation — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`allowPushers` verifies a pusher's EIP-191 consent signature but never marks it as consumed. The only replay guard is the `deadline` field. Within the deadline window a creator can call `allowPushers` a second time with the identical signature to silently re-establish a delegation the pusher already revoked, permanently redirecting the pusher's future slot writes into the creator's namespace until the deadline expires.

### Finding Description

`allowPushers` builds and verifies a consent hash over `(block.chainid, address(this), deadline, pusher, msg.sender)` and then writes `namespaceRemapping[pusher] = msg.sender`: [1](#0-0) 

No used-signature bitmap, nonce, or per-pusher counter is stored. The contract's own NatSpec acknowledges the risk ("an undated signature could re-establish a delegation AFTER the pusher revoked it") and claims the deadline is the mitigation: [2](#0-1) 

However the deadline only prevents replay *after* it expires. Within the window the signature is valid for an unlimited number of `allowPushers` calls. `revokePusher` clears `namespaceRemapping[pusher]` to `address(0)`: [3](#0-2) 

But the creator still holds the original signature and can immediately replay it to restore `namespaceRemapping[pusher] = creator`, overwriting the revocation.

The `fallback` push path resolves the namespace at call time: [4](#0-3) 

So every subsequent push by the pusher lands in the creator's namespace rather than the pusher's own, regardless of the revocation.

### Impact Explanation

A pusher who revokes because they no longer trust a creator cannot prevent that creator from re-establishing the delegation within the deadline window. The pusher's price writes continue to populate the creator's feed namespace. If the creator's feeds are consumed by a pool via a `CompressedOracle`-backed price provider, the pool receives price data the pusher intended to stop delivering — a namespace hijack that can feed stale, redirected, or otherwise unintended bid/ask quotes into live swaps. This maps to the **bad-price execution** and **admin-boundary break** impact categories: an unprivileged path (replaying a public signature) bypasses the revocation control the protocol explicitly provides.

### Likelihood Explanation

The creator retains the original signature bytes from the first `allowPushers` call. Replaying requires only a single transaction before the deadline expires. Deadlines are typically set days in the future (the test suite uses `block.timestamp + 1 days`). Any creator who wants to retain a pusher's data stream after revocation can trivially exploit this.

### Recommendation

Record each accepted consent signature as used. A `mapping(bytes32 => bool) private _usedConsents` keyed on the signature hash (or the full message hash) prevents any signature from being accepted more than once:

```solidity
mapping(bytes32 => bool) private _usedConsents;

// inside allowPushers, after ECDSA.recover succeeds:
require(!_usedConsents[hash], "consent already used");
_usedConsents[hash] = true;
```

Alternatively, include a per-pusher monotonic nonce in the signed payload so each consent is inherently single-use.

### Proof of Concept

```solidity
// 1. Pusher signs consent for creator, deadline = now + 1 day
bytes memory sig = sign(PUSHER_KEY, chainid, oracle, deadline, pusher, creator);

// 2. Creator establishes delegation
oracle.allowPushers(deadline, [pusher], [sig]);
assert(oracle.namespaceRemapping(pusher) == creator);

// 3. Pusher revokes
vm.prank(pusher);
oracle.revokePusher();
assert(oracle.namespaceRemapping(pusher) == address(0));

// 4. Creator replays the SAME signature — deadline still valid
vm.prank(creator);
oracle.allowPushers(deadline, [pusher], [sig]);
// Delegation is silently re-established; revocation is bypassed
assert(oracle.namespaceRemapping(pusher) == creator);

// 5. Pusher's next fallback push lands in creator's namespace, not pusher's own
vm.prank(pusher);
address(oracle).call(slotWord);
assert(oracle.getOracleData(feedIdOf(creator, slotId, pos)).price != 0); // creator gets the data
assert(oracle.getOracleData(feedIdOf(pusher,   slotId, pos)).price == 0); // pusher's own ns empty
```

```text
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
```
