### Title
`PriceProvider.confidenceParam` Uninitialized to Zero Silently Drops Oracle Spread, Enabling Bad-Price Execution — (`File: smart-contracts-poc/contracts/PriceProvider.sol`)

---

### Summary

`PriceProvider` stores `confidenceParam` as a plain storage variable that is **never set in the constructor** and therefore defaults to `0`. Because the oracle's spread is multiplied by `confidenceParam` before being applied to the mid price, a zero value causes the oracle's entire uncertainty signal to be discarded. The resulting bid/ask is shaped only by the immutable `marginStep` bias, not by the oracle's live spread. This is the direct analog of the external "uninitialized discount variable" class: a parameter that governs a safety-critical scaling factor is never initialized, so the scaling factor is permanently zero until the factory explicitly intervenes.

---

### Finding Description

In `PriceProvider._getBidAndAskPrice()`:

```solidity
// 5. Compute bid/ask from mid + confidence-adjusted spread
//    confidenceParam multiplies oracle spread; 0 means no spread
uint256 adjustedSpread = spread * confidenceParam;          // ← always 0
(uint256 bid, uint256 ask) = _getBidAskFrom(mid, adjustedSpread);
``` [1](#0-0) 

`_getBidAskFrom` with `confidence = 0` computes `delta = 0`, returning `bid = mid, ask = mid` — a zero-spread quote:

```solidity
function _getBidAskFrom(uint256 midPrice, uint256 confidence) internal pure returns (uint256 bid, uint256 ask) {
    uint256 delta = midPrice * confidence / CONFIDENCE_BASE;
    bid = delta >= midPrice ? 0 : midPrice - delta;
    ask = midPrice + delta;
}
``` [2](#0-1) 

The constructor accepts `_marginStep` and `_maxTimeDelta` but **no `_confidenceParam`**, and never writes to `confidenceParam`: [3](#0-2) 

The only setter is `setConfidenceParam`, callable exclusively by the factory after deployment: [4](#0-3) 

Until the factory calls it, every deployed `PriceProvider` operates with `confidenceParam = 0`.

---

### Impact Explanation

Two distinct bad outcomes depending on the `marginStep` value chosen at construction:

**Case A — `marginStep = 0`:**
`stepBidFactor = stepAskFactor = BPS_BASE_U`, so `bidOut == askOut`. The invariant check `if (bidOut >= askOut)` fires and the function returns `(0, type(uint128).max)`. `getBidAndAskPrice()` then reverts with `FeedStalled`. [5](#0-4) 

Every swap through the pool reverts — broken core pool functionality, unusable swap flow.

**Case B — `marginStep > 0`:**
`bidOut < askOut` so the pool is live, but the spread is `2 × marginStep / BPS_BASE_U` wide regardless of what the oracle reports. If the oracle's live `spread` is, say, 500 bps (a volatile or off-hours market) and `marginStep` is 50 bps, the pool quotes at ±50 bps while the oracle's own uncertainty is ±500 bps. An informed trader can hit the pool at the tight price and immediately offset against the true market, extracting LP principal. This is a direct bad-price execution / LP loss scenario matching the allowed impact gate.

---

### Likelihood Explanation

Every freshly deployed `PriceProvider` starts with `confidenceParam = 0`. The factory must issue a separate `setConfidenceParam` transaction after deployment; there is no constructor path that sets it. Any pool that goes live before that call — or any pool whose factory owner never calls it — operates in the broken state permanently. Because the default is silent (no revert, no event, no on-chain indicator), operators may not notice.

---

### Recommendation

1. Add `_confidenceParam` as a constructor parameter and assign it to `confidenceParam` during construction, with the same bounds check already present in `setConfidenceParam` (`<= CONFIDENCE_MAX`).
2. Alternatively, add a `require(confidenceParam != 0)` guard in `_getBidAndAskPrice` so the pool fails closed rather than silently quoting with zero spread.
3. Mirror the `AnchoredPriceProvider` pattern, which documents the zero-confidence degenerate case explicitly and clamps to the reference band — `PriceProvider` has no such safety net.

---

### Proof of Concept

```
Deploy PriceProvider(factory, oracle, feedId, marginStep=50_bps_in_BPS_BASE, maxTimeDelta=1h, base, quote)
  → confidenceParam = 0 (default, never set)

Oracle pushes: mid = 2000e8, spread = 500 (5 %)

Call getBidAndAskPrice():
  adjustedSpread = 500 * 0 = 0
  _getBidAskFrom(2000e8, 0) → bid = 2000e8, ask = 2000e8
  _applyBidAdjustments(2000e8) → bidOut = 2000e8 * Q64 * (1e18 - 50e14) / 1e26
  _applyAskAdjustments(2000e8) → askOut = 2000e8 * Q64 * (1e18 + 50e14) / 1e26
  bidOut < askOut → pool returns a ±0.5% spread

True oracle uncertainty = ±5%.
Attacker buys at askOut (pool's ask), sells on external market at true ask (+5%).
Net gain ≈ 4.5% of trade size, extracted from LP reserves.

    function _getBidAndAskPrice() internal returns (uint128, uint128) {
        // 1. Read via the unified price(feedId, pool) path, forwarding the pool (msg.sender).
        //    refTime is already in seconds.
        (uint256 mid, uint256 spread, , uint256 refTime) =
            IPricedOracle(address(offchainOracle)).price(offchainFeedId, msg.sender);

        // 2. Staleness check
        if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA)) {
            return (0, type(uint128).max);
        }

        // 3. Basic validity — price must be positive, spread must not be stalled marker
        if (mid == 0 || spread >= ORACLE_BPS) {
            return (0, type(uint128).max);
        }

        // 4. Price guard check (moved from oracle)
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(offchainFeedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) {
            return (0, type(uint128).max);
        }

        // 5. Compute bid/ask from mid + confidence-adjusted spread
        //    confidenceParam multiplies oracle spread; 0 means no spread
        uint256 adjustedSpread = spread * confidenceParam;
        (uint256 bid, uint256 ask) = _getBidAskFrom(mid, adjustedSpread);

        // 6. Apply marginStep adjustment
        (uint256 bidOut, bool bidOk) = _applyBidAdjustments(bid);
        if (!bidOk || bidOut > type(uint128).max) return (0, type(uint128).max);

        (uint256 askOut, bool askOk) = _applyAskAdjustments(ask);
        if (!askOk || askOut > type(uint128).max) return (0, type(uint128).max);

        // 7. Hard invariant: bid must be strictly less than ask.
        //    Can be violated when marginStep < 0 and confidence is too small.
        if (bidOut >= askOut) return (0, type(uint128).max);

        return (uint128(bidOut), uint128(askOut));
    }
