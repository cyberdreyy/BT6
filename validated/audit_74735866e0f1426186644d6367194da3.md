### Title
`revokePusher()` Is Ineffective Within the Deadline Window Due to Missing Signature-Use Tracking in `allowPushers` — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1.allowPushers` accepts an EIP-191 pusher-consent signature but tracks **no nonce and no used-signature state**. The same signature is valid for every call to `allowPushers` until `block.timestamp > deadline`. A creator who holds a pusher's consent signature can therefore replay it immediately after the pusher calls `revokePusher()`, silently re-establishing the delegation and trapping the pusher in the delegated state for the full lifetime of the deadline.

---

### Finding Description

`allowPushers` verifies the pusher's EIP-191 signature over `(chainid, address(this), deadline, pusher, msg.sender)` and writes `namespaceRemapping[pusher] = msg.sender`. [1](#0-0) 

The only replay guard is the deadline check in `_ensureDeadline`: [2](#0-1) 

There is no mapping of `usedSignatures`, no per-pusher nonce, and no flag that marks a consent as consumed. The same `(deadline, pusher, creator)` tuple — and therefore the same bytes — passes `ECDSA.recover` identically on every call until the deadline expires.

`revokePusher()` clears `namespaceRemapping[msg.sender]` to `address(0)`: [3](#0-2) 

But because the signature is not consumed, the creator can call `allowPushers` again in the same block with the identical signature bytes, writing `namespaceRemapping[pusher] = creator` again. The pusher's revocation is undone atomically.

The NatDoc comment acknowledges the problem but misidentifies the fix: [4](#0-3) 

The comment says "an undated signature could re-establish a delegation after the pusher revoked it" and claims the deadline prevents this. It does not: the deadline only bounds the outer window; within that window the signature is replayable without limit.

The existing test suite verifies that `revokePusher()` clears the mapping and that a post-revoke push lands in the pusher's own namespace, but it never tests that the creator cannot immediately re-delegate using the same signature: [5](#0-4) 

---

### Impact Explanation

The `fallback` push path resolves the namespace at call time from `namespaceRemapping[msg.sender]`: [6](#0-5) 

If the pusher continues to push (e.g., because they are an automated bot that cannot be stopped instantly, or because they are pushing to their own namespace and the delegation is silently re-active), every push lands in the creator's namespace instead of the pusher's own. The creator's pools then consume those prices through `AnchoredPriceProvider.getBidAndAskPrice()` → `_readLeg()` → `offchainOracle.price(feedId, pool)`: [7](#0-6) 

If the pusher's feed is compromised or stale at the moment of revocation, the creator can keep those bad prices live in their namespace — and therefore in any pool bound to that feed — for the full deadline window. This is a bad-price execution path: a stale or manipulated bid/ask quote reaches a live pool swap.

---

### Likelihood Explanation

- The pusher must have previously signed a consent with a non-trivial deadline (common in production deployments where long-lived keys are used).
- The creator must retain the original signature bytes (trivially available from the original `allowPushers` calldata on-chain).
- The creator calls `allowPushers` again in the same block as the pusher's `revokePusher()` — a single transaction, no mempool race required.
- The trigger is the creator (a semi-trusted actor who owns the namespace), which is an explicitly allowed trigger class per the task scope.

---

### Recommendation

Mark each consent signature as consumed after first use. The minimal fix is a `mapping(bytes32 => bool) private _usedConsents` keyed on `keccak256(signature)` (or on the full signed digest), set to `true` inside `allowPushers` and checked before writing `namespaceRemapping`. Alternatively, include a per-pusher nonce in the signed payload and increment it on each successful delegation so that a revoked pusher can invalidate all prior signatures by incrementing their nonce.

---

### Proof of Concept

```solidity
// Foundry test — add to CompressedOracle.t.sol

function testRevokeCanBeUndoneByCreatorReplay() public {
    uint256 deadline = block.timestamp + 365 days; // long-lived consent

    // 1. Pusher signs consent once
    bytes memory sig = _signConsent(PUSHER_KEY, deadline, pusher, creator);
    address[] memory pushers_ = new address[](1);
    pushers_[0] = pusher;
    bytes[] memory sigs_ = new bytes[](1);
    sigs_[0] = sig;

    // 2. Creator delegates pusher
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers_, sigs_);
    assertEq(oracle.namespaceRemapping(pusher), creator);

    // 3. Pusher self-revokes
    vm.prank(pusher);
    oracle.revokePusher();
    assertEq(oracle.namespaceRemapping(pusher), address(0), "revoke cleared");

    // 4. Creator replays the SAME signature — succeeds, revocation undone
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers_, sigs_); // no revert
    assertEq(
        oracle.namespaceRemapping(pusher),
        creator,
        "delegation silently re-established after pusher revoked"
    );

    // 5. Pusher's next push still lands in creator namespace, not pusher's own
    uint56 tsMs = uint56(block.timestamp * 1000);
    uint48 raw  = _packRaw(999_999, 5, 5);
    vm.prank(pusher);
    (bool ok,) = address(oracle).call(_wordAt(0, 0, raw, tsMs));
    assertTrue(ok);
    assertEq(
        oracle.getOracleData(oracle.feedIdOf(creator, 0, 0)).price,
        U64x32.decode(uint32(raw >> 16)),
        "bad price lands in creator namespace despite revocation"
    );
    assertEq(oracle.getOracleData(oracle.feedIdOf(pusher, 0, 0)).price, 0);
}
```

```text
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();

            );
```

```text
    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
```
