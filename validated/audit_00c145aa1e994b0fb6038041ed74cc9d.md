Looking at the L2 provider contracts and their staleness logic to find the analog.

### Title
`ProtectedPriceProviderL2` and `PriceProviderL2` Missing L2 Sequencer Uptime Check Allows Stale Prices to Feed Pool Swaps — (`smart-contracts-poc/contracts/ProtectedPriceProviderL2.sol`, `smart-contracts-poc/contracts/PriceProviderL2.sol`)

---

### Summary

Both L2 price providers check oracle staleness only via a time-delta comparison against `refTime`. Neither checks whether the L2 sequencer itself is live. When the sequencer is down, the last published oracle price appears fresh (within `MAX_TIME_DELTA`) but reflects a pre-downtime market state. That stale bid/ask quote propagates directly into pool swaps.

---

### Finding Description

`ProtectedPriceProviderL2` is explicitly described as the "L2 price provider for the abuse-protected providers oracle (Pyth / Chainlink)." Its `_getBidAndAskPrice` reads the oracle and then delegates to `_computeBidAsk`: [1](#0-0) 

`_computeBidAsk` runs a staleness check: [2](#0-1) 

The `_isStale` helper only compares `refTime` against `block.timestamp ± futureTol / maxDelta`: [3](#0-2) 

There is no call to a Chainlink sequencer uptime feed anywhere in this contract. The identical gap exists in `PriceProviderL2`: [4](#0-3) 

Both contracts acknowledge L2-specific sequencer concerns only for **clock skew** (the `FUTURE_TOLERANCE` immutable), not for **sequencer liveness**: [5](#0-4) 

When the Arbitrum (or any L2) sequencer goes down:

1. Chainlink / Pyth oracle data stops updating on-chain.
2. The last published `refTime` is within `MAX_TIME_DELTA` (which can be up to 7 days by the constructor bound).
3. `_isStale` returns `false` — the price passes as fresh.
4. The stale bid/ask is returned to the pool and used for swap settlement. [6](#0-5) 

---

### Impact Explanation

A stale bid/ask quote reaches `getBidAndAskPrice()` and is consumed by the pool during a swap. Traders can exploit the gap between the stale on-chain price and the true market price to extract value from LPs — a direct loss of LP principal matching the "bad-price execution" impact gate.

---

### Likelihood Explanation

L2 sequencer outages are a documented, recurring event on Arbitrum and other OP-stack chains. No privileged action is required: any user can call the pool's swap function during a sequencer downtime window. The window can last minutes to hours, and `MAX_TIME_DELTA` up to 7 days means the staleness guard never fires during a realistic outage. [7](#0-6) 

---

### Recommendation

Add a Chainlink sequencer uptime check in `_getBidAndAskPrice` (or in a shared internal called before `_isStale`) for both L2 providers, following the Chainlink L2 sequencer feed pattern:

```solidity
// Example — add to ProtectedPriceProviderL2 / PriceProviderL2
AggregatorV2V3Interface sequencerUptimeFeed; // set at construction

function _checkSequencer() internal view {
    (, int256 answer, uint256 startedAt, ,) = sequencerUptimeFeed.latestRoundData();
    // 0 = sequencer up, 1 = sequencer down
    require(answer == 0, "Sequencer down");
    require(block.timestamp - startedAt > GRACE_PERIOD, "Grace period not elapsed");
}
```

Call `_checkSequencer()` at the top of `_getBidAndAskPrice()` before the oracle read. The `sequencerUptimeFeed` address and `GRACE_PERIOD` (Chainlink recommends ≥ 3600 s) should be immutables set at construction.

---

### Proof of Concept

1. Deploy `ProtectedPriceProviderL2` on Arbitrum with `MAX_TIME_DELTA = 86400` (24 h heartbeat feed).
2. Chainlink last updated the feed at `T = block.timestamp - 3600` (1 hour ago). `_isStale` passes.
3. Arbitrum sequencer goes down at `T + 1`. No new Chainlink updates can land.
4. At `T + 7200` (2 hours later, sequencer still down), a user calls the pool's swap.
5. `_getBidAndAskPrice` reads `refTime = T`, computes `nowTs - refTime = 7200 < 86400` → not stale.
6. The 2-hour-old price is returned as the live bid/ask; the pool executes the swap at the stale rate.
7. If the true market moved 5% during the outage, the LP absorbs that loss on every swap until the sequencer recovers and the oracle updates. [8](#0-7)

        uint256 nowTs,
        uint256 maxDelta,
        uint256 futureTol
    ) internal pure returns (bool) {
        if (refTime == 0) return true;

        if (refTime > nowTs) {
            // refTime in the future: tolerate only within futureTol
            return (refTime - nowTs) > futureTol;
        }

        // refTime in the past or equal: check age
        return (nowTs - refTime) > maxDelta;
    }
