### Title
`revokePusher()` Does Not Invalidate the Signed Consent, Allowing Creator to Replay Delegation After Pusher Revokes — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`revokePusher()` clears `namespaceRemapping[msg.sender]` but does not invalidate the pusher's previously signed EIP-191 consent. A creator can immediately replay the original `allowPushers` signature (any time before the deadline) to re-establish the delegation, making the pusher's revocation ineffective for the entire deadline window and routing the pusher's subsequent price pushes into the creator's namespace without the pusher's knowledge.

---

### Finding Description

`allowPushers` accepts a pusher's EIP-191 signature over `(chainid, oracle, deadline, pusher, creator)` and writes `namespaceRemapping[pusher] = creator`:

```solidity
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
);
require(pusher == ECDSA.recover(hash, signatures[i]));
namespaceRemapping[pusher] = msg.sender;
``` [1](#0-0) 

`revokePusher()` clears the mapping entry but performs no further cleanup — there is no nonce increment, no used-signature bitmap, and no per-pusher revocation timestamp:

```solidity
function revokePusher() external {
    address creator = namespaceRemapping[msg.sender];
    if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
    namespaceRemapping[msg.sender] = address(0);
    emit PusherRevoked(msg.sender, creator);
}
``` [2](#0-1) 

Because the signed message contains no revocation nonce and no "used" flag is ever set, the creator can call `allowPushers` again with the identical `(deadline, pusher, sig)` tuple — as many times as desired — until the deadline timestamp passes. The code's own comment acknowledges the replay risk but only addresses the post-deadline case:

> *"The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it."* [3](#0-2) 

The deadline prevents replay only after it expires. Within the deadline window, the creator can replay the signature an unlimited number of times, immediately undoing every `revokePusher()` call the pusher makes.

The `fallback()` push path resolves the target namespace at call time:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [4](#0-3) 

So every push the pusher makes after the creator replays the signature lands in the creator's namespace, not the pusher's own namespace, regardless of the pusher's intent.

---

### Impact Explanation

A pusher who discovers the creator is acting maliciously (e.g., the creator's namespace feeds a pool with manipulated parameters, or the creator is front-running the pusher's data) calls `revokePusher()` to stop contributing to that namespace. The creator immediately replays the original consent signature. The pusher's subsequent price pushes — intended for their own namespace — are silently redirected into the creator's namespace and consumed by any pool registered against that namespace. The pusher has no on-chain recourse until the deadline expires; they must cease all pushing activity entirely to avoid feeding the creator's pool.

This breaks the core invariant that a pusher's revocation is final and immediate. It is the direct analog of the external report: just as `burn()` in CollateralizedDebt.sol leaves the ERC721 token alive (the "burn" is incomplete), `revokePusher()` here leaves the signed consent alive (the revocation is incomplete).

---

### Likelihood Explanation

The creator must have retained the original `(deadline, sig)` tuple — trivially true since they submitted it on-chain and it is visible in calldata. The replay requires a single transaction. The window is as long as the deadline the pusher originally signed (potentially days or weeks). Any creator who wishes to retain a pusher's data feed against the pusher's will can do so at negligible cost.

---

### Recommendation

Track consumed signatures or add a per-pusher revocation nonce included in the signed message. The simplest fix is a `mapping(bytes32 => bool) private usedConsents` keyed on the signature hash, set to `true` in `allowPushers` and checked before accepting a signature. Alternatively, include a per-pusher on-chain nonce (incremented by `revokePusher`) in the signed payload so that any signature issued before the last revocation is automatically invalid.

---

### Proof of Concept

1. Pusher signs consent: `sig = sign(keccak256(abi.encode(chainid, oracle, deadline, pusher, creator)))` with `deadline = block.timestamp + 7 days`.
2. Creator calls `allowPushers(deadline, [pusher], [sig])` → `namespaceRemapping[pusher] = creator`.
3. Pusher calls `revokePusher()` → `namespaceRemapping[pusher] = address(0)`.
4. Creator immediately calls `allowPushers(deadline, [pusher], [sig])` again with the identical arguments → `namespaceRemapping[pusher] = creator` is restored. No revert; the deadline has not passed; no nonce or used-flag exists.
5. Pusher's next `fallback()` push resolves `namespaceRemapping[pusher] == creator` and writes into the creator's storage slot, feeding the creator's pool — contrary to the pusher's explicit revocation. [5](#0-4) [6](#0-5)

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }

    fallback() override external {
        uint256 end;
        uint256 namespace;

        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0)) creator = msg.sender;

        assembly ("memory-safe") {
            end := calldatasize()
            namespace := shl(96, creator) // [creator:20][zeros:12]
        }

        // 4 * 6 + 7 + 1 = 32 bytes per slot
        if (end == 0 || end % 32 != 0) revert BadCalldataLength();

        for (uint256 ptr = 0; ptr < end; ptr += 32) {
            uint256 word;
            assembly ("memory-safe") {
                word := calldataload(ptr)
            }
            // casting to 'uint8' is safe we want LSB
            // forge-lint: disable-next-line(unsafe-typecast)
            uint8 slotId = uint8(word);
            TimeMs timestampMs = toTimeMs(word >> 8 & X56);
            timestampMs.revertIfAfterBlockTimeWithDrift(MAX_TIME_DRIFT);
            bytes32 key = bytes32(namespace | uint256(slotId));
            uint256 old = uint256(_loadStorage(key));
            TimeMs oldTimestampMs = toTimeMs(old >> 8 & X56);

            bool newer = timestampMs.isAfter(oldTimestampMs);
            if (!newer) continue;

            _writeStorage(key, bytes32(bytes32(word & ~uint256(0xff))));
        }
