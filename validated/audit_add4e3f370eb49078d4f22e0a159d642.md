### Title
Stale `allowPushers` Signature Replay Re-establishes Revoked Pusher Delegation, Enabling Bad-Price Injection — (File: smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol)

---

### Summary

`CompressedOracleV1::allowPushers` contains no nonce or used-signature tracking. After a pusher calls `revokePusher()`, the creator can replay the identical EIP-191 consent signature (same deadline, same pusher, same creator) to immediately re-establish delegation, silently undoing the pusher's revocation. If the pusher's key is compromised, this allows an attacker to push bad prices into the creator's namespace and into any pool that reads from it.

---

### Finding Description

`allowPushers` signs consent over `(chainid, oracle, deadline, pusher, creator)`:

```solidity
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
);
require(pusher == ECDSA.recover(hash, signatures[i]));
namespaceRemapping[pusher] = msg.sender;
```

The only replay guard is `_ensureDeadline(deadline)` — `require(block.timestamp <= deadline)`. There is no nonce, no per-pusher revocation counter, and no used-signature bitmap. The same `(deadline, sig)` tuple is accepted an unlimited number of times within the deadline window.

`revokePusher()` clears the mapping:

```solidity
namespaceRemapping[msg.sender] = address(0);
```

But the old signature remains cryptographically valid until `deadline` expires. The creator can immediately call `allowPushers` again with the same arguments, setting `namespaceRemapping[pusher] = creator` again. The code comment itself acknowledges the gap:

> *"The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it."*

The deadline limits the window but does not prevent replay within it. The analog to the external bug is exact: just as `_cumulativeRewardPerLpToken` is not reset when a reward token is re-added (allowing stale accumulator state to persist), the pusher's revocation state is not made permanent — the stale consent signature persists and can re-establish the delegation without fresh pusher consent.

After re-delegation, the `fallback()` push path resolves the namespace from `namespaceRemapping[msg.sender]` and writes directly into the creator's storage slot:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
...
_writeStorage(key, bytes32(bytes32(word & ~uint256(0xff))));
```

Any price word that passes the per-slot monotonicity check (`timestampMs.isAfter(oldTimestampMs)`) is stored and immediately readable by `getOracleData` / `price`, which is the data source for `AnchoredPriceProvider::getBidAndAskPrice` consumed by pool swaps.

---

### Impact Explanation

**High.** A compromised pusher key that has been self-revoked can be re-activated by the creator replaying the old consent signature. The attacker then pushes an arbitrary price word through `fallback()` into the creator's namespace. `AnchoredPriceProvider` reads this via `getOracleData` → `_price` → `getBidAndAskPrice`, and the pool executes swaps at the attacker-controlled bid/ask. This is a direct bad-price execution path: the pool receives a quote it should not have accepted, leading to swap conservation failure or pool insolvency.

---

### Likelihood Explanation

**Low.** The scenario requires: (1) a pusher's key to be compromised, (2) the pusher to call `revokePusher()`, and (3) the creator to call `allowPushers` again with the old signature before the deadline expires — either unaware of the compromise or acting in bad faith. The `PusherRevoked` event is emitted on revocation, so an attentive creator would not replay. However, the deadline window can be up to any future timestamp the pusher originally signed, and there is no on-chain enforcement preventing the replay.

---

### Recommendation

Track a per-pusher revocation nonce or a used-signature bitmap. The simplest fix is a `mapping(address => uint256) public pusherNonce` incremented on every `revokePusher()` / `removePushers()` call, included in the signed digest:

```solidity
keccak256(abi.encode(block.chainid, address(this), deadline, pusherNonce[pusher], pusher, msg.sender))
```

After revocation, `pusherNonce[pusher]++` invalidates all previously issued signatures for that pusher, regardless of their deadline.

---

    /// @notice Delegates pusher wallets into the caller's namespace. The pusher's EIP-191
    ///         signature is REQUIRED — without it anyone could remap a foreign pusher
    ///         wallet into their own namespace and silently swallow its pushes. The
    ///         deadline is likewise required: the signed consent carries no timestamp of
    ///         its own, so an undated signature could re-establish a delegation AFTER the
    ///         pusher revoked it.
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }

    /// @notice Allows a pusher to self-revoke their delegation. After revocation the
    ///         wallet pushes into its OWN namespace again (the registrationless default).
    function revokePusher() external {
        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
        namespaceRemapping[msg.sender] = address(0);
        emit PusherRevoked(msg.sender, creator);

    fallback() override external {
        uint256 end;
        uint256 namespace;

        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0)) creator = msg.sender;

        assembly ("memory-safe") {
            end := calldatasize()
            namespace := shl(96, creator) // [creator:20][zeros:12]
        }

        // 4 * 6 + 7 + 1 = 32 bytes per slot
        if (end == 0 || end % 32 != 0) revert BadCalldataLength();

        for (uint256 ptr = 0; ptr < end; ptr += 32) {
            uint256 word;
            assembly ("memory-safe") {
                word := calldataload(ptr)
            }
            // casting to 'uint8' is safe we want LSB
            // forge-lint: disable-next-line(unsafe-typecast)

            if (!newer) continue;
