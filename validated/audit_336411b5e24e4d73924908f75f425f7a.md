### Title
Pusher Self-Revocation Is Not Final: Creator Can Replay Unexpired Signature to Re-Establish Delegation Without New Pusher Consent — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowPushers` contains no nonce or used-signature tracking. A creator who holds a pusher's still-valid (deadline not yet expired) consent signature can call `allowPushers` again after the pusher has called `revokePusher()`, unconditionally overwriting `namespaceRemapping[pusher] = creator` and restoring the delegation the pusher explicitly terminated.

---

### Finding Description

The signature committed to in `allowPushers` covers only `(block.chainid, address(this), deadline, pusher, msg.sender)`: [1](#0-0) 

There is no nonce, no per-signature consumed flag, and no check that `namespaceRemapping[pusher]` is currently `address(0)` before writing. The only guard is `_ensureDeadline`, which only rejects calls made *after* the deadline: [2](#0-1) 

`revokePusher` clears the mapping to `address(0)`: [3](#0-2) 

But nothing prevents the creator from immediately calling `allowPushers` again with the identical signature, re-setting `namespaceRemapping[pusher] = creator`. The code's own NatSpec acknowledges the intent:

> *"The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it."* [4](#0-3) 

The comment documents the invariant but the implementation does not enforce it — the deadline only prevents replay *after* expiry, not within the deadline window.

---

### Impact Explanation

The pusher's `revokePusher()` right is the only on-chain mechanism for a pusher to stop writing into a creator's namespace. If that right can be silently undone by the creator replaying the original signature, then:

1. A pusher whose key is compromised calls `revokePusher()` to protect the creator's feeds.
2. The creator (unaware of the compromise, or acting adversarially) calls `allowPushers` again with the same signature before the deadline.
3. The compromised pusher resumes writing into the creator's namespace.
4. Pools consuming those feeds receive manipulated prices, enabling bad-price execution.

This satisfies the contest's **bad-price execution** impact gate and violates the documented REVOCATION_FINALITY_INVARIANT.

---

### Likelihood Explanation

- The creator must hold the original signature (they always do — they submitted it in step 1).
- The deadline must not yet have expired (long deadlines, e.g. 1 year, are common in practice).
- No privileged role is required; `allowPushers` is a public function callable by any address.
- The pusher has no on-chain way to prevent this replay short of waiting for the deadline to expire.

---

### Recommendation

Track consumed signatures with a `mapping(bytes32 => bool) private _usedConsents` and mark each signature hash as used on first acceptance. Alternatively, include a per-pusher nonce in the signed message and increment it on each successful `allowPushers` call, so that `revokePusher` can also increment the nonce, invalidating all prior signatures immediately.

---

### Proof of Concept

```solidity
// Foundry unit test (pseudo-code)
function testRevokedPusherResumesByCreatorReplay() public {
    uint256 deadline = block.timestamp + 365 days;
    bytes memory sig = _signConsent(PUSHER_KEY, deadline, pusher, creator);

    // Step 1: creator delegates pusher
    vm.prank(creator);
    oracle.allowPushers(deadline, _arr(pusher), _arr(sig));
    assertEq(oracle.namespaceRemapping(pusher), creator);

    // Step 2: pusher self-revokes
    vm.prank(pusher);
    oracle.revokePusher();
    assertEq(oracle.namespaceRemapping(pusher), address(0));

    // Step 3: creator replays the SAME signature — no new pusher consent
    vm.prank(creator);
    oracle.allowPushers(deadline, _arr(pusher), _arr(sig)); // succeeds
    assertEq(oracle.namespaceRemapping(pusher), creator);   // delegation restored

    // Step 4: pusher (now compromised) pushes bad price into creator namespace
    vm.prank(pusher);
    (bool ok,) = address(oracle).call(_wordAt(0, 0, _packRaw(BAD_PRICE, 0, 0), tsMs));
    assertTrue(ok);
    // creator's feed now contains attacker-controlled price
}
```
