### Title
`MetricOmmPoolFactory.createPool` accepts any arbitrary price provider without verifying factory origin, allowing bad-price execution against pools — (`metric-core/contracts/MetricOmmPoolFactory.sol`)

---

### Summary

`MetricOmmPoolFactory.createPool` is permissionless and validates the `priceProvider` argument only by checking that `token0()`/`token1()` match the pool's tokens. It never verifies that the provider was deployed by `PriceProviderFactory` or `AnchoredProviderFactory`. A pool creator can supply a malicious price provider that passes the token check but returns arbitrary bid/ask prices, causing every swap in that pool to execute at a manipulated rate.

---

### Finding Description

`MetricOmmPoolFactory._validatePriceProvider` is the sole gate on the price provider at pool creation: [1](#0-0) 

```solidity
function _validatePriceProvider(address token0, address token1, address priceProvider) internal view {
    if (priceProvider == address(0)) revert InvalidPriceProvider();
    if (IPriceProvider(priceProvider).token0() != token0 || IPriceProvider(priceProvider).token1() != token1) {
      revert PriceProviderTokenMismatch();
    }
}
```

The check is purely structural: any contract that exposes `token0()` and `token1()` returning the right addresses passes. There is no call to `PriceProviderFactory.isProvider()` or `AnchoredProviderFactory.isProvider()`.

The `AnchoredProviderFactory` explicitly documents `isProvider` as the machine-checkable predicate for public-pool eligibility: [2](#0-1) 

```solidity
/// @notice The public-pool eligibility predicate: deployed by this factory ⇒ clamp-bounded quotes
///         with parameters that were inside the envelope at deploy time.
function isProvider(address provider) external view returns (bool) {
    return _providers.contains(provider);
}
```

`AnchoredProviderFactory` enforces that every legitimate provider is anchored to an admin-curated oracle, has clamp parameters validated against an envelope, and is bound to an allow-listed reference oracle: [3](#0-2) 

None of these invariants are enforced when `createPool` is called. The `createPool` function is permissionless: [4](#0-3) 

A malicious provider needs only to implement:

```solidity
function token0() external view returns (address) { return legitimateToken0; }
function token1() external view returns (address) { return legitimateToken1; }
function getBidAndAskPrice() external returns (uint128 bid, uint128 ask) {
    return (1, type(uint128).max - 1); // or any manipulated value
}
```

This passes `_validatePriceProvider` and is stored as the pool's price provider. Every subsequent swap reads from it.

The pool's swap path calls `getBidAndAskPrice()` on whatever address is stored as the price provider, with no further authenticity check at read time: [5](#0-4) 

---

### Impact Explanation

Every swap in the malicious pool executes at the attacker-controlled bid/ask. Traders receive more or less than the oracle-permitted amount (swap conservation failure / bad-price execution). LPs suffer direct principal loss because the pool's reserves are drained at the wrong rate. The pool can be rendered insolvent: LP claims exceed pool balances. This matches the allowed impact gate: bad-price execution, pool insolvency, and broken core swap functionality causing loss of funds.

---

### Likelihood Explanation

`createPool` is permissionless — any address can call it with zero privilege. The only cost is deploying a two-function mock contract. The pool appears in the factory's `idxToPool` registry and `isPool` returns `true`, giving it the appearance of legitimacy to integrators who only check factory membership. Users who verify `isPool(pool) == true` but do not separately verify `AnchoredProviderFactory.isProvider(pool.priceProvider) == true` are fully exposed.

---

### Recommendation

In `_validatePriceProvider`, add a factory-origin check. The factory should hold a reference to the canonical `AnchoredProviderFactory` (and optionally `PriceProviderFactory`) and require:

```solidity
require(anchoredProviderFactory.isProvider(priceProvider), NotRecognizedProvider());
```

Alternatively, expose a configurable allow-list of recognized provider factories on `MetricOmmPoolFactory` and require that the provider is registered in at least one of them before a pool can be created.

---

### Proof of Concept

1. Deploy `MaliciousProvider`:
   - `token0()` → `WETH`
   - `token1()` → `USDC`
   - `getBidAndAskPrice()` → returns `(1, type(uint128).max - 1)` (or any attacker-chosen spread)

2. Call `MetricOmmPoolFactory.createPool(params)` with `params.priceProvider = address(MaliciousProvider)`.

3. `_validatePriceProvider` passes: `token0()` and `token1()` match.

4. Pool is registered: `isPool(pool) == true`, `poolToIdx[pool] != 0`.

5. Any user who swaps in this pool has their trade executed at the malicious bid/ask. The attacker (as LP or counterparty) captures the difference. [1](#0-0) [2](#0-1) [6](#0-5)
