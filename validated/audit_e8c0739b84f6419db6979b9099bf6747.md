### Title
Pusher revocation can be bypassed by creator replaying the original consent signature — (File: smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol)

### Summary

`allowPushers` uses a deadline to prevent replay of a pusher's signed consent after the pusher has self-revoked. However, the deadline is only checked to be in the future — it is never consumed, stored, or invalidated. A creator who holds a still-valid (non-expired) signed consent can call `allowPushers` repeatedly with the same signature to silently re-establish a delegation that the pusher already revoked via `revokePusher()`. The code's own comment acknowledges the exact invariant that is broken.

### Finding Description

The `allowPushers` function in `CompressedOracle.sol` accepts a `deadline` and a pusher's EIP-191 signature:

```solidity
function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
    _ensureDeadline(deadline);          // only checks block.timestamp <= deadline
    ...
    bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
        keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
    );
    require(pusher == ECDSA.recover(hash, signatures[i]));
    namespaceRemapping[pusher] = msg.sender;   // delegation written, no expiry stored
    ...
}
``` [1](#0-0) 

`_ensureDeadline` only asserts `block.timestamp <= deadline` and returns; it does not mark the signature as used: [2](#0-1) 

`revokePusher` clears the mapping but stores nothing that would prevent the creator from re-submitting the same signed consent: [3](#0-2) 

The code's own NatSpec comment states the invariant that is supposed to hold:

> "an undated signature could re-establish a delegation AFTER the pusher revoked it." [4](#0-3) 

The deadline is the stated mechanism to prevent this, but because it is never consumed, the creator can replay the identical `(deadline, pusher, sig)` tuple an unlimited number of times while `block.timestamp <= deadline`, undoing every `revokePusher` call the pusher makes.

This is the direct analog of the GNTDeposit bug: in GNTDeposit the timelock is set once and not reset by new deposits; here the delegation is set once and not invalidated by the pusher's revocation, because the signed consent carries no consumed-flag.

### Impact Explanation

A pusher who calls `revokePusher()` to stop their data from flowing into a creator's namespace cannot achieve that goal until the deadline they originally signed has expired. During that window:

- Every push the pusher makes (including pushes intended for their own namespace) is silently redirected into the creator's namespace via the re-established `namespaceRemapping`.
- The pusher cannot establish their own oracle feed; `feedIdOf(pusher, slot, pos)` will always read zero because all writes land under the creator's key.
- If the creator's namespace feeds a pool, the pusher's price data continues to reach that pool's swap path even after the pusher has explicitly revoked consent — satisfying the "oracle role check bypassed by an unprivileged path" criterion.

### Likelihood Explanation

The creator is a normal (non-admin) user. The only precondition is that the pusher signed a consent with a deadline that has not yet expired — a routine operational assumption (pushers typically sign long-lived consents so the creator does not need to re-onboard them frequently). No special privilege, no malicious token, no off-chain oracle failure is required.

### Recommendation

One of:

1. **Consume the signature**: store a `mapping(bytes32 => bool) usedConsents` keyed on `keccak256(sig)` and revert if already used.
2. **Store the deadline as the delegation expiry**: write `delegationExpiry[pusher] = deadline` alongside `namespaceRemapping[pusher] = creator`, and check it in the `fallback` push path.

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
