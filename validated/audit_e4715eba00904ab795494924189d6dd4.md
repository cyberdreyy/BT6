### Title
`AnchoredPriceProvider` Uses L1 Staleness Check That Unconditionally Rejects Valid Oracle Prices on L2 Chains, Bricking All Public Pool Swaps — (File: `smart-contracts-poc/contracts/AnchoredPriceProvider.sol`)

---

### Summary

`AnchoredPriceProvider._isStale` unconditionally rejects any oracle `refTime` that exceeds `block.timestamp` by even one second. On L2 chains (Arbitrum, Optimism, Base, Linea — all protocol deployment targets), the sequencer's `block.timestamp` routinely lags behind oracle publication timestamps. This causes every oracle read to return the stale sentinel `(0, type(uint128).max)`, making `getBidAndAskPrice()` revert with `FeedStalled` on every swap and permanently bricking all public pools that use `AnchoredPriceProvider` on L2 chains.

---

### Finding Description

`AnchoredPriceProvider` is described as "the one standard provider for public pools." Its `_isStale` function uses the L1 staleness check:

```solidity
function _isStale(uint256 refTime, uint256 nowTs, uint256 maxDelta) internal pure returns (bool) {
    if (refTime == 0) return true;
    if (refTime > nowTs) return true;   // ← rejects ANY future timestamp, even +1 second
    return (nowTs - refTime) > maxDelta;
}
``` [1](#0-0) 

The protocol already recognizes this L2 clock-skew problem. `PriceProviderL2` carries an explicit comment and a `FUTURE_TOLERANCE` immutable (up to 1 hour) that tolerates oracle timestamps slightly ahead of `block.timestamp`:

```solidity
/// @dev L2 sequencer timestamp can lag behind oracle publication time.
///      Allows refTime up to FUTURE_TOLERANCE seconds ahead of block.timestamp.
uint256 public immutable FUTURE_TOLERANCE;
``` [2](#0-1) 

And its `_isStale` handles the future case gracefully:

```solidity
if (refTime > nowTs) {
    return (refTime - nowTs) > futureTol;
}
``` [3](#0-2) 

Both `PriceProvider` and `ProtectedPriceProvider` have L2 variants (`PriceProviderL2`, `ProtectedPriceProviderL2`). `AnchoredPriceProvider` — the **primary** public-pool provider — has **no L2 variant** and no `FUTURE_TOLERANCE`. The directory confirms this gap:



The failing call chain during a swap on any L2:

1. Pool calls `getBidAndAskPrice()` during swap.
2. `_getBidAndAskPrice()` calls `_readLeg(baseFeedId)`.
3. `_readLeg` calls `_isStale(refTime, block.timestamp, MAX_REF_STALENESS)`.
4. Oracle's `refTime` (Pyth/CompressedOracle publication time) is `block.timestamp + N` seconds due to sequencer lag → `refTime > nowTs` → returns `true`.
5. `_readLeg` returns `ok = false`.
6. `_getBidAndAskPrice` returns `(0, type(uint128).max)`.
7. `getBidAndAskPrice()` hits `if (bid == 0 || ask == type(uint128).max) revert FeedStalled()`. [4](#0-3) [5](#0-4)

### Impact Explanation

Every swap on every public pool using `AnchoredPriceProvider` on L2 chains (Arbitrum, Optimism, Base, Linea) reverts with `FeedStalled` whenever the oracle's `refTime` exceeds `block.timestamp`. Because L2 sequencer timestamps routinely lag behind real-world time, this is not an edge case — it is the normal operating condition. The result is a permanently unusable pool: no swaps, no liquidity removal via swap-path, no fee accrual. This matches the allowed impact gate: **broken core pool functionality causing unusable swap/liquidity flows**.

---

### Likelihood Explanation

**High.** The protocol explicitly targets Arbitrum, Optimism, Base, and Linea. The protocol's own `PriceProviderL2` comment documents that "L2 sequencer timestamp can lag behind oracle publication time," confirming the designers know this happens. `AnchoredPriceProvider` is "the one standard provider for public pools," so every public pool on every L2 deployment is affected. The trigger requires no attacker — it fires on the first swap after any oracle update whose `publishTime` exceeds the sequencer's `block.timestamp`.

---

### Recommendation

Add a `FUTURE_TOLERANCE` immutable to `AnchoredPriceProvider` (set to `0` for L1 deployments, up to `1 hours` for L2) and update `_isStale` to mirror `PriceProviderL2`:

```diff
- function _isStale(uint256 refTime, uint256 nowTs, uint256 maxDelta) internal pure returns (bool) {
-     if (refTime == 0) return true;
-     if (refTime > nowTs) return true;
-     return (nowTs - refTime) > maxDelta;
- }
+ function _isStale(uint256 refTime, uint256 nowTs, uint256 maxDelta, uint256 futureTol) internal pure returns (bool) {
+     if (refTime == 0) return true;
+     if (refTime > nowTs) {
+         return (refTime - nowTs) > futureTol;
+     }
+     return (nowTs - refTime) > maxDelta;
+ }
```

Pass `FUTURE_TOLERANCE` from the constructor (validated `<= 1 hours`) and thread it through `_readLeg` → `_isStale`. For L1 (Ethereum), deploy with `FUTURE_TOLERANCE = 0` to preserve the existing strict behavior.
