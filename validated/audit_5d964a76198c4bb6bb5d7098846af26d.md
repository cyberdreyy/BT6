### Title
`allowContractPushers` consent checked only at registration, never re-verified at push time ŌĆö (`File: smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1.allowContractPushers` verifies a contract pusher's consent via a live `isPusher(creator)` staticcall **once at registration**, then permanently writes `namespaceRemapping[pusherContract] = creator`. The `fallback()` push path reads that mapping and writes price data into the creator's namespace **without ever re-checking `isPusher`**. If the pusher contract is later upgraded, compromised, or its `isPusher` logic changes, the stale delegation remains valid and the pusher can inject arbitrary prices into the creator's feed namespace, which flows directly to pools.

---

### Finding Description

`allowContractPushers` performs a one-time live consent check:

```solidity
// CompressedOracle.sol L217-233
function allowContractPushers(address[] calldata pushers) external {
    ...
    (bool ok, bytes memory res) = pusher.staticcall(
        abi.encodeWithSignature("isPusher(address)", msg.sender)
    );
    require(ok);
    bool allowed = abi.decode(res, (bool));
    require(allowed);

    namespaceRemapping[pusher] = msg.sender;   // permanent delegation
    emit PusherAuthorized(pusher, msg.sender);
}
```

The `fallback()` push path then resolves the namespace from the stored mapping with no re-verification:

```solidity
// CompressedOracle.sol L311-344
fallback() override external {
    address creator = namespaceRemapping[msg.sender];   // stale mapping, never re-checked
    if (creator == address(0)) creator = msg.sender;
    ...
    // writes price data into creator's namespace unconditionally
    _writeStorage(key, bytes32(bytes32(word & ~uint256(0xff))));
}
```

The NatSpec comment explicitly states the design intent: *"consent is proven by a LIVE `isPusher(creator)` staticcall"*. The word "LIVE" implies ongoing verification, but the implementation only checks once. This is the exact analog of the ERC-7484 pattern: a guard is applied at registration but absent at execution.

The EOA pusher path (`allowPushers`) uses a signed message with a **deadline**, so delegations naturally expire. The contract pusher path has **no deadline** and **no re-check**, making it a permanent, irrevocable-by-the-pusher-side delegation.

---

### Impact Explanation

A compromised or upgraded contract pusher (e.g., a proxy-based price aggregator service whose owner changes, or whose `isPusher` logic is altered) retains its `namespaceRemapping` entry. It can call the oracle's `fallback()` with crafted slot words containing:

- An arbitrary `U64x32` price value
- Arbitrary codebook spread indexes (`s0`, `s1`)
- A timestamp just above the stored value (passes the monotonicity check)

These values are written into the creator's feed namespace. Any pool registered against that feed's `feedId` will receive the corrupted mid-price and spread on the next swap, causing:

- **Bad-price execution**: bid/ask derived from a manipulated mid and spread
- **Swap conservation failure**: trader receives more than the oracle curve permits, or the pool receives less input than owed
- **Pool insolvency**: repeated bad-price swaps drain LP reserves below LP claims

---

### Likelihood Explanation

Contract pushers are naturally implemented as upgradeable proxy contracts (price aggregator services, keeper bots). A proxy upgrade, ownership transfer, or internal access-control change can silently alter `isPusher` without any on-chain event visible to the creator. The creator has no notification mechanism and must proactively call `removePushers` to revoke ŌĆö but only after discovering the compromise. The window between compromise and remediation is the attack window. Likelihood is **medium** (requires a contract pusher to be compromised or upgraded), combined with **high** impact yields an overall **High** severity.

---

### Recommendation

Re-verify `isPusher(creator)` inside the `fallback()` push path whenever `namespaceRemapping[msg.sender]` resolves to a non-zero creator. If the live check fails, fall back to the pusher's own namespace (or revert) rather than writing into the creator's namespace:

```solidity
fallback() override external {
    address creator = namespaceRemapping[msg.sender];
    if (creator != address(0)) {
        // Re-verify live consent on every push, matching the stated design intent
        (bool ok, bytes memory res) = msg.sender.staticcall(
            abi.encodeWithSignature("isPusher(address)", creator)
        );
        if (!ok || !abi.decode(res, (bool))) {
            creator = address(0); // fall back to own namespace
        }
    }
    if (creator == address(0)) creator = msg.sender;
    ...
}
```

Alternatively, add an expiry timestamp to contract-pusher delegations (mirroring the EOA `deadline` mechanism) so that stale delegations cannot persist indefinitely.

### Proof of Concept

1. Deploy a `MockPusherAllowed` contract (returns `true` for `isPusher`).
2. Creator calls `allowContractPushers([address(mockPusher)])` ŌĆö `namespaceRemapping[mockPusher] = creator` is set.
3. Upgrade `mockPusher` (or transfer ownership) so `isPusher(creator)` now returns `false`.
4. The creator's `namespaceRemapping` entry is **unchanged** ŌĆö no re-check occurs.
5. Call the oracle's `fallback()` from `mockPusher` with a crafted slot word: price = `U64x32.encode(extremeValue)`, spread indexes = `0xff` (sentinel), timestamp = `currentTimestampMs + 1`.
6. `_writeStorage` writes the corrupted slot into the creator's namespace.
7. Any pool registered for `feedIdOf(creator, slotIndex, positionIndex)` now reads the manipulated price on the next swap.

The existing test `testContractPusherRevokedPushesOwnNamespace` (line 175, `CompressedOracleContractPushers.t.sol`) only tests the case where the **creator** calls `removePushers` ŌĆö it does not test the case where the pusher contract's `isPusher` changes without the creator's knowledge, leaving the mapping intact.     /// @notice Contract-pusher variant: consent is proven by a LIVE `isPusher(creator)`
    ///         staticcall instead of a signature, so there is nothing to replay and no
    ///         deadline is needed.
    function allowContractPushers(address[] calldata pushers) external {
        uint256 l = pushers.length;
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            (bool ok, bytes memory res) = pusher.staticcall(abi.encodeWithSignature("isPusher(address)", msg.sender));
            require(ok);
            bool allowed = abi.decode(res, (bool));
            require(allowed);

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }

    fallback() override external {
        uint256 end;
        uint256 namespace;

        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0)) creator = msg.sender;

        assembly ("memory-safe") {
            end := calldatasize()
            namespace := shl(96, creator) // [creator:20][zeros:12]
        }

        // 4 * 6 + 7 + 1 = 32 bytes per slot
        if (end == 0 || end % 32 != 0) revert BadCalldataLength();

        for (uint256 ptr = 0; ptr < end; ptr += 32) {
            uint256 word;
            assembly ("memory-safe") {
                word := calldataload(ptr)
            }
            // casting to 'uint8' is safe we want LSB
            // forge-lint: disable-next-line(unsafe-typecast)
            uint8 slotId = uint8(word);
            TimeMs timestampMs = toTimeMs(word >> 8 & X56);
            timestampMs.revertIfAfterBlockTimeWithDrift(MAX_TIME_DRIFT);
            bytes32 key = bytes32(namespace | uint256(slotId));
            uint256 old = uint256(_loadStorage(key));
            TimeMs oldTimestampMs = toTimeMs(old >> 8 & X56);

            bool newer = timestampMs.isAfter(oldTimestampMs);
            if (!newer) continue;

            _writeStorage(key, bytes32(bytes32(word & ~uint256(0xff))));
        }

    function testContractPusherRevokedPushesOwnNamespace() public {
        MockPusherAllowed pusherContract = new MockPusherAllowed();

        vm.prank(creator);
        oracle.allowContractPushers(_pushers(address(pusherContract)));
        assertEq(oracle.namespaceRemapping(address(pusherContract)), creator);

        vm.prank(creator);
        oracle.removePushers(_pushers(address(pusherContract)));
        assertEq(oracle.namespaceRemapping(address(pusherContract)), address(0));

        // After revocation the wallet falls back to pushing into its OWN namespace (no revert).
        uint56 tsMs = uint56(block.timestamp * 1000);
        uint48 raw = _packRaw(800_000, 2, 2);
        vm.prank(address(pusherContract));
        (bool ok,) = address(oracle).call(_wordAt(0, 0, raw, tsMs));
        assertTrue(ok, "post-revoke self push should succeed");

        assertEq(
            oracle.getOracleData(oracle.feedIdOf(address(pusherContract), 0, 0)).price,
            U64x32.decode(uint32(raw >> 16)),
            "post-revoke push lands in own namespace"
        );
        assertEq(oracle.getOracleData(oracle.feedIdOf(creator, 0, 0)).price, 0, "creator namespace must stay empty");
    }
