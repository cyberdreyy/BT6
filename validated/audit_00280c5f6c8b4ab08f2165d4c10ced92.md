### Title
`allowContractPushers` checks `isPusher` only at registration, not at push time — stale delegation allows a de-authorized contract pusher to feed bad prices into pools - (File: smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol)

### Summary
`allowContractPushers` verifies consent via a live `isPusher(creator)` staticcall at delegation time and writes `namespaceRemapping[pusher] = creator`. The fallback push path reads `namespaceRemapping[msg.sender]` and routes the slot write into the creator's namespace without re-checking `isPusher`. If the contract pusher's internal consent state later changes (returns `false`), the mapping persists and the pusher retains full write access to the creator's price namespace, enabling bad prices to reach any pool that consumes those feeds.

### Finding Description
`allowContractPushers` (line 217–234) performs a one-time live consent check:

```solidity
(bool ok, bytes memory res) = pusher.staticcall(
    abi.encodeWithSignature("isPusher(address)", msg.sender));
require(ok);
bool allowed = abi.decode(res, (bool));
require(allowed);
namespaceRemapping[pusher] = msg.sender;   // persists indefinitely
```

The NatDoc comment explicitly states the design intent: *"consent is proven by a LIVE `isPusher(creator)` staticcall instead of a signature, so there is nothing to replay and no deadline is needed."* The word "LIVE" implies the check should be dynamic, but the implementation only checks once at registration.

The fallback push path (lines 311–345) resolves the namespace without re-verifying consent:

```solidity
fallback() override external {
    address creator = namespaceRemapping[msg.sender];   // no isPusher re-check
    if (creator == address(0)) creator = msg.sender;
    ...
    _writeStorage(key, bytes32(bytes32(word & ~uint256(0xff))));
}
```

This is the direct analog to `LlamaPolicy.hasRole` not checking `balanceOf(policyholder) == 0`: the stored delegation (`namespaceRemapping`) is treated as permanently valid even after the underlying consent condition (`isPusher`) has changed.

For EOA pushers, the deadline in `allowPushers` bounds the consent window — a revoked EOA pusher cannot re-establish delegation without a fresh signature. Contract pushers have no equivalent bound: once `namespaceRemapping[pusher] = creator` is written, it survives indefinitely unless the creator calls `removePushers` or the pusher calls `revokePusher`. A contract pusher that cannot or does not call `revokePusher` (e.g., lacks that function in its interface) and whose creator is unaware of the consent change retains write access to the creator's entire 256-slot × 4-position namespace.

### Impact Explanation
A de-authorized contract pusher can write arbitrary `(price, s0, s1, timestampMs)` tuples into the creator's namespace. Any `PriceProvider` or `AnchoredPriceProvider` bound to a `feedId` in that namespace will consume the corrupted value. `AnchoredPriceProvider._readLeg` applies staleness, `mid == 0`, `spreadBps >= ORACLE_BPS`, and `priceGuard` checks, but a crafted push that passes those guards (fresh timestamp, plausible mid, spread within `MAX_SPREAD_BPS`, within guard bounds) reaches `_computeBidAsk` and produces a bad bid/ask that the pool executes at swap time. Traders receive more than the oracle curve permits or the pool fails to receive owed input — direct loss of user or LP principal.

### Likelihood Explanation
The trigger is semi-trusted but realistic: any contract pusher whose internal allowlist, ownership, or upgrade changes after delegation. The creator must also fail to call `removePushers` promptly. Both conditions are plausible in production (e.g., a multi-sig contract pusher that rotates signers, or an upgradeable pusher contract). No attacker privilege is required beyond the ability to call the already-delegated contract pusher.

### Recommendation
Re-verify consent at push time for contract pushers. In the fallback, when `namespaceRemapping[msg.sender]` is non-zero and `msg.sender` has code, perform a gas-bounded staticcall to `isPusher(creator)` and revert (or fall back to own namespace) if it returns false:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator != address(0) && msg.sender.code.length > 0) {
    (bool ok, bytes memory res) = msg.sender.staticcall{gas: 30_000}(
        abi.encodeWithSignature("isPusher(address)", creator));
    if (!ok || !abi.decode(res, (bool))) creator = address(0);
}
if (creator == address(0)) creator = msg.sender;
```

Alternatively, add a deadline parameter to `allowContractPushers` (mirroring `allowPushers`) so delegations expire and must be renewed, bounding the window of stale consent.

### Proof of Concept
1. Deploy `MockPusherSelective(creator)` — `isPusher(creator)` returns `true`, `isPusher(other)` returns `false`.
2. Creator calls `oracle.allowContractPushers([address(pusherContract)])` → `namespaceRemapping[pusherContract] = creator`.
3. `MockPusherSelective` is upgraded or its internal state changes so `isPusher(creator)` now returns `false`.
4. `pusherContract` (or an attacker controlling it) calls the oracle fallback with a crafted slot word: `slotId=0`, `tsMs = block.timestamp * 1000`, `price = 1` (near-zero), `s0 = s1 = 0` (tightest spread).
5. The fallback reads `namespaceRemapping[pusherContract] == creator` (no re-check), writes the corrupted slot into `creator`'s namespace.
6. `oracle.getOracleData(oracle.feedIdOf(creator, 0, 0))` returns `price = U64x32.decode(1)` — a near-zero price.
7. `AnchoredPriceProvider._readLeg(feedId)` reads this value; if it passes the `priceGuard` (guard not set → `guardMin = 0, guardMax = type(uint128).max`), `_computeBidAsk` produces a near-zero bid/ask that the pool executes, draining LP assets. [1](#0-0) [2](#0-1) [3](#0-2)

    /// @notice Contract-pusher variant: consent is proven by a LIVE `isPusher(creator)`
    ///         staticcall instead of a signature, so there is nothing to replay and no
    ///         deadline is needed.
    function allowContractPushers(address[] calldata pushers) external {
        uint256 l = pushers.length;
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            (bool ok, bytes memory res) = pusher.staticcall(abi.encodeWithSignature("isPusher(address)", msg.sender));
            require(ok);
            bool allowed = abi.decode(res, (bool));
            require(allowed);

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }

    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
