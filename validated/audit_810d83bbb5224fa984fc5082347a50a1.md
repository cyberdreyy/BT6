### Title
Synthetic-ratio integer truncation to zero in `AnchoredPriceProvider._getBidAndAskPrice()` permanently DoS-es pool swaps for extreme-price pairs — (`smart-contracts-poc/contracts/AnchoredPriceProvider.sol`)

---

### Summary

`AnchoredPriceProvider` supports a two-feed "synthetic ratio" mode (e.g. BTC/USD ÷ ETH/USD = BTC/ETH). The cross-rate is computed as `Math.mulDiv(mid, ORACLE_DECIMALS, mid2)` where `ORACLE_DECIMALS = 1e8`. When the quote feed price (`mid2`) exceeds the base feed price (`mid`) by more than a factor of 1e8, Solidity integer division truncates the result to zero. The downstream `_computeBidAsk(0, …)` then computes `refBid = 0`, hits the `refBid == 0` guard, and returns the `(0, type(uint128).max)` sentinel. `getBidAndAskPrice()` reverts with `FeedStalled`, permanently blocking all swaps through the pool for as long as the price ratio holds.

---

### Finding Description

In `_getBidAndAskPrice()`, when `quoteFeedId != bytes32(0)`, the synthetic mid-price is computed as:

```solidity
mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);   // mid * 1e8 / mid2
```

Both legs pass `_readLeg`'s individual guards (non-zero, fresh, within price-guard bounds). However, no guard checks whether the *ratio* is representable. When `mid2 > mid * 1e8`, the `mulDiv` result is 0 (integer truncation). There is no `mid == 0` check after this line before `_computeBidAsk(mid, spreadBps)` is called.

Inside `_computeBidAsk`:

```solidity
uint256 refBid = _bandEdge(mid, BPS_BASE_U - half, Math.Rounding.Floor);
// _bandEdge(0, …) = Math.mulDiv(0, …) = 0
if (refBid == 0 || …) return (0, type(uint128).max);
```

`getBidAndAskPrice()` then reverts:

```solidity
if (bid == 0 || ask == type(uint128).max) revert FeedStalled();
```

Every swap through the pool calls `getBidAndAskPrice()`, so every swap reverts for the lifetime of the extreme price ratio.

**Concrete example — SHIB/BTC synthetic pair:**
- SHIB/USD feed: $0.00001 → `mid = 1_000` (8-decimal)
- BTC/USD feed: $100,000 → `mid2 = 10_000_000_000_000` (1e13)
- `mid * 1e8 = 1e11 ≪ 1e13 = mid2` → `Math.mulDiv(1_000, 1e8, 1e13) = 0`

The ratio condition `mid2 / mid > 1e8` is satisfied whenever the quote token is more than 1e8 times more expensive than the base token in 8-decimal USD terms. This is reachable for any micro-cap/BTC or micro-cap/ETH synthetic pair, and can also be reached by a previously-healthy pair if the base token crashes sufficiently.

---

### Impact Explanation

All swaps through the affected pool revert with `FeedStalled`. The pool's core swap functionality is permanently broken for the duration of the extreme price ratio. LPs cannot trade; the pool is economically dead even though both underlying oracle feeds are individually valid and fresh. This matches the allowed impact: *"Broken core pool functionality … unusable … swap … flows."*

No bad price executes (the system fails closed), but the pool is rendered completely non-functional for swaps.

---

### Likelihood Explanation

The `AnchoredPriceProvider` with `quoteFeedId` set is an explicitly documented production feature for synthetic cross-rate pairs. The condition `mid2 / mid > 1e8` is reachable for any pair where the quote token is more than 1e8 times more expensive than the base token in 8-decimal USD terms. This is a realistic market condition for micro-cap/BTC or micro-cap/ETH pairs, and can arise from ordinary market price movement on a pair that was initially healthy. No privileged action is required after pool deployment.

---

### Recommendation

Add a zero-check on the computed ratio immediately after the `mulDiv`:

```solidity
mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);
if (mid == 0) return (0, type(uint128).max);   // ratio underflows 1e-8 — fail closed
```

This makes the failure explicit and avoids propagating a silently-wrong zero into `_computeBidAsk`.

---

### Proof of Concept

1. Deploy `AnchoredPriceProvider` with `baseFeedId = SHIB/USD`, `quoteFeedId = BTC/USD`.
2. Oracle reports SHIB/USD `mid = 1_000` (= $0.00001, 8-dec), BTC/USD `mid2 = 1e13` (= $100,000, 8-dec). Both feeds are fresh and pass all per-leg guards.
3. `_getBidAndAskPrice()` computes `Math.mulDiv(1_000, 1e8, 1e13) = 1e11 / 1e13 = 0`.
4. `_computeBidAsk(0, spreadBps)` → `refBid = _bandEdge(0, …) = 0` → returns `(0, type(uint128).max)`.
5. `getBidAndAskPrice()` reverts with `FeedStalled`.
6. Every swap through the pool reverts. The pool is permanently DoS-ed for swaps while the price ratio holds. [1](#0-0) [2](#0-1) [3](#0-2)
