### Title
Delegation consent signature in `allowPushers` can be replayed within the deadline window to re-establish a revoked pusher, routing bad prices into pools — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1.allowPushers` verifies a pusher's EIP-191 consent signature but has no used-signature tracking. The same signature is accepted an unlimited number of times as long as `block.timestamp <= deadline`. A creator can replay a previously-used consent signature to re-establish a delegation that the pusher already revoked via `revokePusher()`, making the pusher's self-revocation permanently ineffective within the deadline window and allowing bad prices to continue flowing into the creator's namespace — and from there into pools.

---

### Finding Description

`allowPushers` in `CompressedOracleV1` verifies the pusher's EIP-191 consent signature over `(block.chainid, address(this), deadline, pusher, msg.sender)`: [1](#0-0) 

The only replay guard is the deadline check via `_ensureDeadline`: [2](#0-1) 

The code's own NatSpec acknowledges the replay concern explicitly — "an undated signature could re-establish a delegation AFTER the pusher revoked it" — but the deadline only prevents replay **after** it expires. Within the deadline window, the identical `(deadline, pusher, creator)` tuple produces the identical hash, and `ECDSA.recover` returns the same pusher address, so `namespaceRemapping[pusher] = msg.sender` is written again unconditionally. [3](#0-2) 

`revokePusher()` clears the mapping to `address(0)`: [4](#0-3) 

But there is no nonce, no `usedSignature[hash]` flag, and no state change that would invalidate the old signature. The creator can call `allowPushers` again with the exact same arguments and signature immediately after the pusher's `revokePusher()` call, restoring `namespaceRemapping[pusher] = creator`.

The `fallback()` push path resolves the effective namespace at call time: [5](#0-4) 

So any push the pusher makes after believing they have revoked will still land in the creator's namespace if the creator has replayed the delegation. The creator's namespace feeds `AnchoredPriceProvider` / `ProtectedPriceProvider` via `oracle.price(feedId, pool)`: [6](#0-5) 

which in turn drives `getBidAndAskPrice()` consumed by pools at swap time. [7](#0-6) 

---

### Impact Explanation

A pusher whose key is compromised (or who simply wants to stop serving a creator) calls `revokePusher()`. The creator replays the old consent signature to immediately re-establish the delegation. The compromised pusher key can then continue pushing arbitrary prices into the creator's namespace. Those prices pass through `AnchoredPriceProvider._readLeg` (staleness, guard, spread checks) and reach pool swaps as the live bid/ask. Traders execute against a bad oracle price; the pool receives less input than the curve permits (swap conservation failure) or LPs are left with under-collateralised positions.

---

### Likelihood Explanation

The creator holds the old consent signature as a matter of course — they submitted it in the original `allowPushers` call. Replaying it requires a single on-chain transaction with no additional knowledge or privilege. The window is as long as the original deadline (commonly days to weeks). The pusher has no on-chain mechanism to invalidate the signature short of waiting for the deadline to expire.

---

### Recommendation

Track consumed signatures with a `mapping(bytes32 => bool) public usedDelegationHash` and reject any hash that has already been recorded:

```solidity
mapping(bytes32 => bool) public usedDelegationHash;

// inside allowPushers, after computing `hash`:
require(!usedDelegationHash[hash], "signature already used");
usedDelegationHash[hash] = true;
```

Alternatively, include a per-pusher nonce in the signed payload so that each consent is single-use by construction, or use a monotonically-increasing `delegationNonce[pusher]` that the pusher increments to invalidate all prior signatures.

### Proof of Concept

```
Step 1 — Setup:
  deadline = block.timestamp + 30 days
  sig = pusher signs keccak256(abi.encode(chainid, oracle, deadline, pusher, creator))

Step 2 — Creator establishes delegation:
  vm.prank(creator);
  oracle.allowPushers(deadline, [pusher], [sig]);
  // namespaceRemapping[pusher] == creator  ✓

Step 3 — Pusher self-revokes:
  vm.prank(pusher);
  oracle.revokePusher();
  // namespaceRemapping[pusher] == address(0)  ✓

Step 4 — Creator replays the SAME signature (deadline still valid):
  vm.prank(creator);
  oracle.allowPushers(deadline, [pusher], [sig]);   // no revert — identical hash accepted again
  // namespaceRemapping[pusher] == creator  ← revocation undone

Step 5 — Compromised pusher key pushes a bad price:
  vm.prank(pusher);   // attacker controls this key
  oracle.call(wordAt(slotId, pos, badPrice, block.timestamp * 1000));
  // price lands in creator's namespace, not pusher's own

Step 6 — Pool reads the bad price at swap time:
  pool.swap(...);
  // AnchoredPriceProvider reads creator's namespace → bad mid → bad bid/ask → swap executes at wrong price

    /// @notice Delegates pusher wallets into the caller's namespace. The pusher's EIP-191
    ///         signature is REQUIRED — without it anyone could remap a foreign pusher
    ///         wallet into their own namespace and silently swallow its pushes. The
    ///         deadline is likewise required: the signed consent carries no timestamp of
    ///         its own, so an undated signature could re-establish a delegation AFTER the
    ///         pusher revoked it.
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }

    /// @notice Allows a pusher to self-revoke their delegation. After revocation the
    ///         wallet pushes into its OWN namespace again (the registrationless default).
    function revokePusher() external {
        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
        namespaceRemapping[msg.sender] = address(0);
        emit PusherRevoked(msg.sender, creator);
    }

        uint256 end;
        uint256 namespace;

        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0)) creator = msg.sender;

        // 4 * 6 + 7 + 1 = 32 bytes per slot

    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
