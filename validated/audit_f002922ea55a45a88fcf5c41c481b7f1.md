### Title
Revoked Pusher Can Replay Unexpired Consent Signature to Re-Establish Delegation and Push Bad Prices Into Pools — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol::allowPushers`)

---

### Summary

`CompressedOracleV1.allowPushers` is a permissionless public function that accepts an EIP-191 pusher-consent signature and writes `namespaceRemapping[pusher] = creator`. The consent signature commits to a deadline but carries **no nonce and no used-signature tracking**. After a creator calls `removePushers` to revoke a pusher, the original consent signature remains cryptographically valid until its deadline expires. Because `allowPushers` is permissionless, the revoked pusher (or any third party holding the signature) can replay the call to silently re-establish the delegation, allowing continued writes into the creator's namespace and feeding bad prices into every pool that reads from those feeds.

---

### Finding Description

`CompressedOracleV1` stores pusher delegation in a single mapping: [1](#0-0) 

`allowPushers` verifies an EIP-191 signature from the pusher and sets `namespaceRemapping[pusher] = creator`. The signed message commits to `(deadline, pusher, creator, oracle address, chainid)` but contains **no nonce** and no per-signature consumption record.

`removePushers` clears the mapping: [2](#0-1) 

Because `allowPushers` is described as "intentionally permissionless" (the `control_hint` in the project's own audit-target metadata), anyone — including the revoked pusher — can replay the original `allowPushers` call with the same signature bytes as long as `block.timestamp < deadline`. The project's own slot-structure documentation explicitly acknowledges this gap:

> *"the signed consent has no data timestamp, so an undated signature could re-establish a delegation after the pusher revoked it"* [3](#0-2) 

After re-establishing delegation, the pusher's `fallback` calls resolve the namespace through `namespaceRemapping[msg.sender]` and overwrite the creator's storage slots with arbitrary packed slot words (price `p`, spread indices `s0`/`s1`, timestamp). These writes are accepted as long as the timestamp is not in the future: [4](#0-3) 

The corrupted slot data is then decoded by `getOracleData` and consumed by `PriceProvider`/`AnchoredPriceProvider` during `MetricOmmPool.swap`, producing bad bid/ask quotes that reach live pool swaps. [5](#0-4) 

The audit-target metadata confirms this exact replay surface is the intended validation focus:

> *"Replay and cross-context-test pusher signatures across creators, deadlines, chain ids, and contract addresses and assert no delegated namespace can be claimed twice."* [6](#0-5) 

---

### Impact Explanation

A revoked pusher re-establishes delegation and pushes an arbitrary packed slot word into the creator's namespace. `getOracleData` decodes the corrupted `p`, `s0`, `s1` fields and returns a fabricated mid-price and spread to `PriceProvider`. The pool's `swap` path consumes this quote, causing traders to receive more output than the true oracle price permits (swap conservation failure) or to execute at an inverted/stale bid/ask. This is a direct loss of user principal and LP assets, matching the "bad-price execution" and "swap conservation failure" impact gates.

---

### Likelihood Explanation

The trigger is semi-privileged: the attacker must be a previously authorized pusher holding an unexpired consent signature. However, once authorized, a pusher routinely holds such a signature. The replay requires a single permissionless transaction with no additional setup. Creators who set long-lived deadlines (days to months, which is operationally normal) are exposed for the full remaining deadline window after revocation.

---

### Recommendation

Track consumed signatures to prevent replay. Add a `mapping(bytes32 => bool) private _usedConsents` keyed on `keccak256(abi.encode(deadline, pusher, creator))` (or a per-pusher nonce). In `allowPushers`, revert if the consent hash has already been used, and mark it used on first acceptance. This closes the replay window without changing the permissionless call model.

---

### Proof of Concept

```solidity
// 1. Creator authorizes pusher with a 30-day deadline
uint256 deadline = block.timestamp + 30 days;
bytes memory sig = _signConsent(PUSHER_KEY, deadline, pusher, creator);
address[] memory pushers = new address[](1); pushers[0] = pusher;
bytes[] memory sigs    = new bytes[](1);    sigs[0]    = sig;

vm.prank(creator);
oracle.allowPushers(deadline, pushers, sigs);
assertEq(oracle.namespaceRemapping(pusher), creator); // delegated

// 2. Creator revokes the pusher
vm.prank(creator);
oracle.removePushers(pushers);
assertEq(oracle.namespaceRemapping(pusher), address(0)); // revoked

// 3. Pusher replays the SAME signature — no nonce, no used-sig check
// allowPushers is permissionless; anyone can call it
oracle.allowPushers(deadline, pushers, sigs);           // succeeds
assertEq(oracle.namespaceRemapping(pusher), creator);   // delegation re-established

// 4. Pusher pushes a fabricated price into the creator's namespace
uint56 tsMs = uint56(block.timestamp * 1000);
uint48 badRaw = _packRaw(type(uint32).max, 0, 0);      // max price, zero spread
vm.prank(pusher);
(bool ok,) = address(oracle).call(_wordAt(0, 0, badRaw, tsMs));
assertTrue(ok);

// 5. Pool swap now reads the fabricated price
bytes32 feedId = oracle.feedIdOf(creator, 0, 0);
IOffchainOracle.OracleData memory data = oracle.getOracleData(feedId);
// data.price == U64x32.decode(type(uint32).max)  — bad price reaches the pool
```

/// @notice Registrationless compressed oracle: a feed's LOCATION is its identity.
///         There is no feed registry — the feedId packs (creator, chainid, slotIndex,
///         positionIndex) and every read decodes its coordinates straight from the id:
///
///           feedId = creator << 96 | block.chainid << 16 | slotIndex << 8 | positionIndex
///
///         A creator owns 256 slots × 4 positions in their own namespace and simply
///         pushes into them (directly, or through pushers delegated via `allowPushers`).
///         A never-pushed position reads as price 0 / timestamp 0, which every consumer
///         already rejects as stale — no seeding or creation step is needed.
contract CompressedOracleV1 is OracleBase, ICompressedOracleV1 {
    /// @notice Oracle family discriminator for off-chain introspection (matches the
    ///         pusher/console `kind` vocabulary).
    string public constant kind = "compressed";

    mapping(address => address) public namespaceRemapping;

    uint256 private constant MAX_PER_SLOT = 4;

    uint256 private constant X48 = 0xFFFFFFFFFFFF;
    uint256 private constant X56 = 0xFFFFFFFFFFFFFF;
    uint256 private constant X80 = 0xFFFFFFFFFFFFFFFFFFFF;

    constructor(address _owner, uint256 maxTimeDrift) OracleBase(_owner, maxTimeDrift) {
        // feedIds reserve 80 bits for the chain id (EIP-2294 caps real ids well below).
        require(block.chainid <= X80);
    }

    function getOracleData(bytes32 feedId) public view override returns (OracleData memory data) {
        (address creator, uint8 slotIndex, uint8 positionIndex) = _unpackFeedId(feedId);

        SlotLayout memory _layout = _loadSlotLayout(_oracleSlot(creator, slotIndex));
        CompressedOracleData memory compressed = _selectCompressedData(_layout, positionIndex);

        if (compressed.s1 == 0xff && compressed.s0 == 0xff) {
            data.spread1 = BPS_BASE;
            data.spread0 = BPS_BASE;
            return data;
        }

        data.price = U64x32.decode(compressed.p);
        data.spread0 = _decodeCodebookIndex(compressed.s0);
        data.spread1 = _decodeCodebookIndex(compressed.s1);
        data.timestampMs = _layout.timestampMs;
    }

    function _loadSlotLayout(bytes32 slotIndex) internal view returns (SlotLayout memory _layout) {
        uint256 slotValue;
        assembly ("memory-safe") {
            slotValue := sload(slotIndex)
        }

        _layout.timestampMs = toTimeMs(slotValue >> 8 & X56);

        _layout.oracle0 = _decodeCompressedOracleData(uint48((slotValue >> 208) & X48));
        _layout.oracle1 = _decodeCompressedOracleData(uint48((slotValue >> 160) & X48));
        _layout.oracle2 = _decodeCompressedOracleData(uint48((slotValue >> 112) & X48));
        _layout.oracle3 = _decodeCompressedOracleData(uint48((slotValue >> 64) & X48));
    }
