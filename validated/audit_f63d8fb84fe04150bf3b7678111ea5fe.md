### Title
Pusher Delegation Replay After Revocation Allows Creator to Permanently Override `revokePusher` — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`allowPushers` signs a message that includes `(chainid, oracle, deadline, pusher, creator)` but **no nonce**. A creator who holds a valid pusher signature can replay it an unlimited number of times before the deadline, immediately re-establishing a delegation that the pusher just revoked via `revokePusher()`. The pusher cannot stop their data from flowing into the creator's namespace until the deadline expires.

### Finding Description

`allowPushers` constructs the signed digest as:

```solidity
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [1](#0-0) 

There is no nonce, no per-pusher revocation counter, and no on-chain record that the signature was already consumed. `revokePusher` sets `namespaceRemapping[msg.sender] = address(0)`: [2](#0-1) 

But the creator can immediately call `allowPushers` again with the **identical** `(deadline, pusher, signature)` tuple, writing `namespaceRemapping[pusher] = creator` again. The revocation is silently undone.

The `fallback` push path resolves the effective namespace at call time:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [3](#0-2) 

So every push the pusher makes after a failed revocation is still written into the creator's namespace, not the pusher's own namespace.

### Impact Explanation

If the pusher, believing revocation succeeded, begins pushing data intended for their **own** namespace (different slot/position layout), those slot words are written into the creator's namespace instead. Because one 32-byte slot word encodes **four** price positions simultaneously:

```
[data0:6][data1:6][data2:6][data3:6][ts:7][slotId:1]
``` [4](#0-3) 

all four positions in the creator's slot are overwritten with the pusher's unrelated price data. Any `PriceProvider` or `AnchoredPriceProvider` reading the creator's `feedId` will receive a corrupted mid-price and spread, which propagates directly into pool `getQuote()` calls and swap execution. Simultaneously, the pusher's own feeds remain stale (zero timestamp), causing pools that depend on the pusher's namespace to reject prices as stale.

### Likelihood Explanation

- The creator already holds the pusher's signed bytes (they submitted them in the original `allowPushers` call).
- Replaying requires a single cheap transaction with no additional off-chain work.
- The window lasts until `block.timestamp > deadline`; deadlines are creator-chosen and can be months or years in the future.
- The pusher has no on-chain mechanism to shorten the window after signing.

### Recommendation

Add a per-pusher revocation nonce to the signed digest and increment it on every successful `revokePusher` / `removePushers` call:

```solidity
mapping(address => uint256) public pusherNonce;

// In allowPushers:
keccak256(abi.encode(block.chainid, address(this), deadline,
                     pusher, msg.sender, pusherNonce[pusher]))

// In revokePusher / removePushers:
pusherNonce[pusher]++;
namespaceRemapping[pusher] = address(0);
```

This invalidates all previously issued signatures the moment the pusher revokes, mirroring the ERC20 pattern of resetting state to zero before accepting a new value.

### Proof of Concept

```
1. Pusher P signs: keccak256(abi.encode(chainid, oracle, deadline=T+365days, P, A))
2. Creator A calls allowPushers(T+365days, [P], [sig]) → namespaceRemapping[P] = A
3. P calls revokePusher() → namespaceRemapping[P] = address(0)
4. Creator A calls allowPushers(T+365days, [P], [sig]) again (same args) → namespaceRemapping[P] = A
5. P pushes slot word for their own ETH/USD feed (slot 0, positions 0-3)
   → fallback() resolves creator = A, writes into A's slot 0
   → A's pool reads feedIdOf(A, 0, 0) and gets P's ETH/USD price in place of A's BTC/USD price
   → swap executes at wrong oracle price; trader or pool absorbs the loss
6. Repeat step 4 indefinitely until block.timestamp > T+365days
``` [5](#0-4) [2](#0-1)

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }

    function revokePusher() external {
        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
        namespaceRemapping[msg.sender] = address(0);
        emit PusherRevoked(msg.sender, creator);
    }
