### Title
Pusher Self-Revocation Can Be Silently Overridden by Creator Replaying the Original Consent Signature — (File: `smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowPushers` has no nonce or used-signature tracking. The only replay guard is a deadline expiry check. After a pusher calls `revokePusher()`, the creator can immediately call `allowPushers` again with the **same** signature and the **same** still-valid deadline to silently re-establish the delegation. The pusher's revocation is therefore ineffective for the entire remaining deadline window, and any pushes the pusher makes (believing they now write to their own namespace) are silently redirected into the creator's namespace and consumed by pools.

---

### Finding Description

`allowPushers` signs consent as:

```
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [1](#0-0) 

The only validation before writing `namespaceRemapping[pusher] = msg.sender` is `_ensureDeadline(deadline)`, which checks `block.timestamp <= deadline` and nothing else. [2](#0-1) 

There is no nonce, no used-signature bitmap, and no per-pusher revocation counter. The contract's own NatSpec comment acknowledges the risk:

> "The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it." [3](#0-2) 

The comment treats the deadline as the fix, but the deadline only prevents replay **after** it expires. Within the window (e.g., 1 day), the creator can call `allowPushers` with the identical `(deadline, pusher, signature)` tuple an unlimited number of times, each time overwriting the `address(0)` that `revokePusher()` wrote. [4](#0-3) 

---

### Impact Explanation

After the creator re-establishes the delegation, the pusher's `fallback` pushes are resolved through:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [5](#0-4) 

The pusher, believing they are writing to their own namespace (e.g., slot 0 = their own ETH/USD feed), is actually writing into the creator's namespace (slot 0 = creator's BTC/USD feed). The corrupted slot is then decoded by `getOracleData` → `price` and consumed by any pool that uses the creator's `feedId`. A pool swap reading a price from a slot that now contains a mismatched or stale value will execute at a bad bid/ask, causing direct loss to traders or LPs. [6](#0-5) 

---

### Likelihood Explanation

The trigger requires the creator to be adversarial (semi-trusted) and the pusher to have signed a consent with a deadline that has not yet expired. Both conditions are realistic in production: deadlines of hours or days are normal, and a creator who wants to keep a pusher's data flowing (even against the pusher's will) has a clear financial incentive. No privileged role beyond being the creator is needed; `allowPushers` is a public function.

---

### Recommendation

Track consumed consent signatures with a per-pusher revocation nonce or a `mapping(bytes32 => bool) usedConsents` keyed on the full consent hash. On `revokePusher()` or `removePushers()`, increment the nonce or mark the hash used so that any previously issued signature is permanently invalidated regardless of its deadline.

---

### Proof of Concept

```solidity
// 1. Pusher signs consent with deadline = now + 1 day
bytes memory sig = pusher.sign(
    keccak256(abi.encode(chainid, oracle, deadline, pusherAddr, creatorAddr))
);

// 2. Creator establishes delegation
oracle.allowPushers(deadline, [pusherAddr], [sig]);
// namespaceRemapping[pusher] == creator  ✓

// 3. Pusher self-revokes
vm.prank(pusherAddr);
oracle.revokePusher();
// namespaceRemapping[pusher] == address(0)  ✓

// 4. Creator immediately replays the SAME signature (deadline still valid)
vm.prank(creatorAddr);
oracle.allowPushers(deadline, [pusherAddr], [sig]);
// namespaceRemapping[pusher] == creator  ← revocation bypassed

// 5. Pusher pushes what they believe is their own ETH/USD slot 0
vm.prank(pusherAddr);
address(oracle).call(wordForSlot0WithBadPrice);
// Data lands in creator's slot 0 (e.g., BTC/USD), corrupting pool prices
``` [7](#0-6)

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
