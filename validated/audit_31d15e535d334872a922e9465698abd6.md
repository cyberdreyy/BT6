### Title
Pusher consent signature is replayable within the deadline window, making `revokePusher()` ineffective and enabling forced namespace hijack — (`File: smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowPushers` in `CompressedOracle.sol` contains no nonce or used-signature tracking. A creator who holds a pusher's consent signature can replay it an unlimited number of times within the deadline window. This makes `revokePusher()` completely ineffective: the creator can re-establish the delegation in the same block the pusher revokes it, permanently redirecting the pusher's price writes into the creator's namespace until the deadline expires.

---

### Finding Description

The `allowPushers` function signs over `(block.chainid, address(this), deadline, pusher, msg.sender)` and checks only that the deadline has not passed: [1](#0-0) 

The code's own NatSpec comment acknowledges the design intent:

> *"The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it."* [2](#0-1) 

The comment implies the deadline prevents re-establishment after revocation. It does not. The deadline only blocks replay **after expiry**. Within the window, the identical `(deadline, pusher, creator)` signature passes `ECDSA.recover` every time, and `namespaceRemapping[pusher] = msg.sender` is written unconditionally with no record that the signature was already consumed.

`revokePusher()` clears the mapping but does not invalidate the signature: [3](#0-2) 

The creator can call `allowPushers` with the same bytes in the next transaction, restoring `namespaceRemapping[pusher] = creator`. The pusher's revocation is silently undone.

The fallback push path resolves the namespace at call time: [4](#0-3) 

So every subsequent push by the pusher lands in the creator's namespace, not the pusher's own, for the entire remaining deadline window.

---

### Impact Explanation

**Direct broken invariant**: A pusher who calls `revokePusher()` cannot stop their price writes from landing in the creator's namespace. The creator can re-establish the delegation atomically (same block, next transaction) using the already-broadcast signature.

**Price-feed consequence**: The pusher's only remaining option to stop feeding the creator's namespace is to cease pushing entirely. If they do:

- The creator's namespace receives no further updates.
- Any `CompressedPriceProvider` or `ProtectedPriceProvider` backed by that feed will see a timestamp that no longer advances.
- The staleness check in `_computeBidAsk` returns `(0, type(uint128).max)`, halting all swaps on every pool that reads from that feed. [5](#0-4) 

This satisfies the "broken core pool functionality causing loss of funds or unusable withdraw/swap/liquidity flows" impact gate: LPs cannot exit at fair value through swaps, and the pool is effectively frozen until the deadline expires and the pusher can safely redirect their writes.

---

### Likelihood Explanation

- The trigger is **unprivileged**: any creator who holds the pusher's consent bytes (which were broadcast on-chain in the original `allowPushers` call) can replay them.
- The signature is **permanently public** once the first `allowPushers` transaction is mined; no off-chain secret is needed.
- The creator has a direct economic incentive to keep the pusher feeding their namespace (their pool depends on it).
- Deadlines in practice are set days to weeks in the future (the test suite uses `block.timestamp + 1 days`), giving a large replay window. [6](#0-5) 

---

### Recommendation

Track consumed signatures with a `mapping(bytes32 => bool) private _usedConsentSigs` and mark each signature hash as used on first acceptance:

```solidity
bytes32 sigHash = keccak256(signatures[i]);
require(!_usedConsentSigs[sigHash], "signature already used");
_usedConsentSigs[sigHash] = true;
```

Alternatively, include a per-pusher nonce in the signed payload so that each consent is single-use by construction, analogous to how EIP-2612 permits work.

---

### Proof of Concept

```solidity
// 1. Pusher signs consent (deadline = now + 1 day)
uint256 deadline = block.timestamp + 1 days;
bytes memory sig = _signConsent(PUSHER_KEY, deadline, pusher, creator);

// 2. Creator establishes delegation — first use of sig
address[] memory pushers = new address[](1);
pushers[0] = pusher;
bytes[] memory sigs = new bytes[](1);
sigs[0] = sig;
vm.prank(creator);
oracle.allowPushers(deadline, pushers, sigs);
assertEq(oracle.namespaceRemapping(pusher), creator); // delegated

// 3. Pusher revokes
vm.prank(pusher);
oracle.revokePusher();
assertEq(oracle.namespaceRemapping(pusher), address(0)); // cleared

// 4. Creator replays the IDENTICAL sig — no nonce, no used-sig check
vm.prank(creator);
oracle.allowPushers(deadline, pushers, sigs); // succeeds — same sig accepted again
assertEq(oracle.namespaceRemapping(pusher), creator); // delegation restored!

// 5. Pusher's subsequent fallback push still lands in creator's namespace
uint56 tsMs = uint56(block.timestamp * 1000);
vm.prank(pusher);
(bool ok,) = address(oracle).call(_wordAt(0, 0, _packRaw(999_999, 3, 3), tsMs));
assertTrue(ok);
// creator namespace updated — pusher's own namespace stays empty
assertGt(oracle.getOracleData(oracle.feedIdOf(creator, 0, 0)).price, 0);
assertEq(oracle.getOracleData(oracle.feedIdOf(pusher,  0, 0)).price, 0);
```

The pusher's only escape is to stop pushing entirely, which starves the creator's feed and freezes any pool backed by it until `deadline` expires.

```text
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
```
