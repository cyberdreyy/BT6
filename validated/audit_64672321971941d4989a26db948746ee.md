### Title
Shared per-slot timestamp in `CompressedOracleV1` allows stale per-position prices to bypass provider staleness checks, enabling bad-price execution in pools - (File: smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol)

### Summary
`CompressedOracleV1` packs four feed positions into one 256-bit storage slot with a **single shared 56-bit timestamp**. Every push overwrites the entire slot word. When a creator/pusher updates one position and carries forward old price data for the other three, the slot timestamp is refreshed for all four positions simultaneously. Price providers (`PriceProvider`, `PriceProviderL2`) use this slot-level timestamp as `refTime` for their staleness check. Sibling positions whose price data has not changed therefore appear fresh, and their stale bid/ask quotes reach pool swaps.

### Finding Description

The slot layout is:

```
bits 255-208 : oracle[0] (48 bits)
bits 207-160 : oracle[1] (48 bits)
bits 159-112 : oracle[2] (48 bits)
bits 111-64  : oracle[3] (48 bits)
bits  63-8   : timestamp (uint56, unix ms) ← ONE timestamp for ALL four positions
bits   7-0   : slotId (cleared before storing)
``` [1](#0-0) 

`_loadSlotLayout` reads the single slot word and assigns `_layout.timestampMs` from bits 63-8 — the same value for every position in the slot.

`getOracleData` then propagates this slot-level timestamp to every per-position read:

```solidity
data.timestampMs = _layout.timestampMs;
``` [2](#0-1) 

`_price` converts it to seconds and returns it as `refTime`:

```solidity
return (uint256(data.price), uint256(data.spread0), data.spread1, data.timestampMs.toSeconds());
``` [3](#0-2) 

`PriceProvider._getBidAndAskPrice` uses this `refTime` as the sole freshness gate:

```solidity
if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA)) {
    return (0, type(uint128).max);
}
``` [4](#0-3) 

The `fallback` push path overwrites the **entire** slot word on every accepted update:

```solidity
_writeStorage(key, bytes32(bytes32(word & ~uint256(0xff))));
``` [5](#0-4) 

The documentation itself acknowledges the whole-slot overwrite requirement:

> "The entire slot is overwritten on update: if you update a single lane, you must still supply correct values for the other lanes." [6](#0-5) 

**Broken invariant:** The staleness check is supposed to reflect the freshness of the *price data for the specific position* being read. Instead it reflects the freshness of the *last write to the slot*, which may have been for a completely different position. A creator who updates position 0 (ETH/USD) at T=2000 must include the old price bytes for positions 1-3 in the new slot word. The timestamp advances to 2000 for all four positions, but positions 1-3 still carry prices from T=1000. Any price provider backed by positions 1-3 passes the staleness check and delivers the stale price to the pool.

The test suite itself documents this shared-timestamp behavior:

```
// A sibling lane in the SAME slot shares the slot's 56-bit timestamp word, but its own
// 48-bit price lane was never written → price 0 (the timestamp is per-slot, not per-lane).
``` [7](#0-6) 

### Impact Explanation

A pool whose `PriceProvider` is backed by a `CompressedOracleV1` feed can execute swaps at a stale price. The staleness guard — the only time-based freshness protection in the provider — is bypassed because the slot timestamp is refreshed by an unrelated position update. In volatile markets the stale price can deviate materially from the true market price, causing the pool to give traders more output than the oracle permits (swap conservation failure) or to accept less input than owed, directly draining LP assets.

### Likelihood Explanation

This is a normal operational scenario, not a malicious one. Any creator who places feeds with different update frequencies in the same slot (e.g., ETH/USD updated every second alongside BTC/USD updated every minute) will routinely push slot words that carry fresh data for one position and old data for the others. No special privilege or malicious intent is required; the vulnerability fires on every such routine update.

### Recommendation

Replace the single per-slot timestamp with per-position timestamps. Each 48-bit lane has 16 bits currently used for two 8-bit codebook indices; alternatively a separate `mapping(bytes32 => uint56) positionTimestamp` keyed by `(creator, slotIndex, positionIndex)` can be maintained alongside the slot word. `getOracleData` should return the per-position timestamp rather than the slot-level one, so that the staleness check in price providers reflects the actual age of the specific price being consumed.

### Proof of Concept

```
T = 1000 ms
Creator pushes slot 0 with all four positions (ETH/USD, BTC/USD, SOL/USD, AVAX/USD).
  → stored slot word: [ETH_price][BTC_price][SOL_price][AVAX_price][ts=1000][0]
  → getOracleData(BTC feedId).timestampMs = 1000  ✓ fresh

T = 2000 ms
Creator pushes slot 0 again, updating only position 0 (ETH/USD) with a new price.
The slot word must include the OLD bytes for positions 1-3:
  → new slot word: [ETH_price_new][BTC_price_OLD][SOL_price_OLD][AVAX_price_OLD][ts=2000][0]
  → getOracleData(BTC feedId).timestampMs = 2000  ← refreshed by ETH update

T = 2000 + MAX_TIME_DELTA - 1
Pool calls PriceProvider.getBidAndAskPrice() for BTC/USD.
  _isStale(refTime=2000, block.timestamp=2000+MAX_TIME_DELTA-1, MAX_TIME_DELTA) → false
  mid = BTC_price_OLD  (from T=1000, potentially far from current market)
  Staleness check passes; stale BTC price reaches pool swap.
``` [8](#0-7) [9](#0-8)

    /// @notice Push path. Calldata = N × 32-byte slot words:
    ///         [data0:6][data1:6][data2:6][data3:6][ts:7][slotId:1]
    ///         The sender pushes into `namespaceRemapping[msg.sender]`, falling back to
    ///         its OWN namespace — a creator needs zero setup transactions to start
    ///         pushing. Each word carries its own timestamp (monotonicity is the only
    ///         freshness gate), so there is no deadline prefix.
    fallback() override external {
        uint256 end;
        uint256 namespace;

        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0)) creator = msg.sender;

        assembly ("memory-safe") {
            end := calldatasize()
            namespace := shl(96, creator) // [creator:20][zeros:12]
        }

        // 4 * 6 + 7 + 1 = 32 bytes per slot
        if (end == 0 || end % 32 != 0) revert BadCalldataLength();

        for (uint256 ptr = 0; ptr < end; ptr += 32) {
            uint256 word;
            assembly ("memory-safe") {
                word := calldataload(ptr)
            }
            // casting to 'uint8' is safe we want LSB
            // forge-lint: disable-next-line(unsafe-typecast)
            uint8 slotId = uint8(word);
            TimeMs timestampMs = toTimeMs(word >> 8 & X56);
            timestampMs.revertIfAfterBlockTimeWithDrift(MAX_TIME_DRIFT);
            bytes32 key = bytes32(namespace | uint256(slotId));
            uint256 old = uint256(_loadStorage(key));
            TimeMs oldTimestampMs = toTimeMs(old >> 8 & X56);

            bool newer = timestampMs.isAfter(oldTimestampMs);
            if (!newer) continue;

            _writeStorage(key, bytes32(bytes32(word & ~uint256(0xff))));
        }
