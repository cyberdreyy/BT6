### Title
`revokePusher` Self-Protection Bypassed via Intra-Deadline Signature Replay in `allowPushers` — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary
A pusher who calls `revokePusher()` to stop their delegation can have it silently re-established by the creator calling `allowPushers()` again with the **same EIP-191 signature**, as long as the deadline has not yet expired. The code comment explicitly acknowledges this risk and claims the deadline is the mitigation, but the deadline only prevents re-establishment *after* it expires — not *within* the deadline window. This is the direct analog of the Pumponomics bypass: a protection mechanism (revocation) is applied to a subset of the attack surface (post-deadline), while the actual operation (re-delegation) can proceed using a pre-existing value (the original signature) that was never invalidated.

### Finding Description

The `allowPushers` function signs over `(chainid, address(this), deadline, pusher, creator)`: [1](#0-0) 

There is no nonce, no revocation flag, and no check for whether `namespaceRemapping[pusher]` was previously cleared by `revokePusher`. The function unconditionally overwrites the mapping: [2](#0-1) 

When a pusher calls `revokePusher()`, the mapping is cleared to `address(0)`: [3](#0-2) 

But the creator, who already holds the pusher's signed consent from the original `allowPushers` call, can immediately re-call `allowPushers()` with the **identical signature** to restore the delegation. The code comment on `allowPushers` explicitly acknowledges this exact risk: [4](#0-3) 

The comment states the deadline prevents re-establishment "after the pusher revoked it," but the implementation enforces no such constraint within the deadline window.

### Impact Explanation

Once the compromised pusher key is re-delegated, its `fallback()` pushes land in the creator's namespace: [5](#0-4) 

Those prices flow through `AnchoredPriceProvider._readLeg()` → `_computeBidAsk()` → pool `swap()`: [6](#0-5) 

The band clamp in `_computeBidAsk` is derived from the same oracle `mid` value — if `mid` is attacker-controlled, the entire band is attacker-controlled. Traders executing swaps against the pool receive quotes at the attacker-set price, causing direct loss of principal.

### Likelihood Explanation

Medium. The creator always holds the original signature (they submitted it in the first `allowPushers` call). Short-lived delegations (hours to days) are the common operational pattern, making the intra-deadline window realistic. The creator may act unintentionally (e.g., an automated re-delegation script) or maliciously. The pusher's emergency revocation — the only self-protection path — is silently nullified.

### Recommendation

Track consumed signatures with a per-pusher nonce or a `usedSignatures` mapping, and reject any `allowPushers` call for a pusher whose `namespaceRemapping` was cleared by `revokePusher()` within the same deadline window. Alternatively, bind the signature to a monotonic per-pusher nonce that `revokePusher()` increments, invalidating all prior consents:

```solidity
mapping(address => uint256) public pusherNonce;

// In revokePusher():
pusherNonce[msg.sender]++;

// Signature covers: (chainid, address(this), deadline, pusher, creator, pusherNonce[pusher])
```

### Proof of Concept

```solidity
function testRevokePusherBypassedByCreatorReplay() public {
    uint256 deadline = block.timestamp + 1 days;

    // 1. Pusher signs consent for creator
    bytes memory sig = _signConsent(PUSHER_KEY, deadline, pusher, creator);

    // 2. Creator delegates pusher
    vm.prank(creator);
    oracle.allowPushers(deadline, _arr(pusher), _arr(sig));
    assertEq(oracle.namespaceRemapping(pusher), creator, "delegated");

    // 3. Pusher self-revokes (key compromise detected)
    vm.prank(pusher);
    oracle.revokePusher();
    assertEq(oracle.namespaceRemapping(pusher), address(0), "revoked");

    // 4. Creator re-establishes with the SAME signature — no revert
    vm.prank(creator);
    oracle.allowPushers(deadline, _arr(pusher), _arr(sig));
    assertEq(oracle.namespaceRemapping(pusher), creator, "re-delegated — bypass confirmed");

    // 5. Compromised key pushes attacker-controlled price into creator namespace
    uint56 tsMs = uint56(block.timestamp * 1000);
    uint48 badPrice = _packRaw(9_999_999, 0, 0); // extreme manipulated price
    vm.prank(pusher); // compromised key
    (bool ok,) = address(oracle).call(_wordAt(slotId, 0, badPrice, tsMs));
    assertTrue(ok, "bad price accepted in creator namespace");

    // Creator namespace now holds attacker price — AnchoredPriceProvider will quote it
    IOffchainOracle.OracleData memory data = oracle.getOracleData(
        oracle.feedIdOf(creator, slotId, 0)
    );
    assertEq(data.price, U64x32.decode(uint32(badPrice >> 16)), "attacker price in creator ns");
}
```

    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
