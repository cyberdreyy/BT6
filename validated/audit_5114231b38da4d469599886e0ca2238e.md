### Title
Shared Slot Timestamp in `CompressedOracleV1` Allows Stale Per-Position Prices to Appear Fresh, Enabling Bad-Price Execution in Pools — (File: `smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1` packs four independent price positions into one 32-byte storage slot with a **single shared `timestampMs`**. When a pusher updates any position in a slot, the shared timestamp is refreshed for **all four positions simultaneously**. A position whose underlying price has not been re-observed (stale) will appear fresh to downstream consumers such as `AnchoredPriceProvider`, bypassing the `MAX_REF_STALENESS` staleness guard and allowing a stale bid/ask quote to reach a live pool swap.

---

### Finding Description

The slot word layout is:

```
[data0:6][data1:6][data2:6][data3:6][ts:7][slotId:1]  (32 bytes total)
```

`_loadSlotLayout` extracts **one** timestamp for the entire slot and assigns it to all four positions: [1](#0-0) 

`getOracleData` then returns that shared slot timestamp as the freshness indicator for whichever individual position is queried: [2](#0-1) 

The `fallback()` push path enforces monotonicity **per slot**, not per position: [3](#0-2) 

Because the protocol has **no per-position update path** — a pusher must always write all four positions in one 32-byte word — a pusher who has fresh data for position 0 but stale data for positions 1–3 is forced to either:

1. Push the slot with the new timestamp and the **old price bytes** for positions 1–3 (making stale positions appear fresh), or  
2. Withhold the push entirely (leaving all four positions stale).

In the first case, `getOracleData(feedId_pos1)` returns `timestampMs = T_new` (fresh) while `data.price` reflects the price last observed at `T_old`.

The `price()` view propagates this shared timestamp directly as `refTime`: [4](#0-3) 

`AnchoredPriceProvider._readLeg()` then performs its staleness check against this `refTime`: [5](#0-4) 

Because `refTime = T_new` (fresh), the check passes and the stale price for position 1 is used to compute the pool's bid/ask, which is then consumed by `getBidAndAskPrice()`: [6](#0-5) 

---

### Impact Explanation

A stale price for any of the three non-updated positions in a slot bypasses `MAX_REF_STALENESS` and reaches the pool's swap path. Depending on how much the underlying asset has moved since the last genuine observation, the pool will execute swaps at an incorrect mid price, widening or inverting the true market spread. LPs suffer impermanent loss on the mispriced side; traders who exploit the stale quote extract value directly from the pool's reserves. This matches the **bad-price execution** and **pool insolvency** impact criteria.

---

### Likelihood Explanation

This is not a malicious-actor scenario. It arises naturally whenever a pusher's data source for one feed in a slot experiences a temporary outage (network partition, API rate-limit, off-hours market closure) while the other feeds in the same slot remain live. The pusher continues pushing fresh data for the live feeds, inadvertently refreshing the shared timestamp for the stale feed. No special privilege or setup is required beyond being a delegated pusher — a role that is explicitly designed to be granted to automated bots.

---

### Recommendation

Store a **per-position timestamp** rather than a single slot-level timestamp. One approach is to reserve 14 bits (two 7-byte fields) per position inside the 48-bit position word, reducing price precision slightly but giving each position an independent freshness anchor. Alternatively, add a dedicated per-position update calldata path (e.g., a 16-byte word: `[data:6][ts:7][posId:1][slotId:1][pad:1]`) that writes only the targeted position's bytes and its own timestamp into the packed slot without touching the other positions' timestamps.

---

### Proof of Concept

1. Creator deploys `CompressedOracleV1` and pushes slot 0 at time `T1` with four valid prices for positions 0–3. Slot timestamp = `T1`.

2. At time `T2 = T1 + MAX_REF_STALENESS + 1`, the pusher's data source for position 1 (e.g., BTC/USD) goes offline. The pusher still has fresh data for positions 0, 2, 3.

3. The pusher constructs a 32-byte slot word with:
   - Position 0 bytes: fresh price at `T2`
   - Position 1 bytes: **old price bytes copied from the on-chain slot** (price from `T1`)
   - Positions 2–3 bytes: fresh prices at `T2`
   - Timestamp field: `T2`

4. The pusher calls `fallback()` with this word. The monotonicity check passes (`T2 > T1`). The slot is written with timestamp `T2`.

5. A pool backed by `AnchoredPriceProvider` (oracle = `CompressedOracleV1`, `baseFeedId` = position 1's feedId) calls `getBidAndAskPrice()`.

6. `_readLeg` calls `price(feedId_pos1, pool)` → `getOracleData` returns `timestampMs = T2`, `price = stale_BTC_price`.

7. `_isStale(T2.toSeconds(), block.timestamp, MAX_REF_STALENESS)` → **false** (T2 is fresh). The stale BTC price passes all guards and is used to compute the pool's bid/ask.

8. Swappers trade against the stale quote; LPs absorb the resulting mispricing loss. [1](#0-0) [2](#0-1) [7](#0-6)

        data.spread1 = _decodeCodebookIndex(compressed.s1);

    function _loadSlotLayout(bytes32 slotIndex) internal view returns (SlotLayout memory _layout) {
        uint256 slotValue;
        assembly ("memory-safe") {
            slotValue := sload(slotIndex)
        }

        _layout.timestampMs = toTimeMs(slotValue >> 8 & X56);

        _layout.oracle0 = _decodeCompressedOracleData(uint48((slotValue >> 208) & X48));
        _layout.oracle1 = _decodeCompressedOracleData(uint48((slotValue >> 160) & X48));
        _layout.oracle2 = _decodeCompressedOracleData(uint48((slotValue >> 112) & X48));
        _layout.oracle3 = _decodeCompressedOracleData(uint48((slotValue >> 64) & X48));
    }

    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
