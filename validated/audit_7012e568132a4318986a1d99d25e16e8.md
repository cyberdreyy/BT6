### Title
Pusher consent signature has no nonce, allowing creator to silently re-establish delegation after pusher self-revokes — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowPushers` in `CompressedOracleV1` verifies a pusher's EIP-191 consent signature but tracks no nonce and no used-signature state. After a pusher calls `revokePusher()` to clear their delegation, the creator can call `allowPushers` again with the **identical** signature (any time before the deadline expires) to silently re-establish the delegation — overriding the pusher's explicit revocation without any new consent from the pusher.

---

### Finding Description

`allowPushers` signs over `(block.chainid, address(this), deadline, pusher, msg.sender)`: [1](#0-0) 

There is no nonce, no per-pusher revocation counter, and no `mapping(bytes32 => bool) usedSignatures` guard. The only replay barrier is the deadline: [2](#0-1) 

The code comment itself acknowledges the deadline is the intended replay guard: [3](#0-2) 

But the deadline only prevents replay **after** it expires. Within the deadline window, the same signature is accepted an unlimited number of times. The revocation path clears `namespaceRemapping[pusher]` to `address(0)`: [4](#0-3) 

Nothing prevents the creator from immediately calling `allowPushers` again with the same signature to write `namespaceRemapping[pusher] = msg.sender` back, undoing the revocation silently.

The fallback push path resolves the namespace at call time: [5](#0-4) 

So any push the pusher makes after the creator replays the signature lands in the creator's namespace, not the pusher's own.

---

### Impact Explanation

The CompressedOracle is the open price-ingestion layer for pools. A creator's namespace feeds directly into `price(feedId, pool)` reads consumed by `AnchoredPriceProvider.getBidAndAskPrice()` during live swaps: [6](#0-5) 

If a pusher is an automated price-pushing service that calls `revokePusher()` because it detected a compromise or wants to stop, and the creator replays the original consent signature to re-establish delegation, the pusher's subsequent pushes (including potentially bad/stale/manipulated prices) continue landing in the creator's namespace. Those prices flow through `getOracleData` → `_price` → `AnchoredPriceProvider._readLeg` → pool swap, producing bad-price execution against live LP positions.

The pusher has no on-chain visibility that their revocation was undone — `namespaceRemapping[pusher]` is public but the pusher has no event or callback to detect the re-establishment.

---

### Likelihood Explanation

- **Trigger is semi-privileged but not trusted**: the creator is the namespace owner, not a protocol-trusted role. The pusher is a separate party whose explicit revocation the creator overrides.
- **Preconditions**: creator already holds the pusher's signature (obtained during the original `allowPushers` call); the deadline has not yet expired; the pusher is an automated service that continues pushing after revocation.
- **No new attacker capability needed**: the creator simply re-submits the same calldata they already used.
- **No test covers this path**: the test suite verifies revocation restores the pusher's own namespace but never tests the creator replaying the original signature post-revocation. [7](#0-6) 

---

### Recommendation

Track used signatures or add a per-pusher revocation nonce. The simplest fix is a `mapping(bytes32 => bool) private _usedConsents` keyed on the full digest, set to `true` on first acceptance and checked before `namespaceRemapping` is written. Alternatively, include a per-pusher monotonic nonce in the signed payload and store the last accepted nonce, so a replayed signature with an old nonce is rejected even within the deadline window.

---

### Proof of Concept

```solidity
// 1. Pusher signs consent with deadline = T+1 day
bytes memory sig = _signConsent(PUSHER_KEY, deadline, pusher, creator);

// 2. Creator establishes delegation
vm.prank(creator);
oracle.allowPushers(deadline, _arr(pusher), _arr(sig));
assertEq(oracle.namespaceRemapping(pusher), creator); // delegated

// 3. Pusher revokes
vm.prank(pusher);
oracle.revokePusher();
assertEq(oracle.namespaceRemapping(pusher), address(0)); // revoked

// 4. Creator replays the SAME signature — no revert, delegation restored
vm.prank(creator);
oracle.allowPushers(deadline, _arr(pusher), _arr(sig));
assertEq(oracle.namespaceRemapping(pusher), creator); // re-established without pusher consent

// 5. Pusher's next push (thinking it goes to own namespace) lands in creator's namespace
uint56 tsMs = uint56(block.timestamp * 1000 + 1);
vm.prank(pusher);
(bool ok,) = address(oracle).call(_wordAt(0, 0, _packRaw(BAD_PRICE, 5, 5), tsMs));
assertTrue(ok);
// BAD_PRICE is now live in creator's namespace, consumed by any pool using that feed
assertEq(oracle.getOracleData(oracle.feedIdOf(creator, 0, 0)).price, U64x32.decode(BAD_PRICE >> 16));
``` [8](#0-7)

    /// @notice Delegates pusher wallets into the caller's namespace. The pusher's EIP-191
    ///         signature is REQUIRED — without it anyone could remap a foreign pusher
    ///         wallet into their own namespace and silently swallow its pushes. The
    ///         deadline is likewise required: the signed consent carries no timestamp of
    ///         its own, so an undated signature could re-establish a delegation AFTER the
    ///         pusher revoked it.
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
