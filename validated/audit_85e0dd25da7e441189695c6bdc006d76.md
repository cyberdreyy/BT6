### Title
Pusher revocation is bypassable via deadline-bounded signature replay, enabling continued bad-price injection into pools - (File: `smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowPushers` in `CompressedOracleV1` binds pusher consent to a `(chainid, oracle, deadline, pusher, creator)` tuple with no nonce. A creator who retains the original signature can call `allowPushers` again after the pusher has self-revoked, atomically restoring `namespaceRemapping[pusher] = creator`. If the pusher's key is compromised, the pusher's only on-chain remedy — `revokePusher()` — is silently undone, and the attacker continues writing into the creator's namespace, which is the live oracle feed consumed by `AnchoredPriceProvider` and downstream pool swaps.

---

### Finding Description

`allowPushers` constructs the signed digest as:

```solidity
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [1](#0-0) 

There is no nonce, no used-signature registry, and no per-pusher revocation counter. `_ensureDeadline` only checks `block.timestamp <= deadline`: [2](#0-1) 

So the same `(deadline, signature)` pair is valid for every call until the deadline expires. The code comment acknowledges the gap — "an undated signature could re-establish a delegation AFTER the pusher revoked it" — but the deadline only bounds the window; it does not make each consent single-use: [3](#0-2) 

`revokePusher` simply zeroes the mapping:

```solidity
namespaceRemapping[msg.sender] = address(0);
``` [4](#0-3) 

Nothing prevents the creator from immediately calling `allowPushers` with the identical signature to restore `namespaceRemapping[pusher] = creator`. The pusher has no counter-move: `revokePusher` is the only self-service path, and it can be replayed away indefinitely until the deadline expires.

---

### Impact Explanation

`CompressedOracleV1.fallback()` resolves the write namespace at call time:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [5](#0-4) 

With the delegation restored, every `fallback` call from the compromised pusher key writes into the creator's namespace. `AnchoredPriceProvider._readLeg` reads that namespace via `IPricedOracle.price(feedId, pool)`: [6](#0-5) 

A crafted slot word that carries a plausible price (non-zero, within `priceGuard` bounds, fresh timestamp, spread below `MAX_SPREAD_BPS`) passes every guard in `_readLeg` and reaches `_computeBidAsk`, producing a bad bid/ask that the pool executes against. Traders receive incorrect amounts; the pool's conservation invariant is broken.

---

### Likelihood Explanation

**Medium.** The preconditions are:

1. A pusher key is compromised (realistic for hot-wallet oracle infrastructure).
2. The creator re-calls `allowPushers` with the old signature — either because they do not know the key is compromised and assume the revocation was accidental, or because they are themselves adversarial.

The window is bounded by the original deadline, which in practice is set far in the future (the docs note the deadline is the only replay guard). No on-chain signal distinguishes a legitimate re-delegation from a malicious one.

---

### Recommendation

**Short term:** Add a per-pusher nonce to the signed digest and increment it on every successful `allowPushers` call:

```solidity
mapping(address => uint256) public pusherNonce;

bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(
        block.chainid, address(this), deadline,
        pusher, msg.sender, pusherNonce[pusher]++
    ))
);
```

This makes each consent single-use: a replayed signature fails because the stored nonce has advanced.

**Long term:** Emit a `PusherRevoked` event that off-chain monitors can use to alert the creator before they inadvertently re-delegate a compromised key.

---

### Proof of Concept

```
1. Pusher signs consent for Creator with deadline T = now + 365 days.
   digest = EIP-191( keccak256(chainid, oracle, T, pusher, creator) )

2. Creator calls allowPushers(T, [pusher], [sig])
   → namespaceRemapping[pusher] = creator  ✓

3. Pusher key is compromised; attacker calls oracle.fallback(badSlotWord)
   → bad price written into creator's namespace

4. Pusher calls revokePusher()
   → namespaceRemapping[pusher] = address(0)

5. Creator calls allowPushers(T, [pusher], [sig])  ← SAME sig, deadline still valid
   → namespaceRemapping[pusher] = creator  ← delegation restored

6. Attacker calls oracle.fallback(badSlotWord) again
   → bad price written into creator's namespace again

7. AnchoredPriceProvider._readLeg reads the bad price (non-zero, fresh ts,
   spread < MAX_SPREAD_BPS, within priceGuard) → ok = true
   → _computeBidAsk produces wrong bid/ask → pool swap executes at bad price
```

    /// @notice Delegates pusher wallets into the caller's namespace. The pusher's EIP-191
    ///         signature is REQUIRED — without it anyone could remap a foreign pusher
    ///         wallet into their own namespace and silently swallow its pushes. The
    ///         deadline is likewise required: the signed consent carries no timestamp of
    ///         its own, so an undated signature could re-establish a delegation AFTER the
    ///         pusher revoked it.
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }

    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
