### Title
Contract-creator feed updates are silently misdirected to the aliased namespace when the L2 sequencer is down and forced inclusion is used — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1.fallback()` resolves the target namespace exclusively from `namespaceRemapping[msg.sender]`, falling back to `msg.sender` itself. On Optimism/Base/Arbitrum, when a **contract** creator uses forced inclusion from L1 (the standard bypass for a downed sequencer), the EVM applies address aliasing: `msg.sender` becomes `originalAddress + 0x1111000000000000000000000000000000001111`. The aliased address is not in `namespaceRemapping`, so the push silently lands in the aliased address's own namespace instead of the creator's namespace. The creator's feed is never updated, goes stale, and every pool reading that feed gets `FeedStalled` on every swap attempt for the entire duration of sequencer downtime.

---

### Finding Description

The `fallback()` push path in `CompressedOracleV1` determines the target namespace as:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [1](#0-0) 

The storage key written is then `bytes32(creator << 96 | slotId)`. [2](#0-1) 

When a contract creator submits a forced-inclusion transaction from L1 (because the L2 sequencer is down), the EVM aliases the sender: `msg.sender = originalCreatorAddress + 0x1111000000000000000000000000000000001111`. `namespaceRemapping[aliasedAddress]` is `address(0)`, so `creator` is set to the aliased address. The slot write goes to `aliasedAddress`'s namespace, not `originalCreatorAddress`'s namespace. The creator's feed is never updated.

The `updateBySignature` path is **not** a viable mitigation for contract creators: it requires an EIP-191 signature from `feedCreator`, which a contract cannot produce. [3](#0-2) 

The `allowContractPushers` path could theoretically pre-register the aliased address, but it requires the creator's contract to implement `isPusher(address)` and the creator to know and register their aliased address before the sequencer goes down — a non-standard, advance-planning requirement that most deployments will not satisfy. [4](#0-3) 

---

### Impact Explanation

Any pool whose price provider reads from a `CompressedOracleV1` feed owned by a contract creator will have all swaps blocked for the entire duration of L2 sequencer downtime. After `MAX_TIME_DELTA` seconds without a valid push, the provider's staleness check returns `true`: [5](#0-4) 

`getBidAndAskPrice()` then reverts with `FeedStalled`: [6](#0-5) 

This matches the allowed impact: **broken core pool functionality — unusable swap flows**.

---

### Likelihood Explanation

- The protocol is explicitly deployed on Base (Optimism-based) and targets Arbitrum — both chains with documented sequencer outages and forced-inclusion support.
- Contract creators are the norm in DeFi (protocols, DAOs, and keeper bots are contracts).
- The aliasing offset (`0x1111000000000000000000000000000000001111`) is deterministic and well-documented in Optimism and Arbitrum specs.
- No existing guard in `CompressedOracleV1` checks for or compensates aliased senders. [7](#0-6) 

---

### Recommendation

In `fallback()`, after the primary `namespaceRemapping[msg.sender]` lookup fails, attempt a de-aliased lookup before falling back to `msg.sender`:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) {
    // L2 forced-inclusion alias: undo the +0x1111...1111 offset
    unchecked {
        address deAliased = address(
            uint160(msg.sender) - uint160(0x1111000000000000000000000000000000001111)
        );
        creator = namespaceRemapping[deAliased];
    }
}
if (creator == address(0)) creator = msg.sender;
```

Apply the same de-aliasing in `allowPushers` and `allowContractPushers` when resolving `msg.sender` as the creator, so delegation setup also works under forced inclusion.

### Proof of Concept

1. `CreatorContract` is deployed on Base (L2) and pushes price data to `CompressedOracleV1` via `fallback()`. Its canonical namespace is `feedIdOf(address(CreatorContract), slotIndex, positionIndex)`.
2. A pool is configured with a `PriceProviderL2` reading from this feed.
3. The Base sequencer goes down.
4. `CreatorContract` submits a fresh price push via forced inclusion from L1.
5. The EVM aliases the sender: `msg.sender = address(CreatorContract) + 0x1111000000000000000000000000000000001111`.
6. `namespaceRemapping[aliasedAddress]` is `address(0)`, so `creator = aliasedAddress`.
7. The slot write lands in `aliasedAddress`'s namespace; `CreatorContract`'s namespace is untouched.
8. After `MAX_TIME_DELTA` seconds, `_isStale` returns `true` for the creator's feed.
9. `getBidAndAskPrice()` reverts with `FeedStalled` on every pool swap attempt.
10. All swaps on the pool are blocked for the full duration of sequencer downtime. [8](#0-7) [9](#0-8) [10](#0-9)

    fallback() override external {
        uint256 end;
        uint256 namespace;

        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0)) creator = msg.sender;

        assembly ("memory-safe") {
            end := calldatasize()
            namespace := shl(96, creator) // [creator:20][zeros:12]
        }

        // 4 * 6 + 7 + 1 = 32 bytes per slot
        if (end == 0 || end % 32 != 0) revert BadCalldataLength();

        for (uint256 ptr = 0; ptr < end; ptr += 32) {
            uint256 word;
            assembly ("memory-safe") {
                word := calldataload(ptr)
            }
            // casting to 'uint8' is safe we want LSB
            // forge-lint: disable-next-line(unsafe-typecast)
            uint8 slotId = uint8(word);
            TimeMs timestampMs = toTimeMs(word >> 8 & X56);
            timestampMs.revertIfAfterBlockTimeWithDrift(MAX_TIME_DRIFT);

            bool newer = timestampMs.isAfter(oldTimestampMs);
            if (!newer) continue;

            _writeStorage(key, bytes32(bytes32(word & ~uint256(0xff))));
