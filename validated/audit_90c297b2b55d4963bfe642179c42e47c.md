### Title
Shared Slot-Level Timestamp in `CompressedOracleV1` Allows Stale Per-Position Prices to Pass Freshness Checks When Multiple Pushers Update the Same Slot ŌĆö (`File: smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1` packs four price positions into a single 256-bit storage slot with **one shared timestamp** for all four positions. Any update to the slot overwrites all four positions and the timestamp atomically. When multiple authorized pushers update the same slot concurrently, each pusher overwrites the other's position data with stale values while advancing the shared slot timestamp. The staleness check in `PriceProvider` and `AnchoredPriceProvider` reads the slot-level timestamp ŌĆö not a per-position timestamp ŌĆö so stale price data for positions not owned by the updating pusher passes the freshness gate and reaches pool swaps.

---

### Finding Description

**Slot structure (the shared state):**

`CompressedOracleV1` stores four oracle lanes in one EVM storage word:

```
bits 255ŌĆ”208 : oracle[0] (48 bits: price + spread0 + spread1)
bits 207ŌĆ”160 : oracle[1]
bits 159ŌĆ”112 : oracle[2]
bits 111ŌĆ” 64 : oracle[3]
bits  63ŌĆ”  8 : timestamp (uint56, unix milliseconds)  ŌåÉ ONE timestamp for all four
bits   7ŌĆ”  0 : reserved
``` [1](#0-0) 

**Atomic whole-slot write:**

Both push paths (`fallback` and `updateBySignature`) overwrite the **entire** storage slot in one `sstore`. There is no mechanism to update a single lane without supplying values for all other lanes. [2](#0-1) 

**Timestamp returned for every position is the slot-level timestamp:**

`getOracleData` assigns `_layout.timestampMs` (the slot-level field) to every position's `OracleData`, regardless of which lane was last updated:

```solidity
data.timestampMs = _layout.timestampMs;
``` [3](#0-2) 

`_loadSlotLayout` reads the single timestamp from bits `[63:8]` of the slot:

```solidity
_layout.timestampMs = toTimeMs(slotValue >> 8 & X56);
``` [4](#0-3) 

**Multiple pushers are explicitly supported:**

`allowPushers` and `allowContractPushers` let a creator delegate any number of EOA or contract pushers into the same namespace. All delegated pushers write to the same set of slots. [5](#0-4) [6](#0-5) 

**The race condition (analog to Ajna's `depositTime = max(prev, new)`):**

1. Creator delegates **Pusher A** (responsible for lanes 0ŌĆō1) and **Pusher B** (responsible for lanes 2ŌĆō3) to the same namespace, both targeting slot `S`.
2. Pusher A reads slot `S` at time `T0`, then pushes at `T1 > T0`: writes `[P0_fresh, P1_fresh, P2_old, P3_old, T1]`. Lanes 2ŌĆō3 carry the values Pusher A read at `T0` ŌĆö it has no knowledge of Pusher B's data.
3. Pusher B reads slot `S` at time `T0` (before Pusher A's write), then pushes at `T2 > T1`: writes `[P0_old, P1_old, P2_fresh, P3_fresh, T2]`. Lanes 0ŌĆō1 carry the values Pusher B read at `T0`.
4. Final slot state: `[P0_old, P1_old, P2_fresh, P3_fresh, T2]`.
5. The slot timestamp is `T2` (fresh). `getOracleData` for lane 0 returns `price = P0_old` with `timestampMs = T2`.

The staleness check in `PriceProvider._isStale` and `AnchoredPriceProvider._isStale` receives `refTime = T2` (seconds), which is within `MAX_TIME_DELTA` / `MAX_REF_STALENESS`, so it returns `false` (not stale). The stale price `P0_old` is accepted and forwarded to the pool. [7](#0-6) [8](#0-7) 

---

### Impact Explanation

A stale price for one or more lanes passes the `_isStale` freshness gate because the slot-level timestamp was advanced by a concurrent pusher updating different lanes. The stale mid-price flows through `PriceProvider._getBidAndAskPrice` or `AnchoredPriceProvider._readLeg` into `_computeBidAsk`, producing a bid/ask pair anchored to an outdated market price. The pool executes swaps at this stale quote: traders receive more output than the current oracle curve permits, or the pool receives less input than owed ŌĆö direct loss of pool LP assets or swap conservation failure. [9](#0-8) [10](#0-9) 

---

### Likelihood Explanation

The protocol explicitly supports and encourages multiple pushers per namespace via `allowPushers` / `allowContractPushers`. A natural deployment pattern is to assign different data sources (e.g., different CEX feeds) to different lanes within the same slot. Each pusher must supply values for all four lanes on every write, but it only has authoritative data for its own lanes. A concurrent write from any other pusher ŌĆö even with no malicious intent ŌĆö overwrites the first pusher's lanes with the stale snapshot the second pusher held at read time. This is a structural race condition inherent to the slot design, not a configuration mistake. [11](#0-10) 

---

### Recommendation

Replace the single slot-level timestamp with **per-lane timestamps**. Each 48-bit lane currently uses `32 + 8 + 8 = 48` bits for `(price, spread0, spread1)`. One option is to reduce price precision slightly and embed a per-lane timestamp delta, or to use a second storage slot for timestamps. Alternatively, expose a **partial-lane update path** that reads the current slot, merges only the updated lane, and writes back ŌĆö preventing pushers from clobbering lanes they do not own. At minimum, document that all pushers sharing a namespace must coordinate to supply the current on-chain values for lanes they do not own, and add an integration test covering the concurrent-pusher scenario. [12](#0-11)

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

import "forge-std/Test.sol";
import {CompressedOracleV1} from "contracts/oracles/compressed/CompressedOracle.sol";
import {IOffchainOracle} from "contracts/interfaces/IOffchainOracle.sol";
import {U64x32} from "contracts/oracles/utils/U64x32.sol";
import {TimeMs} from "contracts/oracles/utils/TimeMs.sol";

contract SharedTimestampPoC is Test {
    CompressedOracleV1 oracle;
    address creator = address(0xC0FFEE);
    address pusherA  = address(0xAAAA);
    address pusherB  = address(0xBBBB);

    // Helpers: pack a 48-bit lane (price=U64x32 encoded, s0, s1)
    function _lane(uint32 p, uint8 s0, uint8 s1) internal pure returns (uint48) {
        return (uint48(p) << 16) | (uint48(s0) << 8) | uint48(s1);
    }

    // Build a full 32-byte slot word: [lane0][lane1][lane2][lane3][ts56][slotId8]
    function _word(uint48 l0, uint48 l1, uint48 l2, uint48 l3, uint56 ts, uint8 sid)
        internal pure returns (bytes memory)
    {
        uint256 w = (uint256(l0) << 208) | (uint256(l1) << 160) |
                    (uint256(l2) << 112) | (uint256(l3) << 64)  |
                    (uint256(ts) << 8)   | uint256(sid);
        return abi.encodePacked(w);
    }

    function setUp() public {
        vm.warp(1_700_000_000);
        oracle = new CompressedOracleV1(address(this), 0);

        // Delegate both pushers to creator's namespace
        address[] memory pushers = new address[](2);
        pushers[0] = pusherA; pushers[1] = pusherB;
        // (signature setup omitted for brevity ŌĆö use allowContractPushers variant in practice)
    }

    function test_sharedTimestampMakesStaleDataAppearFresh() public {
        uint56 T1 = uint56(block.timestamp * 1000);
        uint56 T2 = T1 + 5_000; // 5 seconds later

        // Pusher A pushes at T1: lanes 0-1 fresh (price=100), lanes 2-3 zero (unknown)
        uint48 freshLane  = _lane(U64x32.encode(100e8), 5, 5);
        uint48 staleLane  = _lane(U64x32.encode(0),     0, 0); // zero placeholder

        vm.prank(pusherA);
        (bool ok,) = address(oracle).call(_word(freshLane, freshLane, staleLane, staleLane, T1, 0));
        assertTrue(ok);

        // Pusher B pushes at T2: lanes 2-3 fresh (price=200), lanes 0-1 carry T1 snapshot
        // (Pusher B read the slot BEFORE Pusher A's write ŌĆö race condition)
        uint48 freshLane2 = _lane(U64x32.encode(200e8), 5, 5);
        uint48 staleSnap  = _lane(U64x32.encode(50e8),  5, 5); // old value Pusher B saw

        vm.warp(block.timestamp + 5);
        vm.prank(pusherB);
        (ok,) = address(oracle).call(_word(staleSnap, staleSnap, freshLane2, freshLane2, T2, 0));
        assertTrue(ok);

        // Lane 0 now has stale price (50e8) but slot timestamp = T2 (fresh)
        bytes32 feedId0 = oracle.feedIdOf(creator, 0, 0);
        IOffchainOracle.OracleData memory d = oracle.getOracleData(feedId0);

        // Timestamp appears fresh (T2), but price is Pusher B's stale snapshot
        assertEq(TimeMs.unwrap(d.timestampMs), uint256(T2), "timestamp is T2 (fresh)");
        assertEq(d.price, U64x32.decode(uint32(staleSnap >> 16)), "price is stale snapshot");

        // PriceProvider._isStale(T2/1000, block.timestamp, MAX_TIME_DELTA) ŌåÆ false
        // ŌåÆ stale price passes freshness gate and reaches pool swap
    }
}

    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference ŌåÆ not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity ŌĆö mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
