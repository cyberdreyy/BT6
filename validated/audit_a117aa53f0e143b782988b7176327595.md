**Impact assessment:**

After step 4, P's subsequent fallback pushes resolve `namespaceRemapping[P] = creator` and write into the creator's namespace, not P's own. P has no on-chain signal that their revocation was undone. Two concrete consequences:

### Title
Stale-but-unexpired pusher consent signature can be replayed by the creator to silently re-delegate a self-revoked pusher — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary
`allowPushers` performs no replay-prevention beyond a deadline check. A creator who holds a pusher's original consent signature can call `allowPushers` again with that same signature after the pusher has called `revokePusher()`, re-writing `namespaceRemapping[pusher] = creator` and silently undoing the revocation for the remainder of the deadline window.

### Finding Description
The signature domain for `allowPushers` is `keccak256(abi.encode(chainid, oracle, deadline, pusher, creator))`. [6](#0-5) 

Nothing in this domain encodes a nonce, a revocation epoch, or any state that changes when `revokePusher()` is called. `_ensureDeadline` only checks `block.timestamp <= deadline`; it does not consume or invalidate the signature. [1](#0-0) 

`revokePusher` sets `namespaceRemapping[msg.sender] = address(0)` but does not record any revocation state that `allowPushers` could check. [3](#0-2) 

The code comment at lines 186–191 explicitly claims the deadline prevents re-establishment after revocation, but this is incorrect: the deadline only prevents replay after it expires, not within the window. [5](#0-4) 

### Impact Explanation
After silent re-delegation, the fallback push path resolves `namespaceRemapping[pusher]` at call time and routes all of P's pushes into the creator's namespace. [4](#0-3) 

- P's own pool (`feedIdOf(P, …)`) receives no new data and becomes stale/unusable — broken core pool functionality.
- The creator's pool (`feedIdOf(creator, …)`) consumes P's data without renewed consent. If P revoked because their data was stale or incorrect, the creator's pool executes swaps at bad prices.

### Likelihood Explanation
The creator retains the original signature bytes from the first `allowPushers` call. No privileged access is required. The attack is executable by any creator against any pusher who self-revoked, as long as the original deadline has not yet expired. Deadlines are typically set days or weeks in the future, giving a wide exploitation window.

### Recommendation
Track consumed signatures or introduce a per-pusher revocation nonce. The simplest fix is a `mapping(bytes32 => bool) private _usedConsents` keyed on the full signature hash (or the domain hash), set to `true` on first use in `allowPushers` and checked before writing `namespaceRemapping`. Alternatively, include a monotonically increasing per-pusher nonce in the signed domain so that `revokePusher` increments the nonce, invalidating all prior signatures.

function testRevokeReplayVulnerability() public {

    // 1. Pusher signs consent
    bytes memory sig = _signConsent(PUSHER_KEY, deadline, pusher, creator);

    // 2. Creator establishes delegation

    pushers[0] = pusher;

    sigs[0] = sig;
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs);
    assertEq(oracle.namespaceRemapping(pusher), creator);

    vm.prank(pusher);
    oracle.revokePusher();
    assertEq(oracle.namespaceRemapping(pusher), address(0));

    // 4. Creator replays the SAME signature — succeeds, re-delegates pusher
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs); // no revert
    assertEq(oracle.namespaceRemapping(pusher), creator); // pusher silently re-delegated

    // 5. Pusher's subsequent push lands in creator's namespace, not their own
    vm.prank(pusher);
    (bool ok,) = address(oracle).call(_wordAt(0, 0, _packRaw(999_000, 1, 1),

    assertTrue(ok);
    // Creator's feed has data; pusher's own feed is empty
    assertGt(oracle.getOracleData(oracle.feedIdOf(creator, 0, 0)).price, 0);
    assertEq(oracle.getOracleData(oracle.feedIdOf(pusher,  0, 0)).price, 0);
}

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
