### Title
Pusher Delegation Re-Established After Revocation via Signature Replay Enables Bad-Price Injection — (File: `smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary
`allowPushers` in `CompressedOracleV1` does not check whether `namespaceRemapping[pusher]` is already set before overwriting it. A creator can replay a pusher's original consent signature — which remains cryptographically valid until its deadline — to silently re-establish a delegation the pusher has already revoked. This breaks the revocation invariant and, if the pusher's key is compromised, allows an attacker to continue injecting bad prices into the creator's namespace and onward into pool swaps.

### Finding Description
`allowPushers` verifies the pusher's EIP-191 signature and then unconditionally writes:

```solidity
namespaceRemapping[pusher] = msg.sender;
```

There is no guard of the form `require(namespaceRemapping[pusher] == address(0))` before the write. The signed consent domain is:

```solidity
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
```

Because the deadline is the only time-bound on the signature, any unexpired signature can be replayed by the creator at any time — including after the pusher has called `revokePusher()`. `revokePusher` clears `namespaceRemapping[pusher]` to `address(0)`, but the creator immediately re-establishes it by submitting the same calldata to `allowPushers` again. The code's own comment acknowledges the risk ("an undated signature could re-establish a delegation AFTER the pusher revoked it") but treats the deadline as sufficient mitigation; it is not — the deadline only bounds the window, it does not prevent replay within that window.

The `fallback()` push path resolves the namespace at call time:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
```

So once the mapping is re-established, every subsequent push from the (now-compromised) pusher key lands in the creator's namespace, overwriting live feed slots with attacker-controlled prices.

### Impact Explanation
If a pusher's private key is compromised, the pusher calls `revokePusher()` to stop the attacker. The creator, unaware of the compromise, replays the original consent signature to re-establish the delegation. The attacker (holding the compromised key) then pushes arbitrary `(price, s0, s1)` tuples into the creator's namespace via `fallback()`. Those values are decoded by `getOracleData` → `U64x32.decode` / `Codebook256.decode` and forwarded to `AnchoredPriceProvider._readLeg` → `_computeBidAsk`. Prices within the `priceGuard` bounds and below `MAX_SPREAD_BPS` pass all guards and reach `MetricOmmPool.swap`, causing traders to receive incorrect output amounts — a direct loss of user principal.

### Likelihood Explanation
Low. Requires: (1) a pusher EOA key to be compromised, (2) the pusher to call `revokePusher`, (3) the creator to replay the old signature (e.g., via an automated re-delegation script unaware of the revocation event), and (4) the attacker to craft prices within the `AnchoredPriceProvider`'s guard bounds. The `allowContractPushers` path is not affected because consent is proven by a live `isPusher` staticcall that the pusher contract can update.

### Recommendation
Add a guard in `allowPushers` (and symmetrically in `allowContractPushers`) that rejects re-delegation when the pusher is already mapped:

```solidity
require(
    namespaceRemapping[pusher] == address(0),
    "pusher already delegated; call removePushers first"
);
```

This mirrors the fix recommended in the original report (loop-check before overwrite) and ensures that a revocation is final until the current creator explicitly removes the pusher and the pusher signs fresh consent.

### Proof of Concept

```
1. Creator A calls allowPushers([P], [sig_P_for_A], deadline = now + 365 days)
   → namespaceRemapping[P] = A

2. P's private key is compromised.

3. P calls revokePusher()
   → namespaceRemapping[P] = address(0)   ← revocation recorded

4. A (running an automated re-delegation keeper, unaware of the compromise)
   calls allowPushers([P], [sig_P_for_A], deadline = now + 365 days)
   with the IDENTICAL signature from step 1 (deadline still in the future)
   → namespaceRemapping[P] = A            ← revocation silently undone

5. Attacker (holding P's key) calls oracle.fallback() with a crafted slot word:
   price = attacker-chosen value within priceGuard bounds
   s0, s1 = valid Codebook256 indices
   timestamp = block.timestamp * 1000 (newer than stored)
   → _writeStorage(bytes32(namespace_A | slotId), craftedWord)

6. AnchoredPriceProvider._readLeg(baseFeedId) reads the corrupted slot,
   passes staleness / spread / priceGuard checks, and returns the bad mid.

7. MetricOmmPool.swap executes at the attacker-controlled bid/ask,
   causing traders to receive more or less than the fair market value.
```

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }

    /// @notice Allows a pusher to self-revoke their delegation. After revocation the
    ///         wallet pushes into its OWN namespace again (the registrationless default).
    function revokePusher() external {
        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
        namespaceRemapping[msg.sender] = address(0);
        emit PusherRevoked(msg.sender, creator);
    }

    fallback() override external {
        uint256 end;
        uint256 namespace;

        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0)) creator = msg.sender;

        assembly ("memory-safe") {
            end := calldatasize()
            namespace := shl(96, creator) // [creator:20][zeros:12]
        }

        // 4 * 6 + 7 + 1 = 32 bytes per slot
        if (end == 0 || end % 32 != 0) revert BadCalldataLength();

        for (uint256 ptr = 0; ptr < end; ptr += 32) {
            uint256 word;
            assembly ("memory-safe") {
                word := calldataload(ptr)
            }
            // casting to 'uint8' is safe we want LSB
            // forge-lint: disable-next-line(unsafe-typecast)
            uint8 slotId = uint8(word);
            TimeMs timestampMs = toTimeMs(word >> 8 & X56);
            timestampMs.revertIfAfterBlockTimeWithDrift(MAX_TIME_DRIFT);
            bytes32 key = bytes32(namespace | uint256(slotId));
            uint256 old = uint256(_loadStorage(key));
            TimeMs oldTimestampMs = toTimeMs(old >> 8 & X56);

            bool newer = timestampMs.isAfter(oldTimestampMs);
            if (!newer) continue;

            _writeStorage(key, bytes32(bytes32(word & ~uint256(0xff))));
        }

    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
