### Title
`allowPushers` Delegation Signature Lacks Per-Use Replay Protection, Enabling Post-Revocation Namespace Hijack — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowPushers` verifies a pusher's EIP-191 consent signature but stores no record of used signatures and has no nonce. A creator who saved the original calldata can re-submit the same `(deadline, pusher, signature)` tuple after the pusher has revoked, silently re-establishing the delegation and redirecting the pusher's future price writes back into the creator's namespace.

---

### Finding Description

`allowPushers` reconstructs the signed digest as:

```solidity
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [1](#0-0) 

The only time-based guard is `_ensureDeadline(deadline)`, which rejects calls after the deadline but does **not** prevent the same valid signature from being accepted multiple times before it expires. [2](#0-1) 

There is no `usedSignatures` mapping, no nonce field in the signed payload, and no check that `namespaceRemapping[pusher]` is currently `address(0)` before writing. After a pusher calls `revokePusher()` (which sets `namespaceRemapping[pusher] = address(0)`), the creator can immediately replay the original transaction to restore `namespaceRemapping[pusher] = creator`. [3](#0-2) 

The comment in the code acknowledges the deadline is needed to prevent post-revocation re-establishment, but a deadline alone is insufficient — it only bounds the window, it does not close it. [4](#0-3) 

---

### Impact Explanation

The `fallback` push path resolves the namespace by reading `namespaceRemapping[msg.sender]` at push time:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [5](#0-4) 

If the pusher has revoked and re-delegated to a new creator B (e.g., a legitimate pool operator), creator A can replay the old signature to overwrite `namespaceRemapping[pusher]` back to `creator_A`. Every subsequent push by the pusher then lands in creator A's namespace instead of creator B's. Creator B's feeds become stale; any pool consuming creator B's `feedId` will receive a zero or outdated timestamp, which downstream price providers treat as a stale/bad price. This satisfies the **bad-price execution** and **broken core pool functionality** impact criteria.

---

### Likelihood Explanation

- The original `allowPushers` calldata is permanently visible on-chain; no off-chain secret is needed.
- The creator is a semi-trusted but potentially adversarial party (they control their own namespace and have economic incentive to retain a pusher).
- The deadline is typically set far in the future (tests use `block.timestamp + 1 days`); the replay window is wide.
- No special privilege is required beyond being the original `msg.sender` (creator) in the first delegation call.

---

### Recommendation

Track consumed signatures with a `mapping(bytes32 => bool) private _usedDelegations` and mark each digest as used on first acceptance:

```solidity
bytes32 hash = ...; // existing digest
require(!_usedDelegations[hash], "signature already used");
_usedDelegations[hash] = true;
require(pusher == ECDSA.recover(hash, signatures[i]));
```

Alternatively, include a per-pusher nonce in the signed payload (`keccak256(abi.encode(..., nonces[pusher]++))`) so each consent is single-use by construction.

---

### Proof of Concept

```solidity
// 1. Pusher signs consent for creator with deadline = now + 1 days
bytes memory sig = _signConsent(PUSHER_KEY, deadline, pusher, creator);

// 2. Creator establishes delegation
vm.prank(creator);
oracle.allowPushers(deadline, _arr(pusher), _arr(sig));
assertEq(oracle.namespaceRemapping(pusher), creator); // delegated

// 3. Pusher revokes
vm.prank(pusher);
oracle.revokePusher();
assertEq(oracle.namespaceRemapping(pusher), address(0)); // revoked

// 4. Creator replays the SAME signature — succeeds, no revert
vm.prank(creator);
oracle.allowPushers(deadline, _arr(pusher), _arr(sig));
assertEq(oracle.namespaceRemapping(pusher), creator); // hijacked back
```

The pusher's subsequent fallback pushes now land in `creator`'s namespace, not their own or creator B's, causing stale prices in any pool that reads creator B's feeds.

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
```
