### Title
`allowContractPushers` Allows Unconditional Overwrite of an Existing `namespaceRemapping` Entry, Silently Redirecting a Live Pusher and Staling Pool Feeds - (File: `smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1.allowContractPushers` writes `namespaceRemapping[pusher] = msg.sender` unconditionally. It does not check whether the pusher is already delegated to a different creator. Any caller for whom `pusher.isPusher(caller)` returns `true` can silently redirect an active pusher contract out of the original creator's namespace and into their own, causing the original creator's feeds to stop being updated and every pool that reads those feeds to revert with `FeedStalled`.

---

### Finding Description

`allowContractPushers` proves consent via a live `staticcall` to `pusher.isPusher(msg.sender)`. If that call returns `true`, the function unconditionally overwrites `namespaceRemapping[pusher]`:

```solidity
// CompressedOracle.sol lines 217-234
function allowContractPushers(address[] calldata pushers) external {
    uint256 l = pushers.length;
    for (uint256 i; i < l; i++) {
        address pusher = pushers[i];

        if (pusher == msg.sender) {
            revert NoSelfRemapping();
        }

        (bool ok, bytes memory res) = pusher.staticcall(
            abi.encodeWithSignature("isPusher(address)", msg.sender)
        );
        require(ok);
        bool allowed = abi.decode(res, (bool));
        require(allowed);

        namespaceRemapping[pusher] = msg.sender;   // ← unconditional overwrite
        emit PusherAuthorized(pusher, msg.sender);
    }
}
``` [1](#0-0) 

There is no guard of the form `require(namespaceRemapping[pusher] == address(0) || namespaceRemapping[pusher] == msg.sender)`. Any address for which `pusher.isPusher(addr)` returns `true` — regardless of whether the pusher is already delegated — can steal the delegation.

The fallback push path resolves the namespace at call time:

```solidity
// CompressedOracle.sol lines 315-316
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [2](#0-1) 

After the overwrite, every subsequent push from `pusherContract` lands in the attacker's namespace. The original creator's namespace receives no further updates.

The `feedId` consumed by a pool is hardcoded at provider construction time as `feedIdOf(creator1, slotIndex, positionIndex)`: [3](#0-2) 

Once `creator1`'s feeds stop being updated, `timestampMs` ages past `MAX_TIME_DELTA`. `ProtectedPriceProvider._computeBidAsk` returns the stalled sentinel `(0, type(uint128).max)`, and `getBidAndAskPrice` reverts `FeedStalled`: [4](#0-3) [5](#0-4) 

Every swap through that pool reverts for as long as the delegation remains stolen.

---

### Impact Explanation

Every pool whose price provider reads `feedIdOf(creator1, ...)` becomes unable to execute swaps. `MetricOmmPool.swap` calls `getSellAndBuyPrices`, which calls `getBidAndAskPrice`, which reverts `FeedStalled`. Liquidity providers cannot receive fees; traders cannot swap; the pool is effectively frozen. This satisfies the "broken core pool functionality — unusable swap flows" impact criterion.

---

### Likelihood Explanation

The trigger requires a pusher contract whose `isPusher` function returns `true` for more than one caller. This is a realistic design: a multi-operator pusher, a factory-deployed pusher that whitelists a set of creators, or any contract that does not restrict `isPusher` to a single address. The attacker needs no special privilege — `allowContractPushers` is a public, permissionless function. The original creator has no on-chain mechanism to lock their delegation against re-delegation by a third party.

---

### Recommendation

Before writing `namespaceRemapping[pusher] = msg.sender`, check that the pusher is not already delegated to a different creator:

```solidity
address current = namespaceRemapping[pusher];
require(
    current == address(0) || current == msg.sender,
    AlreadyDelegated(pusher, current)
);
```

This preserves the registrationless design while preventing silent namespace theft. Alternatively, require the current creator to explicitly call `removePushers` before a new creator can claim the pusher.
