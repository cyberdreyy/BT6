### Title
`allowPushers` signature replay within deadline window nullifies `revokePusher` — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary
`allowPushers` has no nonce or used-signature tracking. A pusher consent signature is replayable by the creator any number of times within the deadline window, so a pusher's `revokePusher()` call is silently undone by replaying the original signature. A compromised pusher key that the pusher tried to revoke can be re-delegated and continue pushing bad prices into pools.

### Finding Description

`allowPushers` verifies the pusher's EIP-191 consent signature and writes `namespaceRemapping[pusher] = msg.sender`: [1](#0-0) 

The signed message is `keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))`. The only replay guard is the deadline: [2](#0-1) 

There is no nonce, no used-signature bitmap, and no check that `namespaceRemapping[pusher]` is currently `address(0)` before writing. The code's own comment acknowledges the risk ("an undated signature could re-establish a delegation AFTER the pusher revoked it") and claims the deadline prevents it — but the deadline only prevents use *after* it expires, not repeated use *within* the window.

`revokePusher` clears the mapping: [3](#0-2) 

Because `allowPushers` does not invalidate the signature on first use, the creator can call `allowPushers` again with the identical `(deadline, pusher, sig)` tuple at any point before the deadline, immediately restoring `namespaceRemapping[pusher] = creator` and nullifying the revocation.

### Impact Explanation

Once re-delegated, the pusher (or an attacker holding a compromised pusher key) can call the `fallback` push path and overwrite any slot in the creator's namespace with an arbitrary price, spread, and timestamp: [4](#0-3) 

The `feedId` encodes `(creator, chainid, slotIndex, positionIndex)`, so every feed in the creator's namespace is writable. Any pool whose `PriceProvider` reads from this oracle will receive the manipulated bid/ask price on the next swap, causing bad-price execution and direct loss to LPs or traders.

### Likelihood Explanation

The creator must call `allowPushers` a second time. This can happen:
- **Inadvertently**: the creator's deployment script or keeper re-runs the delegation setup without knowing the pusher revoked.
- **Maliciously**: a creator who wants to keep a pusher active despite the pusher's objection (e.g., after a key-compromise disclosure) simply replays the old signature.

The pusher's signed consent is a public on-chain artifact (emitted in the original transaction's calldata), so the creator does not need to store it separately.

### Recommendation

Track consumed signatures with a `mapping(bytes32 => bool) usedSignatures` and mark each `hash` as used on first acceptance. Alternatively, include a per-pusher nonce in the signed message and increment it on each successful delegation, so a revoked pusher must produce a fresh signature with the new nonce before they can be re-delegated.

### Proof of Concept

```
T=0:   pusher signs hash = keccak256(abi.encode(chainid, oracle, deadline=T+1day, pusher, creator))
T=1:   creator calls allowPushers(T+1day, [pusher], [sig])
         → namespaceRemapping[pusher] = creator  ✓

T=2:   pusher's key is compromised; pusher calls revokePusher()
         → namespaceRemapping[pusher] = address(0)  ✓ (pusher believes they are safe)

T=3:   creator (or attacker who observed the original calldata) calls
       allowPushers(T+1day, [pusher], [sig])  ← SAME signature, deadline still valid
         → namespaceRemapping[pusher] = creator  ← revocation silently undone

T=4:   attacker (holding compromised pusher key) calls fallback() with
       manipulated slot word (arbitrary price, spread indexes, fresh timestamp)
         → creator's namespace slot overwritten with bad price

T=5:   pool.swap() → PriceProvider.getBidAndAskPrice() → oracle.price(feedId, pool)
         → returns attacker-controlled bid/ask → swap executes at wrong price → LP loss
```

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }

    function revokePusher() external {
        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
        namespaceRemapping[msg.sender] = address(0);
        emit PusherRevoked(msg.sender, creator);
    }

    fallback() override external {
        uint256 end;
        uint256 namespace;

        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0)) creator = msg.sender;

        assembly ("memory-safe") {
            end := calldatasize()
            namespace := shl(96, creator) // [creator:20][zeros:12]
        }

        // 4 * 6 + 7 + 1 = 32 bytes per slot
        if (end == 0 || end % 32 != 0) revert BadCalldataLength();

        for (uint256 ptr = 0; ptr < end; ptr += 32) {
            uint256 word;
            assembly ("memory-safe") {
                word := calldataload(ptr)
            }
            // casting to 'uint8' is safe we want LSB
            // forge-lint: disable-next-line(unsafe-typecast)
            uint8 slotId = uint8(word);
            TimeMs timestampMs = toTimeMs(word >> 8 & X56);
            timestampMs.revertIfAfterBlockTimeWithDrift(MAX_TIME_DRIFT);
            bytes32 key = bytes32(namespace | uint256(slotId));
            uint256 old = uint256(_loadStorage(key));
            TimeMs oldTimestampMs = toTimeMs(old >> 8 & X56);

            bool newer = timestampMs.isAfter(oldTimestampMs);
            if (!newer) continue;

            _writeStorage(key, bytes32(bytes32(word & ~uint256(0xff))));
