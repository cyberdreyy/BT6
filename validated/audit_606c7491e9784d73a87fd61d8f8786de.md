### Title
Sibling-Lane Timestamp Inheritance Allows Zero-Price Feed to Bypass Staleness Check — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1` stores one 56-bit timestamp **per storage slot**, shared across all four 48-bit oracle lanes packed into that slot. When a pusher writes a slot word that populates only one lane (leaving the other three at zero), those zero-price lanes inherit the slot's fresh, valid timestamp. `getOracleData` for such a lane returns `price = 0` with a non-stale `timestampMs`, bypassing every downstream staleness guard and feeding a zero mid-price into any `PriceProvider` / `AnchoredPriceProvider` that reads it.

---

### Finding Description

**Root cause — per-slot timestamp, not per-lane:**

`_loadSlotLayout` extracts the timestamp from bits `[63:8]` of the single 256-bit storage word and assigns it to the entire `SlotLayout`: [1](#0-0) 

All four lanes (`oracle0`–`oracle3`) share this one timestamp. There is no per-lane freshness field.

**How a zero-price lane acquires a fresh timestamp:**

The `fallback` push path (and the `_wordAt` test helper) constructs a slot word by ORing a single lane's 48-bit value into the word and leaving the other three lane fields as zero:

```
word = (tsMs << 8) | slotId;
word |= uint256(raw) << (208 - pos * 48);   // only one lane set
```

When this word is stored, lanes at positions other than `pos` have `p = 0, s0 = 0, s1 = 0` but the slot's `timestampMs` is the freshly supplied value.

**`getOracleData` does not reject price = 0 with valid timestamp:**

The only early-exit guard is the stalled sentinel (`s0 == 0xFF && s1 == 0xFF`): [2](#0-1) 

A zero-price lane has `s0 = 0, s1 = 0`, so it **does not** hit the sentinel. Execution falls through to:

```solidity
data.price      = U64x32.decode(0);          // → 0
data.spread0    = _decodeCodebookIndex(0);   // valid codebook entry
data.spread1    = _decodeCodebookIndex(0);   // valid codebook entry
data.timestampMs = _layout.timestampMs;      // ← fresh, non-stale
```

The `price()` view therefore returns `(mid=0, spread0, spread1, refTime=valid)`.

**The existing test acknowledges the behavior but does not check the timestamp:** [3](#0-2) 

The test asserts `price == 0` for the sibling lane but never asserts `timestampMs == 0`. In fact, `timestampMs` equals the pushed timestamp — the lane looks fresh to every consumer.

**Slot-structure documentation confirms the design gap:** [4](#0-3) 

The warning "you must still supply correct values for the other lanes" is documentation only; the contract enforces nothing.

---

### Impact Explanation

Any `PriceProvider` or `AnchoredPriceProvider` that reads a zero-price lane receives `mid = 0` with a timestamp that passes the `maxTimeDrift` staleness check. The provider computes `bid = 0` and `ask = 0` for the pool. A pool operating on those quotes will:

- Accept any amount of the quote token for zero output (or emit the entire reserve for zero input), matching the "bad-price execution" allowed impact: *stale, inverted, unbounded, or unclamped bid/ask quote reaches a pool swap*.
- Result in direct loss of LP principal — pool insolvency — because balances no longer cover LP claims.

---

### Likelihood Explanation

The trigger is **semi-trusted and operationally realistic**:

1. A legitimate creator runs an off-chain pusher that updates one lane at a time (e.g., only the asset whose price changed). If the pusher constructs the slot word without first reading and preserving the current on-chain values of the other three lanes, it zeros them out with a fresh timestamp.
2. The `_wordAt` helper used throughout the test suite does exactly this — it is the natural single-lane update pattern.
3. No malicious intent is required; a routine price update to lane 0 silently invalidates lanes 1–3 for any `PriceProvider` pointed at them.

The `feedIdOf` encoding embeds `positionIndex` in the feed id, so a `PriceProvider` configured for lane 1 will silently receive `price = 0` the moment the creator pushes a single-lane update to the same slot. [5](#0-4) 

---

### Recommendation

**Option A — Per-lane timestamp (preferred):** Store a separate timestamp per 48-bit lane instead of one per slot. This eliminates the inheritance entirely.

**Option B — Zero-price sentinel in `getOracleData`:** Treat `p == 0` as equivalent to the stalled sentinel and return `spread0 = spread1 = BPS_BASE` (and `timestampMs = 0`) so downstream consumers see a stale/invalid feed:

```solidity
if (compressed.p == 0 || (compressed.s1 == 0xff && compressed.s0 == 0xff)) {
    data.spread1 = BPS_BASE;
    data.spread0 = BPS_BASE;
    return data;   // timestampMs stays zero → stale to every consumer
}
```

**Option C — Enforce full-slot writes:** In the `fallback` push path, revert if any lane field in the incoming word is zero while the corresponding on-chain lane is non-zero (analogous to the `UninializedPush` guard present in the older registered-oracle version). [6](#0-5)

### Proof of Concept

```solidity
function testSiblingLaneInheritsTimestampAndBypassesStaleness() public {
    address creator = address(0xC0FFEE);
    uint56 tsMs = uint56(block.timestamp * 1000);

    // Push only lane 1 with a valid price; lanes 0, 2, 3 are left at zero.
    uint48 raw = _packRaw(1_000_000, 5, 3);
    vm.prank(creator);
    (bool ok,) = address(oracle).call(_wordAt(8, 1, raw, tsMs));
    assertTrue(ok);

    // Lane 0 was never written — price is 0.
    bytes32 feedId0 = oracle.feedIdOf(creator, 8, 0);
    IOffchainOracle.OracleData memory data = oracle.getOracleData(feedId0);

    assertEq(data.price, 0, "price is zero");

    // BUG: timestampMs is the slot's fresh timestamp, NOT zero.
    // Every consumer that checks `timestampMs > 0 && age < maxTimeDrift`
    // will accept this as a live, valid price of 0.
    assertEq(
        TimeMs.unwrap(data.timestampMs),
        uint256(tsMs),
        "sibling lane inherits fresh timestamp — staleness check bypassed"
    );

    // price() surfaces the same: mid=0, refTime=valid.
    (uint256 mid,,, uint256 refTime) = oracle.price(feedId0, address(0));
    assertEq(mid, 0);
    assertGt(refTime, 0, "refTime is non-zero — consumer sees a fresh zero price");
}
```

    function _loadSlotLayout(bytes32 slotIndex) internal view returns (SlotLayout memory _layout) {
        uint256 slotValue;
        assembly ("memory-safe") {
            slotValue := sload(slotIndex)
