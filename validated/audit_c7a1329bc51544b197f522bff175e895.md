### Title
`allowPushers` consent signature carries no single-use protection, letting a creator replay a revoked pusher's consent within the deadline window and silently re-establish a delegation the pusher explicitly cancelled — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowPushers` verifies a pusher's EIP-191 consent signature but never marks it as consumed. After a pusher calls `revokePusher()` to clear `namespaceRemapping[pusher]`, the creator can call `allowPushers` again with the **identical original signature** (before the deadline) to silently restore `namespaceRemapping[pusher] = creator`. This is the direct Metric OMM analog of the BMX finalize-window bug: just as auto-voters could change their vote after epoch end but before finalization because the contract read live state at tally time, a pusher's self-revocation can be undone within the deadline window because the delegation consent carries no monotonic counter, allowing bad prices to continue flowing into the creator's namespace and subsequently into pool swaps.

---

### Finding Description

`allowPushers` in `CompressedOracle.sol` verifies the pusher's EIP-191 signature over the domain:

```
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [1](#0-0) 

There is no nonce, no `usedSignatures` bitmap, and no per-pusher revocation counter. The only replay guard is `_ensureDeadline`, which checks `block.timestamp <= deadline`: [2](#0-1) 

This means the same signature is valid for **every call** to `allowPushers` until the deadline expires. The code comment acknowledges the concern explicitly:

> "The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it." [3](#0-2) 

The deadline is intended to *limit* the replay window, but it does not *close* it. Within the window, `revokePusher()` clears `namespaceRemapping[pusher]` to `address(0)`: [4](#0-3) 

But the creator can immediately call `allowPushers` again with the original signature to restore `namespaceRemapping[pusher] = creator`, nullifying the pusher's self-revocation. The `fallback` push path reads `namespaceRemapping[msg.sender]` at push time: [5](#0-4) 

So once the delegation is re-established, any subsequent push from the pusher's address lands in the **creator's namespace**, not the pusher's own.

---

### Impact Explanation

If a pusher's signing key is compromised, the pusher calls `revokePusher()` to stop the attacker from writing bad prices into the creator's namespace. The creator — unaware of the compromise, or acting maliciously — replays the original consent signature before the deadline, restoring the delegation. The attacker (holding the compromised key) then continues pushing manipulated prices into the creator's namespace. Those prices are read by `AnchoredPriceProvider._readLeg` via `IPricedOracle.price(feedId, pool)`: [6](#0-5) 

A manipulated mid or spread that passes the staleness and guard checks reaches `_computeBidAsk`, producing an incorrect bid/ask that the pool uses to settle swaps — causing direct loss of funds to traders or LPs.

---

### Likelihood Explanation

Medium. The trigger requires two concurrent conditions: (1) the pusher's key is compromised and the pusher self-revokes, and (2) the creator replays the original signature before the deadline (either negligently or maliciously). The creator is a semi-trusted party whose `allowPushers` call is a valid, unprivileged-path action. Deadlines in production are typically set days to weeks in the future, leaving a wide replay window.

---

### Recommendation

Track each consent signature as consumed after first use, for example with a `mapping(bytes32 => bool) public usedConsents` keyed on the signature hash, and revert if the hash has already been recorded. Alternatively, add a per-pusher monotonic nonce to the signature domain so that `revokePusher()` increments the nonce and invalidates all prior signatures:

```solidity
// In the signature domain:
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender, pusherNonce[pusher]))

// In revokePusher():
pusherNonce[msg.sender]++;
namespaceRemapping[msg.sender] = address(0);
```

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

import {Test} from "forge-std/Test.sol";
import {CompressedOracleV1} from "src/oracles/compressed/CompressedOracle.sol";
import {MessageHashUtils} from "@openzeppelin/contracts/utils/cryptography/MessageHashUtils.sol";

contract AllowPushersReplayTest is Test {
    CompressedOracleV1 oracle;
    uint256 pusherKey = 0xBEEF;
    address creator;
    address pusher;

    function setUp() public {
        creator = address(0xC0FFEE);
        pusher  = vm.addr(pusherKey);
        oracle  = new CompressedOracleV1(address(this), 0);
        vm.warp(1_700_000_000);
    }

    function test_replayRevokedDelegation() public {
        uint256 deadline = block.timestamp + 1 days;

        // Pusher signs consent once.
        bytes32 digest = MessageHashUtils.toEthSignedMessageHash(
            keccak256(abi.encode(block.chainid, address(oracle), deadline, pusher, creator))
        );
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(pusherKey, digest);
        bytes memory sig = abi.encodePacked(r, s, v);

        address[] memory pushers = new address[](1);
        pushers[0] = pusher;
        bytes[] memory sigs = new bytes[](1);
        sigs[0] = sig;

        // Step 1: creator establishes delegation.
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);
        assertEq(oracle.namespaceRemapping(pusher), creator, "delegation established");

        // Step 2: pusher self-revokes (key compromised).
        vm.prank(pusher);
        oracle.revokePusher();
        assertEq(oracle.namespaceRemapping(pusher), address(0), "delegation cleared");

        // Step 3: creator replays the SAME signature — no nonce, no used-sig check.
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);

        // Delegation is silently restored; pusher's revocation is nullified.
        assertEq(
            oracle.namespaceRemapping(pusher),
            creator,
            "delegation re-established after revoke — attacker can now push bad prices"
        );
    }
}
```
