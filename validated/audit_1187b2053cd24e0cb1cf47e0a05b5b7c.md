### Title
`ProtectedPriceProviderL2` Lacks L2 Sequencer Uptime Check — Stale Prices Reach Pool Swaps After Sequencer Restart - (File: smart-contracts-poc/contracts/ProtectedPriceProviderL2.sol)

### Summary

`ProtectedPriceProviderL2` is the production L2 price provider deployed on Arbitrum, Base, Avalanche, and other L2 chains. It implements a `FUTURE_TOLERANCE` to handle sequencer clock skew but contains no check against a Chainlink sequencer uptime feed. When the L2 sequencer goes down and restarts, the last oracle price pushed before the outage can still pass the `MAX_TIME_DELTA` staleness check and be consumed by `MetricOmmPool.swap`, executing trades at a price that no longer reflects the market.

### Finding Description

`ProtectedPriceProviderL2._computeBidAsk` performs three guards before returning a bid/ask to the pool:

1. `_isStale(refTime, block.timestamp, MAX_TIME_DELTA, FUTURE_TOLERANCE)` — rejects prices older than `MAX_TIME_DELTA` or more than `FUTURE_TOLERANCE` ahead of `block.timestamp`.
2. `price == 0 || spread >= ORACLE_BPS` — rejects zero-price or off-hours marker.
3. `priceGuard(feedId)` — rejects prices outside a static admin-set band. [1](#0-0) 

None of these guards check whether the L2 sequencer is currently live or has recently restarted. The contract has no `sequencerUptimeFeed` field and no `GRACE_PERIOD` constant. [2](#0-1) 

By contrast, the registry ABI confirms that `ChainlinkVerifierL2` — a separate verifier used only by the Chainlink Data Streams oracle path — does carry `sequencerUptimeFeed` and `GRACE_PERIOD`. [3](#0-2) 

When `ProtectedPriceProviderL2` is wired to a Pyth-based `offchainOracle` (the primary non-Data-Streams path), no sequencer uptime check exists anywhere in the call chain: `pool.swap → ProtectedPriceProviderL2.getBidAndAskPrice → IPricedOracle.price(feedId, pool) → _computeBidAsk`. The `FUTURE_TOLERANCE` (up to 1 hour) was added to tolerate sequencer clock skew, but it does not substitute for an uptime check; it actually widens the acceptance window for prices published just before or during a sequencer outage. [4](#0-3) 

`PriceProviderL2` has the identical gap. [5](#0-4) 

### Impact Explanation

When the sequencer goes down, authorized oracle pushers cannot submit new price updates. When it restarts, the last price stored on-chain (from before the outage) is still present in oracle storage. If that price's `refTime` is within `MAX_TIME_DELTA` of the current `block.timestamp`, `_isStale` returns `false` and `_computeBidAsk` returns a bid/ask derived from the pre-outage price. A trader can immediately call `MetricOmmPool.swap` and execute at a price that may be hours stale, extracting value from LPs or the protocol at the expense of the pool's token reserves. This is a direct bad-price execution impact: the pool settles a swap at a quote that does not reflect current market conditions.

### Likelihood Explanation

L2 sequencer outages are documented historical events on Arbitrum and Base. The vulnerability window opens the moment the sequencer restarts and closes only when a pusher submits a fresh price update. During that window — which can last minutes to tens of minutes depending on pusher latency — any public caller can trigger a swap. No privileged access is required; the attacker only needs to monitor the sequencer status and submit a swap transaction immediately after restart.

### Recommendation

Add a sequencer uptime check inside `_computeBidAsk` (or a dedicated internal helper called before the staleness check), following the Chainlink-documented pattern:

```solidity
// Immutable set at construction:
AggregatorV3Interface public immutable sequencerUptimeFeed;
uint256 public constant GRACE_PERIOD = 3600; // 1 hour

function _checkSequencer() internal view {
    (, int256 answer, uint256 startedAt,,) = sequencerUptimeFeed.latestRoundData();
    // answer == 0 → sequencer is up; answer == 1 → sequencer is down
    if (answer != 0) return; // revert or return sentinel
    // Grace period: reject prices published within GRACE_PERIOD of restart
    if (block.timestamp - startedAt < GRACE_PERIOD) revert SequencerGracePeriod();
}
```

Call `_checkSequencer()` at the top of `_computeBidAsk` before the staleness check. If `sequencerUptimeFeed` is `address(0)` (L1 deployment), skip the check. Apply the same fix to `PriceProviderL2`.

### Proof of Concept

1. Deploy `ProtectedPriceProviderL2` on Arbitrum pointing to a Pyth-based oracle with `MAX_TIME_DELTA = 2 hours`.
2. Push a valid price at `T=0` (`refTime = T`).
3. Simulate sequencer downtime from `T+1` to `T+90min` — no new prices can be pushed.
4. At `T+90min` the sequencer restarts. `block.timestamp ≈ T+90min`. The stored price has `refTime = T`, age = 90 min < `MAX_TIME_DELTA` = 2 hours → `_isStale` returns `false`.
5. Call `MetricOmmPool.swap`. `ProtectedPriceProviderL2.getBidAndAskPrice` returns a bid/ask derived from the 90-minute-old price. The swap executes at the stale quote.
6. The real market price has moved during the 90-minute outage; the pool settles at the wrong price, causing LP loss. [6](#0-5)

    /// @dev Pure staleness check. L2-aware: tolerates oracle refTime slightly ahead of block.timestamp.
    function _isStale(
        uint256 refTime,
        uint256 nowTs,
        uint256 maxDelta,
        uint256 futureTol
    ) internal pure returns (bool) {
        if (refTime == 0) return true;

        if (refTime > nowTs) {
            // refTime in the future: tolerate only within futureTol
            return (refTime - nowTs) > futureTol;
        }

        // refTime in the past or equal: check age
        return (nowTs - refTime) > maxDelta;
    }

    function _getBidAndAskPrice() internal returns (uint128, uint128) {
        (uint256 mid, uint256 spread, , uint256 refTime) =
            IPricedOracle(address(offchainOracle)).price(offchainFeedId, msg.sender);
        return _computeBidAsk(mid, spread, refTime);
    }

    /// @dev Downstream pricing: staleness, price guard, confidence spread, marginStep.
    function _computeBidAsk(uint256 price, uint256 spread, uint256 refTime)
        internal view returns (uint128, uint128)
    {
        // 1. Staleness check
        if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA, FUTURE_TOLERANCE)) {
            return (0, type(uint128).max);
        }

        // 2. Basic validity — price must be positive, spread must not be stalled marker
        if (price == 0 || spread >= ORACLE_BPS) {
            return (0, type(uint128).max);
        }

        // 3. Price guard check
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(offchainFeedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (price < guardMin || price > guardMax) {
            return (0, type(uint128).max);
        }

        // 4. Compute bid/ask from mid + confidence-adjusted spread
        uint256 adjustedSpread = spread * confidenceParam;
        (uint256 bid, uint256 ask) = _getBidAskFrom(price, adjustedSpread);

        // 5. Apply marginStep adjustment
        (uint256 bidOut, bool bidOk) = _applyBidAdjustments(bid);
        if (!bidOk || bidOut > type(uint128).max) return (0, type(uint128).max);

        (uint256 askOut, bool askOk) = _applyAskAdjustments(ask);
        if (!askOk || askOut > type(uint128).max) return (0, type(uint128).max);

        // 6. Hard invariant: bid must be strictly less than ask.
        if (bidOut >= askOut) return (0, type(uint128).max);

        return (uint128(bidOut), uint128(askOut));
    }
