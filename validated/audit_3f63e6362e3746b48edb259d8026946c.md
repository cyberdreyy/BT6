### Title
Pusher Consent Signature Replay Allows Creator to Silently Re-establish Revoked Delegation Within Deadline Window — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1.allowPushers()` accepts a pusher's EIP-191 consent signature that commits to `(chainid, oracle, deadline, pusher, creator)`. There is no nonce and no used-signature tracking. A creator who holds a valid, unexpired signature can call `allowPushers` an unlimited number of times with the same signature, re-establishing delegation immediately after the pusher calls `revokePusher()`. The pusher's revocation is therefore not final until the deadline timestamp elapses.

---

### Finding Description

The consent message signed by the pusher is:

```
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [1](#0-0) 

The code's own NatSpec acknowledges the partial risk: *"the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it."* The deadline is offered as the mitigation. [2](#0-1) 

However, the deadline only bounds the outer window; it does not prevent the creator from replaying the same bytes within that window. `allowPushers` contains no mapping of consumed signatures, no nonce increment, and no check that `namespaceRemapping[pusher]` is currently `address(0)` before writing. [3](#0-2) 

`revokePusher` clears the mapping unconditionally: [4](#0-3) 

Because `allowPushers` does not check whether the pusher has already revoked, the creator can immediately overwrite the cleared mapping with the same signature.

The `fallback` push path resolves the namespace at call time from `namespaceRemapping[msg.sender]`: [5](#0-4) 

So every push the pusher sends after the creator's replay lands in the creator's namespace, not the pusher's own, exactly as if revocation had never happened.

---

### Impact Explanation

The pusher's price pushes continue to be written into the creator's namespace and are consumed by any pool registered against the creator's `feedId`. If the pusher is trying to stop feeding prices to the creator's namespace — for example, because the pusher's key is compromised, the pusher wants to redirect to their own namespace, or the pusher disagrees with the creator's pool configuration — the creator can silently nullify every revocation attempt until the deadline expires. Any pool that reads the creator's feed during this window receives prices the pusher did not intend to publish there, which is a bad-price execution path reaching live swaps.

---

### Likelihood Explanation

The trigger requires the creator to be adversarial toward their own pusher, which is a semi-trusted relationship. The creator already holds the signed bytes from the original `allowPushers` call and can replay them in a single transaction immediately after the pusher's `revokePusher` transaction is mined. No special tooling is needed. Deadlines in practice are set to hours or days (the test suite uses `block.timestamp + 1 days`), giving the creator a large replay window. [6](#0-5) 

---

### Recommendation

Track consumed consent signatures. Add a `mapping(bytes32 => bool) private _usedConsents` and mark the digest as used on first acceptance:

```solidity
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
);
require(!_usedConsents[hash], "consent already used");
require(pusher == ECDSA.recover(hash, signatures[i]));
_usedConsents[hash] = true;
namespaceRemapping[pusher] = msg.sender;
```

This makes each signed consent single-use, so a pusher's revocation is immediately and permanently effective regardless of the deadline.

---

### Proof of Concept

```solidity
// 1. Pusher signs consent with deadline = now + 1 day.
uint256 deadline = block.timestamp + 1 days;
bytes memory sig = _signConsent(PUSHER_KEY, deadline, pusher, creator);

// 2. Creator establishes delegation.
vm.prank(creator);
oracle.allowPushers(deadline, _arr(pusher), _arr(sig));
assertEq(oracle.namespaceRemapping(pusher), creator); // delegated

// 3. Pusher revokes.
vm.prank(pusher);
oracle.revokePusher();
assertEq(oracle.namespaceRemapping(pusher), address(0)); // cleared

// 4. Creator replays the SAME signature — still within deadline.
vm.prank(creator);
oracle.allowPushers(deadline, _arr(pusher), _arr(sig)); // succeeds, no revert
assertEq(oracle.namespaceRemapping(pusher), creator);   // delegation re-established

// 5. Pusher's next push lands in creator's namespace, not their own.
vm.prank(pusher);
(bool ok,) = address(oracle).call(_wordAt(slotId, pos, raw, tsMs));
assertTrue(ok);
// creator's feedId has the new price; pusher's own feedId is still zero.
assertGt(oracle.getOracleData(oracle.feedIdOf(creator, slotId, pos)).price, 0);
assertEq(oracle.getOracleData(oracle.feedIdOf(pusher,   slotId, pos)).price, 0);
```

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
```

    function revokePusher() external {
        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
        namespaceRemapping[msg.sender] = address(0);
        emit PusherRevoked(msg.sender, creator);
