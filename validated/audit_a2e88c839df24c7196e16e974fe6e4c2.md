### Title
Missing Sequencer Uptime Check in `PriceProviderL2` and `ProtectedPriceProviderL2` Allows Stale Prices During Sequencer Recovery — (`smart-contracts-poc/contracts/PriceProviderL2.sol`, `smart-contracts-poc/contracts/ProtectedPriceProviderL2.sol`)

---

### Summary

`PriceProviderL2` and `ProtectedPriceProviderL2` are the L2-specific price providers deployed on Arbitrum, Base, and other L2 networks. Neither contract checks the Arbitrum sequencer uptime feed before consuming oracle prices. During and immediately after a sequencer outage, the last pre-outage price remains stored in the oracle (e.g., `CompressedOracle`, `ChainlinkOracle`, `PythOracle`) and passes the existing `_isStale()` guard if the outage duration is shorter than `MAX_TIME_DELTA`. This allows a stale bid/ask quote to reach pool swaps during the sequencer recovery grace period, enabling bad-price execution.

---

### Finding Description

`PriceProviderL2._getBidAndAskPrice()` reads the oracle price and applies only a time-delta staleness check:

```solidity
// PriceProviderL2.sol lines 208–217
function _getBidAndAskPrice() internal returns (uint128, uint128) {
    (uint256 mid, uint256 spread, , uint256 refTime) =
        IPricedOracle(address(offchainOracle)).price(offchainFeedId, msg.sender);

    // 2. Staleness check
    if (_isStale(refTime, block.timestamp, MAX_TIME_DELTA, FUTURE_TOLERANCE)) {
        return (0, type(uint128).max);
    }
    ...
}
``` [1](#0-0) 

The `_isStale()` function only rejects prices older than `MAX_TIME_DELTA` (configurable up to 7 days):

```solidity
// PriceProviderL2.sol lines 135–150
function _isStale(
    uint256 refTime, uint256 nowTs, uint256 maxDelta, uint256 futureTol
) internal pure returns (bool) {
    if (refTime == 0) return true;
    if (refTime > nowTs) {
        return (refTime - nowTs) > futureTol;
    }
    return (nowTs - refTime) > maxDelta;
}
``` [2](#0-1) 

The `FUTURE_TOLERANCE` immutable is explicitly documented as handling L2 clock skew (oracle publication lag), **not** sequencer outages: [3](#0-2) 

The same pattern is present in `ProtectedPriceProviderL2._computeBidAsk()`: [4](#0-3) 

Neither contract's constructor accepts a `_sequencerUptimeFeed` parameter, and `PriceProviderFactoryL2.createPriceProvider()` deploys `PriceProviderL2` without one: [5](#0-4) 

Notably, the contract registry ABI (`registry.json`) exposes a `sequencerUptimeFeed()` getter and a `GRACE_PERIOD` constant on what appears to be a planned L2 provider variant, and a standalone `ChainlinkVerifierL2` contract with both fields — confirming the protocol was aware of the requirement but the current source does not implement it: [6](#0-5) 

The push-based oracles (`CompressedOracle`, `ChainlinkOracle`, `PythOracle`) all store the last verified price on-chain. During a sequencer outage, no new pushes can land. When the sequencer recovers, the stored price is the pre-outage value. If the outage lasted less than `MAX_TIME_DELTA`, `_isStale()` passes and the stale price is consumed by the pool.

---

### Impact Explanation

A stale bid/ask quote reaching `MetricOmmPool.swap()` during the sequencer recovery window constitutes **bad-price execution**: the pool's bin curve is priced against a pre-outage oracle value that may diverge significantly from the true market price. A trader can:

- Buy the underpriced asset at the stale ask (below true market), draining the pool of the base token at a discount.
- Sell the overpriced asset at the stale bid (above true market), receiving more quote token than the pool is owed.

LP principal is directly at risk. The pool's conservation invariant (trader receives no more than the oracle/bin curve permits) is broken for the duration of the grace period.

---

### Likelihood Explanation

Arbitrum sequencer outages have occurred historically. The attack window is the grace period immediately after recovery (Chainlink recommends 1 hour). An attacker monitoring the sequencer status can submit the first swap transaction the moment the sequencer resumes. No privileged access is required — `getBidAndAskPrice()` is called permissionlessly by any pool swap. The `MAX_TIME_DELTA` bound (up to 7 days) means even multi-hour outages leave the stale price valid.

---

### Recommendation

Add a sequencer uptime check in `PriceProviderL2` and `ProtectedPriceProviderL2` following the Chainlink L2 Sequencer Uptime Feeds pattern. Accept a `_sequencerUptimeFeed` address at construction (zero = disabled, for non-Arbitrum L2s) and a `GRACE_PERIOD` constant (e.g., 3600 seconds). In `_getBidAndAskPrice()` / `_computeBidAsk()`, before consuming the oracle price, call `sequencerUptimeFeed.latestRoundData()`, verify `answer == 0` (sequencer is up), and verify `block.timestamp - startedAt >= GRACE_PERIOD`. Revert or return the stale sentinel `(0, type(uint128).max)` on failure. `PriceProviderFactoryL2.createPriceProvider()` should forward the feed address to the constructor.

---

### Proof of Concept

1. Deploy `PriceProviderL2` on Arbitrum with `MAX_TIME_DELTA = 1 days`.
2. Pusher pushes a price of `P₀` at time `T₀` into `CompressedOracle`.
3. Arbitrum sequencer goes offline at `T₁ = T₀ + 1 hour`. No new pushes land.
4. True market price moves to `P₁ = 2 × P₀` during the outage.
5. Sequencer recovers at `T₂ = T₁ + 2 hours`. Stored oracle price is still `P₀`.
6. Attacker immediately calls `MetricOmmPool.swap()` (zero-to-one, buying base token).
7. `PriceProviderL2._getBidAndAskPrice()` reads `refTime = T₀`, `nowTs = T₂`. `nowTs - refTime = 3 hours < MAX_TIME_DELTA = 1 day` → `_isStale()` returns `false`.
8. Pool prices the swap at the stale `P₀` ask. Attacker receives base tokens at half the true market price.
9. Attacker immediately sells on a DEX at `P₁`, extracting the price difference from LP principal.
