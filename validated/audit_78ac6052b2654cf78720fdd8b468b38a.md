### Title
`revokePusher()` is silently nullified by signature replay within the deadline window — (`File: smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowPushers` accepts an EIP-191 signature that commits to `(chainid, oracle, deadline, pusher, creator)` but contains **no nonce and no used-signature bitmap**. Once a pusher signs and the creator calls `allowPushers`, the same bytes can be submitted again any number of times before `deadline` expires. A pusher who calls `revokePusher()` to stop a compromised key from writing bad prices can have their revocation immediately undone by the creator replaying the original signature, restoring `namespaceRemapping[pusher] = creator` and re-opening the push path.

---

### Finding Description

`allowPushers` builds the signed digest as:

```solidity
// CompressedOracle.sol lines 204-207
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
);
require(pusher == ECDSA.recover(hash, signatures[i]));
namespaceRemapping[pusher] = msg.sender;
``` [1](#0-0) 

The only replay guard is `_ensureDeadline(deadline)`, which rejects calls **after** the deadline but permits unlimited re-submissions **before** it. There is no nonce, no per-signature consumed flag, and no mapping that records whether a given `(pusher, creator, deadline)` tuple has already been used.

`revokePusher` zeroes the mapping:

```solidity
// CompressedOracle.sol lines 238-243
function revokePusher() external {
    address creator = namespaceRemapping[msg.sender];
    if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
    namespaceRemapping[msg.sender] = address(0);
    emit PusherRevoked(msg.sender, creator);
}
``` [2](#0-1) 

Because the original signature is still valid (deadline not yet elapsed), the creator can immediately call `allowPushers` again with the identical calldata, writing `namespaceRemapping[pusher] = creator` back. The revocation is silently undone. The code's own inline comment acknowledges the replay risk but incorrectly claims the deadline alone resolves it:

> *"an undated signature could re-establish a delegation AFTER the pusher revoked it"* [3](#0-2) 

The deadline prevents replay **after** expiry; it does nothing to prevent replay **before** expiry, which is exactly the window where revocation is needed.

---

### Impact Explanation

The `fallback()` push path resolves the effective namespace from `namespaceRemapping[msg.sender]`:

```solidity
// CompressedOracle.sol lines 315-316
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [4](#0-3) 

If a pusher's private key is compromised, the pusher calls `revokePusher()` to cut off the attacker's write access. The creator (or an automated keeper) replays the original `allowPushers` calldata, restoring the mapping. The attacker's key is again authorized to overwrite slot data — price, spread codebook indices, and timestamp — for every position in the creator's namespace. Those values flow directly into `getOracleData` → `AnchoredPriceProvider` → pool `getBidAsk`, producing bad bid/ask quotes that execute against real LP capital.

---

### Likelihood Explanation

- The creator holds the original signed bytes off-chain (they submitted them once already).
- Replaying requires a single transaction with no additional cost or privilege beyond being the creator.
- Deadlines are typically set days or weeks in the future (a short deadline defeats the usability purpose of delegation).
- The pusher has no on-chain mechanism to invalidate the signature before the deadline; `revokePusher` is the only tool available and it is fully bypassed.

---

### Recommendation

Add a per-pusher nonce or a consumed-signature set so each signed consent can be used exactly once:

```solidity
mapping(address => uint256) public pusherNonce; // incremented on each allowPushers call

// In allowPushers, include the current nonce in the digest and increment after use:
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender, pusherNonce[pusher]++))
```

Alternatively, track consumed digests:

```solidity
mapping(bytes32 => bool) private _usedDelegations;
// after recovery: require(!_usedDelegations[hash]); _usedDelegations[hash] = true;
```

Either approach ensures that once a pusher revokes via `revokePusher()`, the original signature cannot be replayed to undo the revocation.

---

### Proof of Concept

1. **Setup:** Creator `C` deploys a feed. Pusher `P` signs:
   ```
   sig = sign(keccak256(abi.encode(chainid, oracle, deadline=T+30days, P, C)))
   ```
   `C` calls `allowPushers(T+30days, [P], [sig])` → `namespaceRemapping[P] = C`.

2. **Compromise:** Attacker `A` obtains `P`'s private key and begins pushing manipulated prices into `C`'s slots via the `fallback()` path.

3. **Revocation attempt:** `P` calls `revokePusher()` → `namespaceRemapping[P] = address(0)`. Attacker `A` loses write access.

4. **Replay:** `C` (or an automated keeper) calls `allowPushers(T+30days, [P], [sig])` again with the **identical** calldata. `_ensureDeadline` passes (deadline is still 30 days away). Signature verification passes (same digest, same sig). `namespaceRemapping[P] = C` is written again.

5. **Continued attack:** `A` resumes pushing bad prices. Steps 3–4 can be repeated indefinitely until `T+30days` elapses, giving `A` an effectively unlimited window to feed corrupted bid/ask data into every pool that reads from `C`'s namespace via `AnchoredPriceProvider`. [5](#0-4) [6](#0-5)

    /// @notice Delegates pusher wallets into the caller's namespace. The pusher's EIP-191
    ///         signature is REQUIRED — without it anyone could remap a foreign pusher
    ///         wallet into their own namespace and silently swallow its pushes. The
    ///         deadline is likewise required: the signed consent carries no timestamp of
    ///         its own, so an undated signature could re-establish a delegation AFTER the
    ///         pusher revoked it.
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
