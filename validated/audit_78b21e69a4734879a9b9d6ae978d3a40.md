### Title
Pusher Delegation Signature Replay Bypasses `revokePusher()` Within Deadline Window — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`allowPushers` in `CompressedOracleV1` accepts a pusher's EIP-191 consent signature and binds it only by a deadline. No nonce or consumed-signature record exists. A creator who holds a valid, unexpired signature can call `allowPushers` an unlimited number of times with the same bytes, silently re-establishing a delegation that the pusher already revoked via `revokePusher()`. The pusher's revocation is therefore not final: the creator can restore write authority over the creator's namespace at will, without any fresh consent from the pusher, for the entire lifetime of the original deadline.

---

### Finding Description

`allowPushers` constructs the signed digest as:

```solidity
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
```

and then writes:

```solidity
namespaceRemapping[pusher] = msg.sender;
``` [1](#0-0) 

The only time-bound guard is `_ensureDeadline(deadline)`, which checks `block.timestamp <= deadline`. [2](#0-1) 

There is no nonce, no consumed-signature mapping, and no per-pusher revocation counter. After `revokePusher()` zeroes `namespaceRemapping[pusher]`:

```solidity
namespaceRemapping[msg.sender] = address(0);
``` [3](#0-2) 

the creator can immediately call `allowPushers` again with the identical `(deadline, pusher, signature)` tuple — the signature still verifies, the deadline still passes, and `namespaceRemapping[pusher]` is set back to the creator. The pusher's revocation is silently undone.

The code comment itself acknowledges the deadline is the only replay barrier:

> "The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it." [4](#0-3) 

But the deadline only prevents replay *after* expiry. Within the window it provides zero protection against repeated use of the same bytes.

---

### Impact Explanation

`namespaceRemapping` controls where every fallback-path price push lands. When a pusher is delegated, their `fallback()` writes go into the creator's namespace, which is the namespace a pool's `AnchoredPriceProvider` reads via `feedIdOf(creator, slotIndex, positionIndex)`. [5](#0-4) 

If a pusher revokes because their feed is compromised, they are going offline, or they detected a pricing anomaly — and the creator immediately replays the old signature — the pusher's subsequent pushes (including stale or wrong prices from an automated bot that has not yet halted) continue to land in the creator's namespace and reach any pool anchored to that feed. This is a bad-price execution path: the pusher believed they had withdrawn consent and stopped serving the pool, but the pool continues to consume their data.

The `AnchoredPriceProvider` reads the compressed oracle's `price(feedId, pool)` at swap time and clamps the result to a band, but the band is derived from the same compromised feed's own spread values — it does not independently validate the mid price source. [6](#0-5) 

---

### Likelihood Explanation

- The creator must have retained the original `(deadline, signature)` pair — trivially true since it was submitted on-chain and is visible in calldata.
- The deadline must not have expired. Deadlines are chosen by the creator at the time of the original `allowPushers` call; a creator who anticipates needing to override a revocation will simply set a long deadline (days or weeks).
- The pusher must continue pushing after revoking (e.g., an automated bot that has not yet been halted). This is the normal operational pattern for off-chain oracle bots.
- No privileged role or special setup is required beyond being the creator who originally called `allowPushers`.

---

### Recommendation

Mark each signature as consumed after first use. The simplest approach is a `mapping(bytes32 => bool) private _usedSignatures` keyed on the digest, set to `true` on first acceptance and checked before writing `namespaceRemapping`. Alternatively, adopt OpenZeppelin's `Nonces` contract and include a per-pusher nonce in the signed payload, incrementing it on each successful `allowPushers` call so that `revokePusher()` implicitly invalidates all prior signatures by making their nonce stale.

---

### Proof of Concept

```solidity
// 1. Pusher signs consent for creator with deadline = now + 7 days.
bytes memory sig = pusher.sign(
    keccak256(abi.encode(block.chainid, address(oracle), deadline, pusherAddr, creatorAddr))
);

// 2. Creator establishes delegation.
oracle.allowPushers(deadline, [pusherAddr], [sig]);
// namespaceRemapping[pusher] == creator ✓

// 3. Pusher revokes — intends to stop serving creator's pool.
vm.prank(pusherAddr);
oracle.revokePusher();
// namespaceRemapping[pusher] == address(0) ✓

// 4. Creator replays the IDENTICAL (deadline, sig) — no new consent from pusher.
vm.prank(creatorAddr);
oracle.allowPushers(deadline, [pusherAddr], [sig]);
// namespaceRemapping[pusher] == creator again — revocation bypassed ✓

// 5. Pusher bot continues pushing prices (thinking it's in its own namespace).
//    Prices land in creator's namespace and feed the pool at swap time.
vm.prank(pusherAddr);
(bool ok,) = address(oracle).call(wordAt(slotId, pos, badPrice, tsMs));
assertTrue(ok);
// oracle.getOracleData(oracle.feedIdOf(creator, slotId, pos)).price == badPrice ✓

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
