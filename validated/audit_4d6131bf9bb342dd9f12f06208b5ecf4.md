### Title
Creator Can Replay Pusher EIP-191 Consent Signature After Self-Revocation to Silently Re-Establish Delegation and Capture Unauthorized Price Pushes — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1.allowPushers()` signs pusher consent as `(chainid, address(this), deadline, pusher, creator)` with no nonce or revocation counter. After a pusher calls `revokePusher()` to exit a creator's namespace, the creator can immediately replay the identical signature — before the deadline expires — to silently re-establish the delegation. The pusher's self-revocation is therefore not permanent until the deadline lapses. Any price data the pusher subsequently pushes (believing it targets their own namespace) is instead written into the creator's namespace and consumed by pools registered against that namespace.

---

### Finding Description

`allowPushers` establishes `namespaceRemapping[pusher] = creator` after verifying an EIP-191 signature:

```solidity
// CompressedOracle.sol L204-L210
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
);
require(pusher == ECDSA.recover(hash, signatures[i]));
namespaceRemapping[pusher] = msg.sender;
``` [1](#0-0) 

The signed tuple contains no nonce, no revocation epoch, and no "single-use" marker. The only replay guard is the deadline, which the code's own NatSpec acknowledges:

> *"the deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it."* [2](#0-1) 

`revokePusher()` zeroes the mapping:

```solidity
// CompressedOracle.sol L238-L243
function revokePusher() external {
    address creator = namespaceRemapping[msg.sender];
    if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
    namespaceRemapping[msg.sender] = address(0);
    emit PusherRevoked(msg.sender, creator);
}
``` [3](#0-2) 

But because the signature is replayable, the creator can call `allowPushers` again with the same `(deadline, pusher, signature)` tuple — as long as `block.timestamp <= deadline` — and the mapping is restored to `creator`. The deadline prevents replay only after it expires; it does not prevent replay between revocation and expiry.

The `fallback()` push path reads the mapping on every call:

```solidity
// CompressedOracle.sol L315-L316
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [4](#0-3) 

After the silent re-establishment, every subsequent push from the pusher's wallet is written into the creator's storage slot, not the pusher's own namespace. The pusher has no on-chain signal that the delegation was restored.

---

### Impact Explanation

Pools consume prices from a creator's namespace via a `PriceProvider` that calls `price(feedId, pool)`. The `feedId` encodes the creator address, slot index, and position index:

```solidity
// CompressedOracle.sol L49-L52
function feedIdOf(address creator, uint8 slotIndex, uint8 positionIndex) public view returns (bytes32) {
    return bytes32(
        uint256(uint160(creator)) << 96 | block.chainid << 16 | uint256(slotIndex) << 8 | positionIndex
    );
}
``` [5](#0-4) 

Once the delegation is silently re-established, the pusher's future updates — which the pusher believes target their own namespace — overwrite the creator's slot. If the pusher is now pushing prices for a different asset, a different scale, or simply stale data (because they believe they are no longer responsible for the creator's feed), those values reach the pool's swap path through the `PriceProvider` staleness and spread pipeline. This constitutes bad-price execution: the pool receives a price that was never intended for it, potentially causing traders to receive more output than the oracle curve permits or LPs to suffer losses from mispriced swaps.

---

### Likelihood Explanation

The attack requires:
1. A pusher who signed consent with a deadline far in the future (common for long-running bots).
2. The pusher calling `revokePusher()` to exit the creator's namespace.
3. The creator replaying the original signature before the deadline.
4. The pusher continuing to push (e.g., an automated bot that does not monitor `namespaceRemapping` state).

Steps 1, 3, and 4 are realistic for production bot operators. The creator is a semi-trusted but potentially adversarial party (they control the namespace and the pool). No privileged role is required beyond holding the original signature, which the creator already possesses from the initial `allowPushers` call.

---

### Recommendation

Include a per-pusher revocation nonce in the signed message so that a revocation permanently invalidates all prior signatures:

```solidity
mapping(address => uint256) public pusherNonce; // incremented on every revokePusher / removePushers

// In allowPushers:
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender, pusherNonce[pusher]))

// In revokePusher / removePushers:
pusherNonce[pusher]++;
namespaceRemapping[pusher] = address(0);
```

Alternatively, record a `revokedAt` timestamp per pusher and reject any `allowPushers` call that uses a signature whose deadline predates `revokedAt`.

### Proof of Concept

```
T=0   Pusher signs: hash(chainid, oracle, deadline=T+365d, pusher, creator)
T=1   Creator calls allowPushers(deadline, [pusher], [sig])
        → namespaceRemapping[pusher] = creator  ✓

T=5   Pusher calls revokePusher()
        → namespaceRemapping[pusher] = address(0)
        → Pusher believes they now push into their own namespace

T=6   Creator calls allowPushers(deadline, [pusher], [sig])  ← SAME sig, deadline still valid
        → namespaceRemapping[pusher] = creator  (silently restored)

T=7   Pusher's bot pushes a price update (intended for pusher's own namespace)
        → fallback(): creator = namespaceRemapping[pusher] = creator  ← NOT address(0)
        → Price written to creator's slot

T=7   Pool registered on creator's feedId reads price via PriceProvider
        → Receives pusher's unintended price
        → Swap executes at bad-price

    /// @notice Delegates pusher wallets into the caller's namespace. The pusher's EIP-191
    ///         signature is REQUIRED — without it anyone could remap a foreign pusher
    ///         wallet into their own namespace and silently swallow its pushes. The
    ///         deadline is likewise required: the signed consent carries no timestamp of
    ///         its own, so an undated signature could re-establish a delegation AFTER the
    ///         pusher revoked it.
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }

    fallback() override external {
        uint256 end;
        uint256 namespace;

        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0)) creator = msg.sender;

        assembly ("memory-safe") {
            end := calldatasize()
            namespace := shl(96, creator) // [creator:20][zeros:12]
        }

        // 4 * 6 + 7 + 1 = 32 bytes per slot
        if (end == 0 || end % 32 != 0) revert BadCalldataLength();

        for (uint256 ptr = 0; ptr < end; ptr += 32) {
            uint256 word;
            assembly ("memory-safe") {
                word := calldataload(ptr)
            }
            // casting to 'uint8' is safe we want LSB
            // forge-lint: disable-next-line(unsafe-typecast)
            uint8 slotId = uint8(word);
            TimeMs timestampMs = toTimeMs(word >> 8 & X56);
            timestampMs.revertIfAfterBlockTimeWithDrift(MAX_TIME_DRIFT);
            bytes32 key = bytes32(namespace | uint256(slotId));
            uint256 old = uint256(_loadStorage(key));
            TimeMs oldTimestampMs = toTimeMs(old >> 8 & X56);

            bool newer = timestampMs.isAfter(oldTimestampMs);
            if (!newer) continue;

            _writeStorage(key, bytes32(bytes32(word & ~uint256(0xff))));
        }
