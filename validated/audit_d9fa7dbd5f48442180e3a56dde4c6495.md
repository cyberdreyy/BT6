### Title
`updateBySignature` deadline omitted from signed message hash enables stale-price injection — (File: `smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

The `updateBySignature` function's EIP-191 signed message hash does not include the `deadline` parameter. Because `deadline` is checked against `block.timestamp` but is not cryptographically bound to the signature, any party who obtains a signed slot word can replay it with an arbitrary future deadline, bypassing the pusher's intended expiry and injecting a stale price into pools.

### Finding Description

`allowPushers` correctly binds `deadline` into its signed digest:

```solidity
// CompressedOracle.sol lines 204-207
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
);
```
