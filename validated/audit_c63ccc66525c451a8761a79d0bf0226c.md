### Title
`notionalFeeToken0Scaled`/`notionalFeeToken1Scaled` fully drained to protocol when `notionalSumE8 == 0` but `spreadSumE6 > 0` — (`metric-core/contracts/MetricOmmPool.sol`)

---

### Summary

In `MetricOmmPool.collectFees`, when the factory calls with `protocolNotionalFeeE8_ == 0` and `adminNotionalFeeE8_ == 0` (i.e. `notionalSumE8 == 0`) but with a non-zero spread fee split, the entire accumulated `notionalFeeToken0Scaled` / `notionalFeeToken1Scaled` is silently forwarded to the factory as protocol fees and then zeroed out. The admin's owed share of notional fees is permanently destroyed.

---

### Finding Description

The fee-split arithmetic for notional fees is:

```solidity
uint256 notionalFee0ToAdminScaled =
    notionalSumE8 == 0 ? 0 : (notionalFee0AmountScaled * adminNotionalFeeE8_) / notionalSumE8;

uint256 notionalFee0ToProtocolScaled = notionalFee0AmountScaled - notionalFee0ToAdminScaled;
```

When `notionalSumE8 == 0`, the ternary correctly sets `notionalFee0ToAdminScaled = 0`, but the next line unconditionally computes:

```
notionalFee0ToProtocolScaled = notionalFee0AmountScaled − 0 = notionalFee0AmountScaled
```

The full accumulated notional balance is then added to `totalFee0ToProtocolScaled` and transferred to `FACTORY`, after which both accumulators are zeroed:

```solidity
notionalFeeToken0Scaled = 0;
notionalFeeToken1Scaled = 0;
```

This is the direct analog of the external bug: the accumulator is consumed and reset in one branch (`notionalSumE8 > 0`) but is **also** consumed and reset — incorrectly — in the other branch (`notionalSumE8 == 0, spreadSumE6 > 0`), where it should have been left untouched for a future collection call.

The early-return guard only fires when **both** sums are zero:

```solidity
if (spreadSumE6 == 0 && notionalSumE8 == 0) {
    return;
}
```

So a call with `spreadSumE6 > 0, notionalSumE8 == 0` bypasses the guard, enters the distribution block, and silently drains the notional accumulator.

---

### Impact Explanation

- **Admin fee loss**: Every unit of `notionalFeeToken0Scaled` / `notionalFeeToken1Scaled` accumulated since the last `collectFees` call is sent to the factory (protocol) instead of being preserved for the admin's share. The admin receives zero notional fees for that entire period.
- **Permanent**: After the reset, future `collectFees` calls see `notionalFeeToken0Scaled == 0` and distribute nothing, so the loss is not recoverable.
- **Protocol over-payment**: The factory receives notional fees it did not intend to claim in that call, breaking the intended protocol/admin split.

This constitutes a direct loss of owed protocol fees (admin's portion) — matching the "pool insolvency / owed fees" impact gate.

---

### Likelihood Explanation

The factory is the only caller (`onlyFactory`). A legitimate scenario is the factory collecting spread fees on a regular cadence while deferring notional fee collection to a separate call (e.g. different cadence, different admin destination). Any such call with `protocolNotionalFeeE8_ == 0, adminNotionalFeeE8_ == 0` triggers the bug. Likelihood is low-to-medium depending on factory operational patterns, but the trigger is a valid, non-malicious factory action.

---

### Recommendation

Mirror the guard used for spread fees: skip notional distribution entirely when `notionalSumE8 == 0`, and only zero the accumulators when notional fees are actually distributed.

```solidity
uint256 notionalFee0ToAdminScaled =
    notionalSumE8 == 0 ? 0 : (notionalFee0AmountScaled * adminNotionalFeeE8_) / notionalSumE8;
uint256 notionalFee1ToAdminScaled =
    notionalSumE8 == 0 ? 0 : (notionalFee1AmountScaled * adminNotionalFeeE8_) / notionalSumE8;

// FIX: guard the protocol side the same way
uint256 notionalFee0ToProtocolScaled =
    notionalSumE8 == 0 ? 0 : notionalFee0AmountScaled - notionalFee0ToAdminScaled;
uint256 notionalFee1ToProtocolScaled =
    notionalSumE8 == 0 ? 0 : notionalFee1AmountScaled - notionalFee1ToAdminScaled;

// ... transfers ...

// FIX: only reset when fees were actually distributed
if (notionalSumE8 > 0) {
    notionalFeeToken0Scaled = 0;
    notionalFeeToken1Scaled = 0;
}
```

### Proof of Concept

1. Swaps accumulate `notionalFeeToken0Scaled = N` (N > 0).
2. Factory calls `collectFees(protocolSpreadFeeE6_ > 0, adminSpreadFeeE6_ > 0, 0, 0, adminDest)` — intending to collect only spread fees.
3. `notionalSumE8 = 0 + 0 = 0` → early-return guard **not** triggered.
4. `notionalFee0ToAdminScaled = 0` (ternary).
5. `notionalFee0ToProtocolScaled = N − 0 = N` → entire notional balance forwarded to `FACTORY`.
6. `notionalFeeToken0Scaled = 0` — accumulator wiped.
7. Admin's owed notional fees are permanently lost; factory received N tokens it did not intend to claim. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)
