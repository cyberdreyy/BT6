### Title
`allowPushers` Signature Replay Within Deadline Window Permanently Nullifies Pusher Self-Revocation — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

The `allowPushers` delegation path in `CompressedOracle.sol` uses an EIP-191 signature that commits to a `deadline` but carries no nonce. The code comment explicitly states the deadline is the mechanism that prevents a creator from re-establishing delegation after a pusher revokes it. That claim is false: within the deadline window the identical signature is unconditionally replayable, so a creator can nullify every `revokePusher()` call the pusher makes until the deadline expires. A pusher who signed a long-lived consent (e.g. 30 days) cannot reclaim their own namespace for the full duration, causing their own-namespace feeds to go stale and breaking swaps on any pool that reads those feeds.

---

### Finding Description

`allowPushers` builds its signed digest as:

```solidity
// CompressedOracle.sol L204-207
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
);
require(pusher == ECDSA.recover(hash, signatures[i]));
namespaceRemapping[pusher] = msg.sender;
```

The tuple `(chainid, oracle, deadline, pusher, creator)` is fixed for the lifetime of the deadline. There is no nonce, no "used" flag, and no on-chain record that the signature was already consumed.

The code comment at L186-191 acknowledges the problem and claims the deadline solves it:

> *"The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it."*

The comment is incorrect. The deadline only blocks replay **after** `block.timestamp > deadline`. While `block.timestamp ≤ deadline`, `_ensureDeadline` passes and the same bytes can be submitted again and again.

`revokePusher()` clears the mapping:

```solidity
// CompressedOracle.sol L238-243
function revokePusher() external {
    address creator = namespaceRemapping[msg.sender];
    if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
    namespaceRemapping[msg.sender] = address(0);
    emit PusherRevoked(msg.sender, creator);
}
```

But the creator can immediately call `allowPushers` with the original signature to write `namespaceRemapping[pusher] = creator` again. The pusher's revocation is atomically undone in the next block (or even in the same block via a bundle).

---

### Impact Explanation

Once the delegation is re-established against the pusher's will:

1. Every fallback push the pusher sends lands in the **creator's namespace** (not the pusher's own), because the fallback resolves `namespaceRemapping[msg.sender]` first.
2. The pusher's only escape is to stop pushing entirely.
3. Stopping pushing leaves the pusher's own-namespace feeds at their last stored value. The `_price` path returns `refTime = data.timestampMs.toSeconds()`, which is now stale.
4. `PriceProvider._getBidAndAskPrice` calls `_isStale(refTime, block.timestamp, MAX_TIME_DELTA)` and returns `(0, type(uint128).max)` — the stalled sentinel — which causes `getBidAndAskPrice` to revert with `FeedStalled`.
5. Any pool whose price provider reads from the pusher's namespace can no longer execute swaps: **core pool functionality is broken**.

The corrupted value is the `namespaceRemapping` entry that should be `address(0)` after revocation but is forced back to `creator`, redirecting all subsequent slot writes and leaving the pusher's own feeds permanently stale.

---

### Likelihood Explanation

- **Trigger**: any creator who holds a valid (non-expired) pusher signature — a normal operational artifact.
- **Privilege required**: none beyond being the creator who originally called `allowPushers`; this is a valid semi-trusted role, not a privileged admin.
- **Attacker cost**: a single on-chain transaction replaying the already-known signature bytes.
- **Window**: as long as the deadline the pusher signed. Pushers are incentivised to sign long deadlines to avoid frequent re-authorisation, making the window practically wide.

---

### Recommendation

Introduce a per-pusher nonce into the signed digest and increment it on every successful `allowPushers` call. Record the consumed nonce so the same signature cannot be submitted twice:

```solidity
mapping(address => uint256) public pusherNonce;

// In allowPushers:
uint256 nonce = pusherNonce[pusher]++;
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(this), deadline, nonce, pusher, msg.sender))
);
```

Alternatively, record a `bytes32` bitmap of consumed signature hashes so that each unique `(deadline, pusher, creator)` tuple can only be accepted once.

---

### Proof of Concept

```
State: block.timestamp = T, deadline = T + 30 days

Step 1 — Pusher signs off-chain:
  sig = sign(keccak256(abi.encode(chainid, oracle, T+30days, pusher, creator)))

Step 2 — Creator establishes delegation:
  oracle.allowPushers(T+30days, [pusher], [sig])
  → namespaceRemapping[pusher] == creator  ✓

Step 3 — Pusher revokes:
  oracle.revokePusher()   (called by pusher)
  → namespaceRemapping[pusher] == address(0)  ✓

Step 4 — Creator replays the SAME sig (deadline still valid):
  oracle.allowPushers(T+30days, [pusher], [sig])
  → namespaceRemapping[pusher] == creator  ← revocation nullified

Step 5 — Pusher stops pushing to avoid feeding creator's namespace.
  → pusher's own-namespace feeds go stale (timestampMs frozen at last push)

Step 6 — Pool swap calls PriceProvider.getBidAndAskPrice()
  → _isStale(frozenRefTime, now, MAX_TIME_DELTA) == true
  → returns (0, type(uint128).max)
  → getBidAndAskPrice() reverts FeedStalled
  → all swaps on the pool revert  ← broken core functionality

    ///         signature is REQUIRED — without it anyone could remap a foreign pusher
    ///         wallet into their own namespace and silently swallow its pushes. The
    ///         deadline is likewise required: the signed consent carries no timestamp of
    ///         its own, so an undated signature could re-establish a delegation AFTER the
    ///         pusher revoked it.
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
