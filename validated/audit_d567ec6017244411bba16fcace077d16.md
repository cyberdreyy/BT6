### Title
Silent Namespace Hijack via `allowContractPushers` Overwrites Existing Delegation Without Original Creator's Consent — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`allowContractPushers` allows any caller whose address is approved by a contract pusher's `isPusher()` to overwrite an existing `namespaceRemapping` entry. Unlike the EOA path (`allowPushers`), which binds the pusher's signature to a specific creator address, the contract-pusher path has no such binding. A third-party creator can silently redirect a contract pusher's writes from the original creator's namespace to their own, causing the original feed to go stale and delivering bad prices to pools.

### Finding Description

`allowPushers` (EOA path) signs the consent over `(chainid, oracle_address, deadline, pusher, msg.sender)`: [1](#0-0) 

The `msg.sender` (the creator) is locked into the signed digest. A pusher's signature for creator A cannot be replayed to delegate that pusher to creator B — the recovered signer would not match.

`allowContractPushers` (contract path) only calls `isPusher(msg.sender)` on the pusher contract: [2](#0-1) 

There is **no check** that `namespaceRemapping[pusher]` is currently `address(0)` or already equals `msg.sender`. If a contract pusher returns `true` for multiple creators (e.g., a shared oracle aggregator, a DAO-controlled pusher, or any contract whose `isPusher` is not strictly single-creator), any approved creator can call `allowContractPushers` and atomically overwrite the existing delegation — without the original creator's knowledge or consent.

This is the direct analog to the BridgeRoles inconsistency: `allowPushers` protects against re-delegation to a different creator (signature binds to specific creator), while `allowContractPushers` does not.

### Impact Explanation

Once `namespaceRemapping[pusher]` is overwritten from `creatorA` to `creatorB`, every subsequent fallback push from that contract pusher lands in `creatorB`'s namespace. `creatorA`'s feed stops receiving updates. Any `AnchoredPriceProvider` or pool that reads `creatorA`'s feed will consume a stale price. This satisfies the "Bad-price execution: stale bid/ask quote reaches a pool swap" impact gate.

### Likelihood Explanation

The attack requires the contract pusher to return `true` for the attacker's address. This is realistic for:
- Shared oracle aggregators authorized for multiple creators
- DAO-controlled pushers where multiple members are approved
- Any pusher contract whose `isPusher` is not strictly single-creator

The `allowContractPushers` function is public and permissionless; no privileged setup is required beyond being an approved caller of the target pusher contract.

### Recommendation

Add a guard that prevents overwriting an existing delegation without the original creator's consent:

```solidity
function allowContractPushers(address[] calldata pushers) external {
    uint256 l = pushers.length;
    for (uint256 i; i < l; i++) {
        address pusher = pushers[i];
        if (pusher == msg.sender) revert NoSelfRemapping();

        // Prevent silent overwrite of an existing delegation
        address existing = namespaceRemapping[pusher];
        require(existing == address(0) || existing == msg.sender, AlreadyDelegated(pusher, existing));

        (bool ok, bytes memory res) = pusher.staticcall(
            abi.encodeWithSignature("isPusher(address)", msg.sender)
        );
        require(ok);
        require(abi.decode(res, (bool)));

        namespaceRemapping[pusher] = msg.sender;
        emit PusherAuthorized(pusher, msg.sender);
    }
}
```

Alternatively, align the contract-pusher path with the EOA path by requiring the pusher contract to also encode the intended creator in its consent response, preventing approval for multiple creators simultaneously.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

// Pusher contract that approves ANY caller (realistic for shared aggregators)
contract SharedPusher {
    function isPusher(address) external pure returns (bool) { return true; }
    fallback() external {}
}

contract PoC {
    function run(CompressedOracleV1 oracle) external {
        SharedPusher pusher = new SharedPusher();
        address creatorA = address(0xAAAA);
        address creatorB = address(0xBBBB);

        // Step 1: creatorA legitimately delegates the shared pusher
        address[] memory p = new address[](1);
        p[0] = address(pusher);
        vm.prank(creatorA);
        oracle.allowContractPushers(p);
        // namespaceRemapping[pusher] == creatorA ✓

        // Step 2: creatorB silently overwrites the delegation — no consent from creatorA
        vm.prank(creatorB);
        oracle.allowContractPushers(p);
        // namespaceRemapping[pusher] == creatorB ✓ (creatorA's feed is now orphaned)

        // Step 3: pusher's next update lands in creatorB's namespace
        // creatorA's feed timestamp stays frozen → stale price → bad-price execution in pools
        assert(oracle.namespaceRemapping(address(pusher)) == creatorB);
    }
}
