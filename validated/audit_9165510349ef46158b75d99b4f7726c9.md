### Title
Pool factory accepts arbitrary `IPriceProvider` implementations without verifying factory origin, enabling bad-price execution — (`metric-core/contracts/MetricOmmPoolFactory.sol`)

---

### Summary

`MetricOmmPoolFactory._validatePriceProvider()` only checks that `token0()`/`token1()` match the pool's token pair. It never verifies that the provider was deployed by a recognized factory (`PriceProviderFactory`, `PriceProviderFactoryL2`, or `AnchoredProviderFactory`). Any pool creator can supply a hand-crafted contract that satisfies the two-token check while returning fully arbitrary bid/ask prices, causing every swap in that pool to execute at a manipulated price.

---

### Finding Description

`_validatePriceProvider` is the single gate used both at pool creation and at every price-provider update:

```solidity
// metric-core/contracts/MetricOmmPoolFactory.sol
function _validatePriceProvider(address token0, address token1, address priceProvider) internal view {
    if (priceProvider == address(0)) revert InvalidPriceProvider();
    if (IPriceProvider(priceProvider).token0() != token0 || IPriceProvider(priceProvider).token1() != token1) {
        revert PriceProviderTokenMismatch();
    }
}
``` [1](#0-0) 

It is called unconditionally at `createPool` and at both steps of the mutable-provider update flow: [2](#0-1) [3](#0-2) [4](#0-3) 

The `IPriceProvider` interface requires only three functions — `token0()`, `token1()`, and `getBidAndAskPrice()`: [5](#0-4) 

A fake contract that hard-codes the correct token addresses for `token0()`/`token1()` and returns attacker-chosen values from `getBidAndAskPrice()` passes `_validatePriceProvider` without any further check.

The protocol ships two factory contracts whose entire purpose is to attest that a provider was constructed with safe parameters. `AnchoredProviderFactory` even documents this explicitly:

> *"public-pool eligibility is then the machine-checkable predicate `recognizedFactory.isProvider(p)`"* [6](#0-5) 

`PriceProviderFactory.isProvider()` and `AnchoredProviderFactory.isProvider()` both exist and are queryable on-chain, but `_validatePriceProvider` never calls either of them. [7](#0-6) [8](#0-7) 

---

### Impact Explanation

Pool creation is permissionless — any address can call `createPool` and become the pool admin. A malicious pool admin supplies a fake `IPriceProvider` at creation time (no timelock applies to the initial provider). The fake provider returns an arbitrarily low ask price or arbitrarily high bid price. Every swap in the pool executes against that manipulated quote: buyers receive token1 at a fraction of fair value, draining token0 reserves; sellers receive token0 at an inflated rate, draining token1 reserves. LP shares become under-collateralized and cannot be fully redeemed. This is a direct, quantifiable loss of LP principal — matching the "bad-price execution" and "pool insolvency" impact categories.

---

### Likelihood Explanation

Pool creation is fully permissionless. The attacker needs only to deploy a two-function fake provider and call `createPool` with it. No special role, no admin approval, and no timelock is required for the initial provider. The attack is repeatable across any token pair. Likelihood is **medium-high**: the barrier is a single contract deployment and one factory call.

---

### Recommendation

Add a recognized-factory membership check inside `_validatePriceProvider`. The factory contract should store an admin-curated set of approved provider factories and verify membership before accepting a provider:

```solidity
// In MetricOmmPoolFactory storage:
EnumerableSet.AddressSet private _approvedProviderFactories;

function _validatePriceProvider(address token0, address token1, address priceProvider) internal view {
    if (priceProvider == address(0)) revert InvalidPriceProvider();
    if (IPriceProvider(priceProvider).token0() != token0 || IPriceProvider(priceProvider).token1() != token1) {
        revert PriceProviderTokenMismatch();
    }
    // NEW: require the provider was deployed by a recognized factory
    bool recognized = false;
    uint256 len = _approvedProviderFactories.length();
    for (uint256 i; i < len; ++i) {
        if (IProviderFactory(_approvedProviderFactories.at(i)).isProvider(priceProvider)) {
            recognized = true;
            break;
        }
    }
    if (!recognized) revert ProviderNotFromApprovedFactory();
}
```

Alternatively, store a `mapping(address provider => bool)` populated by the approved factories at deploy time, or require the provider to expose an immutable `factory` field that is checked against the approved set (both `PriceProvider` and `AnchoredPriceProvider` already store `factory` as an immutable).

### Proof of Concept

```solidity
// FakeProvider.sol — passes _validatePriceProvider, returns attacker-chosen prices
contract FakeProvider {
    address public immutable token0;
    address public immutable token1;
    uint128 public bid;
    uint128 public ask;

    constructor(address _t0, address _t1, uint128 _bid, uint128 _ask) {
        token0 = _t0; token1 = _t1; bid = _bid; ask = _ask;
    }

    function getBidAndAskPrice() external view returns (uint128, uint128) {
        return (bid, ask);  // e.g. bid=1, ask=2 when fair price is 1e18
    }
}

// Attack
FakeProvider fake = new FakeProvider(address(token0), address(token1), 1, 2);

PoolParameters memory params = /* normal params */;
params.priceProvider = address(fake);   // passes token0/token1 check
address pool = factory.createPool(params);

// LPs add liquidity at "fair" initial price set by curBinDistFromProvidedPriceE6
// Attacker swaps: pool quotes bid=1/ask=2 instead of fair ~1e18
// Attacker drains reserves at near-zero cost; LP shares become worthless
```
