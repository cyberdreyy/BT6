### Title
Revoked pusher delegation can be silently re-established via signature replay, allowing bad prices to reach pool swaps — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`revokePusher()` sets `namespaceRemapping[msg.sender] = address(0)` but does not invalidate the pusher's original EIP-191 consent signature. Because `allowPushers()` has no nonce or used-signature registry, the creator can replay the exact same signature (before its deadline) to silently re-establish the delegation. A pusher who revokes to stop a compromised key from writing bad prices achieves nothing if the creator replays the old credential.

---

### Finding Description

**Root cause — `revokePusher` does not update the signature-validity state**

`revokePusher` only zeroes the mapping entry:

```solidity
// CompressedOracle.sol L238-243
function revokePusher() external {
    address creator = namespaceRemapping[msg.sender];
    if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
    namespaceRemapping[msg.sender] = address(0);
    emit PusherRevoked(msg.sender, creator);
}
``` [1](#0-0) 

`allowPushers` checks only `block.timestamp <= deadline` and the ECDSA signature. There is no nonce, no used-signature bitmap, and no check that the pusher was not previously revoked:

```solidity
// CompressedOracle.sol L192-212
function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
    _ensureDeadline(deadline);
    ...
    bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
        keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
    );
    require(pusher == ECDSA.recover(hash, signatures[i]));
    namespaceRemapping[pusher] = msg.sender;   // re-establishes delegation
``` [2](#0-1) 

The protocol's own documentation acknowledges the gap:

> *"the signed consent has no data timestamp, so an undated signature could re-establish a delegation after the pusher revoked it."* [3](#0-2) 

**Bad-price write path after re-delegation**

Once `namespaceRemapping[pusher] = creator` is restored, the `fallback()` push path resolves the namespace from the mapping and writes directly into the creator's storage slot:

```solidity
// CompressedOracle.sol L315-343
fallback() override external {
    address creator = namespaceRemapping[msg.sender];
    if (creator == address(0)) creator = msg.sender;
    ...
    _writeStorage(key, bytes32(bytes32(word & ~uint256(0xff))));
}
``` [4](#0-3) 

The only freshness gate is per-slot timestamp monotonicity (`isAfter`). An attacker holding the compromised pusher key can craft a slot word with a timestamp strictly greater than the current stored value and push any price/spread combination.

---

### Impact Explanation

Arbitrary price data written into the creator's namespace is consumed by `getOracleData` → `AnchoredPriceProvider` → `MetricOmmPool.swap`. A manipulated bid/ask quote causes the pool to execute swaps at an attacker-controlled price, draining LP assets or allowing the attacker to extract value beyond what the oracle/bin curve permits. This satisfies the **bad-price execution** and **pool insolvency** impact gates. [5](#0-4) 

---

### Likelihood Explanation

**Medium.** The scenario requires:
1. A pusher key to be compromised (realistic for a long-running bot).
2. The pusher to call `revokePusher()` believing it stops the attacker.
3. The creator to replay the old signature before the deadline — plausible via an automated re-delegation script or manual re-setup after a perceived transient failure.

Deadlines are set by the creator and can be months in the future, leaving a large replay window.

---

### Recommendation

Track a per-pusher revocation nonce or a used-signature set. The simplest fix is a monotonically increasing nonce per `(creator, pusher)` pair included in the signed digest:

```solidity
mapping(address => mapping(address => uint256)) public pusherNonce;

// in allowPushers:
uint256 nonce = pusherNonce[msg.sender][pusher]++;
bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
    keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender, nonce))
);
```

After `revokePusher()` or `removePushers()`, increment the nonce so all previously issued signatures for that `(creator, pusher)` pair become invalid.

---

### Proof of Concept

```
1. creator calls allowPushers(deadline=T+365days, [pusher], [sig])
   → namespaceRemapping[pusher] = creator

2. pusher's private key is compromised; pusher calls revokePusher()
   → namespaceRemapping[pusher] = address(0)
   → attacker can no longer push (for now)

3. creator (unaware of compromise, or via automated script) calls
   allowPushers(deadline=T+365days, [pusher], [SAME sig])
   → _ensureDeadline passes (deadline not expired)
   → ECDSA.recover returns pusher (same sig, same inputs)
   → namespaceRemapping[pusher] = creator  ← delegation restored

4. attacker calls oracle.fallback() with crafted slot word:
   price = 1 (near-zero), timestamp = currentTs+1ms
   → monotonicity check passes (newer than stored)
   → creator's slot overwritten with attacker price

5. pool.swap() reads getOracleData(feedId) → price = 1
   → swap executes at attacker-controlled bid/ask
   → LP assets drained
```

    fallback() override external {
        uint256 end;
        uint256 namespace;

        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0)) creator = msg.sender;

        assembly ("memory-safe") {
            end := calldatasize()
            namespace := shl(96, creator) // [creator:20][zeros:12]
        }

        // 4 * 6 + 7 + 1 = 32 bytes per slot
        if (end == 0 || end % 32 != 0) revert BadCalldataLength();

        for (uint256 ptr = 0; ptr < end; ptr += 32) {
            uint256 word;
            assembly ("memory-safe") {
                word := calldataload(ptr)
            }
            // casting to 'uint8' is safe we want LSB
            // forge-lint: disable-next-line(unsafe-typecast)
