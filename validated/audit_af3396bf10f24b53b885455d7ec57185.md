### Title
Missing Deadline on `updateBySignature` Allows Permissionless Submission of Stale Signed Slot Words, Enabling Bad-Price Execution on Pool Swaps — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

---

### Summary

`CompressedOracleV1.updateBySignature` is a permissionless function that accepts a creator-signed slot word and writes it to storage. The function has **no deadline parameter**, contrary to the documented interface. Any actor who obtains a valid signed slot word (e.g., from the public mempool or a relayer broadcast) can hold it and submit it at any time within the price provider's staleness window (`MAX_TIME_DELTA`), causing pools to execute swaps against a stale, manipulated price.

---

### Finding Description

The documentation in `smart-contracts-poc/docs/en/oracle-packet-structure.md` specifies the `updateBySignature` interface as:

```
updateBySignature(feedCreator, deadline, newSlotValue, signature)
keccak256(abi.encode(chainid, oracleAddress, feedCreator, deadline, newSlotValue))
Required Guards: deadline must be in the future (DeadlineExceeded otherwise).
```

The Russian documentation at `smart-contracts-poc/docs/ru/oracle-packet-structure.md` agrees:

```
updateBySignature(feedCreator, deadline, newSlotValue, signature)
```

The actual deployed code, however, is:

```solidity
function updateBySignature(address feedCreator, uint256 newSlotValue, bytes calldata signature)
    external override returns (bool)
{
    ...
    bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
        keccak256(abi.encode(block.chainid, address(this), feedCreator, newSlotValue))
    );
    require(feedCreator == ECDSA.recover(hash, signature));
    _writeStorage(key, bytes32(newSlotValue & ~uint256(0xff)));
    return true;
}
```

There is **no `deadline` parameter**, **no `DeadlineExceeded` check**, and the signed digest does **not** commit to a deadline. The only freshness gate is:

1. The slot word's embedded timestamp must not exceed `block.timestamp + MAX_TIME_DRIFT` (future guard).
2. The slot word's timestamp must be strictly newer than the stored slot timestamp (monotonicity).

Neither guard prevents a valid signed slot word from being withheld and submitted later — as long as its timestamp is still newer than the stored value and within the price provider's `MAX_TIME_DELTA` staleness window.

The `fallback()` push path has the same documentation mismatch (docs say it requires a `uint32 deadline` prefix; the code requires calldata that is a strict multiple of 32 bytes with no prefix), but `fallback()` is restricted to the creator or authorized pushers. `updateBySignature` is **fully permissionless** — any address may call it with a valid signature.

---

### Impact Explanation

**Bad-price execution reaching live pool swaps.**

The call chain is:

```
MetricOmmPool.swap()
  → PriceProvider.getBidAndAskPrice()
      → CompressedOracleV1.price(feedId, pool)   // open, no in-swap binding
          → getOracleData(feedId)                 // reads the stale slot
      → staleness check: block.timestamp - refTime <= MAX_TIME_DELTA
      → spread / guard / bid-ask computation
  ← stale bid/ask returned to pool
  ← swap executes at wrong price
```

If the attacker submits a signed slot word whose embedded timestamp `T` satisfies `T > stored_timestamp` and `block.timestamp - T <= MAX_TIME_DELTA`, the price provider accepts the stale price as live. The pool then executes swaps at the stale bid/ask, allowing the attacker to extract value from traders or LPs.

---

### Likelihood Explanation

- `updateBySignature` is **permissionless** — no role or whitelist required.
- Signed slot words are routinely broadcast off-chain (relayer model) or visible in the public mempool when submitted as transactions.
- The attacker only needs to hold the signed word and submit it when the price has moved within the `MAX_TIME_DELTA` window.
- No special on-chain state is required; the attack is repeatable across any feed whose creator uses the `updateBySignature` path.

Likelihood: **Medium** (requires access to a valid signed slot word and favorable price movement within the staleness window).

---

### Recommendation

**Short term:** Add a `deadline` parameter to `updateBySignature`, include it in the signed digest, and enforce `block.timestamp <= deadline` before accepting the update:

```solidity
function updateBySignature(
    address feedCreator,
    uint256 deadline,
    uint256 newSlotValue,
    bytes calldata signature
) external override returns (bool) {
    require(block.timestamp <= deadline, DeadlineExceeded());
    ...
    bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
        keccak256(abi.encode(block.chainid, address(this), feedCreator, deadline, newSlotValue))
    );
    require(feedCreator == ECDSA.recover(hash, signature));
    ...
}
```

This matches the documented interface and prevents a withheld signed slot word from being submitted after the creator's intended submission window.

**Long term:** Align the `fallback()` calldata format with the documented deadline-prefix layout, or update the documentation to reflect the actual no-deadline design and explicitly justify why monotonicity alone is sufficient for that path.

    ///         freshness gate), so there is no deadline prefix.

        uint256 end;
        uint256 namespace;

        address creator = namespaceRemapping[msg.sender];
        if (creator == address(0)) creator = msg.sender;

        assembly ("memory-safe") {
            end := calldatasize()
            namespace := shl(96, creator) // [creator:20][zeros:12]

        for (uint256 ptr = 0; ptr < end; ptr += 32) {
            uint256 word;
            assembly ("memory-safe") {
                word := calldataload(ptr)

            if (!newer) continue;

## Required Guards

- `deadline` must be in the future (`DeadlineExceeded` otherwise).
- `timestamp` must not be in the future (`FutureTimestamp` otherwise).
- `timestamp` must be strictly increasing per slot (older updates are ignored).
- For the creator’s *current* (partially filled) slot, uncreated lanes must be zero (`UninializedPush` otherwise).
