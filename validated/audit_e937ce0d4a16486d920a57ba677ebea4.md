### Title
Signature Replay in `allowPushers` Allows Creator to Re-Delegate a Self-Revoked Pusher Without New Consent — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`allowPushers` signs consent over `(chainid, oracle, deadline, pusher, creator)` with no nonce and no revocation-state check. After a pusher calls `revokePusher()`, the creator can immediately replay the original signature — as long as `block.timestamp <= deadline` — to unconditionally overwrite `namespaceRemapping[pusher] = creator`, restoring the delegation without any new pusher consent.

### Finding Description

`allowPushers` constructs the signed digest as:

```solidity
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
``` [1](#0-0) 

The only replay guard is `_ensureDeadline`, which checks `block.timestamp <= deadline`: [2](#0-1) 

`revokePusher` sets `namespaceRemapping[msg.sender] = address(0)`: [3](#0-2) 

But `allowPushers` contains no check that the pusher has not already revoked, and no nonce to invalidate prior signatures. The assignment `namespaceRemapping[pusher] = msg.sender` is unconditional: [4](#0-3) 

The code's own NatSpec acknowledges the deadline is the sole protection against post-revocation replay:

> "The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it." [5](#0-4) 

The comment describes the exact attack but the fix (deadline) only prevents replay **after** the deadline expires, not **before** it. A pusher who signs with `deadline = block.timestamp + 1 year` gives the creator a 1-year replay window that survives any number of `revokePusher()` calls.

### Impact Explanation

After the creator replays the signature, `namespaceRemapping[pusher] == creator` again. Any subsequent push by the pusher — who believes they are writing into their own namespace — is silently redirected into the creator's namespace. If the creator's namespace feeds a pool via `AnchoredPriceProvider`, the pool consumes the pusher's data as the creator's oracle price. This breaks the revocation finality invariant and can route unauthorized price data into live pool swaps.

### Likelihood Explanation

The pusher must have signed with a deadline far in the future (common for operational convenience). The creator must be willing to replay the signature. The pusher has no on-chain way to detect the re-delegation without monitoring `PusherAuthorized` events. The pusher cannot prevent it without switching to a new wallet.

### Recommendation

Add a per-pusher nonce or a revocation flag that `allowPushers` checks before writing:

```solidity
mapping(address => uint256) public pusherNonce;

// In allowPushers, include nonce in the signed digest:
keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender, pusherNonce[pusher]))

// In revokePusher, increment the nonce:
pusherNonce[msg.sender]++;
namespaceRemapping[msg.sender] = address(0);
```

This invalidates all prior signatures on revocation, making revocation truly final.

### Proof of Concept

```solidity
// Foundry unit test
function testRevokedPusherReDelegate() public {
    uint256 deadline = block.timestamp + 365 days;

    // Step 1: pusher signs consent
    bytes memory sig = _signConsent(PUSHER_KEY, deadline, pusher, creator);

    address[] memory pushers = new address[](1);
    pushers[0] = pusher;
    bytes[] memory sigs = new bytes[](1);
    sigs[0] = sig;

    // Step 2: creator delegates
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs);
    assertEq(oracle.namespaceRemapping(pusher), creator);

    // Step 3: pusher self-revokes
    vm.prank(pusher);
    oracle.revokePusher();
    assertEq(oracle.namespaceRemapping(pusher), address(0));

    // Step 4: creator replays the SAME signature — no new pusher consent
    vm.prank(creator);
    oracle.allowPushers(deadline, pushers, sigs); // succeeds, deadline still valid

    // Invariant violated: pusher is re-delegated without consent
    assertEq(oracle.namespaceRemapping(pusher), creator);
}
```
